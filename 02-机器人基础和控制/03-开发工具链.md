## 1. VSCode配置

### 1.1 安装ROS2插件

VSCode是ROS2开发的首选IDE之一，它拥有丰富的插件生态和强大的代码编辑功能。在VSCode中安装ROS2相关插件，可以获得代码补全、语法高亮、编译集成等功能。

![VSCode ROS2插件](./images/vscode_ros_plugin.jpg)
*图注：VSCode中的ROS2插件*

推荐安装的插件包括：

- **ROS**插件：由Microsoft官方提供，提供ROS2的基本支持
- **C/C++**插件：提供C++语言支持，包括IntelliSense和调试功能
- **Python**插件：提供Python语言支持
- **CMake**插件：CMakeLists.txt语法高亮和补全
- **Docker**插件：容器开发支持

安装插件的方法：打开VSCode，按Ctrl+Shift+X打开扩展市场，搜索并安装上述插件。

### 1.2 配置ROS2工作区

安装插件后，需要配置VSCode以正确识别ROS2工作空间。

```json
// .vscode/settings.json
{
    "ROS.Distro": "humble",
    "ROS.Workspace": "${workspaceFolder}",
    "python.autoComplete.addBrackets": true,
    "C_Cpp.default.configurationProvider": "ms-vscode.cmake-tools"
}
```

配置完成后，VSCode会自动识别工作空间中的功能包，提供相应的代码补全和提示。

### 1.3 代码补全配置

代码补全（IntelliSense）是提升开发效率的关键功能。正确配置后，输入代码时会自动出现成员函数、参数提示等信息。

对于C++功能包，需要配置compile_commands.json文件路径：

```bash
# 编译时生成compile_commands.json
colcon build --cmake-args -DCMAKE_EXPORT_COMPILE_COMMANDS=1
```

```json
// .vscode/settings.json
{
    "C_Cpp.default.compileCommands": "${workspaceFolder}/build/compile_commands.json"
}
```

对于Python包，配置Python解释器路径即可：

```json
{
    "python.defaultInterpreterPath": "/usr/bin/python3"
}
```

### 1.4 编译任务配置

VSCode可以集成编译任务，让你在编辑器中直接运行colcon build。

```json
// .vscode/tasks.json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "colcon build",
            "type": "shell",
            "command": "colcon build --symlink-install",
            "problemMatcher": [],
            "group": {
                "kind": "build",
                "isDefault": true
            }
        },
        {
            "label": "colcon test",
            "type": "shell", 
            "command": "colcon test --event-handlers console_direct+",
            "problemMatcher": []
        }
    ]
}
```

配置完成后，按Ctrl+Shift+B即可调用编译任务。

### 1.5 调试配置

VSCode提供了强大的调试功能，可以单步调试ROS2节点。

![VSCode调试配置](./images/vscode_debug.jpg)
*图注：VSCode调试ROS2节点配置*

C++节点调试配置：

```json
// .vscode/launch.json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "ROS: C++ Debug",
            "type": "cppdbg",
            "request": "launch",
            "program": "${workspaceFolder}/install/my_package/lib/my_package/my_node",
            "args": [],
            "stopAtEntry": true,
            "cwd": "${workspaceFolder}",
            "environment": [
                {"name": "ROS_DISTRO", "value": "humble"}
            ],
            "externalConsole": false,
            "MIMode": "gdb",
            "setupCommands": [
                {
                    "description": "Enable pretty-printing for gdb",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ]
        }
    ]
}
```

Python节点调试配置：

```json
{
    "name": "ROS: Python Debug",
    "type": "python",
    "request": "launch",
    "program": "${workspaceFolder}/src/my_package/my_package/my_node.py",
    "console": "integratedTerminal"
}
```

配置完成后，按F5即可启动调试会话。VSCode会在断点处暂停，允许你检查变量值、单步执行代码。

## 2. 日志系统

### 2.1 ROS2日志概述

ROS2提供了完善的日志系统，帮助开发者监控节点运行状态、排查问题。日志系统支持多个日志级别，从最详细的DEBUG到最严重的FATAL，可以根据需要选择合适的级别。

ROS2日志级别按优先级从低到高排列：

| 级别 | 说明 | 用途 |
|------|------|------|
| DEBUG | 最详细的调试信息 | 开发时查看程序运行细节 |
| INFO | 一般信息 | 正常运行状态记录 |
| WARN | 警告信息 | 潜在问题但不影响运行 |
| ERROR | 错误信息 | 功能受损需要关注 |
| FATAL | 致命错误 | 程序即将终止 |

### 2.2 C++日志使用

在C++节点中使用日志，需要包含头文件并创建logger实例。

```cpp
#include <rclcpp/rclcpp.hpp>

class MyNode : public rclcpp::Node
{
public:
    MyNode() : Node("my_node")
    {
        // 使用INFO级别日志
        RCLCPP_INFO(this->get_logger(), "Node started");
        
        // 使用带参数的日志
        int count = 10;
        RCLCPP_INFO(this->get_logger(), "Processing %d items", count);
        
        // 条件日志
        if (count > 5) {
            RCLCPP_WARN(this->get_logger(), "Count exceeds threshold");
        }
        
        // 异常日志
        try {
            // 可能抛出异常的代码
        } catch (const std::exception& e) {
            RCLCPP_ERROR(this->get_logger(), "Exception: %s", e.what());
        }
    }
};
```

### 2.3 Python日志使用

Python节点中使用日志同样简单：

```python
import rclpy
from rclpy.node import Node

class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')
        
        # 使用INFO级别日志
        self.get_logger().info('Node started')
        
        # 使用带参数的日志
        count = 10
        self.get_logger().info(f'Processing {count} items')
        
        # 条件日志
        if count > 5:
            self.get_logger().warn('Count exceeds threshold')
        
        # 异常日志
        try:
            # 可能抛出异常的代码
        except Exception as e:
            self.get_logger().error(f'Exception: {e}')
```

### 2.4 设置日志级别

运行时可以通过参数或命令行设置日志级别：

```bash
# 通过参数设置（推荐）
ros2 param set /my_node ros_logger_level DEBUG

# 通过环境变量设置
export RCUTIL_CONSOLE_OUTPUT_FORMAT="{severity}: {name}: {message}"

# C++代码中设置
rclcpp::Logger logger = node->get_logger();
logger.set_level(rclcpp::Logger::Level::Debug);
```

### 2.5 日志输出到文件

默认情况下，日志输出到控制台。如果需要保存到文件，可以使用ros2launch或配置rclpy：

```python
# Python节点输出日志到文件
import logging

logging.basicConfig(
    level=logging.DEBUG,
    filename='my_node.log',
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```
