## 1. 代价地图概述

代价地图是移动机器人导航系统中的关键组件，它以网格形式表示环境信息，为路径规划算法提供障碍物位置和可通行区域的数据。代价地图的核心思想是将真实环境映射为数值网格，每个单元格存储一个代价值（Cost），代价值越高表示该区域越危险或越不可通行。

### 1.1 代价地图的作用

在机器人导航系统中，代价地图主要承担以下职责：

**环境感知**：代价地图将激光雷达、深度相机等传感器的原始数据转换为统一的二维网格表示，使路径规划算法能够理解环境的几何结构。无论是静态的墙壁还是动态的人员障碍物，都会被标记在代价地图上。

**路径安全**：代价地图不仅标记障碍物本身，还通过膨胀层（Inflation Layer）计算障碍物周围的"缓冲区域"。这确保机器人不会与障碍物发生碰撞，即使路径规划存在一定误差。

**动态更新**：代价地图能够实时反映环境变化。当检测到新的障碍物或障碍物消失时，代价地图会及时更新，确保路径规划基于最新的环境信息。

### 1.2 代价地图坐标系

代价地图使用二维栅格坐标系，原点通常位于地图的左下角。每个栅格单元（Grid Cell）具有固定分辨率，表示实际物理尺寸。代价地图的坐标系与ROS坐标系统一致：

```
y (m)
^
|
|
|
|
+-------------> x (m)
(0,0) 地图原点
```

代价地图中的每个单元格存储的代价值范围为0-255，其中：
- **0 (FREE)**: 完全空闲区域，机器人可以安全通行
- **1-127 (INSCRIBED)**: 膨胀区域，靠近障碍物但尚未完全占据
- **128 (LETHAL)**: 障碍物占据区域，机器人中心不能进入
- **253-255 (UNKNOWN)**: 未知区域，未收到传感器数据

### 1.3 代价地图与导航栈

代价地图是ROS/ROS2导航堆栈的核心组成部分，与其他模块紧密配合：

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   传感器数据     │ ──▶ │    代价地图      │ ──▶ │   路径规划      │
│ (激光雷达/相机)  │     │ (Costmap2D)     │     │ (Navfn/DWA)    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │                        │
        ▼                       ▼                        ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   环境感知      │     │   全局/局部地图   │     │   局部避障      │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

导航堆栈通常使用两个代价地图：
- **全局代价地图（Global Costmap）**：基于静态地图构建，用于全局路径规划
- **局部代价地图（Local Costmap）**：基于实时传感器数据构建，用于局部避障

## 2. 分层结构

ROS/ROS2的代价地图采用**分层架构（Layered Costmap）**，这是其强大灵活性的关键所在。每个图层负责特定类型的环境信息，最终将各层结果叠加生成统一的代价地图。这种设计使得添加新功能（如语义信息、多机器人协调）变得非常简单。

### 2.1 图层类型概述

代价地图支持多种图层类型，每种图层有其特定用途：

| 图层类型 | 用途 | 数据来源 |
|----------|------|----------|
| **Static Layer** | 静态环境 | 地图服务器发布的静态地图 |
| **Obstacle Layer** | 动态障碍物 | 激光雷达、深度相机等传感器 |
| **Inflation Layer** | 障碍物膨胀 | 根据障碍物位置自动计算 |
| **Voxel Layer** | 三维障碍物 | RGB-D相机、三维激光雷达 |
| **Social Layer** | 人类活动区域 | 人员检测传感器 |

### 2.2 Static Layer（静态层）

静态层用于存储环境的静态部分，通常是预先构建的地图。它从地图服务器（Map Server）获取SLAM或手动创建的地图数据，并将地图中的障碍物信息转换为代价地图数据。

**工作原理**：静态层在初始化时加载完整的地图文件（通常是PGM或YAML格式），地图中的障碍物栅格直接映射为代价地图中的致命代价值（LETHAL）。静态层通常不频繁更新，除非收到地图更新通知。

**典型配置**：

```yaml
# costmap_global.yaml
global_costmap:
  global_costmap:
    ros__parameters:
      update_frequency: 1.0
      publish_frequency: 1.0
      global_frame: map
      robot_base_frame: base_link
      
      plugins:
        - name: static
          plugin: nav2_costmap_2d::StaticLayer
```

静态层适用于办公室、仓库等相对固定的环境。对于频繁变化的环境（如动态障碍物较多的场景），可以降低静态层的权重或完全禁用。

### 2.3 Obstacle Layer（障碍物层）

障碍物层是代价地图中最活跃的图层，负责实时检测和跟踪障碍物。它订阅传感器数据（通常是LaserScan或PointCloud消息），将传感器覆盖区域的障碍物信息添加到代价地图。

**工作原理**：障碍物层使用两种标记算法：
- **MARK_OBS**: 将检测到障碍物的栅格标记为致命代价值
- **CLEAR_UNKNOWN**: 将传感器覆盖但无障碍物的区域从未知状态恢复为空闲状态

**障碍物跟踪**：对于动态障碍物（如行人），障碍物层不仅标记当前位置，还会跟踪其运动轨迹。这通过体素柱（Voxel Column）或障碍物数组（Obstacle Array）实现。

**典型配置**：

```yaml
# costmap_local.yaml
local_costmap:
  local_costmap:
    ros__parameters:
      update_frequency: 5.0      # 5Hz更新频率
      publish_frequency: 2.0     # 2Hz发布频率
      global_frame: odom
      robot_base_frame: base_link
      width: 3.0                 # 局部地图宽度（米）
      height: 3.0                # 局部地图高度（米）
      resolution: 0.05           # 栅格分辨率（米/栅格）
      
      plugins:
        - name: obstacle_layer
          plugin: nav2_costmap_2d::ObstacleLayer
          enabled: true
          observation_sources: scan
          
        scan:
          sensor: sensor_msgs/msg/LaserScan
          data_type: LaserScan
          topic: /scan
          marking: true
          clearing: true
          max_obstacle_height: 2.0
          min_obstacle_height: 0.0
```

**关键参数说明**：

| 参数 | 说明 | 建议值 |
|------|------|--------|
| `update_frequency` | 代价地图更新频率 | 局部5Hz，全局1Hz |
| `marking` | 是否将障碍物标记到代价地图 | true |
| `clearing` | 是否清除已消失障碍物 | true |
| `max_obstacle_height` | 最大障碍物高度 | 2.0m（室内） |
| `observation_sources` | 传感器数据源列表 | scan、pointcloud |

### 2.4 Inflation Layer（膨胀层）

膨胀层是确保机器人安全的关键图层。它根据障碍物的位置，计算障碍物周围的"膨胀区域"，将障碍物对机器人的威胁程度映射为连续的代价值梯度。

**工作原理**：膨胀层的核心是距离衰减函数。对于每个障碍物栅格，以障碍物为中心向外辐射，距离越远代价值越低。膨胀层的输出是一个平滑的代价梯度，使得路径规划算法倾向于选择远离障碍物的路径。

**膨胀计算公式**：

$$
cost(c) = \begin{cases} 
c_{lethal} & \text{if } d \leq r_{inscribed} \\
c_{inscribed} \times (1 - \frac{d - r_{inscribed}}{r_{inflation} - r_{inscribed}}) & \text{if } r_{inscribed} < d < r_{inflation} \\
c_{free} & \text{if } d \geq r_{inflation}
\end{cases}
$$

其中：
- $d$ : 单元格到最近障碍物的距离
- $r_{inscribed}$ : 内切半径（机器人能通过的最窄通道）
- $r_{inflation}$ : 膨胀半径（代价衰减的边界）
- $c_{lethal}$ : 致命代价（255）
- $c_{inscribed}$ : 内切代价（254）

**典型配置**：

```yaml
# inflation_layer.yaml
plugins:
  - name: inflation
    plugin: nav2_costmap_2d::InflationLayer
    enabled: true
    inflation_radius: 0.55       # 膨胀半径（米）
    cost_scaling_factor: 3.0    # 代价缩放因子
    inflate_unknown: false       # 是否膨胀未知区域
    inflate_outer: false         # 是否膨胀最外层
```

### 2.5 多层叠加

代价地图的最终输出是各图层叠加的结果。叠加遵循一定规则：

```
最终代价 = max(静态层, 障碍物层, 膨胀层)
```

通常膨胀层会覆盖静态层和障碍物层，因为膨胀后的障碍物区域具有更高的代价值。

**完整配置示例**：

```yaml
# complete_costmap.yaml
global_costmap:
  global_costmap:
    ros__parameters:
      update_frequency: 1.0
      publish_frequency: 1.0
      global_frame: map
      robot_base_frame: base_link
      robot_radius: 0.25
      resolution: 0.05

      plugins:
        - name: static
          plugin: nav2_costmap_2d::StaticLayer
        - name: obstacle
          plugin: nav2_costmap_2d::ObstacleLayer
          enabled: true
          observation_sources: scan
          scan:
            sensor: sensor_msgs/msg/LaserScan
            topic: /scan
            marking: true
            clearing: true
        - name: inflation
          plugin: nav2_costmap_2d::InflationLayer
          enabled: true
          inflation_radius: 0.55
          cost_scaling_factor: 3.0
```

## 3. 膨胀半径

膨胀半径（Inflation Radius）是代价地图中最重要的参数之一，它直接决定了机器人在路径规划时会与障碍物保持多大的安全距离。正确设置膨胀半径是实现安全导航的关键。

### 3.1 膨胀半径与机器人安全

膨胀半径的设置需要考虑以下因素：

**机器人尺寸**：膨胀半径必须大于机器人本体半径，以确保机器人不会与障碍物发生碰撞。通常设置膨胀半径为机器人半径加上一定的安全裕度：

$$
r_{inflation} = r_{robot} + r_{margin}
$$

其中 $r_{margin}$ 建议值为0.15-0.3米。

**运动误差**：机器人定位和控制系统存在一定的误差，较大的膨胀半径可以提供更多的安全缓冲。

**传感器误差**：传感器的测量也存在误差，特别是在远距离区域。

### 3.2 膨胀半径参数详解

```yaml
inflation_radius: 0.55        # 膨胀半径（米）
cost_scaling_factor: 3.0     # 代价衰减速率
```

**inflation_radius（膨胀半径）**：定义障碍物代价值从LETHAL衰减到FREE的距离。膨胀半径越大，机器人与障碍物保持的距离越远，但可用通行区域会减少。

**cost_scaling_factor（代价缩放因子）**：控制代价值随距离衰减的速率。值越大，代价值下降越快，膨胀区域的"硬度"越高。

### 3.3 膨胀半径设置建议

不同场景下膨胀半径的建议值：

| 场景 | 膨胀半径 | 说明 |
|------|----------|------|
| 室内狭窄通道 | 0.3-0.4m | 留出机器人旋转空间 |
| 室内开阔区域 | 0.5-0.6m | 标准安全距离 |
| 室外复杂环境 | 0.7-1.0m | 考虑定位误差和地形 |
| 人机共存环境 | 0.8-1.2m | 确保与人员安全距离 |

**调试技巧**：在RViz中可视化代价地图，观察膨胀区域是否合理。如果膨胀区域覆盖了实际可通行区域，说明膨胀半径设置过大；如果机器人仍然偶尔与障碍物碰撞，说明膨胀半径可能偏小。

## 4. Footprint

Footprint（足迹）是机器人在代价地图中的二维几何表示。正确设置footprint对于代价地图准确计算机器人与障碍物的碰撞关系至关重要。

### 4.1 Footprint的概念

Footprint定义了机器人在二维平面上的占据区域，可以是简单圆形或多边形。在代价地图中，footprint用于两个关键计算：

**碰撞检测**：当footprint与障碍物栅格重叠时，标记相应的代价地图单元格为LETHAL。

**内切半径计算**：内切半径（Inscribed Radius）是footprint内切圆的最大半径，用于计算机器人能够通过的最窄通道宽度。

### 4.2 Footprint配置

**圆形footprint（简单场景）**：

```yaml
# circular_robot.yaml
robot_radius: 0.25          # 圆形机器人半径（米）
```

圆形footprint计算简单，适用于圆形或近似圆形的机器人（如TurtleBot、Pioneer等）。

**多边形footprint（复杂形状）**：

```yaml
# polygon_footprint.yaml
footprint: [
  [-0.3, -0.3],   # 左后
  [-0.3, 0.3],    # 左前
  [0.3, 0.3],     # 右前
  [0.3, -0.3]     # 右后
]
```

多边形footprint能够更准确地表示非圆形机器人（如方形底盘、带机械臂的机器人）。

### 4.3 Footprint与代价地图

Footprint在代价地图中的处理流程：

```
1. 获取机器人当前位置
2. 根据robot_base_frame变换footprint坐标
3. 投影footprint到代价地图栅格
4. 检查footprint覆盖的栅格
5. 对于障碍物栅格，设置LETHAL代价
6. 对于膨胀边界栅格，计算内切距离代价
```

**内切半径计算**：

对于多边形footprint，内切半径通过以下方式计算：

$$
r_{inscribed} = \min_{edges}(distance(robot\_center, edge))
$$

这确保了机器人在通过狭窄通道时，只要通道宽度大于内切直径就能安全通过。

### 4.4 Footprint设置示例

**差速轮机器人（TurtleBot类型）**：

```yaml
# turtlebot3_costmap.yaml
local_costmap:
  local_costmap:
    ros__parameters:
      robot_radius: 0.175      # TurtleBot3 Burger: 0.17m

global_costmap:
  global_costmap:
    ros__parameters:
      robot_radius: 0.175
```

**四轮阿克曼机器人**：

```yaml
# ackermann_robot.yaml
footprint: [
  [-1.0, -0.5],    # 后左
  [-1.0, 0.5],     # 后右
  [0.5, 0.5],      # 前右
  [0.5, -0.5]      # 前左
]
```

**带机械臂的机器人**：

```yaml
# robot_with_arm.yaml
# 考虑机械臂伸展状态的footprint
footprint: [
  # 基础车身
  [-0.4, -0.3],
  [-0.4, 0.3],
  [0.4, 0.3],
  [0.4, -0.3],
  # 机械臂伸展
  [0.8, 0.0],
  [0.4, 0.3],
  [0.4, -0.3]
]
```

## 5. 参数配置实践

本节将通过实际配置示例，帮助你掌握代价地图参数的配置方法。

### 5.1 全局代价地图配置

全局代价地图用于全局路径规划，通常基于静态地图构建：

```yaml
# global_costmap_params.yaml
global_costmap:
  global_costmap:
    ros__parameters:
      # 地图更新参数
      update_frequency: 1.0          # 全局地图更新频率（Hz）
      publish_frequency: 1.0        # 发布频率（Hz）
      
      # 坐标系配置
      global_frame: map              # 全局坐标系
      robot_base_frame: base_link    # 机器人基坐标系
      
      # 地图参数
      plugin_names: [static, obstacle, inflation]
      plugin_types: [
        nav2_costmap_2d::StaticLayer,
        nav2_costmap_2d::ObstacleLayer,
        nav2_costmap_2d::InflationLayer
      ]
      
      # 静态层配置
      static:
        enabled: true
        map_subscribe_transient_local: true
        
      # 障碍物层配置
      obstacle:
        enabled: true
        observation_sources: scan
        scan:
          topic: /scan
          sensor: sensor_msgs/msg/LaserScan
          marking: true
          clearing: true
          max_obstacle_height: 2.5
          min_obstacle_height: 0.0
          
      # 膨胀层配置
      inflation:
        enabled: true
        inflation_radius: 0.55
        cost_scaling_factor: 3.0
        inflate_unknown: false
```

### 5.2 局部代价地图配置

局部代价地图用于局部路径规划和避障：

```yaml
# local_costmap_params.yaml
local_costmap:
  local_costmap:
    ros__parameters:
      # 地图更新参数
      update_frequency: 5.0          # 局部地图更新频率（Hz）
      publish_frequency: 2.0         # 发布频率（Hz）
      
      # 坐标系配置
      global_frame: odom              # 局部地图使用里程计坐标系
      robot_base_frame: base_link
      rolling_window: true           # 滚动窗口模式
      
      # 地图尺寸（以机器人为中心）
      width: 3.0                      # 地图宽度（米）
      height: 3.0                     # 地图高度（米）
      resolution: 0.05               # 栅格分辨率（米/栅格）
      
      # 插件配置
      plugin_names: [obstacle, inflation]
      plugin_types: [
        nav2_costmap_2d::ObstacleLayer,
        nav2_costmap_2d::InflationLayer
      ]
      
      # 障碍物层配置
      obstacle:
        enabled: true
        observation_sources: scan
        scan:
          topic: /scan
          sensor: sensor_msgs/msg/LaserScan
          marking: true
          clearing: true
          max_obstacle_height: 2.0
          max_z: 10.0
          min_z: 0.0
          obstacle_range: 2.5         # 障碍物检测范围

        enabled: true
```

### 5.3 代价地图参数调优指南

**调试流程**：

1. **观察代价地图**：在RViz中可视化代价地图，确认障碍物和膨胀区域正确显示

2. **调整膨胀半径**：
   - 膨胀过大 → 可通行区域减少，可能找不到路径
   - 膨胀过小 → 机器人可能与障碍物碰撞

3. **调整更新频率**：
   - 局部代价地图：5-10Hz（快速响应环境变化）
   - 全局代价地图：1-2Hz（静态环境不需要高频更新）

4. **检查传感器配置**：
   - 确认传感器话题正确
   - 检查`marking`和`clearing`参数

**常见问题与解决方案**：

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 代价地图显示为空 | 传感器数据未接收 | 检查传感器话题 |
| 障碍物不消失 | clearing参数为false | 设置clearing: true |
| 机器人卡住 | 膨胀半径过大 | 减小inflation_radius |
| 机器人碰撞障碍物 | 膨胀半径过小 | 增加inflation_radius |
| 代价地图显示不全 | 地图尺寸设置过小 | 增加width和height |

## 练习题

### 选择题

1. 代价地图中，代价值为255表示什么含义？
   - A) 完全空闲的区域
   - B) 障碍物占据的致命区域
   - C) 未知区域
   - D) 轻微膨胀区域
   
   **答案：B**。代价值255（LETHAL）表示障碍物占据区域，机器人中心不能进入。

2. 在膨胀层中，inflation_radius参数的作用是什么？
   - A) 障碍物代价值开始衰减的边界
   - B) 障碍物代价值衰减到零的距离
   - C) 机器人能通过的最窄通道宽度
   - D) 传感器检测障碍物的最大距离
   
   **答案：B**。inflation_radius定义了障碍物代价值从LETHAL衰减到FREE的距离。

3. 对于圆形机器人，哪个参数用于配置其footprint？
   - A) footprint
   - B) robot_radius
   - C) footprint_padding
   - D) inscribed_radius
   
   **答案：B**。robot_radius用于配置圆形机器人的半径，多边形footprint使用footprint参数。

4. 全局代价地图和局部代价地图的主要区别是什么？
   - A) 使用不同的坐标系
   - B. 全局基于静态地图，局部基于实时传感器数据
   - C) 全局不使用膨胀层
   - D) 局部不更新
   
   **答案：B**。全局代价地图主要基于静态地图，用于全局路径规划；局部代价地图基于实时传感器数据，用于局部避障。

### 实践题

5. 编写一个完整的局部代价地图配置文件，参数要求如下：
   - 地图尺寸：5m × 5m
   - 分辨率：0.05m
   - 更新频率：5Hz
   - 膨胀半径：0.5m
   - 机器人半径：0.3m
   
   **参考答案**：
   
   ```yaml
   local_costmap:
     local_costmap:

         global_frame: odom

           nav2_costmap_2d::ObstacleLayer,
           nav2_costmap_2d::InflationLayer
         ]

           enabled: true
           observation_sources: scan
           scan:
             topic: /scan
             sensor: sensor_msgs/msg/LaserScan

           enabled: true
   ```

6. 如果机器人需要穿过宽度为0.8m的狭窄通道，机器人的footprint应该如何设置？请说明理由。
   
   **参考答案**：
   
   机器人footprint的内切半径必须小于通道宽度的一半：
   
   $$r_{inscribed} < \frac{0.8}{2} = 0.4m$$
   
   因此可以设置：
   - 圆形机器人：`robot_radius: 0.35`
   - 或多边形footprint，确保内切圆半径小于0.4m
   
   膨胀半径也需要相应调整，确保膨胀后的通道宽度仍可通过：
   
   $$通道有效宽度 = 通道宽度 - 2 \times 膨胀半径 > 2 \times r_{robot}$$
   
   膨胀半径不应超过 0.4 - 0.35 = 0.05m，建议设置在0.3m左右以留出安全裕度。
