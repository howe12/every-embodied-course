## 1. 全局路径规划概述

全局路径规划（Global Path Planning）是指在已知环境地图的情况下，计算从机器人当前位置到目标位置的最优路径。全局规划器假设环境是静态的或近似静态的，它不考虑动态障碍物，主要关注路径的最优性和计算效率。

### 1.1 问题定义

全局路径规划可以形式化表示为：

$$\text{给定}: G = (V, E), v_{start}, v_{goal}$$
$$\text{求}: P = \{v_0, v_1, ..., v_n\}$$

其中：
- $G = (V, E)$ 是图，$V$ 是节点集合，$E$ 是边集合
- $v_{start}$ 是起点，$v_{goal}$ 是目标点
- $P$ 是从起点到目标点的路径
- 路径代价 $C(P) = \sum_{i=0}^{n-1} w(v_i, v_{i+1})$ 最小

### 1.2 常用算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| Dijkstra | $O((V+E)\log V)$ | $O(V)$ | 保证最优解 | 探索区域大 |
| A* | $O(E)$ (启发式) | $O(V)$ | 效率高 | 内存消耗大 |
| ARA* | 可变 | $O(V)$ | 可调节解质量 | 需要多次搜索 |
| PRM* | $O(V\log V)$ | $O(V)$ | 概率完备 | 需要后处理 |

## 2. Dijkstra算法

Dijkstra算法是荷兰计算机科学家Edsger Dijkstra于1956年提出的单源最短路径算法。它是目前最经典的全局路径规划算法之一，在ROS的navigation stack中作为默认的全局规划器使用。

### 2.1 算法原理

Dijkstra算法采用贪心策略，从起点开始，逐步向外扩展，直到找到目标点。其核心思想是：每次选择当前代价最小的未访问节点进行扩展。

```
Dijkstra算法伪代码：

function Dijkstra(G, start, goal):
    for each vertex v in G:
        dist[v] = ∞
        prev[v] = undefined
    dist[start] = 0
    
    Q = 所有顶点集合
    
    while Q is not empty:
        u = vertex in Q with minimum dist[u]
        remove u from Q
        
        if u == goal:
            return path reconstruction(prev, start, goal)
        
        for each neighbor v of u:
            if edge(u, v) exists:
                alt = dist[u] + weight(u, v)
                if alt < dist[v]:
                    dist[v] = alt
                    prev[v] = u
    
    return no path exists
```

### 2.2 数学证明

Dijkstra算法保证找到最短路径，其正确性基于以下定理：

**定理**：当节点 $u$ 被从优先队列中取出时，$dist[u]$ 等于从起点到 $u$ 的最短路径长度。

**证明**（简要）：
- 假设当 $u$ 被取出时，存在另一条更短的路径 $P$ 到 $u$
- 设 $y$ 是路径 $P$ 上第一个不在已访问集合中的节点
- $x$ 是 $y$ 的前驱节点（已在访问集合中）
- 根据归纳法，$dist[x]$ 已经是到 $x$ 的最短距离
- 由于边权重非负，$dist[y] \leq dist[x] + w(x,y) < dist[u]$
- 但 $u$ 是队列中最小值，矛盾！

## 3. A*算法

A*（A-Star）算法是1968年由Peter Hart、Nils Nilsson和Bertram Raphael提出的启发式搜索算法。它是Dijkstra算法的改进版本，通过引入启发式函数来指导搜索方向，大幅提高搜索效率。

### 3.1 算法原理

A*算法的核心是评估函数：

$$f(n) = g(n) + h(n)$$

其中：
- $g(n)$：从起点到当前节点 $n$ 的实际代价
- $h(n)$：从当前节点 $n$ 到目标的估计代价（启发式函数）
- $f(n)$：节点 $n$ 的综合评估值

```
A*算法伪代码：

function AStar(G, start, goal, h):
    open_set = {start}
    closed_set = {}
    came_from = {}
    
    g_score = {start: 0}
    f_score = {start: h(start)}
    
    while open_set is not empty:
        current = node in open_set with lowest f_score[current]
        
        if current == goal:
            return reconstruct_path(came_from, current)
        
        open_set.remove(current)
        closed_set.add(current)
        
        for each neighbor of current:
            if neighbor in closed_set:
                continue
            
            tentative_g = g_score[current] + dist(current, neighbor)
            
            if neighbor not in open_set:
                open_set.add(neighbor)
            elif tentative_g >= g_score[neighbor]:
                continue
            
            came_from[neighbor] = current
            g_score[neighbor] = tentative_g
            f_score[neighbor] = g_score[neighbor] + h(neighbor)
    
    return no solution
```

### 3.2 启发式函数

启发式函数 $h(n)$ 的选择对算法性能有决定性影响：

| 启发式函数 | 公式 | 适用场景 |
|------------|------|----------|
| 曼哈顿距离 | $\|x_1-x_2\| + \|y_1-y_2\|$ | 4方向运动 |
| 欧几里得距离 | $\sqrt{(x_1-x_2)^2 + (y_1-y_2)^2}$ | 任意方向运动 |
| 对角线距离 | $\max(\|x_1-x_2\|, \|y_1-y_2\|)$ | 8方向运动 |

**可采纳性**：启发式函数必须是可采纳的，即 $h(n) \leq h^*(n)$（真实代价）。这保证了A*算法能找到最优解。

## 4. 局部路径规划概述

局部路径规划（Local Path Planning）是指在未知或动态环境中，根据机器人实时感知信息（如激光雷达、深度相机）进行短距离的路径规划和避障。局部规划器不考虑全局地图，主要关注实时性和避障能力。

### 4.1 问题定义

局部路径规划可以形式化表示为：

$$\text{给定}: P_{global}, O_{local}, v_{robot}$$
$$\text{求}: v_{cmd}$$

其中：
- $P_{global}$：全局规划器输出的参考路径
- $O_{local}$：局部传感器感知到的障碍物
- $v_{robot}$：机器人当前状态
- $v_{cmd}$：发送给底盘的速度指令

### 4.2 常用算法对比

| 算法 | 计算频率 | 优点 | 缺点 |
|------|----------|------|------|
| TEB | 10-20Hz | 考虑时间最优、动态约束 | 参数多、调优难 |
| DWA | 10Hz | 简单直观、稳定性好 | 可能陷入局部最优 |
| AD* | 可变 | 在线重规划 | 计算复杂 |
| E* | 可变 | 梯度平滑 | 需要密集栅格 |

## 5. TEB局部规划器

TEB（Timed Elastic Band）是一种基于优化的时间弹性带方法，由Rolf Kümmerle等人于2009年提出。TEB将路径规划问题转化为非线性优化问题，能够同时考虑时间最优性和机器人动态约束。

### 5.1 算法原理

TEB的核心思想是在机器人与目标点之间构建一条"弹性带"，然后通过优化算法不断调整这条带，使其：
1. 避开障碍物
2. 满足机器人运动学约束
3. 最小化执行时间

```
TEB优化问题：

min ∑ d_i² + ∑ ΔT_i²

subject to:
  - 障碍物约束：dist(q_i, obstacle) > d_safe
  - 运动学约束：|q_i - q_{i-1}| < v_max * ΔT_i
  - 速度约束：|v_i| < v_max
  - 加速度约束：|a_i| < a_max
```

### 5.3 参数调优指南

TEB参数调优是实际应用中的难点，以下是关键参数的影响和调整建议：

| 参数 | 作用 | 调大影响 | 调小影响 | 建议值 |
|------|------|----------|----------|--------|
| `max_global_plan_lookahead_dist` | 前瞻距离 | 更平滑、计算量大 | 可能规划失败 | 1.5-3.0m |

## 6. DWA局部规划器

DWA（Dynamic Window Approach）是由Dieter Fox等人于1997年提出的局部路径规划算法。DWA算法在速度空间中搜索最优的速度对（线速度、角速度），具有实现简单、运行稳定的特点。

### 6.1 算法原理

DWA算法的核心是在机器人可执行的速度空间中搜索最优速度：

```
DWA算法步骤：

1. 速度采样：
   v_d = {(v, ω) | v ∈ [v_min, v_max], ω ∈ [ω_min, ω_max]}

2. 模拟轨迹：
   对每个(v, ω)模拟dt时间内的轨迹

3. 评估函数：
   score(v, ω) = α·heading(v, ω) + 
                 β·dist(v, ω) + 
                 γ·velocity(v, ω)
   
   其中：
   - heading: 航向得分（朝向目标）
   - dist: 距离障碍物得分
   - velocity: 速度得分

4. 选择最优速度对
```

**评估函数详解**：

$$\text{heading}(v, \omega) = 1 - \frac{\theta_{target} - \theta_{current}}{\pi}$$

$$\text{dist}(v, \omega) = \min_{p \in trajectory} distance(p, obstacle)$$

$$\text{velocity}(v, \omega) = \frac{v}{v_{max}}$$

### 6.3 TEB vs DWA对比

| 特性 | TEB | DWA |
|------|-----|-----|
| **优化目标** | 时间最优 | 多目标加权 |
| **约束处理** | 惩罚函数 | 硬约束过滤 |
| **轨迹形状** | 弹性带 | 圆弧段 |
| **计算速度** | 中等 | 快 |
| **动态避障** | 好 | 一般 |
| **参数数量** | 多 | 少 |
| **适用场景** | 复杂环境 | 简单环境 |

**选择建议**：
- 狭窄环境、多障碍物 → TEB
- 开阔环境、简单避障 → DWA
- 需要精确控制 → TEB
- 追求稳定性 → DWA

## 8. 参数调优实战

### 8.1 全局规划器调优

**Dijkstra/A* 关键参数**：

| 参数 | 说明 | 调整建议 |
|------|------|----------|
| `max_iterations` | 最大搜索迭代次数 | 大地图增加，小地图减小 |
| `max_planning_time` | 单次规划超时时间 | 根据实时性要求调整 |
| `tolerance` | 目标点容差 | 可接受误差越大，规划越快 |
| `allow_unknown` | 是否穿越未知区域 | 取决于地图完整性 |

**调优步骤**：

```bash
# 1. 测试基本规划
ros2 run nav2_util map_loader /path/to/map.yaml

# 2. 调整超时时间
ros2 param set /planner_server GridBased.max_planning_time 10.0

# 3. 观察规划结果
ros2 topic echo /plan
```

### 8.2 局部规划器调优

**TEB调优流程**：

```
步骤1: 基础速度参数
  max_vel_x: 从低值开始(0.1)
  逐步提高到目标速度
  
步骤2: 安全距离
  min_obstacle_dist: 根据机器人尺寸设置
  一般 = 机器人半径 + 0.1m
  
步骤3: 优化权重
  weight_obstacle: 先大后小
  weight_optimisation: 调整平滑度
  
步骤4: 迭代次数
  观察实时性能
  动态调整
```

**常见问题与解决方案**：

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 机器人抖动 | 加速度限制过小 | 增加acc_lim_x, acc_lim_theta |
| 规划失败 | 障碍物距离太近 | 减小min_obstacle_dist或清理障碍物 |
| 路径不平滑 | 优化迭代不足 | 增加迭代次数 |
| 响应慢 | 前瞻距离太大 | 减小max_global_plan_lookahead_dist |
| 碰撞 | 安全距离太小 | 增加inflation_dist |

from rclpy.node import Node
from geometry_msgs.msg import Twist


class MyLocalPlanner(Node):
    """自定义局部规划器示例 - 简化的DWA"""
    
    def __init__(self):
        super().__init__('my_local_planner')
        
        # 参数
        self.declare_parameter('max_vel_x', 0.26)
        self.declare_parameter('max_vel_theta', 1.0)
        self.declare_parameter('min_obstacle_dist', 0.3)
        
        # 发布者
        self.cmd_pub = self.create_publisher(Twist, 'cmd_vel', 1)
        
        self.get_logger().info('My Local Planner 初始化完成')
    
    def compute_velocity_commands(self, pose, velocity, global_plan) -> Twist:
        """计算速度命令"""
        
        # 获取参数
        max_vx = self.get_parameter('max_vel_x').value
        max_vt = self.get_parameter('max_vel_theta').value
        min_dist = self.get_parameter('min_obstacle_dist').value
        
        # 1. 速度采样
        candidates = self._sample_velocities(max_vx, max_vt)
        
        # 2. 评估每个候选
        best_cmd = Twist()
        best_score = float('-inf')
        
        for vx, vt in candidates:
            # 检查障碍物
            if self._collision_check(pose, vx, vt, min_dist):
                continue
            
            # 计算得分
            score = self._evaluate(pose, vx, vt, global_plan)
            
            if score > best_score:
                best_score = score
                best_cmd.linear.x = vx
                best_cmd.angular.z = vt
        
        return best_cmd
    
    def _sample_velocities(self, max_vx, max_vt):
        """采样速度空间"""

### 选择题

1. Dijkstra算法与A*算法的主要区别是什么？
   - A) Dijkstra只能用于静态环境，A*可以用于动态环境
   - B) A*使用了启发式函数来指导搜索
   - C) Dijkstra保证最优解，A*不保证
   - D) A*的时间复杂度高于Dijkstra
   
   **答案：B**。A*算法的核心改进是引入了启发式函数$f(n) = g(n) + h(n)$，这使得搜索更具方向性，效率更高。

2. TEB局部规划器的核心思想是什么？
   - A) 在速度空间中搜索最优速度对
   - B) 构建时间弹性带并进行优化
   - C) 使用梯度下降法平滑路径
   - D) 基于采样的随机路径生成
   
   **答案：B**。TEB（Timed Elastic Band）通过构建连接起点和目标的"弹性带"，并使用优化算法不断调整，以实现时间最优和避障。

3. 在ROS2 Nav2中，哪个参数用于设置局部规划器的前瞻距离？
   - `max_vel_x`
   - `min_obstacle_dist`
   - `max_global_plan_lookahead_dist`
   - `inflation_dist`
   
   **答案：C**。`max_global_plan_lookahead_dist`参数控制局部规划器在全局路径上的前瞻距离。

4. DWA算法中，评估函数通常不包含以下哪个因素？
   - heading（航向得分）、距离得分、速度得分
   - 加速度得分
   - 目标朝向得分

   **答案：B**。DWA的评估函数主要包含三个因素：航向得分（朝向目标）、距离障碍物得分和速度得分，不包含加速度得分。
