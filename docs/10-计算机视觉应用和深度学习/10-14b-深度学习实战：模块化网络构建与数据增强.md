## 1. 课程目标与模块总览

### 1.1 本节目标

本节课程采用 **"不直接给答案"** 的学习方式，目的是让你真正理解每个模块的作用，而不是只会复制粘贴完整代码。

**课程目标：**

- 掌握 PyTorch 中常用网络层的代码表示
- 理解每个模块层的输入输出形状（shape）
- 能够根据任务需求自主选择和组合模块
- 掌握常见的数据增强方法及其适用场景
- 独立设计并训练一个完整的 CNN 模型

### 1.2 模块总览

在正式开始之前，先看一眼我们今天要用的所有"积木"：

| 积木编号 | 模块名称 | PyTorch 对应 | 核心参数 |
|---------|---------|-------------|---------|
| 积木 1 | 全连接层 | `nn.Linear` | in_features, out_features |
| 积木 2 | 卷积层 | `nn.Conv2d` | in_channels, out_channels, kernel_size, stride, padding |
| 积木 3 | 池化层 | `nn.MaxPool2d` / `nn.AvgPool2d` | kernel_size, stride |
| 积木 4 | 激活函数 | `nn.ReLU` / `nn.Sigmoid` / `nn.Tanh` | - |
| 积木 5 | Dropout | `nn.Dropout` | p (丢弃概率) |
| 积木 6 | BatchNorm | `nn.BatchNorm2d` | num_features |
| 数据增强 | 图像变换 | `torchvision.transforms` | 多种参数 |

### 1.3 积木组合思路

一个典型的 CNN 网络就像这样组合：

```
输入图像
  ↓
[积木2: 卷积层] + [积木4: 激活函数]  ← 特征提取
  ↓
[积木3: 池化层]                      ← 降维
  ↓
[积木2: 卷积层] + [积木4: 激活函数]
  ↓
[积木3: 池化层]
  ↓
[积木6: BatchNorm]（可选）
  ↓
展平（flatten）
  ↓
[积木1: 全连接层] + [积木4: 激活函数]
  ↓
[积木5: Dropout]（训练时防止过拟合）
  ↓
[积木1: 全连接层] → 输出类别数
```

下面我们逐个击破每一块积木。

## 2. 数据加载与预处理

### 2.1 MNIST 数据集加载

本节使用 MNIST 手写数字数据集，图片尺寸 28×28，灰度图，10 分类（0~9）。

**标准加载代码：**

```python
import torchvision
import torchvision.transforms as transforms

transform = transforms.Compose([
    transforms.ToTensor(),              # 转为张量 (H, W, C) → (C, H, W)，像素归一化到 [0,1]
    transforms.Normalize((0.5,), (0.5,))  # 标准化到均值0.5、标准差0.5
])

trainset = torchvision.datasets.MNIST(
    root='./data', train=True, download=True, transform=transform
)
trainloader = torchvision.data.DataLoader(
    trainset, batch_size=64, shuffle=True, num_workers=2
)

testset = torchvision.datasets.MNIST(
    root='./data', train=False, download=True, transform=transform
)
testloader = torchvision.data.DataLoader(
    testset, batch_size=100, shuffle=False, num_workers=2
)

print(f"训练集样本数: {len(trainset)}, 测试集样本数: {len(testset)}")
print(f"单张图片尺寸: {trainset[0][0].shape}")  # torch.Size([1, 28, 28])
```

**要点回顾：**
- `transforms.ToTensor()` 自动将 [0, 255] 像素值缩放到 [0.0, 1.0]，并调整通道顺序
- `transforms.Normalize((0.5,), (0.5,))` 将像素范围从 [0, 1] 映射到 [-1, 1]，让数据分布更均衡
- `shuffle=True` 在每个 epoch 开始时打乱数据顺序，有助于防止模型学到一个固定的顺序依赖


## 3. 积木一：全连接层（FC Layer）

### 3.1 什么是全连接层

全连接层（Fully Connected Layer，FC Layer）是神经网络中最基本的层。每个神经元与上一层的所有神经元都有连接，因此称为"全连接"。

**数学公式：**

$$
y = W \cdot x + b
$$

其中 $W$ 是权重矩阵，$b$ 是偏置向量，$x$ 是输入向量，$y$ 是输出向量。

### 3.2 PyTorch 实现

```python
import torch.nn as nn

# nn.Linear(in_features, out_features, bias=True)
# 参数说明：
#   in_features:  输入特征的维度
#   out_features: 输出特征的维度
#   bias:        是否包含偏置项，默认为 True

# 示例：输入 784 维，输出 256 维
fc_layer = nn.Linear(in_features=784, out_features=256)
print(fc_layer)
```

**打印结果：**

```
Linear(in_features=784, out_features=256, bias=True)
```

### 3.3 形状变化

```python
import torch

# 模拟输入: batch_size=32, features=784
x = torch.randn(32, 784)

# 通过全连接层
fc = nn.Linear(784, 256)
y = fc(x)

print(f"输入形状: {x.shape}")   # torch.Size([32, 784])
print(f"输出形状: {y.shape}")   # torch.Size([32, 256])
```

**思考题：** 如果输入是 32 张 28×28 的图片，展平后是多少维？

<details>
<summary>点击查看答案</summary>

28 × 28 = 784 维。32 张图片展平后 shape 为 [32, 784]。

</details>

### 3.4 全连接层在 CNN 中的位置

全连接层通常放在 CNN 的最后几层，负责将学到的特征映射到最终的分类结果。

```
CNN特征提取部分 → flatten → FC层 → FC层 → 输出
```

```python
# 将特征图展平为向量后通过全连接层
x = torch.randn(32, 512, 7, 7)   # 模拟 CNN 输出的特征图
x = x.view(x.size(0), -1)         # 展平: [32, 512, 7, 7] → [32, 25088]
print(f"展平后形状: {x.shape}")    # torch.Size([32, 25088])

fc1 = nn.Linear(25088, 1024)
x = fc1(x)
print(f"FC1输出形状: {x.shape}")   # torch.Size([32, 1024])
```

## 4. 积木二：卷积层（CNN Layer）

### 4.1 什么是卷积层

卷积层（Convolutional Layer）是 CNN 的核心，通过**卷积核（滤波器）**在输入图像上滑动，提取局部特征（如边缘、纹理、形状等）。

**关键概念：**
- **卷积核（Kernel）**：一个小矩阵（如 3×3、5×5），其值是网络需要学习的权重
- **步幅（Stride）**：卷积核每次移动的像素数，默认为 1
- **填充（Padding）**：在输入边缘补零，控制输出尺寸

### 4.2 PyTorch 实现

```python
# nn.Conv2d(in_channels, out_channels, kernel_size, stride=1, padding=0, bias=True)
# 参数说明：
#   in_channels:   输入通道数（灰度图=1，RGB图=3）
#   out_channels: 输出通道数（卷积核的数量，即提取多少种特征）
#   kernel_size:   卷积核大小（可以是 int 或 tuple）
#   stride:        步幅，默认为1
#   padding:       填充像素数，默认为0（'valid'卷积）
#   bias:          是否包含偏置，默认为True

# 示例：输入3通道图像，用32个3×3卷积核卷积，输出32通道特征图
conv_layer = nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3, stride=1, padding=1)
print(conv_layer)
```

### 4.3 形状计算

输出特征图尺寸公式：

$$
H_{out} = \left\lfloor \frac{H_{in} + 2 \times padding - kernel\_size}{stride} \right\rfloor + 1
$$

$$
W_{out} = \left\lfloor \frac{W_{in} + 2 \times padding - kernel\_size}{stride} \right\rfloor + 1
$$

```python
# 模拟输入: batch=1, channels=1, H=28, W=28 (MNIST图片)
x = torch.randn(1, 1, 28, 28)

# 32个3×3卷积核，stride=1, padding=1（保持尺寸）
conv = nn.Conv2d(1, 32, kernel_size=3, stride=1, padding=1)
y = conv(x)

print(f"输入形状: {x.shape}")   # torch.Size([1, 1, 28, 28])
print(f"输出形状: {y.shape}")   # torch.Size([1, 32, 28, 28])
```

### 4.4 常用卷积核配置

| 配置 | 适用场景 | 说明 |
|------|---------|------|
| 3×3, padding=1 | 保持尺寸 | 最常用，堆叠多层也不易丢失空间信息 |
| 5×5, padding=2 | 较大感受野 | 感受野更大，但参数量是 3×3 的约 2.8 倍 |
| 1×1 | 通道压缩 | 用于减少参数量和计算量（ bottleneck 结构） |
| stride=2 | 下采样 | 用卷积实现下采样，减少特征图尺寸 |

## 5. 积木三：池化层（Pooling Layer）

### 5.1 什么是池化层

池化层（Pooling Layer）对特征图进行**下采样**（ Downsampling ），降低空间尺寸，减少计算量，同时增强特征的平移不变性。

**两种常见类型：**
- **最大池化（Max Pooling）**：取池化窗口内的最大值，保留最显著的特征
- **平均池化（Average Pooling）**：取池化窗口内的平均值，保留背景整体信息

### 5.2 PyTorch 实现

```python
# 最大池化
# nn.MaxPool2d(kernel_size, stride=None, padding=0)
# stride 默认为 kernel_size（不重叠池化）

maxpool = nn.MaxPool2d(kernel_size=2, stride=2)   # 2×2 窗口，步幅2 → 尺寸减半
print(maxpool)

# 平均池化
avgpool = nn.AvgPool2d(kernel_size=2, stride=2)   # 2×2 窗口，步幅2
print(avgpool)
```

### 5.3 形状变化

```python
# 模拟输入: 1, 32, 28, 28
x = torch.randn(1, 32, 28, 28)

# 2×2 最大池化，尺寸减半
mp = nn.MaxPool2d(2, 2)
y = mp(x)

print(f"输入形状: {x.shape}")   # torch.Size([1, 32, 28, 28])
print(f"输出形状: {y.shape}")   # torch.Size([1, 32, 14, 14])
```

### 5.4 池化 vs 卷积下采样

| 方式 | 优点 | 缺点 |
|------|------|------|
| 池化 | 计算简单、无参数量、可增强平移不变性 | 丢失部分信息 |
| 卷积下采样（stride=2） | 可学习、有一定参数量 | 计算量稍大 |

**实践建议：** 现代网络（如 ResNet）倾向于用 stride=2 的卷积替代池化，但也有人习惯池化+卷积混用。没有绝对的最佳方案，需要根据任务调试。


## 6. 积木四：激活函数（Activation Functions）

### 6.1 为什么要激活函数

如果没有激活函数，无论多少层的神经网络，都只是线性变换的叠加（线性变换的叠加还是线性变换），表达能力等同于单层网络。**激活函数引入非线性，让网络能够拟合任意复杂的非线性函数。**

### 6.2 常用激活函数

#### 6.2.1 ReLU（最常用）

```python
# nn.ReLU(inplace=False)
# ReLU(x) = max(0, x)
# 优点：计算简单，收敛快，缓解梯度消失
# 缺点：Dead ReLU Problem（某些神经元可能永远不被激活）

relu = nn.ReLU()
x = torch.tensor([-2.0, -1.0, 0.0, 1.0, 2.0])
y = relu(x)
print(y)   # tensor([0., 0., 0., 1., 2.])
```

#### 6.2.2 Sigmoid

```python
# nn.Sigmoid()
# Sigmoid(x) = 1 / (1 + exp(-x))
# 输出范围 (0, 1)，常用于二分类输出层
# 缺点：容易导致梯度消失（导数最大只有0.25）

sigmoid = nn.Sigmoid()
x = torch.tensor([-2.0, 0.0, 2.0])
y = sigmoid(x)
print(y)   # tensor([0.119, 0.500, 0.881])
```

#### 6.2.3 Tanh

```python
# nn.Tanh()
# Tanh(x) = (exp(x) - exp(-x)) / (exp(x) + exp(-x))
# 输出范围 (-1, 1)，零中心
# 缺点：同样容易梯度消失

tanh = nn.Tanh()
x = torch.tensor([-2.0, 0.0, 2.0])
y = tanh(x)
print(y)   # tensor([-0.964, 0.000, 0.964])
```

#### 6.2.4 LeakyReLU

```python
# nn.LeakyReLU(negative_slope=0.01)
# 解决 ReLU Dead Problem：对负值也给一个很小斜率
# LeakyReLU(x) = x if x > 0 else negative_slope * x

leaky_relu = nn.LeakyReLU(negative_slope=0.01)
x = torch.tensor([-2.0, -1.0, 0.0, 1.0, 2.0])
y = leaky_relu(x)
print(y)   # tensor([-0.020, -0.010, 0.000, 1.000, 2.000])
```

#### 6.2.5 Softmax（多分类输出层专用）

```python
# nn.Softmax(dim=1)  按第1维度（特征维度）做 softmax
# 将输出转换为概率分布，各类别概率之和为1
# 通常放在网络的最后一层

softmax = nn.Softmax(dim=1)
x = torch.randn(3, 10)   # 3个样本，10个类别
y = softmax(x)
print(y.sum(dim=1))   # tensor([1., 1., 1.])  每行概率和为1
```

### 6.3 激活函数对比

| 激活函数 | 公式 | 输出范围 | 适用场景 | 注意事项 |
|---------|------|---------|---------|---------|
| ReLU | max(0,x) | [0, +∞) | 隐藏层（默认首选） | Dead ReLU 问题 |
| LeakyReLU | x>0:x : αx | (-∞, +∞) | 隐藏层（ReLU替代） | α 通常取 0.01 |
| Sigmoid | 1/(1+e^-x) | (0,1) | 二分类输出层 | 梯度消失 |
| Tanh | tanh(x) | (-1,1) | 隐藏层（不常用） | 梯度消失 |
| Softmax | e^xi/Σe^xj | (0,1) | 多分类输出层 | 需配合交叉熵损失 |

### 6.4 在模型中使用激活函数

在 `nn.Module` 的 `forward` 方法中：

```python
class DemoNet(nn.Module):
    def __init__(self):
        super(DemoNet, self).__init__()
        self.fc = nn.Linear(784, 256)
        self.relu = nn.ReLU()      # 实例化
        self.fc2 = nn.Linear(256, 10)
    
    def forward(self, x):
        x = self.relu(self.fc(x))  # ReLU 激活
        x = self.fc2(x)
        return x
```

或者更简洁地使用 `torch.nn.functional`（函数式写法）：

```python
import torch.nn.functional as F

class DemoNet(nn.Module):
    def __init__(self):
        super(DemoNet, self).__init__()
        self.fc1 = nn.Linear(784, 256)
        self.fc2 = nn.Linear(256, 10)
    
    def forward(self, x):
        x = F.relu(self.fc1(x))     # 函数式写法，不需要实例化
        x = self.fc2(x)
        return x
```

**思考题：** 输出层为什么通常不使用 ReLU？

<details>
<summary>点击查看答案</summary>

因为输出层通常需要表示概率（Softmax, Sigmoid）或回归值（可能有正负）。ReLU 的输出永远非负，不适合这些场景。如果做多分类，用 Softmax；如果做二分类，用 Sigmoid；如果做回归，输出层通常不加激活函数（线性输出）。

</details>

## 7. 积木五：Dropout 与正则化

### 7.1 什么是 Dropout

Dropout 是防止过拟合的重要技术。在训练时，随机"丢弃"（置零）网络中的一部分神经元，强迫网络不依赖于任何一个特定的神经元，增强泛化能力。

**重要：** 在测试时，所有神经元都参与计算，且权重需要乘以保留概率（PyTorch 中自动处理）。

### 7.2 PyTorch 实现

```python
# nn.Dropout(p=0.5, inplace=False)
# 参数说明：
#   p:           丢弃概率，默认为 0.5（即每个神经元有50%概率被丢弃）
#   inplace:     是否原地修改，默认为 False

dropout = nn.Dropout(p=0.5)

# 训练模式 vs 评估模式
x = torch.randn(1, 10)

dropout.train()   # 训练模式
y_train = dropout(x)
print(f"训练模式输出（随机置零）: {y_train}")

dropout.eval()    # 评估模式
y_eval = dropout(x)
print(f"评估模式输出（全部保留）: {y_eval}")
```

```python
class NetWithDropout(nn.Module):
    def __init__(self):
        super(NetWithDropout, self).__init__()
        self.fc1 = nn.Linear(784, 256)
        self.dropout = nn.Dropout(p=0.5)   # 隐藏层后加 Dropout
        self.fc2 = nn.Linear(256, 10)
    
    def forward(self, x):
        x = F.relu(self.fc1(x))
        x = self.dropout(x)    # 训练时随机丢弃，评估时全部保留
        x = self.fc2(x)
        return x
```

### 7.3 Dropout 使用技巧

| 场景 | 建议 |
|------|------|
| 全连接层后 | 常用 p=0.5 |
| 卷积层后 | 通常不用 Dropout，用 BatchNorm 替代 |
| 最后一层前 | 通常不加 Dropout |
| p 值过大 | 丢弃过多导致欠拟合 |
| p 值过小 | 防止过拟合效果弱，通常 p<0.3 |

## 8. 积木六：BatchNorm 层

### 8.1 什么是 BatchNorm

BatchNorm（Batch Normalization）在每一个 mini-batch 上对数据进行标准化（均值归零、方差归一），使数据分布稳定，从而：
- 加速收敛
- 缓解梯度消失
- 对初始值不那么敏感
- 有轻微正则化效果

**注意：** BatchNorm 在训练和评估时的行为不同，评估时使用训练期间统计的全局均值/方差。

### 8.2 PyTorch 实现

```python
# nn.BatchNorm2d(num_features, eps=1e-05, momentum=0.1, affine=True)
# 用于 Conv2d 之后，对通道维度做归一化
# 参数说明：
#   num_features:  特征图的通道数（C）
#   eps:           防止除零的较小值
#   momentum:      统计均值/方差的动量
#   affine:        是否包含可学习的γ（scale）和β（shift）参数

bn = nn.BatchNorm2d(num_features=32)

# 模拟输入: batch=4, channels=32, H=28, W=28
x = torch.randn(4, 32, 28, 28)
y = bn(x)

print(f"输入形状: {x.shape}")   # torch.Size([4, 32, 28, 28])
print(f"输出形状: {y.shape}")   # torch.Size([4, 32, 28, 28])
# 注意：BatchNorm 不改变空间尺寸，只对通道维度做归一化
```

### 8.3 BatchNorm vs Dropout

| 特性 | BatchNorm | Dropout |
|------|-----------|---------|
| 作用 | 归一化数据分布 | 随机丢弃神经元 |
| 位置 | 卷积层/全连接层之后 | 全连接层之后 |
| 目的 | 加速收敛、稳定训练 | 防止过拟合 |
| 副作用 | 有轻微正则化效果 | 可能丢失有用信息 |

**常见的组合模式：**

```python
# 模式1: Conv → BatchNorm → ReLU（推荐）
x = F.relu(nn.BatchNorm2d(C)(conv(x))))

# 模式2: Conv → ReLU → BatchNorm（也常见）
x = nn.BatchNorm2d(C)(F.relu(conv(x)))

# 模式3: 不使用 BatchNorm，用 Dropout 防止过拟合
x = F.relu(conv(x))
x = nn.Dropout(0.5)(x)
```

## 9. 数据增强工具箱

> **本节核心思想：** 深度学习是"数据驱动"的。数据量不够、数据多样性不足，是导致过拟合的主要原因之一。数据增强（Data Augmentation）通过对训练数据施加随机变换，人工扩充数据的多样性，让模型学到的特征更鲁棒。

### 9.1 高斯模糊（Gaussian Blur）

高斯模糊模拟相机失焦或运动模糊，让模型对图像清晰度不那么敏感。

```python
import torchvision.transforms as T

# 高斯模糊：模糊半径为 r 的高斯卷积核
# kernel_size 必须是奇数
blur_transform = T.GaussianBlur(kernel_size=5, sigma=(0.1, 2.0))
# kernel_size=5: 5×5 卷积核
# sigma=(0.1, 2.0): 模糊程度随机在0.1到2.0之间
```

### 9.2 膨胀/腐蚀（Dilation & Erosion）

膨胀（Dilation）和腐蚀（Erosion）是形态学操作，常用于处理二值图像或增强某些特征。

```python
import cv2
import numpy as np

# OpenCV 实现膨胀
def dilate_image(image, kernel_size=3, iterations=1):
    """膨胀操作：扩大白色区域（前景）"""
    kernel = np.ones((kernel_size, kernel_size), np.uint8)
    dilated = cv2.dilate(image, kernel, iterations=iterations)
    return dilated

# OpenCV 实现腐蚀
def erode_image(image, kernel_size=3, iterations=1):
    """腐蚀操作：缩小白色区域（前景）"""
    kernel = np.ones((kernel_size, kernel_size), np.uint8)
    eroded = cv2.erode(image, kernel, iterations=iterations)
    return eroded

# 示例：对手写数字图像进行随机膨胀或腐蚀
def random_morphology(image, p=0.5):
    """以概率 p 对图像进行随机膨胀或腐蚀"""
    if np.random.rand() < p:
        op = np.random.choice(['dilate', 'erode'])
        if op == 'dilate':
            return dilate_image(image, kernel_size=2, iterations=1)
        else:
            return erode_image(image, kernel_size=2, iterations=1)
    return image
```

### 9.3 噪声（Noise）

向图像中添加噪声，模拟真实环境中的各种干扰。

```python
import numpy as np
import cv2

def add_gaussian_noise(image, mean=0.0, std=0.1):
    """添加高斯噪声"""
    noise = np.random.randn(*image.shape) * std
    noisy = np.clip(image + noise, 0, 255).astype(np.uint8)
    return noisy

def add_salt_pepper_noise(image, amount=0.02):
    """添加椒盐噪声（随机黑白点）"""
    noisy = image.copy()
    salt_mask = np.random.rand(*image.shape[:2]) < amount / 2
    noisy[salt_mask] = 255
    pepper_mask = np.random.rand(*image.shape[:2]) < amount / 2
    noisy[pepper_mask] = 0
    return noisy
```

### 9.4 扭曲/仿射变换（Affine Transformation）

通过随机旋转、平移、缩放、剪切等仿射变换，增加数据的多样性。

```python
import torchvision.transforms as T

# RandomAffine: 随机仿射变换，包含旋转、平移、缩放、剪切
affine_transform = T.RandomAffine(
    degrees=15,          # 旋转角度范围: -15° ~ +15°
    translate=(0.1, 0.1), # 平移: 最多移动图像宽高的10%
    scale=(0.9, 1.1),     # 缩放: 0.9倍 ~ 1.1倍
    shear=5              # 剪切: -5° ~ +5°
)
```

### 9.5 旋转（Rotation）

旋转是数据增强中最有效的手段之一，但需要注意旋转后图像信息是否仍然有效。

```python
# 随机旋转
rotate_transform = T.RandomRotation(degrees=(-30, 30))  # -30° ~ +30° 随机旋转

# 训练时的完整数据增强流水线
train_transform = T.Compose([
    T.RandomRotation(degrees=(-15, 15)),     # 随机旋转 ±15°
    T.RandomAffine(degrees=0, translate=(0.1, 0.1), scale=(0.9, 1.1)),  # 平移+缩放
    T.GaussianBlur(kernel_size=3, sigma=(0.1, 1.0)),  # 轻微模糊
    T.ToTensor(),                             # 转张量
    T.Normalize((0.5,), (0.5,))                # 标准化
])
```

### 9.6 数据增强对比一览

| 增强方法 | 效果 | 适用场景 | 注意事项 |
|---------|------|---------|---------|
| 旋转 | 增加方向多样性 | 手写数字、物体分类 | 旋转角度过大会破坏数字语义 |
| 平移 | 增加位置多样性 | 几乎所有任务 | MNIST 通常移动 ±2 像素即可 |
| 翻转 | 数据翻倍（但MNIST不能水平翻转） | 自然图像分类 | 注意翻转后语义是否一致 |
| 缩放 | 增加尺度多样性 | 物体检测、分类 | 不要缩放太多导致目标消失 |
| 高斯模糊 | 模拟失焦/运动模糊 | 相机图像 | 模糊程度适中 |
| 噪声 | 增强抗干扰能力 | 低光环境、高噪声场景 | 噪声过大反而干扰学习 |
| 膨胀/腐蚀 | 模拟物体大小变化 | 字符/数字识别 | 对MNIST这种细线条效果明显 |
| Cutout/RandomErasing | 随机遮挡 | 提高鲁棒性 | 遮挡比例不要太大 |

### 9.7 实验：高增强 vs 低增强

以下代码对比了"无增强"和"强增强"两种策略对训练的影响：

### 参考网络结构

```python
class ReferenceCNN(nn.Module):
    def __init__(self):
        super(ReferenceCNN, self).__init__()
        # 卷积层1: 1→32 通道, 3×3, padding=1
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3, padding=1)
        self.bn1 = nn.BatchNorm2d(32)
        # 卷积层2: 32→64 通道, 3×3, padding=1
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.bn2 = nn.BatchNorm2d(64)
        # 池化层: 2×2 最大池化
        self.pool = nn.MaxPool2d(2, 2)
        # 全连接层
        # 输入维度: 64 channels × 7×7 = 3136
        self.fc1 = nn.Linear(64 * 7 * 7, 128)
        self.dropout = nn.Dropout(0.5)
        self.fc2 = nn.Linear(128, 10)
    
    def forward(self, x):
        # Conv1 → BN → ReLU → Pool
        x = self.pool(F.relu(self.bn1(self.conv1(x))))
        # Conv2 → BN → ReLU → Pool
        x = self.pool(F.relu(self.bn2(self.conv2(x))))
        # 展平
        x = x.view(x.size(0), -1)
        # FC → ReLU → Dropout → FC
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.fc2(x)
        return x
```

## 附录：常见报错速查

| 报错信息 | 原因 | 解决方法 |
|---------|------|---------|
| `RuntimeError: Expected 4D input to Conv2d` | 展平时机不对 | 检查 forward 中卷积层输入是否为 [B, C, H, W] |
| `size mismatch, m1: [B×N], m2: [D, K]` | FC 层输入维度计算错误 | 重新核算展平后的维度 |
| `Accuracy = 10%`（接近随机猜测） | 学习率太大/太小，或数据没归一化 | 调大学习率，或检查数据归一化 |
| `Loss 不下降` | 激活函数配置错误（如最后一层用了 ReLU） | 检查输出层是否有不必要的激活 |
| `训练准确率高但测试准确率低` | 过拟合 | 加 Dropout、BatchNorm 或数据增强 |
| `CUDA out of memory` | Batch size 太大或网络太深 | 减小 batch_size 或减少通道数 |
