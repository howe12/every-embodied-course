## 1. 计算机视觉概述

### 1.1 定义与发展历史

**计算机视觉**（Computer Vision，CV）是一门研究如何使机器"看"的科学。它的目标是让计算机能够从图像、视频或多维数据中提取出对世界的理解，实现对物体的识别、跟踪、测量、重建等任务。简单来说，计算机视觉就是要让机器拥有和人类一样的视觉能力——不仅能"看到"画面，还能"看懂"画面中的内容。

计算机视觉的发展可以追溯到20世纪60年代。1966年，MIT的Marvin Minsky让他的学生花费一个夏天实现"让机器连接摄像头并描述看到了什么"。当时人们乐观地认为这是一个可以在一个夏天解决的小问题。然而，这个"夏天"持续了半个多世纪，至今仍是活跃的研究领域。

**计算机视觉发展大事记：**

| 年份 | 里程碑事件 | 意义 |
|------|-----------|------|
| 1966 | MIT "Summer Vision Project" 启动 | 计算机视觉作为独立学科的起点 |
| 1970s | Marr 视觉计算理论 | 奠定了计算机视觉的理论基础 |
| 1980s | 反向传播算法（Backpropagation） | 神经网络开始应用于视觉任务 |
| 1990s | E. R. Davies、Singh等学者推动 | 机器学习进入视觉领域 |
| 1999 | OpenCV 发布（Intel） | 开源视觉库降低开发门槛 |
| 2012 | AlexNet 在 ImageNet 大赛夺冠 | 深度学习引爆视觉革命 |
| 2014 | VGGNet、GoogLeNet | 视觉模型走向深层化 |
| 2015 | ResNet 超越人类识别率 | 深度卷积网络在分类任务上超越人类 |
| 2017 | YOLOv3、DensePose | 实时目标检测和人体姿态估计成熟 |
| 2020 | Vision Transformer (ViT) | Transformer架构进入视觉领域 |
| 2023 | Segment Anything Model (SAM) | Foundation Model引爆视觉 |
| 2024 | GPT-4V等多模态大模型 | 视觉与语言深度融合 |

**为什么计算机视觉这么难？** 让我们做一个简单的对比。人类看到一张猫咪的照片，可以轻松说出"这是一只橘猫，它正在沙发上打盹"。但对计算机来说，一张图像仅仅是像素值——比如一张 640×480 的图片，就是一个 307200 个数字组成的矩阵。每个数字代表一个像素的亮度（或颜色值）。计算机需要从这个冰冷的数字矩阵中重建出"语义"，这个过程被称为**语义鸿沟**（Semantic Gap），是计算机视觉面临的核心挑战。

```
人类视角：          计算机视角：
"一只橘猫在沙发上"   →  [23, 45, 67, 89, ...]  (640×480=307200个数值)
                    ↓
              图像理解的过程（语义鸿沟）
```

### 1.2 与机器人领域的关系

计算机视觉和机器人学是一对"孪生兄弟"，它们的关系可以从以下几个维度来理解：

**（2）机器人视觉的独特挑战**

机器人视觉与普通计算机视觉有一个本质区别：**机器人视觉是实时、有物理意义的**。机器人需要：

- **实时性**：毫秒级响应，跟上控制频率（通常 50-100Hz）
- **物理一致性**：感知结果要符合物理世界规律（深度不能是负数，距离要合理）
- **闭环控制**：视觉感知结果直接驱动机器人动作，感知错误会导致动作错误
- **鲁棒性**：要在各种光照、天气、运动模糊下稳定工作

**（3）视觉是具身智能的核心**

具身智能（Embodied AI）强调智能体与物理世界的交互。视觉提供了机器人理解环境的主要输入，典型的任务映射如下：

| 视觉任务 | 机器人行为 |
|---------|-----------|
| 目标检测 | 让机器人找到并接近目标物体 |
| 深度估计 | 让机器人知道目标有多远 |
| 语义分割 | 让机器人理解哪些区域是可行驶的 |
| 人体姿态估计 | 让机器人识别人类意图并安全交互 |
| 3D重建 | 让机器人构建环境地图 |
| 目标跟踪 | 让机器人持续追踪移动的目标 |
| 视觉里程计 | 让机器人通过视觉估算自己的运动 |

## 2. 视觉传感器类型

### 2.1 单目相机

**单目相机**（Monocular Camera）是最简单也最常见的视觉传感器。它本质上是一个成像透镜，将3D世界投影到2D图像平面，就像我们的一只眼睛。

**成像原理——小孔成像模型**

单目相机的成像原理基于**小孔成像模型**（Pinhole Camera Model）。用一个极简的比喻：想象你在一个暗室中，只开一个小孔让光线进入，光线直线传播到对面的墙上形成倒立的图像。

```
        真实世界中的物体
              |
              |  光线（直线传播）
              |
         +---------+
    -----|  小孔   |-----   相机光心 (Pinhole / Optical Center)
         +---------+            |
              |                  | 焦距 f (Focal Length)
              |                  |
         +---------+        +---------+
         | 图像平面 |        | 图像平面 |   实际成像平面
         | (倒立)   |        | (正立)   |   (感光元件/Chip)
         +---------+        +---------+
          距离 f            距离 f
```

**关键公式——投影变换**

3D世界坐标 $(X, Y, Z)$ 到2D像素坐标 $(u, v)$ 的变换关系为：

$$
\begin{bmatrix} u \\ v \\ 1 \end{bmatrix} = \frac{1}{Z} \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix} \begin{bmatrix} X \\ Y \\ Z \end{bmatrix}
$$

其中：
- $f_x, f_y$ 是焦距（以像素为单位，$f_x = f \cdot s_x$，$s_x$ 是像素在x方向的物理尺寸的倒数）
- $c_x, c_y$ 是光心偏移（通常接近图像宽高的一半）
- $Z$ 是深度，即物体到相机的距离

**这是单目相机最核心的局限：无法从单张图像中直接恢复深度 $Z$！** 因为投影过程丢失了深度信息——不同深度的物体，只要满足一定的比例关系，就可以在图像上形成相同的像素位置。这就是所谓的**尺度歧义**（Scale Ambiguity）。

**单目相机的分类：**

| 类型 | 焦距 | 视野角 | 特点 | 机器人应用 |
|------|------|--------|------|-----------|
| 广角/鱼眼 | 短（1-3mm） | 120°-180° | 视野大，畸变大 | 广角避障、全景监控 |
| 普通镜头 | 中等（3-8mm） | 30°-70° | 接近人眼，畸变小 | 通用视觉感知 |
| 长焦镜头 | 长（8mm+） | <30° | 看得远，视野窄 | 远距离目标识别 |
| 远心镜头 | 特殊设计 | 恒定倍率 | 无透视畸变 | 工业精密测量 |

**单目相机的优点：** 成本最低，方案成熟；分辨率高（可达千万像素甚至上亿）；体积小、重量轻、功耗低；技术最成熟，生态最完善。

**单目相机的缺点：** 无法直接获取深度信息，尺度不确定。

**在机器人中的应用：**

```python
# 单目相机的典型应用代码示例
# 使用 OpenCV 读取相机图像
import cv2
import numpy as np

def monocular_camera_example():
    """
    单目相机使用示例
    演示：图像采集 -> 显示 -> 基本处理
    """
    # 创建默认相机（通常为 0，即第一个摄像头）
    cap = cv2.VideoCapture(0)
    
    if not cap.isOpened():
        print("错误：无法打开相机")
        return None
    
    # 设置相机分辨率（可选，根据需求调整）
    cap.set(cv2.CAP_PROP_FRAME_WIDTH, 640)    # 宽度
    cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 480)  # 高度
    cap.set(cv2.CAP_PROP_FPS, 30)             # 帧率
    
    # 获取实际设置的参数
    width = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
    height = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))
    fps = cap.get(cv2.CAP_PROP_FPS)
    
    print(f"相机分辨率：{width} x {height}")
    print(f"帧率：{fps} FPS")
    
    # 读取一帧图像
    ret, frame = cap.read()
    
    if ret:
        # OpenCV 读取的图像是 BGR 格式（Blue-Green-Red）
        # 转换为 RGB 格式用于显示
        frame_rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
        
        print(f"图像形状：{frame.shape}")  # 例如：(480, 640, 3)
        print(f"数据类型：{frame.dtype}")  # 例如：uint8
        print(f"像素值范围：[{frame.min()}, {frame.max()}]")
    else:
        print("错误：无法读取图像帧")
    
    # 释放相机资源
    cap.release()
    
    return frame

# 运行示例
image = monocular_camera_example()
```

### 2.2 双目相机（Stereo Vision）

**双目相机**（Stereo Camera）使用两个具有一定基线距离的相机同时观察同一场景，通过**视差**（Disparity）原理恢复深度信息。

**为什么需要两个眼睛？**

人为什么有两只眼睛？因为单只眼睛看世界时会丢失深度信息。两只眼睛从略微不同的角度观察同一物体，左右眼看到的图像有所差异，大脑利用这个差异计算深度。这就是**立体视觉**（Stereo Vision）的原理。

**视差原理**

```
    左眼位置 (C_l)              右眼位置 (C_r)
         |                         |
         |                         |
    +----x--------------------x----+   图像平面
      物体在左图像的位置        物体在右图像的位置
         x_l                        x_r
      
    基线距离：B = |C_l - C_r|（两个相机光心之间的距离）
    焦距：f
    深度：Z
    
    视差 d = x_l - x_r （同一物体在左右图像中的像素位置差）
```

由相似三角形关系可得深度计算公式：

$$
Z = \frac{f \cdot B}{d}
$$

其中 $Z$ 是深度（物体到相机的距离），$f$ 是焦距，$B$ 是基线距离，$d$ 是视差。

**关键洞察**：视差 $d$ 越大，深度 $Z$ 越小（物体越近）；视差 $d$ 越小（趋近于0），深度 $Z$ 越大（物体越远）。当 $d=0$ 时，物体在无穷远。

**深度误差分析**

双目视觉的深度误差（标准差）公式为：

$$
\sigma_Z = \frac{Z^2}{f \cdot B} \cdot \sigma_d
$$

这个公式揭示了一个重要规律：**深度误差与深度的平方成正比**！这意味着：
- 近处物体深度估计很准确
- 远处物体深度误差迅速增大
- 要提高远距离深度精度，需要增大基线距离 $B$

**常见双目相机产品：**

| 产品 | 特点 | 基线 | 分辨率 | 接口 |
|------|------|------|--------|------|
| Intel RealSense D455 | RGB-D一体，红外双目 | 95mm | 1280×720 | USB-C |
| ZED 2 / ZED 2i | 高分辨率，AI功能强 | 120mm | 3840×1080 | USB 3.0 |
| ELP 双目相机 | 入门级，价格便宜 | 可选 | 多种 | USB |

**双目相机的优点：** 可以获取稠密深度图；不依赖额外光源，室内外都可用；相比RGB-D相机，深度范围更广。

**双目相机的缺点：** 深度精度随距离平方衰减；弱纹理区域（白墙、天空）匹配困难；计算复杂度高。

```python
import numpy as np

def stereo_depth_calculation(focal_length_mm, baseline_mm, disparity):
    """
    根据视差计算深度
    
    参数:
        focal_length_mm: 焦距（毫米）
        baseline_mm: 基线距离（毫米）
        disparity: 视差（像素）
    
    返回:
        depth_m: 深度（米）
    """
    # 传感器单个像素物理尺寸（以 IMX219 为例，单位：毫米）
    pixel_size_mm = 0.00112  # 对于 1/4" 传感器，约 1.12μm
    
    # 将焦距转换为像素单位
    focal_length_px = focal_length_mm / pixel_size_mm
    
    if disparity <= 0:
        return float('inf')  # 视差为0表示物体在无穷远
    
    # 计算深度（毫米）
    depth_mm = (focal_length_px * baseline_mm) / disparity
    
    return depth_mm / 1000.0  # 转换为米

# 示例：Intel RealSense D455 双目相机参数
focal_length = 1.88   # mm（Realsense D455 RGB相机焦距约1.88mm）
baseline = 95          # mm（左右红外相机基线约95mm）

# 假设检测到一个人形物体，视差为50像素
disparity = 50
depth = stereo_depth_calculation(focal_length, baseline, disparity)
print(f"检测到物体，视差 {disparity} 像素 -> 深度 {depth:.2f} 米")

# 视差为10像素（物体较近）
depth_near = stereo_depth_calculation(focal_length, baseline, 10)
```

### 2.3 RGB-D相机

**RGB-D相机**（RGB-Depth Camera）可以同时获取**彩色图像（RGB）**和**深度图（Depth）**。它不是通过视差计算深度，而是通过其他技术手段直接测量深度。

**RGB-D相机的三种技术路线：**

**（1）结构光（Structured Light）**

结构光的工作原理：向目标表面投射一个已知的光模式（通常是黑白条纹或点阵），然后用另一个相机观察这个模式因物体表面起伏而产生的畸变，从而计算出深度。**Kinect v1** 是最著名的结构光相机（2010年）。不过结构光在室外强光下效果很差。

**（2）飞行时间法（ToF - Time of Flight）**

ToF 相机发射一束调制光（如近红外脉冲），测量光从发出到返回的时间：

$$
Z = \frac{c \cdot \Delta t}{2}
$$

其中 $c \approx 3 \times 10^8$ m/s 是光速，$\Delta t$ 是飞行时间。

实际上，由于直接测量皮秒级时间非常困难，ToF 相机通常测量**相位差**——发射光和接收光之间的相位偏移与深度成正比。

**（3）主动双目（Active Stereo）**

在普通双目相机基础上，增加红外投影器主动投射纹理，增强弱纹理区域的匹配效果。**Intel RealSense D400 系列**采用此方案。

```python
import numpy as np

def tof_depth_from_phase(phase_shift, modulation_freq):
    """
    根据相位偏移计算 ToF 深度
    
    参数:
        phase_shift: 相位偏移（弧度）
        modulation_freq: 调制频率（Hz）
    
    返回:
        depth_m: 深度（米）
    """
    speed_of_light = 3e8  # 光速 m/s
    
    # 调制波长 = 光速 / 调制频率
    wavelength = speed_of_light / modulation_freq
    
    # 相位差 → 距离（往返）
    round_trip_distance = (phase_shift / (2 * np.pi)) * wavelength
    
    # 单程深度 = 往返距离 / 2
    depth = round_trip_distance / 2
    
    return depth

# 示例：调制频率 20 MHz
mod_freq = 20e6  # 20 MHz
wavelength = 3e8 / 20e6  # = 15 米（往返调制波长）
print(f"调制波长（往返）：{wavelength:.1f} 米")
print(f"单值距离范围：0 到 {wavelength/2:.2f} 米")

# 假设相位偏移为 π（半个周期）
phase = np.pi
depth = tof_depth_from_phase(phase, mod_freq)
print(f"相位偏移 π → 深度 {depth:.2f} 米")
```

### 2.4 事件相机（Event Camera）

**事件相机**（Event Camera），也叫**动态视觉传感器**（DVS - Dynamic Vision Sensor），是一种全新的视觉传感器范式。与传统相机输出固定频率的帧图像不同，事件相机以**事件**（Event）为基本单位进行异步输出。

**传统相机 vs 事件相机**

传统相机的工作方式：以固定频率（如30fps）输出完整图像，无论场景是否有变化，每帧都要完整传输，高速运动时会产生运动模糊。

事件相机的工作方式：每个像素独立工作，检测光强变化，当某个像素的光强变化超过阈值时，输出一个事件。事件包含：`timestamp`（时间戳）、`x`、`y`（像素位置）、`polarity`（变化方向：变亮/变暗）。

```
传统相机输出（时间 →）：
[帧1] [帧2] [帧3] [帧4] ...  (每帧包含所有像素，不管有无变化)

事件相机输出（时间 →）：
事件1 @ t=1.001s: (x=100, y=200, +1)  ← 像素(100,200)变亮了
事件2 @ t=1.002s: (x=150, y=200, -1)  ← 像素(150,200)变暗了
事件3 @ t=1.005s: (x=101, y=200, +1)  ← 像素(101,200)变亮了
...  ← 只输出有变化的像素
```

**事件相机的数据格式：**

```python
# 事件相机输出的事件数据
# 每个事件通常包含4个字段
events = [
    {"t": 1.001234, "x": 100, "y": 200, "p": 1},   # t=时间戳(s), x/y=像素坐标, p=极性(+1变亮/-1变暗)
    {"t": 1.002567, "x": 150, "y": 200, "p": -1},
    {"t": 1.005891, "x": 101, "y": 200, "p": 1},
]

print("事件相机数据特点：")
print(f"- 每个事件的时间戳精度：微秒级 (μs)")
print(f"- 传统相机帧率：通常 30-120 FPS = 每帧 33ms-8ms")
print(f"- 事件相机时间分辨率：可达 1μs")
print(f"- 数据量：仅在变化时产生数据，静止场景数据量接近零")
```

**事件相机的核心优势：**

| 特性 | 传统相机 | 事件相机 |
|------|---------|---------|
| 动态范围 | 60-90 dB | **120+ dB** |
| 时间分辨率 | 毫秒级（帧率倒数） | **微秒级** |
| 运动模糊 | 高速运动时严重 | **几乎无** |
| 功耗 | 数百毫瓦 | **数十毫瓦** |
| 低延迟 | 数十毫秒 | **微秒级** |

**事件相机的不足：** 数据解读复杂（需要专门算法）；无法直接获取纹理和颜色；成本较高；算法生态不成熟。

**事件相机在机器人领域的应用：**
- **高速运动无人机避障**：高速旋转的螺旋桨，普通相机会模糊，事件相机可以清晰捕捉
- **高速运动机器人**：如竞技机器人乒乓球，需要毫秒级反应
- **高动态范围场景**：从隧道内突然驶出到阳光下，普通相机需要时间调整曝光
- **低功耗嵌入式应用**：事件相机的功耗远低于传统相机，适合电池供电的移动机器人

```python
# 创建一个ROS2图像处理节点（使用 rclpy）
# 文件：image_processing_node.py

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from cv_bridge import CvBridge
import cv2
import numpy as np


class ImageProcessingNode(Node):
    """
    ROS2 图像处理节点示例
    
    功能：
    1. 订阅原始图像话题
    2. 进行图像处理（灰度转换）
    3. 发布处理后的图像
    """
    
    def __init__(self):
        super().__init__('image_processing_node')
        
        # 创建 CvBridge，用于 ROS 图像消息和 OpenCV 图像之间的转换
        self.bridge = CvBridge()
        
        # 订阅原始图像话题
        self.subscription = self.create_subscription(
            Image,
            '/image_raw',              # 原始图像话题名
            self.image_callback,       # 回调函数
            10                         # 队列大小
        )
        self.subscription  # 防止未使用警告
        
        # 发布处理后的图像话题
        self.publisher = self.create_publisher(
            Image,
            '/image_processed',
            10
        )
        
        self.get_logger().info('图像处理节点已启动')
        self.get_logger().info('订阅话题: /image_raw')
        self.get_logger().info('发布话题: /image_processed')
    
    def image_callback(self, msg):
        """
        图像回调函数：每次收到新图像时调用
        
        参数:
            msg: sensor_msgs/Image 类型的消息
        """
        try:
            # 方法1：直接转换（如果 encoding 已经是需要的格式）
            # cv_image = self.bridge.imgmsg_to_cv2(msg, desired_encoding='bgr8')
            
            # 方法2：使用 tryCatch 处理可能的异常
            cv_image = self.bridge.imgmsg_to_cv2(msg, desired_encoding='passthrough')
            
            # 将 BGR 图像转换为灰度图
            if cv_image.ndim == 3:
                gray_image = cv2.cvtColor(cv_image, cv2.COLOR_BGR2GRAY)
            else:
                gray_image = cv_image
            
            # 将处理后的图像转换回 ROS 消息并发布
            processed_msg = self.bridge.cv2_to_imgmsg(gray_image, encoding='mono8')
            processed_msg.header = msg.header  # 保持时间戳和坐标系不变
            
            self.publisher.publish(processed_msg)
            
        except Exception as e:
            self.get_logger().error(f'图像处理失败: {str(e)}')


def main(args=None):
    """
    节点入口函数
    """
    rclpy.init(args=args)
    node = ImageProcessingNode()
    
    try:
        rclpy.spin(node)  # 保持节点运行，持续处理图像
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()
```

**OpenCV 常用图像处理操作：**

```python
import cv2
import numpy as np

def opencv_basic_operations(image_path=None):
    """
    OpenCV 常用基础操作演示
    """
    # 如果没有提供图像路径，创建一个测试用的彩色图
    if image_path is None:
        # 创建一个 480x640 的彩色测试图像
        # 蓝色渐变背景
        image = np.zeros((480, 640, 3), dtype=np.uint8)
        for i in range(480):
            image[i, :] = [int(i / 480 * 255), 100, 200]  # BGR 格式
        print("使用测试彩色图像")
    else:
        # 读取图像
        image = cv2.imread(image_path)
        if image is None:
            print(f"无法读取图像: {image_path}")
```

**OpenCV 标定工具的使用：**

机器人使用相机前，通常需要进行**标定**（Calibration），获取相机的内参和畸变系数：

```python
import numpy as np
import cv2

def calibrate_camera(images_with_checkerboards):
    """
    使用棋盘格进行相机标定
    
    参数:
        images_with_checkerboards: 包含棋盘格视角的图像列表
    
    返回:
        mtx: 相机内参矩阵 (3x3)
        dist: 畸变系数 (1x5)
        rvecs: 旋转向量
        tvecs: 平移向量
    """
    # 棋盘格参数（已知尺寸）
    CHECKERBOARD = (9, 6)  # 内部角点数量 (列 x 行)
    square_size = 0.025    # 每个棋盘格方块的实际尺寸（米）
    
    # 准备对象点 (0,0,0), (1,0,0), (2,0,0) .... (8,5,0)
    objp = np.zeros((CHECKERBOARD[0] * CHECKERBOARD[1], 3), np.float32)
    objp[:, :2] = np.mgrid[0:CHECKERBOARD[0], 0:CHECKERBOARD[1]].T.reshape(-1, 2)
    objp *= square_size
    
    # 存储所有图像的3D点和2D点
    objpoints = []  # 3D点（在真实世界坐标系中）
    imgpoints = []  # 2D点（在图像平面中）
    
    for img in images_with_checkerboards:
        gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
        
        # 查找棋盘格角点
        ret, corners = cv2.findChessboardCorners(gray, CHECKERBOARD, None)
        
        if ret:
            objpoints.append(objp)
            
            # 精确化角点位置（亚像素精度）
            criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
            corners2 = cv2.cornerSubPix(gray, corners, (11, 11), (-1, -1), criteria)
            imgpoints.append(corners2)
    
    # 执行相机标定
    ret, mtx, dist, rvecs, tvecs = cv2.calibrateCamera(
        objpoints, imgpoints, gray.shape[::-1], None, None
    )
    
    print(f"标定结果:")
    print(f"  内参矩阵 (K):\n{mtx}")
    print(f"  畸变系数: {dist.ravel()}")
    print(f"  重投影误差: {ret:.4f} 像素")
    
    return mtx, dist, rvecs, tvecs

def undistort_image(image, mtx, dist):
    """
    根据标定结果校正图像畸变
    
    参数:
        image: 原始图像（有畸变）
        mtx: 相机内参矩阵
        dist: 畸变系数
    
    返回:
        undistorted: 校正后的图像
    """
    h, w = image.shape[:2]
    
    # 优化相机内参矩阵（基于图像尺寸）
    new_mtx, roi = cv2.getOptimalNewCameraMatrix(mtx, dist, (w, h), 1, (w, h))
    
    # 校正畸变
    undistorted = cv2.undistort(image, mtx, dist, None, new_mtx)
    
    # 裁剪有效区域（可选）
    x, y, w, h = roi
    if w > 0 and h > 0:
        undistorted = undistorted[y:y+h, x:x+w]
    
    return undistorted
```

在计算机视觉概述部分，我们了解到计算机视觉是一门让机器"看懂"世界的科学，从1966年MIT的Summer Vision Project发展到今天，已经广泛应用于机器人、自动驾驶、工业检测等众多领域。其核心挑战是跨越"语义鸿沟"——从冰冷的像素数值中重建出语义理解。在机器人领域，视觉是感知系统的核心，负责目标检测、深度估计、人体感知、视觉SLAM、视觉伺服等关键任务。

在视觉传感器类型部分，我们学习了四种主流传感器：单目相机（成本低、应用广，但无法直接获取深度）、双目相机（通过视差原理恢复深度，但精度随距离平方衰减）、RGB-D相机（主动测量深度，但范围受限）、事件相机（异步输出、高动态范围、低延迟，但算法生态不成熟）。每种传感器都有其适用场景，需要根据具体任务进行选择。

在机器人视觉系统架构部分，我们了解了图像从采集到处理的完整流程：物理世界→相机传感器→ISP处理→数据传输→计算平台→视觉算法→感知结果。在ROS2中，图像以`sensor_msgs/Image`消息格式传输，通过`cv_bridge`与OpenCV无缝衔接。OpenCV作为最广泛使用的开源视觉库，提供了从基础图像处理到复杂视觉算法的全套工具。

**题目1.1：** **C. RGB-D相机**

解析：RGB-D相机（如Intel RealSense、Microsoft Kinect）通过结构光或ToF技术可以直接测量每个像素的深度值，输出稠密深度图。单目相机无法直接获取深度（尺度歧义）；双目相机需要通过视差计算深度，但精度随距离衰减；事件相机输出的是动态变化事件，本身不包含深度信息。

**题目1.2：** **B. 视差（Disparity）**

解析：双目视觉利用"视差"原理恢复深度。同一物体在左右相机图像中的像素位置存在差异（视差 $d = x_l - x_r$），根据公式 $Z = \frac{f \cdot B}{d}$ 可以计算深度。视差越大，物体越近；视差越小（趋近于0），物体越远（无穷远）。

```python
import numpy as np

def stereo_depth_from_disparity(focal_length_px, baseline_mm, disparity):
    """
    根据视差计算深度
    
    参数:
        focal_length_px: 焦距（像素）
        baseline_mm: 基线距离（毫米）
        disparity: 视差（像素）

        depth_m: 深度（米）
```

```python
import numpy as np

def pixel_to_3d(u, v, Z, K):
    """
    将图像像素坐标和深度值转换为3D相机坐标系下的坐标
    
    参数:
        u: 像素x坐标（列）
        v: 像素y坐标（行）
        Z: 深度值，即该像素对应点在相机坐标系下的Z坐标（米）
        K: 相机内参矩阵 (3x3 numpy数组)
            K = [[fx, 0, cx],
                 [0, fy, cy],
                 [0,  0,  1]]
    
    返回:
        point_3d: 3D坐标 (X, Y, Z)，单位为米
    
    数学推导:
        像素坐标系到相机坐标系的反投影：
        [X   [fx  0 cx]^-1 [u]
         Y  =           *  [v]
         Z]            [1 ]
    
        即：point_3d = Z * K^(-1) * [u, v, 1]^T
    """
    # 将像素坐标转为齐次坐标
    pixel_homogeneous = np.array([u, v, 1.0])
    
    # 计算相机内参矩阵的逆
    K_inv = np.linalg.inv(K)
    
    # 反投影到归一化相机坐标系
    normalized_coords = K_inv @ pixel_homogeneous
    
    # 乘以深度 Z，得到3D坐标
    point_3d = Z * normalized_coords
    
    return point_3d  # 返回 [X, Y, Z]


def pixel_to_3d_example():
    """
    示例演示：使用本节中的相机内参，计算图像中心点对应的3D坐标
    """
    # 相机内参矩阵（以 640x480 图像为例）
    K = np.array([
        [607.5, 0.0, 324.6],
```

**答案解析：**

本题考察的是小孔成像模型的正逆变换。根据投影公式：

$$
Z \begin{bmatrix} u \\ v \\ 1 \end{bmatrix} = K \begin{bmatrix} X \\ Y \\ Z \end{bmatrix}
$$

其逆过程为：

$$
\begin{bmatrix} X \\ Y \\ Z \end{bmatrix} = Z \cdot K^{-1} \begin{bmatrix} u \\ v \\ 1 \end{bmatrix}
$$

这个变换是机器人视觉中的基础操作——当我们通过RGB-D相机获取了深度图之后，每个像素 $(u, v)$ 都有对应的深度 $Z$，通过这个公式可以恢复该像素对应的3D点在相机坐标系下的位置 $(X, Y, Z)$。这是后续进行抓取规划、避障、3D重建等任务的基础。
