## 1. 深度相机原理

### 1.1 为什么需要深度信息？

在机器人感知中，二维图像虽然包含了丰富的纹理和颜色信息，但存在一个根本性的缺陷：**缺失深度维度**。当机器人看到"前方有一个杯子"时，仅仅知道杯子在图像中的像素位置 $(u, v)$ 是不够的——机器人还需要知道杯子距离自己有多远，才能决定机械臂需要伸多长、以什么角度接近。

```
二维图像感知（单目相机）：
┌─────────────────────┐
│    ○ ← 杯子在图像中  │
│         的某个位置    │
│                     │
│  问题：不知道距离有多远│
└─────────────────────┘

三维点云感知（深度相机）：
┌─────────────────────────────┐
│         3D场景              │
│                             │
│      ○ ← 知道杯子的3D位置   │
│      (x=0.3m, y=0.1m, z=0.5m)│
│                             │
│  ✓ 可以精确规划机械臂抓取   │
└─────────────────────────────┘
```

**深度信息在机器人领域的关键作用：**

| 应用场景 | 为什么需要深度 | 具体任务 |
|---------|--------------|---------|
| 机械臂抓取 | 知道目标距离，决定伸手长度 | 从桌上抓取物体 |
| 移动避障 | 知道障碍物距离，决定绕行 | 室内导航、路径规划 |
| 3D建图 | 构建环境的三维模型 | SLAM建图、场景重建 |
| 人体姿态估计 | 恢复人体的3D骨骼 | 人体动作捕捉 |
| 质量检测 | 测量物体表面3D形状 | 工业零件缺陷检测 |

### 1.2 深度测量的三种主流技术

深度相机根据测量原理不同，主要分为三大类：**结构光（Structured Light）**、**飞行时间法（ToF）** 和 **双目立体视觉（Stereo Vision）**。

#### （1）结构光（Structured Light）

**基本原理：** 结构光系统向目标表面投射一个预先设计好的光模式（通常是黑白条纹、格雷码图案或点阵），然后用另一个相机从不同角度观察这个光模式。由于物体表面凹凸不平，投射的光模式会发生畸变，通过分析畸变程度可以计算出深度信息。

```
结构光工作原理示意图：

        投影器                    相机
          │                        │
          │  投射条纹图案           │
          ▼                        ▼
    ┌──────────────────────────────────┐
    │  ═══════════════════════════════  │  ← 平面：条纹平直
    │  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │
    │  ═══════════════════════════════  │  ← 凸起：条纹错位
    │  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │  ← 凹陷：条纹错位（反向）
    └──────────────────────────────────┘
                    ▲
                    │
              分析条纹错位量
              → 计算深度 Z
```

**代表产品：** Microsoft Kinect v1（2010年）是结构光深度相机的典型代表。

**优缺点：**
- ✅ 精度较高（可达毫米级）
- ✅ 室内表现优秀
- ❌ 室外强光下效果差
- ❌ 容易受到透明、反光表面的干扰

#### （2）飞行时间法（ToF - Time of Flight）

**基本原理：** ToF 相机发射一束调制光（如近红外脉冲或连续波），测量光从发射到返回的时间或相位差，从而计算出距离。

$$Z = \frac{c \cdot \Delta t}{2}$$

其中 $c \approx 3 \times 10^8$ m/s 是光速，$\Delta t$ 是飞行时间。

大多数ToF相机采用间接测量法（测量相位差）：

$$Z = \frac{c \cdot \Delta \phi}{4\pi f}$$

```python
import numpy as np

def tof_depth_calculation(phase_shift_rad, modulation_freq_hz):
    """
    根据相位偏移计算间接ToF深度
    
    参数:
        phase_shift_rad: 发射光与接收光的相位偏移（弧度）
        modulation_freq_hz: 调制频率（Hz）
    
    返回:
        depth_m: 深度（米）
    """
    speed_of_light = 3e8  # 光速 m/s
    
    # 调制波长（往返）
    wavelength_round_trip = speed_of_light / modulation_freq_hz
    
    # 单程距离 = (相位偏移 / 2π) × 调制波长，然后除以2（往返→单程）
    round_trip_distance = (phase_shift_rad / (2 * np.pi)) * wavelength_round_trip
    depth_m = round_trip_distance / 2.0
    
    return depth_m

# 示例：常见ToF相机调制频率
mod_freq = 20e6  # 20 MHz
print(f"调制频率: {mod_freq/1e6:.0f} MHz")
print(f"调制波长（往返）: {3e8/mod_freq:.2f} 米")

# 相位偏移 π（半个周期）
phase_pi = np.pi
depth_pi = tof_depth_calculation(phase_pi, mod_freq)
print(f"相位偏移 π → 深度: {depth_pi:.3f} 米")

# 实际ToF相机的最大无模糊距离
max_range = (3e8 / mod_freq) / 2  # 调制波长/2
print(f"\n最大无模糊距离: {max_range:.2f} 米")
print("(超过这个距离会发生相位缠绕)")
```

**代表产品：** Microsoft Kinect v2（ToF）、Azure Kinect DK（ToF）。

**优缺点：**
- ✅ 测距范围远（可达数十米）
- ✅ 功耗较低，体积较小
- ✅ 不依赖纹理，对白墙等弱纹理区域效果好
- ❌ 精度中等（通常厘米级）
- ❌ 多ToF相机之间会互相干扰

#### （3）双目立体视觉（Stereo Vision）

双目立体视觉模仿人类的双眼，利用左右两个相机从不同视角观察同一场景，通过**视差**（Disparity）计算深度：

$$Z = \frac{f \cdot B}{d}$$

**普通双目 vs. 主动双目：**
- 普通双目在**弱纹理区域**表现很差
- **主动双目**（Active Stereo）通过红外激光投影器主动投射纹理，增强匹配能力
- Intel RealSense D400系列采用主动双目方案

```python
import numpy as np

def stereo_disparity_to_depth(disparity_px, focal_length_mm, baseline_mm, sensor_pixel_size_mm):
    """
    将视差转换为深度
    
    参数:
        disparity_px: 视差（像素）
        focal_length_mm: 焦距（毫米）
        baseline_mm: 基线距离（毫米）
        sensor_pixel_size_mm: 单个像素的物理尺寸（毫米）
    
    返回:
        depth_m: 深度（米）
    """
    if disparity_px <= 0:
        return float('inf')
    
    # 将焦距从毫米转换为像素单位
    focal_length_px = focal_length_mm / sensor_pixel_size_mm
    
    # 深度（毫米）= (焦距像素 × 基线毫米) / 视差像素
    depth_mm = (focal_length_px * baseline_mm) / disparity_px
    
    return depth_mm / 1000.0  # 转换为米

# Intel RealSense D455 参数示例
focal_length = 1.88   # RGB相机焦距 mm
baseline = 95          # 左右红外相机基线 mm
pixel_size = 0.00112  # IMX219 传感器像素尺寸 mm（约1.12μm）

print("=" * 50)
print("Intel RealSense D455 双目深度计算")
print("=" * 50)

for disparity in [10, 50, 100, 200]:
    depth = stereo_disparity_to_depth(disparity, focal_length, baseline, pixel_size)
    print(f"视差 {disparity:3d} 像素 → 深度 {depth:.3f} 米")
```

## 2. 点云基础概念

### 2.1 什么是点云？

**点云**（Point Cloud）是三维空间中一组离散的点的集合，每个点通常包含：
- **位置坐标** $(X, Y, Z)$：点在三维空间中的位置
- **颜色信息** $(R, G, B)$：可选，每个点的颜色值
- **法线向量** $(N_x, N_y, N_z)$：可选，表示点所在表面的朝向

```python
import numpy as np

# 点云的两种常见表示方式

# 方式1：仅包含位置坐标的点云 (N, 3)
points_xyz = np.array([
    [0.1, 0.2, 1.5],   # 点1: x=0.1m, y=0.2m, z=1.5m
    [0.5, -0.3, 2.0],  # 点2
    [-0.2, 0.8, 1.8],  # 点3
    [1.0, 0.0, 3.0],   # 点4
], dtype=np.float32)

# 方式2：包含位置和颜色的点云 (N, 6)，每行 [x, y, z, r, g, b]
points_xyzrgb = np.array([
    [0.1, 0.2, 1.5, 255, 0, 0],    # 红色点
    [0.5, -0.3, 2.0, 0, 255, 0],   # 绿色点
    [-0.2, 0.8, 1.8, 0, 0, 255],  # 蓝色点
    [1.0, 0.0, 3.0, 255, 255, 0], # 黄色点
], dtype=np.float32)

print(f"仅位置点云形状: {points_xyz.shape}")
print(f"带颜色点云形状: {points_xyzrgb.shape}")
```

### 2.2 PCL（Point Cloud Library）简介

**PCL**（Point Cloud Library）是点云处理领域最广泛使用的开源库，类似于图像处理中的OpenCV。PCL提供了从点云读取、可视化、滤波、分割、配准、重建等全套功能。

**常见的点云文件格式：**

| 格式 | 扩展名 | 特点 | 适用场景 |
|------|--------|------|---------|
| PCD | .pcd | PCL原生格式，支持压缩 | PCL开发、实时处理 |
| PLY | .ply | 可存储颜色、法线等多种属性 | 3D扫描、存储 |
| OBJ | .obj | 通用3D格式 | 3D建模软件导入 |
| LAS/LAZ | .las/.laz | 激光雷达标准格式 | 测绘、无人机航测 |
| XYZ | .xyz | 纯文本，简易 | 快速查看、数据交换 |

```python
import numpy as np

def pcl_pointcloud_stats(points):
    """
    计算点云的基本统计信息（模拟PCL的功能）
    
    参数:
        points: 点云数据 (N×3)，每行 [x, y, z]
    
    返回:
        stats: 统计信息字典
    """
    print("=" * 50)
    print("PCL 点云统计信息")
    print("=" * 50)
    print(f"点云点数: {len(points)}")
    
    # 计算质心
    centroid = points.mean(axis=0)
    print(f"\n质心 (Centroid):")
    print(f"  X: {centroid[0]:.4f} m")
    print(f"  Y: {centroid[1]:.4f} m")
    print(f"  Z: {centroid[2]:.4f} m")
    
    # 计算边界框
    min_coords = points.min(axis=0)
    max_coords = points.max(axis=0)
    print(f"\n轴对齐边界框 (AABB):")
    print(f"  最小点: ({min_coords[0]:.4f}, {min_coords[1]:.4f}, {min_coords[2]:.4f})")
    print(f"  最大点: ({max_coords[0]:.4f}, {max_coords[1]:.4f}, {max_coords[2]:.4f})")
    
    # 边界框尺寸
    dimensions = max_coords - min_coords
    print(f"  尺寸: {dimensions[0]:.4f} × {dimensions[1]:.4f} × {dimensions[2]:.4f} m")
    
    # 协方差矩阵（用于计算主成分方向）
    cov_matrix = np.cov(points.T)
    eigenvalues, eigenvectors = np.linalg.eig(cov_matrix)
    print(f"\n协方差特征值（描述点云分布的形状）:")
    for i, ev in enumerate(sorted(eigenvalues, reverse=True)):
        print(f"  主成分{i+1}: {ev:.6f}")
    
    return {
        "centroid": centroid,
        "min": min_coords,
        "max": max_coords,
        "dimensions": dimensions,
    }

# 示例：生成模拟点云并统计
np.random.seed(42)
sample_cloud = np.random.randn(1000, 3).astype(np.float32) * 0.5
sample_cloud[:, 2] += 2.0  # 将Z坐标上移2米
pcl_pointcloud_stats(sample_cloud)
```

### 2.3 点云与深度图的关系

**深度图**（Depth Map）是深度相机的原始输出形式，是一个二维数组，每个元素存储对应像素的深度值。**点云**是将深度图经过三维投影转换后的结果。

```
深度相机输出 → 深度图（2D数组，每个元素是深度值）
                  ↓ 反投影（使用相机内参）
              点云（3D点集合）

举例：
┌────────────┐         ┌─────────────────────┐
│ 深度图     │         │ 3D点云              │
│ (像素)      │   →     │ (相机坐标系下的      │
│ z=1.0 z=2.0 │         │  (X,Y,Z) 点)        │
│ z=0.5 z=3.0 │         │                     │
└────────────┘         └─────────────────────┘
```

**为什么需要转换？**
- **深度图**：密集但信息单一（只有距离），适合做2D像素级别的处理
- **点云**：稀疏但信息完整（XYZ+可选颜色），可以直接用于3D几何计算

## 3. 深度图像到点云的转换

### 3.1 转换原理

深度图到点云的转换基于**小孔成像模型的逆过程**。已知像素坐标 $(u, v)$ 和对应的深度值 $Z$（米），可以反投影到相机坐标系下的3D点 $(X, Y, Z)$：

$$
\begin{bmatrix} X \\ Y \\ Z \end{bmatrix} = Z \cdot K^{-1} \begin{bmatrix} u \\ v \\ 1 \end{bmatrix}
$$

```python
import numpy as np

def depth_to_pointcloud(depth_image, fx, fy, cx, cy):
    """
    将深度图转换为点云
    
    参数:
        depth_image: 深度图 (H×W)，每个元素是深度值（米）
        fx, fy: 焦距（像素单位）
        cx, cy: 光心坐标（像素单位）
    
    返回:
        points: 点云数据 (N×3)，每行 [x, y, z]，单位米
    """
    H, W = depth_image.shape
    
    # 创建像素坐标网格
    u_coords, v_coords = np.meshgrid(np.arange(W), np.arange(H))
    
    # 将像素坐标展平
    u_flat = u_coords.ravel()
    v_flat = v_coords.ravel()
    z_flat = depth_image.ravel()
    
    # 过滤掉无效深度（深度为0或负数通常表示无效）
    valid_mask = z_flat > 0  # 可以根据实际情况设置阈值
    
    u_valid = u_flat[valid_mask]
    v_valid = v_flat[valid_mask]
    z_valid = z_flat[valid_mask]
    
    # 构建相机内参矩阵的逆
    K_inv = np.array([
        [1/fx, 0.0, -cx/fx],
        [0.0, 1/fy, -cy/fy],
        [0.0, 0.0, 1.0]
    ])
    
    # 归一化相机坐标
    normalized_coords = K_inv @ np.vstack([u_valid, v_valid, np.ones_like(u_valid)])
    
    # 乘以深度，得到3D坐标
    X = normalized_coords[0] * z_valid
    Y = normalized_coords[1] * z_valid
    Z = z_valid
    
    # 合并为 N×3 数组
    points = np.column_stack([X, Y, Z]).astype(np.float32)
    
    return points


def depth_to_pointcloud_with_rgb(depth_image, rgb_image, fx, fy, cx, cy):
    """
    将深度图和RGB图像一起转换为带颜色的点云
    
    参数:
        depth_image: 深度图 (H×W)，深度值（米）
        rgb_image: RGB图像 (H×W×3)，uint8，0-255
        fx, fy, cx, cy: 相机内参
    
    返回:
        points: 点云数据 (N×6)，每行 [x, y, z, r, g, b]
    """
    H, W = depth_image.shape
    
    # 创建像素坐标网格
    u_coords, v_coords = np.meshgrid(np.arange(W), np.arange(H))
    
    # 展平
    u_flat = u_coords.ravel()
    v_flat = v_coords.ravel()
    z_flat = depth_image.ravel()
    
    # 过滤无效深度
    valid_mask = z_flat > 0
    u_valid = u_flat[valid_mask]
    v_valid = v_flat[valid_mask]
    z_valid = z_flat[valid_mask]
    
    # 获取对应的RGB颜色
    rgb_flat = rgb_image.reshape(-1, 3)  # (H*W, 3)
    rgb_valid = rgb_flat[valid_mask]
    
    # 计算3D坐标
    K_inv = np.array([
        [1/fx, 0.0, -cx/fx],
        [0.0, 1/fy, -cy/fy],
        [0.0, 0.0, 1.0]
    ])
    
    normalized_coords = K_inv @ np.vstack([u_valid, v_valid, np.ones_like(u_valid)])
    X = normalized_coords[0] * z_valid
    Y = normalized_coords[1] * z_valid
    Z = z_valid
    
    # 合并位置和颜色
    points_xyz = np.column_stack([X, Y, Z]).astype(np.float32)
    points_xyzrgb = np.hstack([points_xyz, rgb_valid.astype(np.float32)])
    
    return points_xyzrgb

    print("=" * 50)

## 4. 点云滤波处理

点云滤波是点云处理的第一步，目的是去除噪声、减少数据量、提取感兴趣区域。

### 4.1 直通滤波（PassThrough Filter）

直通滤波只保留在某个坐标轴范围内（X、Y或Z）的点，类似于2D图像中的**感兴趣区域（ROI）裁剪**。

```python
import numpy as np

def passthrough_filter(points, axis, min_val, max_val):
    """
    直通滤波：只保留指定坐标轴范围内的点
    
    参数:
        points: 点云数据 (N×3)，每行 [x, y, z]
        axis: 滤波轴 ('x', 'y', 'z')
        min_val: 最小值（米）
        max_val: 最大值（米）
    
    返回:
        filtered_points: 滤波后的点云
        mask: 保留点的布尔掩膜
    """
    axis_idx = {'x': 0, 'y': 1, 'z': 2}[axis]
    
    coord = points[:, axis_idx]
    mask = (coord >= min_val) & (coord <= max_val)
    
    filtered_points = points[mask]
    
    return filtered_points, mask

### 4.2 体素滤波（VoxelGrid Filter）

体素滤波将点云划分为三维网格（体素），每个体素只保留一个点（通常是该体素内所有点的质心）。这是点云下采样的标准方法，可以大幅减少点数同时保持点云的形状特征。

```python
import numpy as np

def voxelgrid_filter(points, voxel_size):
    """
    体素滤波：对点云进行下采样
    
    参数:
        points: 点云数据 (N×3)
        voxel_size: 体素大小（米）
    
    返回:
        downsampled_points: 下采样后的点云
    """
    # 将点坐标按体素大小网格化
    voxel_indices = np.floor(points / voxel_size).astype(int)
    
    # 使用 np.unique 获取每个体素的唯一索引
    unique_voxels, inverse = np.unique(voxel_indices, axis=0, return_inverse=True)
    
    # 计算每个体素的质心
    downsampled_points = np.zeros((len(unique_voxels), 3), dtype=np.float32)
    for i in range(len(unique_voxels)):
        mask = np.where((voxel_indices == unique_voxels[i]).all(axis=1))[0]
        downsampled_points[i] = points[mask].mean(axis=0)
    
    return downsampled_points

### 4.3 统计滤波（Statistical Outlier Removal）

统计滤波去除那些在邻域中点密度明显低于平均水平的离群点（噪声点）。每个点的邻域大小由参数 $k$ 指定，计算每个点到其 $k$ 个最近邻的距离分布，去除距离均值过远的点。

```python
import numpy as np
from scipy.spatial import distance_matrix

def statistical_outlier_removal(points, k=50, std_multiplier=1.0):
    """
    统计离群点移除：去除邻域点密度过低的离群点
    
    参数:
        points: 点云数据 (N×3)
        k: 每个点考虑的最近邻数量
        std_multiplier: 标准差倍数阈值（距离均值超过 mean + std_multiplier * std 的点被移除）
    
    返回:
        filtered_points: 滤波后的点云
        mask: 保留点的布尔掩膜
    """
    # 为了效率，对大点云进行采样计算
    if len(points) > 5000:
        # 使用近似方法：随机采样
        sample_size = 3000
        sample_indices = np.random.choice(len(points), sample_size, replace=False)
        sample_points = points[sample_indices]
        
        # 计算        # 计算采样点到所有点的距离
        dist_mat = distance_matrix(sample_points, points)
        # 排序获取k个最近邻（排除自身）
        sorted_dists = np.sort(dist_mat, axis=1)[:, 1:k+1]
        mean_dists = sorted_dists.mean(axis=1)
        
        # 计算全局均值和标准差
        global_mean = mean_dists.mean()
        global_std = mean_dists.std()
        
        # 判断每个采样点是否为离群点
        threshold = global_mean + std_multiplier * global_std
        inlier_mask_sample = mean_dists < threshold
        
        # 对所有点进行判断（近似）
        all_dists = np.linalg.norm(points[:, np.newaxis, :] - sample_points[np.newaxis, :, :], axis=2)
        nearest_dists = np.sort(all_dists, axis=1)[:, 1:k+1].mean(axis=1)
        mask = nearest_dists < threshold
    else:
        # 小点云直接计算
        dist_mat = distance_matrix(points, points)
        sorted_dists = np.sort(dist_mat, axis=1)[:, 1:k+1]
        mean_dists = sorted_dists.mean(axis=1)
        
        global_mean = mean_dists.mean()
        global_std = mean_dists.std()
        threshold = global_mean + std_multiplier * global_std
        
        mask = mean_dists < threshold
    
    filtered_points = points[mask]
    
    return filtered_points, mask

### 4.4 半径滤波（Radius Outlier Removal）

半径滤波去除在给定半径邻域内点数过少的点。与统计滤波不同，半径滤波基于固定半径内的邻居数量来判断是否为离群点。

```python
import numpy as np

def radius_outlier_removal(points, radius, min_neighbors):
    """
    半径离群点移除：去除给定半径邻域内点数过少的点
    
    参数:
        points: 点云数据 (N×3)
        radius: 搜索半径（米）
        min_neighbors: 最小邻居数量阈值（小于此数量的点被移除）
    
    返回:
        filtered_points: 滤波后的点云
        mask: 保留点的布尔掩膜
    """
    # 使用KD-Tree加速最近邻搜索
    from scipy.spatial import cKDTree
    
    tree = cKDTree(points)
    
    # 对每个点查询半径内的邻居数量
    # count_neighbors 统计在给定半径内的点数
    neighbor_counts = np.array([tree.count_neighbors(tree, i, radius) for i in range(len(points))])
    
    # 保留邻居数 >= min_neighbors 的点
    # 注意：count_neighbors 包含自身，所以要 > min_neighbors
    mask = neighbor_counts > min_neighbors
    
    filtered_points = points[mask]
    
    return filtered_points, mask

### 4.5 点云滤波总结

| 滤波方法 | 作用 | 适用场景 | 核心参数 |
|---------|------|---------|---------|
| 直通滤波 | 裁剪感兴趣区域 | 提取某个高度范围、前方区域等 | axis, min, max |
| 体素滤波 | 下采样 | 减少计算量、加速处理 | voxel_size |
| 统计滤波 | 去除离群点 | 去除测量噪声、孤立噪声点 | k, std_multiplier |
| 半径滤波 | 去除稀疏离群点 | 去除边缘稀疏点 | radius, min_neighbors |

**实际使用中，通常组合使用多种滤波：**

    2. 统计滤波：去除测量噪声

    print(f"体素滤波后: {len(points)} 点（压缩比 ~{len(points)/len(points):.1f}x）")
    
    return points

## 5. 点云分割

点云分割是将点云划分为多个有意义的区域的过程，每个区域代表一个独立的物体或表面。在机器人应用中，最常见的分割任务是：**找到地面/平面**和**分割出独立的物体**。

### 5.1 平面分割（RANSAC）

**RANSAC**（Random Sample Consensus）是一种经典的参数估计方法，广泛用于点云平面检测。其核心思想是：
1. 随机选取最少的点来拟合一个模型（平面方程）
2. 计算其他点与该模型的拟合误差
3. 超过阈值的点视为外点，在阈值内的视为内点
4. 重复以上步骤，找到内点数量最多的模型

**平面方程：** $ax + by + cz + d = 0$，归一化条件下 $\sqrt{a^2+b^2+c^2}=1$

```python
import numpy as np

def fit_plane_ransac(points, distance_threshold=0.02, max_iterations=1000, min_inliers=100):
    """
    使用 RANSAC 算法拟合平面
    
    参数:
        points: 点云数据 (N×3)
        distance_threshold: 到平面距离阈值（米），超过视为外点
        max_iterations: 最大迭代次数
        min_inliers: 最小内点数量阈值
    
    返回:
        plane_model: [a, b, c, d] 平面方程系数
        inlier_mask: 内点布尔掩膜
    """
    best_model = None
    best_inliers = 0
    best_mask = None
    
    n_points = len(points)
    
    for iteration in range(max_iterations):
        # 随机选取3个点拟合平面
        indices = np.random.choice(n_points, 3, replace=False)
        pts = points[indices]
        
        # 计算平面法向量（利用叉积）
        v1 = pts[1] - pts[0]
        v2 = pts[2] - pts[0]
        normal = np.cross(v1, v2)
        
        norm = np.linalg.norm(normal)
        if norm < 1e-10:  # 三点共线，跳过
            continue
        
        normal = normal / norm  # 归一化
        
        # 平面方程: normal[0]*x + normal[1]*y + normal[2]*z + d = 0
        # d = -normal · p0
        d = -np.dot(normal, pts[0])
        
        # 计算所有点到平面的距离
        distances = np.abs(np.dot(points, normal) + d)
        
        # 统计内点（距离小于阈值）
        inlier_mask = distances < distance_threshold
        n_inliers = np.sum(inlier_mask)
        
        if n_inliers > best_inliers:
            best_inliers = n_inliers
            best_model = np.array([normal[0], normal[1], normal[2], d])
            best_mask = inlier_mask
    
    if best_inliers < min_inliers:
        print(f"警告: 仅找到 {best_inliers} 个内点，未达到最小阈值 {min_inliers}")
    
    return best_model, best_mask

### 5.2 欧式聚类（Euclidean Clustering）

在去除地面等大平面后，我们需要将剩余点分割成独立的物体。**欧式聚类**（Euclidean Clustering）是基于点与点之间的欧氏距离进行分割的标准方法。

基本思想：从一个未标记的点开始，使用BFS/DFS将与它距离小于阈值的邻居归为同一簇，直到没有新的点可以加入。

```python
import numpy as np
from collections import deque

def euclidean_clustering(points, cluster_tolerance=0.05, min_cluster_size=10, max_cluster_size=10000):
    """
    欧式聚类分割（使用 KD-Tree 加速）
    
    参数:
        points: 点云数据 (N×3)
        cluster_tolerance: 聚类容差（米），同簇点之间的最大距离
        min_cluster_size: 最小簇大小
        max_cluster_size: 最大簇大小
    
    返回:
        clusters: 列表，每个元素是一个簇的点索引数组
    """
    from scipy.spatial import cKDTree
    
    tree = cKDTree(points)
    n_points = len(points)
    
    # 标记每个点是否被访问
    visited = np.zeros(n_points, dtype=bool)
    cluster_labels = np.full(n_points, -1, dtype=int)  # -1 表示未分类
    cluster_id = 0
    
    for i in range(n_points):
        if visited[i]:
            continue
        
        # 开始一个新的簇，使用 BFS
        queue = deque([i])
        visited[i] = True
        cluster_points = []
        
        while queue:
            point_idx = queue.popleft()
            cluster_points.append(point_idx)
            
            # 查找距离小于 cluster_tolerance 的所有邻居
            neighbors = tree.query_ball_point(points[point_idx], cluster_tolerance)
            
            for neighbor_idx in neighbors:
                if not visited[neighbor_idx]:
                    visited[neighbor_idx] = True
                    queue.append(neighbor_idx)
        
        # 检查簇大小是否在有效范围内
        if min_cluster_size <= len(cluster_points) <= max_cluster_size:
            for idx in cluster_points:
                cluster_labels[idx] = cluster_id
            cluster_id += 1
        else:
            # 太小的簇标记为噪声
            for idx in cluster_points:
                cluster_labels[idx] = -1
    
    # 提取每个簇的点
    clusters = []
    for cid in range(cluster_id):
        cluster_mask = cluster_labels == cid
        clusters.append(np.where(cluster_mask)[0])
    
    return clusters, cluster_labels

## 6. 深度学习点云概述

传统的点云处理方法（滤波、分割）主要依靠手工设计的算法和几何特征。近年来，**深度学习**在点云分析领域取得了突破性进展。本节简要介绍点云深度学习的基本思想和两个经典网络。

### 6.1 为什么需要深度学习处理点云？

传统的几何方法（RANSAC、欧式聚类）在简单场景下表现良好，但在复杂场景下会遇到瓶颈：

| 挑战 | 传统方法的局限 |
|------|--------------|
| 遮挡与部分观测 | 手工特征难以处理严重遮挡 |
| 类别多样 | 每个新类别需要设计新的几何特征 |
| 语义理解 | 几何方法无法理解"这是什么物体" |
| 噪声与异常值 | 手工特征对噪声敏感 |

深度学习可以从数据中自动学习特征表示，解决上述问题。

### 6.2 PointNet：开创性的点云深度学习

**PointNet**（2017）是第一个直接处理原始点云（无序点集）的深度学习网络，解决了点云处理的两个核心挑战：

1. **点云的无序性**：N个点有N!种排列，但输出应该对排列不变
2. **点云的几何不变性**：对旋转、平移应该保持输出不变

**核心设计：**

```
PointNet 架构（分类网络）：

输入: N × 3 的原始点云（无序）
    ↓
MLP (共享权重的多层感知机) → N × 64
    ↓
T-Net (空间变换网络，预测3×3旋转矩阵) → 对点云进行对齐
    ↓
MLP → N × 64
    ↓
T-Net (特征变换网络，预测64×64矩阵) → 对特征对齐
    ↓
MLP → N × 1024
    ↓
Max Pooling (对称函数，解决无序性) → 1 × 1024 (全局特征)
    ↓
MLP → 1 × 512 → 1 × 256 → 1 × K (K个类别)

关键创新：
- Max Pooling 是对称函数，保证对输入排列不变
- T-Net 学习对齐，使网络对几何变换更鲁棒
- 全局特征通过 Max Pooling 聚合，适用于分类
```

```python
"""
PointNet 核心概念的伪代码实现

注意：这是概念演示，实际的 PointNet 实现需要 PyTorch/TensorFlow
"""

def pointnet_concept_demo():
    """
    PointNet 核心概念演示
    """
    import numpy as np
    
    print("=" * 50)
    print("PointNet 核心概念")
    print("=" * 50)
    
    # 模拟输入点云 (N=1024, 3D坐标)
    N = 1024
    point_cloud = np.random.randn(N, 3).astype(np.float32)
    
    print(f"\n1. 输入点云: {point_cloud.shape}")
    print(f"   形状: (N, 3)，N={N} 个无序点")
    
    # ========== 共享 MLP (1D卷积，等价于作用在每个点上) ==========
    # 这相当于对每个点应用相同的特征提取函数
    # 输入: (N, 3) → 输出: (N, 64)
    mlp_layer1 = lambda x: np.maximum(0, np.dot(x, np.random.randn(3, 64) * 0.01))
    point_features = mlp_layer1(point_cloud)  # (N, 64)
    
    print(f"\n2. 共享MLP（特征提取）:")
    print(f"   输入: (N, 3) → 输出: (N, 64)")
    print(f"   每个点独立提取特征，但共享权重（排列不变性的第一步）")
    
    # ========== Max Pooling（对称函数，解决无序性） ==========
    # 关键洞察：Max Pooling 是对称函数
    # f(x1, x2, ..., xN) = max(f(x1), f(x2), ..., f(xN))
    # 对任意排列，Max Pooling 输出相同
    global_feature = np.max(point_features, axis=0)  # (64,)
    
    print(f"\n3. Max Pooling（解决无序性）:")
    print(f"   输入: (N, 64) → 输出: (64,)")
    print(f"   取每个特征的最大的点，聚合为全局特征向量")
    print(f"   Max Pooling 是对称函数 → 对输入排列不变")
    
    # ========== 分类头 ==========
    # 从全局特征向量预测类别
    fc1 = lambda x: np.maximum(0, np.dot(x, np.random.randn(64, 32) * 0.01))
    fc2 = lambda x: np.maximum(0, np.dot(x, np.random.randn(32, 16) * 0.01))
    fc3 = lambda x: np.dot(x, np.random.randn(16, 10) * 0.01))  # 假设10类
    
    features = fc1(global_feature)
    features = fc2(features)
    scores = fc3(features)
    
    print(f"\n4. 分类头（从全局特征预测）:")
    print(f"   全局特征 (64,) → 全连接层 → 类别分数 (10,)")
    print(f"   最终输出: {np.argmax(scores)} 类（模拟）")
    
    # ========== 语义分割（需要结合全局和局部特征） ==========
    print(f"\n5. 语义分割（PointNet的另一个任务）:")
    print(f"   分类只能告诉我们'点云里有什么物体'")
    print(f"   分割需要知道'每个点属于哪个物体/类别'")
    print(f"   方法：全局特征广播 + 局部特征 → 每个点的分数")
    print(f"   全局特征复制 N 份，与局部特征拼接: (N, 64+64) → (N, K)")
    
    return {
        "point_cloud": point_cloud,
        "local_features": point_features,
        "global_feature": global_feature,
        "num_classes": 10,
    }

### 6.3 PointNet++：层次化点云学习

**PointNet++**（2017）在 PointNet 基础上引入了**层次化特征学习**（类似 CNN 的卷积层级结构），能够更好地捕捉局部几何结构。

```
PointNet++ 核心思想：

PointNet:    [所有点] → Max Pool → [全局特征]
                单一尺度，无局部聚合

PointNet++:  [小邻域] → PointNet → [局部特征]
                   ↓
              [较大邻域] → PointNet → [更高层特征]
                   ↓
              [更大邻域] → PointNet → [全局特征]
                   ↑
            层次化，逐步扩大感受野
```

**PointNet++ 的关键改进：**

1. **层次化采样+分组**（Hierarchical Point Set Feature Learning）
   - 使用 FPS（Farthest Point Sampling）选择具有代表性的中心点
   - 每个中心点用 Ball Query 找邻域内的点
   - 对每个邻域应用 PointNet 提取局部特征

2. **多尺度分组（MSG）**和**多分辨率分组（MRG）**
   - 解决点云密度不均匀的问题
   - 同一中心点用不同半径的邻域，分辨率互补

```python
def pointnet_plus_plus_concept():

  - 只提取全局特征，丢失了局部几何信息
  - 对于复杂场景（如室内多个房间），效果有限

PointNet++ 的改进：
  - 层次化：先看局部 → 再看整体
  - 类似于 CNN 的层级结构：边缘 → 纹理 → 物体 → 场景

密度自适应：PointNet++ 可以处理点云密度不均匀的问题
  - 室内点云：近处密、远处稀疏
  - PointNet++ 使用多尺度分组，对密度变化更鲁棒
""")
    
    return None

### 7.2 ROS2点云处理节点示例

```python
# ROS2 点云处理节点示例
# 文件：pointcloud_processing_node.py

"""
ROS2 点云处理节点

功能：
1. 订阅深度相机的点云话题
2. 进行滤波处理（直通 + 体素）
3. 进行分割处理（地面去除 + 物体聚类）
4. 发布处理结果

依赖：
- rclpy
- sensor_msgs.msg.PointCloud2
- std_msgs.msg Header
- numpy (用于点云处理)
"""


# 注意：以下代码是完整的 Python 实现，需要在 ROS2 环境中运行
# 由于 PCL 的 Python 支持不完整，实际使用时推荐使用 C++ 或 pclpy

"""
# ============================================================
# ROS2 点云处理节点（C++ 版本参考）
# ============================================================

# 在 ROS2 中使用 C++ 处理点云的典型代码结构：

#include <rclcpp/rclcpp.hpp>
#include <sensor_msgs/msg/point_cloud2.hpp>
#include <pcl/point_cloud.h>
#include <pcl/point_types.h>
#include <pcl_conversions/pcl_conversions.h>
#include <pcl/filters/passthrough.h>
#include <pcl/filters/voxel_grid.h>
#include <pcl/segmentation/sac_segmentation.h>
#include <pcl/segmentation/extract_clusters.h>

class PointCloudProcessor : public rclcpp::Node {
public:
    PointCloudProcessor()
        : Node("pointcloud_processor") {
        
        // 订阅原始点云
        subscription_ = this->create_subscription<sensor_msgs::msg::PointCloud2>(
            "/camera/depth/color/points",  // RealSense 点云话题
            10,
            std::bind(&PointCloudProcessor::cloudCallback, this, std::placeholders::_1)
        );
        
        // 发布滤波后的点云
        publisher_filtered_ = this->create_publisher<sensor_msgs::msg::PointCloud2>(
            "/pointcloud/filtered",
            10
        );
        
        // 发布地面点云
        publisher_floor_ = this->create_publisher<sensor_msgs::msg::PointCloud2>(
            "/pointcloud/floor",
            10
        );
        
        // 发布物体点云（去除地面后）
        publisher_objects_ = this->create_publisher<sensor_msgs::msg::PointCloud2>(
            "/pointcloud/objects",
            10
        );
        
        RCLCPP_INFO(this->get_logger(), "点云处理节点已启动");
    }

private:
    void cloudCallback(const sensor_msgs::msg::PointCloud2::SharedPtr msg) {
        // 1. 将 ROS2 消息转换为 PCL 点云
        pcl::PointCloud<pcl::PointXYZRGB>::Ptr cloud(new pcl::PointCloud<pcl::PointXYZRGB>);
        pcl::fromROSMsg(*msg, *cloud);
        
        // 2. 直通滤波：提取桌面高度范围 (z: 0.6~1.2m)
        pcl::PointCloud<pcl::PointXYZRGB>::Ptr cloud_filtered(new pcl::PointCloud<pcl::PointXYZRGB>);
        pcl::PassThrough<pcl::PointXYZRGB> pass;
        pass.setInputCloud(cloud);
        pass.setFilterFieldName("z");
        pass.setFilterLimits(0.6, 1.2);
        pass.filter(*cloud_filtered);
        
        // 3. 体素滤波：降采样
        pcl::VoxelGrid<pcl::PointXYZRGB> voxel;
        voxel.setInputCloud(cloud_filtered);
        voxel.setLeafSize(0.01f, 0.01f, 0.01f);  // 1cm 体素
        voxel.filter(*cloud_filtered);
        
        // 4. 地面分割 (RANSAC)
        pcl::ModelCoefficients::Ptr coefficients(new pcl::ModelCoefficients);
        pcl::PointIndices::Ptr inliers(new pcl::PointIndices);
        pcl::SACSegmentation<pcl::PointXYZRGB> seg;
        seg.setInputCloud(cloud_filtered);
        seg.setMethodType(pcl::SAC_RANSAC);
        seg.setModelType(pcl::SACMODEL_PLANE);
        seg.setDistanceThreshold(0.02);  // 2cm 阈值
        seg.segment(*inliers, *coefficients);
        
        // 提取地面点
        pcl::ExtractIndices<pcl::PointXYZRGB> extract_floor;
        extract_floor.setInputCloud(cloud_filtered);
        extract_floor.setIndices(inliers);
        pcl::PointCloud<pcl::PointXYZRGB>::Ptr floor_cloud(new pcl::PointCloud<pcl::PointXYZRGB>);
        extract_floor.filter(*floor_cloud);
        
        // 提取非地面点（物体）
        pcl::PointCloud<pcl::PointXYZRGB>::Ptr object_cloud(new pcl::PointCloud<pcl::PointXYZRGB>);
        extract_floor.setNegative(true);
        extract_floor.filter(*object_cloud);
        
        // 5. 欧式聚类（分割独立物体）
        if (object_cloud->size() > 0) {
            pcl::search::KdTree<pcl::PointXYZRGB>::Ptr tree(new pcl::search::KdTree<pcl::PointXYZRGB>);
            tree->setInputCloud(object_cloud);
            
            std::vector<pcl::PointIndices> cluster_indices;
            pcl::EuclideanClusterExtraction<pcl::PointXYZRGB> ec;
            ec.setInputCloud(object_cloud);
            ec.setClusterTolerance(0.03);  // 3cm
            ec.setMinClusterSize(20);
            ec.setMaxClusterSize(2500);
            ec.setSearchMethod(tree);
            ec.extract(cluster_indices);
            
            RCLCPP_INFO(this->get_logger(), 
                "检测到 %lu 个物体", cluster_indices.size());
        }
        
        // 6. 发布结果
        sensor_msgs::msg::PointCloud2 output;
        pcl::toROSMsg(*cloud_filtered, output);
        output.header = msg->header;
        publisher_filtered_->publish(output);
    }
    
    rclcpp::Subscription<sensor_msgs::msg::PointCloud2>::SharedPtr subscription_;
    rclcpp::Publisher<sensor_msgs::msg::PointCloud2>::SharedPtr publisher_filtered_;
    rclcpp::Publisher<sensor_msgs::msg::PointCloud2>::SharedPtr publisher_floor_;
    rclcpp::Publisher<sensor_msgs::msg::PointCloud2>::SharedPtr publisher_objects_;
};


int main(int argc, char** argv) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<PointCloudProcessor>());
    rclcpp::shutdown();
    return 0;
}
"""

## 本章小结

本章我们系统性地学习了点云与深度相机的相关知识。

在深度相机原理部分，我们了解了三种主流深度测量技术：**结构光**（通过分析投影图案畸变计算深度，适合室内近距离）、**ToF飞行时间法**（测量光往返时间，相位法更常见，适合中远距离）、**双目立体视觉**（模仿人眼通过视差计算深度，主动双目增强了弱纹理区域的匹配）。每种技术都有其适用场景，需要根据机器人的具体任务选择合适的深度相机。

在点云基础概念部分，我们学习了点云是三维空间中离散点的集合，每个点包含 $(X, Y, Z)$ 坐标和可选的颜色、法线等信息。PCL（Point Cloud Library）是点云处理的标配开源库，类似于OpenCV在图像处理中的地位。

在深度图到点云转换部分，我们掌握了基于小孔成像模型逆过程的反投影公式：给定像素坐标 $(u, v)$ 和深度值 $Z$，通过相机内参矩阵的逆计算3D坐标。这是所有深度相机数据处理的基础。

在点云滤波部分，我们学习了四种基本滤波方法：**直通滤波**（提取感兴趣区域，如特定高度范围）、**体素滤波**（下采样减少计算量）、**统计滤波**（去除孤立的测量噪声点）、**半径滤波**（去除稀疏边缘点）。实际应用中通常组合使用多种滤波。

在点云分割部分，我们学习了两种核心分割方法：**RANSAC平面分割**（通过随机采样和一致性检验检测平面，如地面、桌面）和**欧式聚类**（基于距离的BFS聚类，将点云分割为独立物体）。这是机器人抓取、分拣等任务的关键步骤。

在深度学习点云部分，我们简要介绍了 PointNet（开创性地直接处理无序点云，通过Max Pooling解决排列不变性）和 PointNet++（层次化特征学习，更好地捕捉局部几何结构）。

在ROS2+PCL实战部分，我们了解了ROS2中点云的标准消息类型 `sensor_msgs/PointCloud2`，以及如何在ROS2系统中订阅、处理、发布点云数据。使用 RealSense 等深度相机时，需要注意坐标系变换（从相机光心到机器人基座）和话题配置。

## 答案

### 1. 选择题答案

**题目1.1：** **C. 主动双目（Active Stereo）**

解析：普通双目立体视觉在弱纹理区域（纯白色墙面）表现很差，因为没有足够的纹理特征供匹配算法找到左右图像中对应的点。结构光同样依赖投射图案的可见性，弱纹理区域效果也不好。ToF 不依赖纹理，但会受到深色表面的影响（吸收红外光）。主动双目通过红外激光投影器主动投射散斑纹理，可以有效增强弱纹理区域的匹配效果，是最适合的方案。

**题目1.2：** **A. 0.0**

解析：使用反投影公式：
- 归一化坐标：$[u, v, 1]^T = [320, 240, 1]^T$
- $K^{-1} = [[1/600, 0, -320/600], [0, 1/600, -240/600], [0, 0, 1]]$
- 归一化坐标：$[0, 0, 1]^T$
- $X = Z \times 0 = 0$ 米

图像中心点 $(u=320, v=240)$ 恰好在光心轴线上，所以 $X=0$。

**题目1.3：** **D. 体素滤波（VoxelGrid）**

解析：体素滤波将点云划分为三维网格，每个体素只保留一个点（通常是质心），可以在大幅减少点云数据量的同时保持点云的空间分布形状特征。其他三种滤波主要是去除噪声或裁剪区域，不是专门用于降采样的方法。

**题目1.4：** **C. 3个**

解析：平面由3个不共线的点唯一确定。RANSAC 每次随机选取最少的点（3个）来拟合模型，然后统计内点数量。选取少于3个点无法确定一个平面。

### 2. 填空题答案

**题目2.1：** **时间**（time）或**往返时间**

解析：ToF（飞行时间法）直接测量光从发射到返回探测器的时间 $\Delta t$，然后用公式 $Z = c \cdot \Delta t / 2$ 计算距离。实际中大多数ToF相机使用间接测量（相位偏移），但本质测量的还是与时间相关的物理量。

**题目2.2：** **sensor_msgs/PointCloud2**

解析：这是 ROS/ROS2 中点云的标准消息类型，定义了点云的数据结构（字段、步长、二进制数据等）。

**题目2.3：** **Max Pooling**（或最大池化）

解析：PointNet 的核心创新之一。使用 Max Pooling 作为对称函数，无论输入点的排列顺序如何，输出结果都相同，解决了点云无序性的挑战。

**题目2.4：** **直通**（PassThrough）

解析：直通滤波通过设定 X、Y 或 Z 轴的范围来裁剪点云，是最简单直接的滤波方法，类似于2D图像中的感兴趣区域（ROI）裁剪。

### 3. 简答题答案

**题目3.1：**

| 技术 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 结构光 | 精度高（亚毫米级）；室内近距离表现优秀；成本相对较低 | 室外强光下失效；易受透明/反光表面干扰；工作距离受限 | 室内近距离扫描（0.5~3m），如Kinect v1；工业3D扫描 |
| ToF | 测距范围广（可达数十米）；不依赖纹理；功耗低体积小；响应速度快 | 精度中等（厘米级）；多相机互相干扰；深色表面效果差 | 中远距离感知；移动机器人导航；手机/AR设备；室内外通用 |

**题目3.2：**

欧式聚类分割独立物体的**核心假设**：属于同一个物体的点之间的距离应该小于某个阈值（聚类容差），而不同物体之间的点距离应该大于该阈值。

**为什么会失效？**
1. **物体之间接触或靠近**：如果两个物体紧贴在一起（距离小于容差），它们会被误分为同一个簇。
2. **物体表面有遮挡**：严重遮挡会导致部分点缺失，同一物体可能分裂成多个簇。
3. **点云密度不均匀**：近处点密集、远处点稀疏，同一物体在不同距离需要不同的容差值。
4. **物体形状复杂**：细长物体（椅子腿）可能被误判为噪声或分裂。

**题目3.3：**

从深度相机到识别独立物体的完整流程：

```
1. 深度图像采集
   深度相机 → 原始深度图 + RGB图像
   
2. 深度图转点云
   深度图 + 相机内参 → 相机坐标系下的3D点云
   
3. 坐标系变换（TF）
   camera_depth_optical_frame → base_link（机器人基座）
   （如果需要在机器人坐标系下处理）
   
4. 点云滤波（前处理）
   - 直通滤波：裁剪到工作空间范围（如 z: 0.5~1.5m）
   - 体素滤波：降采样，减少计算量
   
5. 平面分割（桌面检测）
   - RANSAC 平面拟合 → 检测桌面平面方程
   - 提取桌面内点（属于桌面的点）
   - 反向提取非桌面点（桌面以上的物体点）
   
6. 物体聚类分割
   - 欧式聚类（BFS + 距离阈值）
   - 每个连通区域 = 一个独立物体
   
7. 输出
   - 每个物体的点云集 + 包围盒 + 中心位置
   → 送给后续的物体识别/抓取规划模块
```

### 4. 编程题答案

```python
import numpy as np
from collections import deque

def fit_plane_ransac(points, distance_threshold=0.02, max_iterations=500, min_inliers=50):
    """
    使用 RANSAC 算法拟合平面
    """
    best_model = None
    best_inliers = 0
    best_mask = None
    
    n_points = len(points)
    
    for _ in range(max_iterations):
        # 随机选取3个点
        indices = np.random.choice(n_points, 3, replace=False)
        pts = points[indices]
        
        # 计算平面法向量
        v1 = pts[1] - pts[0]
        v2 = pts[2] - pts[0]
        normal = np.cross(v1, v2)
        
        norm = np.linalg.norm(normal)
        if norm < 1e-10:
            continue
        
        normal = normal / norm
        d = -np.dot(normal, pts[0])
        
        # 计算所有点到平面的距离
        distances = np.abs(np.dot(points, normal) + d)
        
        # 统计内点
        inlier_mask = distances < distance_threshold
        n_inliers = np.sum(inlier_mask)
        
        if n_inliers > best_inliers:
            best_inliers = n_inliers
            best_model = np.array([normal[0], normal[1], normal[2], d])
            best_mask = inlier_mask
    
    return best_model, best_mask


def euclidean_clustering_fast(points, distance_threshold=0.03, min_cluster_size=10):
    """
    快速欧式聚类（使用 KD-Tree）
    """
    from scipy.spatial import cKDTree
    
    tree = cKDTree(points)
    n_points = len(points)
    visited = np.zeros(n_points, dtype=bool)
    cluster_labels = np.full(n_points, -1, dtype=int)
    cluster_id = 0
    
    for i in range(n_points):
        if visited[i]:
            continue
        
        # BFS 聚类
        queue = deque([i])
        visited[i] = True
        cluster_points = []
        
        while queue:
            idx = queue.popleft()
            cluster_points.append(idx)
            
            # 找半径内的邻居
            neighbors = tree.query_ball_point(points[idx], distance_threshold)
            for nb in neighbors:
                if not visited[nb]:
                    visited[nb] = True
                    queue.append(nb)
        
        # 检查簇大小
        if len(cluster_points) >= min_cluster_size:
            for idx in cluster_points:
                cluster_labels[idx] = cluster_id
            cluster_id += 1
    
    # 提取每个簇
    clusters = []
    for cid in range(cluster_id):
        clusters.append(points[cluster_labels == cid])
    
    return clusters


def segment_table_objects(points, verbose=True):
    """
    分割桌面上的独立物体
    
    参数:
        points: 点云数据 (N×3)，假设 Z 坐标约为桌面高度
        verbose: 是否打印详细信息
    
    返回:
        objects: 列表，每个元素是一个物体的点云 (Mi×3)
    """
    if verbose:
        print("=" * 50)
        print("桌面物体分割")
        print("=" * 50)
        print(f"输入点云: {len(points)} 点")
    
    # Step 1: RANSAC 检测桌面平面
    if verbose:
        print("\n[Step 1] RANSAC 平面分割...")
    plane_model, plane_mask = fit_plane_ransac(
        points,
        distance_threshold=0.015,  # 1.5cm 阈值
        max_iterations=500,
        min_inliers=100
    )
    
    # 桌面内点 vs 桌面上方的点（物体）
    table_points = points[plane_mask]
    object_points = points[~plane_mask]
    
    if verbose:
        print(f"  桌面内点: {len(table_points)}")
        print(f"  桌面上方点（物体）: {len(object_points)}")
    
    if len(object_points) < 10:
        if verbose:
            print("警告: 物体点太少")
        return []
    
    # Step 2: 欧式聚类
    if verbose:
        print("\n[Step 2] 欧式聚类...")
    try:
        clusters = euclidean_clustering_fast(object_points, distance_threshold=0.03)
    except ImportError:
        # 如果没有 scipy，手动实现简单的聚类
        clusters = euclidean_clustering_naive(object_points, distance_threshold=0.03)
    
    if verbose:
        print(f"  检测到 {len(clusters)} 个独立物体:")
        for i, cluster in enumerate(clusters):
            center = cluster.mean(axis=0)
            size = cluster.max(axis=0) - cluster.min(axis=0)
            print(f"  物体{i+1}: {len(cluster)} 点, "
                  f"中心({center[0]:.3f}, {center[1]:.3f}, {center[2]:.3f}), "
                  f"尺寸({size[0]:.3f} x {size[1]:.3f} x {size[2]:.3f})m")
    
    return clusters


def euclidean_clustering_naive(points, distance_threshold=0.03, min_cluster_size=10):
    """
    朴素欧式聚类（无 KD-Tree，适合小规模点云）
    """
    n_points = len(points)
    visited = np.zeros(n_points, dtype=bool)
    clusters = []
    
    for i in range(n_points):
        if visited[i]:
            continue
        
        # BFS
        queue = deque([i])
        visited[i] = True
        cluster_indices = []
        
        while queue:
            idx = queue.popleft()
            cluster_indices.append(idx)
            
            # 遍历所有未访问的点（朴素方法，O(N^2)）
            for j in range(n_points):
                if not visited[j]:
                    dist = np.linalg.norm(points[idx] - points[j])
                    if dist < distance_threshold:
                        visited[j] = True
                        queue.append(j)
        
        if len(cluster_indices) >= min_cluster_size:
            clusters.append(points[cluster_indices])
    
    return clusters


def test_segmentation():
    """测试分割功能"""
    np.random.seed(42)
    
    # 模拟桌面点云
    # 桌面: z = 0.8
    table = np.random.uniform([-0.5, -0.5, 0.795], [0.5, 0.5, 0.805], (1000, 3))
    table += np.random.randn(1000, 3) * 0.003
    
    # 物体1: 球
    theta = np.random.uniform(0, 2*np.pi, 300)
    phi = np.random.uniform(0, np.pi, 300)
    r = 0.06
    obj1 = np.column_stack([
        r * np.sin(phi) * np.cos(theta),
        r * np.sin(phi) * np.sin(theta),
        r * np.cos(phi),
    ]) + np.array([0.0, 0.0, 0.86])
    
    # 物体2: 立方体
    obj2 = np.random.uniform([0.2, 0.2, 0.8], [0.3, 0.3, 0.9], (300, 3))
    
    # 合并
    all_points = np.vstack([table, obj1, obj2]).astype(np.float32)
    np.random.shuffle(all_points)
    
    print("测试分割功能...")
    print(f"模拟点云: {len(all_points)} 点")
    
    objects = segment_table_objects(all_points, verbose=True)
    
    return objects
