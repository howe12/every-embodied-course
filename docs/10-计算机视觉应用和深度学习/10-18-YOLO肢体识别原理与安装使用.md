## 1. 肢体识别概述

### 1.1 什么是肢体识别

**肢体识别**（Body Pose Estimation），也称**人体姿态估计**（Human Pose Estimation, HPE），是计算机视觉中的一项核心技术，其目标是从图像或视频中定位出人体关键点（如头部、肩膀、手腕、膝盖等）的位置，并推断出人体的结构姿态。

与目标检测（给出"人在哪里"的矩形框）不同，姿态估计给出了"人的每个部位在哪里"的精细坐标。如下所示：

```
目标检测输出：              姿态估计输出：
┌─────────────┐           ┌─────────────┐
│  ┌───────┐  │           │  ●──────●   │
│  │ ○ ○   │  │           │  │\    /│   │
│  │  ━    │  │           │  ●──●──●   │
│  │ / \   │  │           │  │  │  │   │
│  │ / \   │  │           │  ●──┼──●   │
│  └───────┘  │           │    │      │
│  class:     │           │    ●      │
│  person     │           │   / \     │
│  conf:0.97  │           │  ●   ●     │
└─────────────┘           │ person@0.97 │
                          │ kpts:17    │
                          └─────────────┘
```

**为什么肢体识别对机器人很重要？**

机器人要与人类安全、高效地交互，必须理解人的意图和状态。以下是典型应用场景：

| 应用场景 | 姿态估计的作用 | 机器人行为 |
|---------|--------------|-----------|
| 人机交互 | 检测挥手、指向等手势 | 机器人响应招手、跟随指向方向 |
| 摔倒检测 | 检测人体是否处于非正常姿态 | 机器人报警或上前搀扶 |
| 行为识别 | 检测举手、起立、坐下等动作序列 | 机器人判断用户需求 |
| 运动分析 | 分析人的运动姿态和轨迹 | 辅助康复训练、体育指导 |
| 安全避障 | 检测人的位置和朝向 | 机器人在人群中安全导航 |
| 抓取辅助 | 估计人手位置和朝向 | 机械臂递送物品给用户 |

### 1.2 姿态估计任务分类

姿态估计可以从多个维度进行分类：

**（1）按检测人数分类**

| 类型 | 说明 | 典型算法 |
|------|------|---------|
| **单人姿态估计** | 输入图像中只有一个人，检测该人的所有关键点 | DeepPose, Hourglass |
| **多人姿态估计** | 输入图像中有多个人，需要同时检测所有人的所有关键点 | OpenPose, YOLOv8-pose |

**（2）按输出维度分类**

| 类型 | 说明 | 应用场景 |
|------|------|---------|
| **2D姿态估计** | 在图像平面输出关键点的 (x, y) 像素坐标 | 实时监控、人机交互 |
| **3D姿态估计** | 输出关键点在3D空间中的坐标 (x, y, z) | 运动分析、3D重建 |

**（3）按检测范式分类**

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **自顶向下（Top-Down）** | 先用目标检测器找出每个人，再对每个人单独做姿态估计 | 精度高，可复用检测器 | 人数多时计算量大，检测漏检会导致姿态漏检 |
| **自底向上（Bottom-Up）** | 先检测所有关键点，再通过关联算法将属于同一个人的关键点组合 | 速度快，不受人数影响 | 关键点匹配复杂，遮挡时易出错 |

```
自顶向下（Top-Down）：              自底向上（Bottom-Up）：
┌─────────────────────┐          ┌─────────────────────┐
│  [检测器]            │          │  [关键点检测器]       │
│  ○→人1 ○→人2 ○→人3  │          │  ● ● ● ● ● ● ● ●     │
│     ↓    ↓    ↓     │          │  ● ● ● ● ● ● ● ●    │
│  [单人姿态估计器]     │          │  [关键点关联算法]     │
│  人1姿态 人2姿态 人3  │          │  组合成完整人体       │
└─────────────────────┘          └─────────────────────┘
```

**YOLO姿态估计的范式**：YOLOv8-pose 采用的是**自顶向下**范式，利用YOLO的检测能力先定位每个人体边界框，再预测每个人框内的关键点。

### 1.3 人体骨骼模型

人体姿态估计需要定义一套标准的关键点骨架结构。目前最常用的是 **COCO** 数据集定义的 **17关键点** 模型，也是YOLO姿态估计采用的方案。

**COCO 17关键点定义：**

```
COCO 人体关键点定义（17点）：

        0: nose        (鼻尖)
       / \
      1   2            1: left_eye   (左眼)
      |   |            2: right_eye  (右眼)
      3   4            3: left_ear   (左耳)
          |            4: right_ear  (右耳)
           5           5: left_shoulder  (左肩)
          / \          6: right_shoulder (右肩)
         6   7         7: left_elbow     (左肘)
         |   |         8: right_elbow    (右肘)
         8   9         9: left_wrist     (左手腕)
              \       10: left_hip       (左髋)
              10      11: right_hip      (右髋)
             /  \     12: left_knee      (左膝)
           11    13   13: right_knee     (右膝)
           |     |    14: left_ankle    (左踝)
          12    14   15: right_ankle   (右踝)
          |     |
         15    16   16: left_ankle (同15，为兼容保留)
         |     |     17: right_ankle (同16，为兼容保留)
         ●     ●
        15    16
```

**关键点可见性标志：**

每个关键点还有一个可见性标志，用于处理遮挡情况：

```python

```

## 2. YOLO姿态估计发展与原理

### 2.1 YOLO姿态估计发展历程

YOLO系列模型最初专注于目标检测，随着版本迭代，逐渐扩展到实例分割（YOLOv8-seg）和姿态估计（YOLOv8-pose）。

**YOLO姿态估计发展脉络：**

| 版本 | 推出时间 | 姿态估计模型 | 主要改进 |
|------|---------|------------|---------|
| YOLOv3 | 2018 | 无 | 目标检测，多尺度输出 |
| YOLOv5 | 2020 | YOLOv5-pose | 首次引入姿态估计，CSPDarknet骨干网 |
| YOLOv7 | 2022 | YOLOv7-pose | 高效姿态估计，模型 ensemble |
| YOLOv8 | 2023 | **YOLOv8-pose** | Anchor-free，C2f骨干，精度速度全面提升 |
| DWPose | 2023 | DWPose | 蒸馏方案，更高精度的实时姿态估计 |

**YOLOv8-pose 的定位：**

YOLOv8-pose 是 Ultralytics 官方发布的姿态估计模型，它与目标检测、实例分割模型共享同一套骨干网络和 neck 结构，只是在 head 层针对不同任务做了适配：

```
YOLOv8 模型家族：
                    骨干网络（Backbone）
                         │
                    C2f + SPPF
                         │
                    路径聚合网络（Neck, PANet）
                         │
                    ┌────┴────┐
                    │         │
              检测Head     分割Head     姿态Head
           (YOLOv8n)   (YOLOv8n-seg)  (YOLOv8n-pose)
             80类       32类+掩膜      17关键点
```

### 2.2 与目标检测、实例分割的区别与联系

理解YOLOv8-pose与同系列模型的异同，有助于更好地使用和部署：

**（1）共同点**

| 特性 | 说明 |
|------|------|
| **共享骨干网络** | 都使用 CSPDarknet / YOLOv8 Backbone 进行特征提取 |
| **共享 Neck** | 都使用 PANet 进行多尺度特征融合 |
| **相同的后处理流程** | 同样的 NMS（非极大值抑制）逻辑 |
| **相同的部署生态** | 都可以导出为 ONNX、TensorRT，支持 ROS2 部署 |

**（2）差异对比**

| 维度 | 目标检测 (YOLOv8n) | 实例分割 (YOLOv8n-seg) | 姿态估计 (YOLOv8n-pose) |
|------|-------------------|----------------------|----------------------|
| **输出内容** | 边界框 + 类别 | 边界框 + 类别 + 掩膜 | 边界框 + 关键点坐标 |
| **Head 结构** | 检测头（类别+框回归） | 检测头 + 分割头 | 检测头 + 关键点头 |
| **输出维度** | (batch, num_anchors, 5+num_classes) | (batch, num_anchors, 5+num_classes+32) | (batch, num_anchors, 56) |
| **关键点数量** | 无 | 无 | 17（COCO标准）|
| **应用场景** | 通用目标检测 | 实例分割、抠图 | 人体姿态、行为分析 |
| **模型大小** | ~3.2M 参数 | ~3.4M 参数 | ~3.3M 参数 |

**输出格式对比：**

### 2.3 YOLO姿态估计原理

**（1）整体架构**

YOLOv8-pose 的网络结构可以分为三个部分：

```
输入图像
    │
    ▼
┌───────────────────────────────────────────────┐
│  Backbone（骨干网络，YOLOv8 C2f）              │
│  - 特征提取                                     │
│  - 输出多尺度特征图：P3, P4, P5                  │
└───────────────────────────────────────────────┘
    │
    ▼
┌───────────────────────────────────────────────┐
│  Neck（路径聚合网络，PANet）                    │
│  - 多尺度特征融合                               │
│  - 增强不同尺度人体的检测能力                     │
└───────────────────────────────────────────────┘
    │
    ▼
┌───────────────────────────────────────────────┐
│  Pose Head（姿态估计头）                        │
│  - 边界框检测（与目标检测共享）                  │
│  - 关键点检测（独有）                            │

    │
    ▼
```

**（3）关键点损失函数设计**

YOLOv8-pose 使用 **OKS（Object Keypoint Similarity）** 作为关键点匹配和评估的核心，它同时考虑了关键点的定位精度和置信度。

```python
import numpy as np

def compute_oks(pred_keypoints, gt_keypoints, gt_visibility, sigmas=None):
    """
    计算 OKS（Object Keypoint Similarity）
    
    OKS 是 COCO 姿态估计的标准评估指标，
    类似于 IoU 用于目标检测，OKS 用于衡量预测关键点与真实关键点的相似度
    
    参数:
        pred_keypoints: 预测的关键点坐标，shape=(17, 2)，值为归一化坐标 [0,1]
        gt_keypoints: 真实关键点坐标，shape=(17, 2)
        gt_visibility: 真实关键点可见性标志，shape=(17,)
                      0=标注且可见，1=标注但遮挡，2=未标注
        sigmas: 关键点标准差（与COCO数据集对齐）
    
    返回:
        oks: OKS 值，范围 [0, 1]，越接近1表示预测越准确
    """
    
    # COCO 关键点标准差（经验值，每个关键点精度要求不同）
    # 例如眼睛精度要求高（标准差小），hip精度要求低（标准差大）
    if sigmas is None:
        sigmas = np.array([
            .26, .25, .25, .35, .35, .79, .79, .72, .72, .62, .62, 1.07, 1.07,
            .87, .87, .89, .89
        ]) / 10.0  # 归一化
    
    # 只计算有效关键点（visibility=0 或 1）
    valid_mask = gt_visibility < 2  # visibility=2 表示未标注，跳过
    
    if np.sum(valid_mask) == 0:
        return 0.0
    
    # 计算每个关键点的欧氏距离
    distances = np.sqrt(np.sum((pred_keypoints[valid_mask] - gt_keypoints[valid_mask]) ** 2, axis=1))
    
    # 计算归一化距离
    # 关键点评定时，通常用人体的框大小做归一化
    # 这里简化使用标准差 sigmas 作为归一化因子
    sigmas_valid = sigmas[valid_mask]
    
    # OKS 公式: exp(-d^2 / (2 * s^2 * k^2))
    # d = 欧氏距离, s = 目标尺度因子, k = sigmas
    # 为简化，这里假设 s=1（实际代码中 s 是人体框对角线长度的归一化因子）
    s = 1.0  # 简化：假设归一化尺度因子为1
    
    # 计算 OKS
    oks = np.exp(- (distances ** 2) / (2 * (s ** 2) * (sigmas_valid ** 2)))
    
    # 返回所有有效关键点的平均 OKS
    return np.mean(oks)
```

### 2.4 关键点置信度与匹配

**关键点置信度的含义：**

关键点置信度（keypoint confidence）反映了模型对该关键点预测的确定性。置信度越高，表示该关键点越可能是准确的。

```python
# 置信度阈值处理
confidence_threshold = 0.5  # 常用阈值

def filter_keypoints_by_confidence(keypoints, threshold=0.5):
    """
    根据置信度过滤关键点
    
    参数:
        keypoints: 关键点列表，每个包含 x, y, conf
        threshold: 置信度阈值
    
    返回:
        过滤后的关键点列表
    """
    return [
        kp for kp in keypoints 
        if kp['conf'] >= threshold
    ]

# 实际应用中，不同置信度的处理策略

    "低置信度 (conf < 0.5)": "通常过滤掉，视为遮挡或不可见",
```

## 5. ROS2姿态估计节点

### 5.1 基础ROS2姿态估计节点

以下是一个完整的ROS2姿态估计节点示例，支持图像订阅、YOLOv8-pose姿态估计、关键点发布：

```python
import rclpy
from rclpy.node import Node
from rclpy.qos import QoSProfile, ReliabilityPolicy, HistoryPolicy
from sensor_msgs.msg import Image
from vision_msgs.msg import Detection2DArray, Detection2D, BoundingBox2D
from cv_bridge import CvBridge
import cv2
import numpy as np
from ultralytics import YOLO


class PoseEstimationNode(Node):
    def __init__(self):
        super().__init__('pose_estimation_node')
        self.bridge = CvBridge()
        self.model = YOLO('yolov8n-pose.pt')

        self.declare_parameter('conf_threshold', 0.5)
        self.conf_threshold = self.get_parameter('conf_threshold').value

        qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                          history=HistoryPolicy.KEEP_LAST, depth=1)
        self.create_subscription(Image, '/image_raw', self.image_callback, qos)
        self.pub_image = self.create_publisher(Image, '/pose/image', qos)
        self.pub_kpts = self.create_publisher(Detection2DArray, '/pose/keypoints', qos)
        self.get_logger().info('Pose estimation node started')

    def image_callback(self, msg):
        try:
            img = self.bridge.imgmsg_to_cv2(msg, desired_encoding='bgr8')
            results = self.model(img, conf=self.conf_threshold, verbose=False)
            result = results[0]
            annotated = img.copy()
            if result.keypoints is not None:
                for pk in result.keypoints:
                    k = pk.cpu().numpy()
                    if k.shape[1] == 2:
                        k = np.hstack([k, np.ones((17,1))*0.9])
                    annotated = self.draw_skeleton(annotated, k)
                self.pub_kpts.publish(self.make_detection_msg(result.keypoints, msg.header))
            self.pub_image.publish(self.bridge.cv2_to_imgmsg(annotated, encoding='bgr8'))
        except Exception as e:
            self.get_logger().error(f'Error: {str(e)}')

    def draw_skeleton(self, img, kpts):
        skel = [(5,6),(5,7),(7,9),(6,8),(8,10),(5,11),(6,12),(11,12),(11,13),(13,15),(12,14),(14,16)]
        for s,e in skel:
            k1,k2 = kpts[s],kpts[e]
            if k1[2]>self.conf_threshold and k2[2]>self.conf_threshold:
                cv2.line(img,(int(k1[0]),int(k1[1])),(int(k2[0]),int(k2[1])),(0,255,0),2)
        for k in kpts:
            if k[2]>self.conf_threshold:
                cv2.circle(img,(int(k[0]),int(k[1])),4,(0,0,255),-1)
        return img

    def make_detection_msg(self, kpts_data, header):
        msg = Detection2DArray()
        msg.header = header
        for pk in kpts_data:
            k = pk.cpu().numpy()
            if k.shape[1] == 2:
                k = np.hstack([k, np.ones((17,1))*0.9])
            det = Detection2D()
            v = k[k[:,2]>self.conf_threshold]
            if len(v)>0:
                bbox = BoundingBox2D()
                bbox.center.position.x = (v[:,0].min()+v[:,0].max())/2
                bbox.center.position.y = (v[:,1].min()+v[:,1].max())/2
                bbox.size_x = v[:,0].max()-v[:,0].min()
                bbox.size_y = v[:,1].max()-v[:,1].min()
                det.bbox = bbox
            msg.detections.append(det)
        return msg


def main(args=None):
    rclpy.init(args=args)
    node = PoseEstimationNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()
```

```python
        def bbox(k):
            v = k[k[:,2]>0.3]

        x1,y1,x2,y2 = bbox(k1)

        area = (x2-x1)*(y2-y1)+(x2b-x1b)*(y2b-y1b)-inter
        return inter/area if area>0 else 0.0

        for pid, pdata in self.tracked.items():
            best_i, best_idx = 0, -1
            for idx, dk in enumerate(detections):
                if idx in [m[1] for m in matched]: continue
                iou = self.compute_iou(pdata['keypoints'], dk)
                if iou>best_i and iou>self.iou_threshold:
                    best_i, best_idx = iou, idx
            if best_idx>=0:
                matched.append((pid,best_idx,best_i))
                self.tracked[pid]['keypoints'] = detections[best_idx]
                self.tracked[pid]['age'] += 1
                self.tracked[pid]['last_seen'] = 0
        for idx, k in enumerate(detections):
            if idx not in [m[1] for m in matched]:
                pid = self.next_id
                self.next_id += 1
                self.tracked[pid] = {'keypoints':k,'age':1,'last_seen':0}
        for pid in list(self.tracked.keys()):
            self.tracked[pid]['last_seen'] += 1
            if self.tracked[pid]['last_seen'] > 10:
                del self.tracked[pid]
        return [(pid,p['keypoints']) for pid,p in self.tracked.items()]
```

```python
class PoseNavCoordinator(Node):
    def __init__(self):
        super().__init__('pose_nav_coordinator')
        self.bridge = CvBridge()

        fx,fy,cx,cy = self.cam['fx'],self.cam['fy'],self.cam['cx'],self.cam['cy']
        return (u-cx)*d/fx, (v-cy)*d/fy, d

    def nearest_person(self, kpts_data):
        nearest, min_d = None, float('inf')
        if not kpts_data: return None, None
        for pk in kpts_data:
            k = pk.cpu().numpy()
            if k.shape[1]==2: continue
            n = k[0]
            if n[2]<0.5: continue
            v = k[k[:,2]>0.5]
            if len(v)>0:
                ph = v[:,1].max()-v[:,1].min()
                if ph>10:
                    d = (607.5*1.7)/ph
                    if d<min_d:
                        min_d, nearest = d, k
        return nearest, min_d

    def callback(self, msg):
        try:
            img = self.bridge.imgmsg_to_cv2(msg)
            res = self.model(img, conf=0.5, verbose=False)[0]
            nk, nd = self.nearest_person(res.keypoints)
            if nk is not None and nd is not None:
                n = nk[0]
                u,v = int(n[0]),int(n[1])
                x,y,z = self.px_to_3d(u,v,nd)

                cmd = Twist()
                if z > 1.8: cmd.linear.x = 0.3
                elif z < 1.2: cmd.linear.x = -0.2
                else: cmd.linear.x = 0.0
                if abs(x) > 0.2: cmd.angular.z = -0.5 if x>0 else 0.5
                self.pub_vel.publish(cmd)
                cv2.circle(img,(u,v),10,(0,255,0),-1)
```

## 本章小结

本章我们系统性地学习了 YOLO 肢体识别（人体姿态估计）的原理与实践应用。

在肢体识别概述部分，我们了解到姿态估计是计算机视觉中定位人体关键点位置的技术。相比于目标检测给出"人在哪里"的边界框，姿态估计给出了"人的每个部位在哪里"的精细坐标。姿态估计在机器人领域有着广泛的应用，包括人机交互（挥手、指向）、摔倒检测、行为识别、安全避障等。姿态估计任务可以从多个维度分类：按检测人数分为单人/多人；按输出维度分为2D/3D；按检测范式分为自顶向下/自底向上。YOLOv8-pose 采用的是自顶向下范式，利用 YOLO 的检测能力先定位每个人体边界框，再预测关键点。COCO 数据集定义的17关键点模型是行业标准。

在 YOLO 姿态估计发展与原理部分，我们学习了 YOLO 姿态估计从 YOLOv5-pose 到 YOLOv8-pose 的发展历程。YOLOv8-pose 与目标检测、实例分割共享骨干网络和 Neck 结构，仅在 Head 层针对不同任务做适配。关键点检测 Head 输出 56 维向量：4(边界框) + 1(置信度) + 17x3(关键点坐标和置信度)。OKS（Object Keypoint Similarity）是 COCO 姿态估计的标准评估指标。

在 PyTorch 实践代码部分，我们通过代码实现了：图像姿态估计（带骨骼可视化）、视频/摄像头实时姿态估计、骨骼可视化进阶（多种样式）、行为识别示例（举手检测、跌倒检测、坐下检测）。这些代码可以直接迁移到机器人项目中。

## 答案

### 1. 选择题答案

**题目1.1：** **B. 自顶向下（Top-Down）**

解析：YOLOv8-pose 利用 YOLO 的检测能力先定位每个人体边界框，再对每个人单独做姿态估计。这是典型的自顶向下范式。相比之下，OpenPose 采用的是自底向上范式——先检测所有关键点，再通过关联算法组合成完整人体。

**题目1.2：** **C. 17个**

解析：COCO 数据集定义了17个人体关键点：鼻尖、左眼、右眼、左耳、右耳、左肩、右肩、左肘、右肘、左手腕、右手腕、左髋、右髋、左膝、右膝、左踝、右踝。

**题目1.3：** **C. 人体姿态估计**

解析：OKS（Object Keypoint Similarity）是 COCO 姿态估计挑战的标准评估指标。它综合考虑了关键点的定位精度和可见性，类似于目标检测中的 IoU。OKS 值范围为 [0, 1]，越接近1表示预测与真实越接近。

**题目2.1：** **关键点坐标和置信度**（或"17个关键点的x坐标、y坐标、置信度"）

**题目2.2：** **左肩（left_shoulder）** 和 **右肩（right_shoulder）**

**题目2.3：** **30度**

解析：在跌倒检测中，通常计算躯干（肩膀中点到髋部中点）与水平面的夹角。当夹角小于30度时，表明躯干接近水平状态，人体处于跌倒姿势。

### 3. 简答题答案

**题目3.1：**

**自顶向下（Top-Down）的优缺点：**
优点：精度高（每个人单独估计，不受他人干扰）；可复用（直接使用成熟的目标检测器）；实现简单（检测+单人姿态估计的流水线）
缺点：计算量随人数线性增长；检测漏检会导致姿态漏检；速度不稳定（人数多时速度显著下降）

**自底向上（Bottom-Up）的优缺点：**
优点：速度相对稳定（与人数无直接关系）；检测器漏检不致命
缺点：关键点匹配复杂；遮挡情况处理困难；精度通常低于自顶向下

**YOLOv8-pose 采用自顶向下范式的原因：** YOLOv8-pose 基于 YOLO 检测框架，自然利用检测框来确定每个人的区域。这种方案实现简单，可以复用 YOLO 在目标检测上的成熟技术（如 NMS、anchor matching 等），且精度高。自底向上方案虽然理论速度更稳定，但实现复杂度高，关联算法设计困难，而 YOLO 的设计初衷就是追求精度和速度的平衡，自顶向下范式更符合其架构理念。

### 4. 编程题答案

**题目4.1：**

```python
import numpy as np


def compute_person_height(keypoints, min_confidence=0.5):
    COCO_NAMES = ['nose','left_eye','right_eye','left_ear','right_ear',
                  'left_shoulder','right_shoulder','left_elbow','right_elbow',
                  'left_wrist','right_wrist','left_hip','right_hip',
                  'left_knee','right_knee','left_ankle','right_ankle']
    IDX = {n:i for i,n in enumerate(COCO_NAMES)}

    def get_top():
        ls = keypoints[IDX['left_shoulder']]
        rs = keypoints[IDX['right_shoulder']]
        if ls[2]<min_confidence or rs[2]<min_confidence: return None
        mx,my = (ls[0]+rs[0])/2, (ls[1]+rs[1])/2
        d = np.sqrt((rs[0]-ls[0])**2+(rs[1]-ls[1])**2)
        return (mx, my - 1.8*d)

    def get_bottom():
        la = keypoints[IDX['left_ankle']]
        ra = keypoints[IDX['right_ankle']]
        if la[2]<min_confidence and ra[2]<min_confidence: return None
        if la[2]>=min_confidence and ra[2]>=min_confidence:
            return ((la[0]+ra[0])/2,(la[1]+ra[1])/2)
        return (la[0],la[1]) if la[2]>=min_confidence else (ra[0],ra[1])

    top,bottom = get_top(),get_bottom()
    if top is None or bottom is None: return None
    return np.sqrt((bottom[0]-top[0])**2+(bottom[1]-top[1])**2)
```

**答案解析：** 本题考察对 COCO 关键点定义的理解和关键点数据的处理能力。头顶位置不能直接获取，需要通过肩膀位置推算（头顶大约在肩膀中点上方1.8倍肩距）。脚底使用左右踝关节的中点。计算身高即计算头顶到脚底的欧氏距离。
