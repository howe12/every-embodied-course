## 1. 安装方式对比——哪种适合你？

### 1.1 Debian包安装（推荐）

**Debian包安装**是最简单、最推荐的方式。ROS2官方将编译好的包发布到Ubuntu/Debian的软件源中，你可以像安装普通软件一样用`apt`安装。

![Debian包安装示意图](./images/debian_install.jpg)
*图注：Debian包安装是最简单的ROS2安装方式*

**优点：**
- ✅ 安装简单，一条命令搞定
- ✅ 自动处理依赖
- ✅ 官方维护，及时更新
- ✅ 适合大多数用户

**缺点：**
- ❌ 无法自定义修改ROS2底层代码
- ❌ 更新受限于官方发布节奏

**适用场景：**
- 学习ROS2
- 开发机器人应用
- 产品部署（非深度定制需求）

### 1.2 源码编译安装

**源码编译安装**从源代码构建ROS2，给你最大的自由度。

![源码编译流程图](./images/source_build_flow.jpg)
*图注：源码编译需要下载源码、安装依赖、编译、安装多个步骤*

**优点：**
- ✅ 可以修改ROS2底层代码
- ✅ 可以使用最新功能（nightly版本）
- ✅ 便于调试和问题定位

**缺点：**
- ❌ 安装过程复杂，耗时长（可能需要数小时）
- ❌ 需要手动解决依赖
- ❌ 维护成本高

**适用场景：**
- 需要修改ROS2源码
- 研究ROS2内部机制
- 使用最新的实验性功能

### 1.3 版本选择建议

| 版本 | 类型 | 支持周期 | 适用场景 |
|------|------|----------|----------|
| **Humble Hawksbill** | LTS | 2022.05 - 2027.05 | **推荐**：稳定可靠，5年支持 |
| Iron Irwinton | 正式版 | 2023.05 - 2024.11 | 尝试最新功能 |
| Jazzy Jalisco | 正式版 | 2024.05 - 2025.05 | 最新的正式版 |

**强烈建议选择 Humble** —— 这是目前最稳定、文档最完善、社区支持最好的LTS版本。

## 2. 环境配置——一步步教你搭建

### 2.1 Debian包安装详解（Ubuntu 22.04）

以下步骤在Ubuntu 22.04 LTS上安装ROS2 Humble：

#### 步骤1：设置语言环境

```bash
# 设置UTF-8语言环境
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 验证
locale
```

#### 步骤2：添加ROS2软件源

```bash
# 添加ROS2 GPG密钥
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key | sudo tee /usr/share/keyrings/ros-archive-keyring.gpg > /dev/null

# 添加软件源
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
```

#### 步骤3：安装ROS2

```bash
# 更新软件源
sudo apt update

# 安装桌面版（推荐）：包含ROS2核心 + RViz2 + Gazebo等工具
sudo apt install ros-humble-desktop

# 或者安装基础版：仅包含核心和通信库
# sudo apt install ros-humble-ros-base

# 安装开发工具（可选但推荐）
sudo apt install ros-dev-tools
```

#### 步骤4：安装时间同步工具（重要！）

ROS2对时间同步要求较高，建议安装chrony：

```bash
sudo apt install chrony
sudo systemctl enable chrony
sudo systemctl start chrony
```

### 2.2 源码编译安装详解

如果你需要从源码编译，按以下步骤进行：

#### 步骤1：安装编译工具和依赖

```bash
# 安装编译工具
sudo apt install -y \
    python3-colcon-common-extensions \
    python3-pip \
    python3-rosdep \
    python3-vcstool \
    build-essential \
    cmake \
    git \
    libcunit1-dev

# 安装ROS2依赖
sudo apt install -y \
    libtinyxml2-dev \
    libboost-regex-dev \
    libeigen3-dev \
    libopencv-dev \
    libassimp-dev \
    libconsole-bridge-dev \
    libtinyxml2-dev \
    libyaml-cpp-dev \
    liblz4-dev
```

#### 步骤2：下载ROS2源码

```bash
# 创建工作空间
mkdir -p ~/ros2_humble/src
cd ~/ros2_humble

# 下载ROS2源码（注意选择正确的版本分支）
# 注意：完整源码约4GB，下载时间可能较长
vcs import --input https://raw.githubusercontent.com/ros/rosdistro/master/ros2/humble/ros2 humble.repos

# 如果上述命令失败，可以分步下载
# git clone https://github.com/ros2/ros2.git ~/ros2_humble/src/ros2 --branch humble
```

#### 步骤3：安装系统依赖

```bash
# 使用rosdep安装依赖
cd ~/ros2_humble
sudo rosdep install --from-paths src --ignore-src -r -y
```

#### 步骤4：编译

```bash
# 设置编译选项
export MAKEFLAGS="-j$(nproc)"

# 编译（使用colcon）
# 注意：完整编译可能需要1-2小时或更长
colcon build --symlink-install
```

#### 步骤5：验证编译结果

```bash
# 测试Talker
./install/setup.bash
ros2 run demo_nodes_cpp talker

# 新终端测试Listener
./install/setup.bash
ros2 run demo_nodes_cpp listener
```

### 2.3 环境变量配置

安装完成后，需要配置环境变量：

#### 步骤1：加载ROS2环境

```bash
# 对于Bash用户
source /opt/ros/humble/setup.bash

# 对于Zsh用户
source /opt/ros/humble/setup.zsh
```

#### 步骤2：设置自动加载（推荐）

```bash
# 添加到 ~/.bashrc（每次打开终端自动加载）
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc

# 使当前终端生效
source ~/.bashrc
```

#### 步骤3：验证环境变量

```bash
# 检查ROS2环境变量
echo $ROS_DISTRO

# 应该输出：humble
```

## 3. 验证方法——确保安装成功

### 3.1 版本检查

```bash
# 检查ROS2版本
ros2 --version

# 预期输出：
#   ros2 0.25.3 (或类似版本号)

# 检查环境变量
printenv | grep ROS

# 应该看到类似输出：
#   ROS_DISTRO=humble
#   ROS_LOCALHOST_ONLY=0
```

### 3.2 运行示例程序

#### 实验1：Talker和Listener（C++）

**步骤1：启动Talker（发布者）**

```bash
# 终端1：启动Talker
ros2 run demo_nodes_cpp talker

# 预期输出：
# [INFO] [1234567890.123456789]: Publishing: 'Hello World: 0'
# [INFO] [1234567890.223456789]: Publishing: 'Hello World: 1'
# ...
```

**步骤2：启动Listener（订阅者）**

```bash
# 终端2：启动Listener
ros2 run demo_nodes_cpp listener

# 预期输出：
# [INFO] [1234567890.323456789]: I heard: 'Hello World: 0'
# [INFO] [1234567890.423456789]: I heard: 'Hello World: 1'
# ...
```

#### 实验2：Talker和Listener（Python）

```bash
# 终端1
ros2 run demo_nodes_py talker

# 终端2
ros2 run demo_nodes_py listener
```

#### 实验3：使用小乌龟模拟器

```bash
# 安装turtlesim（如果未安装）
sudo apt install ros-humble-turtlesim

# 启动模拟器
ros2 run turtlesim turtlesim_node

# 新终端启动键盘控制
ros2 run turtlesim turtle_teleop_key
```

### 3.3 工具测试

#### RViz2可视化工具

```bash
# 启动RViz2
rviz2

# 首次使用配置：
# 1. 点击左下角 "Add" 按钮
# 2. 选择 "By Topic"
# 3. 选择要可视化的话题
```

#### rqt_graph查看节点图

```bash
# 启动节点关系图可视化
rqt_graph

# 或者使用命令
ros2 run rqt_graph rqt_graph
```

#### 话题列表查看

```bash
# 列出所有活跃话题
ros2 topic list

# 查看话题类型
ros2 topic type /chatter

# 查看话题详细信息
ros2 topic info /chatter
```

### 3.4 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| `ros2: command not found` | 环境变量未加载 | `source /opt/ros/humble/setup.bash` |
| 无法发现其他节点的话题 | 网络配置问题 | 检查 `ROS_DOMAIN_ID` |
| 通讯延迟高 | 网络或QoS配置问题 | 检查网络，使用Wired连接 |
| 编译失败 | 依赖缺失 | `sudo apt install ros-humble-ros-base` |

## 4. 实训案例：完整安装与验证

### 4.1 实训目标

1. 在Ubuntu 22.04上安装ROS2 Humble
2. 配置开发环境
3. 验证安装成功

### 4.2 实训步骤

#### 步骤1：系统准备

```bash
# 检查Ubuntu版本
lsb_release -a

# 确认是Ubuntu 22.04
# 如果不是，需要先升级或重新安装系统

# 检查磁盘空间（建议至少20GB可用）
df -h
```

#### 步骤2：安装ROS2（采用Debian方式）

```bash
# 按照前面2.1节的步骤安装
# 完整执行以下命令：

# 1. 设置语言
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8

# 2. 添加软件源
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key | sudo tee /usr/share/keyrings/ros-archive-keyring.gpg > /dev/null
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 3. 安装ROS2
sudo apt update
sudo apt install ros-humble-desktop

# 4. 配置环境
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

#### 步骤3：安装开发工具

```bash
# 安装常用开发工具
sudo apt install -y \
    ros-dev-tools \
    python3-colcon-common-extensions \
    python3-rosdep \
    git \
    vim \
    terminator

# 配置terminator（可选）
mkdir -p ~/.config/terminator
cat > ~/.config/terminator/config << 'EOF'
[global]
[layouts]
  [[default]]
    [[[child0]]]
      type = Terminal
      parent = window0
    [[[child1]]]
      type = Terminal
      parent = window0
[profiles]
  [[default]]
    cursor_color = '#aaaaaa'
[plugins]
EOF
```

#### 步骤4：创建ROS2工作空间

```bash
# 创建ROS2工作空间
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws

# 初始化工作空间
colcon build

# 配置环境
echo "source ~/ros2_ws/install/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

#### 步骤5：验证安装

```bash
# 1. 版本检查
ros2 --version

# 2. 运行Talker/Listener
# 终端1
ros2 run demo_nodes_cpp talker

# 终端2
ros2 run demo_nodes_cpp listener

# 3. 启动turtlesim
ros2 run turtlesim turtlesim_node

# 4. 启动rqt_graph查看节点关系
ros2 run rqt_graph rqt_graph
```

#### 步骤6：安装其他常用包

```bash
# 安装常用机器人功能包
sudo apt install -y \
    ros-humble-rosbag2 \
    ros-humble-rviz2 \
    ros-humble-gazebo-ros-pkgs \
    ros-humble-navigation2 \
    ros-humble-slam-toolbox

# 安装额外示例包
sudo apt install -y \
    ros-humble-demo-nodes-cpp \
    ros-humble-demo-nodes-py
```

### 4.3 验收标准

完成实训后，你应该能够：

- [ ] `ros2 --version` 显示 Humble 版本
- [ ] Talker/Listener 能正常通讯
- [ ] turtlesim 能正常启动并响应键盘控制
- [ ] `ros2 topic list` 能看到活跃话题
- [ ] `rqt_graph` 能看到节点关系图

## 本章小结

本章我们详细讲解了ROS2的安装和环境配置。

**核心要点回顾：**

1. **安装方式**：
   - Debian包安装：简单快捷，推荐大多数用户
   - 源码编译：灵活但复杂，适合需要深度定制的用户
   - 推荐选择 Humble LTS 版本

2. **环境配置**：
   - 需要加载 `setup.bash`
   - 建议添加到 `.bashrc` 自动加载
   - 检查 `ROS_DISTRO` 环境变量

3. **验证方法**：
   - `ros2 --version` 检查版本
   - Talker/Listener 验证通讯
   - turtlesim 验证完整功能
   - rqt_graph 可视化节点关系
