## 1. 通信原理

话题通信是ROS2中最基础也是最强大的通信机制。它采用**发布-订阅（Publish-Subscribe）**模式，实现了节点之间的异步数据交换。这种模式使得信息的产生者和消费者之间不需要直接连接，从而实现了松耦合的系统设计。

### 1.1 异步发布-订阅模式

在话题通信中，存在两种关键角色：**发布者（Publisher）**和**订阅者（Subscriber）**。发布者负责发送数据，而订阅者负责接收数据。两者之间没有直接的函数调用关系，而是通过ROS2中间件（DDS）进行数据传递。

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│   传感器    │         │   ROS2     │         │   数据处理   │
│   节点A     │  ────▶  │   话题     │  ────▶  │   节点B     │
│ (发布者)    │         │ /scan      │         │ (订阅者)    │
└─────────────┘         └─────────────┘         └─────────────┘
                                                        │
┌─────────────┐         ┌─────────────┐               ▼
│   传感器    │         │   ROS2     │         ┌─────────────┐
│   节点C     │  ────▶  │   话题     │  ────▶  │   显示节点   │
│ (发布者)    │         │ /scan      │         │ (订阅者)    │
└─────────────┘         └─────────────┘         └─────────────┘
```

这种模式的核心优势在于**解耦**。传感器节点不需要知道有多少个节点在接收数据，也不需要等待响应；数据处理节点可以随时订阅感兴趣的话题，而不需要修改传感器节点的代码。在机器人系统中，这种设计使得添加新的数据处理模块变得非常简单。

异步特性意味着发布者发送消息后不需要等待订阅者的响应，可以立即继续执行其他任务。这对于实时性要求高的机器人系统非常重要，因为它保证了传感器数据能够及时采集和发布，不会因为等待某个慢速处理节点而阻塞。

### 1.2 无连接通信

ROS2话题通信是基于DDS（Data Distribution Service）实现的，它是一种**无连接（Connectionless）**的通信协议。与传统的TCP连接不同，DDS使用UDP协议作为底层传输机制，发布者和订阅者之间不需要建立持久的连接。

无连接通信带来了几个重要优势。首先是**更高的可靠性**：在网络不稳定或节点频繁加入/离开的场景下，无连接通信更加健壮。即使某个节点暂时离线，其他节点仍然可以正常通信，节点重新上线后会自动恢复连接。其次是**更好的实时性**：UDP的无连接特性减少了协议开销，降低了通信延迟，这对于需要高频数据交换的机器人控制系统尤为重要。最后是**分布式优势**：无连接通信天然支持多播，可以更高效地实现一对多的数据分发。

当然，无连接通信也有其挑战。由于UDP本身不保证消息的可靠性，ROS2通过DDS的QoS机制来提供可靠性保证。此外，开发者需要在应用层面处理消息丢失和乱序的问题。这些问题我们会在后续的QoS配置部分详细讨论。

### 1.3 多对多支持

话题通信支持**多对多（Many-to-Many）**的通信模式，这是它区别于服务通信的重要特点。在一个话题上，可以有多个发布者和多个订阅者同时存在。

```
发布者侧                                    订阅者侧
   │                                          │
   ├──────────────────┐                        │
   │                  │                        │
   ▼                  ▼                        ▼
┌─────┐          ┌─────┐              ┌─────────────┐
│ P1  │          │ P2  │              │    /scan    │
└─────┘          └─────┘              └──────┬──────┘
                                             │
                         ┌────────────────────┼────────────────────┐
                         │                    │                    │
                         ▼                    ▼                    ▼
                      ┌─────┐             ┌─────┐             ┌─────┐
                      │ S1  │             │ S2  │             │ S3  │
                      └─────┘             └─────┘             └─────┘
```

多对多支持在实际应用中非常有用。例如，在一个多机器人系统中，多个机器人的激光雷达数据可以发布到同一个话题，监控节点订阅该话题即可获取所有机器人的数据；又例如，导航堆栈中的多个模块（如定位、避障、路径规划）可能都需要订阅激光雷达数据，它们都可以同时订阅同一个话题。

ROS2的DDS中间件会自动处理多个发布者和订阅者之间的数据分发。消息会被复制并发送给所有的订阅者，而发布者不需要知道有多少订阅者存在。这种机制大大简化了分布式系统的开发。

## 2. 消息类型

ROS2使用**接口定义语言（IDL）**来定义消息结构。消息类型分为两大类：标准消息和自定义消息。标准消息是ROS2官方定义的消息类型，涵盖了常见的机器人数据类型；自定义消息则允许开发者根据具体需求定义自己的消息结构。

### 2.1 标准消息（std_msgs, sensor_msgs）

ROS2提供了丰富的标准消息类型，存放在`std_msgs`和`sensor_msgs`等包中。这些消息类型经过充分测试，可以直接在你的项目中使用。

**常用std_msgs消息类型**：

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| `std_msgs/msg/String` | 字符串消息 | 文本数据传输 |
| `std_msgs/msg/Int32` | 32位整数 | 计数器、状态码 |
| `std_msgs/msg/Float64` | 64位浮点数 | 传感器原始数据 |
| `std_msgs/msg/Bool` | 布尔值 | 开关状态 |
| `std_msgs/msg/Empty` | 空消息 | 触发信号 |
| `std_msgs/msg/Header` | 标准头信息 | 时间戳、帧ID |

`Header`是一个非常常用的消息类型，几乎所有复杂的消息都会包含它：

```yaml
# std_msgs/msg/Header.msg
uint32 seq        # 消息序列号
time stamp       # 时间戳
string frame_id   # 坐标系ID
```

**常用sensor_msgs消息类型**：

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| `sensor_msgs/msg/LaserScan` | 激光雷达数据 | 2D激光测距 |
| `sensor_msgs/msg/PointCloud2` | 点云数据 | 3D环境感知 |
| `sensor_msgs/msg/Image` | 图像数据 | 相机图像 |
| `sensor_msgs/msg/Imu` | IMU数据 | 惯性测量 |
| `sensor_msgs/msg/JointState` | 关节状态 | 机器人关节 |
| `sensor_msgs/msg/BatteryState` | 电池状态 | 电源管理 |

例如，一个典型的激光雷达消息结构如下：

```yaml
# sensor_msgs/msg/LaserScan.msg
Header header            # 头信息
float32 angle_min        # 扫描起始角度（弧度）
float32 angle_max        # 扫描结束角度（弧度）
float32 angle_increment # 角度增量
float32 time_increment   # 时间增量
float32 scan_time       # 扫描时间
float32 range_min       # 最小距离
float32 range_max       # 最大距离
float32[] ranges        # 距离数据
float32[] intensities   # 强度数据
```

在实际开发中，优先使用标准消息类型是一个好习惯。标准消息具有良好的兼容性，很多工具（如rviz2、rosbag2）都原生支持这些类型，可以直接可视化或录制。

### 2.2 自定义消息（.msg文件编写）

当标准消息无法满足需求时，ROS2允许开发者创建自定义消息。自定义消息通过`.msg`文件定义，存放在功能包的`msg`目录下。

**创建自定义消息的步骤**：

1. 在功能包的`msg`目录下创建`.msg`文件
2. 在`package.xml`中添加消息依赖
3. 在`CMakeLists.txt`中配置消息生成
4. 重新编译功能包

例如，创建一个机器人状态消息：

```yaml
# msg/RobotStatus.msg
# 机器人基本状态信息

Header header                    # 时间戳和帧ID
string robot_name                # 机器人名称
uint8 state                      # 机器人状态: 0=IDLE, 1=MOVING, 2=ERROR
float32 battery_level            # 电池电量 (0.0-1.0)
float32 temperature             # 温度 (摄氏度)
geometry_msgs/Pose pose          # 当前位置
geometry_msgs/Twist velocity    # 当前速度
string[] active_modules         # 当前活跃的模块列表
```

这个自定义消息引用了`geometry_msgs`包中的类型，所以需要在`package.xml`中添加相应的依赖。

**package.xml配置**：

```xml
<package format="3">
  <name>my_robot_msgs</name>
  <version>0.1.0</version>
  <description>Custom robot messages</description>

  <depend>std_msgs</depend>
  <depend>geometry_msgs</depend>

  <member_of_group>rosidl_interface_packages</member_of_group>

  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

**CMakeLists.txt配置**：

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_robot_msgs)

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

find_package(ament_cmake REQUIRED)
find_package(std_msgs REQUIRED)
find_package(geometry_msgs REQUIRED)
find_package(rosidl_default_generators REQUIRED)

rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/RobotStatus.msg"
  DEPENDENCIES std_msgs geometry_msgs
)

ament_export_dependencies(rosidl_default_runtime)
ament_package()
```

消息文件名必须使用大驼峰命名法（PascalCase），如`RobotStatus.msg`对应类型名`my_robot_msgs/RobotStatus`。消息字段使用小写下划线命名法（snake_case）。

## 3. 跨语言实现

ROS2支持多种编程语言，不同语言的API风格各有特点，但核心概念是统一的。本节将分别介绍C++（rclcpp）和Python（rclpy）的话题通信实现。

### 3.1 C++ (rclcpp)

C++是ROS2开发的主要语言，提供了高性能的API。使用rclcpp实现话题通信需要包含相应的头文件，并遵循特定的编码模式。

**发布者实现**：

```cpp
// talker.cpp - 话题发布者示例
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>
#include <chrono>

class TalkerNode : public rclcpp::Node
{
public:
  TalkerNode() : Node("talker_node"), count_(0)
  {
    // 创建发布者
    publisher_ = this->create_publisher<std_msgs::msg::String>("chatter", 10);
    
    // 创建定时器
    timer_ = this->create_wall_timer(
      std::chrono::milliseconds(500),
      std::bind(&TalkerNode::publish_message, this)
    );
    
    RCLCPP_INFO(this->get_logger(), "Talker node started");
  }

private:
  void publish_message()
  {
    auto message = std_msgs::msg::String();
    message.data = "Hello, ROS2! Count: " + std::to_string(count_++);
    
    RCLCPP_INFO(this->get_logger(), "Publishing: '%s'", message.data.c_str());
    publisher_->publish(message);
  }

  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr publisher_;
  rclcpp::TimerBase::SharedPtr timer_;
  size_t count_;
};

int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<TalkerNode>());
  rclcpp::shutdown();
  return 0;
}
```

**订阅者实现**：

```cpp
// listener.cpp - 话题订阅者示例
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>

class ListenerNode : public rclcpp::Node
{
public:
  ListenerNode() : Node("listener_node")
  {
    // 创建订阅者
    subscription_ = this->create_subscription<std_msgs::msg::String>(
      "chatter",
      10,
      std::bind(&ListenerNode::topic_callback, this, std::placeholders::_1)
    );
    
    RCLCPP_INFO(this->get_logger(), "Listener node started");
  }

private:
  void topic_callback(const std_msgs::msg::String::SharedPtr msg) const
  {
    RCLCPP_INFO(this->get_logger(), "I heard: '%s'", msg->data.c_str());
  }

  rclcpp::Subscription<std_msgs::msg::String>::SharedPtr subscription_;
};

int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<ListenerNode>());
  rclcpp::shutdown();
  return 0;
}
```

**CMakeLists.txt配置**：

```cmake
find_package(rclcpp REQUIRED)
find_package(std_msgs REQUIRED)

add_executable(talker src/talker.cpp)
ament_target_dependencies(talker rclcpp std_msgs)

add_executable(listener src/listener.cpp)
ament_target_dependencies(listener rclcpp std_msgs)

install(TARGETS talker listener
  DESTINATION lib/${PROJECT_NAME})
```

在rclcpp中，创建发布者使用`create_publisher<TopicType>`方法，创建订阅者使用`create_subscription<TopicType>`方法。两个方法都需要指定话题名称和队列大小。回调函数是订阅消息的主要方式，使用`std::bind`或Lambda表达式绑定。

### 3.2 Python (rclpy)

Python是ROS2另一个重要的开发语言，API更加简洁，适合快速原型开发。使用rclpy实现话题通信的代码更加简洁直观。

**发布者实现**：

```python
# talker.py - 话题发布者示例
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class TalkerNode(Node):
    def __init__(self):
        super().__init__('talker_node')
        self.publisher_ = self.create_publisher(String, 'chatter', 10)
        self.timer = self.create_timer(0.5, self.publish_message)
        self.count = 0
        self.get_logger().info('Talker node started')

    def publish_message(self):
        msg = String()
        msg.data = f'Hello, ROS2! Count: {self.count}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')
        self.count += 1

def main(args=None):
    rclpy.init(args=args)
    node = TalkerNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

**订阅者实现**：

```python
# listener.py - 话题订阅者示例
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class ListenerNode(Node):
    def __init__(self):
        super().__init__('listener_node')
        self.subscription = self.create_subscription(
            String,
            'chatter',
            self.listener_callback,
            10
        )
        self.get_logger().info('Listener node started')

    def listener_callback(self, msg):
        self.get_logger().info(f'I heard: "{msg.data}"')

def main(args=None):
    rclpy.init(args=args)
    node = ListenerNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

**package.xml配置**：

```xml
<exec_depend>rclpy</exec_depend>
<exec_depend>std_msgs</exec_depend>
```

**setup.py配置**：

```python
from setuptools import setup

package_name = 'my_py_package'

setup(
    name=package_name,
    version='0.1.0',
    packages=[package_name],
    data_files=[
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='Your Name',
    maintainer_email='you@example.com',
    description='Python topic communication example',
    license='Apache-2.0',
    tests_require=['pytest'],
    entry_points={
        'console_scripts': [
            'talker = ' + package_name + '.talker:main',
            'listener = ' + package_name + '.listener:main',
        ],
    },
)
```

Python版本的代码更加简洁，发布和订阅的逻辑清晰明了。回调函数直接定义在类方法中，不需要使用`std::bind`这样的语法糖。

**运行示例**：

```bash

# 终端2：启动订阅者
ros2 run my_py_package listener

# 终端3：查看话题列表

# 终端4：查看话题数据

```

## 4. QoS配置

ROS2引入了**Quality of Service（QoS）**策略，允许开发者根据应用需求调整通信行为。QoS配置是ROS2区别于ROS1的重要改进，它提供了更灵活的消息传递机制。

### 4.1 可靠性

**可靠性（Reliability）**QoS策略控制消息传递的保证机制：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| `RELIABLE` | 保证消息可靠到达，可能重传 | 重要数据、控制指令 |
| `BEST_EFFORT` | 尽力而为，不保证到达 | 实时传感器数据、视频流 |

```cpp
// C++ 设置可靠性QoS
rclcpp::QoS qos_reliable(10);
qos_reliable.reliable();

rclcpp::QoS qos_best_effort(10);
qos_best_effort.best_effort();

auto publisher = this->create_publisher<std_msgs::msg::String>(
    "chatter", qos_reliable
);
```

```python
# Python 设置可靠性QoS
from rclpy.qos import QoSProfile, ReliabilityPolicy, DurabilityPolicy

qos_reliable = QoSProfile(
    reliability=ReliabilityPolicy.RELIABLE,
    depth=10
)

qos_best_effort = QoSProfile(
    reliability=ReliabilityPolicy.BEST_EFFORT,
    depth=10
)

publisher = self.create_publisher(String, 'chatter', qos_reliable)
```

选择合适的可靠性策略需要权衡性能和可靠性。对于激光雷达点云等高频传感器数据，选择`BEST_EFFORT`可以降低延迟，即使偶尔丢失几个数据点也不会影响整体功能；但对于控制指令或关键状态数据，`RELIABLE`模式是更好的选择。

### 4.2 耐久性

**耐久性（Durability）策略**控制消息是否会被保留以便晚启动的订阅者接收：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| `TRANSIENT_LOCAL` | 保留历史消息，新订阅者收到最后一条 | 配置参数、状态同步 |
| `VOLATILE` | 不保留消息，新订阅者无法收到历史数据 | 实时数据流 |

```cpp
// C++ 设置耐久性QoS
rclcpp::QoS qos_transient_local(10);
qos_transient_local.durability(rclcpp::DurabilityPolicy::TRANSIENT_LOCAL);

rclcpp::QoS qos_volatile(10);
qos_volatile.durability(rclcpp::DurabilityPolicy::VOLATILE);
```

```python
# Python 设置耐久性QoS
qos_transient_local = QoSProfile(
    durability=DurabilityPolicy.TRANSIENT_LOCAL,
    depth=10
)

qos_volatile = QoSProfile(
    durability=DurabilityPolicy.VOLATILE,
    depth=10
)
```

典型的应用场景是：传感器节点先启动并发布数据，监控节点后启动。如果使用`TRANSIENT_LOCAL`，后启动的监控节点会立即收到最新的传感器数据；如果使用`VOLATILE`，监控节点只能收到启动后发布的新数据。

**QoS兼容性**：发布者和订阅者的QoS配置需要兼容才能建立连接。ROS2使用"最宽松优先"的原则：当两个QoS配置不同时，取两者的交集。例如，发布者使用`RELIABLE`而订阅者使用`BEST_EFFORT`，实际通信会使用`BEST_EFFORT`模式。

```bash
# 查看话题的QoS兼容性
ros2 topic info /chatter -v
```

这个命令会显示话题的发布者和订阅者信息，以及它们各自的QoS配置。

## 5. 实战：多传感器数据采集系统

本节我们将实现一个**多传感器数据采集系统**，综合运用话题通信的各种特性。该系统包括：

- 模拟的激光雷达传感器节点
- 模拟的IMU传感器节点  
- 数据融合节点（订阅多个传感器数据）

```python
# sensor_nodes/sensor_nodes/laser_scanner.py
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import LaserScan
import math
import random

class LaserScanner(Node):
    def __init__(self):
        super().__init__('laser_scanner')
        self.publisher = self.create_publisher(LaserScan, '/scan', 10)
        self.timer = self.create_timer(0.1, self.publish_scan)
        self.get_logger().info('Laser scanner started')
        
    def publish_scan(self):
        msg = LaserScan()
        msg.header.stamp = self.get_clock().now().to_msg()
        msg.header.frame_id = 'laser_link'
        msg.angle_min = -math.pi / 2
        msg.angle_max = math.pi / 2
        msg.angle_increment = math.pi / 180  # 1度
        msg.range_min = 0.1
        msg.range_max = 10.0
        
        # 模拟扫描数据
        num_readings = int((msg.angle_max - msg.angle_min) / msg.angle_increment)
        msg.ranges = [random.uniform(0.5, 5.0) for _ in range(num_readings)]
        msg.intensities = [1.0] * num_readings
        
        self.publisher.publish(msg)

def main(args=None):
    rclpy.init(args=args)
    node = LaserScanner()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

import rclpy
from rclpy.node import Node

### 5.5 数据融合节点

```python
# sensor_nodes/sensor_nodes/fusion_node.py
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import LaserScan, Imu
from sensor_fusion.msg import RobotStatus
from rclpy.qos import QoSProfile, ReliabilityPolicy, DurabilityPolicy

class FusionNode(Node):
    def __init__(self):
        super().__init__('fusion_node')
        
        # 使用QoS配置
        qos = QoSProfile(
            reliability=ReliabilityPolicy.RELIABLE,
            durability=DurabilityPolicy.VOLATILE,
            depth=10
        )
        
        # 订阅激光雷达和IMU数据
        self.scan_sub = self.create_subscription(
            LaserScan, '/scan', self.scan_callback, qos)
        self.imu_sub = self.create_subscription(
            Imu, '/imu', self.imu_callback, qos)
        
        # 发布融合后的状态
        self.status_pub = self.create_publisher(RobotStatus, '/robot_status', 10)
        
        # 存储最新数据
        self.latest_scan = None
        self.latest_imu = None
        
        self.get_logger().info('Fusion node started')
        
    def scan_callback(self, msg):
        self.latest_scan = msg
        self.publish_status()
        
    def imu_callback(self, msg):
        self.latest_imu = msg
        
    def publish_status(self):
        if self.latest_scan is None or self.latest_imu is None:
            return
            
        msg = RobotStatus()
        msg.header.stamp = self.get_clock().now().to_msg()
        msg.header.frame_id = 'base_link'
        
        # 计算最近障碍物距离
        valid_ranges = [r for r in self.latest_scan.ranges 
                        if r > self.latest_scan.range_min]
        msg.distance_obstacle = min(valid_ranges) if valid_ranges else 0.0
        
        # 获取IMU姿态
        msg.imu_orientation_x = self.latest_imu.orientation.x
        msg.imu_orientation_y = self.latest_imu.orientation.y
        msg.imu_orientation_z = self.latest_imu.orientation.z
        
        # 碰撞警告
        msg.is_collision_warning = msg.distance_obstacle < 0.5
        
        self.status_pub.publish(msg)

def main(args=None):
    rclpy.init(args=args)
    node = FusionNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

class DisplayNode(Node):

            f'{msg.imu_orientation_y:.2f}, '
            f'{msg.imu_orientation_z:.2f}){warning}'
        )

### 选择题

1. ROS2话题通信采用的是什么模式？
   - A) 同步请求-响应模式
   - B) 异步发布-订阅模式
   - C) 同步远程过程调用
   - D) 异步客户端-服务器模式
   
   **答案：B**。ROS2话题通信采用异步发布-订阅模式，发布者和订阅者之间不需要直接连接，可以独立运行。

2. 下列哪个QoS策略适用于实时传感器数据？
   - A) RELIABLE + TRANSIENT_LOCAL
   - B) BEST_EFFORT + VOLATILE
   - C) RELIABLE + VOLATILE
   - D) BEST_EFFORT + TRANSIENT_LOCAL
   
   **答案：B**。实时传感器数据（如激光雷达点云）适合使用BEST_EFFORT（尽力而为）和VOLATILE（不保留历史），以降低延迟。

3. 在ROS2中，自定义消息文件应该放在哪个目录下？
   - A) `src/`
   - B) `include/`
   - C) `msg/`
   - D) `launch/`
   
   **答案：C**。自定义消息文件应放在功能包的`msg`目录下。
