## 1. 参数特性

参数服务器是ROS2系统中用于集中管理配置参数的基础设施。与话题和服务不同，参数服务器主要解决的是节点配置问题，而不是数据通信问题。

### 1.1 键值对存储

参数服务器采用**键值对（Key-Value）**的存储方式，每个参数由一个唯一的键（Key）和对应的值（Value）组成。键通常采用有意义的命名，如`/robot_name`、`/max_velocity`等，采用分层命名空间（Namespace）组织，便于管理和查找。

```yaml
# 参数键值对示例
/robot_name: "TurtleBot3"
/max_velocity: 0.5
/enable_lidar: true
/laser_scan_topic: "/scan"
```

这种存储方式的优势在于：
- **易于理解**：键名直接反映参数含义
- **层次化**：通过命名空间组织复杂配置
- **类型安全**：每个参数都有明确的类型

### 1.2 节点共享

参数服务器的一个核心特性是**跨节点共享**。所有连接到同一ROS2系统的节点都可以访问和修改参数服务器中的参数。这种设计使得多个节点可以共享配置信息，无需通过话题或服务传递。

```python
# 节点A设置参数
self.declare_parameter('shared_config.robot_name', 'default_bot')
robot_name = self.get_parameter('shared_config.robot_name').value

# 节点B可以读取相同的参数
robot_name = self.get_parameter('shared_config.robot_name').value
```

典型的共享场景包括：
- **机器人配置**：机器人名称、尺寸、重量等
- **算法参数**：PID控制参数、滤波系数等
- **功能开关**：是否启用某个功能模块

### 1.3 支持数据类型

ROS2参数服务器支持丰富的数据类型，满足不同配置需求：

| 类型 | 说明 | 示例 |
|------|------|------|
| `bool` | 布尔值 | `true`, `false` |
| `int` | 整数 | `1024`, `-5` |
| `double` | 浮点数 | `3.14`, `0.5` |
| `string` | 字符串 | `"robot_name"` |
| `byte[]` | 字节数组 | `[0x01, 0x02]` |
| `bool[]` | 布尔数组 | `[true, false, true]` |
| `int[]` | 整数数组 | `[1, 2, 3, 4, 5]` |
| `double[]` | 浮点数数组 | `[0.1, 0.2, 0.3]` |
| `string[]` | 字符串数组 | `["a", "b", "c"]` |

```python
# Python中声明不同类型的参数
self.declare_parameter('bool_param', True)           # 布尔
self.declare_parameter('int_param', 1024)            # 整数
self.declare_parameter('double_param', 3.14)         # 浮点
self.declare_parameter('string_param', 'robot')      # 字符串
self.declare_parameter('int_array', [1, 2, 3])      # 整数数组
self.declare_parameter('string_array', ['a', 'b'])   # 字符串数组
```

```cpp
// C++中声明不同类型的参数
this->declare_parameter<bool>("bool_param", true);
this->declare_parameter<int>("int_param", 1024);
this->declare_parameter<double>("double_param", 3.14);
this->declare_parameter<std::string>("string_param", "robot");
this->declare_parameter<std::vector<int>>("int_array", {1, 2, 3});
this->declare_parameter<std::vector<std::string>>("string_array", {"a", "b"});
```

## 2. 操作接口

ROS2提供了丰富的参数操作接口，包括参数读取、设置、列表查询和回调监听。

### 2.1 get_parameter - 获取参数

获取参数是参数服务器最常用的操作。ROS2提供了多种获取参数的方式：

```python
# 基本获取 - 同步方式
robot_name = self.get_parameter('robot_name').value
max_velocity = self.get_parameter('max_velocity').value

# 获取带默认值的参数（参数不存在时使用默认值）
laser_topic = self.get_parameter_or('laser_topic', rclpy.parameter.Parameter('laser_topic', rclpy.parameter.Parameter.Type.STRING, '/scan')).value

# 批量获取参数
params = self.get_parameters(['robot_name', 'max_velocity', 'enable_lidar'])
robot_name = params['robot_name'].value
max_velocity = params['max_velocity'].value
```

```cpp
// C++ 获取参数
this->get_parameter("robot_name", robot_name);
this->get_parameter("max_velocity", max_velocity);

// 获取带默认值的参数
std::string laser_topic;
this->get_parameter_or("laser_topic", "/scan", laser_topic);

// 批量获取
auto params = this->get_parameters({"robot_name", "max_velocity", "enable_lidar"});
auto robot_name = params[0].as_string();
auto max_velocity = params[1].as_double();
```

### 2.2 set_parameter - 设置参数

设置参数可以动态修改节点配置：

```python
# 设置单个参数
self.set_parameters([rclpy.parameter.Parameter('max_velocity', rclpy.parameter.Parameter.Type.DOUBLE, 0.8)])

# 批量设置参数
self.set_parameters([
    rclpy.parameter.Parameter('max_velocity', rclpy.parameter.Parameter.Type.DOUBLE, 0.8),
    rclpy.parameter.Parameter('enable_lidar', rclpy.parameter.Parameter.Type.BOOL, True),
    rclpy.parameter.Parameter('robot_name', rclpy.parameter.Parameter.Type.STRING, 'TurtleBot4')
])
```

```cpp
// C++ 设置参数
this->set_parameter(rclcpp::Parameter("max_velocity", 0.8));
this->set_parameter(rclcpp::Parameter("enable_lidar", true));

// 批量设置
std::vector<rclcpp::Parameter> params;
params.push_back(rclcpp::Parameter("max_velocity", 0.8));
params.push_back(rclcpp::Parameter("enable_lidar", true));
this->set_parameters(params);
```

### 2.3 list_parameters - 列出参数

列出参数可以查看当前系统中所有可用的参数：

```python
# 列出所有参数
all_params = self.list_parameters(names=[], recursive=True)
print("所有参数:", all_params.names)

# 列出特定命名空间下的参数
laser_params = self.list_parameters(names=['/laser'], recursive=True)
print("激光雷达参数:", laser_params.names)
```

```cpp
// C++ 列出参数
auto all_params = this->list_parameters({}, true);
for (const auto& name : all_params.names) {
    RCLCPP_INFO(this->get_logger(), "参数: %s", name.c_str());
}
```

### 2.4 参数回调 - 动态响应参数变化

参数回调机制允许节点在参数发生变化时自动执行相应逻辑，这是实现动态配置的基础：

```python
# Python 参数回调
import rclpy
from rclpy.node import Node
from rclpy.parameter import Parameter


class ParameterCallbackNode(Node):
    def __init__(self):
        super().__init__('parameter_callback_node')
        
        # 声明参数并添加回调
        self.declare_parameter('max_velocity', 0.5)
        self.declare_parameter('enable_debug', False)
        
        # 添加参数变化回调
        self.add_on_set_parameters_callback(self.parameter_callback)
        
        self.get_logger().info('参数回调节点已启动')

    def parameter_callback(self, params):
        """参数变化回调函数"""
        for param in params:
            if param.name == 'max_velocity':
                self.get_logger().info(f'max_velocity 已更改: {param.value}')
                # 这里可以执行参数应用逻辑
                self.apply_velocity_param(param.value)
            elif param.name == 'enable_debug':
                self.get_logger().info(f'enable_debug 已更改: {param.value}')
                # 启用/禁用调试模式
                self.set_debug_mode(param.value)
                
        return rclpy.parameter.SetParametersResult(successful=True)
    
    def apply_velocity_param(self, value):
        """应用速度参数"""
        self.get_logger().info(f'应用新速度限制: {value}')
    
    def set_debug_mode(self, enabled):
        """设置调试模式"""
        if enabled:
            self.get_logger().info('调试模式已启用')
        else:
            self.get_logger().info('调试模式已禁用')


def main(args=None):
    rclpy.init(args=args)
    node = ParameterCallbackNode()
    
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

```cpp
// C++ 参数回调
#include <rclcpp/rclcpp.hpp>
#include <rclcpp/parameter.hpp>
#include <vector>


class ParameterCallbackNode : public rclcpp::Node
{
public:
    ParameterCallbackNode() : Node("parameter_callback_node")
    {
        // 声明参数
        this->declare_parameter<double>("max_velocity", 0.5);
        this->declare_parameter<bool>("enable_debug", false);
        
        // 添加参数变化回调
        param_callback_handle_ = this->add_on_set_parameters_callback(
            std::bind(&ParameterCallbackNode::parameter_callback, this, std::placeholders::_1)
        );
        
        RCLCPP_INFO(this->get_logger(), "参数回调节点已启动");
    }

private:
    rclcpp::OnSetParametersCallbackHandle::SharedPtr param_callback_handle_;

    rcl_interfaces::msg::SetParametersResult parameter_callback(
        const std::vector<rclcpp::Parameter> & params)
    {
        rcl_interfaces::msg::SetParametersResult result;
        result.successful = true;
        result.reason = "OK";

        for (const auto & param : params) {
            if (param.get_name() == "max_velocity") {
                RCLCPP_INFO(this->get_logger(), "max_velocity 已更改: %.2f", param.as_double());
                apply_velocity_param(param.as_double());
            } else if (param.get_name() == "enable_debug") {
                RCLCPP_INFO(this->get_logger(), "enable_debug 已更改: %s", 
                           param.as_bool() ? "true" : "false");
                set_debug_mode(param.as_bool());
            }
        }

        return result;
    }

    void apply_velocity_param(double value)
    {
        RCLCPP_INFO(this->get_logger(), "应用新速度限制: %.2f", value);
    }

    void set_debug_mode(bool enabled)
    {
        if (enabled) {
            RCLCPP_INFO(this->get_logger(), "调试模式已启用");
        } else {
            RCLCPP_INFO(this->get_logger(), "调试模式已禁用");
        }
    }
};

int main(int argc, char ** argv)
{
    rclcpp::init(argc, argv);
    auto node = std::make_shared<ParameterCallbackNode>();
    
    rclcpp::spin(node);
    rclcpp::shutdown();
    return 0;
}
```

## 3. 动态配置

动态配置是ROS2的一个重要特性，允许在节点运行时修改参数而无需重启节点。这对于调试、参数调优和运行时适应非常有用。

### 3.1 rqt_reconfigure - 图形化动态配置

`rqt_reconfigure`是ROS2提供的图形化动态配置工具，可以通过GUI实时修改运行中节点的参数。

**安装和启动**：

```bash
# 安装 rqt_reconfigure（通常随桌面版ROS2安装）
sudo apt install ros-humble-rqt-reconfigure

# 启动动态配置GUI
ros2 run rqt_reconfigure rqt_reconfigure
```

**使用步骤**：

1. 启动需要动态配置的节点（节点需要有声明的参数）
2. 在终端运行 `ros2 run rqt_reconfigure rqt_reconfigure`
3. 在GUI中选择对应的节点
4. 修改参数值，实时生效

**示例节点**：

```python
# dynamic_config_node.py
import rclpy
from rclpy.node import Node
from rclpy.parameter import Parameter


class DynamicConfigNode(Node):
    """支持动态配置的示例节点"""
    
    def __init__(self):
        super().__init__('dynamic_config_node')
        
        # 声明可动态配置的参数
        self.declare_parameter('robot_name', 'TurtleBot3')
        self.declare_parameter('max_linear_speed', 0.5)    # 最大线速度
        self.declare_parameter('max_angular_speed', 1.0)    # 最大角速度
        self.declare_parameter('wheel_separation', 0.3)     # 轮间距
        self.declare_parameter('wheel_radius', 0.1)         # 轮半径
        self.declare_parameter('enable_debug', False)       # 调试模式
        self.declare_parameter('update_rate', 10.0)        # 更新频率
        
        # 添加参数回调
        self.add_on_set_parameters_callback(self.parameter_callback)
        
        self.get_logger().info('动态配置节点已启动')
        self.get_logger().info('使用 ros2 run rqt_reconfigure rqt_reconfigure 进行动态配置')

    def parameter_callback(self, params):
        """处理参数变化"""
        for param in params:
            self.get_logger().info(f'参数 {param.name} 从之前值更新为 {param.value}')
        
        return Parameter.SetParametersResult(successful=True)
    
    def get_config(self):
        """获取当前配置"""
        return {
            'robot_name': self.get_parameter('robot_name').value,
            'max_linear_speed': self.get_parameter('max_linear_speed').value,
            'max_angular_speed': self.get_parameter('max_angular_speed').value,
            'wheel_separation': self.get_parameter('wheel_separation').value,
            'wheel_radius': self.get_parameter('wheel_radius').value,
            'enable_debug': self.get_parameter('enable_debug').value,
            'update_rate': self.get_parameter('update_rate').value,
        }


def main(args=None):
    rclpy.init(args=args)
    node = DynamicConfigNode()
    
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 3.2 程序内动态更新

除了使用rqt_reconfigure，还可以通过编程方式在程序内部动态更新参数：

```python
# 程序内动态更新参数
import rclpy
from rclpy.node import Node


class ParameterManager(Node):
    """参数管理器示例"""
    
    def __init__(self):
        super().__init__('parameter_manager')
        
        # 声明参数
        self.declare_parameter('target_velocity', 0.5)
        self.declare_parameter('kp', 1.0)
        self.declare_parameter('ki', 0.1)
        self.declare_parameter('kd', 0.05)
        
        # 创建定时器周期性调整参数
        self.timer = self.create_timer(5.0, self.adjust_parameters)
        
        self.get_logger().info('参数管理器已启动')

    def adjust_parameters):
        """动态调整参数示例"""
        # 根据某些条件动态调整参数
        current_time = self.get_clock().now().seconds_nanoseconds()[0]
        
        if current_time % 10 == 0:
            # 动态调整速度
            new_velocity = 0.5 + (current_time % 20) / 20.0
            self.set_parameters([
                rclpy.parameter.Parameter('target_velocity', rclpy.parameter.Parameter.Type.DOUBLE, new_velocity)
            ])
            self.get_logger().info(f'动态调整 target_velocity 为 {new_velocity}')
            
        if current_time % 15 == 0:
            # 动态调整PID参数
            new_kp = 1.0 + (current_time % 10) / 10.0
            self.set_parameters([
                rclpy.parameter.Parameter('kp', rclpy.parameter.Parameter.Type.DOUBLE, new_kp)
            ])
            self.get_logger().info(f'动态调整 kp 为 {new_kp}')


def main(args=None):
    rclpy.init(args=args)
    node = ParameterManager()
    
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

**动态配置的实际应用场景**：

1. **PID参数调优**：在控制过程中实时调整PID参数，观察控制效果
2. **速度限制调整**：根据环境变化动态调整机器人速度
3. **调试模式开关**：运行时启用/禁用调试输出
4. **滤波器参数调整**：根据传感器噪声特性调整滤波参数

## 4. 参数持久化

参数持久化允许将参数保存到文件，以便在节点重启后自动恢复配置。这对于保持机器人配置一致性非常有用。

### 4.3 程序保存参数

```python
# 保存参数到YAML文件
import yaml
import rclpy
from rclpy.node import Node


class ParameterPersistence(Node):
    """参数持久化示例"""
    
    def __init__(self):
        super().__init__('parameter_persistence')
        
        # 声明参数
        self.declare_parameter('config_file', 'robot_config.yaml')
        
    def save_parameters(self, filename):
        """保存当前参数到文件"""
        # 获取所有参数
        param_names = self.list_parameters(names=[], recursive=True).names
        
        params_dict = {}
        for name in param_names:
            param = self.get_parameter(name)
            # 简化处理：只处理基本类型
            if param.type_ == Parameter.Type.STRING:
                params_dict[name] = param.value
            elif param.type_ == Parameter.Type.DOUBLE:
                params_dict[name] = param.value
            elif param.type_ == Parameter.Type.INTEGER:
                params_dict[name] = param.value
            elif param.type_ == Parameter.Type.BOOL:
                params_dict[name] = param.value
        
        # 写入YAML文件
        with open(filename, 'w') as f:
            yaml.dump(params_dict, f, default_flow_style=False)
        
        self.get_logger().info(f'参数已保存到 {filename}')
    
    def load_parameters(self, filename):
        """从文件加载参数"""
        with open(filename, 'r') as f:
            params_dict = yaml.safe_load(f)
        
        # 转换为ROS2参数格式
        params = []
        for name, value in params_dict.items():
            param_type = type(value).__name__
            if param_type == 'str':
                params.append(rclpy.parameter.Parameter(name, rclpy.parameter.Parameter.Type.STRING, value))
            elif param_type in ['int', 'int32', 'int64']:
                params.append(rclpy.parameter.Parameter(name, rclpy.parameter.Parameter.Type.INTEGER, value))
            elif param_type in ['float', 'float64']:
                params.append(rclpy.parameter.Parameter(name, rclpy.parameter.Parameter.Type.DOUBLE, value))
            elif param_type == 'bool':
                params.append(rclpy.parameter.Parameter(name, rclpy.parameter.Parameter.Type.BOOL, value))
        
        # 设置参数
        self.set_parameters(params)
        self.get_logger().info(f'参数已从 {filename} 加载')


def main(args=None):
    rclpy.init(args=args)
    node = ParameterPersistence()
    
    # 示例：保存参数
    node.save_parameters('/tmp/robot_config.yaml')
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 5.2 参数服务器节点

```python
# robot_param_server.py
import rclpy
from rclpy.node import Node
from rclpy.parameter import Parameter


class RobotParamServer(Node):
    """机器人参数服务器 - 集中管理所有机器人参数"""
    
    def __init__(self):
        super().__init__('robot_param_server')
        
        self.get_logger().info('机器人参数服务器启动中...')
        
        # ===== 机器人基础参数 =====
        self.declare_parameter('robot.name', 'TurtleBot3')
        self.declare_parameter('robot.model', 'Burger')
        self.declare_parameter('robot.id', 0)
        
        # ===== 运动参数 =====
        self.declare_parameter('motion.max_linear_speed', 0.5)      # 最大线速度 m/s
        self.declare_parameter('motion.max_angular_speed', 1.0)    # 最大角速度 rad/s
```

```python
# robot_controller.py
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist


class RobotController(Node):
    """机器人控制器 - 使用参数服务器的配置"""
    
    def __init__(self):
        super().__init__('robot_controller')
        
        self.get_logger().info('机器人控制器启动中...')
        
        # 从参数服务器获取配置
        self.max_linear_speed = self.get_parameter('motion.max_linear_speed').value
        self.max_angular_speed = self.get_parameter('motion.max_angular_speed').value
        self.wheel_separation = self.get_parameter('motion.wheel_separation').value
        self.wheel_radius = self.get_parameter('motion.wheel_radius').value
        
        # PID参数
        self.linear_kp = self.get_parameter('pid.linear.kp').value
        self.linear_ki = self.get_parameter('pid.linear.ki').value
        self.linear_kd = self.get_parameter('pid.linear.kd').value
        
        # 打印初始配置
        self.get_logger().info(f'最大线速度: {self.max_linear_speed} m/s')
        self.get_logger().info(f'最大角速度: {self.max_angular_speed} rad/s')
        self.get_logger().info(f'轮间距: {self.wheel_separation} m')
        self.get_logger().info(f'轮半径: {self.wheel_radius} m')
        
        # 创建发布者
        self.cmd_vel_pub = self.create_publisher(Twist, '/cmd_vel', 10)
        
        # 定时器 - 定期检查参数变化
        self.timer = self.create_timer(1.0, self.check_parameters)
        
        # PID控制器状态
        self.target_linear = 0.0
        self.target_angular = 0.0
        
        self.get_logger().info('机器人控制器已就绪')

    def check_parameters(self):
        """检查参数变化并更新"""
        # 检查关键参数
        new_max_linear = self.get_parameter('motion.max_linear_speed').value
        if new_max_linear != self.max_linear_speed:
            self.get_logger().info(f'最大线速度已更新: {self.max_linear_speed} -> {new_max_linear}')
            self.max_linear_speed = new_max_linear
        
        new_max_angular = self.get_parameter('motion.max_angular_speed').value
        if new_max_angular != self.max_angular_speed:
            self.get_logger().info(f'最大角速度已更新: {self.max_angular_speed} -> {new_max_angular}')
            self.max_angular_speed = new_max_angular
        
        # 检查PID参数
        new_kp = self.get_parameter('pid.linear.kp').value
        if new_kp != self.linear_kp:
            self.get_logger().info(f'Linear KP 更新: {self.linear_kp} -> {new_kp}')
            self.linear_kp = new_kp
        
        # 检查功能开关
        enable_debug = self.get_parameter('features.enable_debug').value
        if enable_debug:
            self.get_logger().debug('调试模式已启用')

    def set_velocity(self, linear, angular):
        """设置目标速度（带限幅）"""
        # 限幅
        linear = max(-self.max_linear_speed, min(self.max_linear_speed, linear))
        angular = max(-self.max_angular_speed, min(self.max_angular_speed, angular))
        
        self.target_linear = linear
        self.target_angular = angular
        
        # 发布速度命令
        cmd = Twist()
        cmd.linear.x = linear
        cmd.angular.z = angular
        self.cmd_vel_pub.publish(cmd)


def main(args=None):
    rclpy.init(args=args)
    controller = RobotController()
    
    # 示例：设置速度
    controller.set_velocity(0.3, 0.0)  # 前进 0.3 m/s
    
    try:
        rclpy.spin(controller)
    except KeyboardInterrupt:
        pass
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 基础题

1. **参数服务器的主要特性是什么？**
   
   答案：参数服务器采用键值对存储，支持节点间共享配置，支持多种数据类型（bool、int、double、string、数组等）。

2. **ROS2参数服务器支持哪些基本数据类型？**
   
   答案：支持bool、int、double、string、byte[]以及对应的数组类型（bool[]、int[]、double[]、string[]）。

3. **如何获取参数的值？列举至少两种方法。**
   
   答案：
   - 使用`get_parameter('param_name').value`获取单个参数
   - 使用`get_parameter_or('param_name', default_value)`获取带默认值的参数
   - 使用`get_parameters(['param1', 'param2'])`批量获取参数

### 进阶题

4. **如何实现参数变化时的回调处理？请给出Python示例。**
   
   答案：
   ```python
   # 声明参数
   self.declare_parameter('max_velocity', 0.5)
   
   # 添加参数回调
   self.add_on_set_parameters_callback(self.parameter_callback)
   
   def parameter_callback(self, params):
       for param in params:
           self.get_logger().info(f'{param.name} 更新为 {param.value}')
       return rclpy.parameter.SetParametersResult(successful=True)
   ```

5. **参数持久化的意义是什么？如何实现？**
   
   答案：参数持久化可以将参数保存到YAML文件，使得节点重启后能自动恢复配置。实现方式包括：
   - 使用launch文件的`parameters`参数加载YAML文件
   - 编写代码将参数写入YAML文件保存

### 实战题

6. **设计一个移动机器人参数管理系统，包含哪些核心参数？请列出并说明其作用。**
   
   答案：
   - 机器人基础参数：名称、型号、ID

   - PID参数：线速度和角速度的kp、ki、kd

   - 功能开关：调试模式、避障功能、定位功能

   - 使用rqt_reconfigure工具实时修改参数

   - 或者使用定时器定期检查参数变化
