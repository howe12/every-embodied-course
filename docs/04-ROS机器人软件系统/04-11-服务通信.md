## 1. 通信原理

服务通信是ROS2中实现**同步请求-响应**模式的通信机制。与话题通信的异步发布-订阅模式不同，服务通信允许客户端发送请求后阻塞等待服务端的响应，这种模式非常适合需要立即获取结果的操作，如查询传感器状态、执行控制命令等。

### 1.1 同步请求-响应模式

服务通信采用典型的**客户端-服务端（C/S）**架构：

```
客户端                                    服务端
  │                                        │
  │───── 请求 (Request) ─────────────────▶│
  │                                        │
  │                                        ▼
  │                                        │
  │◀───── 响应 (Response) ────────────────│
  │                                        │
```

**工作流程**：

1. **服务端启动**：服务端节点创建服务，等待客户端请求
2. **客户端发送请求**：客户端调用服务，发送请求数据
3. **等待响应**：客户端阻塞等待（同步调用）或继续执行（异步调用）
4. **服务端处理**：服务端接收请求，执行处理逻辑
5. **返回响应**：服务端将处理结果返回给客户端
6. **客户端接收**：客户端获得响应，继续后续操作

**适用场景**：

| 场景 | 适用性 | 原因 |
|------|--------|------|
| 查询传感器数据 | ✅ 适合 | 需要即时获取当前状态 |
| 控制机器人执行动作 | ✅ 适合 | 需要确认动作是否执行 |
| 读取配置参数 | ✅ 适合 | 需要返回配置值 |
| 发布传感器数据 | ❌ 不适合 | 应使用话题通信 |
| 播放音频/视频 | ❌ 不适合 | 应使用话题通信 |

**与话题通信对比**：

| 特性 | 话题通信 | 服务通信 |
|------|----------|----------|
| 通信模式 | 发布-订阅 | 请求-响应 |
| 同步性 | 异步 | 同步 |
| 数据流向 | 单向 | 双向 |
| 响应性 | 无响应 | 有响应 |
| 持续性 | 持续发布 | 按需调用 |
| 适用场景 | 持续数据流 | 短时任务 |

### 1.2 有连接通信

服务通信是**有连接**的通信方式，这意味着每次服务调用都会建立专门的通信连接。

**连接建立过程**：

```
1. 服务注册
   服务端向ROS2 DDS中间件注册服务名称和类型
   
2. 服务发现
   客户端通过服务名称查找对应的服务端
   
3. 建立连接
   DDS为客户端和服务端建立专用通信通道
   
4. 数据交换
   通过建立的连接进行请求-响应数据交换
   
5. 释放连接
   响应完成后，连接可以被释放或保持
```

**连接特点**：

1. **一对一通信**：每个服务调用建立客户端与服务端之间的直接连接
2. **阻塞等待**：同步调用时，客户端会阻塞等待服务端响应
3. **短连接**：每次请求-响应完成后可以释放连接（相比动作的长时间连接）
4. **可靠性**：ROS2 DDS提供可靠传输保证

**服务质量（QoS）配置**：

```cpp
// C++ - 配置服务QoS
rclcpp::QoS qos_profile(10);
qos_profile.reliability(RMW_QOS_POLICY_RELIABILITY_RELIABLE);
qos_profile.durability(RMW_QOS_POLICY_DURABILITY_TRANSIENT_LOCAL);

// 创建服务端时应用QoS
auto service = this->create_service<MyService>(
    "service_name",
    callback,
    qos_profile
);
```

```python
# Python - 配置服务QoS
from rclpy.qos import QoSProfile, ReliabilityPolicy, DurabilityPolicy

qos_profile = QoSProfile(
    depth=10,
    reliability=ReliabilityPolicy.RELIABLE,
    durability=DurabilityPolicy.TRANSIENT_LOCAL
)

# 创建服务端时应用QoS
service = self.create_service(
    MyService,
    'service_name',
    callback,
    qos_profile=qos_profile
)
```

## 2. 服务类型

ROS2服务通信使用服务类型（Service Type）定义请求和响应的数据结构。与话题类型类似，服务类型也使用`.srv`文件定义，但包含两个部分：请求（Request）和响应（Response）。

### 2.1 标准服务（std_srvs）

ROS2提供了标准服务类型，存放在`std_srvs`包中。最常用的是`Empty`和`Trigger`服务。

**Empty 服务**（无请求无响应）：

```srv
---
```

**Trigger 服务**（标准触发请求-响应）：

```srv
---
bool success   # 执行是否成功
string message # 状态消息
```

**使用标准服务**：

```cpp
// C++ - 使用 std_srvs/srv/Empty
#include <std_srvs/srv/empty.hpp>

// 服务端
auto service = this->create_service<std_srvs::srv::Empty>(
    "reset",
    [](const std::shared_ptr<std_srvs::srv::Empty::Request> request,
       std::shared_ptr<std_srvs::srv::Empty::Response> response) {
        // 重置系统
        RCLCPP_INFO(this->get_logger(), "系统已重置");
    }
);
```

```python
# Python - 使用 std_srvs/srv/Empty
from std_srvs.srv import Empty

# 服务端
def reset_callback(self, request, response):
    # 重置系统
    self.get_logger().info('系统已重置')
    return response

self.reset_service = self.create_service(Empty, 'reset', reset_callback)
```

```cpp
// C++ - 使用 std_srvs/srv/Trigger
#include <std_srvs/srv/trigger.hpp>

// 服务端
auto service = this->create_service<std_srvs::srv::Trigger>(
    "start_process",
    [](const std::shared_ptr<std_srvs::srv::Trigger::Request> request,
       std::shared_ptr<std_srvs::srv::Trigger::Response> response) {
        // 启动处理
        response->success = true;
        response->message = "处理已启动";
    }
);
```

```python
# Python - 使用 std_srvs/srv/Trigger
from std_srvs.srv import Trigger

# 服务端
def start_process_callback(self, request, response):
    # 启动处理
    response.success = True
    response.message = "处理已启动"
    return response

self.start_service = self.create_service(Trigger, 'start_process', start_process_callback)
```

**常用标准服务类型**：

| 服务类型 | 包 | 用途 |
|----------|-----|------|
| std_srvs/Empty | std_srvs | 无参数的服务调用 |
| std_srvs/Trigger | std_srvs | 标准触发操作 |
| std_srvs/SetBool | std_srvs | 布尔参数控制 |

### 2.2 自定义服务（.srv文件）

自定义服务允许我们定义特定应用场景的请求和响应数据结构。

#### 2.2.1 .srv文件语法

`.srv`文件使用特定语法，包含三部分：请求（Request）、分隔线、响应（Response）。

```srv
# MyCustomService.srv

# ===== 请求部分 =====
# 注释以 # 开头
string name      # 字符串类型
int32 age        # 整型
float64 height   # 浮点型
bool enabled     # 布尔型
---
# ===== 响应部分 =====
bool success     # 操作是否成功
string message   # 返回消息
int32 result_code # 结果代码
```

**支持的数据类型**：

| 类型 | 说明 |
|------|------|
| bool | 布尔值 |
| byte | 8位无符号整数 |
| char | 8位有符号整数 |
| int8 | 8位整数 |
| uint8 | 8位无符号整数 |
| int16 | 16位整数 |
| uint16 | 16位无符号整数 |
| int32 | 32位整数 |
| uint32 | 32位无符号整数 |
| int64 | 64位整数 |
| uint64 | 64位无符号整数 |
| float32 | 32位浮点数 |
| float64 | 64位浮点数 |
| string | 字符串 |
| time | 时间类型（sec/nsec） |
| duration | 持续时间类型 |

**嵌套类型**：

```srv
# 嵌套使用其他消息类型
geometry_msgs/PoseStamped target_pose
sensor_msgs/Image camera_image
---
bool success
string message
```

**数组类型**：

```srv
# 固定长度数组
int32[5] coordinates
# 可变长度数组
string[] names
---
int32[] results
```

#### 2.2.3 使用自定义服务

```cpp
// C++ - 使用自定义服务
#include "my_package/srv/robot_control.hpp"

class RobotControlServer : public rclcpp::Node
{
public:
    RobotControlServer() : Node("robot_control_server")
    {
        service_ = this->create_service<my_package::srv::RobotControl>(
            "robot_control",
            std::bind(&RobotControlServer::handle_control, this, 
                     std::placeholders::_1, std::placeholders::_2)
        );
    }

private:
    rclcpp_service<my_package::srv::RobotControl>::SharedPtr service_;

    void handle_control(
        const std::shared_ptr<my_package::srv::RobotControl::Request> request,
        std::shared_ptr<my_package::srv::RobotControl::Response> response)
    {
        RCLCPP_INFO(this->get_logger(), "收到命令: %s", request->command.c_str());
        
        if (request->command == "move") {
            response->success = true;
            response->message = "机器人开始移动";
            response->error_code = 0;
        } else if (request->command == "stop") {
            response->success = true;
            response->message = "机器人已停止";
            response->error_code = 0;
        } else {
            response->success = false;
            response->message = "无效命令";
            response->error_code = 1;
        }
    }
};
```

```python
# Python - 使用自定义服务
from my_package.srv import RobotControl

class RobotControlServer(Node):

            RobotControl,

    def handle_control_callback(self, request, response):
        self.get_logger().info(f'收到命令: {request.command}')
        
        if request.command == 'move':
            response.success = True
            response.message = "机器人开始移动"
            response.error_code = 0
        elif request.command == 'stop':
            response.success = True
            response.message = "机器人已停止"
            response.error_code = 0
        else:
            response.success = False
            response.message = "无效命令"
            response.error_code = 1
            
        return response
```

## 3. 调用方式

ROS2服务支持两种调用方式：**同步调用**和**异步调用**。选择合适的调用方式取决于应用场景的需求。

### 3.1 同步调用

同步调用是最直接的调用方式，客户端发送请求后会阻塞等待服务端响应，直到收到响应或超时才会继续执行。

**特点**：
- 简单直观，符合常规编程思维
- 请求-响应顺序明确
- 适合短时、确定性的服务调用

**C++ 同步调用**：

```cpp
// client_sync.cpp
#include <rclcpp/rclcpp.hpp>
#include <example_interfaces/srv/add_two_ints.hpp>

class SyncClient : public rclcpp::Node
{
public:
    SyncClient() : Node("sync_client")
    {
        client_ = this->create_client<example_interfaces::srv::AddTwoInts>("add_two_ints");
    }

    bool call_service(int a, int b)
    {
        // 等待服务可用
        while (!client_->wait_for_service(std::chrono::seconds(1))) {
            if (!rclcpp::ok()) {
                RCLCPP_ERROR(this->get_logger(), "等待服务时中断");
                return false;
            }
            RCLCPP_INFO(this->get_logger(), "等待服务上线...");
        }

        // 创建请求
        auto request = std::make_shared<example_interfaces::srv::AddTwoInts::Request>();
        request->a = a;
        request->b = b;

        // 同步调用（等待响应）
        auto future = client_->async_send_request(request);
        
        try {
            // 等待并获取响应
            auto response = rclcpp::future_until(future, std::chrono::seconds(5));
            if (response.get()) {
                RCLCPP_INFO(this->get_logger(), "结果: %ld", response.get()->sum);
                return true;
            }
        } catch (const std::exception& e) {
            RCLCPP_ERROR(this->get_logger(), "服务调用失败: %s", e.what());
        }
        
        return false;
    }

private:
    rclcpp_client<example_interfaces::srv::AddTwoInts>::SharedPtr client_;
};

int main(int argc, char ** argv)
{
    rclcpp::init(argc, argv);
    auto node = std::make_shared<SyncClient>();
    
    // 调用服务
    bool result = node->call_service(10, 20);
    
    rclcpp::shutdown();
    return result ? 0 : 1;
}
```

**Python 同步调用**：

```python
# client_sync.py
import rclpy
from rclpy.node import Node
from example_interfaces.srv import AddTwoInts


class SyncClient(Node):
    def __init__(self):
        super().__init__('sync_client')
        self.client = self.create_client(AddTwoInts, 'add_two_ints')

    def call_service(self, a, b):
        # 等待服务可用
        while not self.client.wait_for_service(timeout_sec=1.0):
            if not rclpy.ok():
                self.get_logger().error('等待服务时中断')
                return False
            self.get_logger().info('等待服务上线...')

        # 创建请求
        request = AddTwoInts.Request()
        request.a = a
        request.b = b

        # 同步调用
        future = self.client.call_async(request)
        
        try:
            # 等待响应（阻塞）
            rclpy.spin_until_future_complete(self, future, timeout_sec=5.0)
            
            if future.done():
                response = future.result()
                self.get_logger().info(f'结果: {response.sum}')
                return True
        except Exception as e:
            self.get_logger().error(f'服务调用失败: {e}')
        
        return False


def main(args=None):
    rclpy.init(args=args)
    node = SyncClient()
    
    # 调用服务
    result = node.call_service(10, 20)
    
    rclpy.shutdown()
    return 0 if result else 1


if __name__ == '__main__':
    main()
```

### 3.2 异步调用

异步调用允许客户端发送请求后不阻塞等待，而是继续执行其他任务，通过回调函数处理响应。

**特点**：
- 非阻塞，提高程序并发性
- 适合需要同时调用多个服务或处理多个任务的场景
- 需要使用回调函数处理响应

**C++ 异步调用**：

```cpp
// client_async.cpp
#include <rclcpp/rclcpp.hpp>
#include <example_interfaces/srv/add_two_ints.hpp>

class AsyncClient : public rclcpp::Node
{
public:
    AsyncClient() : Node("async_client")
    {
        client_ = this->create_client<example_interfaces::srv::AddTwoInts>("add_two_ints");
    }

    void call_service_async(int a, int b)
    {
        // 等待服务可用
        while (!client_->wait_for_service(std::chrono::seconds(1))) {
            if (!rclcpp::ok()) {
                RCLCPP_ERROR(this->get_logger(), "等待服务时中断");
                return;
            }
            RCLCPP_INFO(this->get_logger(), "等待服务上线...");
        }

        // 创建请求
        auto request = std::make_shared<example_interfaces::srv::AddTwoInts::Request>();
        request->a = a;
        request->b = b;

        // 异步调用
        auto future = client_->async_send_request(
            request,
            std::bind(&AsyncClient::response_callback, this, std::placeholders::_1)
        );
        
        RCLCPP_INFO(this->get_logger(), "请求已发送，等待响应...");
    }

private:
    rclcpp_client<example_interfaces::srv::AddTwoInts>::SharedPtr client_;

    // 响应回调函数
    void response_callback(rclcpp::Client<example_interfaces::srv::AddTwoInts>::SharedFuture future)
    {
        try {
            auto response = future.get();
            RCLCPP_INFO(this->get_logger(), "异步结果: %ld", response->sum);
        } catch (const std::exception& e) {
            RCLCPP_ERROR(this->get_logger(), "服务调用失败: %s", e.what());
        }
    }
};

int main(int argc, char ** argv)
{
    rclcpp::init(argc, argv);
    auto node = std::make_shared<AsyncClient>();
    
    // 异步调用服务
    node->call_service_async(10, 20);
    
    // 处理回调
    rclcpp::spin(node);
    
    rclcpp::shutdown();
    return 0;
}
```

**Python 异步调用**：

```python
# client_async.py
import rclpy
from rclpy.node import Node
from rclpy.callback_groups import ReentrantCallbackGroup
from example_interfaces.srv import AddTwoInts


class AsyncClient(Node):
    def __init__(self):
        super().__init__('async_client')
        
        # 创建回调组以支持多线程
        self.callback_group = ReentrantCallbackGroup()
        
        self.client = self.create_client(
            AddTwoInts, 
            'add_two_ints',
            callback_group=self.callback_group
        )

    def call_service_async(self, a, b):
        # 等待服务可用
        while not self.client.wait_for_service(timeout_sec=1.0):
            if not rclpy.ok():
                self.get_logger().error('等待服务时中断')
                return
            self.get_logger().info('等待服务上线...')

        # 创建请求
        request = AddTwoInts.Request()
        request.a = a
        request.b = b

        # 异步调用 - 传入回调函数
        future = self.client.call_async(request)
        future.add_done_callback(self.response_callback)
        
        self.get_logger().info('请求已发送，等待响应...')

    def response_callback(self, future):
        """响应回调函数"""
        try:
            response = future.result()
            self.get_logger().info(f'异步结果: {response.sum}')
        except Exception as e:
            self.get_logger().error(f'服务调用失败: {e}')


def main(args=None):
    rclpy.init(args=args)
    node = AsyncClient()
    
    # 异步调用服务
    node.call_service_async(10, 20)
    
    # 处理回调
    rclpy.spin(node)
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 3.3 同步 vs 异步对比

| 特性 | 同步调用 | 异步调用 |
|------|----------|----------|
| 阻塞行为 | 阻塞等待响应 | 非阻塞，立即返回 |
| 代码复杂度 | 简单 | 稍复杂（需回调） |
| 并发能力 | 低 | 高 |
| 适用场景 | 简单流程、顺序执行 | 并发任务、实时响应 |
| 资源利用 | 等待时CPU空闲 | 可处理其他任务 |

## 4. 异常处理

服务通信过程中可能出现各种异常情况，如服务不可用、超时、服务端错误等。良好的异常处理机制是构建健壮机器人应用的关键。

### 4.1 服务不可用

当客户端尝试连接不存在的服务时，会抛出异常或返回错误。

**C++ 处理**：

```cpp
void handle_service_unavailable()
{
    auto client = this->create_client<MyService>("my_service");
```

### 4.2 调用超时

网络延迟或服务端处理时间过长可能导致调用超时。

**C++ 超时处理**：

```cpp
#include <future>

bool call_with_timeout(rclcpp::Client<MyService>::SharedPtr client, 
                       std::shared_ptr<MyService::Request> request,
                       int timeout_seconds)
{
    auto future = client->async_send_request(request);
    
    // 使用std::future_status检测超时
    auto status = future.wait_for(std::chrono::seconds(timeout_seconds));
    
    if (status == std::future_status::timeout) {
        RCLCPP_ERROR(this->get_logger(), "服务调用超时（%d秒）", timeout_seconds);
        return false;
    } else if (status == std::future_status::ready) {
        try {
            auto response = future.get();
            return response->success;
        } catch (const std::exception& e) {
            RCLCPP_ERROR(this->get_logger(), "调用失败: %s", e.what());
            return false;
        }
    }
    
    return false;
}
```

### 4.4 重试机制

对于可能瞬时失败的服务调用，实现重试机制可以提高系统可靠性。

**C++ 重试机制**：

```cpp
#include <thread>
#include <chrono>

bool call_with_retry(rclcpp::Client<MyService>::SharedPtr client,
                     std::shared_ptr<MyService::Request> request,
                     int max_retries = 3,
                     int retry_delay_ms = 1000)
{
    for (int attempt = 0; attempt < max_retries; attempt++) {
        try {
            auto future = client->async_send_request(request);
            
            if (rclcpp::future_until(future, std::chrono::seconds(5))) {
                auto response = future.get();
                if (response->success) {
                    return true;
                }
            }
            
        } catch (const std::exception& e) {
            RCLCPP_WARN(this->get_logger(), "尝试 %d/%d 失败: %s", 
                       attempt + 1, max_retries, e.what());
        }
        
        // 等待后重试
        if (attempt < max_retries - 1) {
            RCLCPP_INFO(this->get_logger(), "等待 %dms 后重试...", retry_delay_ms);
            std::this_thread::sleep_for(std::chrono::milliseconds(retry_delay_ms));
        }
    }
    
    RCLCPP_ERROR(this->get_logger(), "达到最大重试次数");
    return false;
}
```

```cpp
            robot_status_ = "moving";

                stop_cmd.angular.z = 0.0;
```

```python
                return False
            self.get_logger().info('等待控制服务上线...')

        # 创建请求
        request = RobotControl.Request()
        request.command = command
        request.speed = speed
        request.duration = duration

        # 发送请求
        future = self.control_client.call_async(request)
        
        # 等待响应
        try:
            rclpy.spin_until_future_complete(self, future, timeout_sec=5.0)
            
            if future.done() and future.result():
                response = future.result()
                self.get_logger().info(f'响应: {response.message}, 状态: {response.robot_status}')
                return response.success
        except Exception as e:
            self.get_logger().error(f'调用失败: {e}')
        
        return False

    def get_status(self):
        # 等待服务可用
        while not self.status_client.wait_for_service(timeout_sec=1.0):
            if not rclpy.ok():
                self.get_logger().error('等待服务时中断')
                return False
            self.get_logger().info('等待状态服务上线...')

        # 创建空请求
        request = RobotStatus.Request()

        # 发送请求
        future = self.status_client.call_async(request)
        
        # 等待响应
        try:
            rclpy.spin_until_future_complete(self, future, timeout_sec=5.0)
            
            if future.done() and future.result():
                response = future.result()
                self.get_logger().info(
                    f'状态: {response.status}, 位置: ({response.position.position.x:.2f}, '
                    f'{response.position.position.y:.2f}), 电量: {response.battery_level:.1f}%'
                )
                return True
        except Exception as e:
            self.get_logger().error(f'查询失败: {e}')
        
        return False


def main(args=None):
    rclpy.init(args=args)
    node = RobotControlClient()
    
    # 测试控制命令
    node.send_command('forward', 0.5, 2.0)  # 前进 0.5m/s，持续2秒
    
    # 等待执行完成
    rclpy.sleep_for(seconds=3)
    
    # 查询状态
    node.get_status()
    
    # 停止
    node.send_command('stop', 0.0)
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 简答题

7. **服务通信的工作流程**：
   ```
   1. 服务端启动，注册服务到ROS2中间件
   2. 客户端查找并连接服务端
   3. 客户端发送请求数据
   4. 服务端接收请求，处理业务逻辑
   5. 服务端返回响应结果
   6. 客户端接收响应，继续后续操作
   ```

8. **同步调用 vs 异步调用**：
   - **同步调用**：
     - 优点：代码简单直观，易于理解和调试
     - 缺点：阻塞等待，CPU空闲，并发能力低
     - 适用：简单流程、顺序执行的任务
   - **异步调用**：
     - 优点：非阻塞，并发能力强，资源利用率高
     - 缺点：代码复杂，需要回调处理
     - 适用：并发任务、实时响应场景

9. **服务通信适用场景**：
   - 查询传感器当前状态（需要立即获取结果）
   - 控制机器人执行动作（需要确认执行结果）
   - 读取/修改配置参数（需要返回值）
   - **举例**：查询电池电量、获取机器人位置、控制机械臂运动到指定位置

```python
# add_two_ints_client_async.py
import rclpy
from rclpy.node import Node
from example_interfaces.srv import AddTwoInts


class AddTwoIntsClient(Node):
    def __init__(self):
        super().__init__('add_two_ints_client')
        self.client = self.create_client(AddTwoInts, 'add_two_ints')

    def call_async(self, a, b):
        # 等待服务可用
        while not self.client.wait_for_service(timeout_sec=1.0):
            if not rclpy.ok():
                self.get_logger().error('等待服务时中断')
                return
            self.get_logger().info('等待服务上线...')

        # 创建请求
        request = AddTwoInts.Request()
        request.a = a
        request.b = b

        # 异步调用
        future = self.client.call_async(request)
        future.add_done_callback(self.response_callback)
        
        self.get_logger().info(f'发送请求: {a} + {b}')

    def response_callback(self, future):
        try:
            response = future.result()
            self.get_logger().info(f'异步结果: {response.sum}')
        except Exception as e:
            self.get_logger().error(f'调用失败: {e}')


def main(args=None):
    rclpy.init(args=args)
    node = AddTwoIntsClient()
    
    # 发送异步请求
    node.call_async(10, 20)
    node.call_async(30, 40)
    
    # 处理回调
    rclpy.spin(node)
    
    rclpy.shutdown()
```

```cpp
// add_two_ints_client_async.cpp
#include <rclcpp/rclcpp.hpp>
#include <example_interfaces/srv/add_two_ints.hpp>

class AddTwoIntsClient : public rclcpp::Node
{
public:
    AddTwoIntsClient() : Node("add_two_ints_client")
    {
        client_ = this->create_client<example_interfaces::srv::AddTwoInts>("add_two_ints");
    }

    void call_async(int a, int b)
    {
        while (!client_->wait_for_service(std::chrono::seconds(1))) {
            if (!rclcpp::ok()) {
                RCLCPP_ERROR(this->get_logger(), "等待服务时中断");
                return;
            }
            RCLCPP_INFO(this->get_logger(), "等待服务上线...");

        request->a = a;
        request->b = b;

        RCLCPP_INFO(this->get_logger(), "发送请求: %d + %d", a, b);

        client_->async_send_request(
            request,
            std::bind(&AddTwoIntsClient::response_callback, this, std::placeholders::_1)

        } catch (const std::exception& e) {
            RCLCPP_ERROR(this->get_logger(), "调用失败: %s", e.what());
        }
    }
};

{
```
