## 1. 通信原理

动作通信（Action）是ROS2专门为长时间运行任务设计的通信机制。与服务通信的一次性请求-响应模式不同，动作通信允许客户端在任务执行过程中接收实时反馈，并且可以在任务完成前随时取消。

### 1.1 目标-反馈-结果模式

动作通信采用**目标（Goal）-反馈（Feedback）-结果（Result）**的三阶段模式：

```
客户端                                    服务端
  │                                        │
  │───── 目标请求 (Goal) ─────────────────▶│
  │                                        │
  │                                        ▼
  │◀───── 接受/拒绝 (Goal Accepted) ─────────│
  │                                        │
  │◀───── 反馈1 (Feedback) ─────────────────│
  │                                        │
  │◀───── 反馈2 (Feedback) ─────────────────│
  │                                        │
  │◀───── 结果 (Result) ────────────────────│
  │                                        │
```

- **目标（Goal）**：客户端发送任务目标给服务端，包含任务的参数
- **反馈（Feedback）**：服务端在任务执行过程中周期性发送进度信息
- **结果（Result）**：任务完成后服务端返回最终结果

这种模式特别适合机器人导航、机械臂抓取等需要长时间执行且需要反馈的任务。

### 1.2 可取消

动作通信支持任务取消机制，这是服务通信无法实现的重要特性。客户端可以在任务执行过程中发送取消请求，服务端收到请求后需要：

1. 停止当前任务的执行
2. 清理相关资源
3. 返回取消状态

```python
# Python 取消示例
from actionlib_msgs.msg import GoalStatus

# 发送取消请求
client.cancel_goal()
```

取消机制确保了系统的灵活性，用户可以在任务执行过程中随时改变主意或纠正错误。

### 1.3 可查询状态

动作客户端可以随时查询当前任务的状态，而无需等待反馈消息。常见的状态包括：

| 状态 | 值 | 说明 |
|------|-----|------|
| PENDING | 0 | 目标等待被处理 |
| ACTIVE | 1 | 目标正在执行 |
| PREEMPTED | 2 | 目标被抢占（取消） |
| SUCCEEDED | 3 | 目标成功完成 |
| ABORTED | 4 | 目标执行失败 |
| REJECTED | 5 | 目标被拒绝 |
| PREEMPTING | 6 | 正在取消中 |
| RECALLING | 7 | 正在召回中 |
| RECALLED | 8 | 目标已召回 |
| LOST | 9 | 目标丢失 |

通过状态查询，客户端可以实现更复杂的上层逻辑，如显示任务进度、处理超时等。

### 1.4 适配长时间任务

动作通信的设计初衷就是解决长时间任务的问题。相比服务通信：

| 特性 | 服务通信 | 动作通信 |
|------|----------|----------|
| 反馈机制 | 无 | 周期性反馈 |
| 取消支持 | 无 | 支持取消 |
| 状态查询 | 无 | 支持查询 |
| 任务时长 | 短（秒级） | 长（分钟/小时级） |
| 通信模式 | 同步 | 异步 |

对于一个需要10分钟完成的机器人路径跟踪任务，动作通信可以实时反馈当前位置，让用户了解任务进度，同时允许用户在任何时刻取消任务。

## 2. 动作类型

在ROS2中，动作类型需要通过`.action`文件定义。定义文件通常存放在功能包的`action`目录下。

### 2.1 自定义动作：.action文件编写

`.action`文件使用特殊的语法格式，包含三个部分：目标（Goal）、反馈（Feedback）和结果（Result）。每部分用`---`分隔。

```action
# action/NavigateToPoint.action

# 目标定义
geometry_msgs/PoseStamped target_pose
---
# 结果定义
geometry_msgs/PoseStamped final_pose
bool success
string message
---
# 反馈定义
geometry_msgs/PoseStamped current_pose
float64 distance_remaining
float64 progress_percent
```

**语法说明**：
- `#` 开头为注释
- `---` 分隔目标、结果、反馈三个部分
- 字段格式：`类型 字段名`

### 2.2 目标/反馈/结果字段定义

动作消息的三部分各有其用途：

**目标（Goal）** - 任务的输入参数：
```action
# 输入参数
geometry_msgs/PoseStamped target_pose  # 目标位置
float64 speed_limit                     # 速度限制
```

**结果（Result）** - 任务的最终输出：
```action
# 输出结果
geometry_msgs/PoseStamped final_pose    # 最终位置
bool success                             # 是否成功
string message                           # 状态消息
float64 execution_time                   # 执行时间
```

**反馈（Feedback）** - 任务执行过程中的状态：
```action
# 进度反馈
geometry_msgs/PoseStamped current_pose  # 当前位置
float64 distance_remaining              # 剩余距离
float64 progress_percent                 # 完成百分比
```

ROS2提供了内置动作类型，如`actionlib_msgs`、`nav2_msgs`等，可以直接使用：
- `actionlib_msgs/GoalID`
- `actionlib_msgs/GoalStatus`
- `nav2_msgs/action/NavigateToPose`

## 3. 节点实现

下面分别介绍动作服务端和动作客户端的实现方法，提供C++和Python两种语言的示例。

### 3.1 动作服务端

动作服务端负责接收目标、执行任务、发送反馈并返回结果。

#### C++ 实现

```cpp
// action_server.cpp
#include <rclcpp/rclcpp.hpp>
#include <geometry_msgs/msg/pose_stamped.hpp>
#include <nav2_msgs/action/navigate_to_pose.hpp>
#include <rclcpp_action/rclcpp_action.hpp>

class NavigateActionServer : public rclcpp::Node
{
public:
    using NavigateToPose = nav2_msgs::action::NavigateToPose;
    using GoalHandle = rclcpp_action::ServerGoalHandle<NavigateToPose>;

    NavigateActionServer() : Node("navigate_action_server")
    {
        // 创建动作服务端
        this->action_server_ = rclcpp_action::create_server<NavigateToPose>(
            this,
            "navigate_to_pose",
            std::bind(&NavigateActionServer::handle_goal, this, std::placeholders::_1, std::placeholders::_2),
            std::bind(&NavigateActionServer::handle_cancel, this, std::placeholders::_1),
            std::bind(&NavigateActionServer::handle_accepted, this, std::placeholders::_1)
        );
        
        RCLCPP_INFO(this->get_logger(), "导航动作服务端已启动");
    }

private:
    rclcpp_action::Server<NavigateToPose>::SharedPtr action_server_;
    bool cancel_requested_ = false;

    // 处理目标请求
    rclcpp_action::GoalResponse handle_goal(
        const rclcpp_action::GoalUUID & uuid,
        std::shared_ptr<const NavigateToPose::Goal> goal)
    {
        RCLCPP_INFO(this->get_logger(), "收到目标请求");
        (void)uuid;
        (void)goal;
        return rclcpp_action::GoalResponse::ACCEPT_AND_EXECUTE;
    }

    // 处理取消请求
    rclcpp_action::CancelResponse handle_cancel(
        std::shared_ptr<GoalHandle> goal_handle)
    {
        RCLCPP_INFO(this->get_logger(), "收到取消请求");
        cancel_requested_ = true;
        (void)goal_handle;
        return rclcpp_action::CancelResponse::ACCEPT;
    }

    // 处理接受的目标
    void handle_accepted(std::shared_ptr<GoalHandle> goal_handle)
    {
        // 在新线程中执行任务
        std::thread{std::bind(&NavigateActionServer::execute, this, goal_handle)}.detach();
    }

    // 执行导航任务
    void execute(std::shared_ptr<GoalHandle> goal_handle)
    {
        RCLCPP_INFO(this->get_logger(), "开始执行导航任务");
        
        const auto goal = goal_handle->get_goal();
        auto feedback = std::make_shared<NavigateToPose::Feedback>();
        auto result = std::make_shared<NavigateToPose::Result>();
        
        // 模拟导航过程
        double total_distance = 10.0;  // 假设总距离10米
        double current_distance = 0.0;
        cancel_requested_ = false;
        
        while (current_distance < total_distance && !cancel_requested_)
        {
            // 检查是否被取消
            if (goal_handle->is_canceling()) {
                result->pose = geometry_msgs::msg::PoseStamped();
                result->pose.header.stamp = this->now();
                result->pose.header.frame_id = "map";
                result->pose.pose.position.x = current_distance;
                goal_handle->canceled(result);
                RCLCPP_INFO(this->get_logger(), "任务已取消");
                return;
            }
            
            // 更新位置（模拟）
            current_distance += 0.1;
            
            // 发送反馈
            feedback->pose = geometry_msgs::msg::PoseStamped();
            feedback->pose.header.stamp = this->now();
            feedback->pose.header.frame_id = "map";
            feedback->pose.pose.position.x = current_distance;
            goal_handle->publish_feedback(feedback);
            
            RCLCPP_INFO(this->get_logger(), "当前位置: %.2f 米, 进度: %.1f%%", 
                       current_distance, (current_distance/total_distance)*100);
            
            // 模拟执行时间
            std::this_thread::sleep_for(std::chrono::milliseconds(100));
        }
        
        // 任务完成
        if (cancel_requested_) {
            result->pose = geometry_msgs::msg::PoseStamped();
            goal_handle->canceled(result);
        } else {
            result->pose = geometry_msgs::msg::PoseStamped();
            result->pose.header.stamp = this->now();
            result->pose.header.frame_id = "map";
            result->pose.pose.position.x = total_distance;
            goal_handle->succeed(result);
            RCLCPP_INFO(this->get_logger(), "导航任务完成");
        }
    }
};

int main(int argc, char ** argv)
{
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<NavigateActionServer>());
    rclcpp::shutdown();
    return 0;
}
```

#### Python 实现

```python
# action_server.py
import rclpy
from rclpy.node import Node
from rclpy.action import ActionServer
from rclpy.action.server import GoalResponse, CancelResponse
from geometry_msgs.msg import PoseStamped
from nav2_msgs.action import NavigateToPose


class NavigateActionServer(Node):
    def __init__(self):
        super().__init__('navigate_action_server')
        
        # 创建动作服务端
        self._action_server = ActionServer(
            self,
            NavigateToPose,
            'navigate_to_pose',
            goal_callback=self._goal_callback,
            cancel_callback=self._cancel_callback,
            execute_callback=self._execute_callback
        )
        
        self.get_logger().info('导航动作服务端已启动')
        self._cancel_requested = False

    def _goal_callback(self, goal_request):
        """处理目标请求"""
        self.get_logger().info(f'收到目标请求')
        return GoalResponse.ACCEPT_AND_EXECUTE

    def _cancel_callback(self, goal_handle):
        """处理取消请求"""
        self.get_logger().info('收到取消请求')
        self._cancel_requested = True
        return CancelResponse.ACCEPT

    async def _execute_callback(self, goal_handle):
        """执行导航任务"""
        self.get_logger().info('开始执行导航任务')
        
        goal = goal_handle.request
        feedback = NavigateToPose.Feedback()
        result = NavigateToPose.Result()
        
        # 模拟导航过程
        total_distance = 10.0  # 假设总距离10米
        current_distance = 0.0
        self._cancel_requested = False
        
        while current_distance < total_distance and not self._cancel_requested:
            # 检查是否被取消
            if goal_handle.is_canceling:
                result.pose = PoseStamped()
                result.pose.header.stamp = self.get_clock().now().to_msg()
                result.pose.header.frame_id = 'map'
                goal_handle.canceled(result)
                self.get_logger().info('任务已取消')
                return result
            
            # 更新位置（模拟）
            current_distance += 0.1
            
            # 发送反馈
            feedback.pose = PoseStamped()
            feedback.pose.header.stamp = self.get_clock().now().to_msg()
            feedback.pose.header.frame_id = 'map'
            feedback.pose.pose.position.x = current_distance
            goal_handle.publish_feedback(feedback)
            
            self.get_logger().info(f'当前位置: {current_distance:.2f}米, 进度: {(current_distance/total_distance)*100:.1f}%')
            
            # 模拟执行时间
            await rclpy.sleep(0.1)
        
        # 任务完成
        if self._cancel_requested:
            result.pose = PoseStamped()
            goal_handle.canceled(result)
        else:
            result.pose = PoseStamped()
            result.pose.header.stamp = self.get_clock().now().to_msg()
            result.pose.header.frame_id = 'map'
            result.pose.pose.position.x = total_distance
            goal_handle.succeed(result)
            self.get_logger().info('导航任务完成')
        
        return result


def main(args=None):
    rclpy.init(args=args)
    action_server = NavigateActionServer()
    rclpy.spin(action_server)
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

动作客户端负责发送目标、处理反馈并接收结果。

```cpp
// action_client.cpp
#include <rclcpp/rclcpp.hpp>
#include <geometry_msgs/msg/pose_stamped.hpp>
#include <nav2_msgs/action/navigate_to_pose.hpp>
#include <rclcpp_action/rclcpp_action.hpp>

class NavigateActionClient : public rclcpp::Node
{
public:
    using NavigateToPose = nav2_msgs::action::NavigateToPose;
    using GoalHandle = rclcpp_action::ClientGoalHandle<NavigateToPose>;

    NavigateActionClient() : Node("navigate_action_client")
    {
        // 创建动作客户端
        this->action_client_ = rclcpp_action::create_client<NavigateToPose>(
            this,
            "navigate_to_pose"
        );
        
        RCLCPP_INFO(this->get_logger(), "导航动作客户端已启动");
    }

    void send_goal()
    {
        if (!this->action_client_->wait_for_action_server(std::chrono::seconds(10))) {
            RCLCPP_ERROR(this->get_logger(), "动作服务端不可用");
            return;
        }

        auto goal = NavigateToPose::Goal();
        goal.pose.header.frame_id = "map";
        goal.pose.header.stamp = this->now();
        goal.pose.pose.position.x = 10.0;
        goal.pose.pose.orientation.w = 1.0;

        auto send_goal_options = rclcpp_action::Client<NavigateToPose>::SendGoalOptions();
        send_goal_options.goal_response_callback = 
            std::bind(&NavigateActionClient::goal_response_callback, this, std::placeholders::_1);
        send_goal_options.feedback_callback = 
            std::bind(&NavigateActionClient::feedback_callback, this, std::placeholders::_1, std::placeholders::_2);
        send_goal_options.result_callback = 
            std::bind(&NavigateActionClient::result_callback, this, std::placeholders::_1);

        RCLCPP_INFO(this->get_logger(), "发送目标...");
        this->action_client_->async_send_goal(goal, send_goal_options);
    }

    void cancel_goal()
    {
        RCLCPP_INFO(this->get_logger(), "发送取消请求...");
        this->action_client_->async_cancel_all_goals();
    }

private:
    rclcpp_action::Client<NavigateToPose>::SharedPtr action_client_;
    std::shared_future<GoalHandle::SharedPtr> future_goal_handle_;

    void goal_response_callback(GoalHandle::SharedPtr future)
    {
        auto goal_handle = future.get();
        if (!goal_handle) {
            RCLCPP_WARN(this->get_logger(), "目标被拒绝");
        } else {
            RCLCPP_INFO(this->get_logger(), "目标已被接受");
        }
    }

    void feedback_callback(
        GoalHandle::SharedPtr future,
        const std::shared_ptr<const NavigateToPose::Feedback> feedback)
    {
        (void)future;
        RCLCPP_INFO(this->get_logger(), "当前位置: %.2f", 
                   feedback->pose.pose.position.x);
    }

    void result_callback(const GoalHandle::WrappedResult & result)
    {
        switch (result.code) {
            case rclcpp_action::ResultCode::SUCCEEDED:
                RCLCPP_INFO(this->get_logger(), "任务成功完成");
                break;
            case rclcpp_action::ResultCode::ABORTED:
                RCLCPP_ERROR(this->get_logger(), "任务被中止");
                break;
            case rclcpp_action::ResultCode::CANCELED:
                RCLCPP_INFO(this->get_logger(), "任务被取消");
                break;
            default:
                RCLCPP_ERROR(this->get_logger(), "未知结果");
                break;
        }
        
        // 输出最终位置
        RCLCPP_INFO(this->get_logger(), "最终位置: %.2f", 
                   result.result->pose.pose.position.x);
    }
};

int main(int argc, char ** argv)
{
    rclcpp::init(argc, argv);
    auto node = std::make_shared<NavigateActionClient>();
    
    // 发送目标
    node->send_goal();
    
    // 等待一段时间后取消（模拟用户取消）
    // std::this_thread::sleep_for(std::chrono::seconds(3));
    // node->cancel_goal();
    
    rclcpp::spin(node);
    rclcpp::shutdown();
    return 0;
}
```

#### Python 实现

```python
# action_client.py
import rclpy
from rclpy.node import Node
from rclpy.action import ActionClient
from rclpy.action.client import GoalStatus
from geometry_msgs.msg import PoseStamped
from nav2_msgs.action import NavigateToPose


class NavigateActionClient(Node):
    def __init__(self):
        super().__init__('navigate_action_client')
        
        # 创建动作客户端
        self._action_client = ActionClient(self, NavigateToPose, 'navigate_to_pose')
        
        self.get_logger().info('导航动作客户端已启动')
        self._goal_handle = None

    def send_goal(self):
        """发送目标"""
        # 等待服务端可用
        if not self._action_client.wait_for_server(timeout_sec=10.0):
            self.get_logger().error('动作服务端不可用')
            return

        # 创建目标
        goal_msg = NavigateToPose.Goal()
        goal_msg.pose.header.frame_id = 'map'
        goal_msg.pose.header.stamp = self.get_clock().now().to_msg()
        goal_msg.pose.pose.position.x = 10.0
        goal_msg.pose.pose.orientation.w = 1.0

        self.get_logger().info('发送目标...')
        
        # 发送目标并指定回调
        self._goal_handle = self._action_client.send_goal_async(
            goal_msg,
            feedback_callback=self._feedback_callback
        )
        
        self._goal_handle.add_done_callback(self._goal_response_callback)

    def _goal_response_callback(self, future):
        """处理目标响应"""
        goal_handle = future.result()
        if goal_handle.accepted:
            self.get_logger().info('目标已被接受')
            goal_handle.get_result_async().add_done_callback(self._result_callback)
        else:
            self.get_logger().warn('目标被拒绝')

    def _feedback_callback(self, feedback_msg):
        """处理反馈"""
        feedback = feedback_msg.feedback
        self.get_logger().info(f'当前位置: {feedback.pose.pose.position.x:.2f}米')

    def _result_callback(self, future):
        """处理结果"""
        result = future.result().result
        status = future.result().status
        
        if status == GoalStatus.STATUS_SUCCEEDED:
            self.get_logger().info('任务成功完成')
        elif status == GoalStatus.STATUS_CANCELED:
            self.get_logger().info('任务被取消')
        elif status == GoalStatus.STATUS_ABORTED:
            self.get_logger().error('任务被中止')
        
        self.get_logger().info(f'最终位置: {result.pose.pose.position.x:.2f}米')

    def cancel_goal(self):
        """取消目标"""
        if self._goal_handle:
            self.get_logger().info('发送取消请求...')
            # 取消所有目标
            self._action_client._cancel_goal(self._goal_handle)


def main(args=None):
    rclpy.init(args=args)
    action_client = NavigateActionClient()
    
    # 发送目标
    action_client.send_goal()
    
    # 保持节点运行
    try:
        rclpy.spin(action_client)
    except KeyboardInterrupt:
        pass
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## 4. 任务控制

动作通信提供了丰富的任务控制功能，包括任务取消、状态查询和超时处理。

### 4.1 任务取消

任务取消是动作通信的核心特性之一。以下是实现取消功能的完整示例：

**服务端取消处理**：
```python
# 检查取消状态并处理
if goal_handle.is_canceling:
    # 执行清理操作
    cleanup_resources()
    
    # 返回取消状态
    result = MyAction.Result()
    result.message = "任务已被取消"
    goal_handle.canceled(result)
    return result
```

**客户端发送取消**：
```python
# 客户端取消正在执行的目标
goal_handle = future.result()
goal_handle.cancel_goal_async()
```

### 4.2 状态查询

动作客户端可以查询当前任务状态：

```python
# 获取目标状态
goal_handle = future.result()

# 检查各种状态
if goal_handle.status == GoalStatus.STATUS_SUCCEEDED:
    print("任务成功")
elif goal_handle.status == GoalStatus.STATUS_CANCELED:
    print("任务已取消")
elif goal_handle.status == GoalStatus.STATUS_ABORTED:
    print("任务失败")
```

**服务端获取客户端目标状态**：
```cpp
// 在服务端获取所有活动目标
auto goals = this->action_server_->get_all_goals();
for (const auto& goal : goals) {
    RCLCPP_INFO(this->get_logger(), "活动目标数: %zu", goals.size());
}
```

### 4.3 结果超时处理

为防止客户端无限等待，可以设置结果超时：

```python
# 设置超时为30秒
from rclpy.duration import Duration

future = goal_handle.get_result_async()
try:
    # 等待结果，最多30秒
    result = rclpy.wait_for_future(future, timeout=Duration(seconds=30))
    if result:
        print("获取到结果")
    else:
        print("等待超时，取消目标")
        goal_handle.cancel_goal_async()
except TimeoutError:
    print("操作超时")
```

### 5.1 动作服务端 - 执行路径规划，实时反馈位置进度

首先创建自定义动作类型：

```action
# action/PathTracking.action

# 目标：路径点序列
geometry_msgs/PoseStamped[] waypoints
float32 max_velocity
---
# 结果：最终状态
geometry_msgs/PoseStamped final_pose
bool success
float32 total_distance
float32 execution_time
---
# 反馈：当前位置和进度
geometry_msgs/PoseStamped current_pose

float32 current_velocity
```

服务端实现（C++）：

```cpp
// path_tracking_server.cpp
#include <rclcpp/rclcpp.hpp>
#include <rclcpp_action/rclcpp_action.hpp>
#include <geometry_msgs/msg/pose_stamped.hpp>
// 注意：需要生成对应的动作消息头文件
// #include "path_tracking/action/path_tracking.hpp"

class PathTrackingServer : public rclcpp::Node
{
public:
    // 使用自定义动作类型
    // using PathTracking = path_tracking::action::PathTracking;
    // 临时使用内置类型作为示例
    using PathTracking = nav2_msgs::action::NavigateToPose;
    using GoalHandle = rclcpp_action::ServerGoalHandle<PathTracking>;

    PathTrackingServer() : Node("path_tracking_server")
    {
        action_server_ = rclcpp_action::create_server<PathTracking>(
            this,
            "path_tracking",
            std::bind(&PathTrackingServer::handle_goal, this, std::placeholders::_1, std::placeholders::_2),
            std::bind(&PathTrackingServer::handle_cancel, this, std::placeholders::_1),
            std::bind(&PathTrackingServer::handle_accepted, this, std::placeholders::_1)
        );
        
        RCLCPP_INFO(this->get_logger(), "路径跟踪服务端已启动");
    }

private:
    rclcpp_action::Server<PathTracking>::SharedPtr action_server_;
    bool cancel_requested_ = false;

    rclcpp_action::GoalResponse handle_goal(
        const rclcpp_action::GoalUUID & uuid,
        std::shared_ptr<const PathTracking::Goal> goal)
    {
        RCLCPP_INFO(this->get_logger(), "收到路径跟踪目标"); 
        (void)uuid;
        (void)goal;
        return rclcpp_action::GoalResponse::ACCEPT_AND_EXECUTE;
    }

    rclcpp_action::CancelResponse handle_cancel(std::shared_ptr<GoalHandle> goal_handle)
    {
        RCLCPP_INFO(this->get_logger(), "收到取消请求");
        cancel_requested_ = true;
        (void)goal_handle;
        return rclcpp_action::CancelResponse::ACCEPT;
    }

    void handle_accepted(std::shared_ptr<GoalHandle> goal_handle)
    {
        std::thread{std::bind(&PathTrackingServer::execute, this, goal_handle)}.detach();
    }

    void execute(std::shared_ptr<GoalHandle> goal_handle)
    {
        auto goal = goal_handle->get_goal();
        auto feedback = std::make_shared<PathTracking::Feedback>();
        auto result = std::make_shared<PathTracking::Result>();
        
        cancel_requested_ = false;
        
        // 模拟沿路径移动（简化版）
        float total_distance = 10.0;
        float distance_traveled = 0.0;
        rclcpp::Time start_time = this->now();
        
        // 模拟移动过程
        while (distance_traveled < total_distance && !cancel_requested_)
        {
            // 检查取消
            if (goal_handle->is_canceling()) {
                result->pose = geometry_msgs::msg::PoseStamped();
                result->pose.header.stamp = this->now();
                goal_handle->canceled(result);
                RCLCPP_INFO(this->get_logger(), "路径跟踪已取消");
                return;
            }
            
            // 更新位置
            distance_traveled += 0.1;
            
            // 发送反馈
            feedback->pose = geometry_msgs::msg::PoseStamped();
            feedback->pose.header.stamp = this->now();
            feedback->pose.header.frame_id = "map";
            feedback->pose.pose.position.x = distance_traveled;
            
            goal_handle->publish_feedback(feedback);
            
            RCLCPP_INFO(this->get_logger(), 
                       "进度: %.1f%%, 剩余距离: %.2f米",
                       (distance_traveled/total_distance)*100,
                       total_distance - distance_traveled);
            
            std::this_thread::sleep_for(std::chrono::milliseconds(100));
        }
        
        // 任务完成
        rclcpp::Time end_time = this->now();
        
        if (cancel_requested_) {
            goal_handle->canceled(result);
        } else {
            result->pose = geometry_msgs::msg::PoseStamped();
            result->pose.header.stamp = end_time;
            result->pose.header.frame_id = "map";
            result->pose.pose.position.x = total_distance;
            goal_handle->succeed(result);
            
            RCLCPP_INFO(this->get_logger(), "路径跟踪完成");
        }
    }
};

int main(int argc, char ** argv)
{
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<PathTrackingServer>());
    rclcpp::shutdown();
    return 0;
}
```

### 5.2 客户端 - 发送路径目标，处理反馈并支持任务取消

客户端实现（Python）：

```python
# path_tracking_client.py
import rclpy
from rclpy.node import Node
from rclpy.action import ActionClient
from rclpy.action.client import GoalStatus
from geometry_msgs.msg import PoseStamped
from nav2_msgs.action import NavigateToPose


class PathTrackingClient(Node):
    def __init__(self):
        super().__init__('path_tracking_client')
        # 使用NavigateToPose作为示例，实际使用自定义动作类型
        self._action_client = ActionClient(self, NavigateToPose, 'path_tracking')
        self._goal_handle = None
        self.get_logger().info('路径跟踪客户端已启动')

    def send_path_goal(self, target_x, max_velocity=0.5):
        """发送路径跟踪目标"""
        if not self._action_client.wait_for_server(timeout_sec=10.0):
            self.get_logger().error('服务端不可用')
            return

        goal_msg = NavigateToPose.Goal()
        goal_msg.pose.header.frame_id = 'map'
        goal_msg.pose.header.stamp = self.get_clock().now().to_msg()
        goal_msg.pose.pose.position.x = target_x
        goal_msg.pose.pose.orientation.w = 1.0

        self.get_logger().info(f'发送路径目标，目标位置: {target_x}米')
        
        self._goal_handle = self._action_client.send_goal_async(
            goal_msg,
            feedback_callback=self._feedback_callback
        )
        
        self._goal_handle.add_done_callback(self._goal_response_callback)

    def _goal_response_callback(self, future):
        """处理目标响应"""
        goal_handle = future.result()
        if goal_handle.accepted:
            self.get_logger().info('路径目标已被接受，开始跟踪')
            goal_handle.get_result_async().add_done_callback(self._result_callback)
        else:
            self.get_logger().warn('路径目标被拒绝')

    def _feedback_callback(self, feedback_msg):
        """处理位置反馈"""
        feedback = feedback_msg.feedback
        self.get_logger().info(
            f'当前位置: {feedback.pose.pose.position.x:.2f}米'
        )

    def _result_callback(self, future):
        """处理最终结果"""
        result = future.result().result
        status = future.result().status
        
        if status == GoalStatus.STATUS_SUCCEEDED:
            self.get_logger().info('路径跟踪成功完成')
        elif status == GoalStatus.STATUS_CANCELED:
            self.get_logger().info('路径跟踪被取消')
        elif status == GoalStatus.STATUS_ABORTED:
            self.get_logger().error('路径跟踪失败')

    def cancel_goal(self):
        """取消当前目标"""
        if self._goal_handle:
            future = self._goal_handle.result()
            if future and future.accepted:
                self.get_logger().info('发送取消请求')
                future.cancel_goal_async()


def main(args=None):
    rclpy.init(args=args)
    client = PathTrackingClient()
    
    # 发送目标（10米）
    client.send_path_goal(target_x=10.0, max_velocity=0.3)
    
    # 保持运行
    try:
        rclpy.spin(client)
    except KeyboardInterrupt:
        pass
    
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## 练习题

### 基础题

1. **动作通信与服务通信的主要区别是什么？**
   
   答案：动作通信相比服务通信增加了反馈机制、支持任务取消、支持状态查询，特别适合长时间运行的任务。

2. **`.action`文件的三个部分是什么？**
   
   答案：`.action`文件包含目标（Goal）、结果（Result）和反馈（Feedback）三部分，用`---`分隔。

3. **动作通信的五个基本状态是什么？**
   
   答案：PENDING（等待）、ACTIVE（执行中）、SUCCEEDED（成功）、CANCELED（取消）、ABORTED（失败）。

### 进阶题

4. **编写一个动作服务端，当目标被取消时如何正确处理？**
   
   答案：需要在执行循环中检查`goal_handle->is_canceling()`（C++）或`goal_handle.is_canceling`（Python），如果返回true，则执行清理并调用`goal_handle.canceled(result)`。

5. **如何在动作客户端设置结果超时？**
   
   答案：可以使用`rclpy.wait_for_future()`配合`Duration`设置超时时间，或使用`get_result_async()`配合定时器实现超时检测。
