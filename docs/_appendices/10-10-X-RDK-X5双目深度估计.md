## 1. 双目视觉概述

### 1.1 什么是双目立体视觉

双目立体视觉（Binoocular Stereo Vision）是模仿人类双眼的深度感知机制，通过两个位于不同位置的相机同时拍摄同一场景，利用**视差（Disparity）** 原理计算每个像素的三维深度信息。与单目深度估计不同，双目深度估计属于**几何方法**，不依赖深度学习模型，因此在精度和实时性上具有独特优势。

```
┌─────────────────────────────────────────────────────────────────────┐
│                        双目立体视觉原理                               │
│                                                                     │
│         左相机                    右相机                             │
│        (Cam L)                  (Cam R)                             │
│           \                        /                                │
│            \   同一物体 P          /                                 │
│             \      ●              /                                 │
│              \    /|\             /                                 │
│               \  / | \           /                                  │
│                \/  |  \         /                                   │
│            ┌──────────────┐                                        │
│            │   深度图 Z    │                                        │
│            └──────────────┘                                        │
│                                                                     │
│   视差 d = xl - xr（同名点在左右图像的像素坐标差）                      │
│   深度 Z = f × B / d（三角测量核心公式）                              │
│                                                                     │
│   其中：f=焦距，B=基线（两相机距离），d=视差                          │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 视差与深度的几何关系

**核心公式——三角测量法**：

```
Z（深度）= f（焦距）× B（基线） / d（视差）
```

| 参数 | 符号 | 说明 | 典型值 |
|------|------|------|--------|
| 焦距 | f | 相机内参（像素） | ~600px |
| 基线 | B | 左右相机光心距离（mm） | 60~120mm |
| 视差 | d | 同一3D点在左右图像的x坐标差（像素） | 0~255px |
| 深度 | Z | 目标到相机的实际距离（mm） | 0.3~20m |

**视差与深度的关系特点**：

- **视差越大 → 深度越近**（最近处视差可达 100+ 像素）
- **视差越小 → 深度越远**（远处物体视差趋近于 0）
- 深度与视差成**反比**关系，而非线性关系
- 当深度趋于无穷远时，视差趋于 0，此时深度估计误差极大

### 1.3 双目相机内参与外参

**内参（Intrinsics）**：描述单个相机的成像几何特性，与相机外部位置无关。

| 内参 | 符号 | 说明 |
|------|------|------|
| 焦距 | fx, fy | 沿 x、y 方向焦距（像素） |
| 主点 | cx, cy | 光轴与图像平面的交点 |
| 畸变系数 | k1, k2, p1, p2, k3 | 镜头畸变参数 |

**外参（Extrinsics）**：描述两个相机之间的相对位置和姿态关系。

| 外参 | 符号 | 说明 |
|------|------|------|
| 旋转矩阵 | R | 右相机相对左相机的旋转（3×3） |
| 平移向量 | T | 右相机相对左相机的平移（B 基线） |
| 本质矩阵 | E | 旋转×平移，描述两相机相对位姿 |
| 基本矩阵 | F | 图像平面上的对极几何关系 |

**理想双目模型（Rectified）**：经过校正（Rectification）后，左右相机共面、水平对齐、行对准，此时匹配搜索只需在同一行（水平 scanline）内进行，大大简化了立体匹配算法。

### 1.4 双目匹配算法分类

| 类别 | 代表算法 | 特点 | 实时性 |
|------|---------|------|--------|
| 全局方法 | Graph Cuts、Belief Propagation | 精度高、计算慢 | 差 |
| 局部方法 | BM（Block Matching）、SAD、SSD | 速度快、精度一般 | 好 |
| 半全局方法 | **SGBM（Semi-Global BM）** | 精度速度均衡 | 较好 |
| 深度学习方法 | StereoNet、PSMNet、RAFT-Stereo | 精度最高、需GPU | 取决于硬件 |

**本课程重点**：SGBM（Semi-Global Block Matching）是工程实践中最常用的双目匹配算法，在 RDK-X5 的 BPU 上可通过地平线算法库高效运行。

### 1.5 RDK-X5 双目相机接口

## 3. 双目相机标定

### 3.1 标定板准备

双目相机标定需要使用**棋盘格标定板**（Checkerboard）：

| 参数 | 推荐值 |
|------|--------|
| 内角点数 | 9×6 或 11×9（行列内角点数） |
| 单格尺寸 | 30mm × 30mm（建议打印在 A4/A3 纸上） |
| 打印精度 | 600 DPI 以上 |
| 材质 | 平整硬板（避免弯曲变形） |
| 数量 | 建议 20~30 张不同角度的标定图像 |

### 3.2 ROS2 双目标定工具

地平线 RDK-X5 的 ROS2 环境已包含 `stereo_image_proc` 和标定工具集。

**Step 1：安装标定依赖包**

```bash
# SSH 登录 RDK-X5
ssh root@<RDK-X5-IP>

# 安装标定相关工具
apt update && apt install -y ros-humble-camera-calibration ros-humble-image-view
```

**Step 2：启动双目相机驱动**

```bash
# 如果是 USB 双目相机（Leopard Imaging）
source /opt/hobot/ros2/setup.bash
ros2 launch rdk_deploy leopard_stereo_launch.py

# 如果是两个 MIPI 单目相机，需要配置双目录像头节点
# 参考 10-1-X 课程的 MIPI 摄像头接入方法
```

**Step 3：采集标定图像**

```bash
# 在 RDK-X5 或 PC 上打开图像查看器
ros2 run image_view stereo_view --ros-args -r image:=<left_image_topic> -r right/image:=<right_image_topic>

# 或者使用 rqt_image_view
ros2 run rqt_image_view rqt_image_view
```

> **标定图像采集要求**：
> - 标定板占据画面 20%~80% 区域
> - 采集 20~30 组图像，覆盖不同距离（0.4m ~ 2.0m）
> - 标定板角度多样：绕 X 轴（俯仰）、绕 Y 轴（偏航）、绕 Z 轴（翻滚）
> - 左右相机各自覆盖完整标定画面

**Step 4：执行双目标定**

```bash
# 使用 camera_calibration 包进行双目标定
ros2 run camera_calibration cameracalibrator.py \
    --approximate 0.0 \
    --size 9x6 \
    --square 0.03 \
    /stereo/left:=/stereo/left/image_raw \
    /stereo/right:=/stereo/right/image_raw \
    --no-service-check
```

> **参数说明**：
> - `--size 9x6`：标定板内角点行列数
> - `--square 0.03`：每个棋盘格方格的实际尺寸（米）
> - `--approximate 0.0`：时间同步容差（设为 0 表示严格同步）

标定过程中，UI 窗口会实时显示标定进度：
- **X**：左右方向移动
- **Y**：前后方向移动
- **Size**：标定板大小（距离远近）
- **Skew**：标定板倾斜角度

当所有指标都变成绿色后，点击 **CALIBRATE** 按钮开始标定计算（约需 1~3 分钟）。

**Step 5：保存标定结果**

点击 **SAVE** 按钮保存标定结果，文件保存路径默认为 `~/.ros/camera_info/`。

**Step 6：导出标定参数文件**

```bash
# 标定结果文件格式为 .yaml
# 查看左相机内参
cat ~/.ros/camera_info/left_camera.yaml

# 查看右相机内参

## 4. 双目深度估计算法

### 4.1 块匹配（Block Matching, BM）

**BM 算法**是最经典的局部立体匹配算法，速度快，适合嵌入式实时应用。

**核心思想**：对于左图像中的每个像素，以该像素为中心取一个固定大小的匹配窗口，在右图像的同一行（校正后）中搜索最佳匹配位置，找到的位移即为视差。

```
┌──────────────────────────────────────────────────────────────┐
│                     BM 块匹配算法原理                          │
│                                                              │
│  左图像窗口           右图像扫描          匹配代价计算           │
│   ┌───────┐    ┌────────────────┐    SAD / SSD              │
│   │ ●  ●  │ vs │ ●  ●  ●  ●  ● │ =>  Sum(|I左 - I右|)       │
│   └───────┘    └────────────────┘    找到最小代价位置         │
│   中心像素                                                   │
│   xl = 100     xr 扫描范围           视差 d                   │
│                                                              │
│   关键参数：                                                   │
│   • blocksize：匹配窗口大小（奇数，如 15）                    │
│   • numDisparities：搜索视差范围（16 的倍数，如 128）          │
└──────────────────────────────────────────────────────────────┘
```

**OpenCV BM 参数配置**：

```python
import cv2

# 创建 BM 对象并设置参数
stereo = cv2.StereoBM_create(
    numDisparities=128,    # 视差搜索范围（16的倍数）
    blockSize=15           # 匹配窗口大小（奇数）
)

# 可选参数调整
stereo.setMinDisparity(0)          # 最小视差
stereo.setUniquenessRatio(15)      # 唯一性约束（防止歧义匹配）
stereo.setSpeckleRange(0)          # 散粒噪声过滤范围
stereo.setSpeckleWindowSize(0)    # 散粒噪声过滤窗口
```

**BM 算法优缺点**：

| 优点 | 缺点 |
|------|------|
| 计算速度快，适合实时 | 对无纹理区域处理差 |
| 实现简单，参数少 | 对光照变化敏感 |
| 嵌入式友好 | 视差精度有限 |

### 4.2 半全局块匹配（SGBM）

**SGBM（Semi-Global Block Matching）** 是 BM 的改进版本，通过多方向（8 或 16 个方向）动态规划代价聚合，兼顾速度和精度，是工程中最常用的算法。

**与 BM 的核心区别**：BM 只做当前行的匹配；SGBM 在多个方向（行、列、对角线）上累计匹配代价，减少了噪声和误匹配。

```
┌──────────────────────────────────────────────────────────────┐
│                   SGBM 多方向代价聚合                          │
│                                                              │
│     ←  ←  ←  ●  →  →  →   (水平方向)                        │
│     ↑        ↑         (垂直方向)                          │
│     ↑        ↓         (对角线方向)                         │
│     ●                                                           │
│   累计 8 个方向的匹配代价：                                   │
│   C(p, d) = Σ C_direction(p, d)                             │
│   视差 = argmin C(p, d)                                      │
└──────────────────────────────────────────────────────────────┘
```

**OpenCV SGBM 参数配置**：

```python
# 创建 SGBM 对象
stereo = cv2.StereoSGBM_create(
    minDisparity=0,              # 最小视差
    numDisparities=128,          # 视差搜索范围（16的倍数）
    blockSize=5,                 # 匹配窗口大小（奇数）
    P1=8 * 3 * 5**2,             # 平滑项参数1（惩罚小视差变化）
    P2=32 * 3 * 5**2,            # 平滑项参数2（惩罚大视差变化）
    disp12MaxDiff=1,             # 左右视差一致性检查阈值
    uniquenessRatio=10,          # 唯一性约束（百分比）
    speckleWindowSize=100,       # 散粒噪声过滤窗口
    speckleRange=2,              # 散粒噪声过滤范围
    mode=cv2.STEREO_SGBM_MODE_HH # HH = 所有方向动态规划（精度最高）
)

# SGBM 各参数含义：
# P1, P2：平滑惩罚项，值越大深度图越平滑
# disp12MaxDiff：左右视差一致性检查，超过该阈值判定为错误匹配
# speckleWindowSize：连通区域过滤，移除孤立噪点
```

**BM vs SGBM 性能对比**：

| 指标 | BM | SGBM |
|------|-----|------|
| 处理速度 | ★★★★★ | ★★★★ |
| 视差精度 | ★★★ | ★★★★ |
| 边缘保持 | ★★ | ★★★★ |
| 无纹理区域 | ★★ | ★★★ |
| 推荐场景 | 高速、低成本 | 工程应用首选 |

### 4.3 深度学习双目方法（简介）

| 算法 | 年份 | 特点 | 精度 |
|------|------|------|------|
| StereoNet | 2018 | 轻量级网络，适合嵌入式 | 中 |
| PSMNet | 2018 | 空间金字塔池化，多尺度 | 高 |
| RAFT-Stereo | 2021 | 迭代优化，精度最高 | 很高 |
| HITNet | 2021 | 高速，多线程友好 | 高 |

> **RDK-X5 BPU 部署**：地平线支持将 StereoNet 等模型量化部署到 BPU 加速，详细内容见课程 10-16-X（YOLO 模型转换与 RDK-X5 部署）。

### 4.4 视差图转深度图

得到视差图后，需要转换为深度图：

```python
# 深度计算公式
# Z（米）= (fx × B) / d（像素）
# 其中：fx=焦距，B=基线（米），d=视差值

# 视差为 0 或无效时（d = 0），深度值为无穷大，需要过滤
depth = np.where(disparity > 0, fx * baseline / disparity, 0)
```

## 5. ROS2 双目深度节点

### 5.1 ROS2 双目图像处理流水线

典型的 ROS2 双目深度处理流程：

```
┌──────────────────────────────────────────────────────────────────┐
│                    ROS2 双目深度处理流水线                          │
│                                                                  │
│  /stereo/left/image_raw    /stereo/right/image_raw               │
│         │                           │                            │
│         └────────┬───────────────────┘                           │
│                  ▼                                                │
│         ┌───────────────┐                                         │
│         │ image_proc     │  校正（Rectification）                  │
│         │ (单目校正)      │ 畸变去除 + 极线对齐                     │
│         └───────┬───────┘                                         │
│                 │                                                 │
│         ┌───────▼───────────┐                                     │
│         │  stereo_image_proc │  立体匹配（SGBM/BM）                │

"""
双目相机校正
功能：读取标定参数，对左右图像进行畸变校正和极线对齐
"""
import cv2
import numpy as np

# ========== 1. 标定参数（替换为实际标定结果） ==========
LEFT_CAMERA_MATRIX = np.array([
    [615.23, 0.0, 641.5],
    [0.0, 615.23, 361.2],
    [0.0, 0.0, 1.0]
], dtype=np.float64)

LEFT_DIST_COEFFS = np.array([-0.0123, 0.0234, -0.0012, 0.0034, 0.0], dtype=np.float64)

RIGHT_CAMERA_MATRIX = np.array([
    [615.23, 0.0, 638.2],
    [0.0, 615.23, 361.2],
    [0.0, 0.0, 1.0]
], dtype=np.float64)

RIGHT_DIST_COEFFS = np.array([-0.0118, 0.0219, -0.0015, 0.0031, 0.0], dtype=np.float64)

# 双目外参（从标定获得）
R = np.array([
    [1.0, 0.0001, -0.0002],
    [-0.0001, 1.0, 0.0003],
    [0.0002, -0.0003, 1.0]
], dtype=np.float64)

T = np.array([-0.12, 0.0, 0.0], dtype=np.float64)  # 基线 120mm


def compute_rectify_maps(image_size=(1280, 720)):
    """
    计算双目校正的映射表
    校正后，左右图像平面共面且行对齐
    """
    w, h = image_size

    # 立体校正：Bouguet 校正（保持两张图像重叠区域最大化）
    R1, R2, P1, P2, Q, roi1, roi2 = cv2.stereoRectify(
        cameraMatrix1=LEFT_CAMERA_MATRIX,
        distCoeffs1=LEFT_DIST_COEFFS,
        cameraMatrix2=RIGHT_CAMERA_MATRIX,
        distCoeffs2=RIGHT_DIST_COEFFS,
        imageSize=(w, h),
        R=R,  # 旋转矩阵
        T=T,  # 平移向量
        flags=cv2.CALIB_ZERO_DISPARITY,  # 零视差约束
        newImageSize=(w, h)
    )

    print(f"Q 矩阵（视差-深度映射）:\n{Q}")

    # 计算校正映射表（用于 cv2.remap）
    mapLx, mapLy = cv2.initUndistortRectifyMap(
        cameraMatrix=LEFT_CAMERA_MATRIX,
        distCoeffs=LEFT_DIST_COEFFS,
        R=R1,
        newCameraMatrix=P1[:, :3],
        size=(w, h),
        m1type=cv2.CV_16SC2
    )

    # 右相机校正映射
    mapRx, mapRy = cv2.initUndistortRectifyMap(
        cameraMatrix=RIGHT_CAMERA_MATRIX,
        distCoeffs=RIGHT_DIST_COEFFS,
        R=R2,
        newCameraMatrix=P2[:, :3],
        size=(w, h),
        m1type=cv2.CV_16SC2
    )

    return mapLx, mapLy, mapRx, mapRy, Q


def rectify_images(left_img, right_img, mapLx, mapLy, mapRx, mapRy):
    """
    对左右图像进行校正
    校正后同名点位于同一行（水平对准）
    """
    left_rect = cv2.remap(left_img, mapLx, mapLy, cv2.INTER_LINEAR)
    right_rect = cv2.remap(right_img, mapRx, mapRy, cv2.INTER_LINEAR)

    return left_rect, right_rect


if __name__ == "__main__":

### 6.2 SGBM 视差计算

**文件路径**：`~/robot_ws/src/rdk_deploy/scripts/stereo_sgbm.py`

```python
#!/usr/bin/env python3
"""
双目深度估计：SGBM 算法实现
功能：读取校正后的左右图像，计算视差图和深度图
"""
import cv2
import numpy as np
import os


class StereoProcessor:
    """双目深度处理器"""

    def __init__(self, baseline=0.12, focal_length=615.23):
        """
        初始化双目处理器

        参数：
            baseline：基线长度（米）
            focal_length：焦距（像素）
        """
        self.baseline = baseline  # 基线 B（米）
        self.focal_length = focal_length  # 焦距 fx（像素）

        # ========== 创建 SGBM 对象 ==========
        # SGBM 参数详解：
        # minDisparity：最小视差，通常为 0
        # numDisparities：视差搜索范围，必须是 16 的倍数（如 64, 128, 256）
        # blockSize：匹配窗口大小，奇数（5, 7, 9, 11...），越大越平滑
        # P1, P2：平滑惩罚项，控制深度图平滑度
        # P1 = 8 * 通道数 * 块大小^2（小视差变化惩罚）
        # P2 = 32 * 通道数 * 块大小^2（大视差变化惩罚）
        # disp12MaxDiff：左右一致性检查阈值，1 通常足够
        # uniquenessRatio：唯一性约束百分比，防止歧义匹配，5~15 合适
        # speckleWindowSize：散粒噪声过滤窗口大小，50~200
        # speckleRange：散粒噪声范围，1~3
        # mode：STEREO_SGBM_MODE_HH = 所有方向动态规划，精度最高
        self.stereo = cv2.StereoSGBM_create(
            minDisparity=0,
            numDisparities=128,          # 视差搜索范围
            blockSize=5,                  # 匹配窗口大小
            P1=600,                       # P1 = 8 * 3 * 5^2
            P2=2400,                      # P2 = 32 * 3 * 5^2
            disp12MaxDiff=1,             # 左右一致性检查
            uniquenessRatio=10,           # 唯一性约束
            speckleWindowSize=100,        # 散粒噪声过滤
            speckleRange=2,               # 散粒噪声范围
            mode=cv2.STEREO_SGBM_MODE_HH  # 全方向动态规划
        )

    def compute_disparity(self, left_img, right_img):
        """
        计算视差图

        参数：
            left_img：左校正图像（灰度）
            right_img：右校正图像（灰度）

        返回：
            disparity：视差图（16位，1/16像素精度）
        """
        # 确保输入为灰度图
        if len(left_img.shape) == 3:
            left_gray = cv2.cvtColor(left_img, cv2.COLOR_BGR2GRAY)
        else:
            left_gray = left_img

        if len(right_img.shape) == 3:
            right_gray = cv2.cvtColor(right_img, cv2.COLOR_BGR2GRAY)
        else:
            right_gray = right_img

        # SGBM 计算视差
        # 注意：OpenCV 的视差图是 16SC1（16位带符号字符）格式
        # 实际视差值 = disparity / 16.0（因为 SGBM 使用 1/16 像素精度）
        disparity = self.stereo.compute(left_gray, right_gray)

        return disparity

    def disparity_to_depth(self, disparity, eps=1e-6):
        """
        将视差图转换为深度图

        公式：Z = f * B / d

        参数：
            disparity：视差图
            eps：避免除零的小值

        返回：
            depth：深度图（米）
        """
        # 过滤无效视差（小于等于0）
        valid_disparity = np.where(disparity > 0, disparity.astype(np.float32) / 16.0, 0)

        # 深度计算：Z = f * B / d
        # 当视差趋于0时（远处），深度趋于无穷大
        depth = np.where(
            valid_disparity > eps,
            self.focal_length * self.baseline / valid_disparity,
            0
        )

        return depth



    def visualize_disparity(self, disparity):
        """
        将视差图转换为伪彩色图像（用于显示）
        """
        disparity_float = disparity.astype(np.float32)
        disparity_float = np.maximum(disparity_float, 0)
        disparity_normalized = (disparity_float / 128.0 * 255).astype(np.uint8)
        disparity_color = cv2.applyColorMap(disparity_normalized, cv2.COLORMAP_JET)
        return disparity_color

#!/usr/bin/env python3
"""
双目深度图可视化
功能：将深度图渲染为伪彩色，并生成 3D 点云
"""
import cv2
import numpy as np
import open3d as o3d


def visualize_depth_map(depth, max_depth=10.0):
    """
    将深度图转换为伪彩色可视化
    """
    depth_scaled = np.clip(depth / max_depth * 255, 0, 255).astype(np.uint8)
    depth_color = cv2.applyColorMap(depth_scaled, cv2.COLORMAP_JET)
    return depth_color


def depth_to_pointcloud(depth, fx, fy, cx, cy):
    """
    将深度图转换为 3D 点云
    """
    h, w = depth.shape
    u_coords, v_coords = np.meshgrid(np.arange(w), np.arange(h))

    valid_mask = depth > 0
    u_valid = u_coords[valid_mask]
    v_valid = v_coords[valid_mask]
    d_valid = depth[valid_mask]

    x_3d = (u_valid - cx) * d_valid / fx
    y_3d = (v_valid - cy) * d_valid / fy
    z_3d = d_valid

    points = np.stack([x_3d, y_3d, z_3d], axis=-1)

    pcd = o3d.geometry.PointCloud()
    pcd.points = o3d.utility.Vector3dVector(points)

    return pcd

## 7. 深度估计后处理

### 7.1 深度滤波

深度图往往包含噪声和空洞，需要滤波处理。

**时序滤波（Temporal Filter）**：

```python
import numpy as np

class TemporalMedianFilter:
    """时序中值滤波：对连续帧的深度值取中值，减少瞬时噪声"""

    def __init__(self, window_size=5):
        self.window_size = window_size
        self.buffer = []

    def filter(self, depth_frame):
        self.buffer.append(depth_frame.copy())
        if len(self.buffer) > self.window_size:
            self.buffer.pop(0)
        filtered = np.median(np.stack(self.buffer), axis=0)
        return filtered.astype(np.float32)
```

**空间滤波（Spatial Filter）**：

```python
import cv2

def spatial_median_filter(depth, kernel_size=5):
    """空间域中值滤波，去除孤立噪点"""
    filtered = cv2.medianBlur(depth.astype(np.float32), kernel_size)
    return filtered


def bilateral_filter_depth(depth, d=5, sigmaColor=1000, sigmaSpace=5):
    """双边滤波：保边去噪"""
    filtered = cv2.bilateralFilter(depth.astype(np.float32), d, sigmaColor, sigmaSpace)
    return filtered
```

### 7.2 深度图插值

深度图中存在空洞（无效深度），需要进行插值填补。

```python
def fill_depth_holes(depth, method='median'):
    """填补深度图中的空洞"""
    mask = (depth > 0).astype(np.uint8) * 255

    if method == 'median':
        kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
        mask_dilated = cv2.dilate(mask, kernel, iterations=2)
        depth_median = cv2.medianBlur(depth, 5)
        filled = np.where(mask == 0, depth_median, depth)
    elif method == 'linear':
        kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (15, 15))
        mask_dilated = cv2.dilate(mask, kernel, iterations=3)
        depth_expanded = depth.copy()
        depth_expanded[mask == 0] = 0
        filled = np.where(mask == 0, depth_expanded, depth)
    else:
        filled = depth

    return filled
```

def evaluate_depth_accuracy(our_depth, reference_depth, valid_mask=None):
    """评估深度估计精度"""
    if valid_mask is None:
        valid_mask = (our_depth > 0) & (reference_depth > 0)

    our_valid = our_depth[valid_mask]
    ref_valid = reference_depth[valid_mask]

    abs_error = np.abs(our_valid - ref_valid)
    rel_error = abs_error / ref_valid

    metrics = {
        'MAE': np.mean(abs_error),
        'RMSE': np.sqrt(np.mean(abs_error**2)),
        'REL': np.mean(rel_error),
    }

    print(f"MAE: {metrics['MAE']*1000:.2f} mm")
    print(f"RMSE: {metrics['RMSE']*1000:.2f} mm")
    print(f"REL: {metrics['REL']*100:.2f}%")

    return metrics
```

## 答案

### 练习 1 答案

1. **核心公式** `Z = f x B / d`：
   - Z：深度（米）
   - f：相机焦距（像素）
   - B：基线长度（米）
   - d：视差值（像素）

2. 由公式 Z = f x B / d 可知，Z 与 d 成反比。物体靠近相机时，视差变大，因此视差越大深度越近。

3. **理想双目模型**：经过校正后，左右相机共面且行对准。校正的目的是将 2D 匹配搜索从整个图像简化为沿水平扫描线进行，降低匹配复杂度。

4. **SGBM 与 BM 的区别**：BM 只在当前像素所在的水平行上进行匹配搜索（一维）；SGBM 在 8 个方向上累计匹配代价，能够更好地处理遮挡和边缘，精度更高。

```python
class TemporalMedianFilter:
    def __init__(self, window_size=5):
        self.window_size = window_size
        self.buffer = []
    def filter(self, depth_frame):
        self.buffer.append(depth_frame.copy())
        if len(self.buffer) > self.window_size:
            self.buffer.pop(0)
        return np.median(np.stack(self.buffer), axis=0).astype(np.float32)

def fill_depth_holes(depth, kernel_size=5):
    mask = (depth > 0).astype(np.uint8)
    kernel = np.ones((kernel_size, kernel_size), np.uint8)
    depth_dilated = cv2.dilate(depth.astype(np.float32), kernel)
    return np.where(mask == 0, depth_dilated, depth)
```

import numpy as np
import matplotlib.pyplot as plt

def evaluate_depth(our_depth, ref_depth):
    valid = (our_depth > 0) & (ref_depth > 0)
    our_v = our_depth[valid]
    ref_v = ref_depth[valid]
    abs_err = np.abs(our_v - ref_v)
    rel_err = abs_err / ref_v
    print(f"MAE: {np.mean(abs_err)*1000:.2f} mm")
    print(f"RMSE: {np.sqrt(np.mean(abs_err**2))*1000:.2f} mm")
    print(f"REL: {np.mean(rel_err)*100:.2f}%")
    plt.hist(rel_err * 100, bins=50, range=(0, 20))
    plt.xlabel('Relative Error (%)')
    plt.show()
