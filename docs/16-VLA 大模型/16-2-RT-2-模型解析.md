## 1. RT-2概述

### 1.1 Google Physical Intelligence团队

**RT-2**（Robotics Transformer 2）是由Google DeepMind旗下的**Physical Intelligence**团队于2023年7月发布的视觉-语言-动作（VLA）模型。该团队专注于将大规模语言模型和多模态模型的研究成果应用于机器人控制领域，致力于打造**通用具身智能体**（Generalist Robot Agent）。

Physical Intelligence团队的核心研究方向包括：
- 将互联网规模的预训练知识迁移到机器人控制
- 视觉-语言-动作统一建模
- 人形机器人通用控制策略
- 机器人模仿学习与强化学习

该团队的主要作品包括RT系列（RT-1、RT-2、RT-2-X）、π0系列等，是VLA领域最具影响力的研究团队之一。

### 1.2 RT-1到RT-2的演进

理解RT-2的创新，需要先了解其前身**RT-1**的设计思路及其局限性。

**RT-1（Robotics Transformer 1，2022）**

RT-1是Google在2022年发布的开创性工作，首次将Transformer架构应用于机器人控制。RT-1的核心设计：

- **FiLM条件网络**：使用Feature-wise Linear Modulation将语言指令嵌入到视觉特征中
- **RT-1 Transformer**：17个动作维度（7-DOF臂 + 2-DOF底座 + 1维夹爪 + 其他）通过Transformer编码器处理
- **数据规模**：在13台机器人上历时17个月采集的约15万个演示片段

RT-1的架构可表示为：

$$\mathbf{h}_{out} = \mathbf{h}_{in} \cdot (1 + \text{FiLM}(\mathbf{c}))$$

其中 $\mathbf{c}$ 是语言指令的嵌入，FiLM层对视觉特征进行调制。

**RT-1的局限性**：

尽管RT-1展示了跨任务和跨机器人的泛化能力，但其核心问题在于**缺乏互联网规模的预训练知识**。RT-1从零开始训练，无法利用GPT-4V、LLaVA等视觉-语言模型已经学到的丰富语义理解能力。

**RT-2的核心突破（2023）**

RT-2在RT-1的基础上，引入了**视觉-语言模型预训练**的思路：

| 对比维度 | RT-1 | RT-2 |
|----------|------|------|
| **预训练** | 无，从零训练 | 基于VLM预训练后动作微调 |
| **模型规模** | 13M参数 | 55B参数（Vision Transformer + T5-XL） |
| **知识来源** | 仅机器人演示数据 | 机器人数据 + 互联网图文数据 |
| **泛化能力** | 中等（依赖特定数据分布） | 强（VLM知识迁移） |
| **零样本能力** | 无 | 涌现零样本视觉推理能力 |

**从RT-1到RT-2的演进公式**：

RT-2的核心思想可以概括为：

$$p(\mathbf{a} | \mathbf{X}_{img}, \mathbf{X}_{text}) = \text{VLA-Transformer}(\mathbf{X}_{img}, \mathbf{X}_{text})$$

即直接将视觉图像和文本指令作为输入，输出机器人动作，而不再使用RT-1中的FiLM条件调制机制。

### 1.3 视觉-语言-动作模型的开创性

RT-2是**第一个真正意义上的视觉-语言-动作（VLA）统一模型**，其开创性体现在以下几个方面：

**（1）统一Token空间**

RT-2将动作预测任务重新定义为**序列到序列的生成任务**：将视觉Token、文本Token和动作Token统一到同一个词汇表中，使用标准的语言建模目标进行训练。

这与RT-1的两阶段设计（视觉→语言条件→动作）有本质区别：

```
RT-1:  图像 → [视觉编码器] → 视觉特征 → [FiLM条件注入] → [动作解码器] → 动作
       文本 → [语言编码器] → 语言嵌入 → (调制视觉特征)

RT-2:  图像 + 文本 + 动作 → [统一VLA Transformer] → 动作序列
                         (三者共享同一Token空间和模型参数)
```

**（2）互联网知识迁移**

RT-2能够将预训练VLM中学习的互联网知识迁移到机器人控制任务中。例如：

- **符号理解**：理解"把苹果放到印有水果图案的碗里"中的图案匹配概念
- **视觉推理**：理解"将物体推到与其颜色相同的区域"这类需要颜色推理的指令
- **常识知识**：理解"拿起一个可以被食用的东西"需要对物体属性的常识判断

**（3）涌现能力（Emergent Capabilities）**

RT-2展示了典型的**涌现能力**现象：随着模型规模增大和训练数据增加，模型自发涌现出在训练数据中从未明确监督的能力。

## 2. RT-2架构

### 2.1 基于PaLM-E和ViTGAN

RT-2的架构设计直接借鉴了**PaLM-E**（Google具身多模态语言模型）和**ViTGAN**（视觉Transformer与GAN结合）的技术方案。

**（1）视觉编码器：ViT（Vision Transformer）**

RT-2使用**Vision Transformer（ViT）** 作为视觉编码器，将输入图像分割成固定大小的Patch，映射为一系列视觉Token：

$$\mathbf{X}_{img} \in \mathbb{R}^{H \times W \times C} \xrightarrow{\text{ViT}} \mathbf{V} = [\mathbf{v}_1, \mathbf{v}_2, ..., \mathbf{v}_N]$$

RT-2使用的ViT规格：
- **ViT-L**：大尺寸版本，用于55B参数的RT-2
- **Patch Size**：16×16像素
- **预训练方式**：在海量互联网图像-文本对数据上对比学习预训练（如CLIP）

**（2）语言模型基座：T5-XL（或PaLM-E）**

RT-2的语言模型基座是Google的**T5-XL**（50B参数）或**PaLM-E**（具身多模态版本）：

$$\mathbf{X}_{text} \xrightarrow{\text{Tokenizer}} \mathbf{L} = [l_1, l_2, ..., l_M]$$

预训练目标为标准的语言建模损失。

**（3）架构集成：PaLM-E风格**

RT-2借鉴了PaLM-E的架构风格，将视觉编码器输出直接拼接到语言模型的输入序列中：

```
                    ┌─────────────────────────────────────────┐
图像 → [ViT编码器] → 视觉Token序列 [v_1, v_2, ..., v_N] ─────┐
                                                          │
文本 → [T5编码器] → 语言Token序列 [l_1, l_2, ..., l_M] ──┼→ [统一Transformer解码器] → 输出序列
                                                          │        (视觉+语言+动作Token统一处理)
动作 → 动作Token序列 [a_1, a_2, ..., a_D] ────────────────┘
```

### 2.2 动作表示方法

动作表示是RT-2的核心技术创新之一。RT-2提出了**将连续动作离散化到256个Token桶**的方法，使动作预测可以被建模为离散的分类问题。

**（1）动作维度定义**

| 维度编号 | 动作类型 | 描述 | 典型范围 |
|----------|----------|------|----------|
| 1-3 | 末端执行器位置增量 | Δx, Δy, Δz（相对于当前位置的位移） | [-0.05, 0.05] m |
| 4-6 | 末端执行器旋转增量 | Δroll, Δpitch, Δyaw（旋转变量） | [-0.2, 0.2] rad |
| 7 | 夹爪开合动作 | gripper连续开度 | [0, 1]（0=闭合，1=打开） |
| 8 | 底座线速度 | vx（前进/后退速度） | [-0.5, 0.5] m/s |
| 9 | 底座角速度 | ω（转向角速度） | [-1.0, 1.0] rad/s |
| 10 | 终止动作 | terminate（判断是否结束当前任务） | 0或1 |

共**10个动作维度**，其中terminate维度用于任务终止判断。

**（2）动作离散化方法**

对于每个连续动作维度 $a \in [a_{min}, a_{max}]$，RT-2的离散化公式为：

$$a_{discrete} = \text{Clamp}\left( \left\lfloor \frac{a - a_{min}}{a_{max} - a_{min}} \times 255 \right\rfloor, 0, 255 \right)$$

这里的关键设计：
- 将动作范围均匀划分为**256个桶**（对应8位二进制）
- 每个桶用一个离散的类别标签表示，$a_{discrete} \in \{0, 1, 2, ..., 255\}$

**（3）连续动作还原**

在推理阶段，模型输出离散动作索引后，需要还原为连续动作值：

$$a_{continuous} = a_{min} + \frac{a_{discrete}}{255} \times (a_{max} - a_{min})$$

**动作离散化的优势**：

| 优势 | 说明 |
|------|------|
| 与LLM统一 | 动作Token可直接复用语言Token的建模框架 |
| VLM知识迁移 | 预训练的视觉-语言对齐知识直接服务于动作预测 |
| 语义化动作 | 不同桶可能学到有意义的动作语义 |
| 多峰分布 | softmax天然支持一个动作维度的多峰分布 |

### 2.3 神经网络结构

**（1）整体结构图**

```
                    ┌──────────────────────────────────────────────────┐
                    │                   ViT 视觉编码器                    │
图像输入 → [Patch化] → [线性投影] → [ViT Transformer块] → 视觉Token序列 │
                    └──────────────────────────────────────────────────┘
                                              ↓
                    ┌──────────────────────────────────────────────────┐
                    │              T5/PaLM-E 语言模型主干                │
文本输入 → [Tokenizer] → 语言Token序列 ──────────────────────────────┼→ [统一Transformer解码器] → 动作Token序列
                    └──────────────────────────────────────────────────┘
```

**（2）模型参数规模对比**

| 组件 | RT-2（ViT-L + T5-XL） | RT-2（ViT-L + PaLM-E 8B） |
|------|----------------------|--------------------------|
| 视觉编码器 | ~300M | ~300M |
| 语言模型 | ~50B | ~8B |
| 动作预测头 | ~3M | ~3M |
| **总参数量** | **~55B** | **~12B** |

## 3. 训练方法

### 3.1 协同微调策略

RT-2的核心训练方法是**协同微调（Co-fine-tuning）**，即同时在机器人演示数据和视觉-语言数据上进行微调。

**（1）预训练基础**

预训练损失为标准的语言建模损失和图文对比损失：

$$\mathcal{L}_{pretrain} = \mathcal{L}_{LM} + \lambda \cdot \mathcal{L}_{contrastive}$$

**（2）协同微调的两类数据**

| 数据类型 | 来源 | 作用 |
|----------|------|------|
| **机器人演示数据** | 真实机器人遥操作采集 | 注入动作预测能力 |
| **视觉-语言数据** | 互联网图文对 | 保持和增强通用视觉-语言理解 |

**（3）训练目标**

$$\mathcal{L}_{RT-2} = \mathbb{E}_{(\mathbf{X}_{img}, \mathbf{X}_{text}, \mathbf{a}) \sim \mathcal{D}_{robot}} \left[ -\sum_{i=1}^{D} \sum_{k=0}^{255} \mathbb{1}[a_i = k] \log p_{\theta}(a_i = k | \mathbf{X}_{img}, \mathbf{X}_{text}) \right]$$

**（4）训练数据配比**

| 数据类型 | 配比 | 说明 |
|----------|------|------|
| 机器人演示数据 | ~1-5% | 核心动作预测能力来源 |
| 视觉-语言数据 | ~95-99% | 保持VLM能力，提供泛化 |

### 3.2 互联网规模数据

RT-2使用的视觉-语言预训练数据主要来自Google的**WebLI**（Web Language-Image）数据集：

| 统计项 | 数值 |
|--------|------|
| 图像-文本对数量 | ~100亿对 |
| 覆盖语言 | 100+种 |
| 图像来源 | 网页Alt文本配对 |
| 质量过滤 | 基于CLIP分数和质量分类器 |

**数据配比实验结果**：

| 实验 | 机器人数据比例 | VL数据比例 | 泛化性能 | 动作精度 |
|------|---------------|-----------|----------|----------|
| 仅机器人数据 | 100% | 0% | 低 | 高 |
| 1%机器人 + 99%VL | 1% | 99% | 最高 | 中等 |
| 5%机器人 + 95%VL | 5% | 95% | 次高 | 较高 |

结论：即使只使用1%的机器人数据，协同微调也能实现比纯机器人数据训练更好的泛化性能。

### 3.3 符号理解和推理能力

RT-2最引人注目的能力之一是**涌现出的符号理解和推理能力**：

| 推理类型 | 示例指令 | 所需能力 |
|----------|----------|----------|
| 颜色推理 | "把物体放到与其颜色相同的区域" | 颜色识别、物体-区域匹配 |
| 空间推理 | "把物体放到最左边的格子" | 空间位置理解、排序 |
| 属性推理 | "拿起一个可以用来盛水的容器" | 物体属性理解、功能推理 |
| 类比推理 | "像推多米诺骨牌一样推动这个物体" | 物理因果理解 |

涌现机制可用信息论解释：

$$I(\text{VLM知识}; \text{动作预测}) > 0$$

即VLM中存储的互联网知识与机器人动作预测任务存在信息重叠。

## 5. RT-2-X与后续版本

### 5.1 RT-2 Web灵敏度

### 5.3 后续版本发展路线

| 版本 | 时间 | 核心创新 |

## 6. 代码实战

### 6.1 RT-2核心组件概念实现

本节提供RT-2核心组件的概念性实现，帮助理解其工作原理。由于RT-2是Google内部模型，实际部署请使用OpenVLA等开源替代方案。

```python
import torch
import torch.nn as nn
import numpy as np
from PIL import Image
import json

# ==================== 1. 视觉编码器 (ViT) ====================
class VisionTransformer(nn.Module):
    """
    Vision Transformer 实现（简化版）

    图像 → Patch化 → 线性投影 → Transformer块 → 视觉Token序列
    """
    def __init__(self, image_size=224, patch_size=16, embed_dim=1024, num_layers=24):
        super().__init__()
        self.image_size = image_size
        self.patch_size = patch_size
        self.num_patches = (image_size // patch_size) ** 2  # 14×14 = 196
        self.embed_dim = embed_dim

        # Patch嵌入层：将每个patch映射到嵌入空间
        self.patch_embed = nn.Conv2d(
            in_channels=3,
            out_channels=embed_dim,
            kernel_size=patch_size,
            stride=patch_size
        )

        # 可学习的[CLS]Token（用于聚合全局信息）
        self.cls_token = nn.Parameter(torch.zeros(1, 1, embed_dim))

        # 位置编码
        self.pos_embed = nn.Parameter(
            torch.zeros(1, self.num_patches + 1, embed_dim)
        )

        # Transformer编码器块
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=embed_dim,
            nhead=16,
            dim_feedforward=embed_dim * 4,
            dropout=0.1,
            activation='gelu',
            batch_first=True
        )
        self.transformer = nn.TransformerEncoder(
            encoder_layer,
            num_layers=num_layers
        )

        # 输出投影层
        self.projection = nn.Linear(embed_dim, embed_dim)

    def forward(self, x):
        """
        前向传播

        参数:
            x: 输入图像张量 [B, C, H, W] = [B, 3, 224, 224]
        返回:
            visual_features: 视觉特征 [B, num_patches, embed_dim]
        """
        B = x.shape[0]

        # Patch嵌入：[B, 3, 224, 224] → [B, embed_dim, 14, 14] → [B, 196, embed_dim]
        x = self.patch_embed(x)  # [B, embed_dim, 14, 14]
        x = x.flatten(2).transpose(1, 2)  # [B, 196, embed_dim]

        # 添加[CLS]Token
        cls_tokens = self.cls_token.expand(B, -1, -1)  # [B, 1, embed_dim]
        x = torch.cat([cls_tokens, x], dim=1)  # [B, 197, embed_dim]

        # 添加位置编码
        x = x + self.pos_embed

        # 通过Transformer编码器
        x = self.transformer(x)

        # 移除[CLS]Token，返回所有patch的表示
        visual_features = x[:, 1:, :]  # [B, 196, embed_dim]

        return self.projection(visual_features)


# ==================== 2. 动作离散化器 ====================
class ActionDiscretizer:
    """
    RT-2动作离散化器

    将连续动作值映射到256个离散桶中
    """

    def __init__(self, action_bounds, num_bins=256):
        """
        初始化离散化器

        参数:
            action_bounds: list of (min_val, max_val) 每个动作维度的范围
            num_bins: 离散桶数量，默认256
        """
        self.action_bounds = action_bounds
        self.num_bins = num_bins

    def discretize(self, action):
        """
        将连续动作离散化为桶索引

        参数:
            action: 连续动作数组，shape = [D]
        返回:
            discretized: 离散动作索引数组，shape = [D]
        """
        discretized = []
        for i, (a, (a_min, a_max)) in enumerate(zip(action, self.action_bounds)):
            # 缩放到 [0, 255] 范围
            scaled = (a - a_min) / (a_max - a_min) * (self.num_bins - 1)
            # 取整并限制在 [0, 255]
            clipped = int(np.clip(scaled, 0, self.num_bins - 1))
            discretized.append(clipped)
        return np.array(discretized, dtype=np.int64)

    def continuousize(self, discretized_action):
        """
        将离散动作索引还原为连续值

        参数:
            discretized_action: 离散动作索引数组，shape = [D]
        返回:
            continuous: 连续动作数组，shape = [D]
        """
        continuous = []
        for i, (a_disc, (a_min, a_max)) in enumerate(
            zip(discretized_action, self.action_bounds)
        ):
            # 从桶索引还原到 [a_min, a_max] 范围
            a_cont = a_min + (a_disc / (self.num_bins - 1)) * (a_max - a_min)
            continuous.append(a_cont)
        return np.array(continuous, dtype=np.float32)


# ==================== 3. RT-2 动作预测模型（概念实现） ====================
class RT2ConceptualModel(nn.Module):
    """
    RT-2概念模型实现（简化版，仅用于教学）

    将视觉编码器、语言模型和动作预测头组合在一起
    """

    def __init__(
        self,
        visual_dim=1024,
        lang_dim=4096,
        hidden_dim=4096,
        num_action_dims=10,
        num_bins=256,
        vocab_size=32000,
    ):
        super().__init__()
        self.num_action_dims = num_action_dims
        self.num_bins = num_bins
        self.vocab_size = vocab_size

        # 视觉编码器
        self.visual_encoder = VisionTransformer(
            image_size=224,
            patch_size=16,
            embed_dim=visual_dim,
            num_layers=24
        )

        # 跨模态投影：将视觉特征映射到语言模型空间
        self.visual_projection = nn.Linear(visual_dim, hidden_dim)

        # 语言Embedding层
        self.token_embedding = nn.Embedding(vocab_size, hidden_dim)

        # Transformer解码器（融合视觉和语言信息）
        decoder_layer = nn.TransformerDecoderLayer(
            d_model=hidden_dim,
            nhead=16,
            dim_feedforward=hidden_dim * 4,
            dropout=0.1,
            activation='gelu',
            batch_first=True
        )
        self.transformer_decoder = nn.TransformerDecoder(
            decoder_layer,
            num_layers=24
        )

        # 动作预测头：对每个动作维度输出256类分类
        self.action_heads = nn.ModuleList([
            nn.Sequential(
                nn.Linear(hidden_dim, hidden_dim // 2),
                nn.GELU(),
                nn.Linear(hidden_dim // 2, num_bins)
            )
            for _ in range(num_action_dims)
        ])

    def forward_visual(self, images):
        """
        处理图像输入，提取视觉特征

        参数:
            images: 图像张量 [B, 3, 224, 224]
        返回:
            visual_features: 视觉Token序列 [B, 196, hidden_dim]
        """
        # 通过ViT编码器
        visual_tokens = self.visual_encoder(images)  # [B, 196, visual_dim]
        # 投影到语言模型空间
        visual_features = self.visual_projection(visual_tokens)  # [B, 196, hidden_dim]
        return visual_features

    def forward_action(self, visual_features, text_tokens):
        """
        动作预测前向传播

        参数:
            visual_features: 视觉特征 [B, 196, hidden_dim]
            text_tokens: 文本Token [B, seq_len]
        返回:
            action_logits: 每个动作维度的分类logits [B, num_action_dims, num_bins]
        """
        # 文本嵌入
        text_embeddings = self.token_embedding(text_tokens)  # [B, seq_len, hidden_dim]

        # 拼接视觉和文本特征作为解码器输入
        memory = visual_features  # [B, 196, hidden_dim]
        tgt = text_embeddings     # [B, seq_len, hidden_dim]

        # 通过Transformer解码器
        decoder_output = self.transformer_decoder(
            tgt=tgt,
            memory=memory
        )  # [B, seq_len, hidden_dim]

        # 取最后一个token的表示作为动作预测依据
        last_hidden = decoder_output[:, -1, :]  # [B, hidden_dim]

        # 对每个动作维度预测256类
        action_logits = torch.stack([
            head(last_hidden) for head in self.action_heads
        ], dim=1)  # [B, num_action_dims, num_bins]

        return action_logits

    def predict_action(self, visual_features, text_tokens):
        """
        推理时预测动作

        参数:
            visual_features: 视觉特征
            text_tokens: 文本Token
        返回:
            discretized_actions: 离散动作索引 [num_action_dims]
        """
        self.eval()
        with torch.no_grad():
            action_logits = self.forward_action(visual_features, text_tokens)
            # 贪心解码：取每个维度概率最高的桶
            discretized_actions = torch.argmax(action_logits, dim=-1)  # [num_action_dims]
        return discretized_actions.squeeze(0).cpu().numpy()

        0.8,                     # gripper
        0.1,                     # vx
        0.05,                    # ω
        0.0,                     # terminate
    ])

    print("原始连续动作:")
    for name, val in zip(action_names, continuous_action):
        print(f"  {name}: {val:.4f}")

    # 离散化
    discretized_action = discretizer.discretize(continuous_action)
    print("\n离散化后的动作索引 (0-255):")
    for name, val in zip(action_names, discretized_action):
        print(f"  {name}: {val}")

    # 还原为连续动作
    restored_action = discretizer.continuousize(discretized_action)
    print("\n还原后的连续动作:")
    for name, val in zip(action_names, restored_action):
        print(f"  {name}: {val:.4f}")

    # 计算还原误差
    error = np.abs(continuous_action - restored_action)
    print("\n还原误差:")
    for name, err in zip(action_names, error):
        print(f"  {name}: {err:.6f}")

def compute_rt2_loss():
    """
    演示RT-2动作预测损失计算过程
    """
    print("\n=== RT-2 动作预测损失计算演示 ===")

    # 模拟参数
    batch_size = 4
    num_action_dims = 10
    num_bins = 256

    # 模拟真实动作标签（离散索引）
    # 假设每个动作维度的真实桶索引
    target_actions = torch.randint(0, num_bins, (batch_size, num_action_dims))
    print(f"目标动作索引 shape: {target_actions.shape}")
    print(f"目标动作示例（第一个样本）: {target_actions[0].tolist()}")

    # 模拟模型输出的logits [B, num_action_dims, num_bins]
    # 初始化为均匀分布，然后给目标类别稍高的分数
    logits = torch.randn(batch_size, num_action_dims, num_bins) * 0.1

    # 对目标类别添加偏移，模拟模型有一定预测能力
    for b in range(batch_size):
        for d in range(num_action_dims):
            target = target_actions[b, d].item()
            logits[b, d, target] += 1.5  # 提高目标类别的logit

    print(f"动作预测logits shape: {logits.shape}")

    # 计算交叉熵损失
    # 每个动作维度独立计算256类分类损失
    loss = nn.CrossEntropyLoss(reduction='none')

    total_loss = 0.0
    for d in range(num_action_dims):
        # 取第d个动作维度的logits和目标
        logits_d = logits[:, d, :]           # [B, 256]
        target_d = target_actions[:, d]       # [B]
        loss_d = loss(logits_d, target_d)     # [B]
        total_loss += loss_d.mean()

    # RT-2损失 = 所有动作维度损失之和
    print(f"\n总动作预测损失: {total_loss.item():.4f}")
    print("（此损失在反向传播中用于更新模型参数）")

        执行一步控制

        参数:
            observation_image: 当前观察图像 [224, 224, 3]

        # 预测离散动作
        discretized_action = self.model.predict_action(visual_features, text_tokens)

        # 还原为连续动作
        continuous_action = self.discretizer.continuousize(discretized_action)

        return continuous_action
```

```python
import torch
import numpy as np

# ==================== RT-2 推理流程总览 ====================
"""
RT-2 推理流程分为以下几个步骤：

1. 图像预处理：将相机拍摄的RGB图像resize并归一化
2. 视觉编码：通过ViT编码器提取视觉特征
3. 文本编码：通过Tokenizer将语言指令转为Token序列
4. 跨模态融合：将视觉Token和文本Token拼接输入Transformer
5. 动作预测：解码器输出每个动作维度的256类分类logits
6. 动作离散化解码：取每个维度概率最高的桶索引
7. 动作还原：将离散索引还原为连续动作值
8. 机器人执行：将连续动作发送给机器人控制器执行

流程图：

相机图像 → [预处理] → 归一化张量
                           ↓
                   [ViT视觉编码器] → 视觉Token序列
                                            ↓
语言指令 → [Tokenizer] → 文本Token序列 ──→ [Transformer融合] → [D×256分类器] → 离散动作索引
                                                                                    ↓
                                                                         [动作还原] → 连续动作
                                                                                    ↓
                                                                           机器人执行器
"""

def rt2_inference_step(observation_image, language_instruction, model, tokenizer, discretizer):
    """
    RT-2单步推理函数

    参数:
        observation_image: 相机观察图像 [H, W, 3] 或 [3, H, W] 张量
        language_instruction: 自然语言指令字符串
        model: RT-2模型（PyTorch nn.Module）
        tokenizer: 文本Tokenizer
        discretizer: 动作离散化器
    返回:
        action: 连续动作向量 [num_action_dims]
    """
    # Step 1: 图像预处理
    if isinstance(observation_image, np.ndarray):
        # numpy数组转PyTorch张量
        img_tensor = torch.from_numpy(observation_image).float()
        # 确保是 [C, H, W] 格式
        if img_tensor.ndim == 3 and img_tensor.shape[-1] == 3:
            img_tensor = img_tensor.permute(2, 0, 1)
        # 归一化到 [0, 1]
        img_tensor = img_tensor / 255.0
        # 添加batch维度
        img_tensor = img_tensor.unsqueeze(0)  # [1, 3, H, W]
    else:
        img_tensor = observation_image

    # Step 2: 视觉编码
    visual_features = model.forward_visual(img_tensor)  # [1, 196, hidden_dim]

    # Step 3: 文本编码
    text_tokens = tokenizer.encode(language_instruction)
    text_tokens = torch.tensor([text_tokens], dtype=torch.long)  # [1, seq_len]

    # Step 4-6: 动作预测
    discretized_action = model.predict_action(visual_features, text_tokens)

    # Step 7: 动作还原
    continuous_action = discretizer.continuousize(discretized_action)

    return continuous_action


# ==================== 动作执行循环 ====================
class RT2RobotController:
    """
    RT-2 机器人控制器

    封装RT-2模型调用和闭环控制逻辑
    """

    def __init__(self, model, tokenizer, discretizer, control_frequency=10):
        """
        初始化控制器

        参数:
            model: RT-2模型
            tokenizer: 文本Tokenizer
            discretizer: 动作离散化器
            control_frequency: 控制频率（Hz）
        """
        self.model = model
        self.tokenizer = tokenizer
        self.discretizer = discretizer
        self.control_frequency = control_frequency
        self.step_interval = 1.0 / control_frequency

    def run_episode(self, get_observation_fn, execute_action_fn, instruction, max_steps=100):
        """
        运行一个完整的任务片段

        参数:
            get_observation_fn: 获取当前观察的函数，返回(图像, 指令)
            execute_action_fn: 执行动作的函数，输入连续动作向量
            instruction: 任务指令
            max_steps: 最大步数
        返回:
            episode_reward: 累计奖励
            success: 是否成功完成任务
        """
        import time

        episode_reward = 0.0
        success = False

        for step in range(max_steps):
            step_start = time.time()

            # 获取当前观察
            observation_image = get_observation_fn()

            # RT-2推理
            action = rt2_inference_step(
                observation_image,
                instruction,
                self.model,
                self.tokenizer,
                self.discretizer
            )

            # 执行动作
            reward = execute_action_fn(action)
            episode_reward += reward

            # 检查是否成功（terminate维度）
            if len(action) >= 10 and action[9] > 0.5:
                success = True
                break

            # 控制频率同步
            elapsed = time.time() - step_start
            sleep_time = max(0, self.step_interval - elapsed)
            time.sleep(sleep_time)

            if step % 10 == 0:
                print(f"步骤 {step}/{max_steps} | 累计奖励: {episode_reward:.2f}")

        return episode_reward, success
```

### 简答题答案

**16. RT-1与RT-2架构区别及零样本泛化原因：**

| 区别 | RT-1 | RT-2 |
|------|------|------|
| 架构 | FiLM条件网络调制视觉特征 | 统一VLA架构，视觉+语言+动作Token统一处理 |
| 预训练 | 无预训练，从零训练 | 基于VLM预训练后动作微调 |
| 知识来源 | 仅机器人数据 | 机器人数据 + 互联网图文数据 |

RT-2零样本泛化能力的原因：预训练的VLM已经从互联网图文数据中学习了丰富的视觉概念和语言对应关系。当RT-2在机器人数据上微调时，这些知识并没有被覆盖，而是被泛化到了新的动作预测任务中。

**17. RT-2动作离散化及VLM知识迁移机制：**

动作离散化过程：将连续动作 $a \in [a_{min}, a_{max}]$ 映射到 $a_{discrete} \in \{0, 1, ..., 255\}$

$$a_{discrete} = \text{Clamp}\left( \left\lfloor \frac{a - a_{min}}{a_{max} - a_{min}} \times 255 \right\rfloor, 0, 255 \right)$$

VLM知识迁移的原理：
1. 动作Token与文本Token共享同一词汇表和Embedding空间
2. 预训练的LLM已经学会了视觉概念→语言→动作的隐式关联（如"红色"→"拿起红色物体"）
3. 离散化后，动作预测被建模为256类分类，与语言建模任务形式一致
4. VLM的注意力机制可以在视觉Token和动作Token之间建立直接联系

**18. 协同微调"少数据高能力"的原因：**

1-5%的机器人数据之所以有效，是因为：
- **语义先验**：VLM已经学过"拿起"、"放下"等概念的视觉表达，机器人数据只需微调动作精度
- **动作空间简单**：仅10个维度，256^10 的可能动作空间通过语义聚类大幅简化
- **信息重叠**：VLM知识与机器人动作存在大量信息重叠（如物体识别、空间关系）
- **表示复用**：视觉编码器和语言模型的表示在协同微调中被复用

**19. RT-2泛化能力提升的分析：**

| 角度 | RT-1 | RT-2 |
|------|------|------|
| **数据** | 仅机器人演示数据（~150K片段） | 机器人数据 + 互联网图文数据（~10亿图文对） |
| **架构** | FiLM条件调制，视觉和语言分离 | 统一VLA，视觉、语言、动作Token深度融合 |
| **预训练** | 无预训练 | VLM预训练，继承互联网知识 |

**20. 移动Manipulator机器人RT-2部署方案：**

**a) 动作空间定义：**

| 动作维度 | 表示 | 范围 | 说明 |
|----------|------|------|------|
| 1-6 | 关节角度增量 Δq | [-0.05, 0.05] rad | 6-DOF机械臂 |
| 7 | 夹爪开合 g | [0, 1] | 0=闭合，1=打开 |
| 8-9 | 底盘线速度 vx, vy | [-0.3, 0.3] m/s | 麦克纳姆轮全向移动 |
| 10 | 底盘角速度 ω | [-1.0, 1.0] rad/s | 转向 |
| 11 | terminate | [0, 1] | 任务终止 |

共11个动作维度。

**b) 桶边界设计：**

- 均匀分桶：对于关节角度增量、夹爪等动作，使用均匀分桶
- 数据驱动分桶：对于底盘速度，根据实际运动数据调整桶边界，使高频动作区域有更细的分桶

**c) 实时性优化方案（目标30Hz）：**

| 优化方法 | 预期效果 | 说明 |
|----------|----------|------|
| INT4量化 | 延迟降低50-60% | 55B→约14B显存 |
| 蒸馏到7B | 延迟降低4-5倍 | 用OpenVLA等开源模型 |
| 异步执行 | 主频由控制器决定 | VLA推理与高频控制解耦 |
| 动作缓存 | 静态场景跳过推理 | 动作执行时间>推理延迟时有效 |
