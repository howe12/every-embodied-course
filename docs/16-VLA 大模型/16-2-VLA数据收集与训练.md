## 1. VLA数据收集概述

### 1.1 数据是VLA的核心瓶颈

VLA（Vision-Language-Action）模型的性能高度依赖于**数据质量与数据规模**。与语言模型类似，VLA遵循"缩放定律"（Scaling Law）：在一定范围内，数据量越大、模型规模越大，VLA的泛化能力越强。然而，与纯文本数据不同，VLA所需的**视觉-语言-动作三模态同步数据**极为稀缺和昂贵，这成为制约VLA发展的核心瓶颈。

VLA数据收集面临以下独特挑战：

| 挑战 | 描述 | 影响 |
|------|------|------|
| **数据采集成本高** | 需要真实机器人或高保真仿真器 + 人类操作员 | 规模化困难 |
| **动作标注困难** | 机器人动作难以自动化标注，需要专家参与 | 标注成本高 |
| **多模态同步难** | 视觉、语言、动作三者在时序上精确对齐 | 数据质量受限 |
| **领域差异大** | 不同机器人形态、数据格式差异大 | 跨机器人泛化难 |
| **安全约束** | 真实机器人操作存在碰撞风险 | 限制数据多样性 |

### 1.2 VLA数据采集方法分类

VLA数据采集方法可以按照数据来源分为四大类：

**（1）真实世界遥操作数据**

通过人类操作员使用各种设备远程控制机器人，采集示范轨迹。这是当前最高质量的数据来源，但成本最高。

**（2）动捕设备数据**

使用光学或惯性动捕系统捕捉人体动作，将人类动作映射到机器人动作。适合大规模采集人体操作数据。

**（3）仿真数据（Sim2Real）**

在仿真环境中生成数据，利用域随机化和物理引擎生成多样化机器人数据。成本低、可规模化，但存在"sim2real gap"。

**（4）视频数据**

从互联网视频中学习人类动作模式（如YouTube烹饪视频）。规模最大但缺乏精确动作标签。

```
┌─────────────────────────────────────────────────────┐
│                   VLA数据采集方法                     │
├─────────────┬─────────────┬─────────────┬───────────┤
│  遥操作数据   │   动捕数据   │   仿真数据   │  视频数据  │
│   (Teleop)   │  (Mocap)   │ (Sim2Real)  │  (Video)  │
├─────────────┼─────────────┼─────────────┼───────────┤
│   质量最高   │   质量高    │   质量中    │  质量低   │
│   成本最高   │   成本中高  │   成本低    │  成本最低  │
│   规模小    │   规模中    │   规模大    │  规模极大  │
└─────────────┴─────────────┴─────────────┴───────────┘
```

## 2. 遥操作数据采集

### 2.1 遥操作原理

遥操作（Teleoperation）是指人类操作员在不同物理位置或使用不同交互设备，远程控制机器人执行任务的数据采集方式。操作员的视觉观察和决策被实时转化为机器人的动作指令，这些（观察 → 动作）对构成了VLA模仿学习的最直接监督信号。

遥操作数据采集的数学表示：

$$\mathcal{D}_{teleop} = \{(\mathbf{X}_{img}^{(t)}, \mathbf{X}_{text}^{(t)}, \mathbf{a}^{(t)})\}_{t=0}^{T}$$

其中：
- $\mathbf{X}_{img}^{(t)}$：$t$ 时刻机器人相机拍摄的图像
- $\mathbf{X}_{text}^{(t)}$：$t$ 时刻对应的语言指令或描述
- $\mathbf{a}^{(t)}$：$t$ 时刻操作员控制机器人执行的动作
- $T$：一个演示片段的总步数

### 2.2 遥操作设备

**（1）示教器（Teach Pendant）**

示教器是最传统的机器人编程方式，操作员通过手持设备手动设置机器人关节角度或末端位置，记录关键点位后回放。

| 特点 | 说明 |
|------|------|
| 精度 | 高（直接控制关节/末端） |
| 易用性 | 低（需要专业培训） |
| 效率 | 低（点位录制耗时） |
| 成本 | 低（机器人自带） |
| 适用场景 | 工业机械臂编程 |

**（2）VR遥操作**

使用VR头显和手柄进行遥操作，操作员的自然手部动作通过运动学映射到机器人末端执行器。

```
VR遥操作流程：
操作员手部动作（VR手柄追踪）
        ↓
    运动学映射
        ↓
机器人末端目标位姿
        ↓
低层伺服控制器跟踪
        ↓
执行动作 + 采集数据
```

代表系统：
- **Meta Quest + controller**：消费级VR方案
- **HTC Vive**：高精度追踪方案
- **Manus VR**：专业动捕手套

**（3）Teleop via Camera**

使用相机捕捉操作员手部/身体姿态，通过姿态估计算法映射到机器人动作。

| 方法 | 描述 | 精度 |
|------|------|------|
| 单目RGB | 2D姿态估计算法（如MediaPipe） | 中等 |
| 深度相机 | Azure Kinect、RealSense深度姿态 | 较高 |
| 多视角 | 多相机协同捕捉 | 高 |
| 手套传感器 | 数据手套直接测量关节角度 | 高 |

**（4）Shadow Hand / 灵巧手遥操作**

使用仿人型五指灵巧手（如Shadow Robot Hand）进行精细操作数据的采集，适合需要精细抓取的任务。

### 2.3 遥操作数据采集系统

一个完整的遥操作数据采集系统包含以下组件：

**硬件层**：
- 机器人本体（机械臂 + 夹爪/灵巧手）
- 相机系统（眼到手/眼固定）
- 遥操作设备（VR/示教器/相机）
- 计算单元（实时处理）

**软件层**：
- 机器人控制接口（ROS/ROS2）
- 遥操作映射模块
- 数据记录模块
- 质量控制模块

**数据记录格式**：

```python
# 遥操作采集的原始数据结构
teleop_sample = {
    "timestamp": 1700000000.123,          # Unix时间戳（秒）
    "episode_id": "ep_001",               # 片段ID
    "timestep": 50,                       # 当前时间步
    "rgb_image": "path/to/rgb_050.jpg",   # RGB图像路径
    "depth_image": "path/to/depth_050.png", # 深度图像路径（可选）
    "instruction": "把红色积木放到蓝色盒子左边",  # 任务语言指令
    "robot_state": {                      # 机器人本体状态
        "joint_positions": [0.1, -0.3, 0.5, 0.8, -0.2, 0.6, 0.0],  # 7个关节角度
        "joint_velocities": [0.01, -0.02, 0.01, 0.0, 0.0, 0.0, 0.0], # 关节速度
        "gripper_position": 0.5,          # 夹爪开合程度 [0=闭合, 1=打开]
        "gripper_force": 5.2,             # 夹爪抓力（N）
        "end_effector_pose": [0.3, 0.1, 0.5, 0.0, 0.0, 0.0, 1.0], # 末端位姿 (x,y,z,qx,qy,qz,qw)
    },
    "action": {                           # 操作员执行的动作（标签）
        "arm_delta": [0.001, 0.002, 0.0005, 0.0, 0.0, 0.0],  # 末端位置增量（相对）
        "gripper_delta": 0.0,              # 夹爪开合增量
    },
    "operator_hand_pose": [[...], [...]], # 操作员手部21关键点3D坐标（可选）
    "vr_controller_pose": [[...], [...]], # VR手柄位姿（可选）
}
```

## 3. 仿真数据生成

### 3.1 Sim2Real概述

Sim2Real（Simulation to Reality）是利用仿真环境生成机器人训练数据，再迁移到真实机器人的技术范式。其核心动机是：**真实机器人数据采集成本高、耗时长、安全风险大，而仿真可以低成本、大规模、零风险地生成数据**。

Sim2Real的核心挑战是**域差距（Domain Gap）**：

| 域差异来源 | 仿真 | 真实 |
|------------|------|------|
| 视觉外观 | 纹理简单、材质模型不准确 | 复杂光照、遮挡、噪声 |
| 物理 dynamics | 参数不精确、简化模型 | 复杂摩擦、柔性、延迟 |
| 传感器噪声 | 可建模但可能不准确 | 复杂噪声特性 |
| 接触力学 | 简化碰撞模型 | 复杂接触摩擦 |

域随机化（Domain Randomization）是解决Sim2Real Gap的主要方法。

### 3.2 域随机化

域随机化（Domain Randomization, DR）的核心思想是：**在仿真中随机化尽可能多的视觉和物理参数，使真实世界成为仿真分布的一个样本，从而迫使模型学习到域不变特征**。

**视觉域随机化**：

| 参数类别 | 随机化范围 | 说明 |
|----------|------------|------|
| 光照 | 强度 ×[0.5, 2.0]，色温变化 | 模拟不同照明条件 |
| 背景纹理 | 从纹理库随机采样 | 消除背景依赖 |
| 物体纹理 | 颜色、材质随机 | 提高物体识别鲁棒性 |
| 相机参数 | 内参、畸变系数随机 | 模拟不同相机 |
| 添加噪声 | 高斯噪声、运动模糊 | 提高抗噪能力 |
| 遮挡 | 随机添加干扰物 | 模拟真实遮挡 |

**物理域随机化**：

| 参数类别 | 随机化范围 | 说明 |
|----------|------------|------|
| 物体质量 | ±20% 随机 | 适应不同重量物体 |
| 摩擦系数 | 0.3 ~ 1.2 | 适应不同表面 |
| 关节刚度 | ±10% 随机 | 适应关节特性差异 |
| 动作延迟 | 0 ~ 50ms 随机 | 模拟控制延迟 |
| 观察噪声 | 高斯噪声注入 | 传感器不确定性 |

**域随机化的数学表示**：

设真实数据分布为 $p_{real}(\mathbf{x})$，仿真数据分布为 $p_{sim}(\mathbf{x}|\phi)$，其中 $\phi$ 是随机化参数。域随机化通过最大化 $\phi$ 的多样性，使：

$$p_{sim}(\mathbf{x}) = \mathbb{E}_{\phi \sim p(\phi)}[p_{sim}(\mathbf{x}|\phi)]$$

覆盖真实分布 $p_{real}(\mathbf{x})$。

## 5. 动作标注与预处理

### 5.1 动作空间定义

动作空间定义是VLA数据处理的第一步，决定了模型输出的物理意义。主要有两种动作空间定义方式：

**（1）关节空间（Joint Space）**

直接输出机器人各关节的目标角度或增量：

$$\mathbf{a}_t^{joint} = [q_1, q_2, ..., q_J, g]$$

其中 $q_j$ 是第 $j$ 个关节的角度，$g$ 是夹爪开合度，$J$ 是关节数。

| 优点 | 缺点 |
|------|------|
| 与机器人控制直接对应 | 高维（7-DOF臂 = 8维） |
| 无奇异位形问题（相对增量） | 需要精确的关节控制 |
| 碰撞检测在关节空间更精确 | 动作语义不直观 |

**（2）末端执行器空间（End-Effector Space）**

输出末端执行器的位姿增量或速度：

$$\mathbf{a}_t^{EE} = [\Delta x, \Delta y, \Delta z, \Delta roll, \Delta pitch, \Delta yaw, g]$$

| 优点 | 缺点 |
|------|------|
| 动作语义直观（移动、旋转） | 可能遇到奇异位形 |
| 与视觉伺服控制天然结合 | 逆运动学求解可能无解 |
| 低维（7维 vs 8维关节空间） | 需要额外的IK求解器 |

**（3）混合表示**

部分系统同时输出末端执行器姿态和夹爪状态，在高层使用EE空间，在低层转换为关节空间。

### 5.2 动作时序对齐

多模态数据在时序上的精确对齐是数据预处理的关键。常见问题：

**（1）动作延迟**

由于控制延迟和传感器延迟，动作标签 $\mathbf{a}_t$ 与观察 $\mathbf{X}_{img}^{(t)}$ 之间存在时间偏移：

$$\mathbf{a}_t^{true} \approx \mathbf{a}_{t-\tau}^{recorded}$$

解决方案：使用时序对齐技术（Time Alignment），常用方法：
- **手动对齐**：根据硬件延迟补偿
- **互相关对齐**：通过统计方法估计延迟 $\tau$
- **额外传感器**：使用精确时钟同步

**（2）多模态数据频率不一致**

| 模态 | 典型频率 |
|------|----------|
| RGB图像 | 10-30 Hz |
| 关节状态 | 100-1000 Hz |
| 语言指令 | 一次性（无频率） |
| 动作输出 | 10-100 Hz |

解决方案：插值 + 下采样到统一频率（如10Hz）

```python
# 动作时序对齐示例
def align_trajectory_timesteps(
    rgb_timestamps,      # 图像时间戳列表
    joint_timestamps,     # 关节状态时间戳列表
    action_timestamps,    # 动作时间戳列表
    target_frequency=10   # 目标频率 Hz
):
    """
    将多模态数据对齐到统一时间轴

    参数:
        rgb_timestamps: 图像时间戳
        joint_timestamps: 关节状态时间戳
        action_timestamps: 动作标签时间戳
        target_frequency: 目标采样频率
    返回:
        aligned_data: 对齐后的数据数组
    """
    # 计算统一时间轴
    t_start = max(rgb_timestamps[0], joint_timestamps[0], action_timestamps[0])
    t_end = min(rgb_timestamps[-1], joint_timestamps[-1], action_timestamps[-1])
    dt = 1.0 / target_frequency
    aligned_timestamps = np.arange(t_start, t_end, dt)

    # 对每个模态进行插值
    aligned_rgb = interpolate_to_timestamps(rgb_timestamps, aligned_timestamps, mode='nearest')
    aligned_joints = interpolate_to_timestamps(joint_timestamps, aligned_timestamps, mode='linear')
    aligned_actions = interpolate_to_timestamps(action_timestamps, aligned_timestamps, mode='linear')

    return {
        'timestamps': aligned_timestamps,
        'rgb_indices': aligned_rgb,       # 索引列表，指向最近的原始图像
        'joint_states': aligned_joints,
        'actions': aligned_actions,
    }
```

### 5.3 多模态数据同步

VLA训练需要将视觉、语言、动作三者在样本级别对齐。常见的多模态同步策略：

**策略1：每个时间步独立对齐**

每个训练样本 $(\mathbf{X}_{img}^{(t)}, \mathbf{X}_{text}^{(t)}, \mathbf{a}^{(t)})$ 来自同一时刻 $t$，是标准做法。

**策略2：语言指令跨步广播**

语言指令在整个片段内保持不变：

```
Episode: [动作: 拿起 → 移动 → 放下]

样本1: (图像_0, "拿起红色积木放到蓝色盒子中", 动作_0)
样本2: (图像_1, "拿起红色积木放到蓝色盒子中", 动作_1)
样本3: (图像_2, "拿起红色积木放到蓝色盒子中", 动作_2)
```

**策略3：密集动作标注**

每隔 $k$ 帧提供一次动作标签，中间帧通过插值获得：

```python
def dense_action_labeling(episode_images, sparse_actions, sparse_indices, k=5):
    """
    密集动作标注：将稀疏动作标签插值到每帧

    参数:
        episode_images: 片段图像列表
        sparse_actions: 稀疏动作标签列表
        sparse_indices: 稀疏动作对应的图像索引
        k: 每隔k帧标注一次
    返回:
        dense_actions: 密集动作标签（每帧一个）
    """
    n_frames = len(episode_images)
    dense_actions = []

    for i in range(n_frames):
        # 找到最近的两个稀疏标注点
        left_idx = max([j for j in sparse_indices if j <= i], default=0)
        right_idx = min([j for j in sparse_indices if j > i], default=sparse_indices[-1])

        if left_idx == right_idx:
            # 正好落在标注点上
            action = sparse_actions[sparse_indices.index(left_idx)]
        else:
            # 线性插值
            alpha = (i - left_idx) / (right_idx - left_idx)
            action = (1 - alpha) * sparse_actions[sparse_indices.index(left_idx)] + \
                     alpha * sparse_actions[sparse_indices.index(right_idx)]

        dense_actions.append(action)

    return dense_actions
```

### 5.4 数据质量筛选与清洗

原始采集数据通常包含大量低质量样本，需要系统性的筛选和清洗。

**常见质量问题**：

| 问题类型 | 描述 | 筛选方法 |
|----------|------|----------|
| 轨迹中断 | 机器人急停或通信中断 | 检测连续性 |
| 动作异常 | 动作值超出合理范围 | 阈值过滤 |
| 视觉失效 | 相机遮挡、曝光过度 | 图像质量检测 |
| 任务失败 | 没有完成预期目标 | 成功率过滤 |
| 重复帧 | 多帧内容完全相同 | 哈希去重 |
| 速度异常 | 关节速度突变 | 平滑性检测 |

**自动化筛选流水线**：

```python
def filter_trajectory_quality(trajectory, config):
    """
    自动筛选高质量轨迹

    参数:
        trajectory: 原始轨迹数据
        config: 筛选配置
    返回:
        is_valid: 是否通过筛选
        reason: 失败原因（如果未通过）
        quality_score: 质量分数
    """
    # 1. 长度检查：轨迹不能太短
    min_length = config.get('min_length', 10)
    if len(trajectory['actions']) < min_length:
        return False, "轨迹长度不足", 0.0

    # 2. 动作范围检查：动作值应在合理范围内
    for i, action in enumerate(trajectory['actions']):
        if np.any(np.abs(action) > config.get('action_bound', 100)):
            return False, f"动作值异常（第{i}帧）", 0.0

    # 3. 速度平滑性检查：关节速度不应突变
    velocities = compute_velocities(trajectory['actions'])
    max_velocity = config.get('max_velocity', 10.0)
    if np.any(np.abs(velocities) > max_velocity):
        return False, "动作速度异常（可能急停）", 0.0

    # 4. 视觉质量检查
    for img_path in trajectory['images']:
        img_quality = compute_image_quality(img_path)
        if img_quality < config.get('min_image_quality', 0.5):
            return False, f"图像质量过低: {img_path}", 0.0

    # 5. 任务成功率（如果有）
    if 'success' in trajectory:
        if not trajectory['success']:
            return False, "任务未成功完成", 0.0

    # 计算综合质量分数
    quality_score = min(1.0, len(trajectory['actions']) / 100)  # 简单示例
    return True, "通过", quality_score
```

## 6. VLA训练策略

### 6.1 预训练阶段

VLA预训练的目标是学习通用的**视觉-语言对齐表示**，为后续的动作微调奠定基础。

**预训练数据来源**：

| 数据类型 | 规模 | 作用 |
|----------|------|------|
| 图文对（LAION-2B等） | 亿级 | 视觉-语言基础对齐 |
| 视频-文本（WebVid等） | 百万级 | 动态视觉理解 |
| 机器人图像（无动作） | 万级 | 领域适应 |

**预训练损失函数**：

VLA预训练通常使用标准的VLM预训练损失：

$$\mathcal{L}_{pretrain} = \mathcal{L}_{LM} + \lambda_1 \cdot \mathcal{L}_{contrastive} + \lambda_2 \cdot \mathcal{L}_{ITM}$$

其中：
- $\mathcal{L}_{LM}$：语言建模损失（Next Token Prediction）
- $\mathcal{L}_{contrastive}$：图文对比损失（InfoNCE）
- $\mathcal{L}_{ITM}$：图文匹配损失（二分类）

**预训练参数策略**：

| 模块 | 参数策略 | 说明 |
|------|----------|------|
| 视觉编码器 | 冻结或低学习率微调 | 保持CLIP预训练知识 |
| 语言模型 | 冻结 | 保持语言能力 |
| 模态对齐层 | 全量训练 | 学习视觉-语言对齐 |

### 6.2 动作微调阶段

动作微调（Action Fine-tuning）是VLA训练的核心阶段，将动作预测能力从机器人演示数据中注入到模型中。

**动作微调的核心问题**：

如何将动作预测融入基于语言建模的框架？主要有两种方法：

**方法1：动作离散化（RT-2/OpenVLA路线）**

将连续动作离散化为256个桶，每个桶对应一个Token：

$$a_i^{discrete} = \text{Clip}\left( \left\lfloor \frac{a_i - a_i^{min}}{a_i^{max} - a_i^{min}} \times 255 \right\rfloor, 0, 255 \right)$$

然后使用标准的语言建模损失：

$$\mathcal{L}_{action} = -\sum_{i=1}^{D} \log p_{\theta}(a_i^{discrete} | \mathbf{X}_{img}, \mathbf{X}_{text})$$

**方法2：连续动作回归（ACT/π0路线）**

使用回归损失直接预测连续动作：

$$\mathcal{L}_{action} = \sum_{i=1}^{D} | \hat{a}_i - a_i^{gt} |$$

或使用扩散模型建模动作分布。

**动作微调参数策略**：

| 策略 | 视觉编码器 | 语言模型 | 动作头 |
|------|-----------|----------|--------|
| 全冻结（Adapter） | 冻结 | 冻结 | 训练 |
| 视觉微调 | 低学习率 | 冻结 | 训练 |
| LLM LoRA | 冻结 | LoRA训练 | 训练 |
| 全量微调 | 微调 | 微调 | 训练 |

OpenVLA的策略是：**视觉编码器全量微调 + LLM LoRA + 动作头全量训练**，在效果和效率之间取得平衡。

### 6.3 扩散模型训练（π0路线）

**π0**（Physical Intelligence）采用扩散模型（Diffusion Model）来建模连续动作分布，这是VLA动作预测的另一条重要技术路线。

**扩散模型基本原理**：

扩散模型通过逐步去噪生成数据。设真实动作分布为 $q(\mathbf{a})$，扩散过程向数据中逐步添加噪声：

$$q(\mathbf{a}_t | \mathbf{a}_{t-1}) = \mathcal{N}(\mathbf{a}_t; \sqrt{1-\beta_t}\mathbf{a}_{t-1}, \beta_t \mathbf{I})$$

去噪过程学习反向条件分布：

$$p_{\theta}(\mathbf{a}_{t-1} | \mathbf{a}_t, \mathbf{X}_{img}, \mathbf{X}_{text}) = \mathcal{N}(\mathbf{a}_{t-1}; \mu_{\theta}(\mathbf{a}_t, t, \mathbf{X}_{img}, \mathbf{X}_{text}), \sigma_t^2 \mathbf{I})$$

**π0的训练目标**：

π0使用简化损失（DDPM简化版）：

$$\mathcal{L} = \mathbb{E}_{t, \mathbf{a}_0, \epsilon} \left[ \left\| \epsilon - \epsilon_{\theta}(\sqrt{\bar{\alpha}_t}\mathbf{a}_0 + \sqrt{1-\bar{\alpha}_t}\epsilon, t, \mathbf{X}_{img}, \mathbf{X}_{text}) \right\|^2 \right]$$

其中 $\epsilon_{\theta}$ 是去噪网络（由Transformer实现）。

**π0的独特设计**：

| 特点 | 说明 |
|------|------|
| 双系统架构 | π0-Base（100Hz，高频控制）+ π0-High（1-5Hz，任务规划） |
| 动作先验 | 使用互联网视频预训练学习人类动作先验 |
| 扩散调度 | 使用有余弦调度的扩散，适合连续动作 |
| 多任务统一 | 同一模型处理多种任务，通过文本指令指定 |

### 6.4 多任务联合训练

VLA通常需要在一个模型中同时处理多种任务，这带来独特的训练挑战。

**多任务训练的核心问题**：

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| 任务冲突 | 不同任务需要不同的动作模式 | 任务平衡采样 |
| 灾难遗忘 | 学习新任务遗忘旧任务 | 经验回放、LoRA |
| 动作空间异构 | 不同任务的动作维度不同 | 动作补齐/掩码 |
| 指令歧义 | 同一指令在不同任务中有不同含义 | 任务描述前缀 |
