## 1. 扩散模型基础

### 1.1 从生成模型到扩散模型

在深度生成模型领域，我们经历了 GAN（生成对抗网络）、VAE（变分自编码器）、Flow-based Model 等多个范式。**扩散概率模型（Diffusion Probabilistic Model）** 是近年来异军突起的一种生成模型，以其稳定的训练过程和高质量的生成效果，迅速成为图像、音频、代码生成的主流方法。

扩散模型的核心思想可以用一个物理过程来类比：**墨水在水中的扩散**。想象你把一滴墨水滴入清水中，墨水分子会逐渐扩散，最终均匀分布在整个水体中——这是**前向扩散过程（Forward Process）**。如果我们能记录下这个过程中每个时间步的分子分布，就有可能逆转这个过程，从完全均匀分布的状态"凝聚"回那滴墨水——这是**反向扩散过程（Reverse Process）**。

在数学上，扩散模型正是基于这个原理：
- **前向过程**：向真实数据中逐步添加高斯噪声，直到数据变成纯噪声
- **反向过程**：学习一个神经网络，从纯噪声中逐步去噪，最终恢复出真实数据

### 1.2 去噪扩散概率模型（DDPM）

**DDPM（Denoising Diffusion Probabilistic Models）** 是扩散模型的奠基性工作，由 Ho 等人在 2020 年提出。其核心是一个马尔可夫链，通过变分推断进行训练。

#### 1.2.1 前向扩散过程

前向过程定义为从真实数据 $\mathbf{x}_0$ 开始，逐步添加高斯噪声直到 $\mathbf{x}_T$：

$$
q(\mathbf{x}_t \mid \mathbf{x}_{t-1}) = \mathcal{N}\left(\mathbf{x}_t; \sqrt{1 - \beta_t} \mathbf{x}_{t-1}, \beta_t \mathbf{I}\right)
$$

其中 $\beta_t$ 是噪声调度（noise schedule）参数，随着 $t$ 从小到大递增（通常从 $10^{-4}$ 到 $0.02$）。这个公式的物理含义是：每一步都以一定比例保留前一状态的信息，同时添加新的噪声。

通过重参数化技巧，我们可以直接计算任意时间步 $t$ 的 $\mathbf{x}_t$：

$$
q(\mathbf{x}_t \mid \mathbf{x}_0) = \mathcal{N}\left(\mathbf{x}_t; \sqrt{\bar{\alpha}_t} \mathbf{x}_0, (1 - \bar{\alpha}_t) \mathbf{I}\right)
$$

其中 $\alpha_t = 1 - \beta_t$，$\bar{\alpha}_t = \prod_{i=1}^t \alpha_i$。这意味着我们不需要逐步迭代，可以直接求出任意时间步的噪声数据。

#### 1.2.2 反向扩散过程

反向过程是学习的核心。我们训练一个神经网络 $p_\theta(\mathbf{x}_{t-1} \mid \mathbf{x}_t)$ 来近似真实的反向过程 $q(\mathbf{x}_{t-1} \mid \mathbf{x}_t)$：

$$
p_\theta(\mathbf{x}_{t-1} \mid \mathbf{x}_t) = \mathcal{N}\left(\mathbf{x}_{t-1}; \boldsymbol{\mu}_\theta(\mathbf{x}_t, t), \boldsymbol{\Sigma}_\theta(\mathbf{x}_t, t)\right)
$$

DDPM 的一个关键洞察是：**我们不需要直接预测 $\mathbf{x}_{t-1}$，而是让它预测噪声 $\boldsymbol{\epsilon}$**。可以证明：

$$
\boldsymbol{\mu}_\theta(\mathbf{x}_t, t) = \frac{1}{\sqrt{\alpha_t}}\left(\mathbf{x}_t - \frac{\beta_t}{\sqrt{1 - \bar{\alpha}_t}} \boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)\right)
$$

其中 $\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)$ 是神经网络预测的噪声。因此，DDPM 的训练目标简化为：

$$
\mathcal{L} = \mathbb{E}_{t, \mathbf{x}_0, \boldsymbol{\epsilon}}\left[\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta(\sqrt{\bar{\alpha}_t} \mathbf{x}_0 + \sqrt{1 - \bar{\alpha}_t} \boldsymbol{\epsilon}, t)\|^2\right]
$$

这是一个简单的 MSE（均方误差）损失，训练过程非常稳定。

#### 1.2.3 DDPM 采样过程

给定训练好的噪声预测网络 $\boldsymbol{\epsilon}_\theta$，我们可以通过以下步骤从纯噪声生成样本：

$$
\mathbf{x}_{t-1} = \frac{1}{\sqrt{\alpha_t}}\left(\mathbf{x}_t - \frac{\beta_t}{\sqrt{1 - \bar{\alpha}_t}} \boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)\right) + \sigma_t \mathbf{z}
$$

其中 $\mathbf{z} \sim \mathcal{N}(0, \mathbf{I})$，$\sigma_t$ 是方差。这个过程需要迭代 $T$ 步（通常 $T = 1000$），是 DDPM 的主要计算瓶颈。

### 1.3 条件扩散模型

在实际应用中，我们通常需要**条件生成**：给定一个条件输入（如文本描述、类别标签、图像等），生成对应的输出。这在具身智能中尤为重要——我们希望根据当前观察（图像）和语言指令，生成对应的机器人动作序列。

#### 1.3.1 Classifier-Free Guidance（CFG）

**Classifier-Free Guidance** 是目前最流行的条件扩散训练策略，由 Ho 等人在 2022 年提出。其核心思想是：不再训练一个单独的分类器来引导生成，而是将条件信息直接注入到扩散模型的输入中。

具体做法是：
1. 训练一个同时支持条件输入和无条件输入的扩散模型
2. 在训练时，随机将条件信号替换为 null（空）输入（通常以 10-20% 的概率）
3. 在推理时，结合条件预测和无条件预测来增强生成质量

数学上，CFG 的输出为：

$$
\hat{\boldsymbol{\epsilon}} = (1 + w) \cdot \boldsymbol{\epsilon}_\theta(\mathbf{x}_t, c) - w \cdot \boldsymbol{\epsilon}_\theta(\mathbf{x}_t, \emptyset)
$$

其中 $c$ 是条件输入（如文本），$\emptyset$ 是 null 条件，$w$ 是**引导强度（guidance scale）**。当 $w > 0$ 时，生成结果更符合条件；当 $w = 0$ 时，退化为无条件生成。典型的 $w$ 值在 1.0 到 10.0 之间。

CFG 的优点是：
- 不需要额外的分类器网络
- 可以使用任意类型的条件信号（文本、图像、类别等）
- 引导强度可以动态调节

#### 1.3.2 交叉注意力机制

在 VLA 场景中，条件通常是多模态的：视觉观察 + 语言指令。为了将语言条件注入扩散模型，通常使用**交叉注意力（Cross-Attention）**机制。

扩散模型的 UNet 骨干网络中，每个注意力层除了自注意力外，还包含一个交叉注意力层，接收来自语言编码器的输出：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right) V
$$

其中 $Q$ 来自图像/动作特征，$K$ 和 $V$ 来自语言编码器。这使得模型能够在生成过程中动态关注与当前任务相关的语言信息。

### 1.4 扩散模型 vs 自回归模型

在 VLA 领域，扩散模型和自回归（Autoregressive, AR）模型是两种主要的动作生成范式：

| 特性 | 自回归模型（RT-2、OpenVLA） | 扩散模型（π0、Diffusion Policy） |
|------|---------------------------|--------------------------------|
| **生成方式** | 逐个 token 自回归生成 | 并行去噪，一次生成整个序列 |
| **动作序列** | 离散化后自回归输出 | 连续空间直接建模 |
| **推理速度** | 需要 $N$ 次前向传递（$N$=序列长度） | 需要 $T$ 次迭代去噪（$T$=扩散步数） |
| **多模态动作分布** | 近似单峰分布 | 天然支持多峰分布 |
| **训练稳定性** | 依赖教师-forcing | 训练稳定，MSE 损失 |
| **条件控制** | 通过语言/视觉 token 注入 | 通过 CFG 或交叉注意力注入 |

**扩散模型的核心优势**在于：
1. **多峰分布建模**：机器人任务中，给定同一观察可能有多种合理动作。扩散模型能够同时建模这种多模态性，而自回归模型倾向于坍缩到单峰分布。
2. **连续动作空间的自然建模**：不像自回归模型需要将动作离散化（精度损失），扩散模型直接在连续空间中操作。
3. **训练稳定性**：扩散模型的 MSE 损失不像 GAN 那样有训练不稳定问题，也不像自回归模型那样需要复杂的教师-forcing。

## 2. π0模型详解

### 2.1 Physical Intelligence π0 架构

π0 是 Physical Intelligence 公司开发的通用机器人模型，其核心创新在于将**扩散模型**引入 VLA 架构，实现了高效、多任务、灵巧的机器人控制。

π0 的架构设计遵循以下核心原则：
- **动作频率分离（Action Frequency Separation）**：不同任务的动作频率差异巨大（如抽屉开关需要慢速精确动作，而物体抓取需要快速反应）。π0 通过分离不同频率的动作头来解决这一问题。
- **双系统架构**：π0 包含一个"慢速"系统（处理需要深思熟虑的任务）和一个"快速"系统（处理反射性、快速的反应），两者协同工作。
- **动作扩散生成**：使用扩散模型生成连续动作序列，而非离散 token。

#### 2.1.1 模型结构

π0 的整体架构可以分为以下几个组件：

```
┌─────────────────────────────────────────────────────┐
│                    π0 整体架构                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────────┐   ┌─────────────┐                 │
│  │  视觉编码器  │   │  语言编码器  │                 │
│  │ (SigLIP So400m)│ │ (PaLM 2)   │                 │
│  └──────┬──────┘   └──────┬──────┘                 │
│         │                  │                        │
│         └────────┬─────────┘                        │
│                  ▼                                  │
│         ┌───────────────┐                          │
│         │   特征融合层    │                          │
│         │  (MLP / FiLM)  │                          │
│         └───────┬───────┘                          │
│                 │                                   │
│    ┌────────────┴────────────┐                    │
│    ▼                         ▼                      │
│ ┌──────────┐           ┌──────────┐               │
│ │ 慢速动作头 │           │ 快速动作头 │               │
│ │ (扩散模型) │           │ (MLP)    │               │
│ │ 手臂/移动  │           │ 末端执行器  │               │
│ └──────────┘           └──────────┘               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**视觉编码器**：π0 使用 SigLIP So400m 作为视觉编码器，这是一个大规模预训练的视觉-语言模型，具有强大的图像理解能力。视觉编码器将 RGB 图像转换为高维特征向量。

**语言编码器**：π0 使用 PaLM 2 作为语言编码器，处理自然语言指令。语言编码器将文本指令转换为语义特征向量。

**特征融合**：视觉和语言特征通过 MLP（多层感知机）或 FiLM（Feature-wise Linear Modulation）层进行融合。FiLM 层根据语言特征动态调制视觉特征，实现语言对视觉的理解引导。

**动作头**：这是 π0 的核心创新。π0 包含多个动作头，每个动作头负责不同频率、不同部件的动作：
- **慢速动作头（Slow Policy）**：基于扩散模型，负责手臂关节角度、末端执行器姿态等需要精确规划的动作
- **快速动作头（Fast Policy）**：基于简单 MLP，负责末端执行器位置、夹爪开合等需要快速响应的反射性动作

#### 2.1.2 动作频率分离机制

传统的 VLA 模型（如 RT-2）使用统一的 action chunk（动作块），以固定频率（如 1Hz 或 5Hz）输出所有动作。这种设计在处理需要不同动作频率的任务时存在矛盾：

- **精细操作任务**（如折叠毛巾、系鞋带）：需要高频（10-50Hz）的末端执行器控制
- **移动操作任务**（如导航到桌子、抓取杯子）：需要低频（1-5Hz）的臂体运动

π0 的动作频率分离解决了这一问题：

| 动作类型 | 频率 | 生成方式 | 例子 |
|---------|------|---------|------|
| **慢速动作** | 1-5 Hz | 扩散模型 | 臂体运动、末端姿态 |
| **快速动作** | 10-50 Hz | MLP | 夹爪开合、位置调整 |
| **低频控制** | 0.1-1 Hz | 扩散模型 | 移动平台控制 |

慢速动作头使用**扩散模型**生成，因为它需要精确的关节配置和多模态规划；快速动作头使用简单的**MLP**，因为它只需要快速映射观察到的状态到预定义的动作模式。

### 2.2 π0 的动作扩散生成

π0 的慢速动作头采用**条件扩散模型**生成动作序列。具体而言，给定当前观察 $\mathbf{o}_t$（图像 + 状态向量）和语言指令 $\mathcal{L}$，π0 生成未来 $H$ 步的动作序列 $\mathbf{a}_{t:t+H}$。

#### 2.2.1 条件扩散过程

π0 的动作扩散过程与标准 DDPM 类似，但条件信息（观察 $\mathbf{o}$ 和语言 $\mathcal{L}$）通过**交叉注意力**注入：

$$
\boldsymbol{\epsilon}_\theta(\mathbf{a}_t, t, \mathbf{o}, \mathcal{L}) = \text{UNet}_\theta(\mathbf{a}_t, t) + \text{CrossAttention}(\text{FLAN-UL2}(\mathbf{o}, \mathcal{L}))
$$

训练时，以 MSE 损失进行去噪：

$$
\mathcal{L}_{\pi_0} = \mathbb{E}_{t, \mathbf{a}_0, \mathbf{o}, \mathcal{L}, \boldsymbol{\epsilon}}\left[\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta(\sqrt{\bar{\alpha}_t} \mathbf{a}_0 + \sqrt{1 - \bar{\alpha}_t} \boldsymbol{\epsilon}, t, \mathbf{o}, \mathcal{L})\|^2\right]
$$

#### 2.2.2 动作序列长度与chunk设计

π0 生成的不是单个动作，而是一个**动作序列（Action Chunk）**。典型配置下，每个 chunk 包含 16-32 个时间步的动作，对应 1-2 秒的机器人行为。

chunk 设计的好处是：
1. **时序一致性**：整个 chunk 内的动作来自同一次扩散采样，保证动作的连贯性
2. **计算效率**：一次扩散生成多个动作，减少推理次数
3. **多模态建模**：扩散模型天然建模动作序列上的多模态分布

### 2.3 多任务学习策略

π0 的核心能力之一是**真正的多任务通用性**。它在数千种不同的任务上进行训练，涵盖了桌面操作、衣物折叠、物体抓取、环境导航等多种场景。

#### 2.3.1 数据混合策略

π0 的训练数据来自多种来源：

| 数据来源 | 描述 | 规模 |
|---------|------|------|
| **BridgeData** | 遥操作采集的厨房任务数据 | ~50K 条轨迹 |
| **RT-X** | 多个实验室机器人数据的聚合 | ~1M 条轨迹 |
| **Open X-Embodiment** | 具身智能数据联盟数据 | ~2M 条轨迹 |
| **PI 内部采集数据** | 灵巧手、双臂任务数据 | ~100K 条轨迹 |

在训练时，π0 使用**加权采样**策略，平衡不同任务类型的数据比例，避免某些任务被忽视。

#### 2.3.2 预训练-微调范式

π0 遵循大规模预训练 + 任务特定微调的范式：

1. **预训练阶段**：使用大规模互联网数据和机器人数据联合预训练，学习通用的视觉-语言理解和动作规划能力
2. **动作微调阶段**：在机器人数据上微调扩散动作头，学习具体的动作模式
3. **任务微调阶段**：在特定任务（如折叠特定衣物）上进行少量微调，进一步优化

这种范式使得 π0 能够"站在巨人的肩膀上"——预训练阶段学习的通用能力 + 动作微调获得的具体操作能力。

### 2.4 π0 Variants

Physical Intelligence 发布了多个 π0 的变体：

| 模型 | 参数量 | 描述 | 适用场景 |
|------|--------|------|----------|
| **π0** | 7B | 基础版本，双臂操作 | 通用双臂机器人 |
| **π0.5** | 3B | 轻量版本 | 边缘部署、实时控制 |
| **π0-Humanoid** | 7B | 人形机器人专用 | 双臂+双腿人形机器人 |
| **π0-Hand** | 7B | 灵巧手版本 | 精细手部操作任务 |

### 2.5 π0 与其他VLA范式的对比

| 特性 | RT-2 | OpenVLA | π0 |
|------|------|---------|-----|
| **动作生成** | 离散token自回归 | 离散token自回归 | 连续扩散 |
| **动作空间** | 末端执行器 | 末端执行器 | 全身关节+末端 |

## 3. 其他扩散VLA模型

### 3.1 Diffusion Policy（哥伦比亚大学）

**Diffusion Policy** 是由哥伦比亚大学 CHAI Lab 于 2023 年提出的开创性工作，首次将扩散模型应用于机器人动作生成。

Diffusion Policy 的核心贡献：
- 将机器人动作建模为**条件扩散过程**，给定视觉观察生成动作序列
- 在多个模拟和真实机器人任务上验证了扩散方法的有效性
- 提出了 **Diffusion KV Player**，用于分析扩散策略学到的多模态行为

#### 3.1.1 架构设计

Diffusion Policy 使用标准的 **Conditional DDPM** 架构。关键组件包括：
- **视觉观察编码器**：CNN（ResNet18/50）或 ViT，将图像序列编码为观察嵌入
- **MLP 条件注入层**：将观察嵌入映射为噪声预测网络的调制信号
- **UNet 噪声预测网络**：核心扩散模型，输入带噪动作，输出预测噪声
- **扩散调度器**：管理噪声添加和去除的调度

### 3.2 ACT（斯坦福）

**ACT（Action Chunking with Transformers）** 是斯坦福大学提出的基于 Transformer 的动作分块模型，虽然名字中没有"扩散"，但其思想与扩散模型有密切联系——两者都强调生成**动作序列**而非单个动作。

ACT 的核心设计：
- 使用 **Transformer 编码器**处理观察序列
- 使用 **Transformer 解码器**自回归生成动作序列
- 引入 **EKF（扩展卡尔曼滤波）** 进行时序过滤，减少观测噪声

ACT 的动作分块机制允许一次生成多个时间步的动作，提高了时序一致性，与 Diffusion Policy 的 chunk 思想异曲同工。

### 3.3 UniPi / Composer

**UniPi**（Physical Intelligence 等）提出了**统一生成模型**的思想，使用扩散模型同时生成视频计划、机器人动作和文本描述。UniPi 的核心创新是：

## 4. 扩散VLA训练与推理

### 4.1 动作序列扩散训练

扩散VLA的训练目标是学习一个条件扩散模型 $p_\theta(\mathbf{a} \mid \mathbf{o}, \mathcal{L})$，其中 $\mathbf{a}$ 是动作序列，$\mathbf{o}$ 是视觉观察，$\mathcal{L}$ 是语言指令。

#### 4.1.1 训练数据格式

一条典型的训练样本包含：

```python
# 训练样本结构
sample = {
    'image': np.random.randn(3, 224, 224),           # 当前RGB图像
    'state': np.random.randn(14),                   # 机器人关节状态 (7 DOF arm × 2)
    'language': "pick up the red cup on the table",  # 自然语言指令
    'action': np.random.randn(16, 14),              # 未来16步动作序列
    # action: [arm_joint(7) × 2 arms + gripper(1) × 2] = 14维
}
```

#### 4.1.2 训练流程

扩散VLA的标准训练流程：

1. **观察编码**：将图像通过视觉编码器得到观察嵌入 $\mathbf{z}_o = \text{Enc}_o(\mathbf{o})$
2. **语言编码**：将语言指令通过语言编码器得到语言嵌入 $\mathbf{z}_l = \text{Enc}_l(\mathcal{L})$
3. **特征融合**：通过 FiLM 或 MLP 融合 $\mathbf{z}_o$ 和 $\mathbf{z}_l$ 得到条件嵌入 $\mathbf{c}$
4. **噪声添加**：从数据分布中采样动作 $\mathbf{a}_0$，随机采样 timestep $t$，添加噪声 $\boldsymbol{\epsilon}$
5. **噪声预测**：通过扩散UNet预测噪声 $\hat{\boldsymbol{\epsilon}} = \boldsymbol{\epsilon}_\theta(\mathbf{a}_t, t, \mathbf{c})$
6. **损失计算**：$\mathcal{L} = \|\boldsymbol{\epsilon} - \hat{\boldsymbol{\epsilon}}\|^2$

典型训练配置：

| 参数 | 典型值 |
|------|--------|
| 批量大小 | 64-256 |
| 学习率 | 1e-4（AdamW） |
| 扩散步数 $T$ | 100-1000 |
| 动作序列长度 $H$ | 8-32 |
| 训练轮数 | 10-100 epochs |
| GPU | 8× A100 80GB |

### 4.2 条件引导生成

#### 4.2.1 视觉条件引导

在推理时，给定当前视觉观察，生成对应的动作。核心流程是：
1. 编码视觉观察得到条件嵌入
2. 从纯噪声初始化动作序列
3. 通过 DDPM/DDIM 迭代去噪
4. 提取第一步动作执行

#### 4.2.2 CFG 引导强度调节

在推理时，可以通过调节 CFG 引导强度 $w$ 来控制生成动作的"听话程度"：

```python
# CFG 引导推理核心公式
noise_pred = (1 + guidance_scale) * noise_cond - guidance_scale * noise_uncond
```

- $w = 0$：无条件生成，动作随机
- $w = 1$：标准条件生成
- $w = 3\sim7$：强引导，动作更符合指令但可能缺乏多样性
- $w > 10$：过强引导，可能导致生成质量下降

### 4.3 实时推理优化

扩散模型的一个主要挑战是**推理速度**——标准 DDPM 需要 1000 步迭代去噪，这在与物理世界交互的实时控制场景中是不可接受的。

#### 4.3.1 加速采样策略

| 方法 | 迭代步数 | 质量损失 | 说明 |
|------|---------|---------|------|
| **DDIM** | 20-50 步 | 极小 | 确定性轨迹，加速效果明显 |
| **DPM-Solver** | 10-20 步 | 极小 | 基于ODE的求解器 |
| **原始 DDPM** | 1000 步 | 无 | 基准方法 |

**DDIM（Denoising Diffusion Implicit Models）** 是目前最流行的加速方法。DDIM 重新参数化了去噪过程，使得可以在远少于 $T$ 的步数内完成高质量采样。通过选择性地"跳步"（每隔 $S$ 步采样一次），DDIM 可以将推理步数从 1000 降至 20-50 步，而视觉质量几乎不变。

#### 4.3.2 动作延迟补偿

在实时控制中，从观察到动作执行存在延迟（传感器处理 + 模型推理 + 控制信号传输）。典型的延迟约为 **50-200ms**，需要通过以下方式补偿：

1. **动作递推（Action Extrapolation）**：基于当前速度估计，递推动作到实际执行时刻的位置
2. **观察缓冲（Observation Buffer）**：维护一个短窗口（如最近3帧）的观察序列，平滑动作输出
3. **异步控制**：使用独立线程进行推理，PID 控制器在主线程中以高频执行

#### 4.3.3 ONNX / TensorRT 部署

为实现实时推理，通常需要将模型导出为 ONNX 格式或使用 TensorRT 优化：

```bash
# ONNX 导出示例
python -m torch.onnx.export \
    policy \
    (obs_images,) \
    policy.onnx \
    input_names=['obs'] \
    output_names=['action'] \
    dynamic_axes={'obs': {0: 'batch'}, 'action': {0: 'batch'}}

# TensorRT 半精度优化（典型加速 2-4x）
trtexec --onnx=policy.onnx \
        --saveEngine=policy.trt \
        --fp16
```

## 5. 代码实战

### 5.1 简化扩散策略实现

以下是一个简化版的 Diffusion Policy 实现，帮助理解扩散VLA的核心机制：

```python
"""
简化版 Diffusion Policy 实现
基于 DDPM 的机器人动作序列生成
"""

import torch
import torch.nn as nn
import torch.nn.functional as F
import math
import numpy as np


# ============ 1. 扩散调度器 ============

class DiffusionSchedule:
    """
    扩散调度器：管理噪声调度（beta schedule）
    支持 linear 和 cosine 两种调度方式
    """

    def __init__(self, num_steps=1000, schedule_type='linear',
                 beta_start=1e-4, beta_end=0.02):
        self.num_steps = num_steps
        self.schedule_type = schedule_type

        if schedule_type == 'linear':
            # 线性调度：beta 从 beta_start 线性增加到 beta_end
            self.betas = torch.linspace(beta_start, beta_end, num_steps)
        elif schedule_type == 'cosine':
            # Cosine 调度：更平滑的噪声添加
            self.betas = self._cosine_beta_schedule(num_steps)
        else:
            raise ValueError(f"Unknown schedule type: {schedule_type}")

        # 预计算 alpha = 1 - beta 和累积 alpha
        self.alphas = 1.0 - self.betas
        self.alphas_cumprod = torch.cumprod(self.alphas, dim=0)  # \bar{\alpha}_t
        self.alphas_cumprod_prev = torch.cat([torch.ones(1), self.alphas_cumprod[:-1]])

        # 缓存计算所需的 sqrt 项
        self.sqrt_alphas_cumprod = torch.sqrt(self.alphas_cumprod)
        self.sqrt_one_minus_alphas_cumprod = torch.sqrt(1.0 - self.alphas_cumprod)

        # 后验方差（用于 DDPM 采样）
        self.posterior_variance = (
            self.betas * (1.0 - self.alphas_cumprod_prev) / (1.0 - self.alphas_cumprod)
        )

    def _cosine_beta_schedule(self, num_steps, s=0.008):
        """
        Cosine 调度：按照 cosine 函数的形式安排 beta
        论文: "Improved Techniques for Training Consistency Models"
        """
        steps = torch.arange(num_steps + 1, dtype=torch.float32)
        alphas_cumprod = torch.cos(((steps / num_steps) + s) / (1 + s) * math.pi * 0.5) ** 2
        alphas_cumprod = alphas_cumprod / alphas_cumprod[0]  # 归一化
        betas = 1 - (alphas_cumprod[1:] / alphas_cumprod[:-1])
        return torch.clip(betas, 0.0001, 0.9999)

    def add_noise(self, x0, t, noise=None):
        """
        前向扩散：向数据添加噪声
        参数:
            x0: 原始数据 (batch_size, action_dim)
            t: 时间步 (batch_size,) 整数张量
            noise: 可选的预定义噪声
        返回:
            xt: 加噪后的数据
        """
        batch_size = x0.shape[0]
        if noise is None:
            noise = torch.randn_like(x0)  # 生成标准高斯噪声

        # 获取对应时间步的 sqrt_alpha 和 sqrt_one_minus_alpha
        sqrt_alphas_cumprod_t = self._get_index_from_list(
            self.sqrt_alphas_cumprod, t, x0.shape
        )
        sqrt_one_minus_alphas_cumprod_t = self._get_index_from_list(
            self.sqrt_one_minus_alphas_cumprod, t, x0.shape
        )

        # 重参数化：x_t = sqrt(\bar{a}_t) * x_0 + sqrt(1 - \bar{a}_t) * epsilon
        xt = sqrt_alphas_cumprod_t * x0 + sqrt_one_minus_alphas_cumprod_t * noise
        return xt

    def _get_index_from_list(self, vals, t, x_shape):
        """根据时间步 t 索引查找对应的值，处理成正确的形状"""
        batch_size = t.shape[0]
        # gather: 根据 t 索引从 vals 中取值
        out = vals.gather(-1, t.cpu())
        # reshape 为 (batch_size, 1, 1, ...) 以便广播
        return out.reshape(batch_size, *((1,) * (len(x_shape) - 1))).to(t.device)


# ============ 2. 噪声预测网络（UNet）============

class SinusoidalPositionEmbedding(nn.Module):
    """
    位置编码：将 timestep t 映射为高维向量
    用于让网络知道当前处于去噪的哪个阶段
    """

    def __init__(self, dim):
        super().__init__()
        self.dim = dim

    def forward(self, t):
        """
        参数:
            t: (batch_size,) 时间步整数
        返回:
            embedding: (batch_size, dim) 位置编码
        """
        device = t.device
        half_dim = self.dim // 2
        # 计算频率衰减因子
        embeddings = math.log(10000) / (half_dim - 1)
        embeddings = torch.exp(torch.arange(half_dim, device=device) * -embeddings)
        embeddings = t[:, None] * embeddings[None, :]  # (batch_size, half_dim)
        embeddings = torch.cat([torch.sin(embeddings), torch.cos(embeddings)], dim=-1)
        return embeddings


class ResidualBlock(nn.Module):
    """
    残差块：包含两个线性层 + LayerNorm + 残差连接
    """

    def __init__(self, dim, time_dim):
        super().__init__()
        self.fc1 = nn.Linear(dim, dim * 4)
        self.fc2 = nn.Linear(dim * 4, dim)
        self.time_mlp = nn.Linear(time_dim, dim * 4)  # 时间步嵌入映射
        self.norm = nn.LayerNorm(dim)
        self.activation = nn.SiLU()  # SiLU/Swish 激活函数

    def forward(self, x, t_emb):
        """
        参数:
            x: (batch, dim) 隐藏特征
            t_emb: (batch, time_dim) 时间步嵌入
        返回:
            out: (batch, dim)
        """
        h = self.norm(x)
        h = self.fc1(h)
        h = self.activation(h)
        # 注入时间步信息（FiLM 风格调制）
        h = h + self.time_mlp(self.activation(t_emb))
        h = self.fc2(h)
        h = self.activation(h)
        # 残差连接
        return x + h


class ConditionalUNet(nn.Module):
    """
    条件 UNet：用于噪声预测
    输入: 带噪动作序列 + timestep + 条件嵌入
    输出: 预测的噪声
    """

    def __init__(self, action_dim, cond_dim=512, hidden_dim=256, time_dim=128):
        super().__init__()
        self.action_dim = action_dim

        # 时间步嵌入
        self.time_embed = SinusoidalPositionEmbedding(time_dim)
        self.time_fc = nn.Linear(time_dim, hidden_dim)

        # 动作序列的输入投影：将动作维度映射到隐藏维度
        self.input_proj = nn.Linear(action_dim, hidden_dim)

        # 条件注入层：将条件嵌入投影到与时间步嵌入相同的空间
        self.cond_proj = nn.Linear(cond_dim, hidden_dim)

        # 主干网络：多层残差块
        self.res_blocks = nn.ModuleList([
            ResidualBlock(hidden_dim, time_dim) for _ in range(4)
        ])

        # 输出层：隐藏维度 -> 动作维度（预测噪声）
        self.output_proj = nn.Linear(hidden_dim, action_dim)

        # 初始化输出层为零，以加速训练收敛
        self._init_weights()

    def _init_weights(self):
        nn.init.zeros_(self.output_proj.weight)
        nn.init.zeros_(self.output_proj.bias)

    def forward(self, noise_action, t, cond):
        """
        前向传播：预测噪声
        参数:
            noise_action: (batch, action_dim) 带噪的动作
            t: (batch,) 时间步
            cond: (batch, cond_dim) 条件嵌入（观察/语言）
        返回:
            noise_pred: (batch, action_dim) 预测的噪声
        """
        # 时间步嵌入
        t_emb = self.time_embed(t)
        t_emb = self.time_fc(t_emb)  # (batch, hidden_dim)

        # 条件嵌入
        cond_emb = self.cond_proj(cond)  # (batch, hidden_dim)

        # 融合时间步和条件（相加）
        h = t_emb + cond_emb

        # 输入投影
        x = self.input_proj(noise_action)

        # 通过残差块
        for block in self.res_blocks:
            x = block(x, h)


        # 输出预测的噪声
        noise_pred = self.output_proj(x)
        return noise_pred


# ============ 3. 简化版 Diffusion Policy ============

class SimpleDiffusionPolicy(nn.Module):
    """
    简化版 Diffusion Policy
    用于机器人动作生成的扩散策略模型
    """

    def __init__(self, action_dim=14, cond_dim=512,
                 num_diffusion_steps=100, hidden_dim=256):
        super().__init__()
        self.action_dim = action_dim
        self.num_diffusion_steps = num_diffusion_steps

        # 扩散调度器
        self.diffusion = DiffusionSchedule(
            num_steps=num_diffusion_steps,
            schedule_type='cosine'
        )

        # 噪声预测网络
        self.noise_pred_net = ConditionalUNet(
            action_dim=action_dim,
            cond_dim=cond_dim,
            hidden_dim=hidden_dim
        )

    def forward(self, actions, cond, t=None):
        """
        训练时的前向传播：计算扩散损失
        参数:
            actions: (batch_size, action_dim) 原始动作
            cond: (batch_size, cond_dim) 条件嵌入
            t: 可选，指定的时间步
        返回:
            loss: MSE 损失
        """
        batch_size = actions.shape[0]

        # 采样时间步
        if t is None:
            t = torch.randint(
                0, self.num_diffusion_steps,
                (batch_size,), device=actions.device
            )

        # 采样噪声
        noise = torch.randn_like(actions)

        # 添加噪声
        noisy_actions = self.diffusion.add_noise(actions, t, noise)

        # 预测噪声
        noise_pred = self.noise_pred_net(noisy_actions, t, cond)

        # MSE 损失
        loss = F.mse_loss(noise_pred, noise)
        return loss

    @torch.no_grad()
    def sample(self, cond, num_steps=20, guidance_scale=3.0):
        """
        推理：从噪声生成动作（使用 DDIM 加速采样）
        参数:
            cond: (batch_size, cond_dim) 条件嵌入
            num_steps: DDIM 采样步数
            guidance_scale: CFG 引导强度
        返回:
            action: (batch_size, action_dim) 生成的动作
        """
        batch_size = cond.shape[0]
        device = cond.device

        # 初始化为纯噪声
        action = torch.randn(batch_size, self.action_dim, device=device)

        # DDIM 采样：使用扩展时间步序列（稀疏采样）
        # 从 num_steps 到 0，每一步跳过他 num_steps/步数 个时间步
        step_size = self.num_diffusion_steps // num_steps
        timesteps = list(range(self.num_diffusion_steps, 0, -step_size))

        for i, t in enumerate(timesteps):
            t_tensor = torch.full((batch_size,), t, device=device, dtype=torch.long)

            # 预测噪声（带 CFG）
            if guidance_scale > 0:
                # 条件预测
                noise_pred_cond = self.noise_pred_net(action, t_tensor, cond)
                # 无条件预测（用零条件）
                zero_cond = torch.zeros_like(cond)
                noise_pred_uncond = self.noise_pred_net(action, t_tensor, zero_cond)
                # CFG 组合
                noise_pred = (1 + guidance_scale) * noise_pred_cond - guidance_scale * noise_pred_uncond
            else:
                noise_pred = self.noise_pred_net(action, t_tensor, cond)

            # DDIM 去噪一步
            alpha_t = self.diffusion.alphas_cumprod[t - 1]
            alpha_t_prev = self.diffusion.alphas_cumprod[t - step_size - 1] if t - step_size > 0 else torch.ones_like(alpha_t)

            # 重参数化去噪
            pred_x0 = (action - torch.sqrt(1 - alpha_t) * noise_pred) / torch.sqrt(alpha_t)
            direction_pointing_to_xt = torch.sqrt(1 - alpha_t_prev) * noise_pred
            action = torch.sqrt(alpha_t_prev) * pred_x0 + direction_pointing_to_xt

        return action

# ============ 5. DDPM 动作生成代码 ============

class DDPMActionGenerator:
    """
    纯 DDPM 动作生成器（不依赖 DDIM 加速）
    用于理解底层扩散机制
    """

    def __init__(self, noise_pred_net, diffusion_schedule, device='cuda'):
        self.net = noise_pred_net
        self.diffusion = diffusion_schedule
        self.device = device

    @torch.no_grad()
    def generate(self, cond, return_seq=False):
        """
        DDPM 纯采样生成动作
        参数:
            cond: 条件嵌入
            return_seq: 是否返回完整去噪序列
        返回:
            生成的干净动作
        """
        self.net.eval()
        batch_size = cond.shape[0]
        action_dim = self.net.action_dim

        # 从纯噪声开始
        action = torch.randn(batch_size, action_dim, device=self.device)

        # 逐步去噪（T 步）
        seq = [action.cpu()] if return_seq else None

        for t in reversed(range(self.diffusion.num_steps)):
            t_batch = torch.full((batch_size,), t, device=self.device, dtype=torch.long)

            # 预测噪声
            noise_pred = self.net(action, t_batch, cond)

            # DDPM 采样公式
            alpha_t = self.diffusion.alphas[t]
            alpha_cumprod_t = self.diffusion.alphas_cumprod[t]
            beta_t = self.diffusion.betas[t]

            # 预测 x0
            pred_x0 = (action - torch.sqrt(1 - alpha_cumprod_t) * noise_pred) / torch.sqrt(alpha_cumprod_t)

            # 计算 x_{t-1}
            if t > 0:
                noise = torch.randn_like(action)
                variance = self.diffusion.posterior_variance[t]
                action = torch.sqrt(alpha_t) * pred_x0 + torch.sqrt(1 - alpha_t - variance) * noise_pred + torch.sqrt(variance) * noise
            else:
                action = pred_x0

            if return_seq:
                seq.append(action.cpu())

        return action if not return_seq else torch.stack(seq, dim=1)
```

### 7.2 简答题答案

**6. DDPM 训练目标解析**

DDPM 的原始变分下界（ELBO）目标涉及最大化似然，但直接优化是困难的。关键洞察来自对 $q(\mathbf{x}_{t-1} \mid \mathbf{x}_t, \mathbf{x}_0)$ 的条件高斯分析。

真实反向过程的后验分布为：

$$
q(\mathbf{x}_{t-1} \mid \mathbf{x}_t, \mathbf{x}_0) = \mathcal{N}\left(\mathbf{x}_{t-1}; \tilde{\boldsymbol{\mu}}_t(\mathbf{x}_t, \mathbf{x}_0), \tilde{\beta}_t \mathbf{I}\right)
$$

可以推导（Ho et al., 2020）：

$$
\tilde{\boldsymbol{\mu}}_t(\mathbf{x}_t, \mathbf{x}_0) = \frac{1}{\sqrt{\alpha_t}}\left(\mathbf{x}_t - \frac{\beta_t}{\sqrt{1 - \bar{\alpha}_t}} \boldsymbol{\epsilon}_t\right)
$$

其中 $\boldsymbol{\epsilon}_t$ 是添加到 $\mathbf{x}_0$ 得到 $\mathbf{x}_t$ 的噪声。因此，如果我们让网络预测这个噪声 $\boldsymbol{\epsilon}_t$，就可以计算均值，从而训练目标简化为：

$$
\mathcal{L} = \mathbb{E}_{t, \mathbf{x}_0, \boldsymbol{\epsilon}}\left[\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)\|^2\right]
$$

这是一个简单的 MSE 损失，与 GAN 相比训练极为稳定，不需要对抗性优化。

**7. FiLM 机制原理与优势**

**FiLM（Feature-wise Linear Modulation）** 的全称是"特征级线性调制"。给定一个特征图 $\mathbf{f}$ 和一个条件向量 $\mathbf{c}$，FiLM 操作定义为：

$$
\text{FiLM}(\mathbf{f}, \gamma, \beta) = \gamma \odot \mathbf{f} + \beta
$$

其中：
- $\gamma, \beta$ 是由条件 $\mathbf{c}$ 通过线性变换得到的缩放和偏置参数
- $\odot$ 表示逐元素乘法（Hadamard 积）

在 π0 中，语言特征 $\mathbf{z}_l$ 通过一个小型 MLP 生成 $\gamma$ 和 $\beta$，用于调制视觉特征 $\mathbf{z}_v$：

$$
\tilde{\mathbf{z}}_v = \gamma(\mathbf{z}_l) \odot \mathbf{z}_v + \beta(\mathbf{z}_l)
$$

**FiLM 相比简单拼接的优势**：
1. **参数效率**：拼接需要增加输入维度导致参数爆炸；FiLM 只增加 2 倍的线性变换参数
2. **条件调制更直接**：FiLM 允许条件信号在通道维度上"选择性地强调或抑制"特定特征，而不是简单叠加
3. **条件与特征的交互更丰富**：FiLM 实现了乘法交互（$\gamma \odot \mathbf{f}$），而拼接是加性交互（$\mathbf{f} + \mathbf{c}$），乘法交互天然适合建模特征间的依赖关系

**8. 推理延迟优化策略**

将扩散 VLA 推理延迟从 500ms 降至 50ms，需要约 **10 倍加速**，以下是三种可行策略：

| 策略 | 原理 | 优点 | 潜在代价 |

| **DDIM 加速采样** | 将 1000 步 DDPM 降至 20-50 步 DDIM 采样 | 加速 20-50 倍，质量损失极小 | 可能轻微影响动作平滑性；需要调优跳步策略 |

**9. 多峰分布建模的重要性**

在机器人操作中，给定相同的视觉观察和语言指令，往往存在**多种合理且正确的动作执行方式**。

**具体例子**：假设任务指令是"把杯子放到托盘上"。

在某个时间步 $t$，机器人视觉观察到：
- 杯子当前在桌面左侧
- 托盘在桌面右侧

**可能的多种合理动作序列**：
1. **路线A（顺时针）**：先向前伸手 → 抓住杯子 → 向右移动 → 放到托盘左侧
2. **路线B（逆时针）**：先向右移动到托盘附近 → 伸手向左抓杯 → 带着杯子移回托盘
3. **路线C（直接）**：伸直手臂直接向前 → 抓住杯子 → 直线路径移到托盘上方 → 放下

这三种策略都是合理的（都能完成任务），但动作序列完全不同。**扩散模型**能够在动作空间上建模这种多模态分布——训练数据中同时包含这三种演示，扩散模型在生成时会根据噪声初始化随机"选择"其中一种路线。

相比之下，**自回归模型**（如 RT-2）倾向于学习所有演示的"平均"，输出一个在各种路线之间折中的动作——这可能导致机器人做出"四不像"的动作，既不像路线A也不像路线B，在实际执行中可能失败。

这就是为什么扩散模型在需要"灵巧操作"、"多样策略"的机器人任务中表现优于自回归模型。
