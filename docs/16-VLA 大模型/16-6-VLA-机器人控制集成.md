## 1. VLA机器人集成概述

### 1.1 VLA与ROS/ROS2的关系

ROS（Robot Operating System）和ROS2是机器人软件开发的实际标准框架，为机器人提供了硬件抽象、消息传递、工具链等基础能力。VLA模型本身是一个推理引擎，负责根据视觉输入和语言指令输出动作决策。将VLA集成到ROS/ROS2系统中，意味着让VLA成为机器人控制pipeline中的一个计算节点。

**ROS与ROS2的关键区别**对VLA集成有重要影响：

| 特性 | ROS | ROS2 |
|------|-----|------|
| 通信机制 | TCPROS/UDPROS | DDS（数据分发服务） |
| 实时性 | 非实时，需要补丁 | 原生实时支持 |
| 生命周期管理 | 节点独立 | 节点有生命周期状态机 |
| QoS质量服务 | 不支持 | 支持，可配置可靠性、持久性 |
| 构建系统 | catkin | ament/colcon |

对于VLA机器人控制，**ROS2是更推荐的选择**，原因如下：
- DDS的实时性更适合控制循环的确定性要求
- QoS机制可以保证视觉数据传递的可靠性
- 生命周期管理便于优雅地启动/停止VLA推理节点
- ROS2的Action机制天然适合长时间的运动任务

### 1.2 控制循环架构

VLA机器人控制系统的核心是一个多层嵌套的控制循环架构：

```
┌─────────────────────────────────────────────────┐
│           任务规划层（Task Planning）              │
│  接收高层指令 → 分解为可执行的动作序列              │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│          VLA 推理层（VLA Inference）              │
│  视觉+语言 → 动作预测（action prediction）         │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│         运动控制层（Motion Control）              │
│  动作命令 → 关节力矩/末端速度 → 电机驱动           │
└─────────────────────┬───────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────┐
│         传感反馈层（Sensor Feedback）             │
│  视觉/力觉/关节状态 → 更新环境模型                 │
└─────────────────────────────────────────────────┘
```

**控制频率的层级差异**是架构设计的关键考量：

| 层级 | 频率 | 示例 | 典型延迟容忍 |
|------|------|------|-------------|
| 任务规划层 | 0.1-1 Hz | "去厨房拿杯子" | 允许秒级延迟 |
| VLA推理层 | 1-10 Hz | 每帧图像推理 | 10-100ms可接受 |
| 运动控制层 | 100-1000 Hz | PID控制 | 必须<10ms |
| 传感反馈层 | 30-120 Hz | 相机图像采集 | 实时要求 |

VLA推理层处于中间位置，承上启下：它需要足够快地响应环境变化（至少1-2Hz），又不能像底层控制那样要求极高的实时性。

### 1.3 实时性要求分析

虽然VLA推理本身不需要毫秒级实时性，但整个系统的响应延迟对机器人性能至关重要。延迟来源分析：

$$L_{total} = L_{capture} + L_{preprocess} + L_{inference} + L_{postprocess} + L_{control}$$

其中：
- $L_{capture}$：图像采集延迟（相机驱动、传输）
- $L_{preprocess}$：图像预处理延迟（resize、normalize）
- $L_{inference}$：VLA模型推理延迟（核心计算）
- $L_{postprocess}$：动作后处理延迟（动作映射、限幅）
- $L_{control}$：控制命令发送延迟（通信传输）

典型的各环节延迟占比如下：

| 环节 | 平均延迟 | P99延迟 | 优化手段 |
|------|---------|---------|---------|
| 图像采集 | 5-15ms | 30ms | USB3.0/GigE相机、硬件触发 |
| 图像预处理 | 2-5ms | 10ms | GPU加速、批量处理 |
| VLA推理 | 50-200ms | 500ms | TensorRT/ONNX加速、模型量化 |
| 动作后处理 | 1-3ms | 5ms | 查表、并行计算 |
| 控制命令发送 | 1-5ms | 10ms | ROS2 QoS、本地通信 |

**总延迟目标**：对于桌面级机械臂，100-300ms的感知-动作延迟是基本可用的；移动机器人由于环境动态性更高，通常要求50-150ms。

## 2. 视觉感知模块

### 2.1 相机配置

机器人视觉系统通常配备多种相机，以满足不同任务需求：

**单目相机**：成本低、部署简单，适合物体识别、场景理解等以二维图像为主的任务。分辨率选择通常在640×480到1920×1080之间，帧率30-60fps。

**深度相机**：通过结构光（Intel RealSense）、TOF（Time of Flight）或双摄立体匹配获取深度信息，适合抓取、避障等需要三维信息的任务。典型产品包括：
- Intel RealSense D415/D435i：RGBD，深度精度约2mm@2m
- Azure Kinect DK：RGBD+IMU，深度精度1mm@1m
- Orbbec Astra：经济型RGBD方案

**腕部相机**：安装在机械臂末端，接近工作物体，提供近距离、高分辨率的视觉信息，对于精细操作（如插孔、装配）至关重要。

ROS2中相机的标准接口是`image_transport`，支持raw、compressed、theora等多种传输方式。对于VLA推理节点，订阅相机的`/image_raw`话题即可获取原始图像。

```python
# ROS2相机节点配置示例
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from cv_bridge import CvBridge

class CameraNode(Node):
    """相机采集节点
    
    负责从相机获取图像并发布到ROS2话题，
    VLA推理节点订阅该话题获取视觉输入。
    """
    
    def __init__(self):
        super().__init__('camera_node')
        # 图像转OpenCV格式的桥接器
        self.bridge = CvBridge()
        
        # 发布图像话题（raw格式）
        self.publisher = self.create_publisher(
            Image,
            '/camera/image_raw',
            10
        )
        
        # 订阅相机驱动话题（不同相机驱动话题不同）
        self.subscription = self.create_subscription(
            Image,
            '/device_0/sensor__COLOR/image_color',  # 根据实际相机调整
            self.image_callback,
            10
        )
        
        self.get_logger().info('相机节点已初始化')
    
    def image_callback(self, msg):
        """图像回调函数"""
        # 图像消息转换为OpenCV格式
        cv_image = self.bridge.imgmsg_to_cv2(msg, desired_encoding='bgr8')
        
        # 发布原始图像
        self.publisher.publish(
            self.bridge.cv2_to_imgmsg(cv_image, encoding='bgr8')
        )
```

### 2.2 图像预处理

VLA模型对输入图像有特定的格式要求，通常需要预处理步骤：

**尺寸调整**：VLA模型的视觉编码器通常接受固定尺寸输入（如224×224或384×384）。需要将相机采集的图像resize到目标尺寸。使用双线性插值即可，高质量resize对性能影响不大。

**归一化**：将像素值从[0, 255]归一化到[0, 1]或标准化到均值0、标准差1。归一化参数应与VLA模型预训练时一致。

**通道顺序**：OpenCV采集的图像是BGR格式，而大多数VLA模型使用RGB格式，需要转换。

```python
import numpy as np
import cv2

class ImagePreprocessor:
    """VLA图像预处理器
    
    将相机采集的原始图像预处理为VLA模型所需的格式。
    预处理参数应与模型预训练保持一致。
    """
    
    def __init__(self, target_size=(224, 224), 
                 mean=(0.485, 0.456, 0.406),
                 std=(0.229, 0.224, 0.225)):
        """
        Args:
            target_size: VLA模型输入尺寸 (宽, 高)
            mean: ImageNet归一化均值
            std: ImageNet归一化标准差
        """
        self.target_size = target_size
        self.mean = np.array(mean).reshape(1, 1, 3)
        self.std = np.array(std).reshape(1, 1, 3)
    
    def preprocess(self, image):
        """
        预处理单张图像
        
        Args:
            image: OpenCV格式BGR图像，numpy数组
            
        Returns:
            preprocessed: 预处理后的浮点数组，形状 (H, W, 3)
        """
        # BGR转RGB
        image = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)
        
        # 尺寸调整：双线性插值
        image = cv2.resize(image, self.target_size, 
                          interpolation=cv2.INTER_LINEAR)
        
        # 归一化到[0, 1]
        image = image.astype(np.float32) / 255.0
        
        # 标准化
        image = (image - self.mean) / self.std
        
        return image
    
    def preprocess_batch(self, images):
        """批量预处理多张图像"""
        return np.stack([self.preprocess(img) for img in images])
```

### 2.3 物体检测与跟踪

VLA模型虽然具备场景理解能力，但在实际系统中，专门的物体检测与跟踪模块可以提供额外的鲁棒性：

**物体检测**：使用YOLO等实时检测模型定位图像中的物体及其边界框。检测结果可以：
- 作为VLA的额外输入（拼接在语言指令中）
- 用于视觉伺服（Visual Servoing）的特征跟踪
- 提供物体空间位置用于运动规划

**物体跟踪**：在连续帧之间跟踪特定物体，解决VLA推理频率有限（通常1-10Hz）时的运动插值问题。常用算法包括：
- SORT/DeepSORT：基于检测的多目标跟踪
- ByteTrack：高性能跟踪器
- 光流法（Optical Flow）：连续帧间的像素级跟踪

```python
import numpy as np
from collections import deque

class ObjectTracker:
    """物体跟踪器
    
    使用IOU匹配和卡尔曼滤波实现多目标跟踪，
    为VLA提供稳定的目标物体位置估计。
    """
    
    def __init__(self, max_disappeared=5, iou_threshold=0.3):
        """
        Args:
            max_disappeared: 目标连续消失多少帧后删除
            iou_threshold: IOU匹配阈值
        """
        self.max_disappeared = max_disappeared
        self.iou_threshold = iou_threshold
        
        # 跟踪器状态：{object_id: {"bbox": [...], "center": [...], "disappeared": n}}
        self.tracked_objects = {}
        self.next_object_id = 0
        self.frame_count = 0
    
    def compute_iou(self, box1, box2):
        """计算两个边界框的IOU"""
        x1_min, y1_min, x1_max, y1_max = box1
        x2_min, y2_min, x2_max, y2_max = box2
        
        # 交集区域
        inter_xmin = max(x1_min, x2_min)
        inter_ymin = max(y1_min, y2_min)
        inter_xmax = min(x1_max, x2_max)
        inter_ymax = min(y1_max, y2_max)
        
        if inter_xmax < inter_xmin or inter_ymax < inter_ymin:
            return 0.0
        
        inter_area = (inter_xmax - inter_xmin) * (inter_ymax - inter_ymin)
        
        # 并集区域
        box1_area = (x1_max - x1_min) * (y1_max - y1_min)
        box2_area = (x2_max - x2_min) * (y2_max - y2_min)
        union_area = box1_area + box2_area - inter_area
        
        return inter_area / union_area if union_area > 0 else 0.0
    
    def update(self, detections):
        """
        更新跟踪器状态
        
        Args:
            detections: 检测结果列表，每个元素为 [x_min, y_min, x_max, y_max, confidence, class_id]
            
        Returns:
            tracked: dict，键为object_id，值为 {"bbox": [...], "center": [...]}
        """
        self.frame_count += 1
        
        if len(detections) == 0:
            # 没有检测到物体，标记所有现有目标为消失
            for obj_id in list(self.tracked_objects.keys()):
                self.tracked_objects[obj_id]["disappeared"] += 1
                if self.tracked_objects[obj_id]["disappeared"] > self.max_disappeared:
                    del self.tracked_objects[obj_id]
            return self.tracked_objects
        
        # 提取检测框
        detected_boxes = [det[:4] for det in detections]
        
        if len(self.tracked_objects) == 0:
            # 第一帧，初始化所有检测目标
            for det in detections:
                obj_id = self.next_object_id
                self.next_object_id += 1
                x_min, y_min, x_max, y_max = det[:4]
                self.tracked_objects[obj_id] = {
                    "bbox": det[:4],
                    "center": [(x_min + x_max) / 2, (y_min + y_max) / 2],
                    "disappeared": 0
                }
        else:
            # 匹配检测框与跟踪目标
            object_ids = list(self.tracked_objects.keys())
            object_boxes = [self.tracked_objects[oid]["bbox"] for oid in object_ids]
            
            # 计算IOU矩阵
            iou_matrix = np.zeros((len(object_ids), len(detected_boxes)))
            for i, obj_box in enumerate(object_boxes):
                for j, det_box in enumerate(detected_boxes):
                    iou_matrix[i, j] = self.compute_iou(obj_box, det_box)
            
            # 贪心匹配
            matched_objects = set()
            matched_detections = set()
            
            for _ in range(min(len(object_ids), len(detected_boxes))):
                max_iou_idx = np.unravel_index(np.argmax(iou_matrix), iou_matrix.shape)
                if iou_matrix[max_iou_idx] < self.iou_threshold:
                    break
                
                obj_idx, det_idx = max_iou_idx
                obj_id = object_ids[obj_idx]
                detected_boxes[det_idx]
                
                # 更新跟踪目标
                x_min, y_min, x_max, y_max = detected_boxes[det_idx]
                self.tracked_objects[obj_id] = {
                    "bbox": detected_boxes[det_idx],
                    "center": [(x_min + x_max) / 2, (y_min + y_max) / 2],
                    "disappeared": 0
                }
                
                matched_objects.add(obj_idx)
                matched_detections.add(det_idx)
                iou_matrix[max_iou_idx] = 0  # 避免重复匹配
            
            # 未匹配的目标标记为消失
            for i, obj_id in enumerate(object_ids):
                if i not in matched_objects:
                    self.tracked_objects[obj_id]["disappeared"] += 1
                    if self.tracked_objects[obj_id]["disappeared"] > self.max_disappeared:
                        del self.tracked_objects[obj_id]
            
            # 新检测到的目标初始化为新跟踪目标
            for j, det_box in enumerate(detected_boxes):
                if j not in matched_detections:
                    obj_id = self.next_object_id
                    self.next_object_id += 1
                    x_min, y_min, x_max, y_max = det_box
                    self.tracked_objects[obj_id] = {
                        "bbox": det_box,
                        "center": [(x_min + x_max) / 2, (y_min + y_max) / 2],
                        "disappeared": 0
                    }
        
        return self.tracked_objects
```

## 3. 语音指令解析

### 3.1 语音识别

语音识别（ASR，Automatic Speech Recognition）是将用户的语音输入转换为文本的技术。对于机器人控制场景，常用的ASR方案包括：

**在线方案**：
- Whisper（OpenAI）：支持多语言、中英文混说效果好、开源可本地部署
- WhisperX：在Whisper基础上增加了词级时间戳对齐
- Paraformer（阿里）：中文识别效果好

**ROS2集成方式**：通过ROS2的action或service机制调用ASR服务，保持与非ROS2的ASR引擎解耦。

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String
from sensor_msgs.msg import AudioData

class SpeechRecognitionNode(Node):
    """语音识别节点
    
    订阅音频数据流，通过Whisper模型进行语音识别，
    识别结果以文本形式发布到指令话题。
    """
    
    def __init__(self, model_name="base"):
        super().__init__('speech_recognition_node')
        
        # Whisper模型加载（首次调用时下载）
        # 注意：实际部署时建议使用torch.compile优化
        self.get_logger().info(f'加载Whisper模型: {model_name}')
        import whisper
        self.model = whisper.load_model(model_name)
        self.get_logger().info('Whisper模型加载完成')
        
        # 音频缓存，用于积累足够的音频片段
        self.audio_buffer = []
        self.sample_rate = 16000
        self.max_buffer_duration = 30  # 最大缓存30秒
        
        # 订阅音频话题（麦克风节点发布）
        self.audio_sub = self.create_subscription(
            AudioData,
            '/audio/raw',
            self.audio_callback,
            10
        )
        
        # 发布识别结果
        self.text_pub = self.create_publisher(
            String,
            '/speech/text',
            10
        )
        
        # 静音检测参数
        self.silence_threshold = 0.01
        self.silence_frames = 0
        self.speech_frames = 0
        self.is_speaking = False
        
        self.get_logger().info('语音识别节点已初始化')
    
    def audio_callback(self, audio_msg):
        """音频数据回调"""
        import numpy as np
        
        # 将音频数据转换为numpy数组
        audio_data = np.frombuffer(audio_msg.data, dtype=np.int16)
        audio_float = audio_data.astype(np.float32) / 32768.0
        
        # 计算音频能量
        energy = np.sqrt(np.mean(audio_float ** 2))
        
        # 简单的语音活动检测（VAD）
        if energy > self.silence_threshold:
            self.speech_frames += 1
            self.silence_frames = 0
            self.is_speaking = True
            self.audio_buffer.append(audio_float)
        else:
            self.silence_frames += 1
            if self.silence_frames > 30:  # 约0.5秒静音
                if self.is_speaking and len(self.audio_buffer) > 0:
                    # 静音触发识别
                    self.process_audio()
                    self.audio_buffer = []
                    self.is_speaking = False
                self.silence_frames = 0
    
    def process_audio(self):
        """处理缓存的音频并进行识别"""
        import numpy as np
        
        if not self.audio_buffer:
            return
        
        # 拼接音频片段
        audio = np.concatenate(self.audio_buffer)
        
        # 限制最大长度
        if len(audio) > self.sample_rate * self.max_buffer_duration:
            audio = audio[:self.sample_rate * self.max_buffer_duration]
        
        # Whisper推理
        result = self.model.transcribe(
            audio,
            language='zh',  # 中文识别
            fp16=False  # CPU模式使用fp32
        )
        
        text = result["text"].strip()
        
        if text:
            self.get_logger().info(f'识别结果: {text}')
            
            # 发布识别文本
            msg = String()
            msg.data = text
            self.text_pub.publish(msg)
```

### 3.2 指令理解

识别出的文本需要进一步解析为机器人可执行的指令。这一步通常涉及：

**意图识别（Intent Detection）**：判断用户指令属于哪类操作，如"抓取"、"移动"、"放置"等。

**实体抽取（Entity Extraction）**：从指令中提取关键信息：
- 目标物体："拿起**红色的杯子**"
- 位置信息："放到**桌子左边**"
- 动作参数："用**较慢的**速度移动"

**任务分解（Task Decomposition）**：复杂指令拆分为子动作序列。例如"把桌上的杯子放到柜子里"可分解为：
1. 移动到桌子附近
2. 定位杯子
3. 抓取杯子
4. 移动到柜子附近
5. 放置杯子

```python
import re
from enum import Enum
from dataclasses import dataclass
from typing import List, Optional

class RobotAction(Enum):
    """机器人动作枚举"""
    MOVE_TO = "move_to"
    GRASP = "grasp"
    PLACE = "place"
    PUSH = "push"
    PULL = "pull"
    POUR = "pour"
    OPEN = "open"
    CLOSE = "close"
    LOOK_AT = "look_at"
    WAIT = "wait"
    HOME = "home"

@dataclass
class ParsedInstruction:
    """解析后的指令数据结构"""
    action: RobotAction
    target_object: Optional[str] = None
    location: Optional[str] = None
    speed: Optional[str] = None
    confidence: float = 1.0
    raw_text: str = ""

class InstructionParser:
    """自然语言指令解析器
    
    将用户的自然语言指令解析为结构化的机器人动作指令。
    使用规则匹配实现，可扩展为LLM-based解析。
    """
    
    # 动作关键词映射
    ACTION_PATTERNS = {
        RobotAction.GRASP: [r"抓[取取]?", r"拿起", r"拾取", r"拿.*起"],
        RobotAction.PLACE: [r"放下", r"放置", r"放.*到", r"放到"],
        RobotAction.MOVE_TO: [r"移动到", r"走到", r"去.*那边", r"导航到"],
        RobotAction.PUSH: [r"推", r"按"],
        RobotAction.PULL: [r"拉", r"拽"],
        RobotAction.POUR: [r"倒", r"倾倒"],
        RobotAction.OPEN: [r"打开", r"开"],
        RobotAction.CLOSE: [r"关闭", r"关上", r"合上"],
        RobotAction.LOOK_AT: [r"看", r"观察", r"查看"],
        RobotAction.WAIT: [r"等[待一下]", r"停[止留]"],
        RobotAction.HOME: [r"回零", r"复位", r"归位", r"回家"],
    }
    
    # 速度关键词
    SPEED_PATTERNS = {
        "slow": [r"慢[速的]", r"慢一点", r"小心"],
        "medium": [r"正常", r"中速", r"一般"],
        "fast": [r"快[速的]?", r"快点", r"迅速"],
    }
    
    # 物体颜色修饰词
    COLOR_WORDS = [r"(红|黄|蓝|绿|白|黑|紫|橙)(色)的?"]
    
    def __init__(self):
        # 编译正则表达式
        self.action_regexes = {
            action: [re.compile(p, re.IGNORECASE) for p in patterns]
            for action, patterns in self.ACTION_PATTERNS.items()
        }
        
        self.speed_regexes = {
            speed: [re.compile(p, re.IGNORECASE) for p in patterns]
            for speed, patterns in self.SPEED_PATTERNS.items()
        }
    
    def parse(self, text: str) -> List[ParsedInstruction]:
        """
        解析文本指令
        
        Args:
            text: 用户输入的自然语言文本
            
        Returns:
            解析后的指令列表，每个指令包含动作类型和参数
        """
        results = []
        text = text.strip()
        
        # 检测动作类型
        detected_actions = []
        for action, regexes in self.action_regexes.items():
            for regex in regexes:
                match = regex.search(text)
                if match:
                    detected_actions.append((action, match.start()))
                    break
        
        if not detected_actions:
            # 无法识别动作，返回wait指令
            return [ParsedInstruction(
                action=RobotAction.WAIT,
                confidence=0.0,
                raw_text=text
            )]
        
        # 按文本位置排序
        detected_actions.sort(key=lambda x: x[1])
        
        for action, start_pos in detected_actions:
            # 提取动作后的文本（可能包含目标对象）
            action_text = text[start_pos:]
            
            # 提取颜色修饰词
            target_object = None
            for color_regex in self.COLOR_WORDS:
                color_match = re.search(color_regex, action_text)
                if color_match:
                    target_object = color_match.group(0)
                    break
            
            # 提取速度
            speed = None
            for speed_val, speed_regexes in self.speed_regexes.items():
                for regex in speed_regexes:
                    if regex.search(text):
                        speed = speed_val
                        break
            
            # 提取位置信息（简单实现：寻找"到"、"在"等介词后的内容）
            location = None
            location_match = re.search(r"(到|在|去)(.{1,20})", action_text)
            if location_match:
                location = location_match.group(2).rstrip("的")
            
            results.append(ParsedInstruction(
                action=action,
                target_object=target_object,
                location=location,
                speed=speed,
                confidence=0.8,  # 简化：固定置信度
                raw_text=text
            ))
        
        return results

# 使用示例
if __name__ == "__main__":
    parser = InstructionParser()
    
    test_instructions = [
        "把红色的杯子放到桌子上",
        "慢一点拿起蓝色的方块",
        "移动到厨房那边",
        "打开抽屉",
        "等待一下",
    ]
    
    for instr in test_instructions:
        results = parser.parse(instr)
        print(f"\n原始指令: {instr}")
        for r in results:
            print(f"  动作: {r.action.value}")
            print(f"  目标: {r.target_object}")
            print(f"  位置: {r.location}")
            print(f"  速度: {r.speed}")
```

### 3.3 任务分解

复杂任务需要分解为VLA可执行的子动作序列。任务分解可以在两个层面进行：

**高层规划（LLM/规则引擎）**：使用大语言模型或规则系统将自然语言指令分解为动作序列。这种方法灵活、可处理复杂指令，但延迟较高。

**低层执行（VLA）**：VLA接收高层规划的子目标（如"抓取杯子"），输出具体的控制动作（如关节角度序列）。VLA的每一步推理对应一个短时动作。

```python
from typing import List, Callable
import time

class TaskDecomposer:
    """任务分解器
    
    将高层用户指令分解为可执行的子动作序列。
    当前实现为基于规则的分解，可扩展为LLM调用。
    """
    
    # 复合动作模板
    TASK_TEMPLATES = {
        "抓取并放置": [
            {"action": "MOVE_TO", "target": "approach_{object}"},  # 移动到物体附近
            {"action": "MOVE_TO", "target": "above_{object}"},      # 对准物体上方
            {"action": "LOWER"},                                    # 下降
            {"action": "GRASP", "target": "{object}"},              # 抓取
            {"action": "RAISE"},                                    # 上升
            {"action": "MOVE_TO", "target": "approach_{location}"},# 移动到目标位置
            {"action": "MOVE_TO", "target": "above_{location}"},   # 对准目标上方
            {"action": "LOWER"},                                    # 下降
            {"action": "RELEASE"},                                  # 释放
            {"action": "RAISE"},                                    # 上升
            {"action": "HOME"},                                     # 回归初始位姿
        ],
        "打开抽屉": [
            {"action": "MOVE_TO", "target": "near_drawer"},
            {"action": "LOOK_AT", "target": "drawer_handle"},
            {"action": "MOVE_TO", "target": "drawer_handle"},
            {"action": "GRASP", "target": "drawer_handle"},
            {"action": "PULL", "target": "drawer", "distance": 0.3},
            {"action": "RELEASE"},
        ],
        "倒水": [
            {"action": "MOVE_TO", "target": "near_cup"},
            {"action": "GRASP", "target": "cup"},
            {"action": "MOVE_TO", "target": "above_kettle"},
            {"action": "TILT", "angle": 45},
            {"action": "WAIT", "duration": 3.0},
            {"action": "TILT", "angle": 0},
            {"action": "MOVE_TO", "target": "near_person"},
            {"action": "PLACE", "target": "cup"},
        ]
    }
    
    def __init__(self, llm_client=None):
        """
        Args:
            llm_client: 可选的LLM客户端，用于复杂任务分解
        """
        self.llm_client = llm_client
    
    def decompose(self, instruction: str) -> List[dict]:
        """
        分解高层指令为子动作序列
        
        Args:
            instruction: 解析后的指令
            
        Returns:
            子动作序列列表
        """
        action = instruction.action.value
        target = instruction.target_object or ""
        location = instruction.location or ""
        
        # 模板匹配
        template_key = None
        for key in self.TASK_TEMPLATES:
            if key in instruction.raw_text:
                template_key = key
                break
        
        if template_key:
            template = self.TASK_TEMPLATES[template_key]
            # 填充模板参数
            sub_actions = []
            for step in template:
                step_copy = step.copy()
                if "{object}" in str(step_copy):
                    step_copy["target"] = step_copy["target"].format(object=target)
                if "{location}" in str(step_copy):
                    step_copy["target"] = step_copy["target"].format(location=location)
                sub_actions.append(step_copy)
            return sub_actions
        
        # 无模板匹配时的简单处理
        if action == "grasp":
            return [
                {"action": "MOVE_TO", "target": f"above_{target}"},
                {"action": "LOWER"},
                {"action": "GRASP", "target": target},
                {"action": "RAISE"},
            ]
        elif action == "place":
            return [
                {"action": "MOVE_TO", "target": f"above_{location}"},
                {"action": "LOWER"},
                {"action": "RELEASE"},
                {"action": "RAISE"},
            ]
        elif action == "move_to":
            return [
                {"action": "MOVE_TO", "target": location or target},
            ]
        else:
            return [{"action": action.upper(), "target": target}]
```

## 4. 动作执行

### 4.1 关节控制

机械臂的关节控制是最底层的运动控制，直接控制各关节的角度或力矩。ROS2中通过`joint_trajectory_controller`或直接发送`JointTrajectory`消息实现。

**关节空间控制**：直接指定各关节的目标角度，控制器通过轨迹插值平滑过渡。

**速度控制**：指定关节的转动速度，适用于连续轨迹跟踪。

**力矩控制**：指定关节的输出力矩，适用于力控场景（如打磨、装配）。
