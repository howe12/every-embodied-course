## 1. 概述：从ChatGPT for Robotics到π0.6

### 1.1 背景

从2023年ChatGPT for Robotics到2025年π0.6的RECAP，具身智能操作（Manipulation）领域两年经历了至少**三次范式跃迁**。本期视频系统梳理这条技术路线从头到尾的全貌，每一代技术讲了什么问题、怎么解决的、为什么重要。

### 1.2 时间线总览

| 时间节点 | 技术阶段 | 代表工作 |
|----------|----------|----------|
| 2023年初 | 开局：SayCan时代 | Google奠定基础 |
| 2023年中 | 第一跃迁：Action Chunking | ACT + ALOHA |
| 2023年底 | 第二跃迁：扩散模型 | Diffusion Policy |
| 2024年 | 第三跃迁：VLA统一 | OpenVLA / SmolVLA / π0 |
| 2025年 | 强化学习突破 | RECAP |

### 1.3 三次范式跃迁

```
第一跃迁 (2023): 视觉-语言-动作统一（Action Chunking）
第二跃迁 (2023): 扩散模型生成动作（Diffusion Policy）  
第三跃迁 (2024): VLA端到端（OpenVLA / π0）
```

### 1.4 如何判断是否为范式跃迁

#### 范式跃迁的定义
在具身智能领域，"范式跃迁"指的是**解决问题的主导方法发生了根本性变化**，而非仅仅是性能提升或局部改进。参考Kuhn范式理论，真正的范式跃迁需要满足以下特征：

| 特征 | 说明 | 判断标准 |
|------|------|----------|
| **问题定义重构** | 之前无法解决的问题现在可以被解决 | 新范式能处理之前方法的 Hard Case |
| **架构/方法根本变化** | 核心模块的连接方式发生质变 | 不是微调而是重新设计 |
| **数据利用方式变革** | 对数据的假设或使用方式改变 | 能利用之前方法无法使用的数据 |
| **涌现新能力** | 出现之前方法不可能涌现的能力 | 零样本推理、跨任务泛化等 |

#### 判断方法：四个问题测试

拿到一篇新论文或新工作时，用以下四个问题判断是否属于范式跃迁：

**问题1：它解决了之前方法完全无法解决的问题吗？**
- ✅ 是 → 可能是范式跃迁
- ❌ 否（只是性能提升）→ 不是范式跃迁

**问题2：它的核心架构/方法是否引入了之前领域不存在的设计？**
- ✅ 是（例如 Transformer、扩散模型）→ 可能是范式跃迁
- ❌ 否（只是组合已有模块）→ 不是范式跃迁

**问题3：它的训练方式或数据利用方式是否发生了根本变化？**
- ✅ 是 → 可能是范式跃迁
- ❌ 否 → 不是范式跃迁

**问题4：它是否涌现出了不可能在之前框架下出现的全新能力？**
- ✅ 是（例如 RT-2 的零样本推理）→ 一定是范式跃迁
- ❌ 否 → 可能不是范式跃迁

#### 历史范式跃迁案例分析

| 跃迁 | 问题1 | 问题2 | 问题3 | 问题4 | 结论 |
|------|-------|-------|-------|-------|------|
| **SayCan (2022)** | ✅ LLM+机器人 | ✅ 视觉-动作统一 | ✅ 互联网预训练 | ❌ | 局部跃迁 |
| **RT-1 (2022)** | ❌ 性能提升 | ✅ Transformer统一 | ❌ | ❌ | 不是跃迁 |
| **ACT (2023)** | ✅ 长时序误差 | ✅ Action Chunking | ❌ | ✅ 零样本 | **是范式跃迁** |
| **Diffusion Policy (2023)** | ✅ 多模态动作 | ✅ 扩散生成 | ❌ | ✅ 复杂分布 | **是范式跃迁** |
| **RT-2 (2023)** | ❌ 性能提升 | ✅ 动作离散化 | ✅ VLM统一 | ✅ 零样本推理 | **是范式跃迁** |
| **π0 (2024)** | ✅ 人形精细操作 | ✅ Flow Matching | ✅ 互联网视频预训练 | ✅ 双系统架构 | **是范式跃迁** |
| **RECAP (2025)** | ✅ 超越演示者 | ✅ RL+模仿融合 | ✅ 对比动作提议 | ✅ 超人类水平 | **是范式跃迁** |

#### 警惕「伪范式跃迁」

以下情况容易被误判为范式跃迁，实际上只是技术进步：

| 伪跃迁特征 | 典型案例 | 辨别方法 |
|------------|----------|----------|
| **数据规模驱动** | 更大的模型 + 更多数据 | 架构没有本质变化，只是 scale |
| **benchmark 刷点** | 在特定数据集上提升 10% | 换个数据集就退步 |
| **工程优化** | 量化加速、蒸馏压缩 | 方法论没有创新 |

## 2. 第一次范式跃迁：Google奠基工作

### 2.1 SayCan：语言指令落地

**SayCan**（2022）是Google提出的将大型语言模型（LLM）与机器人控制结合的工作。核心思想：

- **Say**：用LLM理解高层语言指令
- **Can**：用价值函数评估每个动作的可行性
- **结合**：选择既"想说"又"能做"的动作

### 2.2 RT-1：统一视觉-语言-动作

**RT-1**（Robotics Transformer 1，2022）首次用**Transformer架构**统一处理视觉观察和语言指令：

- 输入：图像序列 + 文本指令
- 输出：机器人动作序列
- 数据：13万台机器人执行数据
- 特点：能够泛化到新任务



#### 2.2.1 什么是 Transformer 架构

**Transformer**（2017）是 Google 提出的革命性序列建模架构，核心是**自注意力机制（Self-Attention）**。

**传统 RNN 的问题**：顺序处理无法并行，长距离依赖需要一步步传递。

**自注意力机制**：
对于输入序列中的每个位置，Self-Attention 计算它与所有其他位置的相关性，然后加权求和：

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中：
- $Q$（Query）：当前词的查询向量
- $K$（Key）：每个词的键向量
- $V$（Value）：每个词的值向量



**为什么 Transformer 适合机器人控制**：并行计算、长距离依赖、可扩展。

#### 2.2.2 RT-1 如何统一处理视觉和语言

RT-1 使用 **FiLM（Feature-wise Linear Modulation）** 条件网络融合视觉和语言：

$$\text{FiLM}(\mathbf{h}; \gamma, \beta) = \gamma \odot \mathbf{h} + \beta$$



#### 2.2.3 RT-1 如何生成动作序列

RT-1 动作维度：7维机械臂 + 2维底座 + 1维夹爪 = 10维动作。



泛化能力来源：大规模13万台机器人数据 + Transformer归纳偏置 + 语言指令条件调制。

### 2.3 RT-2：VLM+动作统一训练

**RT-2**（2023）将视觉语言模型（VLM）与动作预测**统一训练**：

- 核心创新：将动作离散化为token，与语言token一起训练
- 效果：涌现出零样本推理能力
- 缺点：计算成本极高

### 2.4 Open X-Embodiment：数据共享

**Open X-Embodiment**（2023）由Google和DeepMind联合发布：

- 汇总了21种机器人、100+万条真实机器人轨迹
- 建立了机器人领域的"ImageNet时刻"
- 为后续模仿学习研究提供了统一数据基础

```
Google奠基工作演进路线：
SayCan → RT-1 → RT-2 → Open X-Embodiment
```

## 3. 第二次范式跃迁：模仿学习的巅峰

### 3.1 ACT与ALOHA：Action Chunking解决累积误差

**ACT**（Action Chunking with Transformers，2023）和**ALOHA**（2023）是斯坦福提出的模仿学习方法：

**核心问题**：模仿学习在长时序任务中存在**累积误差**——每个动作的微小偏差会随着时间累积，导致任务失败。

**解决方案**：Action Chunking（动作分块）

- 不预测单个动作，而是预测**一系列动作块**（chunk）
- 每个chunk包含多个时间步的动作序列
- 减少了误差累积

**ALOHA硬件系统**：
- 特点：低成本双手操作平台（总成本<2万美元）
- 录制真实演示数据
- 与ACT算法结合

**代码示例**：

```python
# ACT 动作分块示意
class ACT(nn.Module):
    def __init__(self, obs_dim, action_dim, seq_len=10, chunk_size=8):
        super().__init__()
        self.encoder = TransformerEncoder(obs_dim)
        self.policy = TransformerDecoder(action_dim, chunk_size)
    
    def forward(self, obs):
        # 编码观察
        features = self.encoder(obs)
        # 预测动作块（一次性输出chunk_size个动作）
        action_chunk = self.policy(features)  # [batch, chunk_size, action_dim]
        return action_chunk
```



#### 3.1.1 Action Chunking 的数学形式与误差分析

**累积误差问题（Compounding Error）**：

标准模仿学习中，每个时刻的误差会累积：

\$\$e_t = \| \hat{a}_t - a_t^* \|$$

经过 \$T\$ 步后，总误差约为 \$O(T \cdot ar{e})\$。对于长时序任务（50+步），累积误差会导致任务失败。

**Action Chunking 解决方案**：

不预测单个动作，而是预测一个 chunk 的动作序列 \$[a_t, a_{t+1}, ..., a_{t+K-1}]\$：

\$\$\hat{a}_{t:t+K-1} = \pi_{chunk}(o_t)\$$

这样误差被分散到 K 个动作上，每个动作的误差影响范围只有 1/K。

```python
class ActionChunkingPolicy(nn.Module):
    def __init__(self, obs_dim, action_dim, chunk_size=8):
        super().__init__()
        self.chunk_size = chunk_size
        self.encoder = TransformerEncoder(obs_dim, d_model=512, nhead=8, num_layers=6)
        self.chunk_decoder = TransformerDecoder(
            d_model=512, nhead=8, num_layers=6,
            output_dim=action_dim * chunk_size
        )
    
    def forward(self, obs):
        encoded = self.encoder(obs)
        chunk_output = self.chunk_decoder(encoded)
        chunk_actions = chunk_output.view(-1, self.chunk_size, obs.shape[1])
        current_action = chunk_actions[:, 0, :]
        return current_action, chunk_actions
```

### 3.2 Diffusion Policy：扩散模型生成动作

**Diffusion Policy**（2023）将**扩散模型**引入机器人动作生成：

**核心思想**：
- 将动作生成建模为**去噪扩散过程**
- 从随机噪声开始，逐步去噪生成高质量动作序列
- 解决了传统方法难以处理多模态动作分布的问题

**技术优势**：

| 指标 | 传统BC | Diffusion Policy |
|------|--------|------------------|
| 15个任务平均提升 | 基准 | **+46.9%** |
| 多模态动作分布 | 难以建模 | 自然建模 |
| 动作平滑性 | 一般 | 优秀 |
| 长时序任务 | 累积误差大 | 显著改善 |

**数学形式**：

DDPM（去噪扩散概率模型）的动作生成过程：

$$p_\theta(\mathbf{a}_{0:T} | \mathbf{x}) = \prod_{t=0}^{T} p_\theta^{(t)}(\mathbf{a}_t | \mathbf{a}_{t+1:T}, \mathbf{x})$$

其中 $p_\theta^{(t)}$ 是在 timestep $t$ 的去噪分布。

**代码示例**：

```python
# Diffusion Policy 核心去噪循环
def denoise(self, noisy_action, timestep, condition):
    """去噪过程"""
    noise_pred = self.noise_pred_net(noisy_action, timestep, condition)
    # 减去预测噪声
    denoised = noisy_action - self.alpha_bar[timestep] * noise_pred
    return denoised

def sample(self, condition, num_steps=50):
    """从噪声生成动作序列"""
    action = torch.randn(condition.shape[0], self.action_dim)
    for t in reversed(range(num_steps)):
        action = self.denoise(action, t, condition)
    return action
```

## 4. 第三次范式跃迁：VLA统一架构

### 4.1 VLA的核心思想

**VLA（Vision-Language-Action Model）** 是将视觉感知、语言理解和机器人动作控制统一到**同一个Transformer模型**中的端到端架构。

与之前的两阶段方法（感知→规划→控制）不同，VLA实现了**端到端**的直接映射：

```
传统方法：图像 → 感知模块 → 规划模块 → 控制模块 → 动作
VLA方法：图像 + 文本 → [VLA Transformer] → 动作
```

### 4.2 OpenVLA：开源7B VLA

**OpenVLA**（2024）由Stanford和UIUC联合发布：

- **参数量**：7B参数
- **数据**：开源数据集训练（Open X-Embodiment子集）
- **发布平台**：HuggingFace
- **特点**：完全开源，降低研究门槛

**架构**：
- 视觉编码器：SigLIP / DFN
- 语言模型：Llama 2 / Vicuna
- 动作头：线性投影层

### 4.3 SmolVLA：轻量级VLA

**SmolVLA**（2024）是OpenVLA的轻量版本：

- **参数量**：7B（但经过剪枝蒸馏）
- **推理速度**：比OpenVLA快3倍
- **硬件需求**：单卡A100即可运行
- **适合场景**：资源受限的边缘部署

### 4.4 π0：面向人形机器人的VLA

**π0**（Physical Intelligence，2024）是专门面向**人形机器人**设计的VLA：

- **架构**：扩散+流匹配（Flow Matching）
- **创新点**：将扩散模型的表达能力与VLA架构结合
- **控制频率**：50Hz实时控制
- **泛化能力**：零样本泛化到新任务

**与OpenVLA的核心区别**：

| 特性 | OpenVLA | π0 |
|------|---------|-----|
| 训练数据 | 机器人数据 | 人形机器人专项数据 |
| 动作生成 | 离散动作token | 连续动作（流匹配） |
| 控制频率 | 较低 | 50Hz |
| 开源程度 | 完全开源 | 部分开源 |

## 6. RECAP强化学习：突破模仿学习天花板

### 6.1 模仿学习的局限

模仿学习（Imitation Learning）存在**天花板效应**：

- 智能体的能力上限是演示数据分布
- 无法突破演示者的水平
- 对新场景泛化能力有限

### 6.2 RECAP的核心思想

**RECAP**（Reinforcement Learning with Contrastent Action Proposals，2025）是Physical Intelligence提出的强化学习方法：

- **核心创新**：用强化学习在模仿学习基础上继续优化
- **对比动作提议**：生成多样化的动作候选，对比选择最优
- **突破天花板**：智能体可以超越演示者水平

### 6.3 技术细节

RECAP的工作流程：

1. **动作提议生成**：基于当前观察生成多个候选动作
2. **价值评估**：用 critic 网络评估每个候选的价值
3. **优势计算**：对比选最优
4. **策略更新**：用优势信号更新策略网络

**数学形式**：

$$L_{RECAP} = \mathbb{E}_{(\mathbf{o}, \mathbf{a}) \sim D} \left[ \sum_{t} A(\mathbf{a}_t, \mathbf{a}'_t | \mathbf{o}_t) \cdot \log \pi(\mathbf{a}_t | \mathbf{o}_t) \right]$$

其中 $A(\mathbf{a}_t, \mathbf{a}'_erth | \mathbf{o}_t)$ 是优势函数，$\mathbf{a}'_t$ 是候选动作。

### 6.4 为什么是2025年最重要进展

RECAP代表了具身智能的**范式转变**：

- 从"学习模仿"到"学习+优化"
- 打破了模仿学习的天花板
- 为通用机器人操作开辟了新道路

---



#### 6.3.1 RECAP 详细机制

**为什么模仿学习有天花板**：

模仿学习的数据来自专家演示，策略只能学到演示数据分布内的行为。即使环境中存在更好的动作，如果专家没有演示过，模仿学习也无法学到。

**RECAP 解决思路**：

不只模仿专家动作，而是**生成多样化的动作候选池，通过强化学习选择最优**。

**三个核心组件**：

1. **动作提议生成器**：基于观察 \$o_t\$ 生成 \$K\$ 个候选动作 \$a_t^1, ..., a_t^K\$，使用对比学习确保多样性

2. **价值评估网络**：\$V(a_t^k | o_t) = \text{Critic}(o_t, a_t^k)\$

3. **优势函数与策略更新**：
\$\pi(a_t | o_t) \propto \sum_k \exp(\beta \cdot V(a_t^k | o_t)) \cdot \delta(a_t = a_t^k)\$

```python
for epoch in range(num_epochs):
    for batch in dataloader:
        obs, actions, rewards = batch
        proposals = proposal_generator(obs, num_proposals=8)
        values = critic(obs, proposals)
        best_idx = values.argmax(dim=1)
        advantages = values - values.mean(dim=1, keepdim=True)
        policy_loss = -advantages.mean()
        policy_loss.backward()
        optimizer.step()
```

**为什么能突破天花板**：

1. **多样化的候选动作**：即使专家没演示过的动作，只要在候选池中就能被选择
2. **价值函数的泛化**：即使某动作没在专家数据中出现，只要价值函数能评估就能被选中
3. **持续优化**：每轮训练后，价值函数和策略都会更新，选择越来越好的动作

## 7. 开源VLA对比：π0 vs GR00T N1

### 7.1 GR00T N1

## 8. 世界模型：UniSim / DreamerV3 / Cosmos / GigaBrain

### 8.1 什么是世界模型

**世界模型（World Model）** 是智能体对环境动态的内部表示，用于：

- 预测动作的后果
- 规划长时序任务
- 减少真实机器人交互需求

### 8.2 代表工作

#### UniSim（NVIDIA）

- **功能**：视频生成+动作控制
- **特点**：生成逼真的机器人操作视频
- **应用**：数据扩增、规划验证

#### DreamerV3（Google）

- **架构**：世界模型+强化学习统一框架
- **创新**：从像素学习，无需人工设计的表示
- **应用**：Atari游戏、机器人操作

#### Cosmos（NVIDIA）

- **规模**：大规模视频生成模型
- **特点**：物理真实的视频生成
- **应用**：仿真数据生成

#### GigaBrain（Meta）

- **目标**：构建通用世界模型
- **数据**：海量互联网视频预训练
- **特点**：强大的零样本泛化能力

### 8.3 世界模型 vs VLA

| 对比 | 世界模型 | VLA |
|------|----------|-----|
| 目标 | 预测环境动态 | 直接输出动作 |
| 训练方式 | 自监督（视频预测） | 模仿/强化学习 |
| 应用方式 | 规划+仿真 | 端到端控制 |
| 代表 | UniSim, DreamerV3 | RT-2, π0 |
