## 1. 赛道概述

### 1.1 什么是 World Model（世界模型）？

World Model 是机器人领域的重要概念。简单来说：

```
世界模型 = 让 AI 理解物理世界如何运作

举例：
- 推一下杯子，杯子会倒
- 拿起苹果，苹果在手的上方
- 水倒入杯子，杯中水位上升
```

**在比赛中：** World Model 的任务是给定当前状态和动作，预测未来的视觉结果。

### 1.2 赛道目标

| 项目 | 内容 |
|------|------|
| **输入** | 首帧图像 + 动作信号 |
| **输出** | 未来若干帧的视频 |
| **核心能力** | 理解动作与视觉变化的对应关系 |

### 1.3 为什么这个赛道重要？

```
传统方法：直接执行动作
     ↓
     可能出错（物体没抓住、碰撞等）

World Model：先预测再执行
     ↓
     在虚拟环境中验证动作安全性
     ↓
     选出最优动作执行
```

**应用场景：**
- 自动驾驶预测
- 机器人规划
- 游戏模拟
- 数字孪生

## 3. 技术背景

### 3.1 EVAC 模型架构

EVAC (Envisioning Embodied Environments with Action Condition) 是一种**动作条件的视频生成模型**。

**核心思想：**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  图像编码器  │     │  动作编码器  │     │  视频解码器  │
│ (CLIP ViT) │     │ (多层感知机) │     │ (扩散模型)  │
└──────┬──────┘     └──────┬──────┘     └──────┬──────┘
       │                   │                   │
       └────────┬──────────┘                   │
                ↓                              │
         条件向量 C                             │
                ↓                              │
         扩散模型 U-Net                         │
                ↓                              │
         预测下一帧 ──────────────────────────→│
```

**关键组件：**

| 组件 | 作用 |
|------|------|
| CLIP Image Encoder | 将首帧图像编码为特征 |
| Action Encoder | 将动作信号编码为特征 |
| Latent Diffusion | 在隐空间进行扩散生成 |
| Autoencoder (AE) | 图像 ↔ 隐空间 转换 |

### 3.2 扩散模型基础

扩散模型是当前 SOTA 的生成模型技术：

**前向过程（加噪声）：**
```
图像 → 加噪声 → 加噪声 → ... → 纯噪声
  x₀      x₁      x₂           xₜ
```

**反向过程（去噪声）：**
```
纯噪声 → 去噪声 → 去噪声 → ... → 生成图像
  xₜ      xₜ₋₁    xₜ₋₂         x₀
```

**动作条件：** 在去噪声过程中加入动作信息，引导生成符合动作的视频。

### 3.3 隐空间扩散

EVAC 不直接在像素空间生成，而是在**隐空间**操作：

```
原始图像 (512x512x3)
       ↓ Autoencoder
隐变量 (32x32x4)
       ↓ 扩散 + 动作条件
生成隐变量
       ↓ Autoencoder
生成图像 (512x512x3)
```

**优势：** 计算效率高，生成质量好

## 4. 评测指标

### 4.1 官方指标

| 指标 | 含义 | EVAC 基线 |
|------|------|-----------|
| **PSNR** | 峰值信噪比，越高越好 | 20.98 |
| **Scene Consistency** | 场景一致性，值越接近1越好 | 0.90 |
| **nDTW** | 轨迹动态时间规整，越高越好 | 0.91 |

### 4.2 指标详解

#### PSNR (Peak Signal-to-Noise Ratio)

```python
PSNR = 10 * log10(MAX² / MSE)
# MAX = 255 (8位图像)
# MSE = 均方误差
```

**物理意义：** 生成图像与真值的像素级差异
- PSNR > 30：图像非常相似
- PSNR 20-30：有一定差异
- PSNR < 20：差异明显

#### Scene Consistency

衡量视频中场景元素的一致性：

```
场景一致性低 ←→ 物体消失、出现、闪烁
场景一致性高 ←→ 物体稳定、连续
```

**计算方法：** 检测视频中物体是否在帧间保持一致

#### nDTW (normalized Dynamic Time Warping)

动态时间规整，用于比较轨迹相似度：

```
真值轨迹：→ → → ↓
预测轨迹：→ → ↓ →

DTW 可以对齐这两个轨迹，计算相似度
```

## 6. 知识点详解

### 6.1 视频生成基础

**关键概念：**
- 帧率 (FPS)：视频每秒帧数
- 时序一致性：帧间内容连贯
- 动作连贯性：动作与视觉变化匹配

**挑战：**
- 长视频生成质量下降
- 复杂动作难以精确预测
- 物体持久性维护

### 6.2 动作编码

动作信号需要编码为模型可理解的向量：

```python
# 动作包含：
# - 末端位置 (x, y, z)
# - 末端姿态 (四元数 qx, qy, qz, qw)
# - 夹爪状态 (open/close)

action = {
    'position': [x, y, z],
    'orientation': [qx, qy, qz, qw],  # 四元数
    'gripper': 0.0  # 0=关闭, 1=打开
}
```

**编码方式：**
1. 归一化处理
2. MLP 多层感知机编码
3. 与图像特征融合

### 6.3 相机参数

**内参 (Intrinsic)：**
```python
{
    'fx': 焦距 x,
    'fy': 焦距 y,
    'ppx': 主点 x,
    'ppy': 主点 y
}
```

**外参 (Extrinsic)：**
- 相机到机器人基座的变换矩阵
- 用于坐标系对齐

### 6.4 隐变量扩散

```
隐空间 = 压缩后的图像表示
- 原始: 512x512x3 = 786,432 维
- 隐变量: 32x32x4 = 4,096 维
- 压缩比: ~192x
```

**优势：**
- 计算效率高
- 特征更紧凑
- 有利于生成一致性

## 8. 常见问题

### Q1: 为什么要生成视频而不是单帧？

**A:** 视频能展示动作的完整执行过程，评委可以看到：
- 动作是否合理
- 时序是否连贯
- 场景是否一致

### Q2: n_pred=3 是什么意思？

**A:** 每个输入样本生成 3 个不同的预测结果。官方评测会从多个预测中选择最好的或计算平均。

### Q3: 显存不足怎么办？

**A:** 
1. 减少 `batch_size`
2. 减少 `chunk` 参数
3. 使用 `fp16` 推理

### Q4: 如何提升 PSNR？

**A:** 
1. 提升图像生成质量
2. 减少模糊和伪影
3. 确保时序平滑

### Q5: Scene Consistency 低怎么办？

**A:**
1. 检查物体是否闪烁
2. 确保相机参数正确
3. 增强空间一致性约束
