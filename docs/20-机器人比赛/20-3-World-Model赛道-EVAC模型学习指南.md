## 第一章：基础概念

### 1.1 什么是 World Model？

#### 🎯 一句话理解

**World Model = 让 AI 理解"做什么动作会产生什么结果"**

就像人类婴儿通过不断尝试学会理解世界：
- 推杯子 → 杯子倒了
- 抓苹果 → 苹果在手心里
- 倒水 → 水流入杯中

#### 🎮 游戏比喻

想象你在玩电子游戏：

```
你看到的画面 ← 这是「视觉感知」
你按下的按钮 ← 这是「动作」
角色的反应 ← 这是「预测结果」

World Model 就是游戏引擎，能预测：
- 按这个键 → 角色会向右走
- 按那个键 → 角色会跳跃
```

#### 🏭 在机器人领域的意义

```
传统方法：直接执行动作
     ↓
     可能出错（物体没抓住、碰撞等）
     ↓
     试错成本高

有 World Model：先预测再执行
     ↓
     在虚拟环境中验证动作安全性
     ↓
     选出最优动作执行
     ↓
     试错成本低
```

#### 📊 比赛中的具体任务

```
输入：
┌─────────────┐
│ 摄像头画面   │ ← 机器人看到的
│ 关节角度     │ ← 机器人当前状态
│ 目标动作     │ ← 要执行的动作
└─────────────┘
     ↓
World Model
     ↓
输出：
┌─────────────┐
│ 执行动作后   │ ← 预测未来的画面
│ 的画面       │
└─────────────┘
```

### 1.2 视频生成 vs 图像生成

#### 🖼️ 图像生成

```
输入：文字描述 "一只可爱的猫"
     ↓
AI 生成：一张猫的图片
     ↓
输出：静态图片
```

#### 🎬 视频生成

```
输入：文字描述 "一个人在跑步"
     ↓
AI 生成：一系列连续的帧
     ↓
输出：动态视频（多帧图像按时间排列）
```

#### 🎯 视频预测（我们的任务）

```
输入：过去的画面 + 动作指令
     ↓
AI 预测：未来的画面
     ↓
输出：后续若干帧
```

#### 📈 时间维度的重要性

```
视频 = 空间信息 + 时间信息

空间信息：这帧里有什么（猫、杯子、桌子）
时间信息：它们如何变化（猫在移动、杯子在倒下）

预测未来 = 理解物理规律 + 动作效果
```

### 1.3 扩散模型直观理解

#### 💧 最通俗的解释：墨水扩散

想象你有一杯清水和一滴墨水：

**正向过程（加噪声）🌊：**
```
清水
  ↓ 滴入一滴墨水
颜色开始扩散
  ↓ 继续扩散
均匀的淡墨水（看不清原来的样子）
```

**反向过程（去噪声）🔄：**
```
均匀的淡墨水
  ↓ AI 猜测："墨水最初在哪里？"
颜色开始聚集
  ↓ 继续猜测
还原的清水（恢复了原样）
```

#### 🎯 核心思想

```
扩散 = 逐渐加入随机噪声，直到看不清原图
反扩散 = 逐渐去除噪声，恢复清晰图像

AI 的任务：学习"如何反向去噪"
```

#### 🔢 逐步图解

```
正向过程（加噪声，越来越乱）：
🖼️ → 🌀 → 🌫️ → ❓ → 🔮
  x₀     x₁     x₂    xₜ

反向过程（去噪声，越来越清晰）：
🔮 → ❓ → 🌫️ → 🌀 → 🖼️
  xₜ    xₜ₋₁   xₜ₋₂   x₀
```

#### 🧠 为什么叫"扩散"？

这个词来自物理学中的**扩散现象**：
- 墨水滴入水中会逐渐扩散
- 烟雾在空气中会扩散
- 噪声在图像中会"扩散"到每个像素

## 第二章：核心概念

### 2.1 动作条件是什么？

#### 📝 条件的概念

普通扩散模型：输入噪声 → 生成图像

**带条件的扩散模型：**
```
输入噪声 + 条件信息 → 生成符合该条件的图像

例子：
- 文字条件：噪声 + "猫" → 生成猫的图片
- 图像条件：噪声 + 首帧 → 生成后续视频
- 动作条件：噪声 + 图像 + 动作 → 预测未来
```

#### 🎮 我们比赛中的条件

```
条件 = 首帧图像 + 动作信号

首帧图像：告诉模型"现在是什么场景"
动作信号：告诉模型"要做什么动作"
```

#### 🔄 为什么需要条件？

没有条件 = 随机生成（生成什么都有可能）

有条件 = 定向生成（按照给定的条件生成）

```
无条件生成：噪声 → 随机画面（可能是猫、狗、人...）
条件生成：噪声 + 动作"伸手" → 伸手后的画面
```

---

### 2.2 动作信号的表示

#### 📊 机器人动作的组成

机器人动作通常用一串数字表示：

```python
动作 = [x, y, z, qx, qy, qz, qw, gripper]
         位置   四元数姿态      夹爪
```

| 维度 | 名称 | 含义 | 范围 |
|------|------|------|------|
| x, y, z | 末端位置 | 机械臂末端的xyz坐标 | -1 ~ 1 (归一化) |
| qx, qy, qz, qw | 四元数 | 末端的旋转姿态 | -1 ~ 1 |
| gripper | 夹爪状态 | 张开程度 | 0(完全关闭) ~ 1(完全张开) |

#### 🔢 四元数是什么？

四元数是表示三维旋转的数学工具，比欧拉角更稳定：

```
不需要知道具体数学原理
只需要理解：
四元数 = 一种表示"朝向"的数学方式

就像：
- 角度表示 = "转 90 度"（可能有万向锁问题）
- 四元数表示 = 更稳定的朝向表示
```

#### 📈 动作序列

动作不是单独的数值，而是一个**时间序列**：

```
时刻 0: [0.1, 0.2, 0.3, 0, 0, 0, 1, 0]  ← 手臂伸出去
时刻 1: [0.2, 0.2, 0.3, 0, 0, 0, 1, 0]  ← 继续伸出
时刻 2: [0.3, 0.2, 0.2, 0, 0, 0, 1, 0]  ← 到达目标
时刻 3: [0.3, 0.2, 0.2, 0, 0, 0, 0, 1]  ← 夹爪闭合
```

### 2.3 什么是隐变量？

#### 📐 直观理解

```
原始图像：512 × 512 × 3 = 786,432 个数值
隐变量：  32 × 32 × 4 = 4,096 个数值

压缩了 192 倍！
```

#### 🎯 为什么需要隐变量？

如果直接在 786,432 维的空间做扩散：
- 计算量巨大
- 显存不够

在 4,096 维的隐空间做扩散：
- 计算快
- 显存友好
- 生成质量依然很好

#### 🔄 编码-解码过程

```
原始图像 (512×512)
     ↓ Encoder 编码
隐变量 (32×32)
     ↓ 扩散过程
去噪后的隐变量
     ↓ Decoder 解码
生成的图像 (512×512)
```

#### 🎨 类比：马赛克艺术

```
原始图像 = 高清照片
编码 = 转换成马赛克（压缩信息）
隐变量 = 马赛克表示
解码 = 从马赛克还原回高清
```

### 3.2 图像编码器 (CLIP)

#### 🤖 CLIP 是什么？

CLIP = **C**ontrastive **L**anguage-**I**mage **P**re-training

**一句话理解：** 一个能理解"图像"和"文字"对应关系的模型

#### 🎯 CLIP 的能力

```
输入：一张猫的照片
     ↓
CLIP 编码
     ↓
输出：一个向量（比如 [0.1, 0.3, ..., 0.8]）

这个向量代表了"猫"这个概念
```

#### 🔍 对比学习

CLIP 通过对比学习训练：
- 猫的图片 → 应该和"猫"这个文字很接近
- 狗的图片 → 应该和"狗"这个文字很接近
- 猫的图片 → 应该和"狗"这个文字很远

#### 💡 为什么用 CLIP？

1. **预训练知识**：CLIP 在 4 亿张图片上训练过
2. **语义理解**：不仅认像素，还理解语义
3. **零样本能力**：没见过的新类别也能处理

#### 📐 在 EVAC 中的作用

```
首帧图像 → CLIP → 语义特征向量
                    ↓
            帮助模型理解"现在是什么场景"
            （桌子上的杯子、机器人手臂的位置等）
```

### 3.3 动作编码器 (MLP)

#### 🧠 MLP 是什么？

MLP = **M**ulti-**L**ayer **P**erceptron（多层感知机）

最简单的神经网络结构：
```
输入 → 线性变换 → 激活 → 线性变换 → 输出
```

#### 📝 动作编码器的结构

```python
动作编码器 = nn.Sequential(
    nn.Linear(7, 128),    # 7维动作 → 128维
    nn.ReLU(),             # 激活
    nn.Linear(128, 256),  # 128 → 256
    nn.ReLU(),             # 激活
    nn.Linear(256, 14),   # 256 → 14维
)
```

#### 🎯 为什么要编码？

原始动作信号：
- 位置单位是米，姿态单位是弧度
- 数值范围差异大

编码后：
- 归一化到相似范围
- 神经网络更容易学习

#### 🔧 可优化点

**问题**：简单的 MLP 可能不够表达复杂的动作模式

**可能的优化**：
- 加入时间维度（LSTM/Transformer）
- 加入位置编码
- 多模态融合

### 3.4 视频解码器 (VAE)

#### 🎨 VAE 是什么？

VAE = **V**ariational **A**uto**E**ncoder（变分自编码器）

一种能够学习"压缩"和"重建"的网络

#### 🔄 编码-解码过程

```
原始图像
     ↓ Encoder
隐变量（压缩）z
     ↓ Decoder
重建图像（接近原始）
```

#### 📊 VAE vs AE

| 类型 | 区别 |
|------|------|
| AE（自编码器） | 压缩后直接重建 |
| VAE（变分自编码器） | 压缩到分布参数，重建更平滑 |

#### 🎯 在 EVAC 中的作用

```
U-Net 输出（去噪后的隐变量）
     ↓ VAE Decoder
最终的视频帧图像（512×512×3）
```

## 第四章：扩散原理

### 4.1 扩散的数学表示

#### 📈 正向过程（加噪声）

```python
# 给定原始图像 x₀，生成任意时刻 t 的图像 xₜ
xₜ = √(ᾱₜ) * x₀ + √(1 - ᾱₜ) * ε

# 其中：
# ᾱₜ = α₁ × α₂ × ... × αₜ （累积噪声率，逐渐减小）
# ε ~ N(0, I) （标准正态随机噪声）
```

**直观理解：**
- 当 t=0 时，ᾱ₀=1，xₜ = x₀（无噪声）
- 当 t=T 时，ᾱₜ≈0，xₜ ≈ ε（纯噪声）

#### 🔄 反向过程（去噪声）

```python
# U-Net 预测噪声
εθ(xₜ, t, c) = 预测的噪声

# 重建原始图像
x̂₀ = (xₜ - √(1-ᾱₜ) * εθ) / √ᾱₜ

# 逐步去噪
xₜ₋₁ = αₜ * x̂₀ + √(1-αₜ) * εθ
```

#### 📉 噪声调度

```python
# α 值的例子（逐渐减小）
α = [0.999, 0.998, 0.997, ..., 0.001]

# 对应的 ᾱ（累积乘积）
ᾱ = [0.999, 0.999×0.998, 0.999×0.998×0.997, ...]
```

### 4.2 DDIM 加速采样

#### ⚡ DDIM 是什么？

DDIM = **D**enoising **D**iffusion **I**mplicit **M**odels

**核心思想**：用更少的步数达到相近的质量

#### 📊 DDPM vs DDIM

| 方法 | 步数 | 速度 | 质量 |
|------|------|------|------|
| DDPM | 1000 步 | 慢 | 最好 |
| DDIM | 10-50 步 | 快 | 几乎一样 |

#### 🎯 DDIM 原理

```python
# DDIM 的关键：ᾱ 调度
# 不是线性减小，而是按照某种策略跳跃

# DDPM：每步都走
# xₜ → xₜ₋₁ → xₜ₋₂ → ... → x₀

# DDIM：跳跃式
# x₁₀₀₀ → x₉₇₃ → x₉₄₆ → ... → x₀
#        跳27步就到终点了
```

#### 🎛️ ddim_steps 参数

```python
# ddim_steps = 27 意味着：
# 从 t=1000 到 t=0，只需要 27 步

# 时间点序列（大约）
timesteps = [1000, 973, 946, ..., 27, 0]
```

**为什么 ddim_steps=10 质量差？**
- 步数太少，每次跨越太大
- 去噪不精细，很多细节丢失

**为什么 ddim_steps=27 质量好？**
- 步数适中，每步变化不大
- 去噪过程精细，保留细节

---

### 4.3 条件融合机制

#### 🔗 融合方式

```python
# 方式1：拼接 (Concat)
condition = concat([image_feature, action_feature])
# 1024 + 14 = 1038 维

# 方式2：交叉注意力 (Cross Attention)
query = image_feature
key = action_feature
output = Attention(query, key, value)
```

#### 📍 条件注入位置

```
在 U-Net 的多个层中注入条件：
- 每层都有 Cross Attention 层
- 让去噪过程始终"知道"动作信息
```

#### 🎯 条件的作用

```
无条件的去噪：只知道"要生成一个物体"
有条件的去噪：知道"要生成伸手后的画面"

条件让生成更有针对性
```

## 第五章：模型组件

### 5.1 U-Net 结构

#### 🏗️ U-Net 架构图

```
        输入 (32×32)
             ↓
    ┌───────────────────────┐
    │   Encoder (收缩路径)    │
    │  32→64→128→256→320   │
    │  分辨率减半，通道翻倍   │
    └───────────────────────┘
             ↓
    ┌───────────────────────┐
    │   Bottleneck           │
    │  320 + Self-Attention  │
    └───────────────────────┘
             ↓
    ┌───────────────────────┐
    │   Decoder (扩张路径)    │
    │  320→256→128→64→32    │
    │  分辨率翻倍，通道减半   │
    └───────────────────────┘
             ↓
        输出 (32×32)
```

#### 🔗 跳跃连接

```
跳跃连接 = Encoder 和 Decoder 同层之间的"快捷通道"

好处：
1. 保留空间细节（低层特征）
2. 加速训练
3. 防止梯度消失
```

#### 🔄 残差连接

```python
# 每个卷积块内部
output = Conv(input) + input  # 残差连接

# 好处：让网络更容易学习恒等映射
```

---

### 5.2 Attention 机制

#### 🧠 Attention 是什么？

**一句话理解**：让每个位置都能"关注"到所有其他位置

#### 🔢 原理

```python
# Q, K, V 都是输入的线性变换
Q = input @ Wq  # Query（查询）
K = input @ Wk  # Key（键）
V = input @ Wv  # Value（值）

# 计算注意力分数
score = softmax(Q @ K.T / √d)

# 加权求和
output = score @ V
```

#### 🎯 直观例子

```
句子："机器人的手在移动"

"手" 这个词应该更多地关注：
- "机器人"（手的拥有者）
- "移动"（手的动作）

而不应该关注：
- "在"（语法词）
```

#### 📐 在 EVAC 中的作用

```
1. 空间 Attention：理解图像不同区域的关系
2. 时间 Attention：理解不同帧之间的关系
3. 条件 Attention：融合图像和动作信息
```

---

### 5.3 时间注意力

#### 🎬 为什么需要时间注意力？

视频不仅有空间信息，还有**时间信息**：

```
帧1: 手的左边位置
帧2: 手的中间位置  
帧3: 手的右边位置

时间注意力帮助理解：
"手是从左到右移动的"
```

#### 🔄 因果注意力的概念

```
因果 = 只能看过去，不能看未来

帧1 只能看 帧1（自己）
帧2 只能看 帧1, 帧2
帧3 只能看 帧1, 帧2, 帧3
...
```

**原因**：预测时无法知道未来的帧

#### 📊 实现方式

```python
# 对多个帧的时间维度做 Attention
# [B, T, C, H, W] = [批次, 时间, 通道, 高, 宽]

temporal_attn = Attention(
    query = frame_features,
    key = frame_features,
    value = frame_features,
    causal=True  # 因果模式
)
```

## 第六章：实践优化

### 6.1 推理流程解析

#### 🔄 完整推理流程

```
1️⃣ 准备输入
   读取首帧图像 + 动作信号
   ↓
2️⃣ 编码
   图像 → CLIP → 1024维
   动作 → MLP → 14维
   ↓
3️⃣ 初始化
   生成纯噪声隐变量 zₜ
   ↓
4️⃣ DDIM 循环 (27步)
   for t in reversed(timesteps):
       条件融合
       U-Net 预测噪声
       DDIM 去噪一步
   ↓
5️⃣ 解码
   隐变量 → VAE → 图像
   ↓
6️⃣ 保存
   保存为 JPEG
```

#### ⏱️ 各部分时间分布

```
总时间 ≈ 27秒/预测

图像编码 (CLIP):     ~1秒  (5%)
动作编码 (MLP):      ~0.5秒 (2%)
DDIM采样 (U-Net):   ~18秒  (70%) ← 主要瓶颈
VAE解码:            ~2秒  (8%)
I/O保存:            ~5秒  (20%)
```

### 6.2 参数作用与调优

#### 🎛️ ddim_steps

| 值 | 速度 | 质量 | 适用场景 |
|----|------|------|----------|
| 10 | 最快 | ❌ 很差 | 不可用 |
| 15 | 快 | ✅ 可接受 | 开发调试 |
| 20 | 中等 | ✅ 较好 | 快速验证 |
| 27 | 慢 | ✅ 最好 | 官方提交 |

**原理**：DDIM 步数越多，去噪越精细

#### 🎛️ n_pred

| 值 | 速度 | 稳定性 | 适用场景 |
|----|------|--------|----------|
| 1 | 最快 | 一般 | 测试 |
| 3 | 3x | 较好 | 官方提交 |
| 5 | 5x | 最好 | 追求稳定 |

**原理**：多次预测可以取最优或平均

### 6.3 优化方向指南

#### 📊 优化分类

| 类别 | 例子 | 难度 | 预期提升 |
|------|------|------|----------|
| 调参 | ddim_steps、n_pred | ⭐ | 小 |
| 后处理 | 质量筛选 | ⭐ | 小 |
| 组件改进 | 动作编码器 | ⭐⭐ | 中 |
| 模型替换 | 更换 CLIP | ⭐⭐⭐ | 大 |

#### 🎯 优化决策

```
1. 我的目标是什么？
   ├── 速度优先 → 减少 ddim_steps
   └── 质量优先 → 继续下面

2. 优化哪个指标？
   ├── PSNR → 图像生成质量
   ├── Scene Consistency → 空间一致性
   └── nDTW → 动作轨迹准确性

3. 愿意投入多少精力？
   ├── 少 → 调参
   ├── 中 → 改组件
   └── 多 → 重新训练
```
