## 1. Gazebo仿真概述

Gazebo是ROS2中最常用的机器人仿真平台，它提供了高保真的物理仿真环境，能够准确模拟各种传感器和执行器的行为。理解Gazebo的插件系统是掌握传感器与执行器仿真的关键。

### 1.1 Gazebo与ROS2的集成

Gazebo通过**插件（Plugin）**机制与ROS2进行交互。插件是一种共享库，可以在Gazebo启动时加载，用于模拟传感器数据生成、物理力计算等功能。

Gazebo-ROS2 Bridge是连接Gazebo和ROS2的核心组件，它负责在Gazebo话题和ROS2话题之间转发数据。通过配置不同的插件，机器人模型可以拥有各种传感器和执行器。

### 1.2 插件类型分类

Gazebo插件主要分为以下几类：

| 插件类型 | 功能 | 典型应用 |
|----------|------|----------|

## 2. 传感器仿真

传感器仿真是机器人仿真的核心组成部分。本节将详细介绍三种常用传感器（激光雷达、相机、IMU）在Gazebo中的仿真方法。

### 2.1 激光雷达仿真

激光雷达（LiDAR）是机器人导航中最重要的传感器之一。在Gazebo中，激光雷达通过`RaySensor`插件进行仿真，该插件模拟激光束的发射和接收过程。

#### 2.1.1 激光雷达插件配置

在URDF/Xacro文件中配置激光雷达传感器：

```xml
<?xml version="1.0"?>
<robot xmlns:xacro="http://www.ros.org/wiki/xacro"
       name="robot_with_lidar">

  <!-- 激光雷达物理属性定义 -->
  <xacro:property name="lidar_radius" value="0.03"/>
  <xacro:property name="lidar_height" value="0.02"/>
  
  <!-- 激光雷达连杆 -->
  <link name="lidar_link">
    <visual>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="${lidar_radius}" length="${lidar_height}"/>
      </geometry>
      <material name="black">
        <color rgba="0.1 0.1 0.1 1.0"/>
      </material>
    </visual>
    <collision>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <cylinder radius="${lidar_radius}" length="${lidar_height}"/>
      </geometry>
    </collision>
    <inertial>
      <mass value="0.1"/>
      <inertia ixx="0.00001" ixy="0" ixz="0" iyy="0.00001" iyz="0" izz="0.00001"/>
    </inertial>
  </link>

  <!-- 激光雷达关节 -->
  <joint name="lidar_joint" type="fixed">
    <parent link="base_link"/>
    <child link="lidar_link"/>
    <origin xyz="0 0 0.15" rpy="0 0 0"/>
  </joint>

  <!-- 激光雷达传感器插件 -->
  <gazebo reference="lidar_link">
    <sensor type="ray" name="rplidar_sensor">
      <!-- 更新频率 -->
      <update_rate>10</update_rate>
      <pose>0 0 0 0 0 0</pose>
      
      <!-- 视觉插件：显示激光束 -->
      <visualize>true</visualize>
      
      <!-- 激光雷达参数 -->
      <ray>
        <!-- 扫描范围设置 -->
        <scan>
          <horizontal>
            <samples>360</samples>
            <resolution>1</resolution>
            <min_angle>-3.14159</min_angle>
            <max_angle>3.14159</max_angle>
          </horizontal>
        </scan>
        
        <!-- 距离范围 -->
        <range>
          <min>0.1</min>
          <max>10.0</max>
          <resolution>0.01</resolution>
        </range>
        
        <!-- 噪声模型 -->
        <noise>
          <type>gaussian</type>
          <mean>0.0</mean>
          <stddev>0.01</stddev>
        </noise>
      </ray>
      
      <!-- ROS2话题配置 -->
      <plugin name="gazebo_ros_laser" filename="libgazebo_ros_ray_sensor.so">
        <ros>
          <namespace>/robot</namespace>
          <remapping>~/out:=scan</remapping>
        </ros>
        <output_type>sensor_msgs/LaserScan</output_type>
      </plugin>
    </sensor>
  </gazebo>

</robot>
```

**关键参数说明**：

| 参数 | 说明 | 常用值 |
|------|------|--------|
| `samples` | 激光束数量 | 360/720 |
| `min_angle/max_angle` | 扫描角度范围 | -π 到 π |
| `min/max` | 测距范围 | 0.1-10m |
| `update_rate` | 更新频率 | 10-40Hz |
| `noise/stddev` | 噪声标准差 | 0.01-0.05 |

#### 2.1.2 激光雷达数学模型

激光雷达的测距原理基于**飞行时间（Time of Flight, ToF）**：

$$d = \frac{c \cdot \Delta t}{2}$$

其中：
- $d$ 为目标距离
- $c$ 为光速（约 $3 \times 10^8$ m/s）
- $\Delta t$ 为发射和接收之间的时间间隔

在仿真中，Gazebo通过光线投射（Ray Casting）计算距离：

$$d_{measured} = d_{true} + \mathcal{N}(0, \sigma^2)$$

其中 $\mathcal{N}(0, \sigma^2)$ 是高斯噪声，$\sigma$ 由噪声参数设置。

### 2.2 相机仿真

 Gazebo支持多种相机仿真，包括普通相机、深度相机和多目相机。相机仿真基于**针孔相机模型**，将3D世界投影到2D图像平面。

#### 2.2.1 普通RGB相机配置

```xml
<!-- RGB相机传感器 -->
<gazebo reference="camera_link">
  <sensor type="camera" name="rgb_camera">
    <update_rate>30</update_rate>
    <camera name="front_camera">
      <!-- 相机内参 -->
      <horizontal_fov>1.3962634</horizontal_fov>  <!-- 80度 -->
      <image>
        <width>640</width>
        <height>480</height>
        <format>R8G8B8</format>
      </image>
      <clip>
        <near>0.02</near>
        <far>300</far>
      </clip>
      <!-- 噪声参数 -->
      <noise>
        <type>gaussian</type>
        <stddev>0.007</stddev>
      </noise>
    </camera>
    
    <plugin name="gazebo_ros_camera" filename="libgazebo_ros_camera.so">
      <ros>
        <namespace>/robot</namespace>
        <remapping>~/image_raw:=camera/image_raw</remapping>
        <remapping>~/camera_info:=camera/camera_info</remapping>
      </ros>
      <camera_name>front_camera</camera_name>
      <frame_name>camera_link</frame_name>
      <hack_baseline>0.07</hack_baseline>
    </plugin>
  </sensor>
</gazebo>
```

#### 2.2.3 相机数学模型

相机成像遵循**针孔相机模型**，其投影关系为：

$$\begin{bmatrix} u \\ v \\ 1 \end{bmatrix} = \frac{1}{Z} \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix} \begin{bmatrix} X \\ Y \\ Z \end{bmatrix}$$

其中：
- $(X, Y, Z)$ 为3D点在相机坐标系下的坐标
- $(u, v)$ 为2D图像坐标
- $f_x, f_y$ 为焦距
- $c_x, c_y$ 为主点坐标

相机的内参矩阵 $K$：

$$K = \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}$$

在仿真中，Gazebo自动完成3D到2D的投影，并添加噪声模拟真实相机的特性。

### 2.3 IMU仿真

IMU（Inertial Measurement Unit，惯性测量单元）是测量物体姿态的重要传感器，通常包含加速度计和陀螺仪。在Gazebo中，IMU插件通过读取机器人刚体运动来计算惯性数据。

#### 2.3.1 IMU传感器配置

```xml
  </inertial>

<joint name="imu_joint" type="fixed">
  <parent link="base_link"/>
  <child link="imu_link"/>
  <origin xyz="0 0 0.1" rpy="0 0 0"/>

<!-- IMU传感器插件 -->
<gazebo reference="imu_link">
  <sensor type="imu" name="imu_sensor">
    <update_rate>100</update_rate>
    <always_on>true</always_on>
    <pose>0 0 0 0 0 0</pose>
    
    <plugin name="gazebo_ros_imu" filename="libgazebo_ros_imu.so">
      <ros>
        <namespace>/robot</namespace>
        <remapping>~/out:=imu</remapping>
```

#### 2.3.3 IMU数学模型

IMU的测量模型可以表示为：

**加速度计**：
$$a_{measured} = a_{true} + b_a + \eta_a$$

其中：
- $a_{measured}$ 为测量值
- $a_{true}$ 为真实值
- $b_a$ 为加速度计偏置（随机游走）
- $\eta_a$ 为高斯白噪声

**陀螺仪**：
$$\omega_{measured} = \omega_{true} + b_g + \eta_g$$

其中：
- $\omega_{measured}$ 为测量角速度
- $\omega_{true}$ 为真实角速度
- $b_g$ 为陀螺仪偏置
- $\eta_g$ 为高斯白噪声

## 3. 执行器仿真

执行器是机器人的动力来源，常见的有电机、舵机、液压缸等。在Gazebo中，执行器仿真通过关节控制器实现。

### 3.1 差速驱动轮机器人

差速驱动是移动机器人最常用的底盘形式，通过控制左右轮速来实现转向。


#### 3.1.2 差速驱动运动学

差速驱动机器人的运动学模型：

$$v = \frac{v_R + v_L}{2}$$

$$\omega = \frac{v_R - v_L}{L}$$

其中：
- $v$ 为线速度
- $\omega$ 为角速度
- $v_R, v_L$ 为左右轮速度
- $L$ 为轮间距

机器人的运动方程：

$$\begin{bmatrix} x \\ y \\ \theta \end{bmatrix} = \begin{bmatrix} \cos\theta & 0 \\ \sin\theta & 0 \\ 0 & 1 \end{bmatrix} \begin{bmatrix} v \\ \omega \end{bmatrix} \Delta t$$

#### 3.1.3 里程计计算

```python
# 差速驱动里程计计算
class DiffDriveOdom:
    def __init__(self, wheel_separation, wheel_radius):
        self.wheel_separation = wheel_separation
        self.wheel_radius = wheel_radius
        self.x = 0.0
        self.y = 0.0
        self.theta = 0.0
        self.left_wheel_pos = 0.0
        self.right_wheel_pos = 0.0
        
    def update(self, left_wheel_vel, right_wheel_vel, dt):
        # 计算左右轮位移
        left_delta = left_wheel_vel * dt * self.wheel_radius
        right_delta = right_wheel_vel * dt * self.wheel_radius
        
        # 更新轮位置
        self.left_wheel_pos += left_delta
        self.right_wheel_pos += right_delta
        
        # 计算里程计
        center_delta = (left_delta + right_delta) / 2.0
        theta_delta = (right_delta - left_delta) / self.wheel_separation
        
        # 更新位姿
        self.theta += theta_delta
        self.x += center_delta * math.cos(self.theta)
        self.y += center_delta * math.sin(self.theta)
        
        return self.x, self.y, self.theta
```

```python
# 关节轨迹控制示例
import rclpy
from rclpy.node import Node
from trajectory_msgs.msg import JointTrajectory, JointTrajectoryPoint

class ArmController(Node):
    def __init__(self):
        super().__init__('arm_controller')
        self.publisher = self.create_publisher(
            JointTrajectory, 
            '/robot/joint_trajectory_controller/joint_trajectory',
            10
        )
        
    def move_to_position(self, positions, duration=2.0):
        msg = JointTrajectory()
        msg.joint_names = [
            'arm_joint_1', 'arm_joint_2', 'arm_joint_3',
            'arm_joint_4', 'arm_joint_5', 'arm_joint_6'
        ]
        
        point = JointTrajectoryPoint()
        point.positions = positions
        point.time_from_start = rclpy.duration.Duration(seconds=duration).to_msg()
        
        msg.points.append(point)
        self.publisher.publish(msg)
        
    def home_position(self):
        self.move_to_position([0.0] * 6)
        
    def fold_position(self):
        self.move_to_position([0.0, -1.57, 1.57, 0.0, 0.785, 0.0])

def main(args=None):
    rclpy.init(args=args)
    controller = ArmController()
    
    # 移动到折叠位置
    controller.fold_position()
    
    rclpy.spin(controller)
    controller.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

```python
# 舵机云台控制示例
import rclpy
from rclpy.node import Node
from std_msgs.msg import Float64

class GimbalController(Node):
    def __init__(self):
        super().__init__('gimbal_controller')
        
        # 创建位置命令发布者
        self.pitch_pub = self.create_publisher(
            Float64, 
            '/robot/gimbal_pitch_controller/command',
            10
        )
        self.yaw_pub = self.create_publisher(
            Float64, 
            '/robot/gimbal_yaw_controller/command',
            10
        )
        
        self.get_logger().info('Gimbal controller initialized')
        
    def set_pitch(self, angle):
        """设置俯仰角（弧度）"""
        msg = Float64()
        msg.data = float(angle)
        self.pitch_pub.publish(msg)
        
    def set_yaw(self, angle):
        """设置偏航角（弧度）"""
        msg = Float64()
        msg.data = float(angle)
        self.yaw_pub.publish(msg)
        
    def look_at(self, pitch, yaw):
        """同时设置俯仰和偏航"""
        self.set_pitch(pitch)
        self.set_yaw(yaw)

def main(args=None):
    rclpy.init(args=args)
    gimbal = GimbalController()
    
    # 示例：让云台向下看45度
    gimbal.set_pitch(-0.785)  # -45度
    
    rclpy.spin(gimbal)
    gimbal.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 选择题

1. Gazebo中激光雷达传感器使用什么类型的插件进行仿真？
   - A) gazebo_ros_camera
   - B) gazebo_ros_laser
   - C) gazebo_ros_imu
   - D) gazebo_ros_diff_drive
   
   **答案：B**。激光雷达使用`gazebo_ros_laser`或`libgazebo_ros_ray_sensor.so`插件进行仿真。

2. 在差速驱动模型中，机器人角速度与左右轮速度的关系是什么？
   - A) $\omega = \frac{v_R + v_L}{2}$
   - B) $\omega = \frac{v_R - v_L}{L}$
   - C) $\omega = \frac{v_R + v_L}{L}$
   - D) $\omega = \frac{v_R \cdot v_L}{L}$
   
   **答案：B**。角速度等于左右轮速度差除以轮间距。

3. IMU传感器的测量模型中包含哪些噪声？
   - A) 只有高斯白噪声
   - B) 只有偏置
   - C) 偏置和高斯白噪声
   - D) 没有噪声
   
   **答案：C**。IMU测量包含真实值、偏置（随机游走）和高斯白噪声。

4. 相机内参矩阵中的焦距参数是什么？
   - $f_x, f_y$
   - $c_x, c_y$
   - $k_1, k_2$
   - $p_1, p_2$
   
   **答案：A**。$f_x, f_y$是相机内参矩阵中的焦距参数。

5. 在ROS2中使用ros2_control控制机械臂时，发布关节轨迹消息的类型是什么？
   - sensor_msgs/JointState
   - trajectory_msgs/JointTrajectory
   - control_msgs/JointController
   - nav_msgs/Odometry
   
   **答案：B**。使用trajectory_msgs/JointTrajectory发布关节轨迹命令。

   ```python
   import rclpy
   from rclpy.node import Node
   from sensor_msgs.msg import LaserScan
   
   class LidarWarning(Node):
       def __init__(self):
           super().__init__('lidar_warning')
           self.subscription = self.create_subscription(
               LaserScan,
               '/robot/scan',
               self.scan_callback,
               10
           )
           self.warning_threshold = 0.5  # 50cm警告阈值
           
       def scan_callback(self, msg):
           valid_ranges = [
               r for r in msg.ranges 
               if msg.range_min < r < msg.range_max
           ]
           
           if valid_ranges:
               min_distance = min(valid_ranges)
               if min_distance < self.warning_threshold:
                   self.get_logger().warn(
                       f'⚠️ COLLISION WARNING: {min_distance:.2f}m'
                   )
               else:
                   self.get_logger().info(
                       f'Safe: closest obstacle {min_distance:.2f}m'
                   )
   
   def main(args=None):
       rclpy.init(args=args)
       node = LidarWarning()
       rclpy.spin(node)
       node.destroy_node()
       rclpy.shutdown()
   
   if __name__ == '__main__':
       main()
   ```
