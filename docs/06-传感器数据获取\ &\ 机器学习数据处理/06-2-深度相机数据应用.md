## 1. 深度相机原理

深度相机（Depth Camera）是一种能够获取场景三维信息的相机系统。与普通RGB相机只记录二维图像不同，深度相机还能测量空间中每个像素点到相机的距离，从而生成包含深度信息的数据。

### 1.1 深度感知技术

目前主流的深度感知技术有三种：**结构光（Structured Light）**、**飞行时间（Time of Flight, ToF）**和**双目立体视觉（Stereo Vision）**。

**结构光技术**通过向场景投射已知的光模式（如光斑或光条纹），然后分析被物体表面调制后的图案变形来计算深度。微软Kinect第一代和Intel RealSense D400系列就采用了这种技术。结构光的优势在于近距离精度高，但受环境光照影响较大，在户外强光下效果不佳。

**飞行时间（ToF）技术**通过测量光脉冲从发射到返回的时间来计算距离。传感器发射调制红外光，然后接收从物体反射回来的光，通过相位差计算距离。微软Kinect V2、RealSense D500系列采用ToF技术。ToF的优势在于响应速度快、测距范围远，但精度受多路径效应和环境反射影响。

**双目立体视觉**使用两个已知位置的摄像头同时捕获图像，通过三角测量法计算视差从而得到深度。这是最接近人眼感知深度的技术。双目视觉的优势是不需要主动光源，功耗低，适合户外使用；但在纹理缺失或光照不足的区域，深度的计算会变得困难。

三种技术对比如下：

| 技术 | 代表产品 | 优点 | 缺点 |
|------|---------|------|------|
| 结构光 | RealSense D400 | 近距离精度高 | 户外效果差 |
| ToF | Kinect V2, Azure Kinect | 响应快、范围远 | 多路径干扰 |
| 双目视觉 | ZED, DUO | 无需主动光源 | 低纹理区域失效 |

### 1.2 深度图像与点云

深度相机输出两种核心数据：**深度图像（Depth Image）**和**点云（Point Cloud）**。

深度图像是一张二维图像，每个像素的值不是颜色，而是该像素对应的空间点到相机平面的距离（单位通常是米）。可以用伪彩色显示深度，蓝色表示近，红色表示远。深度图像的优点是存储紧凑、处理速度快，适合用于目标检测和避障的实时计算。

点云是三维空间中一系列点的集合，每个点包含`(x, y, z)`三维坐标信息，有些还包含颜色（RGB）或反射强度等信息。点云完整保留了三维几何信息，适合用于建图、导航规划、三维重建等需要精确几何信息的场景。

两者的转换关系如下：已知相机的内参（焦距、光心位置）和深度值，可以将深度图像中每个像素投影到三维空间，生成点云：

```
x = (u - cx) * depth / fx
y = (v - cy) * depth / fy
z = depth
```

其中`(u, v)`是像素坐标，`(cx, cy)`是光心位置，`(fx, fy)`是焦距，`depth`是深度值。

### 1.3 常见深度相机产品

在机器人领域，最常用的深度相机是Intel RealSense系列和Azure Kinect DK。

**Intel RealSense D455**是一款双目深度相机，深度范围0.6-6米，深度精度约2%（2米处）。它同时输出深度图像、彩色图像和IMU数据，ROS2有官方支持包`realsense2_camera`，开箱即用。D455的RGB-D方案在室内外场景都有不错的表现，是科研和教育的热门选择。

**Azure Kinect DK**是微软推出的深度相机，采用ToF技术，深度范围0.25-5.5米，深度分辨率可达1MP。它还配备了四麦克风阵列和IMU，支持同步录制多个Kinect的数据。Azure Kinect在人体姿态识别和三维重建领域有广泛应用。

## 2. ROS2中的消息类型

ROS2为深度相机数据提供了标准化的消息类型，主要存放在`sensor_msgs`包中。掌握这些消息类型是处理深度相机数据的基础。

### 2.1 Image消息

`sensor_msgs/Image`是图像数据的基本消息类型，用于传输彩色图像和深度图像：

```yaml
# sensor_msgs/msg/Image.msg
std_msgs/Header header        # 时间戳和坐标系
uint32 height                 # 图像高度（行数）
uint32 width                  # 图像宽度（列数）
string encoding               # 像素编码
uint8 is_bigendian           # 数据字节序
uint32 step                  # 一行数据的字节数
uint8[] data                 # 图像数据数组
```

`encoding`字段决定了如何解析`data`数组。常见的编码格式：

| encoding | 说明 | 通道数 | 每像素字节 |
|----------|------|--------|-----------|
| `rgb8` | RGB彩色 | 3 | 3 |
| `rgba8` | RGBA彩色 | 4 | 4 |
| `mono8` | 8位灰度 | 1 | 1 |
| `mono16` | 16位灰度 | 1 | 2 |
| `16UC1` | 16位深度图 | 1 | 2 |
| `32FC1` | 32位浮点深度图 | 1 | 4 |

对于深度图像，RealSense相机通常输出`16UC1`格式（毫米级整数深度），而Open3D等库处理时通常使用`32FC1`格式（米级浮点数）。

### 2.2 CameraInfo消息

`sensor_msgs/CameraInfo`包含相机的内参和外参信息，是将图像与深度数据进行几何校正的关键：

```yaml
# sensor_msgs/msg/CameraInfo.msg
std_msgs/Header header        # 时间戳和坐标系
uint32 height                 # 图像高度
uint32 width                  # 图像宽度
string distortion_model       # 畸变模型：plumb_bob, equidistant
float64[] D                   # 畸变参数
float64[9] K                  # 相机内参矩阵 [fx, 0, cx, 0, fy, cy, 0, 0, 1]
float64[9] R                  # rectification matrix (stereo)
float64[12] P                 # projection matrix
uint32 binning_x              # 像素合并
uint32 binning_y              # 像素合并
RegionOfInterest roi          # 感兴趣区域
```

其中`K`矩阵是最常用的内参矩阵：
- `K[0] = fx`：X方向的焦距
- `K[2] = cx`：光心X坐标
- `K[4] = fy`：Y方向的焦距
- `K[5] = cy`：光心Y坐标

使用Python的`opencv-python`库，可以通过`K`矩阵和`D`参数矫正图像畸变：

```python
import cv2
import numpy as np

# 从CameraInfo获取内参
K = np.array(camera_info.K).reshape(3, 3)
D = np.array(camera_info.D)

# 矫正深度图像畸变
mapx, mapy = cv2.initUndistortRectifyMap(K, D, None, K, (width, height), cv2.CV_32FC1)
undistorted = cv2.remap(depth_image, mapx, mapy, cv2.INTER_LINEAR)
```

### 2.3 PointCloud2消息

`sensor_msgs/PointCloud2`是传输点云数据的主力消息类型，支持携带每个点的坐标、颜色、法向量等多种属性：

```yaml
# sensor_msgs/msg/PointCloud2.msg
std_msgs/Header header       # 时间戳和坐标系
uint32 height                # 点云高度（非结构化为1）
uint32 width                 # 点云宽度（点数）
PointField[] fields          # 点字段描述
bool is_bigendian            # 数据字节序
uint32 point_step            # 一个点的字节数
uint32 row_step              # 一行的字节数
uint8[] data                 # 点云数据
bool is_dense                # 是否所有数据都有效
```

`fields`数组描述了每个点的数据结构，例如一个包含XYZ和RGB的点云，其fields定义为：

```yaml
# fields示例
- name: x    offset: 0   datatype: 7  # FLOAT32
- name: y    offset: 4   datatype: 7
- name: z    offset: 8   datatype: 7
- name: rgb  offset: 16  datatype: 7
```

常用的`datatype`值：1=INT8、2=UINT8、3=INT16、4=UINT16、5=INT32、6=UINT32、7=FLOAT32、8=FLOAT64。

`rgb`字段通常将三个字节的颜色值打包成一个32位浮点数存储，颜色按BGR顺序排列。提取RGB值的Python方法：

```python
import struct

def read_rgb_from_float(float_val):
    # 从点云Float32类型的rgb字段提取颜色
    packed = struct.unpack('I', struct.pack('f', float_val))[0]
    r = (packed >> 16) & 0xFF
    g = (packed >> 8) & 0xFF
    b = packed & 0xFF
    return r, g, b
```

### 2.4 深度相机关联话题

以Intel RealSense为例，深度相机通常发布以下话题：

| 话题 | 消息类型 | 说明 |
|------|----------|------|
| `/camera/depth/image_rect_raw` | `Image` | 原始深度图像 |
| `/camera/depth/camera_info` | `CameraInfo` | 深度相机内参 |
| `/camera/color/image_raw` | `Image` | 彩色图像 |
| `/camera/color/camera_info` | `CameraInfo` | 彩色相机内参 |
| `/camera/points` | `PointCloud2` | 合成点云（已配准到彩色图） |

当使用已配准（Aligned）的深度图时，深度图像中每个像素都能与彩色图像中对应位置的像素匹配，这在目标检测和语义分割任务中非常有用。

## 3. 跨语言实现

深度相机数据的处理涉及图像滤波、点云转换、坐标变换等操作。下面分别介绍C++（PCL库）和Python（Open3D）两种主流实现方式。

### 3.1 C++ (PCL)

Point Cloud Library（PCL）是C++处理点云的标准库，功能强大但API相对复杂。使用PCL处理RealSense深度相机数据是工业级应用的常见选择。

**订阅深度图像并转换为点云**：

```cpp
// depth_to_cloud.cpp - 深度图像转点云
#include <rclcpp/rclcpp.hpp>
#include <sensor_msgs/msg/image.hpp>
#include <sensor_msgs/msg/camera_info.hpp>
#include <pcl/point_cloud.h>
#include <pcl/point_types.h>
#include <pcl_conversions/pcl_conversions.h>
#include <image_transport/image_transport.hpp>

class DepthToCloudNode : public rclcpp::Node
{
public:
  DepthToCloudNode() : Node("depth_to_cloud_node"), camera_info_received_(false)
  {
    image_transport::ImageTransport it(*this);
    depth_sub_ = it.subscribe("/camera/depth/image_rect_raw", 10,
        &DepthToCloudNode::depthCallback, this);
    info_sub_ = this->create_subscription<sensor_msgs::msg::CameraInfo>(
        "/camera/depth/camera_info", 10,
        &DepthToCloudNode::infoCallback, this);
    cloud_pub_ = this->create_publisher<pcl::PointCloud<pcl::PointXYZ>::SharedPtr>(
        "/output/points", 10);

    RCLCPP_INFO(this->get_logger(), "Depth to cloud node started");
  }

private:
  void infoCallback(const sensor_msgs::msg::CameraInfo::SharedPtr info)
  {
    fx_ = info->k[0]; fy_ = info->k[4];
    cx_ = info->k[2]; cy_ = info->k[5];
    camera_info_received_ = true;
  }

  void depthCallback(const sensor_msgs::msg::Image::SharedPtr msg)
  {
    if (!camera_info_received_) {
      RCLCPP_WARN_ONCE(this->get_logger(), "Waiting for camera info...");
      return;
    }

    auto cloud = std::make_shared<pcl::PointCloud<pcl::PointXYZ>>();
    cloud->header.frame_id = msg->header.frame_id;
    cloud->header.stamp = pcl_conversions::toPCL(msg->header).stamp;
    cloud->width = msg->width;
    cloud->height = msg->height;
    cloud->is_dense = false;

    const uint16_t* depth_data = reinterpret_cast<const uint16_t*>(&msg->data[0]);
    int row_step = msg->step / sizeof(uint16_t);

    for (size_t v = 0; v < msg->height; ++v) {
      for (size_t u = 0; u < msg->width; ++u) {
        uint16_t depth = depth_data[v * row_step + u];
        if (depth == 0) continue;

        pcl::PointXYZ point;
        float z = depth * 0.001f;  // mm -> m
        point.x = (u - cx_) * z / fx_;
        point.y = (v - cy_) * z / fy_;
        point.z = z;
        cloud->push_back(point);
      }
    }

    cloud_pub_->publish(cloud);
  }

  rclcpp::Subscription<sensor_msgs::msg::CameraInfo>::SharedPtr info_sub_;
  rclcpp::Publisher<pcl::PointCloud<pcl::PointXYZ>::SharedPtr>::SharedPtr cloud_pub_;
  image_transport::Subscriber depth_sub_;
  double fx_, fy_, cx_, cy_;
  bool camera_info_received_;
};

int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<DepthToCloudNode>());
  rclcpp::shutdown();
  return 0;
}
```

**PCL点云滤波（直通滤波+降采样）**：

```cpp
// pcl_filter.cpp - 点云滤波示例
#include <pcl/point_cloud.h>
#include <pcl/point_types.h>
#include <pcl/filters/passthrough.h>
#include <pcl/filters/voxel_grid.h>

void filterPointCloud(pcl::PointCloud<pcl::PointXYZ>::Ptr input,
                       pcl::PointCloud<pcl::PointXYZ>::Ptr output)
{
  // 1. 直通滤波：裁剪掉深度范围外的点
  pcl::PassThrough<pcl::PointXYZ> pass;
  pass.setInputCloud(input);
  pass.setFilterFieldName("z");
  pass.setFilterLimits(0.5, 5.0);  // 保留0.5-5米范围内的点
  pass.filter(*output);

  // 2. 体素网格降采样：减少点数
  pcl::VoxelGrid<pcl::PointXYZ> voxel;
  voxel.setInputCloud(output);
  voxel.setLeafSize(0.05f, 0.05f, 0.05f);  // 5cm体素
  voxel.filter(*output);
}
```

### 3.2 Python (Open3D)

Open3D是一个现代化的3D数据处理库，提供了简洁的Python接口，非常适合快速原型开发。

**深度图像转点云**：

```python
# depth_to_cloud.py - 深度图像转点云
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image, CameraInfo, PointCloud2, PointField
import numpy as np
import open3d as o3d

class DepthToCloudNode(Node):
    def __init__(self):
        super().__init__('depth_to_cloud_node')

        self.depth_sub = self.create_subscription(
            Image, '/camera/depth/image_rect_raw',
            self.depth_callback, 10)
        self.info_sub = self.create_subscription(
            CameraInfo, '/camera/depth/camera_info',
            self.info_callback, 10)
        self.publisher = self.create_publisher(PointCloud2, '/output/points', 10)

        self.camera_info = None
        self.get_logger().info('Depth to cloud node started')

    def info_callback(self, msg):
        self.fx = msg.k[0]
        self.fy = msg.k[4]
        self.cx = msg.k[2]
        self.cy = msg.k[5]
        self.frame_id = msg.header.frame_id
        self.camera_info = msg

    def depth_callback(self, msg):
        if self.camera_info is None:
            self.get_logger().warn_once('Waiting for camera info...')
            return

        # 将ROS图像消息转换为numpy数组（深度图单位：毫米）
        depth = np.frombuffer(msg.data, dtype=np.uint16).reshape(
            msg.height, msg.width).astype(np.float32) * 0.001  # 转换为米

        pcd = self.create_point_cloud(depth)
        cloud_msg = self.o3d_to_ros2(pcd, msg.header.stamp)
        self.publisher.publish(cloud_msg)

    def create_point_cloud(self, depth):
        # 将深度图像转换为Open3D点云
        h, w = depth.shape
        y, x = np.meshgrid(np.arange(h), np.arange(w), indexing='ij')

        z = depth
        x_3d = (x - self.cx) * z / self.fx
        y_3d = (y - self.cy) * z / self.fy

        valid = (z > 0) & (z < 10.0)

        points = np.stack([x_3d[valid], y_3d[valid], z[valid]], axis=-1)
        pcd = o3d.geometry.PointCloud()
        pcd.points = o3d.utility.Vector3dVector(points)
        return pcd

    def o3d_to_ros2(self, pcd, stamp):
        # Open3D点云转换为ROS2 PointCloud2
        points = np.asarray(pcd.points)

        ros_cloud = PointCloud2()
        ros_cloud.header.stamp = stamp
        ros_cloud.header.frame_id = self.frame_id
        ros_cloud.height = 1
        ros_cloud.width = points.shape[0]
        ros_cloud.fields = [
            PointField(name='x', offset=0, datatype=PointField.FLOAT32, count=1),
            PointField(name='y', offset=4, datatype=PointField.FLOAT32, count=1),
            PointField(name='z', offset=8, datatype=PointField.FLOAT32, count=1),
        ]
        ros_cloud.is_bigendian = False
        ros_cloud.point_step = 12
        ros_cloud.row_step = ros_cloud.point_step * points.shape[0]
        ros_cloud.data = points.astype(np.float32).tobytes()
        ros_cloud.is_dense = True
        return ros_cloud

def main(args=None):
    rclpy.init(args=args)
    node = DepthToCloudNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

**Open3D点云滤波**：

```python
# pcl_filter.py - Open3D点云滤波
import numpy as np
import open3d as o3d

def filter_point_cloud(pcd, z_min=0.5, z_max=5.0, voxel_size=0.05):
    # 点云滤波：直通滤波 + 体素降采样

    # 1. 直通滤波
    points = np.asarray(pcd.points)
    valid_indices = (points[:, 2] > z_min) & (points[:, 2] < z_max)
    pcd.points = o3d.utility.Vector3dVector(points[valid_indices])

    # 2. 降采样
    pcd_down = pcd.voxel_down_sample(voxel_size)
    return pcd_down

def compute_normals(pcd, radius=0.1):
    # 估计点云法向量
    pcd.estimate_normals(
        search_param=o3d.geometry.KDTreeSearchParamHybrid(radius=radius, max_nn=30)
    )
    pcd.orient_normals_towards_camera_location(camera_location=np.array([0., 0., 0.]))
    return pcd
```

## 4. 实战：深度相机的实时障碍物检测

本节我们将实现一个**实时障碍物检测系统**，利用深度相机数据进行障碍物识别和距离测量。该系统包括深度图像预处理和障碍物区域提取两个节点。

### 4.2 深度预处理节点

```python
# obstacle_detection/obstacle_detection/depth_preprocess.py
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image, CameraInfo
import cv2
import numpy as np

class DepthPreprocessNode(Node):
    # 深度图像预处理：滤波、缩放、单位转换

    def __init__(self):
        super().__init__('depth_preprocess_node')

        self.declare_parameter('depth_min', 0.1)
        self.declare_parameter('depth_max', 10.0)

        self.depth_min = self.get_parameter('depth_min').value
        self.depth_max = self.get_parameter('depth_max').value

        self.info_sub = self.create_subscription(
            CameraInfo, '/camera/depth/camera_info',
            self.info_callback, 10)

        self.depth_sub = self.create_subscription(
            Image, '/camera/depth/image_rect_raw',
            self.depth_callback, 10)

        self.depth_pub = self.create_publisher(
            Image, '/preprocessed/depth', 10)

        self.debug_pub = self.create_publisher(
            Image, '/debug/depth_normalized', 10)

        self.get_logger().info('Depth preprocess node started')

    def info_callback(self, msg):
        self.fx = msg.k[0]
        self.fy = msg.k[4]
        self.cx = msg.k[2]
        self.cy = msg.k[5]
        self.frame_id = msg.header.frame_id

    def depth_callback(self, msg):
        # 原始深度图：uint16 (毫米)
        depth_mm = np.frombuffer(msg.data, dtype=np.uint16).reshape(
            msg.height, msg.width)
        depth_m = depth_mm.astype(np.float32) * 0.001  # 转米

        # 应用深度范围掩码
        mask = (depth_m > self.depth_min) & (depth_m < self.depth_max)
        depth_filtered = np.where(mask, depth_m, 0)

        # 中值滤波去噪
        depth_filtered = cv2.medianBlur(depth_filtered.astype(np.float32), 5)

        # 发布预处理后的深度图 (32FC1)
        out_msg = self.numpy_to_ros_image(depth_filtered, msg.header)
        self.depth_pub.publish(out_msg)

        # 发布调试用伪彩色图
        depth_debug = self.depth_to_colormap(depth_filtered)
        debug_msg = self.numpy_to_ros_image(depth_debug, msg.header, encoding='bgr8')
        self.debug_pub.publish(debug_msg)

    def depth_to_colormap(self, depth):
        # 深度图转伪彩色用于可视化
        depth_vis = np.clip(depth, 0, 5.0) / 5.0 * 255
        depth_vis = depth_vis.astype(np.uint8)
        return cv2.applyColorMap(depth_vis, cv2.COLORMAP_JET)

    def numpy_to_ros_image(self, data, header, encoding='32FC1'):
        ros_img = Image()
        ros_img.header = header
        ros_img.height = data.shape[0]
        ros_img.width = data.shape[1]
        ros_img.encoding = encoding
        ros_img.is_bigendian = False
        ros_img.step = data.shape[1] * data.dtype.itemsize
        ros_img.data = data.tobytes()
        return ros_img

def main(args=None):
    rclpy.init(args=args)
    node = DepthPreprocessNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 4.3 障碍物检测节点

```python
# obstacle_detection/obstacle_detection/obstacle_detector.py
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image
from std_msgs.msg import Float32MultiArray
import numpy as np

class ObstacleDetectorNode(Node):
    # 基于深度图像的障碍物检测

    def __init__(self):
        super().__init__('obstacle_detector_node')

        self.declare_parameter('grid_width', 8)
        self.declare_parameter('grid_height', 6)
        self.declare_parameter('danger_threshold', 0.5)  # 米

        self.grid_w = self.get_parameter('grid_width').value
        self.grid_h = self.get_parameter('grid_height').value
        self.danger_threshold = self.get_parameter('danger_threshold').value

        self.depth_sub = self.create_subscription(
            Image, '/preprocessed/depth',
            self.detect_obstacles, 10)

        self.obstacle_pub = self.create_publisher(
            Float32MultiArray, '/obstacle/grid_distances', 10)

        self.get_logger().info('Obstacle detector started')

    def detect_obstacles(self, msg):
        depth = np.frombuffer(msg.data, dtype=np.float32).reshape(
            msg.height, msg.width)

        h, w = depth.shape
        cell_h = h // self.grid_h
        cell_w = w // self.grid_w

        distances = []
        danger_flags = []

        for i in range(self.grid_h):
            for j in range(self.grid_w):
                y_start, y_end = i * cell_h, (i + 1) * cell_h
                x_start, x_end = j * cell_w, (j + 1) * cell_w
                region = depth[y_start:y_end, x_start:x_end]

                valid_depths = region[region > 0]
                if len(valid_depths) > 0:
                    avg_dist = float(np.mean(valid_depths))
                else:
                    avg_dist = 0.0

                distances.append(avg_dist)
                danger_flags.append(avg_dist < self.danger_threshold and avg_dist > 0)

        grid_msg = Float32MultiArray()
        grid_msg.data = distances
        self.obstacle_pub.publish(grid_msg)

        if any(danger_flags):
            danger_count = sum(danger_flags)
            self.get_logger().warn(f'Warning: {danger_count} grid cells with obstacles!')

def main(args=None):
    rclpy.init(args=args)
    node = ObstacleDetectorNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

1. 深度相机的ToF（飞行时间）技术是通过什么方式测量深度的？
   - A) 测量光脉冲从发射到返回的时间
   - B) 分析投射光图案的变形
   - C) 计算两个摄像头视差
   - D) 测量物体反射光的强度

   **答案：A**。ToF技术通过测量调制红外光脉冲从发射到返回的时间来计算距离。

2. RealSense深度相机输出的原始深度图像编码格式是什么？
   - A) `32FC1`
   - B) `rgb8`
   - C) `16UC1`
   - D) `mono8`

   **答案：C**。RealSense相机通常输出`16UC1`格式的深度图，单位为毫米。

3. 将深度图像像素坐标转换为3D点云时，需要用到什么信息？
   - A) 只需要深度值
   - B) 相机内参（焦距、光心）
   - C) 相机外参
   - D) 不需要任何参数

   **答案：B**。将深度图像转换为点云需要相机内参（K矩阵），包括焦距和光心位置。

4. PCL中直通滤波的作用是什么？
   - A) 将点云压缩
   - B) 裁剪掉指定范围内的点
   - C) 增加点云密度
   - D) 平滑点云表面

   **答案：B**。直通滤波（PassThrough）根据某一维度（如z轴）的范围裁剪点云，保留指定范围内的点。
