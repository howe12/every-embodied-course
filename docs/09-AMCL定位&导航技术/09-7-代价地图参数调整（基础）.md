## 1. 代价地图概述

代价地图（Costmap）是Nav2导航系统中用于环境表示的核心组件，它以栅格形式存储环境的障碍物信息，为路径规划器提供安全的通行区域。代价地图将现实世界的空间划分为一个个小格子，每个格子分配一个代价值，表示机器人在该位置通行的危险程度。

### 1.1 什么是代价地图

代价地图本质上是一个二维栅格地图，每个单元格（cell）存储一个无符号字节值（0-255），表示该位置的代价：

| 代价值 | 含义 | 颜色表示（rviz2） |
|--------|------|-------------------|
| 0 | 未知区域（Unknown） | 灰色 |
| 1-127 | 自由区域（Free） | 黑色到深灰色渐变 |
| 128 | 机器人足印（Inscribed） | 浅灰色 |
| 129-252 | 膨胀区域（Inflation） | 浅灰色到红色渐变 |
| 253 | 障碍物足印（Lethal） | 红色 |
| 254 | 膨胀障碍物（Beyond Inscribed） | 红色 |
| 255 | 被占区域（Occupied） | 红色 |

代价地图的设计理念是：为路径规划器提供足够的安全缓冲区，使机器人不会与障碍物发生碰撞。通过将障碍物进行膨胀处理，路径规划器会自动避开障碍物周围的区域。

### 1.2 代价地图的作用

代价地图在导航系统中承担着关键作用：

1. **环境建模**：将传感器数据（激光雷达、深度相机、超声波等）转换为统一的栅格表示
2. **安全屏障**：通过膨胀层为机器人提供安全缓冲区域
3. **路径约束**：为全局和局部规划器提供可通行区域的约束
4. **动态更新**：实时反映环境变化，支持动态障碍物的处理

### 1.3 与ROS1的区别

ROS2的Nav2框架对代价地图进行了重大改进：

| 特性 | ROS1 (navigation stack) | ROS2 (Nav2) |
|------|-------------------------|-------------|
| 架构 | 单层代价地图 | 多层代价地图（插件化） |
| 实时性 | 依赖tf静态变换 | 更好的实时性支持 |
| 膨胀算法 | 基于距离的简单膨胀 | 支持多种膨胀算法 |
| 开发者API | C++ API | C++/Python API，支持插件化 |
| 性能 | 中等 | 优化的高性能实现 |

Nav2的代价地图采用插件化设计，允许开发者自定义代价地图层，这大大提高了系统的灵活性和可扩展性。

## 2. 分层结构

Nav2的代价地图采用分层（Layered）架构，每一层负责特定类型的环境信息。这种设计使得不同来源的环境数据可以独立处理，最后合并为统一的代价地图。

### 2.1 分层架构原理

代价地图的分层结构就像 Photoshop 的图层一样，每一层保存特定类型的信息，最终叠加形成完整的代价地图：

**优先级规则**：代价地图层按优先级顺序叠加，高优先级层的代价值会覆盖低优先级层的值。通常：
- 静态地图层（Static Layer）：优先级最低，通常为1
- 障碍物层（Obstacle Layer）：优先级中等，通常为2
- 膨胀层（Inflation Layer）：优先级最高，通常为3

### 2.2 Static Layer（静态地图层）

静态地图层用于存储环境的静态表示，通常来自SLAM建图阶段生成的地图文件（.pgm/.yaml）。这一层在导航过程中通常不会发生变化，除非重新定位或更新地图。

**主要功能**：
- 加载并解析SLAM生成的地图文件
- 将地图坐标转换为代价地图坐标
- 提供静态环境的基础层

**配置参数**：

```yaml
global_costmap:
  global_costmap:
    ros__parameters:
      # 静态地图层插件
      plugins: ["static_layer"]
      
      # 地图订阅配置
      map_subscribe_transient_local: True
      
  static_layer:
    plugin: "nav2_costmap_2d::StaticLayer"
    ros__parameters:
      # 地图文件路径
      map_yaml_file: "map.yaml"
      
      # 是否订阅地图主题
      subscribe_to_map: True
      
      # 是否使用地图的原始尺寸
      use_maximum: False
      
      # 未知空间是否视为自由空间
      unknown_cost_value: -1
      
      # 是否将未知空间视为可通行（255）
      treat_unknown_as_free: False
```

**关键参数说明**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `map_yaml_file` | string | "" | 地图文件路径 |
| `subscribe_to_map` | bool | True | 是否订阅地图话题 |
| `use_maximum` | bool | False | 使用地图的最大尺寸 |
| `treat_unknown_as_free` | bool | False | 未知区域是否视为自由 |

### 2.3 Obstacle Layer（障碍物层）

障碍物层负责实时检测和更新环境中的动态障碍物。它订阅传感器话题（激光雷达、深度相机等），将传感器数据转换为障碍物信息。

**主要功能**：
- 订阅传感器数据（LaserScan、PointCloud2等）
- 实时检测障碍物
- 标记占用单元格的代价

**配置参数**：

```yaml
local_costmap:
  local_costmap:
    ros__parameters:
      plugins: ["obstacle_layer"]
      
  obstacle_layer:
    plugin: "nav2_costmap_2d::ObstacleLayer"
    ros__parameters:
      # 激光雷达障碍物层配置
      observation_sources: scan
      
      scan:
        sensor: sensor_msgs/LaserScan
        topic: /scan
        data_type: LaserScan
        marking: True
        clearing: True
        # 观察范围（米）
        observation_persistance: 0.0
        max_obstacle_height: 2.0
        min_obstacle_height: 0.0
        obstacle_range: 3.0
        raytrace_range: 3.0
        
      # 障碍物层全局配置
      enabled: True
      footprint_clearing_enabled: True
      max_obstacle_keepin_time: 10.0
      max_obstacle_keepout_time: 10.0
```

**关键参数说明**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `observation_sources` | string | "" | 传感器源列表（空格分隔） |
| `marking` | bool | True | 是否标记障碍物 |
| `clearing` | bool | True | 是否清除障碍物 |
| `obstacle_range` | double | 3.0 | 最大障碍物检测距离（米） |
| `raytrace_range` | double | 3.0 | 光线投射清除范围（米） |
| `max_obstacle_height` | double | 2.0 | 障碍物最大高度 |
| `min_obstacle_height` | double | 0.0 | 障碍物最小高度 |
| `footprint_clearing_enabled` | bool | True | 是否清除机器人footprint区域 |

### 2.4 Inflation Layer（膨胀层）

膨胀层是代价地图中最重要的安全层，它将障碍物进行"膨胀"处理，在障碍物周围创建缓冲区。这一层直接决定了机器人的避障安全距离。

**主要功能**：
- 根据障碍物生成膨胀区域
- 计算每个单元格到最近障碍物的距离
- 将距离转换为代价值

**膨胀算法**：

膨胀层的核心是将每个障碍物单元格按照一定规则向周围扩展。代价值的计算公式为：

$$cost = \begin{cases} 0 & \text{if } d > d_{inflation} \\ c_{max} \cdot \left(1 - \frac{d}{d_{inflation}}\right) & \text{if } 0 < d \leq d_{inflation} \\ c_{inscribed} & \text{if } d = 0 \end{cases}$$

其中：
- $d$ ：单元格到最近障碍物的距离
- $d_{inflation}$ ：膨胀半径（inflation_radius）
- $c_{max}$ ：最大代价（通常为254）
- $c_{inscribed}$ ：机器人足印代价（128）

**配置参数**：

```yaml
global_costmap:
  global_costmap:
    ros__parameters:
      plugins: ["static_layer", "obstacle_layer", "inflation_layer"]
      
  inflation_layer:
    plugin: "nav2_costmap_2d::InflationLayer"
    ros__parameters:
      # 膨胀半径（米）
      inflation_radius: 0.55
      
      # 代价衰减权重
      cost_scaling_factor: 10.0
      
      # 膨胀是否启用
      enabled: True
      
      # 机器人footprint是否 inflation
      inflate_footprint: True
      
      # 是否将未知空间视为自由空间
      treat_unknown_as_free: False
```

## 3. 参数配置详解

代价地图的正确配置对于导航系统的性能至关重要。本节将详细介绍各类参数的作用和调整方法。

### 3.1 全局与局部代价地图

Nav2中通常配置两个代价地图：

1. **全局代价地图（Global Costmap）**：用于全局路径规划，覆盖整个导航环境
2. **局部代价地图（Local Costmap）**：用于局部路径规划，只覆盖机器人周围区域

**两者的区别**：

| 特性 | 全局代价地图 | 局部代价地图 |
|------|-------------|-------------|
| 覆盖范围 | 整个地图 | 机器人周围局部区域 |
| 尺寸 | 固定（等于地图尺寸） | 可配置（如5m×5m） |
| 更新频率 | 较低 | 较高 |
| 用途 | 全局路径规划 | 局部避障 |
| 层组成 | 静态层+障碍物层+膨胀层 | 障碍物层+膨胀层 |

**完整配置示例**：

```yaml
# 全局代价地图配置
global_costmap:
  global_costmap:
    ros__parameters:
      # 全局坐标系
      global_frame: map
      # 机器人基坐标系
      robot_base_frame: base_link
      # 更新频率
      update_frequency: 1.0
      # 发布频率
      publish_frequency: 1.0
      # 代价地图插件列表
      plugins: ["static_layer", "obstacle_layer", "inflation_layer"]
      
  static_layer:
    plugin: "nav2_costmap_2d::StaticLayer"
    ros__parameters:
      map_yaml_file: "map.yaml"
      subscribe_to_map: True
      
  obstacle_layer:
    plugin: "nav2_costmap_2d::ObstacleLayer"
    ros__parameters:
      observation_sources: scan
      scan:
        sensor: sensor_msgs/LaserScan
        topic: /scan
        marking: True
        clearing: True
        obstacle_range: 3.0
        raytrace_range: 3.0
        
  inflation_layer:
    plugin: "nav2_costmap_2d::InflationLayer"
    ros__parameters:
      enabled: True
      inflation_radius: 0.55
      cost_scaling_factor: 10.0

# 局部代价地图配置
local_costmap:
  local_costmap:
    ros__parameters:
      global_frame: odom
      robot_base_frame: base_link
      update_frequency: 5.0
      publish_frequency: 2.0
      plugins: ["obstacle_layer", "inflation_layer"]
      # 局部地图尺寸（米）
      width: 5
      height: 5
      # 分辨率（米/单元格）
      resolution: 0.05
      static_map: False
      
  obstacle_layer:
    plugin: "nav2_costmap_2d::ObstacleLayer"
    ros__parameters:
      observation_sources: scan
      scan:
        sensor: sensor_msgs/LaserScan
        topic: /scan
        marking: True
        clearing: True
        obstacle_range: 2.5
        raytrace_range: 2.5
        
  inflation_layer:
    plugin: "nav2_costmap_2d::InflationLayer"
    ros__parameters:
      enabled: True
      inflation_radius: 0.25
      cost_scaling_factor: 10.0
```

### 3.2 核心参数说明

以下是代价地图最常用的核心参数及其调整建议：

| 参数 | 作用域 | 推荐值 | 说明 |
|------|--------|--------|------|
| `inflation_radius` | 膨胀层 | 0.3~1.0m | 障碍物膨胀半径 |
| `cost_scaling_factor` | 膨胀层 | 5~15 | 代价衰减因子，越大衰减越快 |
| `resolution` | 全局/局部 | 0.05m | 栅格分辨率 |
| `width/height` | 局部 | 3~10m | 局部地图尺寸 |
| `update_frequency` | 全局/局部 | 1~10Hz | 更新频率 |
| `obstacle_range` | 障碍物层 | 2~5m | 障碍物检测最大距离 |
| `raytrace_range` | 障碍物层 | 2~5m | 光线投射清除范围 |

### 3.3 参数调整指南

**膨胀半径（inflation_radius）调整**：

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 狭窄环境 | 0.3~0.4m | 减小膨胀区域，增加通行空间 |
| 宽敞环境 | 0.5~0.7m | 适度膨胀，保证安全 |
| 复杂环境 | 0.7~1.0m | 增大膨胀区域，预留更多安全距离 |

## 4. 膨胀半径

膨胀半径是代价地图中最重要的参数之一，它决定了障碍物周围"危险区域"的大小。

### 4.1 膨胀半径的概念

膨胀半径（inflation_radius）定义了从障碍物边缘开始到代价降为0的最大距离。在这个范围内，代价值从障碍物边缘的LETHAL（253）逐渐衰减到边缘的0。

### 4.2 如何确定膨胀半径

膨胀半径应该至少等于**机器人最大尺寸的一半**，以确保机器人的任何部分都不会与障碍物发生碰撞。

**计算公式**：

$$r_{inflation} \geq \frac{D_{robot}}{2} + d_{safety}$$

其中：
- $r_{inflation}$：膨胀半径
- $D_{robot}$：机器人最大直径（考虑旋转）
- $d_{safety}$：额外安全距离（建议0.1~0.2m）

**实例计算**：

假设机器人尺寸为0.4m × 0.3m（长×宽），考虑旋转时的最大对角线：

$$D_{max} = \sqrt{0.4^2 + 0.3^2} = 0.5m$$

最小膨胀半径：

$$r_{inflation} = \frac{0.5}{2} + 0.15 = 0.4m$$

考虑到地图分辨率和传感器误差，推荐使用更大的值：

$$r_{inflation} = 0.5 \sim 0.6m$$

### 4.3 rviz2中可视化膨胀效果

在调试代价地图时，可以使用rviz2直观查看膨胀效果：

```bash

```

1. **颜色主题**：使用"costmap"主题可以看到代价值的颜色编码

## 5. Footprint

Footprint（足印）是机器人在地图上的占用区域，用于碰撞检测和障碍物清除。

### 5.1 Footprint的概念

Footprint是机器人底部的投影形状，通常用一个多边形表示。在代价地图中，footprint用于：

1. **碰撞检测**：检查机器人是否会与障碍物碰撞
2. **障碍物清除**：将footprint覆盖的区域标记为自由
3. **代价计算**：确定机器人在当前位置的代价

### 5.2 Footprint配置

**参数形式**：

Footprint可以用两种形式配置：

1. **圆形**：指定半径
2. **多边形**：指定顶点列表

```yaml
# 圆形 footprint 配置
robot_radius: 0.3  # 米

# 多边形 footprint 配置
footprint: [[-0.2, -0.15], [-0.2, 0.15], [0.2, 0.15], [0.2, -0.15]]
# 解释：[x, y] 表示顶点坐标（相对于机器人中心）
# 这是一个 0.4m × 0.3m 的矩形
```

**完整配置示例**：

```yaml
# 在 robot_description 中配置
robot_state_publisher:
  ros__parameters:
    robot_description:
      robot:
        footprint:
          # 矩形机器人：0.4m × 0.3m
          default: [[-0.2, -0.15], [-0.2, 0.15], [0.2, 0.15], [0.2, -0.15]]
          
# 在代价地图中启用 footprint 清除
obstacle_layer:
  ros__parameters:
    footprint_clearing_enabled: True
```

### 5.3 Footprint与膨胀层

Footprint与膨胀层的关系非常密切。关键原则是：

$$d_{inflation} > d_{footprint}$$

即膨胀半径应该大于机器人footprint的对角线长度的一半，否则机器人的footprint会与膨胀后的障碍物重叠，导致无法找到有效路径。

### 5.4 Footprint动态配置

Nav2支持运行时动态配置footprint：

```bash
# 查看当前 footprint
ros2 topic echo /robot_radius

# 动态调整膨胀半径（运行时）
ros2 param set /global_costmap inflation_layer.inflation_radius 0.6

# 启用/禁用层
ros2 param set /global_costmap inflation_layer.enabled False
```

## 6. 实战：代价地图参数调优

本节将通过一个实际案例，展示如何根据机器人特点和环境要求调整代价地图参数。

### 6.1 场景描述

**机器人参数**：
- 尺寸：0.45m × 0.35m（长 × 宽）
- 最大速度：0.5m/s
- 传感器：激光雷达（范围3m）

**环境要求**：
- 办公室环境，有狭窄通道
- 需要躲避动态障碍物（人、桌椅）
- 导航精度要求较高

### 6.2 参数计算

**Step 1：计算footprint对角线**

$$D_{diagonal} = \sqrt{0.45^2 + 0.35^2} = 0.57m$$

**Step 2：确定最小膨胀半径**

$$r_{inflation_{min}} = \frac{0.57}{2} + 0.1 = 0.38m$$

考虑到狭窄环境和安全余量，选择：

$$r_{inflation} = 0.5m$$

**Step 3：配置参数**

```yaml
# 自定义参数文件 custom_costmap_params.yaml

global_costmap:
  global_costmap:
    ros__parameters:
      plugins: ["static_layer", "obstacle_layer", "inflation_layer"]
      
  static_layer:
    plugin: "nav2_costmap_2d::StaticLayer"
    ros__parameters:
      map_yaml_file: "warehouse_map.yaml"
      
  obstacle_layer:
    plugin: "nav2_costmap_2d::ObstacleLayer"
    ros__parameters:
      observation_sources: scan
      scan:
        topic: /scan
        marking: True
        clearing: True
        obstacle_range: 3.0
        raytrace_range: 3.0
        
  inflation_layer:
    ros__parameters:
      enabled: True
      inflation_radius: 0.5
      cost_scaling_factor: 8.0

local_costmap:
  local_costmap:
    ros__parameters:
      plugins: ["obstacle_layer", "inflation_layer"]
      width: 4.0
      height: 4.0
      
  obstacle_layer:
    ros__parameters:
      observation_sources: scan
      scan:
        topic: /scan
        obstacle_range: 2.5
        raytrace_range: 2.5
        
  inflation_layer:
    ros__parameters:
      enabled: True
      inflation_radius: 0.35
      cost_scaling_factor: 10.0

robot_radius: 0.3
```

## 本章小结

本章我们全面学习了ROS2导航系统中代价地图的参数配置。代价地图是Nav2导航系统的核心组件，它通过分层架构将静态地图、动态障碍物和膨胀区域有机结合，为路径规划器提供安全的环境表示。

在分层结构部分，我们详细讲解了静态地图层（Static Layer）、障碍物层（Obstacle Layer）和膨胀层（Inflation Layer）的工作原理和配置方法。在参数配置部分，我们介绍了全局和局部代价地图的区别，以及各个核心参数的作用和调整建议。

膨胀半径（inflation_radius）是代价地图中最重要的参数，它决定了障碍物周围的"安全缓冲区"大小。确定膨胀半径的原則是：膨胀半径必须大于机器人footprint对角线的一半加上安全距离。

Footprint是机器人在地图上的占用形状，用于碰撞检测和障碍物清除。Nav2支持圆形和多边形两种footprint形式，可以根据机器人实际形状进行配置。

掌握代价地图的参数调整，对于实现安全、可靠的机器人导航至关重要。建议在实际项目中根据机器人尺寸、传感器能力和环境特点进行针对性调整。
