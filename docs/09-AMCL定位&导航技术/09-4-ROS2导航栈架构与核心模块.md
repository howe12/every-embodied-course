## 1. Nav2概述

Nav2是ROS2生态系统中的核心导航框架，专门为移动机器人提供自主导航能力。它是一个功能完整、可扩展性强的导航解决方案，支持从简单的点对点导航到复杂的多目标任务规划。

### 1.1 从ROS1到ROS2的演进

ROS1的Navigation Stack是机器人导航领域的经典框架，但在ROS2时代，它面临着一些挑战。ROS2引入了许多新特性，如分布式发现、更好的实时性能、跨平台支持等，Nav2正是为充分利用这些新特性而设计的。

| 特性 | ROS1 Navigation Stack | ROS2 Nav2 |
|------|----------------------|-----------|
| 通信机制 | ROS1话题/服务 | ROS2动作/DDS |
| 配置文件 | YAML + launch | YAML + lifecycle |
| 行为控制 | 状态机 | 行为树（BT） |
| 可扩展性 | 有限 | 插件化架构 |
| 多机器人支持 | 复杂 | 原生支持 |

Nav2的核心设计理念包括：**模块化**（每个功能都是可替换的插件）、**可组合性**（通过行为树灵活组合行为）、**生命周期管理**（节点状态可控）以及**标准化接口**（统一的插件接口）。

### 1.2 Nav2主要功能

Nav2提供了完整的机器人导航功能集：

- **定位**：基于AMCL的自适应蒙特卡洛定位，支持多种地图类型
- **全局规划**：基于搜索（A*、Dijkstra）和采样（PRM、RRT）的路径规划
- **局部规划**：基于时间最优、障碍物规避的局部轨迹生成
- **运动控制**：PID控制器实现精确的机器人速度命令
- **情景恢复**：多种恢复策略处理卡死、迷失等情况
- **动态避障**：实时检测并规避动态障碍物
- **多目标导航**：支持巡航模式和多目标点顺序导航

### 1.3 Nav2软件栈架构

Nav2的软件栈采用分层设计，每一层都有明确的职责：

```
┌─────────────────────────────────────────────────────────────┐
│                    应用层（Application）                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │
│  │  Nav2 Stack │  │   BT Server │  │  外部控制系统        │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│                    行为树层（BT Layer）                       │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────────────┐   │
│  │ Compute │ │  Pass   │ │ Drive   │ │   Recovery      │   │
│  │ Path    │ │ Through │ │ To Pose │ │   Behaviors     │   │
│  └─────────┘ └─────────┘ └─────────┘ └─────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                    核心服务器层（Core Servers）               │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────┐   │
│  │  Planner    │ │ Controller  │ │   Recovery Server   │   │
│  │  Server     │ │ Server      │ │                     │   │
│  └─────────────┘ └─────────────┘ └─────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                      工具层（Utils）                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────────────┐   │
│  │  AMCL   │ │  Map    │ │ Costmap │ │    Lifecycle    │   │
│  │  Server │ │  Server │ │  2D/3D  │ │    Manager      │   │
│  └─────────┘ └─────────┘ └─────────┘ └─────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

这种分层架构的优势在于每一层都可以独立开发、测试和替换，使得系统更加灵活和可维护。

## 2. 核心架构组件

Nav2的核心功能由多个服务器（Server）提供，每个服务器负责特定的功能域。这种设计使得系统具有良好的模块化和可扩展性。

### 2.1 规划器服务器（Planner Server）

规划器服务器负责全局路径规划，它是导航系统的大脑，决定机器人从当前位置到目标位置的最佳路径。

**全局规划器接口**：

```cpp
// nav2_core/include/nav2_core/planner.hpp
namespace nav2_core
{
  class Planner
  {
  public:
    virtual void configure(
      rclcpp_lifecycle::LifecycleNode::SharedPtr parent,
      std::string name,
      std::shared_ptr<tf2_ros::Buffer> tf,
      std::shared_ptr<nav2_costmap_2d::Costmap2DROS> costmap_ros) = 0;

    virtual nav_msgs::msg::Path createPlan(
      const geometry_msgs::msg::PoseStamped & start,
      const geometry_msgs::msg::PoseStamped & goal) = 0;

    virtual ~Planner() = default;
  };
}
```

全局规划器接收起点和终点pose，计算出代价最低的路径。路径规划的质量直接影响导航的效率和安全性。

**常用全局规划器**：

| 规划器 | 特点 | 适用场景 |
|--------|------|----------|
| `NavFn` | 基于Dijkstra/A*，经典可靠 | 室内环境、静态地图 |
| `SmacPlanner2D` | 混合A*，支持非方形机器人 | 复杂形状机器人 |
| `SmacPlannerHybrid` | 混合A*，考虑车辆运动学 | 阿克曼转向车辆 |
| `ThetaStar2D` | 角度感知的A*，路径更平滑 | 需要平滑轨迹的场合 |

**配置示例**：

```yaml
planner_server:
  ros__parameters:
    expected_planner_frequency: 20.0
    use_lifecycle: true
    planner_plugins: ["grid_based"]

    grid_based:
      plugin: "nav2_navfn_planner/NavfnPlanner"
      tolerance: 0.5
      use_astar: false
      allow_unknown: true
```

### 2.2 控制器服务器（Controller Server）

控制器服务器负责局部路径跟踪和运动控制。它接收全局路径，生成实时的速度命令发送给机器人底座。

**控制器接口**：

```cpp
// nav2_core/include/nav2_core/controller.hpp
namespace nav2_core
{
  class Controller
  {
  public:
    virtual void configure(
      rclcpp_lifecycle::LifecycleNode::SharedPtr parent,
      std::string name,
      std::shared_ptr<tf2_ros::Buffer> tf,
      std::shared_ptr<nav2_costmap_2d::Costmap2DROS> costmap_ros) = 0;

    virtual geometry_msgs::msg::TwistStamped computeVelocityCommands(
      const geometry_msgs::msg::PoseStamped & pose,
      const geometry_msgs::msg::Twist & velocity) = 0;

    virtual void setPlan(const nav_msgs::msg::Path & path) = 0;

    virtual void reset() = 0;

    virtual ~Controller() = default;
  };
}
```

控制器是机器人运动的直接执行者，需要具备快速响应、平稳跟踪、避障能力。

**常用控制器**：

| 控制器 | 特点 | 适用场景 |
|--------|------|----------|
| `DWB Controller` | 基于DWA的动态窗口法 | 差速机器人、通用场景 |
| `TEB Controller` | 时间弹性带法 | 阿克曼车辆、复杂轨迹 |
| `MPPI Controller` | 基于模型预测的随机优化 | 高性能要求场景 |
| `Regulated Pure Pursuit` | 纯追踪+速度调节 | 农业、仓储机器人 |

**配置示例**：

```yaml
controller_server:
  ros__parameters:
    use_lifecycle: true
    controller_plugins: ["FollowPath"]

    FollowPath:
      plugin: "dwb_core/DWBLocalPlanner"
      min_vel_x: 0.0
      max_vel_x: 0.26
      max_vel_theta: 1.0
      min_speed_xy: 0.0
      max_speed_xy: 0.26
      acc_lim_x: 2.5
      acc_lim_theta: 3.2
```

### 2.3 恢复行为服务器（Recovery Server）

恢复行为服务器负责处理机器人卡死或导航失败的情况。当检测到异常时，系统会自动执行恢复策略。

**恢复行为类型**：

| 行为 | 功能 | 触发条件 |
|------|------|----------|
| `spin` | 原地旋转寻找出路 | 局部规划失败 |
| `backup` | 后退尝试脱困 | 前进受阻 |
| `wait` | 暂停等待障碍物移开 | 传感器被遮挡 |
| `clear_costmap` | 清除代价地图障碍物 | 代价地图数据过时 |
| `reinitialize` | 重新初始化定位 | 定位丢失 |

**恢复行为配置**：

```yaml
recoveries_server:
  ros__parameters:
    use_lifecycle: true
    recovery_plugins: ["spin", "backup", "clear_costmap"]

    spin:
      plugin: "nav2_recoveries/Spin"
      max_rotational_vel: 1.0
      min_rotational_vel: 0.4
      rotational_acc_lim: 2.0

    backup:
      plugin: "nav2_recoveries/BackUp"
      max_backup_vel: 0.1
      min_backup_vel: 0.05
      backup_dist: 0.5

    clear_costmap:
      plugin: "nav2_recoveries/ClearCostmap"
      reset_distance: 3.0
```

**恢复策略执行流程**：

```
┌─────────────────┐
│   导航执行中    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐     是     ┌─────────────────┐
│ 检测到异常?     │ ───────▶ │ 执行恢复行为   │
└────────┬────────┘           └────────┬────────┘
         │                            │
         │ 否                         ▼
         │                    ┌─────────────────┐
         │                    │ 恢复成功?       │
         └───────────────────▶└────────┬────────┘
                              否       │ 是
                                      ▼
                              ┌─────────────────┐
                              │   继续导航     │
                              └─────────────────┘
```

## 3. 行为树导航（BT Navigation）

行为树（Behavior Tree）是Nav2的行为控制核心，它取代了ROS1中的状态机，提供了更灵活、可读性更强的行为编排方式。

### 3.1 行为树基础

行为树是一种用于控制系统执行流程的树形结构。每个节点都有特定的功能，父节点的执行结果决定子节点是否被执行。

**节点类型**：

| 节点类型 | 功能 | 执行逻辑 |
|----------|------|----------|
| **Control Nodes** | 控制子节点执行顺序 | Sequence, Fallback, Parallel |
| **Decorator Nodes** | 修饰子节点行为 | Inverter, Repeater, Retry |
| **Action Nodes** | 执行具体动作 | ComputePath, FollowPath |
| **Condition Nodes** | 检查条件 | IsBatteryLow, IsPathValid |

**控制节点说明**：

- **Sequence（序列）**：按顺序执行子节点，全部成功则成功，任一失败则失败
- **Fallback（备选）**：尝试执行子节点，第一个成功的子节点使整体成功，全部失败则失败
- **Parallel（并行）**：同时执行多个子节点

### 3.2 Nav2行为树结构

Nav2的默认行为树结构如下：

```xml
<root main_tree_to_execute="MainTree">
  <BehaviorTree ID="MainTree">
    <RecoveryNode>
      <PipelineSequence>
        <RateController hz="10">
          <DriveWith/>
        </RateController>
        <PathExpireTimer hz="1">
          <PathTruncater/>
        </PathExpireTimer>
      </PipelineSequence>
      <ReactiveFallback>
        <IsBatteryCharged/>
        <RecoverySpinner/>
      </ReactiveFallback>
    </RecoveryNode>
  </BehaviorTree>
</root>
```

简化版本的行为树如下：

```
                    ┌─────────────┐
                    │   Recovery  │
                    │   (根节点)  │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  Pipeline   │
                    │  Sequence   │
                    └──────┬──────┘
                           │
         ┌─────────────────┼─────────────────┐
         │                 │                 │
    ┌────▼────┐      ┌────▼────┐      ┌────▼────┐
    │ Compute │      │  Path   │      │ Drive   │
    │  Path   │ ────▶│ Expire  │ ────▶│ To Pose │
    └─────────┘      │         │      └─────────┘
                    └─────────┘
```

### 3.3 自定义行为树

Nav2允许用户创建自定义行为树以满足特定需求。

**创建自定义行为树文件**：

```xml
<!-- ~/ros2_ws/src/nav2_custom/behavior_trees/my_navigate_to_pose.xml -->
<root main_tree_to_execute="NavigateToPose">
  <BehaviorTree ID="NavigateToPose">
    <RecoveryNode>
      <PipelineSequence>
        <!-- 计算全局路径 -->
        <ComputePathToPose goal="{goal}" path="{path}" planner_id="grid_based"/>
        <!-- 跟踪路径 -->
        <FollowPath path="{path}" controller_id="FollowPath"/>
      </PipelineSequence>
      <!-- 恢复行为 -->
      <Fallback>
        <!-- 尝试清理代价地图 -->
        <ClearCostmap service_name="clear_costmap"/>
        <!-- 原地旋转 -->
        <Spin/>
        <!-- 后退 -->
        <BackUp/>
      </Fallback>
    </RecoveryNode>
  </BehaviorTree>
</root>
```

**使用自定义行为树**：

```bash
# 通过参数指定行为树文件
ros2 launch nav2_bringup navigation_launch.py \
    bt_xml_file:=/path/to/my_navigate_to_pose.xml
```

**配置Nav2使用自定义BT**：

```yaml
bt_navigator:
  ros__parameters:
    use_lifecycle: true
    bt_xml_filename: /path/to/my_navigate_to_pose.xml
    plugin_lib_names:
      - nav2_compute_path_to_pose_action_bt_node
      - nav2_follow_path_action_bt_node
      - nav2_back_up_action_bt_node
      - nav2_spin_action_bt_node
      - nav2_wait_action_bt_node
      - nav2_clear_costmap_service_bt_node
```

### 3.4 行为树节点插件开发

开发者可以创建自定义的BT节点来扩展功能。

**创建自定义BT节点（C++）**：

```cpp
// custom_bt_node.cpp
#include <behaviortree_cpp_v3/behavior_tree.h>
#include <behaviortree_cpp_v3/bt_factory.h>
#include <rclcpp/rclcpp.hpp>
#include <geometry_msgs/msg/twist.hpp>

class CheckObstacle : public BT::ConditionNode
{
public:
  CheckObstacle(const std::string & name, const BT::NodeConfiguration & config)
    : BT::ConditionNode(name, config)
  {
    node_ = rclcpp::Node::make_shared("check_obstacle_client");
    client_ = node_->create_client<sensor_msgs::msg::LaserScan>("/scan");
  }

  BT::NodeStatus tick() override
  {
    auto request = std::make_shared<sensor_msgs::msg::LaserScan>();
    // 检查前方障碍物
    // 返回 BT::NodeStatus::SUCCESS 或 FAILURE
  }

private:
  rclcpp::Node::SharedPtr node_;
  rclcpp::Client<sensor_msgs::msg::LaserScan>::SharedPtr client_;
};

// 注册节点
BT_REGISTER_NODES(factory)
{
  factory.registerNodeType<CheckObstacle>("CheckObstacle");
}
```

## 4. 生命周期管理

Nav2采用ROS2的生命周期（Lifecycle）管理机制，使得节点的启动、停止、配置等操作更加可控和可靠。

### 4.1 生命周期状态机

ROS2.lifecycle定义了节点可能处于的状态以及状态之间的转换：

```
        ┌─────────────┐
        │  Uncreated  │
        └──────┬──────┘
               │ create
               ▼
        ┌─────────────┐
        │  Unconfigured│◀────────────┐
        └──────┬──────┘             │
               │ configure          │
               ▼                     │
        ┌─────────────┐              │
        │  Inactive   │──────────┐   │
        └──────┬──────┘          │   │
               │ activate         │   │
               ▼                  │ cleanup
        ┌─────────────┐          │   │
        │   Active    │──────────┴───┘
        └──────┬──────┘
               │ deactivate
               ▼
        ┌─────────────┐
        │ Inactive    │
        └─────────────┘
```

**生命周期状态说明**：

| 状态 | 说明 | 主要操作 |
|------|------|----------|
| `Unconfigured` | 初始状态，未配置 | 加载参数、初始化资源 |
| `Inactive` | 已配置但未运行 | 加载话题、服务 |
| `Active` | 正常运行 | 执行导航逻辑 |
| `Finalized` | 节点关闭 | 释放资源 |

### 4.2 Nav2生命周期管理

Nav2中的每个服务器都遵循生命周期管理规范，确保系统组件按正确顺序启动和停止。

**启动顺序**：

```bash
# 1. 启动生命周期管理器
ros2 lifecycle set /nav2_controller_server configure
ros2 lifecycle set /nav2_controller_server activate

# 2. 查看所有Nav2节点状态
ros2 lifecycle list /nav2_controller_server

# 3. 启动完整的Nav2栈
ros2 launch nav2_bringup navigation_launch.py
```

**Nav2生命周期转换流程**：

```
launch文件启动
     │
     ▼
┌────────────┐     ┌────────────┐     ┌────────────┐
│ map_server │ ──▶ │ amcl       │ ──▶ │ controller │
│ (config)   │     │ (config)   │     │ (config)   │
└────────────┘     └────────────┘     └────────────┘
     │                  │                  │
     ▼                  ▼                  ▼
┌────────────┐     ┌────────────┐     ┌────────────┐
│ map_server │     │ amcl       │     │ controller │
│ (activate) │     │ (activate) │     │ (activate) │
└────────────┘     └────────────┘     └────────────┘
     │                  │                  │
     └──────────────────┴──────────────────┘
                        │
                        ▼
              ┌────────────────────┐
              │   Nav2 全部就绪     │
              └────────────────────┘
```

### 4.3 生命周期参数配置

每个Nav2组件都支持通过参数文件进行生命周期配置：

```yaml

  ros__parameters:

    # ... 其他参数
```

### 4.4 手动控制生命周期

在实际开发中，可能需要手动控制生命周期节点：

```bash
# 查看生命周期状态
ros2 lifecycle list /nav2_controller_server

# 配置节点
ros2 lifecycle set /nav2_controller_server configure

# 激活节点
ros2 lifecycle set /nav2_controller_server activate

# 停用节点
ros2 lifecycle set /nav2_controller_server deactivate

# 清理节点
ros2 lifecycle set /nav2_controller_server cleanup

# 关闭节点
ros2 lifecycle set /nav2_controller_server shutdown
```

**使用rclpy控制生命周期**：

```python
import rclpy
from rclpy.lifecycle import LifecycleNode
from rclpy.lifecycle import State
from rclpy.lifecycle import TransitionCallbackReturn

class MyLifecycleNode(LifecycleNode):
    def __init__(self):
        super().__init__('my_lifecycle_node')

    def on_configure(self, state: State) -> TransitionCallbackReturn:
        self.get_logger().info('Configuring...')
        # 加载参数、初始化资源
        return TransitionCallbackReturn.SUCCESS

    def on_activate(self, state: State) -> TransitionCallbackReturn:
        self.get_logger().info('Activating...')
        # 启动发布者、订阅者
        return TransitionCallbackReturn.SUCCESS

    def on_deactivate(self, state: State) -> TransitionCallbackReturn:
        self.get_logger().info('Deactivating...')
        # 停止发布者、订阅者
        return TransitionCallbackReturn.SUCCESS

    def on_cleanup(self, state: State) -> TransitionCallbackReturn:
        self.get_logger().info('Cleaning up...')
        # 释放资源
        return TransitionCallbackReturn.SUCCESS
```

### 5.4 通过编程控制导航

```python
# navigate_to_pose.py
import rclpy
from rclpy.action import ActionClient
from rclpy.node import Node
from nav2_msgs.action import NavigateToPose
from geometry_msgs.msg import PoseStamped

class NavigateToPoseClient(Node):
    def __init__(self):
        super().__init__('navigate_to_pose_client')
        self._action_client = ActionClient(self, NavigateToPose, 'navigate_to_pose')

    def send_goal(self, x, y, yaw=0.0):
        goal_msg = NavigateToPose.Goal()
        goal_msg.pose = PoseStamped()
        goal_msg.pose.header.frame_id = 'map'
        goal_msg.pose.header.stamp = self.get_clock().now().to_msg()
        goal_msg.pose.pose.position.x = x
        goal_msg.pose.pose.position.y = y
        goal_msg.pose.pose.orientation.z = yaw

        self._action_client.wait_for_server()
        self._send_goal_future = self._action_client.send_goal_async(
            goal_msg,
            feedback_callback=self.feedback_callback
        )
        self._send_goal_future.add_done_callback(self.goal_response_callback)

    def goal_response_callback(self, future):
        goal_handle = future.result()
        if not goal_handle.accepted:
            self.get_logger().info('Goal rejected')
            return
        self.get_logger().info('Goal accepted')
        self._get_result_future = goal_handle.get_result_async()
        self._get_result_future.add_done_callback(self.get_result_callback)

    def get_result_callback(self, future):
        self.get_logger().info('Navigation completed')
        rclpy.shutdown()

    def feedback_callback(self, feedback_msg):
        feedback = feedback_msg.feedback
        self.get_logger().info(f'Navigation progress: {feedback.distance_remaining:.2f}m')

def main(args=None):
    rclpy.init(args=args)
    client = NavigateToPoseClient()
    client.send_goal(2.0, 1.0)  # 导航到 (2.0, 1.0)
    rclpy.spin(client)

if __name__ == '__main__':
    main()
```

### 选择题

1. Nav2中，哪个组件负责将全局路径转换为机器人的实时速度命令？
   - A) Planner Server
   - B) Controller Server
   - C) Recovery Server
   - D) BT Navigator
   
   **答案：B**。Controller Server负责局部路径跟踪和运动控制，将全局路径转换为实时速度命令。

2. 在Nav2的行为树中，哪个节点类型用于检查条件并返回成功/失败状态？
   - A) Action Node
   - B) Condition Node
   - C) Decorator Node
   - D) Control Node
   
   **答案：B**。Condition Node用于检查条件，返回SUCCESS或FAILURE状态。

3. ROS2.lifecycle中，节点从Active状态可以转换到哪个状态？
   - A) Unconfigured
   - B) Inactive
   - C) Finalized
   - D) Uncreated
   
   **答案：B**。Active状态可以通过deactivate转换到Inactive状态。

4. 下列哪个不是Nav2的恢复行为？
   - A) Spin
   - B) BackUp
   - C) Recharge
   - D) ClearCostmap
   
   **答案：C**。Spin、BackUp和ClearCostmap都是Nav2的恢复行为，Recharge不是内置恢复行为。
