## 一、课程概述

行为克隆通过监督学习直接模仿专家动作，但存在分布偏移（compounding error）问题；逆强化学习从专家轨迹推断奖励函数，但内外层双层优化计算代价极高。2016年，Ho 和 Ermon 在论文《Generative Adversarial Imitation Learning》中提出了 **GAIL（生成对抗模仿学习）**，将 GAN 的对抗博弈思想引入模仿学习，用判别器隐式区分专家轨迹与策略轨迹，以判别器输出作为奖励信号驱动策略优化——无需显式建模奖励函数，无需求解内层 RL 问题，实现了端到端的模仿学习。

GAIL 的核心洞察是：**不直接模仿动作，而是学习一个"判断行为是否像专家"的隐式奖励函数**。判别器与策略的对抗训练，使策略自发地向专家行为分布靠拢。本课程将从 GAN 基础出发，系统讲解 GAIL 的数学原理、网络设计、训练技巧与代码实现，并扩展到 AIRL、Wasserstein GAIL 等前沿变体。

学完本章后，你将理解 GAN 的对抗训练机制与模式崩溃根源，掌握 GAIL 从 GAN 到模仿学习的思想迁移，深入理解判别器作为隐式奖励函数的设计原理，能够实现完整的 GAIL 训练 pipeline，并掌握 AIRL、Wasserstein GAIL 等变体的改进思路。

## 三、理论内容

### 3.1 GAN 基础（生成对抗网络）

#### 3.1.1 生成对抗网络原理

**生成对抗网络（GAN）** 由 Goodfellow 等人于 2014 年提出，核心思想是通过**对抗训练**让生成器 $G$ 学会模拟真实数据分布。GAN 包含两个神经网络——生成器 $G$ 和判别器 $D$——在一个零和博弈中相互对抗：

```
真实数据 x ~ p_data(x)
生成数据 G(z), 其中 z ~ p_z(z) （随机噪声）

判别器 D(x): 输入 x，输出 x 来自真实数据的概率 D(x) ∈ [0,1]
生成器 G(z): 输入噪声 z，输出生成的"假"数据 G(z)
```

**博弈过程**：

| 角色 | 目标 | 行动 |
|------|------|------|
| **判别器 $D$** | 最大化分类准确率 | 尽可能区分真实数据与生成数据 |
| **生成器 $G$** | 最小化判别器准确率 | 生成尽可能逼真的数据骗过判别器 |

判别器试图最大化对真实数据 $x \sim p_{\text{data}}$ 的输出 $D(x)$，同时最小化对生成数据 $G(z)$ 的输出 $D(G(z))$。生成器则相反，试图让 $D(G(z))$ 尽可能大，即让判别器认为生成数据是真实的。

#### 3.1.2 生成器 vs 判别器

**判别器（Discriminator）**：

判别器是一个二分类器，接收真实数据 $x$ 或生成数据 $G(z)$，输出一个概率值表示"输入来自真实数据分布"的可能性。

```python
class Discriminator(nn.Module):
    """
    GAN 判别器：二分类网络
    输入：数据 x（图片、向量等）
    输出：属于真实数据的概率 D(x) ∈ [0, 1]
    """
    def __init__(self, data_dim, hidden_dims=[256, 256]):
        super().__init__()
        layers = []
        prev_dim = data_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.LeakyReLU(0.2)
            ])
            prev_dim = h_dim
        layers.append(nn.Linear(prev_dim, 1))
        layers.append(nn.Sigmoid())  # 输出概率
        self.network = nn.Sequential(*layers)
    
    def forward(self, x):
        return self.network(x)
```

**生成器（Generator）**：

生成器从简单噪声分布 $p_z(z)$ 中采样，通过神经网络变换为复杂数据分布 $p_g$ 中的样本。生成器本身不直接接触真实数据，而是通过判别器的反馈（梯度）间接学习。

```python
class Generator(nn.Module):
    """
    GAN 生成器：从噪声分布生成数据
    输入：随机噪声 z ~ p_z(z)
    输出：生成数据 G(z)
    """
    def __init__(self, noise_dim, data_dim, hidden_dims=[256, 256]):
        super().__init__()
        layers = []
        prev_dim = noise_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        layers.append(nn.Linear(prev_dim, data_dim))
        layers.append(nn.Tanh())  # 输出归一化到 [-1, 1]
        self.network = nn.Sequential(*layers)
    
    def forward(self, z):
        return self.network(z)
```

**两者对比**：

| 维度 | 生成器 $G$ | 判别器 $D$ |
|------|-----------|-----------|
| **输入** | 随机噪声 $z \sim p_z(z)$ | 数据 $x$（真实或生成） |
| **输出** | 生成数据 $G(z)$ | 真实概率 $D(x) \in [0,1]$ |
| **目标** | 最大化 $D(G(z))$（骗过判别器） | 最大化 $D(x)$，最小化 $D(G(z))$ |
| **结构** | 通常与判别器对称（转置卷积） | 标准分类网络（全卷积） |
| **训练信号** | 来自判别器的梯度 | 来自真实标签的交叉熵损失 |

#### 3.1.3 损失函数与训练过程

**GAN 的价值函数（Value Function）**：

GAN 的训练可以形式化为一个极小极大博弈（Minimax Game）：

$$
\min_G \max_D \mathcal{L}(D, G) = \mathbb{E}_{x \sim p_{\text{data}}(x)}[\log D(x)] + \mathbb{E}_{z \sim p_z(z)}[\log(1 - D(G(z)))]
$$

**直觉解释**：
- **最大化 $\log D(x)$**：判别器在真实数据上的对数似然越高越好
- **最大化 $\log(1 - D(G(z)))$**：判别器在生成数据上的对数似然（$D(G(z))$ 越小越好）越高越好
- **生成器最小化 $\log(1 - D(G(z)))$**：生成器试图让 $D(G(z))$ 变大

**训练过程（交替优化）**：

```
1. 固定生成器 G，更新判别器 D：
   - 从真实数据采样: x ~ p_data
   - 从噪声分布采样: z ~ p_z
   - 生成假数据: x_fake = G(z)
   - 计算判别器损失: L_D = -log D(x) - log(1 - D(x_fake))
   - 更新 D 的参数（梯度上升）

2. 固定判别器 D，更新生成器 G：
   - 从噪声分布采样: z ~ p_z
   - 生成假数据: x_fake = G(z)
   - 计算生成器损失: L_G = -log D(x_fake)
   - 更新 G 的参数（梯度下降）

3. 重复步骤 1-2 直到收敛
```

**训练技巧**：实际训练中，生成器损失通常使用 $\min(-\log D(G(z)))$ 而非 $\min(\log(1 - D(G(z))))$，因为后者在训练初期梯度较小（判别器容易区分，$D(G(z)) \approx 0$，梯度消失）。

```python
def training_step(D, G, real_data, noise_dim, optimizer_D, optimizer_G):
    """
    GAN 一步训练
    """
    batch_size = real_data.size(0)
    
    # ====== 1. 训练判别器 ======
    real_labels = torch.ones(batch_size, 1)
    fake_labels = torch.zeros(batch_size, 1)
    
    # 真实数据损失
    real_output = D(real_data)
    loss_D_real = F.binary_cross_entropy(real_output, real_labels)
    
    # 生成数据损失
    noise = torch.randn(batch_size, noise_dim)
    fake_data = G(noise)
    fake_output = D(fake_data.detach())  # detach 避免更新 G
    loss_D_fake = F.binary_cross_entropy(fake_output, fake_labels)
    
    loss_D = loss_D_real + loss_D_fake
    optimizer_D.zero_grad()
    loss_D.backward()
    optimizer_D.step()
    
    # ====== 2. 训练生成器 ======
    noise = torch.randn(batch_size, noise_dim)
    fake_data = G(noise)
    fake_output = D(fake_data)
    # 生成器损失：让判别器认为生成数据是真实的
    loss_G = F.binary_cross_entropy(fake_output, real_labels)
    optimizer_G.zero_grad()
    loss_G.backward()
    optimizer_G.step()
    
    return loss_D.item(), loss_G.item()
```

#### 3.1.4 GAN 模式崩溃问题

**模式崩溃（Mode Collapse）** 是 GAN 训练中最常见的问题，指生成器学会"骗过"判别器而不真正学习数据分布的多样性——生成器输出趋向于单一模式，忽略其他模式。

**问题表现**：

| 现象 | 说明 |
|------|------|
| **多样性丧失** | 生成器只生成某一种类别的样本（如只生成某一数字的手写体） |
| **模式坍缩** | 数据分布有多个峰值，生成器只捕获其中一个 |
| **训练震荡** | 判别器和生成器损失剧烈波动，无法收敛 |
| **梯度消失** | 判别器过强时，生成器梯度接近零 |

**模式崩溃的数学分析**：

设真实数据分布 $p_{\text{data}}$ 有 $K$ 个模式（mode），理想生成器应生成所有 $K$ 个模式。但在对抗训练中：

1. 判别器 $D$ 逐步变强，能精确区分每个模式
2. 生成器 $G$ 发现"欺骗"判别器的捷径——只生成最容易骗过的单个模式
3. 该模式对应的判别器梯度变弱，$G$ 不再有动力探索其他模式
4. 最终 $G$ 收敛到某个模式，而非完整分布

**解决方案**：

| 方法 | 核心思想 | 代表算法 |
|------|---------|---------|
| **特征匹配** | 在特征空间而非输出空间匹配分布 | Feature Matching GAN |
| **小批次判别** | 判别器同时看多个样本判断真实性 | Minibatch Discrimination |
| **经验重放** | 保留历史生成样本，打破模式锁定 | - |
| **Wasserstein 距离** | 用连续 Wasserstein 距离替代 JS 散度 | WGAN |
| **光谱归一化** | 对判别器权重做谱归一化，稳定训练 | SN-GAN |
| **双时序更新** | 生成器更新次数多于判别器 | - |

```python
# 缓解模式崩溃：小批次判别示例
class MinibatchDiscrimination(nn.Module):
    """
    小批次判别层：让判别器同时感知多个样本的差异
    减少生成器只生成单一模式的动力
    """
    def __init__(self, input_dim, num_kernels=50, kernel_dim=5):
        super().__init__()
        self.num_kernels = num_kernels
        self.kernel_dim = kernel_dim
        # 可学习的变换矩阵 T：每个 kernel 提取一种跨样本特征
        self.T = nn.Parameter(torch.randn(input_dim, num_kernels * kernel_dim))
    
    def forward(self, x):
        # x: (batch_size, input_dim)
        matrices = x @ self.T  # (batch_size, num_kernels * kernel_dim)
        matrices = matrices.view(-1, self.num_kernels, self.kernel_dim)
        
        # 计算批次内样本间 L1 距离
        x1 = x.unsqueeze(0)  # (1, batch, dim)
        x2 = x.unsqueeze(1)  # (batch, 1, dim)
        diff = (x1 - x2).abs().sum(dim=-1)  # (batch, batch)
        
        # 跨样本距离作为额外特征
        return diff
```

### 3.2 GAIL 原理详解（生成对抗模仿学习）

#### 3.2.1 从 GAN 到 GAIL 的思想迁移

GAIL 的核心创新是将 GAN 的对抗框架迁移到模仿学习场景：

| GAN | GAIL |
|-----|------|
| 生成器 $G$ 生成图片 | **策略网络 $\pi$ 生成轨迹** |
| 判别器 $D$ 区分真实/生成图片 | **判别器 $D$ 区分专家/策略轨迹** |
| 真实数据分布 $p_{\text{data}}$ | 专家轨迹分布 $\pi_E$ |
| 生成器学着骗过判别器 | 策略学着骗过判别器 |

```
GAN:  真实图片 x ──→ D(x) ──→ 输出：真实概率
      生成图片 G(z) ─→ D ──→ 输出：真实概率
                                ↓ 对抗优化
                           G 学着生成更像真实的图片

GAIL: 专家轨迹 (s,a) ──→ D(s,a) ──→ 输出：专家概率
      策略轨迹 π(s,a) ─→ D ──→ 输出：专家概率
                                ↓ 对抗优化
                          π 学着生成更像专家的轨迹
```

**GAIL 的关键洞察**：

行为克隆直接让策略记忆"状态 $s$ 下的动作 $a$"，这种记忆式学习无法泛化到未见过的状态。GAIL 换了一个思路：**让策略学会"判断一个 (s,a) 对有多像专家行为"**。判别器 $D(s,a)$ 的输出可以被当作奖励信号，引导策略向专家分布方向优化——这比记忆具体动作更具泛化能力。

#### 3.2.2 GAIL 的核心思想

**不直接模仿动作，而是学习一个隐式的奖励函数。**

GAIL 的训练目标是：

$$
\min_{\pi} \max_{D} \mathbb{E}_{\pi}[\log(1 - D(s,a))] + \mathbb{E}_{\pi_E}[\log D(s,a)]
$$

这个目标与 GAN 几乎完全相同，区别仅在于：
- GAN 的生成器从噪声 $z$ 生成数据
- GAIL 的"生成器"是策略网络 $\pi$，从状态 $s$ 生成动作 $a$，构成轨迹 $(s,a)$

**GAIL 的工作流程**：

```
┌──────────────────────────────────────────────────────┐
│                    GAIL 训练循环                       │
│                                                       │
│  专家轨迹 D_E ──┐                                     │
│                 ├──→ 判别器 D ──→ 二分类损失           │
│  策略轨迹 D_π ──┘                                     │
│                 ↓                                      │
│          D 的输出作为奖励 r(s,a)                       │
│                 ↓                                      │
│          用 PPO/TRPO 优化策略 π                        │
│                 ↓                                      │
│          新轨迹收集 → 继续对抗训练                       │
└──────────────────────────────────────────────────────┘
```

#### 3.2.3 奖励函数学习

GAIL 的核心创新在于**奖励函数是由判别器隐式定义的**。

**判别器设计**：

判别器 $D_\omega(s,a)$ 是一个二分类神经网络，输入状态-动作对 $(s,a)$，输出该对来自专家轨迹的概率。GAIL 沿用了标准 GAN 的 Sigmoid 判别器结构：

```python
class Discriminator(nn.Module):
    """
    GAIL 判别器网络
    输入：状态-动作对 (s, a)
    输出：属于专家的概率 D(s,a) ∈ [0, 1]
    """
    def __init__(self, state_dim, action_dim, hidden_dims=[256, 256]):
        super().__init__()
        input_dim = state_dim + action_dim
        layers = []
        prev_dim = input_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        layers.extend([
            nn.Linear(prev_dim, 1),
            nn.Sigmoid()
        ])
        self.network = nn.Sequential(*layers)
    
    def forward(self, state, action):
        """
        Args:
            state: 状态张量 (batch_size, state_dim)
            action: 动作张量 (batch_size, action_dim)
        Returns:
            D(s,a): 专家概率 (batch_size, 1)
        """
        sa = torch.cat([state, action], dim=-1)
        return self.network(sa)
    
    def get_reward(self, state, action):
        """
        GAIL 奖励函数：从判别器输出计算奖励
        r(s,a) = log(D) - log(1-D)
        当 D→1（像专家）→ r → +∞（高奖励）
        当 D→0（不像专家）→ r → -∞（负奖励）
        当 D=0.5（无法区分）→ r = 0（中立）
        """
        d = self.forward(state, action)
        reward = torch.log(d + 1e-8) - torch.log(1 - d + 1e-8)
        return reward
```

**判别器损失函数**：

$$
\mathcal{L}(\omega) = -\mathbb{E}_{(s,a) \sim \pi_E}[\log D_\omega(s,a)] - \mathbb{E}_{(s,a) \sim \pi_\theta}[\log(1 - D_\omega(s,a))]
$$

```python
def compute_discriminator_loss(D, states_exp, actions_exp, states_pol, actions_pol):
    """
    计算 GAIL 判别器损失（二元交叉熵）
    Args:
        D: 判别器网络
        states_exp, actions_exp: 专家状态-动作对（标签=1）
        states_pol, actions_pol: 策略状态-动作对（标签=0）
    Returns:
        判别器损失 (标量)
    """
    # 专家样本：标签为 1（真实）
    exp_output = D(states_exp, actions_exp)
    loss_exp = F.binary_cross_entropy(exp_output, torch.ones_like(exp_output))
    
    # 策略样本：标签为 0（生成/假）
    pol_output = D(states_pol, actions_pol)
    loss_pol = F.binary_cross_entropy(pol_output, torch.zeros_like(pol_output))
    
    return loss_exp + loss_pol
```

**奖励函数分析**：

GAIL 的奖励函数 $r(s,a) = \log D(s,a) - \log(1 - D(s,a))$ 有几个重要性质：

| 判别器输出 $D(s,a)$ | 奖励 $r(s,a)$ | 物理含义 |
|---------------------|--------------|---------|
| $D \to 1$（判别器确信是专家） | $r \to +\infty$ | 策略应该多做这类行为 |
| $D \to 0$（判别器确信是策略） | $r \to -\infty$ | 策略应该避免这类行为 |
| $D = 0.5$（无法区分） | $r = 0$ | 中立，引导探索 |
| $D = 0.9$（高概率专家） | $r \approx 2.197$ | 正奖励 |
| $D = 0.1$（低概率专家） | $r \approx -2.197$ | 负奖励 |

#### 3.2.4 GAIL vs Behavior Cloning

| 维度 | 行为克隆（BC） | GAIL |
|------|--------------|------|
| **学习方式** | 监督分类/回归 | 生成对抗 |
| **是否需要奖励** | 否 | 否（判别器隐式定义） |
| **泛化能力** | 受限于专家数据覆盖范围 | 通过奖励泛化到新状态 |
| **分布偏移** | 存在（compounding error） | 通过对抗训练逐步修正 |
| **训练稳定性** | 高（标准监督学习） | 中等（需要平衡 D 和 π 的训练） |
| **样本效率** | 高（一次性收集数据） | 中等（需要持续收集策略轨迹） |
| **计算复杂度** | 低 | 中等（额外判别器网络） |

**BC 的问题**：
- 策略在专家见过的状态上表现好，但一旦偏离，误差累积导致进入从未训练的区域
- 策略不会"理解"任务目标，只是机械地复制动作

**GAIL 的优势**：
- 不逐状态复制动作，通过奖励泛化到未见过的状态
- 不需要在线专家（对抗信号来自判别器自身）
- 策略学到的是"什么行为像专家"而非"什么动作是对的"

#### 3.2.5 收敛性分析

**GAIL 的联合优化目标**：

GAIL 的完整优化目标是一个极小极大问题：

$$
\min_{\pi} \max_{D} \mathbb{E}_{\pi}[\log(1 - D(s,a))] + \mathbb{E}_{\pi_E}[\log D(s,a)]
$$

**收敛条件**：

当系统收敛时，判别器 $D$ 无法区分专家轨迹与策略轨迹，即 $D^*(s,a) = 0.5$ 对所有 $(s,a)$ 成立。此时：

$$
\pi = \pi^* \quad \text{（策略分布等于专家分布）}
$$

这意味着 **GAIL 的最优解是让策略轨迹分布与专家轨迹分布完全重合**，这比 BC 只匹配动作均值/分布具有更强的理论保证。

**实践中收敛困难的原因**：

1. **判别器过强**：判别器在策略轨迹分布上学得太快，奖励信号消失（梯度消失）
2. **策略崩溃**：策略在判别器奖励信号的局部最优处停止探索
3. **模式崩溃**：策略只覆盖专家分布的部分模式

这些问题与 GAN 的训练困难高度相似，也是 Wasserstein GAIL、GAIL-LS 等变体试图解决的。

### 3.3 其他对抗模仿学习方法

#### 3.3.1 AIRL（对抗逆强化学习）

**问题**：GAIL 学到的奖励是隐式的，无法解释，难以迁移到新环境。GAIL 的最优策略对应某个奖励函数，但这个奖励函数本身没有被显式建模。

**解决**：AIRL（Adversarial Inverse Reinforcement Learning）由 Fu 等人于 2018 年提出，同时学习一个**可解释的显式奖励函数** $r_\theta(s,a)$ 和一个势函数 $\phi(s)$。

**AIRL 的判别器结构**：

$$
f_\theta(s,a,s') = r_\theta(s,a) + \gamma \phi(s') - \phi(s)
$$

$$
D = \frac{\exp(f_\theta)}{\exp(f_\theta) + \pi(a|s)} = \ sigmoid(f_\theta - \log \pi(a|s))
$$

```python
class AIRLDiscriminator(nn.Module):
    """
    AIRL 判别器：同时学习奖励函数和势函数
    奖励可解释：r(s,a) = f - log π(a|s) 的主要项
    """
    def __init__(self, state_dim, action_dim, gamma=0.99, hidden_dims=[256, 256]):
        super().__init__()
        self.gamma = gamma
        
        # 奖励网络 r(s,a)：可解释的显式奖励
        reward_layers = []
        prev_dim = state_dim + action_dim
        for h_dim in hidden_dims:
            reward_layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        reward_layers.append(nn.Linear(prev_dim, 1))
        self.reward_net = nn.Sequential(*reward_layers)
        
        # 势函数网络 φ(s)：状态的价值估计
        phi_layers = []
        prev_dim = state_dim
        for h_dim in hidden_dims:
            phi_layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        phi_layers.append(nn.Linear(prev_dim, 1))
        self.phi_net = nn.Sequential(*phi_layers)
    
    def forward(self, state, action, next_state, log_pi=None):
        """
        计算 AIRL f 函数：f = r + γ*φ(s') - φ(s)
        """
        r = self.reward_net(torch.cat([state, action], dim=-1))
        phi_s = self.phi_net(state)
        phi_sp1 = self.phi_net(next_state)
        
        f = r + self.gamma * phi_sp1 - phi_s
        
        if log_pi is not None:
            # D = sigmoid(f - log_pi)，稳定实现
            d = torch.sigmoid(f - log_pi.unsqueeze(-1))
        else:
            d = torch.sigmoid(f)
        
        return d
    
    def get_explicit_reward(self, state, action):
        """获取可解释的显式奖励（AIRL 的核心输出）"""
        return self.reward_net(torch.cat([state, action], dim=-1))
```

**AIRL vs GAIL**：

| 维度 | AIRL | GAIL |
|------|------|------|
| **奖励函数** | 显式 $r_\theta(s,a)$ | 隐式 $r = \log D - \log(1-D)$ |
| **可解释性** | 高（可直接分析奖励） | 低（隐式奖励难以解释） |
| **可迁移性** | 高（奖励函数可跨环境） | 低 |
| **训练稳定性** | 较高（势函数提供额外约束） | 中等 |

#### 3.3.2 Discriminator-Critic Design

**问题**：标准 GAIL 判别器是 Sigmoid 输出，在类别重叠严重时梯度消失，训练不稳定。

**解决**：引入 **Value-Based Discriminator（Critic）** 设计，将判别器与价值函数结合。

**两种改进方向**：

**方向一：最小二乘判别器（GAIL-LS）**

用最小二乘损失替代交叉熵损失，输出从 Sigmoid 概率改为实数值：

$$
\min_D \mathbb{E}_\pi[(D(s,a) - 1)^2] + \mathbb{E}_{\pi_E}[(D(s,a))^2]
$$

```python
class DiscriminatorLS(nn.Module):
    """
    GAIL-LS 判别器：Least Squares 版本
    输出为实数值，不经过 Sigmoid
    避免梯度消失问题
    """
    def __init__(self, state_dim, action_dim, hidden_dims=[256, 256]):
        super().__init__()
        input_dim = state_dim + action_dim
        layers = []
        prev_dim = input_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        layers.append(nn.Linear(prev_dim, 1))
        self.network = nn.Sequential(*layers)  # 无 Sigmoid，直接输出实数
    
    def forward(self, state, action):
        sa = torch.cat([state, action], dim=-1)
        return self.network(sa)  # 输出实数
    
    def get_reward(self, state, action):
        """奖励 = -D²，当 D→0（像专家）奖励变高"""
        d = self.forward(state, action)
        return -d.pow(2)
```

**方向二：能量模型判别器**

用能量函数 $E(s,a)$ 替代概率输出，$p(a|s) \propto \exp(-E(s,a))$：

```python
class EnergyBasedDiscriminator(nn.Module):
    """
    能量模型判别器
    能量越低 → 越像专家行为
    奖励 = -energy
    """
    def __init__(self, state_dim, action_dim, hidden_dims=[256, 256]):
        super().__init__()
        input_dim = state_dim + action_dim
        layers = []
        prev_dim = input_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        layers.append(nn.Linear(prev_dim, 1))
        self.network = nn.Sequential(*layers)  # 无 Sigmoid
    
    def forward(self, state, action):
        """输出能量值，越低越像专家"""
        sa = torch.cat([state, action], dim=-1)
        return self.network(sa)
    
    def get_reward(self, state, action):
        """奖励 = -能量"""
        energy = self.forward(state, action)
        return -energy
```

#### 3.3.3 Dropout 技术在判别器训练中的作用

**问题**：判别器在有限专家数据上容易过拟合，导致奖励信号过于稀疏。

**解决**：在判别器中引入 **Dropout** 作为正则化和不确定性估计。

```python
class DropoutDiscriminator(nn.Module):
    """
    带 Dropout 的 GAIL 判别器
    Dropout 提供两种作用：
    1. 正则化：防止判别器在专家数据上过拟合
    2. 不确定性估计：多次采样估计奖励的方差
    """
    def __init__(self, state_dim, action_dim, hidden_dims=[256, 256], dropout_p=0.5):
        super().__init__()
        input_dim = state_dim + action_dim
        layers = []
        prev_dim = input_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU(),
                nn.Dropout(dropout_p)  # 每层后加 Dropout
            ])
            prev_dim = h_dim
        layers.extend([
            nn.Linear(prev_dim, 1),
            nn.Sigmoid()
        ])
        self.network = nn.Sequential(*layers)
    
    def forward(self, state, action, use_dropout=True):
        """
        Args:
            use_dropout: 推理时是否使用 Dropout（用于不确定性估计）
        """
        if not self.training and not use_dropout:
            # 关闭 Dropout 进行确定性推理
            self.network.eval()
        else:
            self.network.train()
        
        sa = torch.cat([state, action], dim=-1)
        return self.network(sa)
    
    def estimate_reward_uncertainty(self, state, action, num_samples=10):
        """
        通过多次采样估计奖励不确定性
        用于检测策略是否在"安全区域"（低不确定性）
        """
        rewards = []
        for _ in range(num_samples):
            r = self.forward(state, action, use_dropout=True)
            rewards.append(r)
        
        rewards = torch.stack(rewards, dim=0)  # (num_samples, batch, 1)
        mean_reward = rewards.mean(dim=0)
        std_reward = rewards.std(dim=0)  # 不确定性指标
        
        return mean_reward, std_reward
```

**Dropout 的核心作用总结**：

| 作用 | 机制 | 效果 |
|------|------|------|
| **防止过拟合** | 随机丢弃神经元，降低判别器对特定专家样本的记忆 | 判别器泛化更好 |
| **不确定性估计** | MC Dropout：用多个 Dropout mask 采样估计奖励方差 | 检测策略是否在安全区 |
| **模式探索** | Dropout 的随机性让判别器对相似状态-动作对更鲁棒 | 减少模式崩溃风险 |

#### 3.3.4 Wasserstein GAIL

**问题**：JS 散度在分布不重叠时梯度消失，导致训练停滞。

**解决**：用 Wasserstein-1 距离（Earth-Mover Distance）替代 JS 散度。

Wasserstein GAIL 的判别器不再输出 Sigmoid 概率，而是输出实数值（类似 WGAN 的 Critic），并使用梯度惩罚（Gradient Penalty）确保 Lipschitz 约束：

$$
\mathcal{L} = \mathbb{E}_{\pi}[D(s,a)] - \mathbb{E}_{\pi_E}[D(s,a)] + \lambda \mathbb{E}_{\hat{x}}[(\|\nabla_{\hat{x}} D(\hat{x})\| - 1)^2]
$$

#### 3.4.1 专家数据要求

| 参数 | 建议值 | 说明 |

#### 3.4.3 稳定训练方法

GAIL 训练不稳定是公认难题，以下是经过实践验证的稳定训练策略：

**1. 判别器学习率保守设置**

判别器学习率通常应低于策略学习率（建议 $3 \times 10^{-4}$ vs $3 \times 10^{-4}$）。判别器训练过快会导致奖励信号消失。

```python
# 推荐：判别器学习率略低于策略
discriminator_optimizer = optim.Adam(discriminator.parameters(), lr=2e-4)
policy_optimizer = optim.Adam(policy.parameters(), lr=3e-4)
```

**2. 标签平滑（Label Smoothing）**

将专家标签从 1.0 平滑到 0.9-0.95，减少判别器对专家数据的过拟合：

```python
# 原始标签：专家=1.0，策略=0.0
# 平滑标签：专家=0.9，策略=0.1
exp_labels = torch.full_like(exp_output, 0.95)
pol_labels = torch.full_like(pol_output, 0.05)
loss_exp = F.binary_cross_entropy(exp_output, exp_labels)
loss_pol = F.binary_cross_entropy(pol_output, pol_labels)
```

**3. 经验重放（Experience Replay）**

保留历史策略轨迹，混合当前轨迹一起训练判别器：

```python
class ReplayBuffer:
    """策略轨迹缓冲区：混合历史和当前轨迹训练判别器"""
    def __init__(self, max_size=50000):
        self.buffer = deque(maxlen=max_size)
    
    def add(self, states, actions, is_expert=False):
        for s, a in zip(states, actions):
            self.buffer.append((s, a, is_expert))
    
    def sample(self, batch_size):
        batch = random.sample(self.buffer, min(batch_size, len(self.buffer)))
        states, actions, labels = zip(*batch)
        return torch.FloatTensor(states), torch.FloatTensor(actions), torch.FloatTensor(labels)
    
    def __len__(self):
        return len(self.buffer)
```

**4. 梯度惩罚（Gradient Penalty）**

在判别器损失中加入梯度惩罚项，防止判别器过于陡峭：

```python
def compute_gradient_penalty(D, real_data, fake_data):
    """WGAN-GP 风格的梯度惩罚"""
    batch_size = real_data.size(0)
    alpha = torch.rand(batch_size, 1)
    
    # 在真实数据和生成数据之间插值
    interpolates = alpha * real_data + (1 - alpha) * fake_data
    interpolates = interpolates.requires_grad_(True)
    
    d_interpolates = D(interpolates)
    
    gradients = torch.autograd.grad(
        outputs=d_interpolates,
        inputs=interpolates,
        grad_outputs=torch.ones_like(d_interpolates),
        create_graph=True
    )[0]
    
    gradients = gradients.view(batch_size, -1)
    gradient_norm = gradients.norm(2, dim=1)
    penalty = ((gradient_norm - 1) ** 2).mean()
    
    return penalty
```

#### 3.4.4 收敛判断标准

GAIL 收敛的判断需要综合多个指标：

| 指标 | 判断方法 | 收敛标准 |
|------|---------|---------|
| **判别器 loss** | 趋于 0.5-0.7（无法区分专家和策略） | $\mathcal{L}_D \approx -\log(0.5) \approx 0.693$ |
| **策略 episode reward** | 逐步提升并趋于稳定 | 连续 10 回合奖励波动 < 5% |
| **D 输出分布重叠** | 专家和策略的 $D$ 均值趋于 0.5 | $\|D_{\text{exp}} - D_{\text{pol}}\| < 0.1$ |
| **轨迹相似度** | 计算轨迹间 KL/JS 散度 | 逐步下降 |

```python
def check_convergence(discriminator, expert_states, expert_actions,
                      policy_states, policy_actions, threshold=0.1):
    """
    检查 GAIL 是否收敛
    Args:
        threshold: 分布重叠阈值
    Returns:
        converged: 是否收敛
        metrics: 诊断指标字典
    """
    with torch.no_grad():
        exp_d = discriminator(expert_states, expert_actions).mean().item()
        pol_d = discriminator(policy_states, policy_actions).mean().item()
    
    d_gap = abs(exp_d - pol_d)
    
    metrics = {
        'expert_D_mean': exp_d,      # 理想值 ≈ 0.5
        'policy_D_mean': pol_d,      # 理想值 ≈ 0.5
        'D_gap': d_gap,              # 理想值 → 0
    }
    
    converged = d_gap < threshold
    return converged, metrics
```

#### 3.4.5 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 判别器快速过拟合 | 专家数据太少 | 增加专家数据、使用标签平滑、启用 Dropout |
| 策略崩溃（reward 爆炸） | 判别器梯度不稳定 | 使用 GAIL-LS、Wasserstein GAIL |
| 策略无法骗过判别器 | 策略探索不足 | 增加 PPO 的熵系数、使用更大网络 |
| 训练震荡 | 学习率过大 | 降低学习率、使用学习率调度 |
| 专家数据与策略数据分布差异大 | 数据收集策略不当 | 归一化状态、使用相同初始状态分布 |
| 判别器 loss = 0（完全判别） | 策略与专家分布完全不重叠 | 检查专家数据质量、减少判别器更新次数 |

#### 4.1.2 网络定义

```python
"""
gail/networks.py - GAIL 网络定义
包含 Actor-Critic 策略网络和 GAIL 判别器网络
"""
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np


# ============ Actor-Critic 策略网络 ============
class ActorCritic(nn.Module):
    """
    Actor-Critic 策略网络（用于连续控制任务）
    - Actor: 输出动作均值和方差（高斯策略）
    - Critic: 输出状态价值 V(s)
    """
    def __init__(self, state_dim, action_dim, hidden_dims=[256, 256]):
        super().__init__()
        self.action_dim = action_dim
        
        # ===== Actor：策略网络 =====
        actor_layers = []
        prev_dim = state_dim
        for h_dim in hidden_dims:
            actor_layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        self.actor = nn.Sequential(*actor_layers)
        self.actor_head = nn.Linear(prev_dim, action_dim)
        
        # ===== Critic：价值网络 =====
        critic_layers = []
        prev_dim = state_dim
        for h_dim in hidden_dims:
            critic_layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        self.critic = nn.Sequential(*critic_layers)
        self.critic_head = nn.Linear(prev_dim, 1)
        
        # 动作输出层：初始化为较小值（输出方差的对数）
        self.log_std = nn.Parameter(torch.zeros(action_dim))
    
    def forward(self, state):
        """
        前向传播，同时返回动作分布和状态价值
        Args:
            state: 状态张量 (batch_size, state_dim)
        Returns:
            action_mean: 动作均值 (batch_size, action_dim)
            action_std: 动作标准差 (action_dim,)
            value: 状态价值 (batch_size, 1)
        """
        action_mean = self.actor_head(self.actor(state))
        action_std = torch.exp(self.log_std)
        value = self.critic_head(self.critic(state))
        return action_mean, action_std, value
    
    def get_action(self, state, deterministic=False):
        """
        采样一个动作
        Args:
            state: 状态张量 (state_dim,) 或 (batch_size, state_dim)
            deterministic: 是否使用确定性策略（仅返回均值）
        Returns:
            action: 动作 numpy 数组
            log_prob: 对数概率
            value: 状态价值
        """
        if state.dim() == 1:
            state = state.unsqueeze(0)
        
        action_mean, action_std, value = self.forward(state)
        
        if deterministic:
            action = action_mean
            log_prob = None
        else:
            dist = torch.distributions.Normal(action_mean, action_std)
            action = dist.sample()
            log_prob = dist.log_prob(action).sum(dim=-1, keepdim=True)
        
        return action.squeeze(0).cpu().numpy(), log_prob, value.squeeze(0)
    
    def get_distribution(self, state):
        """获取当前状态下的动作分布"""
        action_mean, action_std, _ = self.forward(state)
        return torch.distributions.Normal(action_mean, action_std)
    
    def get_log_prob(self, state, action):
        """计算给定状态-动作对的对数概率"""
        dist = self.get_distribution(state)
        return dist.log_prob(action).sum(dim=-1, keepdim=True)
    
    def get_value(self, state):
        """获取状态价值"""
        if state.dim() == 1:
            state = state.unsqueeze(0)
        _, _, value = self.forward(state)
        return value.squeeze(-1)


# ============ GAIL 判别器网络 ============
class Discriminator(nn.Module):
    """
    GAIL 判别器网络
    输入：状态-动作对 (s, a)
    输出：属于专家的概率 D(s,a) ∈ [0, 1]
    """
    def __init__(self, state_dim, action_dim, hidden_dims=[256, 256]):
        super().__init__()
        input_dim = state_dim + action_dim
        layers = []
        prev_dim = input_dim
        for h_dim in hidden_dims:
            layers.extend([
                nn.Linear(prev_dim, h_dim),
                nn.ReLU()
            ])
            prev_dim = h_dim
        layers.extend([
            nn.Linear(prev_dim, 1),
            nn.Sigmoid()
        ])
        self.network = nn.Sequential(*layers)
    
    def forward(self, state, action):
        """
        Args:
            state: 状态张量 (batch, state_dim)
            action: 动作张量 (batch, action_dim)
        Returns:
            D(s,a): 专家概率 (batch, 1)
        """
        sa = torch.cat([state, action], dim=-1)
        return self.network(sa)
    
    def get_reward(self, state, action):
        """
        GAIL 奖励函数：从判别器输出计算奖励
        r(s,a) = log(D) - log(1-D)
        当 D→1（像专家）→ 奖励高
        当 D→0（不像专家）→ 奖励低
        """
        d = self.forward(state, action)
        reward = torch.log(d + 1e-8) - torch.log(1 - d + 1e-8)
        return reward
```

```python
"""
gail/utils.py - GAIL 训练工具函数
"""
import torch
import numpy as np


def compute_gae(rewards, dones, last_value, gamma=0.99, lambda_=0.95):
    """
    计算 GAE（Generalized Advantage Estimation）
    用于估计优势函数 A(s,a) = Q(s,a) - V(s)
    Args:
        rewards: 奖励列表 [step0, step1, ..., stepT]
        dones: done 标志列表 [done0, done1, ..., doneT]
        last_value: 最后一个状态的价值估计
        gamma: 折扣因子
        lambda_: GAE 参数（值越大越偏向蒙特卡洛估计）
    Returns:
        advantages: 优势估计（用于 PPO 策略更新）
        returns: 回报（用于价值网络训练）
    """
    advantages = []
    gae = 0
    # values 的长度比 rewards 多 1（包含最后一个状态的价值）
    values = [last_value] + list(rewards)
    
    # 反向计算 GAE
    for t in reversed(range(len(rewards))):
        # TD 误差：delta = r_t + gamma * V(s_{t+1}) * (1-done_t) - V(s_t)
        delta = rewards[t] + gamma * values[t+1] * (1 - dones[t]) - values[t]
        # GAE 累积：gae_t = delta_t + gamma * lambda_ * (1-done_t) * gae_{t+1}
        gae = delta + gamma * lambda_ * (1 - dones[t]) * gae
        advantages.insert(0, gae)
    
    advantages = torch.FloatTensor(advantages)
    returns = advantages + torch.FloatTensor(values[:-1])  # R = A + V
    
    return advantages, returns


def normalize(tensor, eps=1e-8):
    """对张量做标准化"""
    return (tensor - tensor.mean()) / (tensor.std() + eps)


def compute_discriminator_loss(D, states_exp, actions_exp, states_pol, actions_pol,
                                 label_smoothing=True):
    """
    计算 GAIL 判别器损失（二元交叉熵）
    Args:
        D: 判别器网络
        states_exp, actions_exp: 专家状态-动作对
        states_pol, actions_pol: 策略状态-动作对
        label_smoothing: 是否使用标签平滑（缓解过拟合）
    Returns:
        判别器损失 (标量)
    """
    # 专家样本：标签为 1（真实）
    exp_output = D(states_exp, actions_exp)
    if label_smoothing:
        exp_labels = torch.full_like(exp_output, 0.9)  # 标签平滑
    else:
        exp_labels = torch.ones_like(exp_output)
    loss_exp = F.binary_cross_entropy(exp_output, exp_labels)
    
    # 策略样本：标签为 0（生成/假）
    pol_output = D(states_pol, actions_pol)
    if label_smoothing:
        pol_labels = torch.full_like(pol_output, 0.1)
    else:
        pol_labels = torch.zeros_like(pol_output)
    loss_pol = F.binary_cross_entropy(pol_output, pol_labels)
    
    return loss_exp + loss_pol


def collect_rollouts(env, policy, num_episodes=10, max_steps=1000,
                     reward_func=None, state_normalizer=None):
    """
    收集策略轨迹
    Args:
        env: Gymnasium 环境
        policy: 策略网络
        num_episodes: 收集的回合数
        max_steps: 每回合最大步数
        reward_func: 可选的自定义奖励函数（用于 GAIL 判别器奖励）
        state_normalizer: 可选的归一化器
    Returns:
        rollouts: 包含 states, actions, rewards, log_probs, dones 的字典
    """
    rollouts = {
        'states': [], 'actions': [], 'rewards': [],
        'log_probs': [], 'dones': [], 'values': []
    }
    
    for _ in range(num_episodes):
        state, _ = env.reset()
        if state_normalizer is not None:
            state = state_normalizer.normalize(state)
        
        episode_states, episode_actions = [], []
        episode_rewards, episode_log_probs = [], []
        episode_dones = [], episode_values = []
        
        for step in range(max_steps):
            state_t = torch.FloatTensor(state).unsqueeze(0)
            
            # 获取动作（带随机性）
            action, log_prob, value = policy.get_action(torch.FloatTensor(state).unsqueeze(0))
            action_np = action.squeeze().cpu().numpy() if action.dim() > 1 else action.cpu().numpy()
            
            # 环境一步交互
            next_state, env_reward, terminated, truncated, _ = env.step(action_np)
            done = terminated or truncated
            
            if state_normalizer is not None:
                next_state_norm = state_normalizer.normalize(next_state)
            else:
                next_state_norm = next_state
            
            # 如果提供了 reward_func，用它替换环境奖励（用于 GAIL）
            if reward_func is not None:
                reward = reward_func(
                    torch.FloatTensor(state).unsqueeze(0),
                    torch.FloatTensor(action_np).unsqueeze(0) if action_np.ndim == 1
                    else torch.FloatTensor(action_np).unsqueeze(0)
                ).item()
            else:
                reward = env_reward
            
            episode_states.append(state)
            episode_actions.append(action_np)
            episode_rewards.append(reward)
            episode_log_probs.append(log_prob.item() if log_prob is not None else 0.0)
            episode_dones.append(done)
            episode_values.append(value.item() if value is not None else 0.0)
            
            state = next_state
            if done:
                break
        
        rollouts['states'].extend(episode_states)
        rollouts['actions'].extend(episode_actions)
        rollouts['rewards'].extend(episode_rewards)
        rollouts['log_probs'].extend(episode_log_probs)
        rollouts['dones'].extend(episode_dones)
        rollouts['values'].extend(episode_values)
    
    # 转换为 numpy 数组
    rollouts['states'] = np.array(rollouts['states'])
    rollouts['actions'] = np.array(rollouts['actions'])
    rollouts['rewards'] = np.array(rollouts['rewards'])
    rollouts['log_probs'] = np.array(rollouts['log_probs'])
    rollouts['dones'] = np.array(rollouts['dones'])
    rollouts['values'] = np.array(rollouts['values'])
    
    return rollouts
```

```python
"""
gail/gail.py - GAIL 主训练脚本
基于 PyTorch + Gymnasium，在 MuJoCo HalfCheetah-v5 环境中训练
"""
import os
import time
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
import gymnasium as gym

from networks import ActorCritic, Discriminator
from utils import compute_gae, compute_discriminator_loss, collect_rollouts


def train_gail(env_id="HalfCheetah-v5",
               expert_trajectories=None,
               state_dim=None,
               action_dim=None,
               discriminator_iters=5,
               ppo_epochs=10,
               total_timesteps=1000000,
               rollout_episodes=10,
               batch_size=256,
               disc_lr=3e-4,
               policy_lr=3e-4,
               clip_epsilon=0.2,
               entropy_coef=0.01,
               gamma=0.99,
               lambda_=0.95,
               eval_interval=10000,
               save_interval=50000,
               device='cuda'):
    """
    GAIL 训练主函数
    Args:
        env_id: Gymnasium 环境 ID
        expert_trajectories: 专家轨迹数据 {'states': np.array, 'actions': np.array}
        state_dim: 状态维度（从环境自动获取）
        action_dim: 动作维度（从环境自动获取）
        discriminator_iters: 每回合判别器更新次数
        ppo_epochs: PPO 策略更新轮数
        total_timesteps: 总训练步数
        rollout_episodes: 每回合收集的轨迹数量
        batch_size: 批次大小
        disc_lr: 判别器学习率
        policy_lr: 策略学习率
        clip_epsilon: PPO clip 参数
        entropy_coef: 熵正则化系数
        gamma: 折扣因子
        lambda_: GAE 参数
        eval_interval: 评估间隔（步数）
        save_interval: 模型保存间隔
        device: 计算设备
    """
    device = torch.device(device if torch.cuda.is_available() else 'cpu')
    print(f"Using device: {device}")
    
    # ===== 准备专家数据 =====
    if expert_trajectories is not None:
        exp_states = torch.FloatTensor(expert_trajectories['states']).to(device)
        exp_actions = torch.FloatTensor(expert_trajectories['actions']).to(device)
        print(f"专家数据: {len(exp_states)} 样本")
    else:
        # 如果没有专家数据，使用随机初始化的策略预训练
        print("警告: 未提供专家数据，使用随机策略收集初始轨迹")
        exp_states = torch.zeros(1, state_dim).to(device)
        exp_actions = torch.zeros(1, action_dim).to(device)
    
    # ===== 初始化网络 =====
    policy = ActorCritic(state_dim, action_dim).to(device)
    discriminator = Discriminator(state_dim, action_dim).to(device)
    
    disc_optimizer = optim.Adam(discriminator.parameters(), lr=disc_lr)
    policy_optimizer = optim.Adam(policy.parameters(), lr=policy_lr)
    
    print(f"策略网络参数: {sum(p.numel() for p in policy.parameters()):,}")
    print(f"判别器参数: {sum(p.numel() for p in discriminator.parameters()):,}")
    
    # ===== 训练循环 =====
    total_steps = 0
    iteration = 0
    episode_rewards = []
    
    env = gym.make(env_id)
    
    while total_steps < total_timesteps:
        iteration += 1
        
        # ====== 1. 用当前策略收集轨迹 ======
        rollouts = collect_rollouts(env, policy, num_episodes=rollout_episodes,
                                     state_normalizer=None)
        total_steps += len(rollouts['states'])
        
        # 计算 GAE 优势
        last_value = policy.get_value(torch.FloatTensor(rollouts['states'][-1]).unsqueeze(0).to(device)).item()
        advantages, returns = compute_gae(
            rollouts['rewards'], rollouts['dones'], last_value,
            gamma=gamma, lambda_=lambda_
        )
        advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)
        
        # 转换为张量
        states_t = torch.FloatTensor(rollouts['states']).to(device)
        actions_t = torch.FloatTensor(rollouts['actions']).to(device)
        old_log_probs_t = torch.FloatTensor(rollouts['log_probs']).to(device)
        advantages_t = advantages.to(device)
        returns_t = returns.to(device)
        
        episode_reward = sum(rollouts['rewards'])
        episode_rewards.append(episode_reward)
        
        # ====== 2. 更新判别器 ======
        for _ in range(discriminator_iters):
            # 随机采样专家数据（与策略数据量相同）
            exp_idx = torch.randperm(len(exp_states))[:len(states_t)]
            exp_s = exp_states[exp_idx]
            exp_a = exp_actions[exp_idx]
            
            # 随机采样策略数据
            pol_idx = torch.randperm(len(states_t))[:batch_size]
            pol_s = states_t[pol_idx]
            pol_a = actions_t[pol_idx]
            
            d_loss = compute_discriminator_loss(
                discriminator, exp_s, exp_a, pol_s, pol_a, label_smoothing=True
            )
            
            disc_optimizer.zero_grad()
            d_loss.backward()
            disc_optimizer.step()
        
        # ====== 3. 用判别器奖励更新策略（PPO） ======
        for _ in range(ppo_epochs):
            # 重新计算对数概率
            new_log_probs = policy.get_log_prob(states_t, actions_t)
            
            # PPO clipped surrogate objective
            ratio = torch.exp(new_log_probs - old_log_probs_t.unsqueeze(-1))
            ratio = ratio.squeeze(-1)
            advantages_t_flat = advantages_t
            
            surr1 = ratio * advantages_t_flat
            surr2 = torch.clamp(ratio, 1 - clip_epsilon, 1 + clip_epsilon) * advantages_t_flat
            policy_loss = -torch.min(surr1, surr2).mean()
            
            # 熵正则化（鼓励探索）
            dist = policy.get_distribution(states_t)
            entropy_loss = -entropy_coef * dist.entropy().mean()
            
            # 价值损失
            values_pred = policy.get_value(states_t)
            value_loss = 0.5 * F.mse_loss(values_pred.squeeze(-1), returns_t)
            
            total_loss = policy_loss + entropy_loss + value_loss
            
            policy_optimizer.zero_grad()
            total_loss.backward()
            torch.nn.utils.clip_grad_norm_(policy.parameters(), 0.5)
            policy_optimizer.step()
        
        # ====== 4. 日志与评估 ======
        if total_steps % eval_interval == 0 or total_steps >= total_timesteps:
            avg_reward = np.mean(episode_rewards[-rollout_episodes:])
            print(f"[Iter {iteration}] Steps: {total_steps:7d}/{total_timesteps} | "
                  f"Avg Reward: {avg_reward:8.2f} | "
                  f"D_Loss: {d_loss.item():.4f} | "
                  f"Exp_D: {discriminator(exp_s[:32], exp_a[:32]).mean().item():.3f} | "
                  f"Pol_D: {discriminator(pol_s[:32], pol_a[:32]).mean().item():.3f}")
        
        # ====== 5. 保存模型 ======
        if total_steps % save_interval == 0 or total_steps >= total_timesteps:
            os.makedirs("gail_models", exist_ok=True)
            torch.save({
                'policy': policy.state_dict(),
                'discriminator': discriminator.state_dict(),
                'iteration': iteration,
                'steps': total_steps,
            }, f"gail_models/gail_step{total_steps}.pt")
            print(f"  [保存] gail_models/gail_step{total_steps}.pt")
    
    env.close()
    return policy, discriminator


if __name__ == "__main__":

    parser = argparse.ArgumentParser(description='GAIL 训练脚本')

    parser.add_argument('--total_steps', type=int, default=500000)
    parser.add_argument('--device', type=str, default='cuda')
    args = parser.parse_args()
    
    env = gym.make(args.env)
    state_dim = env.observation_space.shape[0]
    action_dim = env.action_space.shape[0]
    env.close()
    
    # 生成模拟专家数据（实际应用中应使用真实人类示教数据）
    print("生成模拟专家数据...")
    np.random.seed(42)
    expert_states = np.random.randn(5000, state_dim).astype(np.float32)
    expert_actions = np.random.randn(5000, action_dim).astype(np.float32) * 0.5
    expert_trajectories = {'states': expert_states, 'actions': expert_actions}
    
    print(f"环境: {args.env} | 状态维度: {state_dim} | 动作维度: {action_dim}")
    print(f"总训练步数: {args.total_steps}")
    print("=" * 60)
    
    policy, discriminator = train_gail(

        expert_trajectories=expert_trajectories,

        action_dim=action_dim,
        total_timesteps=args.total_steps,
        device=args.device,
    )
    
    print("\n训练完成！")
```

```python
    # 3. 训练 GAIL
    print("\n" + "=" * 60)
    print("第 2 步：GAIL 训练")
    print("=" * 60)
    
    from gail import train_gail
    
    policy, discriminator = train_gail(
        env_id=env_id,
        expert_trajectories=expert_trajectories,
        state_dim=state_dim,
        action_dim=action_dim,
        total_timesteps=200000,
        rollout_episodes=5,
        disc_lr=2e-4,           # 判别器学习率略低
        policy_lr=3e-4,
        discriminator_iters=3,
        ppo_epochs=5,
        entropy_coef=0.001,      # MuJoCo 任务熵系数较小
        eval_interval=10000,
        device='cuda' if torch.cuda.is_available() else 'cpu'
    )
    
    # 4. 最终评估
    print("\n" + "=" * 60)
    print("第 3 步：策略评估")
    print("=" * 60)
    avg_reward = evaluate_policy(env_id, policy, num_episodes=10)
    print(f"GAIL 策略平均 Reward: {avg_reward:.2f}")
    
    # 保存最终模型
    os.makedirs("gail_models", exist_ok=True)
    torch.save({
        'policy': policy.state_dict(),
        'discriminator': discriminator.state_dict(),
    }, "gail_models/gail_halfcheetah_final.pt")
    print("最终模型已保存: gail_models/gail_halfcheetah_final.pt")
```

## 六、参考答案

### 选择题答案

| 题号 | 答案 | 解析 |
|------|------|------|
| 1 | **B** | GAN 的核心是生成器 $G$ 与判别器 $D$ 的对抗博弈：$G$ 试图生成逼真的假数据骗过 $D$，$D$ 试图准确区分真实数据与生成数据。两者交替优化，最终达到纳什均衡。 |
| 2 | **C** | 模式崩溃指生成器找到"骗过"判别器的捷径——只生成某个单一模式的数据（如手写数字只生成数字"1"），而不是学习完整的数据分布。根源是判别器过强导致生成器缺乏探索所有模式的压力。 |
| 3 | **C** | GAIL 通过对抗训练让策略学习隐式奖励函数，而非逐状态记忆动作。这使策略能够在未见过的状态下通过奖励泛化做出合理决策，从根本上避免了 BC 的分布偏移（compounding error）问题。 |
| 4 | **B** | AIRL 的核心创新是将 GAIL 的隐式奖励分离为可解释的显式奖励函数 $r_\theta(s,a)$ 和势函数 $\phi(s)$，即 $f = r + \gamma\phi(s') - \phi(s)$。这使得学到的奖励可解释、可迁移，而 GAIL 的奖励是隐式由判别器定义的。 |
| 5 | **B** | Dropout 在判别器中有两个核心作用：1）作为正则化减少过拟合（随机丢弃神经元，降低对特定专家样本的记忆）；2）MC Dropout 通过多次采样估计奖励方差，用于检测策略是否在"安全区域"（低不确定性）。 |
