## 1. IRL 概述

### 1.1 什么是 IRL？

**逆向强化学习（Inverse Reinforcement Learning, IRL）** 是强化学习的"逆向"问题。

| 正向强化学习 (RL) | 逆向强化学习 (IRL) |
|------------------|-------------------|
| **已知**：奖励函数 $r(s,a)$ | **已知**：专家轨迹 $\{(s_0,a_0), (s_1,a_1), ...\}$ |
| **求解**：最优策略 $\pi^*$ | **求解**：奖励函数 $r(s,a)$ |
| 回答：*怎么做？* | 回答：*为什么这样做？* |

> 核心思想：观察专家行为，反推其背后的奖励逻辑。

### 1.2 与 RL / 模仿学习的关系

```
┌─────────────────────────────────────────────────────────────┐
│                        模仿学习 (IL)                          │
│  直接学习：专家状态-动作对 → 策略 π(a|s)                       │
│  缺点：只能模仿"怎么做"，无法理解"为什么"                       │
└─────────────────────────────────────────────────────────────┘
                              ↕
┌─────────────────────────────────────────────────────────────┐
│                      逆向强化学习 (IRL)                        │
│  推断奖励：专家轨迹 → 奖励函数 r(s,a) → 用 RL 求解策略          │
│  优点：学到的奖励函数可迁移、可解释                              │
└─────────────────────────────────────────────────────────────┘
                              ↕
┌─────────────────────────────────────────────────────────────┐
│                      正向强化学习 (RL)                         │
│  优化策略：给定奖励函数 → 找到最优策略                          │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 应用场景

| 场景 | 说明 |
|------|------|
| **机器人示教** | 观察人类操作，推断奖励函数，让机器人自主学习技能 |
| **自动驾驶** | 从人类驾驶轨迹学习奖励（安全、舒适、效率） |
| **游戏 AI** | 从职业选手行为学习 reward shaping |
| **医疗决策** | 从专家医生行为推断治疗目标 |

## 2. IRL 核心思想

### 2.1 问题定义

给定：
- 专家轨迹集合：$\tau_E = \{(\tau_1, \tau_2, ..., \tau_N)\}$
- 每个轨迹：$\tau = (s_0, a_0, s_1, a_1, ..., s_T)$

求解：
- 奖励函数 $r_\theta(s, a)$，使得专家轨迹的收益高于其他轨迹

### 2.2 关键洞察

**专家轨迹之所以最优，是因为它遵循了潜在的奖励函数。**

举例：观察人类司机在路口减速 → 推断"靠近障碍物 = 负奖励" → 学到安全奖励函数

### 2.3 形式化定义

对于参数化的奖励函数 $r_\theta(s,a)$，IRL 寻找 $\theta$ 使得：

$$P(\tau_E | \theta) = \frac{\exp(\frac{1}{Z} R(\tau_E))}{\sum_{\tau} \exp(\frac{1}{Z} R(\tau))}$$

其中 $R(\tau) = \sum_{t=0}^{T} r(s_t, a_t)$ 是轨迹总奖励，$Z$ 是配分函数。

## 3. 算法实现

### 3.1 MaxEnt IRL（最大熵逆向强化学习）

#### 3.1.1 核心思想

**最大熵原理**：在所有符合专家行为的奖励函数中，选择熵最大（最不确定）的那个，避免过度自信。

#### 3.1.2 softmax 公式

$$P_\theta(\tau) = \frac{\exp(\theta \cdot f(\tau))}{\sum_{\tau'} \exp(\theta \cdot f(\tau'))}$$

其中 $f(\tau)$ 是轨迹特征（如状态访问频率），$\theta$ 是奖励权重。

#### 3.1.3 算法步骤

```
输入：专家轨迹 {τ_E}
输出：奖励函数权重 θ

1. 初始化 θ
2. 循环直到收敛：
   a. 计算特征期望：
      - μ_E = 所有专家轨迹的特征均值
      - μ_π = 当前策略 π_θ 下轨迹的特征均值
   b. 计算梯度：
      ▽_θ J = μ_E - μ_π
   c. 更新权重：
      θ ← θ + α ▽_θ J
3. 返回奖励函数 r_θ(s,a) = θ · φ(s,a)
```

#### 3.1.4 特征设计

常用的轨迹特征 $\phi(\tau)$：

| 特征 | 含义 |
|------|------|
| 状态访问频率 | 机器人去往各状态的次数 |
| 动作分布 | 各动作执行频率 |
| 目标达成率 | 达到目标状态的次数 |

### 3.2 IRL 神经网络

#### 3.2.1 奖励网络

用神经网络直接从状态-动作对预测奖励：

$$r_\theta(s, a) = \text{NeuralNet}(s, a; \theta)$$

#### 3.2.2 对抗式 IRL（GAN-IRL / GAIL）

类似 GAN 的框架：

```
┌──────────────┐     ┌──────────────┐
│  奖励网络     │ ←→  │   判别器      │
│  r_θ(s,a)    │     │  D_ω(τ)      │
└──────────────┘     └──────────────┘
      ↓                    ↓
  估计优势             判断轨迹来源
      ↓                    ↓
  更新策略 π         判断专家 vs 学徒
```

- **判别器**：区分专家轨迹 vs 学习策略生成的轨迹
- **奖励信号**：来自判别器的 log-likelihood

#### 3.2.3 GAIL 判别器公式

$$r_t = -\log(1 - D_\omega(s_t, a_t))$$

判别器输出轨迹来自专家的概率。

### 4.1 MaxEnt IRL 实现

```python
"""
MaxEnt IRL (最大熵逆向强化学习)
基于 Ziebart et al. 2008 "Maximum Entropy Inverse Reinforcement Learning"
"""

import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
from typing import List, Tuple, Dict

# ============================================================
# 1. 奖励网络定义
# ============================================================

class RewardNetwork(nn.Module):
    """用神经网络从状态-动作对预测奖励"""
    
    def __init__(self, state_dim: int, action_dim: int, hidden_dim: int = 128):
        super().__init__()
        # 特征提取
        self.state_net = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )
        self.action_net = nn.Sequential(
            nn.Linear(action_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )
        # 奖励输出
        self.reward_head = nn.Sequential(
            nn.Linear(hidden_dim * 2, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)
        )
    
    def forward(self, state: torch.Tensor, action: torch.Tensor) -> torch.Tensor:
        """
        Args:
            state: (batch, state_dim)
            action: (batch, action_dim)
        Returns:
            reward: (batch, 1)
        """
        s_feat = self.state_net(state)
        a_feat = self.action_net(action)
        combined = torch.cat([s_feat, a_feat], dim=-1)
        return self.reward_head(combined)


class FeatureExtractor(nn.Module):
    """轨迹特征提取器：统计状态-动作访问频率"""
    
    def __init__(self, state_dim: int, action_dim: int, n_bins: int = 10):
        super().__init__()
        self.n_bins = n_bins
        
        # 将连续状态离散化为 n_bins 个区间
        # 实际中可用更复杂的方法如 VAE
        self.state_bins = nn.Parameter(
            torch.linspace(-5, 5, n_bins + 1), requires_grad=False
        )
        
    def compute_features(self, states: torch.Tensor, actions: torch.Tensor) -> torch.Tensor:
        """
        计算状态-动作对的特征（直方图统计）
        Args:
            states: (traj_len, state_dim)
            actions: (traj_len, action_dim)
        Returns:
            features: (feature_dim,)
        """
        # 简化：用状态均值 + 动作均值 + 协方差作为特征
        state_mean = states.mean(dim=0)          # (state_dim,)
        action_mean = actions.mean(dim=0)         # (action_dim,)
        state_std = states.std(dim=0)             # (state_dim,)
        
        features = torch.cat([state_mean, action_mean, state_std])
        return features


# ============================================================
# 2. 软策略迭代（用于计算状态价值）
# ============================================================

class SoftPolicyIteration:
    """最大熵 IRL 中的软策略迭代"""
    
    def __init__(self, reward_net: RewardNetwork, state_dim: int, 
                 action_dim: int, gamma: float = 0.99, beta: float = 1.0):
        self.reward_net = reward_net
        self.gamma = gamma
        self.beta = beta  # 温度参数
        self.state_dim = state_dim
        self.action_dim = action_dim
        
        # 简化的状态空间（网格世界）
        self.n_states = 100
        self.n_actions = 4
        self._init_transitions()
    
    def _init_transitions(self):
        """初始化转移矩阵（网格世界）"""
        # P[s, a] = [(s', prob), ...]
        self.transitions = {}
        for s in range(self.n_states):
            self.transitions[s] = {}
            for a in range(self.n_actions):
                # 简化的转移：每个动作有概率转向邻居状态
                next_s = max(0, min(self.n_states - 1, s + (a - 1)))
                self.transitions[s][a] = [(next_s, 0.9), (s, 0.1)]
    
    def soft_value_iteration(self, states: torch.Tensor) -> torch.Tensor:
        """
        软价值迭代，计算 V^π(s)
        V^π(s) = sum_a π(a|s) * [r(s,a) + gamma * sum_{s'} P(s'|s,a) V^π(s')]
        """
        with torch.no_grad():
            # 状态张量
            s_tensor = torch.FloatTensor(states).unsqueeze(0)  # (1, n_states, dim)
            
            # 计算所有状态-动作对的奖励
            all_rewards = []
            for a in range(self.n_actions):
                a_tensor = torch.full((states.shape[0], 1), a, dtype=torch.float)
                r = self.reward_net(states, a_tensor)  # (n_states, 1)
                all_rewards.append(r)
            rewards = torch.stack(all_rewards, dim=1).squeeze(-1)  # (n_states, n_actions)
            
            # 简化的价值迭代（固定迭代次数）
            V = torch.zeros(states.shape[0])
            for _ in range(50):
                new_V = torch.zeros_like(V)
                for s in range(states.shape[0]):
                    # 计算每个动作的价值
                    action_values = []
                    for a in range(self.n_actions):
                        r_sa = rewards[s, a].item()
                        # 转移期望
                        v_next = 0.0
                        for s_next, prob in self.transitions[s % self.n_states][a]:
                            s_idx = min(s_next, states.shape[0] - 1)
                            v_next += prob * V[s_idx].item()
                        action_values.append(r_sa + self.gamma * v_next)
                    
                    # softmax 策略
                    action_values = torch.tensor(action_values) / self.beta
                    log_pi = action_values - torch.logsumexp(action_values, dim=0)
                    new_V[s] = torch.sum(torch.exp(log_pi) * action_values).item()
                
                V = new_V
            
            return V


# ============================================================
# 3. MaxEnt IRL 主算法
# ============================================================

class MaxEntIRL:
    """
    最大熵逆向强化学习主算法
    
    核心思想：
    1. 初始化随机奖励网络
    2. 用奖励网络运行软策略迭代
    3. 计算专家轨迹特征期望 vs 策略轨迹特征期望
    4. 梯度上升更新奖励网络参数
    """
    
    def __init__(self, state_dim: int, action_dim: int, 
                 hidden_dim: int = 128, gamma: float = 0.99, beta: float = 1.0,
                 lr: float = 0.001):
        self.state_dim = state_dim
        self.action_dim = action_dim
        self.gamma = gamma
        self.beta = beta
        
        # 奖励网络
        self.reward_net = RewardNetwork(state_dim, action_dim, hidden_dim)
        self.optimizer = optim.Adam(self.reward_net.parameters(), lr=lr)
        
        # 特征提取器
        self.feature_extractor = FeatureExtractor(state_dim, action_dim)
        
        # 软策略迭代器
        self.policy_iter = SoftPolicyIteration(
            self.reward_net, state_dim, action_dim, gamma, beta
        )
    
    def get_trajectory_features(self, states: torch.Tensor, 
                                actions: torch.Tensor) -> torch.Tensor:
        """计算轨迹特征"""
        return self.feature_extractor.compute_features(states, actions)
    
    def sample_trajectories(self, n_trajectories: int, 
                            trajectory_length: int) -> List[Tuple[torch.Tensor, torch.Tensor]]:
        """从当前策略采样轨迹（简化版本：随机采样）"""
        trajectories = []
        for _ in range(n_trajectories):
            # 随机生成轨迹（实际应用中应该用策略采样）
            states = torch.randn(trajectory_length, self.state_dim)
            actions = torch.randint(0, self.action_dim, (trajectory_length,))
            trajectories.append((states, actions))
        return trajectories
    
    def compute_gradient(self, expert_trajectories: List[Tuple[torch.Tensor, torch.Tensor]],
                         policy_trajectories: List[Tuple[torch.Tensor, torch.Tensor]]) -> torch.Tensor:
        """
        计算 IRL 梯度
        
        ▽_θ J = (1/N) Σ μ_Expert(τ) - (1/M) Σ μ_Policy(τ)
        
        其中 μ(τ) 是轨迹特征期望
        """
        # 专家轨迹特征均值
        expert_features = []
        for states, actions in expert_trajectories:
            features = self.get_trajectory_features(states, actions)
            expert_features.append(features)
        mu_expert = torch.stack(expert_features).mean(dim=0)
        
        # 策略轨迹特征均值
        policy_features = []
        for states, actions in policy_trajectories:
            features = self.get_trajectory_features(states, actions)
            policy_features.append(features)
        mu_policy = torch.stack(policy_features).mean(dim=0)
        
        # 梯度 = 专家期望 - 策略期望
        gradient = mu_expert - mu_policy
        return gradient
    
    def train_step(self, expert_trajectories: List[Tuple[torch.Tensor, torch.Tensor]],
                   n_policy_trajectories: int = 10,
                   trajectory_length: int = 20) -> Dict[str, float]:
        """单步训练"""
        # 1. 采样策略轨迹
        policy_trajectories = self.sample_trajectories(
            n_policy_trajectories, trajectory_length
        )
        
        # 2. 计算梯度
        gradient = self.compute_gradient(expert_trajectories, policy_trajectories)
        
        # 3. 更新奖励网络（梯度上升）
        self.optimizer.zero_grad()
        
        # 将梯度应用到奖励网络的最后一层
        # 简化：直接用特征梯度更新
        for param in self.reward_net.reward_head.parameters():
            if param.grad is None:
                param.grad = torch.zeros_like(param)
        
        # 手动应用梯度（简化版本）
        loss = -torch.norm(gradient)  # 最大化 expert - policy
        loss.backward()
        self.optimizer.step()
        
        return {
            'gradient_norm': torch.norm(gradient).item(),
            'loss': loss.item()
        }
    
    def train(self, expert_trajectories: List[Tuple[torch.Tensor, torch.Tensor]],
              n_epochs: int = 100, 
              n_policy_trajectories: int = 10,
              trajectory_length: int = 20,
              verbose: bool = True) -> List[Dict[str, float]]:
        """完整训练流程"""
        history = []
        
        for epoch in range(n_epochs):
            metrics = self.train_step(
                expert_trajectories,
                n_policy_trajectories,
                trajectory_length
            )
            history.append(metrics)
            
            if verbose and (epoch + 1) % 20 == 0:
                print(f"Epoch {epoch+1}/{n_epochs} | "
                      f"Grad Norm: {metrics['gradient_norm']:.4f} | "
                      f"Loss: {metrics['loss']:.4f}")
        
        return history
```

### 4.2 GAIL 风格 IRL 实现

```python
"""
GAIL 风格的 IRL 实现（简化版）
使用判别器区分专家轨迹 vs 策略轨迹
"""

import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
import numpy as np
from typing import List, Tuple

# ============================================================
# 1. 轨迹判别器
# ============================================================

class TrajectoryDiscriminator(nn.Module):
    """判别轨迹是来自专家还是策略"""
    
    def __init__(self, state_dim: int, action_dim: int, hidden_dim: int = 128):
        super().__init__()
        
        # 输入：状态-动作序列（展平）
        input_dim = state_dim + action_dim  # 每个时间步
        
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1),
            nn.Sigmoid()  # 输出 [0, 1] 概率
        )
    
    def forward(self, states: torch.Tensor, actions: torch.Tensor) -> torch.Tensor:
        """
        Args:
            states: (batch, seq_len, state_dim)
            actions: (batch, seq_len, action_dim)
        Returns:
            prob: (batch, 1) 专家概率
        """
        # 拼接状态和动作
        combined = torch.cat([states, actions], dim=-1)  # (batch, seq_len, state_dim + action_dim)
        
        # 逐时间步判别后平均
        scores = []
        for t in range(combined.shape[1]):
            step_input = combined[:, t, :]  # (batch, input_dim)
            score = self.net(step_input)     # (batch, 1)
            scores.append(score)
        
        # 平均所有时间步
        avg_score = torch.stack(scores, dim=1).mean(dim=1)  # (batch, 1)
        return avg_score


# ============================================================
# 2. 简化的策略网络（用于生成轨迹）
# ============================================================

class SimplePolicy(nn.Module):
    """简化的随机策略"""
    
    def __init__(self, state_dim: int, action_dim: int, hidden_dim: int = 128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim)
        )
    
    def forward(self, state: torch.Tensor) -> torch.Tensor:
        """返回动作 logits"""
        return self.net(state)
    
    def sample_action(self, state: torch.Tensor) -> Tuple[int, torch.Tensor]:
        """采样动作"""
        logits = self.forward(state)
        probs = F.softmax(logits, dim=-1)
        action = torch.multinomial(probs, 1).item()
        return action, probs[0, action]


# ============================================================
# 3. GAIL IRL
# ============================================================

class GAIL IRL:
    """
    GAIL 风格的 IRL
    
    训练流程：
    1. 专家轨迹训练判别器
    2. 判别器提供奖励信号
    3. 用奖励信号更新策略
    """
    
    def __init__(self, state_dim: int, action_dim: int, 
                 hidden_dim: int = 128, lr: float = 0.001):
        self.state_dim = state_dim
        self.action_dim = action_dim
        
        # 判别器
        self.discriminator = TrajectoryDiscriminator(
            state_dim, action_dim, hidden_dim
        )
        self.disc_opt = optim.Adam(
            self.discriminator.parameters(), lr=lr
        )
        
        # 策略
        self.policy = SimplePolicy(state_dim, action_dim, hidden_dim)
        self.policy_opt = optim.Adam(
            self.policy.parameters(), lr=lr
        )
    
    def discriminator_loss(self, expert_states: torch.Tensor,
                          expert_actions: torch.Tensor,
                          policy_states: torch.Tensor,
                          policy_actions: torch.Tensor) -> torch.Tensor:
        """
        判别器损失：让专家轨迹输出 1，策略轨迹输出 0
        """
        # 专家轨迹：label = 1
        expert_prob = self.discriminator(expert_states, expert_actions)
        expert_loss = F.binary_cross_entropy(
            expert_prob, torch.ones_like(expert_prob)
        )
        
        # 策略轨迹：label = 0
        policy_prob = self.discriminator(policy_states, policy_actions)
        policy_loss = F.binary_cross_entropy(
            policy_prob, torch.zeros_like(policy_prob)
        )
        
        return expert_loss + policy_loss
    
    def get_irl_reward(self, states: torch.Tensor, actions: torch.Tensor) -> torch.Tensor:
        """
        从判别器获取 IRL 奖励
        r = -log(1 - D(s,a))
        """
        d_prob = self.discriminator(states, actions)
        reward = -torch.log(1 - d_prob + 1e-8)
        return reward
    
    def update_discriminator(self, expert_trajectories: List[Tuple[torch.Tensor, torch.Tensor]],
                             policy_trajectories: List[Tuple[torch.Tensor, torch.Tensor]]):
        """更新判别器"""
        self.disc_opt.zero_grad()
        
        # 专家轨迹
        expert_states = torch.stack([t[0] for t in expert_trajectories])
        expert_actions = torch.stack([t[1] for t in expert_trajectories])
        
        # 策略轨迹
        policy_states = torch.stack([t[0] for t in policy_trajectories])
        policy_actions = torch.stack([t[1] for t in policy_trajectories])
        
        loss = self.discriminator_loss(
            expert_states, expert_actions,
            policy_states, policy_actions
        )
        
        loss.backward()
        self.disc_opt.step()
        
        return loss.item()
    
    def update_policy(self, trajectories: List[Tuple[torch.Tensor, torch.Tensor]]):
        """用 IRL 奖励更新策略（简化的策略梯度）"""
        self.policy_opt.zero_grad()
        
        total_loss = 0.0
        for states, actions in trajectories:
            # 获取 IRL 奖励
            rewards = self.get_irl_reward(states, actions)
            
            # 简化的策略梯度：最大化 IRL 奖励
            # 实际应用中应该用 PPO/TRPO 等算法
            loss = -rewards.mean()
            loss.backward()
        
        self.policy_opt.step()
    
    def train_step(self, expert_trajectories: List[Tuple[torch.Tensor, torch.Tensor]],
                   policy_trajectories: List[Tuple[torch.Tensor, torch.Tensor]],
                   n_disc_updates: int = 1) -> dict:
        """单步训练"""
        disc_losses = []
        for _ in range(n_disc_updates):
            disc_loss = self.update_discriminator(
                expert_trajectories, policy_trajectories
            )
            disc_losses.append(disc_loss)
        
        self.update_policy(policy_trajectories)
        
        return {
            'discriminator_loss': np.mean(disc_losses)
        }
    
    def train(self, expert_trajectories: List[Tuple[torch.Tensor, torch.Tensor]],
              n_epochs: int = 100,
              trajectories_per_epoch: int = 10,
              trajectory_length: int = 20,
              verbose: bool = True):
        """训练流程"""
        for epoch in range(n_epochs):
            # 生成策略轨迹
            policy_trajectories = []
            for _ in range(trajectories_per_epoch):
                states_list = []
                actions_list = []
                s = torch.randn(self.state_dim)  # 初始状态
                
                for t in range(trajectory_length):
                    a, _ = self.policy.sample_action(s)
                    states_list.append(s)
                    actions_list.append(torch.tensor([a]))
                    # 简化的状态转移
                    s = s + torch.randn(self.state_dim) * 0.1
                
                states = torch.stack(states_list)
                actions = torch.stack(actions_list).float()
                policy_trajectories.append((states, actions))
            
            # 训练
            metrics = self.train_step(expert_trajectories, policy_trajectories)
            
            if verbose and (epoch + 1) % 20 == 0:
                print(f"Epoch {epoch+1}/{n_epochs} | Disc Loss: {metrics['discriminator_loss']:.4f}")
```

```
场景：让机器人学会折叠衣服

1. 人类演示：操作员演示 50 次折叠动作
```

## 6. GAIL vs IRL 对比

| 维度 | MaxEnt IRL | GAIL (GAN-IRL) |
|------|-----------|----------------|
| **核心思想** | 最大熵分布匹配 | 对抗式轨迹分布匹配 |
| **输出** | 显式奖励函数 | 隐式奖励（判别器信号） |
| **可解释性** | 高（可直接查看奖励） | 低（端到端学习） |
| **计算复杂度** | 高（需计算配分函数） | 中（判别器训练） |
| **样本效率** | 低 | 高 |
| **收敛稳定性** | 较好 | 依赖 GAN 训练技巧 |
| **适用场景** | 需要可解释奖励 | 需要端到端策略 |

### 6.1 何时选 IRL vs GAIL？

```
选择 MaxEnt IRL：
├── 需要解释学到的行为
├── 奖励函数需要迁移到新任务
└── 样本充足，计算资源充裕

选择 GAIL：
├── 追求端到端策略性能
├── 样本有限，需要高效利用
└── 不需要显式奖励函数
```

## 7. 练习题

### 7.1 选择题

!!! question "Q1: IRL 的核心问题是什么？"
    A. 给定奖励函数，求最优策略  
    B. 给定专家轨迹，推断奖励函数  
    C. 直接模仿专家动作  
    D. 最大化折扣回报

??? check "答案"
    **B**。IRL 的核心是逆向问题：从专家行为反推奖励函数。

!!! question "Q2: MaxEnt IRL 中的"最大熵"指的是什么？"
    A. 策略的熵最大  
    B. 状态分布的熵最大  
    C. 在符合专家行为的分布中，选择熵最大的那个  
    D. 奖励函数的熵最大

??? check "答案"
    **C**。最大熵原理确保在所有符合专家行为的分布中选择最不确定的，避免过度假设。

!!! question "Q3: GAIL 中判别器的作用是什么？"
    A. 生成新轨迹  
    B. 区分专家轨迹和策略轨迹，提供奖励信号  
    C. 估计价值函数  
    D. 优化策略参数

??? check "答案"
    **B**。判别器区分轨迹来源，其输出 log-likelihood 作为 IRL 奖励。

### 7.2 简答题

!!! question "Q4: 说明 IRL 与行为克隆（BC）的区别。"
??? check "答案"
    **行为克隆（BC）**：
    - 直接监督学习：$\min_\theta \sum_{(s,a) \in \tau_E} \ell(\pi_\theta(s), a)$
    - 缺点：只能模仿"怎么做"，分布偏移会导致错误累积
    
    **IRL**：
    - 先推断奖励函数，再用 RL 学习策略
    - 优点：学到的奖励函数可迁移、可解释，能理解"为什么"

!!! question "Q5: 为什么 IRL 比正向 RL 更难？"
??? check "答案"
    1. **多解性**：同一专家行为可能由多个不同的奖励函数产生
    2. **计算复杂度**：需要计算配分函数（路径积分），状态空间大时难以精确计算
    3. **循环依赖**：推断奖励需要策略，策略学习又依赖奖励

### 7.3 编程题

!!! question "Q6: 修改上面的 MaxEnt IRL 代码，实现自定义的特征函数来区分不同类型的专家行为。"
??? check "参考答案"
    ```python
    # 参考实现：自定义特征函数
    
    class CustomFeatureExtractor(nn.Module):
        """自定义特征提取器"""
        
        def __init__(self, state_dim: int, action_dim: int):
            super().__init__()
            self.state_dim = state_dim
            self.action_dim = action_dim
            
            # 可学习的特征权重
            self.feature_weights = nn.Parameter(torch.ones(state_dim + action_dim))
        
        def compute_features(self, states: torch.Tensor, 
                            actions: torch.Tensor) -> torch.Tensor:
            """
            自定义特征：
            1. 状态统计特征（均值、标准差、极值）
            2. 动作统计特征
            3. 状态变化趋势
            """
            features = []
            
            # 状态均值
            state_mean = states.mean(dim=0)
            features.append(state_mean)
            
            # 状态标准差
            state_std = states.std(dim=0)
            features.append(state_std)
            
            # 动作均值
            action_mean = actions.mean(dim=0).float()
            features.append(action_mean)
            
            # 状态变化（一阶差分）
            if len(states) > 1:
                state_diff = torch.diff(states, dim=0).mean(dim=0)
            else:
                state_diff = torch.zeros(self.state_dim)
            features.append(state_diff)
            
            # 拼接所有特征
            all_features = torch.cat(features)
            
            # 加权
            weighted_features = all_features * self.feature_weights
            
            return weighted_features
    ```

## 8. 小结

| 概念 | 关键点 |
|------|--------|
| **IRL 定义** | 从专家轨迹推断潜在奖励函数 |
| **MaxEnt IRL** | 最大熵原理，避免过度假设 |
| **GAIL** | 对抗式 IRL，判别器提供奖励信号 |
| **应用** | 机器人示教、自动驾驶、游戏 AI |
| **优缺点** | IRL 可解释但计算复杂；GAIL 高效但黑盒 |
