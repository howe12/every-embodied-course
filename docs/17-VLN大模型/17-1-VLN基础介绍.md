## 1. 什么是视觉语言导航（VLN）？

### 1.1 定义——让机器"看懂"指令并行动

想象一下这样的场景：你走进一家陌生的博物馆，想找某个特定的展厅。你可能会问工作人员："请问中国瓷器展厅怎么走？"工作人员会告诉你："从这边直走，经过青铜器展厅后右转，走廊尽头就是。"你根据这段描述，一边观察周围环境，一边按照指示行动，最终到达目的地。

这个对人类来说稀松平常的过程，对机器人来说却是一个巨大的挑战。**视觉语言导航（Vision-Language Navigation，简称VLN）**正是要解决这个问题：让智能体根据自然语言指令，在未知的环境中自主导航到目标位置。

从技术角度定义：

**VLN = Vision（视觉）+ Language（语言）+ Navigation（导航）**

这是一个典型的多模态任务，需要智能体同时理解两种截然不同的信息：
- **视觉信息**：智能体通过摄像头看到的周围环境
- **语言信息**：人类给出的导航指令

然后在"理解"的基础上做出决策——下一步应该往哪里走。

### 1.2 任务示例——体验VLN的实际挑战

让我们看一个具体的VLN任务示例：

**人类指令**：
> "Walk past the kitchen and turn left into the bedroom. Go straight until you see a bed on your right, then stop."

（走过厨房，左转进入卧室。一直走到你看到右边有一张床，然后停下。）

这是一个典型的VLN任务指令。智能体需要：
1. 理解"厨房"、"卧室"、"床"这些语义概念
2. 理解"走过"、"左转"、"直走"这些动作指令
3. 将语言描述与视觉观察对应起来
4. 在适当的时候执行"停止"动作

这个过程涉及**跨模态理解**——你得知道语言描述的"厨房"在视觉上长什么样，也得能从视觉中判断"这是厨房"。

### 1.3 与传统导航的本质区别

很多人可能会问：这不就是导航吗？现在的GPS导航不是很成熟了吗？

事实上，VLN与传统的机器人导航有本质区别。让我们通过一个对比表格来理解：

| 维度 | 传统导航 | VLN（视觉语言导航） |
|------|----------|---------------------|
| **地图** | 已知地图 | 未知地图 |
| **目标表示** | GPS坐标、路标 | 自然语言描述 |
| **感知方式** | GPS、激光雷达 | 视觉摄像头 |
| **环境理解** | 几何层面 | 语义层面 |
| **典型应用** | 车载GPS、机器人定位 | 室内导航、机器人伴侣 |

**传统导航的局限**：

想象一下，你告诉传统导航系统："带我去最近的中国餐厅"。它会怎么做？
1. 你需要在手机上输入"中国餐厅"
2. 系统通过GPS和地图数据找到餐厅的精确坐标
3. 然后规划路线

这个过程有几个问题：
- 你得知道餐厅的名字或者在地图上标注它
- 如果你不知道餐厅名字，只是想吃"宫保鸡丁"，传统导航就无法处理
- 在室内环境中，GPS信号往往不可用

**VLN的优势**：

VLN可以处理更灵活、更自然的指令：
- "带我去那个有大幅海报的咖啡厅"
- "跟着蓝色指示牌走"
- "去那个有很多植物的房间"

这种能力对于家庭机器人、服务机器人来说至关重要。毕竟，我们不会给机器人输入精确的坐标——我们只会像指挥另一个人类一样，用自然语言下达指令。

![传统导航 vs VLN](./images/traditional_vs_vln.jpg)
*图注：传统导航依赖已知地图和GPS坐标，VLN依赖视觉感知和自然语言理解*

## 2. VLN核心要素

### 2.1 三大输入——智能体需要什么信息？

VLN任务中，智能体需要同时处理三类输入信息。这就像人类导航时需要同时运用多种感官和知识：

#### （1）视觉输入——智能体的"眼睛"

VLN中的视觉输入主要包括：
- **RGB图像**：彩色图像，提供环境的视觉外观
- **深度图**：每个像素点到摄像头的距离，帮助理解空间结构
- **有时还包括**：全景图像（360度视角）

这些视觉信息让智能体能够"看到"周围环境。在连续VLN（VLN-CE）任务中，智能体通常每一步都能获取一个第一人称视角的RGB-D图像，分辨率约为224×224，视野角度约90度。

**生活类比**：这就像你走进一个陌生的博物馆，你会用眼睛观察周围的画作、雕塑、指示牌，用余光注意脚下的地面和周围的墙壁。视觉信息是你导航的基础。

#### （2）语言输入——智能体的"耳朵"

语言输入是VLN的核心特色。它是一段自然语言描述，告诉智能体要做什么：

**分步导航指令**（Step-by-step）：
> "Go down the stairs, turn left at the picture and enter the bedroom."
> （下楼，在画那里左转，进入卧室。）

**过程导向指令**（Process-oriented）：
> "Standing on the rug, turn right and face the wooden cabinet. Walk past the cabinet, keeping it on your left, and enter the hallway."
> （站在地毯上，右转面向木柜。走过柜子，保持它在你的左手边，然后进入走廊。）

这些指令往往包含：
- **空间关系**："左转"、"直走"、"经过"
- **语义地标**："厨房"、"卧室"、"画"
- **目标描述**："看到床就停下"

#### （3）状态输入——智能体的"记忆"

智能体还需要知道自己的"状态"，这包括：
- **当前位置**：智能体当前在环境中的位置
- **历史轨迹**：从起点到当前位置走过的路径
- **当前朝向**：面向哪个方向

这些状态信息帮助智能体理解"我已经走了多远"、"我现在在哪里"。

**生活类比**：这就像你在一个迷宫里导航，你不仅要看前面，还要记住自己是怎么走过來的，哪些路走过了、哪些没有。状态信息就是你的"空间记忆"。

### 2.2 输出——智能体需要做什么决定？

在理解了三类输入之后，VLN智能体需要输出动作决策。根据任务设定不同，动作空间也有区别：

#### 离散VLN的动作空间

在Matterport3D等离散环境中，智能体被限制在一个预定义的"导航图"上。每个位置是一个"节点"，节点之间有连边表示可通行。

**动作选择**：从当前节点的所有相邻节点中选择一个作为下一个目的地

#### 连续VLN的动作空间

在VLN-CE等连续环境中，智能体可以在空间中自由移动：

| 动作 | 描述 |
|------|------|
| **MoveForward** | 向前移动0.25米 |
| **TurnLeft** | 向左旋转15度 |
| **TurnRight** | 向右旋转15度 |
| **Stop** | 停止（表示任务完成） |

**生活类比**：这就像你在用一个遥控机器人。你只能发出简单的指令："往前走"、"往左转"、"往右转"、"停下"。机器人需要自己决定什么时候该执行什么动作。

## 3. VLN数据集

### 3.1 主流数据集——VLN研究的基石

数据是VLN研究的基础。一个好的数据集需要包含：
- 逼真的3D环境
- 多样化的导航指令
- 明确的成功标准

下面介绍几个主流的VLN数据集：

#### （1）R2R数据集——VLN的"Hello World"

**R2R**（Room-to-Room）是VLN领域最经典的数据集，由Anderson等人于2018年提出。它基于**Matterport3D**数据集的环境，该数据集包含90个真实的室内场景（房屋、办公室、商店等）的3D扫描。

**数据集规模**：
- 路径数量：约6,000+条
- 指令数量：约21,000条（每条路径3-5条指令）
- 平均路径长度：约10米

**特点**：
- 指令为"分步导航"类型
- 环境为离散导航图（预定义连通节点）
- 被广泛用作VLN的基准数据集

![R2R数据集示例](./images/r2r_dataset.jpg)
*图注：R2R数据集示例。左图为Matterport3D的3D环境，右图为路径和指令*

#### （2）REVERIE数据集——目标指派任务

**REVERIE**（REmote Embodied Vision-to-Expressions in Real Indoor Environments）是一个更具挑战性的VLN变体。

**任务特点**：
- 不仅要导航到目标房间
- 还要在房间中找到指定的物体并"指派"（point out）
- 指令更简洁，通常只给出目标物体的描述

**数据集规模**：
- 约10,000条导航+指派指令
- 涵盖多种室内场景

#### （3）R2R-CE数据集——连续环境VLN

**R2R-CE**（R2R-Continuous Environment）是将R2R扩展到连续环境的数据集。

**核心创新**：
- 使用**Habitat模拟器**加载Matterport3D场景
- 智能体可以在空间中连续移动（不再限制在预定义节点）
- 动作空间从"选择节点"变为"低级动作原语"（前进、转向等）

**数据集规模**：
- 约4,475条连续轨迹
- 约13,425条指令
- 平均路径长度：约9.89米

#### （4）RxR-CE数据集——多语言VLN

**RxR**（Room-across-Room）是另一个重要的VLN数据集：

**特点**：
- **多语言**：支持英语、印地语、泰卢固语
- **过程导向**：指令非常详尽，像"导游"一样描述沿途细节
- **时间对齐**：指令中的词语与路径位姿时间对齐

**数据集规模**：
- 约42,000+条连续轨迹
- 约126,069条指令
- 平均路径长度：约15米

### 3.2 评价指标——如何衡量智能体表现？

VLN任务有多个评价指标，每个指标侧重不同方面：

| 指标 | 全称 | 中文 | 说明 |
|------|------|------|------|
| **NE** | Navigation Error | 导航误差 | 智能体最终位置与目标位置的平均距离（米） |
| **SR** | Success Rate | 成功率 | 智能体在距离目标3米内停止的比例 |
| **TL** | Trajectory Length | 轨迹长度 | 智能体走过的平均路径长度 |
| **SPL** | Success weighted by Path Length | 加权成功率 | 综合考虑成功率和路径效率的指标 |
| **OSR** | Oracle Success Rate | Oracle成功率 | 轨迹任意一点经过目标位置的比例 |
| **nDTW** | normalized Dynamic Time Warping | 归一化动态时间规整 | 预测轨迹与参考轨迹的相似度 |
| **SDTW** | Success weighted by nDTW | 加权轨迹相似度 | 综合成功和轨迹相似度的指标 |

**指标详解**：

**成功率（SR）**是最直观的指标：
$$SR = \frac{1}{N} \sum_{i=1}^{N} \mathbb{I}(dist(p_{final}^{(i)}, p_{target}^{(i)}) < 3m)$$

简单说，就是智能体最终停在距离目标3米以内的比例。

**SPL**（加权成功率）更全面：
$$SPL = \frac{1}{N} \sum_{i=1}^{N} S_i \frac{l_i}{\max(l_i, p_i)}$$

其中$S_i$表示第i个任务是否成功，$l_i$是最短路径长度，$p_i$是智能体实际走的路径长度。

**SPL的核心思想**：成功很重要，但用更短的路径成功更厉害！

**生活类比**：
- **SR**就像考试只看有没有及格
- **SPL**就像考试成绩还要考虑完成时间——你虽然做完了，但如果磨蹭太久，分数也会打折

## 4. VLN方法分类

### 4.1 按训练方式分类

VLN方法可以从训练方式角度分为三类：

#### （1）行为克隆（Behavior Clone）

**核心思想**：模仿学习，让智能体"看"专家是怎么做的，然后学会执行相同的动作。

**训练方式**：
- 收集专家演示数据（人类标注的轨迹+指令）
- 用监督学习的方式训练策略网络
- 输入：当前观测+指令 → 输出：动作

**优点**：
- 训练稳定
- 不需要与环境交互

**缺点**：
- 容易出现"分布偏移"问题
  - 训练时走的是专家轨迹
  - 测试时如果走偏了，决策可能越来越差
- 难以纠正错误

**经典工作**：Seq2Seq模型（2018 R2R论文）

#### （2）强化学习（Reinforcement Learning）

**核心思想**：通过与环境交互，让智能体自己"试错"学习最优策略。

**训练方式**：
- 定义奖励函数（如：靠近目标+1，碰撞-1，成功+10）
- 智能体不断尝试，根据奖励调整策略

**优点**：
- 可以学习更复杂的决策
- 能够自我纠正错误

**缺点**：
- 训练不稳定
- 样本效率低（需要大量交互）
- 奖励函数设计困难

**经典工作**：Speaker-Follower系列、EnvDrop

#### （3）预训练（Pretraining）

**核心思想**：先在大规模数据上进行"通用"学习，然后再在VLN任务上微调。

**预训练任务**：
- 图像-文本匹配
- 掩码语言建模
- 掩码图像建模
- 轨迹-指令对齐

**优点**：
- 利用大规模数据学习通用表示
- 泛化能力强

**经典工作**：Pretrained VLN、ViLBERT、Pix2Seq等

### 4.2 按架构分类

#### （1）跨模态融合架构

**核心思想**：将视觉和语言信息在早期就进行融合，生成统一的表示。

**典型方法**：
- 早期融合：直接将图像特征和文本特征拼接
- 交叉注意力：让视觉查询语言，语言查询视觉

**代表模型**：
- **Seq2Seq + Attention**：用注意力机制对齐视觉和语言
- **CMA（Cross-Modal Attention）**：跨模态注意力

![跨模态融合架构](./images/crossmodal_fusion.jpg)
*图注：跨模态融合架构示意图。视觉特征和语言特征通过注意力机制交互*

#### （2）端到端架构

**核心思想**：从输入到输出，一个神经网络全部搞定。

**特点**：
- 简单直接
- 全局优化
- 训练端到端的梯度

**代表模型**：
- **VLN-BERT**：基于Transformer的端到端模型

#### （3）分层规划架构

**核心思想**：分而治之，将导航问题分解为多个层次。

**层次划分**：
- **高层规划**：根据指令预测下一个"航点"（waypoint）
- **低层执行**：控制智能体到达航点

**优点**：
- 可解释性强
- 更容易处理长程任务

**代表工作**：
- **ETPNav**：基于拓扑地图的分层规划
- **两阶段VLN方法**：航点预测 + 路径执行

![分层规划架构](./images/hierarchical_planning.jpg)

## 5. VLN发展历程

### 5.1 里程碑事件——VLN如何走到今天？

| 时间 | 里程碑 | 意义 |
|------|--------|------|
| **2018** | R2R数据集发布 | VLN作为独立任务正式诞生 |
| **2019** | Seq2Seq + Attention | 首次将序列到序列模型和注意力机制应用于VLN |
| **2020** | EnvDrop | 提出环境 dropout 技术，提高泛化能力 |
| **2020** | VLN-CE | 连续环境VLN，更接近真实场景 |
| **2021** | PRETRAIN模型 | 大规模预训练应用于VLN |
| **2022** | 两阶段VLN | 引入航点预测器，分层规划成为主流 |
| **2023** | 大语言模型时代 | LLM开始应用于VLN任务规划 |
| **2024** | VLA + VLN | Vision-Language-Action统一框架 |

### 5.2 关键技术演进

**第一阶段（2018-2019）：开创期**

这一阶段的核心问题是：如何让模型理解视觉和语言的对应关系？

主要工作是：
- 设计合适的模型架构（CNN + LSTM + Attention）
- 建立第一个基准数据集（R2R）
- 验证端到端学习的可行性

**第二阶段（2020-2021）：突破期**

这一阶段关注的核心问题是：如何提高模型的泛化能力？

关键突破：
- **EnvDrop**：通过随机遮挡视觉输入来模拟"未知环境"
- **VLN-CE**：从离散环境扩展到连续环境
- **预训练**：利用大规模视觉-语言数据

**第三阶段（2022-至今）：融合期**

这一阶段的核心趋势是：
- 分层规划成为主流架构
- 大语言模型开始应用于任务规划
- VLA（Vision-Language-Action）统一框架的出现

**2. SR和SPL的区别是什么？**

<details>
<summary>点击查看答案</summary>

- **SR（Success Rate，成功率）**：只关心智能体是否成功到达目标（距离目标3米内停止），不考虑路径效率
- **SPL（Success weighted by Path Length，加权成功率）**：综合考虑成功率和路径效率，成功且路径越短分数越高

简单说：SR只看"有没有到"，SPL还看"走得快不快"
</details>

**3. R2R数据集在哪个环境采集？**

<details>
<summary>点击查看答案</summary>

R2R数据集基于**Matterport3D**数据集的环境。Materport3D包含90个真实室内场景（房屋、办公室、商店等）的3D扫描，VLN任务在这些逼真的三维环境中进行。
</details>

**4. VLN连续环境和离散环境的主要区别是什么？**

<details>
<summary>点击查看答案</summary>

- **离散环境**：智能体被限制在预定义的导航图节点上，移动是"跳跃式"的
- **连续环境**：智能体可以在空间中连续移动，动作是"前进0.25米"、"左转15度"这样的低级原语

连续环境更接近真实机器人的运动方式，也更具挑战性。
</details>

### 进阶题

**1. 为什么VLN-CE比传统R2R更难？**

<details>
<summary>点击查看答案</summary>

VLN-CE（连续环境VLN）比传统R2R（离散环境VLN）更难的原因：

1. **动作空间更大**：
   - R2R：只需从候选节点中选择下一个节点
   - VLN-CE：需要选择具体的低级动作（前进多少米、转多少度）

2. **状态空间连续**：
   - R2R：状态是离散的节点
   - VLN-CE：状态是连续的三维坐标和朝向

3. **物理约束**：
   - R2R：可以"瞬移"到目标节点
   - VLN-CE：必须真实移动，可能遇到障碍、死胡同

4. **误差累积**：
   - R2R：每一步都在预定义节点
   - VLN-CE：连续移动会有定位误差累积

5. **碰撞处理**：
   - R2R：不需要考虑碰撞
   - VLN-CE：需要处理与障碍物的碰撞

**生活类比**：就像从"在地铁图上规划路线"变成"真的在商场里找人"
</details>

**2. 分层规划方法相比端到端方法有什么优势？**

<details>
<summary>点击查看答案</summary>

分层规划方法的**优势**：

1. **可解释性强**：可以明确看到"高层的决策"和"低层的执行"
2. **处理长程任务更稳定**：将复杂任务分解为多个简单子任务
3. **更容易调试**：哪一层出问题改哪一层
4. **可复用性**：高层规划可以复用，低层执行可以更换

**劣势**：
1. 架构更复杂
2. 需要额外训练多个模块
3. 层与层之间可能有信息丢失

**端到端方法的优缺点则相反**。
</details>

**3. 为什么Sim2Real是VLN落地的重要挑战？**

<details>
<summary>点击查看答案</summary>

Sim2Real（从仿真到真实）是VLN落地的重要挑战，因为：

1. **训练效率**：在真实环境中训练机器人成本高、风险大，所以通常在仿真器中训练

2. **环境差异**：
   - 视觉差异：仿真器渲染的图像vs真实相机拍摄的图像
   - 物理差异：仿真中的理想条件vs真实世界的噪声和干扰
   - 控制差异：仿真中的精确执行vs真实电机的不确定性

3. **泛化问题**：在仿真中表现好的模型，可能在真实环境中完全失效

**解决方向**：
- 域随机化
- 对抗训练
- 真实数据微调
- 改善仿真器的真实感
</details>

## 6. VLN完整实战流程：从数据到部署

想象你有一个移动机器人，想让它听懂"去厨房帮我拿水杯"这样的指令，你需要做什么？这一节将介绍完整的VLN实战流程。

### 6.1 数据收集阶段

如果你想训练一个VLN模型，首先需要数据。那么需要收集什么数据？

#### 6.1.1 需要收集哪些数据？

| 数据类型 | 说明 | 收集方式 |
|----------|------|----------|
| **视觉数据** | RGB图像、深度图 | 机器人摄像头 |
| **语言指令** | 自然语言导航指令 | 人工标注 |
| **轨迹数据** | 专家演示路径 | 遥控/远程操控 |
| **环境数据** | 3D扫描、语义地图 | RGB-D相机+SLAM |
| **奖励标签** | 成功/失败、路径质量 | 后处理 |


**数据收集流程**：
```
1. 环境扫描 → 2. 轨迹采集 → 3. 指令标注 → 4. 数据清洗 → 5. 数据集构建
```

#### 6.1.2 主流数据集

如果你没有条件自己收集数据，可以使用公开数据集：

| 数据集 | 环境 | 轨迹数 | 语言指令 |
|------|------|--------|----------|
| R2R | Matterport3D | ~15k | 英文 |
| REVERIE | Matterport3D | ~21k | 英文 |
| VLN-CE | Habitat | ~50k | 英文 |
| RxR | Matterport3D | ~100k | 多语言 |


#### 6.1.3 自建数据集建议

如果你需要在特定环境（如你的实验室、家庭）部署：

1. **环境扫描**：使用RGB-D相机（如Realsense）扫描环境，构建3D地图
2. **轨迹采集**：
   - 方法1：遥控机器人采集（人类遥控机器人走一遍）
   - 方法2：远程操控（通过VR手柄控制）
3. **指令标注**：让标注员根据轨迹写指令，如"从起点出发，经过餐桌，最后停在沙发旁边"
4. **数据增强**：
   - 同一条轨迹写多条不同指令
   - 数据增强（图像crop、颜色抖动）

### 6.2 模型训练阶段

#### 6.2.1 训练流程

```
数据准备 → 特征提取 → 模型训练 → 验证调优 → 保存模型
```

**训练配置示例**：
- 预训练模型：CLIP、ViT
- 训练框架：PyTorch + Lightning
- 优化器：AdamW
- 学习率：1e-4
- Batch Size：32
- Epochs：100

#### 6.2.2 训练方式与方法的对应

| 训练方式 | 对应方法 | 优点 | 缺点 |
|----------|----------|------|------|
| **行为克隆** | Seq2Seq、Speaker-Follower | 简单直接 | 分布偏移 |
| **强化学习** | RLM、Actor-Critic | 能探索 | 训练不稳定 |
| **预训练+微调** | PREVALENT、VLN-BERT | 泛化好 | 需要大量数据 |


**选择建议**：
- 数据少 → 行为克隆
- 数据中等 → 预训练+微调
- 数据多 → 强化学习

#### 6.2.3 训练技巧

1. **课程学习**：先学短距离指令，再学长距离
2. **数据增强**：图像增强、指令改写
3. **正则化**：Dropout、Label Smoothing
4. **多任务学习**：同时学习预测动作和视觉语言对齐

### 6.3 部署测试阶段

#### 6.3.1 部署流程

```
模型导出 → 性能优化 → 机器人集成 → 现场测试 → 迭代优化
```

#### 6.3.2 部署方案

| 方案 | 适用场景 | 延迟 | 算力要求 |
|------|----------|------|----------|
| **云端部署** | 网络条件好 | ~200ms | GPU服务器 |
| **边缘部署** | 本地机器人 | ~50ms | Jetson/Orin |
| **混合部署** | 复杂任务 | ~100ms | 边缘+云端 |


**边缘部署优化**：
- 模型量化（INT8）
- TensorRT加速
- 知识蒸馏

#### 6.3.3 测试评估

**测试指标**：
- 成功率（SR）
- 加权成功率（SPL）
- 导航误差（NE）
- 路径效率（TL）

**测试环境**：
- 仿真环境：Habitat、WebVLN
- 真实环境：你的机器人

### 6.4 完整方案示例

#### 方案一：实验室科研方案

**目标**：发论文、验证算法
- 数据：使用R2R/VLN-CE
- 模型：预训练+微调
- 部署：云端推理
- 成本：GPU服务器

**流程**：
```
公开数据集 → 预训练模型 → 微调 → 仿真评估 → 论文
```

#### 方案二：产品落地方案

**目标**：实际机器人应用
- 数据：自建特定环境数据集
- 模型：轻量模型+边缘部署
- 部署：Jetson Orin
- 成本：实体机器人

**流程**：
```
环境扫描 → 数据采集 → 训练 → 边缘部署 → 现场测试 → 迭代
```

#### 方案三：低成本学习方案

**目标**：学习VLN技术
- 数据：公开数据集
- 模型：小模型（如MiniGPT-4）
- 部署：笔记本CPU
- 成本：免费

**流程**：
```
公开数据集 → 小模型训练 → Docker部署 → 本地测试
```

### 6.5 方法分类与实战流程的关系

前面我们介绍了VLN的方法分类，现在来看看它与实战流程的关系：

| 阶段 | 行为克隆 | 强化学习 | 预训练+微调 |
|------|-----------|-----------|-------------|
| **数据需求** | 中等 | 高 | 中等 |
| **训练难度** | 低 | 高 | 中等 |
| **泛化能力** | 弱 | 强 | 强 |
| **适用场景** | 快速原型 | 复杂任务 | 通用部署 |


**实战建议**：
1. **入门阶段**：从行为克隆开始，理解VLN基本流程
2. **进阶阶段**：使用预训练模型，如CLIP+Transformer
3. **高级阶段**：引入强化学习，提升泛化能力

### 6.6 常见问题与解决

**Q1：数据不够怎么办？**
- 使用公开数据集预训练
- 数据增强（同轨迹多条指令）
- 仿真器生成数据

**Q2：仿真到真实（Sim2Real）差距大？**
- 域随机化（随机化光照、纹理、物体）
- 真实数据微调
- 改善仿真器物理真实性

**Q3：机器人响应慢？**
- 模型量化（FP16/INT8）
- 边缘部署（Jetson）
- 减少输入分辨率

**Q4：指令理解错误？**
- 检查语言模型能力
- 使用更大的VLM
- 后处理规则过滤

## 本章小结

| 概念 | 说明 |
|------|------|
| **VLN** | Vision-Language-Navigation，视觉语言导航，让智能体根据语言指令在环境中导航 |
| **R2R** | Room-to-Room，经典的VLN数据集，基于Matterport3D |
| **SR** | Success Rate，成功率，智能体在目标3米内停止的比例 |
| **SPL** | Success weighted by Path Length，加权成功率，综合成功和路径效率的指标 |
| **VLN-CE** | Continuous Environment VLN，连续环境VLN，更接近真实机器人 |
| **NE** | Navigation Error，导航误差，最终位置与目标的平均距离 |
| **行为克隆** | 通过模仿专家演示学习策略 |
| **强化学习** | 通过与环境交互试错学习 |
| **分层规划** | 将导航分解为高层规划和低层执行 |
