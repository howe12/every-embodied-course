## 一、课程概述

前两节课程（17-1、17-2）我们介绍了 VLN 的基础模型、预训练范式和训练策略。然而，这些方法都假设**一次性指令**——用户给出一条完整指令，智能体从始至终按照该指令执行。这种设定与真实场景存在显著差距：

1. **指令不完整**：用户难以一次性描述所有细节（如"经过红色的门"但实际有多个门）
2. **环境不确定**：语言描述与实际场景存在歧义（如"往前走，左转"——往左还是往右）
3. **目标动态变化**：用户可能在导航中途改变目的地

**对话式导航（Dialogue-based Navigation）** 和 **交互式导航（Interactive Navigation）** 应运而生——它们允许智能体在导航过程中**主动与用户对话**，通过多轮交互消除歧义、获取额外信息、纠正错误。

本课程聚焦以下四大方向：

1. **对话式VLN** — CVDN 数据集、多轮对话上下文建模、多轮交互决策
2. **交互式导航** — 人类反馈机制、主动询问策略、视觉参考点解析
3. **典型模型** — HELPER、VLN+Dialog、Active Navigation Agents、协同式导航
4. **多模态对话理解** — 上下文注意力、对话状态跟踪、生成式询问

## 三、理论内容

### 3.1 对话式VLN概述

#### 3.1.1 从一次性指令到多轮对话

传统 VLN（如 R2R 数据集）的指令是**静态且完整**的——用户在起点就知道完整路径的描述。真实场景则完全不同：

```
用户：去厨房帮我拿杯水
智能体：好的，请问是哪个厨房？家里有两个厨房
用户：主卧旁边的那个
智能体：了解，我现在出发
...
智能体：请问厨房门是开着的还是关着的？
用户：开着的，你直接进去就行
```

这种**多轮交互**带来了全新的挑战：

| 维度 | 传统VLN | 对话式VLN |
|------|---------|-----------|
| **指令形式** | 一次性完整指令 | 多轮对话，逐步澄清 |
| **信息获取** | 仅依赖初始指令 | 可在导航中主动询问 |
| **歧义处理** | 硬编码规则或忽略 | 对话澄清机制 |
| **用户参与** | 被动（仅提供初始指令） | 主动（持续参与） |

---

#### 3.1.2 CVDN 数据集

**CVDN**（Cooperative Vision-and-Dialog Navigation，Thomason et al., 2020）是对话式VLN的开创性数据集，基于 Matterport3D 环境。

**数据集特点**：

- **对话轨迹**：每条数据包含 Navigator（智能体）和 Oracle（用户）之间的多轮对话
- **导航历史**：包含完整的导航路径（轨迹 + 每步视觉观测）
- **目标指称**：最终目标位置用自然语言描述（作为对话焦点）

**对话结构**：

CVDN 中的对话通常围绕**指代歧义**展开：

```
Navigator：请问您说的"红色的椅子"是指哪个房间的？
Oracle：就是在你现在所在的客厅里，靠窗的那个
Navigator：好的，我看到窗外有蓝色的窗帘
Oracle：是的，旁边就是红色的椅子
```

**数据集统计**：

| 统计项 | 数值 |
|--------|------|
| 训练集轨迹 | 5,139 条 |
| 验证集轨迹 | 1,034 条 |
| 测试集轨迹 | 1,020 条 |
| 平均对话轮数 | 6.2 轮 |
| 平均指令长度 | 18.3 词 |
| 平均轨迹长度 | 5.2 步 |

**评估指标**：

CVDN 使用以下指标：

1. **任务完成率（Task Success Rate）**：是否到达目标位置（距离 < 3m）
2. **对话效率（Dialog Efficiency）**：完成任务所需的最少对话轮数
3. **目标定位精度（Goaloracle Accuracy）**：给定完整对话历史，判断目标位置的准确率

#### 3.1.3 对话上下文建模

对话式VLN的核心是如何有效建模**多轮对话历史**。对话上下文包含：

- **历史对话轮次**：用户和智能体之间的所有问答
- **历史视觉观测**：每个对话轮次对应的视觉图像
- **历史动作**：智能体已经执行的动作序列

**上下文编码器设计**：

一个典型的对话上下文编码器包含三个组件：

```
┌─────────────────────────────────────────────────────────────┐
│                    对话上下文编码器                            │
├──────────────────┬──────────────────┬───────────────────────┤
│  对话历史编码器    │  视觉历史编码器    │  动作历史编码器        │
│  (Dialogue LSTM)  │  (Vision Encoder) │  (Action LSTM)       │
│  编码对话文本      │  编码历史图像      │  编码历史动作          │
└────────┬─────────┴────────┬────────┴────────┬──────────────┘
         │                   │                  │
         └───────────────────┼──────────────────┘
                             ↓
                  ┌────────────────────┐
                  │  跨模态融合层       │
                  │  (Cross-Modal Fuse)│
                  └────────┬───────────┘
                           ↓
                  ┌────────────────────┐
                  │  上下文表示 C_t     │
                  └────────────────────┘
```

**对话历史编码**：

使用双层 LSTM 编码对话历史：

$$h_t^{\text{dial}} = \text{LSTM}\left([\text{UTTERANCE}_t; a_{t-1}], h_{t-1}^{\text{dial}}\right)$$

其中 $\text{UTTERANCE}_t$ 是第 $t$ 轮对话的文本嵌入（来自用户或智能体），$a_{t-1}$ 是上一轮智能体的动作。

**视觉历史编码**：

将历史图像序列通过视觉编码器处理后，用 LSTM 聚合：

$$h_t^{\text{vis}} = \text{LSTM}\left([v_t^{\text{panorama}}; v_t^{\text{local}}], h_{t-1}^{\text{vis}}\right)$$

**融合策略**：

最终的上下文表示通过门控机制融合：

$$C_t = g_t \cdot h_t^{\text{dial}} + (1 - g_t) \cdot h_t^{\text{vis}}$$

其中 $g_t = \sigma(W_g \cdot [h_t^{\text{dial}}; h_t^{\text{vis}}; v_t] + b_g)$ 是动态门控，控制对话信息和视觉信息的比重。

#### 3.1.4 多轮交互决策

对话式VLN的决策过程比传统VLN更加复杂——智能体需要在**导航动作**和**询问动作**之间做出选择。

**动作空间扩展**：

传统 VLN 的动作空间：$\mathcal{A} = \{\text{Forward}, \text{TurnLeft}, \text{TurnRight}, \text{Stop}\}$

对话式 VLN 的动作空间：$\mathcal{A} = \{\text{Forward}, \text{TurnLeft}, \text{TurnRight}, \text{Stop}\} \cup \mathcal{Q}$

其中 $\mathcal{Q}$ 是询问动作集合，如：
- $\text{AskLocation}$：询问具体位置
- $\text{AskAppearance}$：询问物体外观
- $\text{AskColor}$：询问物体颜色
- $\text{AskSize}$：询问物体大小

**决策策略**：

多轮交互决策的目标是在每个时间步选择最优动作：

$$a_t^* = \arg\max_{a \in \mathcal{A}} Q\left(s_t, a; \theta\right)$$

其中 $s_t = (v_t, L, D_t, H_t)$ 包含当前视觉观测 $v_t$、原始指令 $L$、对话历史 $D_t$ 和动作历史 $H_t$。

**不确定性感知决策**：

当智能体对当前状态不确定时，应该选择询问而非盲目行动：

$$\text{uncertainty}(s_t) = -\max_a P(a | s_t) \quad \text{（策略熵）}$$

当 $\text{uncertainty}(s_t) > \tau_{\text{ask}}$ 时，优先选择询问动作。

### 3.2 交互式导航

#### 3.2.1 人类反馈机制

交互式导航的核心是引入**人类反馈（Human Feedback）**作为额外的信息来源。相比于纯视觉-语言对齐，人类反馈能提供更直接、更具体的指导。

**反馈的类型**：

| 类型 | 描述 | 示例 |
|------|------|------|
| **二元反馈** | "是/否"确认 | "对了，继续往前走" / "不对，左边那条路" |
| **距离反馈** | "近了/远了" | "再远一点就到了" |
| **方向反馈** | 方向纠正 | "往右转，不要往左" |
| **参照反馈** | 提供视觉参照 | "你经过一扇蓝色的门" |
| **语义反馈** | 语义级别纠正 | "你说的厨房不对，应该是客厅" |

**反馈整合方式**：

将人类反馈 $f_t$ 作为额外输入融入决策：

$$a_t = \pi\left(v_t, L, f_t, h_{t-1}\right)$$

具体实现中，反馈通过嵌入层映射为向量，与视觉特征和语言特征拼接：

$$x_t = \left[v_t; \text{Embed}(L); \text{Embed}(f_t); h_{t-1}\right]$$

---

#### 3.2.2 主动询问（Active Learning）

**主动学习（Active Learning）** 是指智能体主动选择**最有价值的问题**进行询问，以最少的对话轮数获取最多的有效信息。

**信息价值评估**：

智能体需要评估每个候选问题的**期望信息增益（Expected Information Gain）**：

$$\text{EIG}(q) = \mathbb{E}_{p(\text{answer}|\text{context}, q)}\left[\text{IG}(\text{answer})\right]$$

其中 $\text{IG}(\text{answer}) = H(\text{belief before}) - H(\text{belief after} | \text{answer})$ 是信息增益。

**问题类型空间**：

典型的询问类型包括：

- **location**：目标在哪里？
- **appearance**：目标物体长什么样？
- **color**：目标是什么颜色的？
- **size**：目标有多大？
- **room**：目标在哪个房间？
- **distance**：目标离这里多远？
- **direction**：应该往哪个方向走？

**策略实现**：

主动询问策略通过一个**问题选择器**实现：

$$q_t^* = \arg\max_{q \in \mathcal{Q}_{\text{cand}}} \eta(q; s_t, \theta_q)$$

其中 $\eta(\cdot)$ 是问题质量评估函数，由神经网络参数化。

---

#### 3.2.3 视觉参考点解析

视觉参考点（Visual Landmarks）是语言指令中提到的可辨识的视觉元素。准确解析视觉参考点对导航至关重要。

**参考点识别**：

从对话历史和当前观测中识别候选参考点：

$$\text{landmarks} = \text{LandmarkDetector}(v_t, L, D_t)$$

候选参考点通过物体检测器（ Faster R-CNN / CLIP）提取。

**参考点定位**：

当用户提到"经过蓝色的门"，智能体需要：

1. **检测**：在当前图像中检测蓝色的门
2. **定位**：判断门在图像中的相对位置（左/中/右）
3. **关联**：将视觉位置与导航动作关联（如"门的右边有条路"）

### 3.3 典型模型

#### 3.3.1 HELPER 模型

**HELPER**（Humannn et al., 2020, *Heterogeneous Encompassing Language-Enhanced Procedure*）是 CVDN 数据集上的经典模型，首次将对话式导航系统化。

**核心思想**：将对话历史和视觉观测联合编码，使用**不完全信息下的决策**框架处理歧义。

**架构**：

```
┌──────────────────────────────────────────────────────────────┐
│                         HELPER                               │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  语言指令 L ──→ Embed ──→ ┌──────────────────────────┐      │
│                            │  Instruction Encoder     │      │
│                            │  (2层BiLSTM)             │      │
│                            └────────────┬─────────────┘      │
│                                         ↓                    │
│  对话历史 D_t ──→ Dialog Encoder ──┐    │                   │
│                                      ↓    ↓                   │
│  视觉观测 v_t ──→ CNN+FC ────────┼────┴──→ Cross-Modal ─→ │
│                                   │          Fusion         │
│  历史动作 H_t ──→ Action Encoder ─┘            │             │
│                                                ↓             │
│                              ┌───────────────────────┐     │
│                              │  策略网络 π(a|s_t)   │     │
│                              │  输出动作分布         │     │
│                              └───────────────────────┘     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**损失函数**：

HELPER 使用**交叉熵损失**进行动作预测，结合**对话预测损失**：

$$\mathcal{L}{\text{HELPER}} = -\sum_t \log \pi_\theta(a_t^* | s_t) - \lambda \sum_t \log \pi_\theta(d_t^* | s_t)$$

其中 $d_t^*$ 是真实的对话回应（或询问），$\lambda$ 是权衡权重。

**关键创新**：

1. **不完全信息建模**：通过贝叶斯更新维护对目标位置的后验分布
2. **对话动作双预测**：同时预测导航动作和询问动作
3. **历史感知**：通过注意力机制关注历史对话中的关键信息

---

#### 3.3.2 VLN+Dialog 模型

**VLN+Dialog**（Nguyen et al., 2019）是另一篇开创性工作，将对话理解显式融入 VLN 框架。

**核心思想**：将对话视为"动态指令修正"过程——每轮对话都可能修正对目标位置的理解。

**信息传递机制**：

$$\text{belief}_t = \text{belief}{t-1} + \text{DialogUpdate}(d_t, \text{belief}_{t-1})$$

其中 $\text{DialogUpdate}$ 是一个 GRU 单元，接收当前对话 $d_t$ 并更新目标 belief：

$$\text{belief}_t = (1 - z_t) \odot \text{belief}_{t-1} + z_t \odot \tilde{\text{belief}}_t$$

$$\tilde{\text{belief}}_t = \tanh\left(W_h \cdot [\text{belief}_{t-1}; d_t]\right)$$

$$z_t = \sigma\left(W_z \cdot [\text{belief}_{t-1}; d_t]\right)$$

**动态指令更新**：

每轮对话后，原始指令被动态更新为"修正后的指令"：

$$L_t^{\text{updated}} = \text{Update}(L_{t-1}^{\text{updated}}, d_t)$$

#### 3.3.3 Active Navigation Agents

**Active Navigation Agents**（Chen et al., 2021）专注于**何时询问**和**询问什么**的最优策略学习。

**主动询问框架**：

```
┌─────────────────────────────────────────────────────────────┐
│           Active Navigation Agent 架构                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  当前状态 s_t ──→ ┌──────────────────┐                       │
│                  │  不确定性估计器    │ → uncertainty_t     │
│                  │  (置信度网络)     │                       │
│                  └────────┬─────────┘                       │
│                           ↓                                  │
│  uncertainty_t > τ_ask? ──┼── Yes → 选择最佳问题 q*         │
│                           │                                  │
│                           No → 执行导航动作 a*               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**问题选择器**：

候选问题集合为 $\mathcal{Q} = \{q_1, q_2, ..., q_K\}$，每个问题通过价值网络评估：

$$V(q; s_t, \theta) = \mathbb{E}\left[\text{SuccessRate}(\text{after asking } q) - \text{SuccessRate}(\text{no asking})\right]$$

选择期望价值最高的问题：

$$q^* = \arg\max_{q \in \mathcal{Q}} V(q; s_t, \theta)$$

**训练方法**：

使用**强化学习**直接优化询问策略，以任务成功率为奖励：

$$\mathcal{L}(\theta) = -\mathbb{E}_{q \sim \pi_\theta}[\nabla_\theta \log \pi_\theta(q | s_t) \cdot R_t]$$

其中 $R_t$ 是长期折扣奖励，包含询问代价（避免过度询问）。

#### 3.3.4 协同式导航

**协同式导航（Collaborative Navigation）** 扩展了对话式导航的设定：智能体不仅与用户对话，还可能与**其他智能体**协同完成导航任务。

**协同式导航的典型场景**：

```
场景：两个机器人在同一建筑中协作
Robot A：负责主导航（类似传统VLN）
Robot B：作为"活地图"提供定位信息（已知全局信息）

对话示例：
Robot A：我现在在二楼走廊，面前有两个门
Robot B：你左边那个门通向会议室，右边那个通向休息室
Robot A：明白了，我去会议室，左转
Robot B：好的，祝顺利
```

**信息共享机制**：

协同式导航需要解决**多智能体通信**问题：

1. **通信协议**：定义智能体之间传递信息的形式（如物体指代、区域描述）
2. **状态对齐**：确保多个智能体对环境的理解一致
3. **任务分解**：将复杂任务分解为多个子任务分配给不同智能体

### 3.4 多模态对话理解

#### 3.4.1 上下文注意力

对话式VLN中的上下文注意力（Contextual Attention）需要同时处理**多种模态**的上下文信息。

**多层级注意力机制**：

```
┌────────────────────────────────────────────────────────────┐
│                   多模态上下文注意力                        │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  Layer 1: 对话层注意力                                      │
│  → 关注当前对话与历史对话的相关性                           │
│                                                            │
│  Layer 2: 视觉-对话交叉注意力                              │
│  → 用对话内容指导视觉区域的选择性关注                       │
│                                                            │
│  Layer 3: 动作-状态注意力                                  │
│  → 根据当前状态决定关注哪些上下文                           │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**数学表达**：

在第 $l$ 层中，上下文注意力定义为：

$$\alpha_i^{(l)} = \frac{\exp\left(e_i^{(l)}\right)}{\sum_j \exp\left(e_j^{(l)}\right)}$$

$$e_i^{(l)} = \frac{(W_Q^{(l)} h_i^{(l-1)})^\top \cdot (W_K^{(l)} c_j^{(l)})}{\sqrt{d_k}}$$

其中 $h_i^{(l-1)}$ 是上一层的隐藏状态，$c_j^{(l)}$ 是上下文表示。

---

#### 3.4.2 对话状态跟踪

**对话状态跟踪（Dialogue State Tracking, DST）** 是对话式VLN的核心组件——它维护一个对**目标位置**的 belief（信念）分布。

**状态表示**：

对话状态包含多个槽位（slot）：

$$B_t = \{\text{goal_location}, \text{landmark_color}, \text{landmark_type}, \text{direction}, \text{distance}\}$$

每个槽位维护一个概率分布：

$$P_t(\text{slot}_k) = \text{Softmax}(W_{\text{slot}_k} \cdot [v_t; L; D_t])$$

**贝叶斯更新**：

每轮对话后，用新的对话信息更新 belief：

$$P(\text{goal} | D_t) \propto P(D_t | \text{goal}) \cdot P(\text{goal} | D_{t-1})$$

当用户确认某个位置时，该位置的概率大幅提升：

$$P_t(\text{goal}_i) = \begin{cases}P_{t-1}(\text{goal}_i) \cdot \beta & \text{if confirmed} \\ P_{t-1}(\text{goal}_i) \cdot (1-\beta) & \text{if denied}\end{cases}$$

---

#### 3.4.3 生成式询问

**生成式询问（Generative Questioning）** 是指智能体能够**生成自然语言问题**而非从预设问题池中选择。

**Seq2Seq 询问生成**：

使用标准的 Seq2Seq 模型生成询问：

$$P(q_t | s_t) = \prod_{i=1}^{T_q} P(w_i | w_{<i}, s_t; \theta_q)$$

**条件控制**：

生成的问题需要满足：
1. **语法正确**：符合自然语言的语法规则
2. **语义相关**：与当前导航上下文相关
3. **信息增益**：期望能提供有价值的信息

**无限制生成的问题**：

预定式询问（从池中选择）："请问目标物体是什么颜色的？"

生成式询问（自由生成）：
- "那扇红色门的旁边是什么？"
- "我能看到地上有一块蓝色的地毯，目标是在那里吗？"
- "从这里到厨房还有多远？"

**训练目标**：

$$\mathcal{L}{\text{gen}} = -\sum_{(q, c) \in \text{aligned}} \log P(q | c; \theta) + \lambda \cdot \text{Reward}(q, c)$$

其中 $\text{Reward}$ 是任务相关的奖励函数，评估生成问题对任务完成的有效性。

## 四、代码实战

### 4.1 对话上下文编码实现

本节实现一个完整的**对话上下文编码器**，用于建模多轮对话历史。

```python
"""
对话上下文编码器实现
用于对话式VLN视觉-语言导航模型中的多轮对话历史建模

作者：Wendy
功能：
1. 对话历史编码（Dialogue LSTM）
2. 视觉历史编码（Vision Encoder + History LSTM）
3. 多模态上下文融合（Cross-Modal Fusion）
4. 动态门控机制（Dynamic Gating）
"""

import torch
import torch.nn as nn
import torch.nn.functional as F


class DialogueHistoryEncoder(nn.Module):
    """
    对话历史编码器
    使用双层LSTM编码用户和智能体之间的多轮对话历史
    """

    def __init__(self, embed_dim=128, hidden_dim=256, num_layers=2, dropout=0.1):
        super().__init__()
        self.embed_dim = embed_dim
        self.hidden_dim = hidden_dim
        self.num_layers = num_layers

        # 词嵌入层（词表大小10000）
        self.embedding = nn.Embedding(num_embeddings=10000, embedding_dim=embed_dim)

        # 对话编码LSTM：输入=[词嵌入; 说话者标签; 动作标签]
        self.dialogue_lstm = nn.LSTM(
            input_size=embed_dim + 2,  # +2: 说话者(1) + 动作(1)
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0,
            bidirectional=False
        )

        self.dropout = nn.Dropout(dropout)
        self.projection = nn.Linear(hidden_dim, hidden_dim)

    def forward(self, dialogue_history, speaker_ids, action_ids):
        """
        参数：
        - dialogue_history: [batch_size, max_dialogue_len] 文本token ids
        - speaker_ids: [batch_size, max_dialogue_len] 0=系统, 1=用户, 2=智能体
        - action_ids: [batch_size, max_dialogue_len] 动作ID

        返回：
        - dialogue_features: [batch_size, hidden_dim]
        """
        # 1. 词嵌入
        word_embeds = self.embedding(dialogue_history)  # [B, L, embed_dim]

        # 2. 拼接说话者标签和动作标签
        speaker_embeds = F.one_hot(speaker_ids, num_classes=3).float()  # [B, L, 3]
        action_embeds = F.one_hot(action_ids.clamp(0, 4), num_classes=5).float()  # [B, L, 5]

        # 拼接：[B, L, embed_dim + 8]
        dialogue_input = torch.cat([word_embeds, speaker_embeds, action_embeds], dim=-1)

        # 3. LSTM编码
        _, (h_n, _) = self.dialogue_lstm(dialogue_input)

        # 4. 取最后一层隐藏状态
        dialogue_features = h_n[-1]  # [batch_size, hidden_dim]

        # 5. 投影和Dropout
        dialogue_features = self.dropout(self.projection(dialogue_features))

        return dialogue_features


class VisionHistoryEncoder(nn.Module):
    """
    视觉历史编码器
    编码历史视觉观测序列，用注意力加权聚合
    """

    def __init__(self, vision_dim=512, hidden_dim=256, num_history=10):
        super().__init__()
        self.vision_dim = vision_dim
        self.hidden_dim = hidden_dim
        self.num_history = num_history

        self.vision_projection = nn.Linear(vision_dim, hidden_dim)
        self.vision_lstm = nn.LSTM(
            input_size=hidden_dim + 6,  # +6: 相机姿态(3) + 动作编码(3)
            hidden_size=hidden_dim,
            num_layers=1,
            batch_first=True
        )

        # 注意力网络：为历史观测分配重要性权重
        self.attention_weights = nn.Sequential(
            nn.Linear(hidden_dim, hidden_dim // 2),
            nn.Tanh(),
            nn.Linear(hidden_dim // 2, 1)
        )
        self.dropout = nn.Dropout(0.1)

    def forward(self, vision_history, camera_poses, past_actions):
        """
        参数：
        - vision_history: 历史视觉特征列表，长度=num_history
          每元素：[batch_size, num_views, vision_dim]
        - camera_poses: 相机姿态列表，长度=num_history
          每元素：[batch_size, 6]
        - past_actions: 历史动作列表，长度=num_history
          每元素：[batch_size]

        返回：
        - vision_features: [batch_size, hidden_dim]
        """
        projected_features = []
        for obs in vision_history:
            # 投影并池化多视角特征
            proj = self.vision_projection(obs)  # [B, num_views, hidden_dim]
            proj = proj.mean(dim=1)  # [B, hidden_dim]
            projected_features.append(proj)

        # 拼接相机姿态和动作编码
        fused_features = []
        for vis_feat, pose, action in zip(projected_features, camera_poses, past_actions):
            action_onehot = F.one_hot(action.clamp(0, 4), num_classes=5).float()
            action_emb = action_onehot[:, :3]  # [B, 3]
            # 拼接：[B, hidden_dim + 6 + 3]
            fused = torch.cat([vis_feat, pose, action_emb], dim=-1)
            fused_features.append(fused)

        # 打包序列：[B, num_history, hidden_dim + 9]
        history_sequence = torch.stack(fused_features, dim=1)

        # LSTM编码
        vision_output, (h_n, _) = self.vision_lstm(history_sequence)  # [B, num_hist, hidden_dim]

        # 注意力加权聚合
        attention_scores = self.attention_weights(vision_output)  # [B, num_hist, 1]
        attention_weights = F.softmax(attention_scores, dim=1)
        vision_features = torch.sum(vision_output * attention_weights, dim=1)  # [B, hidden_dim]

        return self.dropout(vision_features)


class DynamicContextFusion(nn.Module):
    """
    动态上下文融合模块
    用门控机制动态融合对话历史和视觉历史的表示
    """

    def __init__(self, hidden_dim=256, vision_dim=512):
        super().__init__()
        self.hidden_dim = hidden_dim
        self.vision_projection = nn.Linear(vision_dim, hidden_dim)

        # 门控网络：决定对话信息和视觉信息的融合比例
        self.gate_network = nn.Sequential(
            nn.Linear(hidden_dim * 3, hidden_dim),  # [dialogue; vision; current_obs]
            nn.Tanh(),
            nn.Linear(hidden_dim, 1),
            nn.Sigmoid()
        )

        self.output_layer = nn.Sequential(
            nn.Linear(hidden_dim * 2, hidden_dim),
            nn.LayerNorm(hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.1)
        )

    def forward(self, dialogue_features, vision_features, current_obs):
        """
        参数：
        - dialogue_features: [batch_size, hidden_dim]
        - vision_features: [batch_size, vision_dim]
        - current_obs: [batch_size, vision_dim]

        返回：
        - context_features: [batch_size, hidden_dim]
        """
        # 投影当前观测和视觉历史
        current_obs_proj = self.vision_projection(current_obs)  # [B, hidden_dim]
        vision_features_proj = self.vision_projection(vision_features)  # [B, hidden_dim]

        # 计算动态门控权重
        gate_input = torch.cat([dialogue_features, vision_features_proj, current_obs_proj], dim=-1)
        gate_value = self.gate_network(gate_input)  # [B, 1]

        # 门控融合：g * dialogue + (1 - g) * vision
        fused = gate_value * dialogue_features + (1 - gate_value) * vision_features_proj

        # 最终融合（拼接后投影）
        context_features = self.output_layer(
            torch.cat([fused, current_obs_proj], dim=-1)
        )

        return context_features


class DialogueContextEncoder(nn.Module):
    """
    对话上下文编码器（顶层模块）
    整合对话历史、视觉历史和当前观测，输出统一的上下文表示
    """

    def __init__(self, vocab_size=10000, embed_dim=128, hidden_dim=256,
                 vision_dim=512, num_history=10):
        super().__init__()
        self.hidden_dim = hidden_dim

        self.dialogue_encoder = DialogueHistoryEncoder(embed_dim, hidden_dim, num_layers=2)
        self.vision_encoder = VisionHistoryEncoder(vision_dim, hidden_dim, num_history)
        self.context_fusion = DynamicContextFusion(hidden_dim, vision_dim)

    def forward(self, dialogue_history, speaker_ids, action_ids,
                vision_history, camera_poses, past_actions, current_obs):
        """
        完整前向传播

        参数：
        - dialogue_history: [batch_size, max_dial_len]
        - speaker_ids: [batch_size, max_dial_len]
        - action_ids: [batch_size, max_dial_len]
        - vision_history: 历史视觉特征列表，长度=num_history
        - camera_poses: 相机姿态列表，长度=num_history
        - past_actions: 历史动作列表，长度=num_history
        - current_obs: [batch_size, vision_dim]

        返回：
        - context_features: [batch_size, hidden_dim]
        """
        # 1. 编码对话历史
        dialogue_features = self.dialogue_encoder(dialogue_history, speaker_ids, action_ids)

        # 2. 编码视觉历史
        vision_features = self.vision_encoder(vision_history, camera_poses, past_actions)

        # 3. 动态融合
        context_features = self.context_fusion(dialogue_features, vision_features, current_obs)

        return context_features
```

##

本节实现一个**主动询问策略模块**，用于决定何时询问以及询问什么问题。

```python
"""
主动询问策略实现
用于对话式VLN中的主动学习——决定何时询问以及询问什么问题

作者：Wendy
功能：
1. 不确定性估计器（判断何时需要询问）
2. 问题选择器（决定询问什么问题）
3. 询问策略网络（整合导航动作和询问动作）
"""

import torch
import torch.nn as nn
import torch.nn.functional as F


class UncertaintyEstimator(nn.Module):
    """
    不确定性估计器
    通过策略熵估计当前决策的不确定性
    当不确定性高时，智能体应选择询问而非盲目行动
    """

    def __init__(self, state_dim, hidden_dim=128):
        super().__init__()
        self.state_dim = state_dim

        # 状态编码网络
        self.state_encoder = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )

        # 置信度网络（估计每个动作的概率）
        self.confidence_head = nn.Linear(hidden_dim, 1)  # 输出 logit

    def forward(self, state_features, candidate_action_logits):
        """
        参数：
        - state_features: 状态特征，[batch_size, state_dim]
        - candidate_action_logits: 候选动作的logits，[batch_size, num_actions]

        返回：
        - uncertainty: 不确定性得分，[batch_size, 1]
          值越大表示越不确定，越应该询问
        """
        # 1. 编码状态
        encoded = self.state_encoder(state_features)  # [B, hidden_dim]

        # 2. 计算动作分布的熵
        # 策略熵：H(P) = -sum(p * log(p))
        action_probs = F.softmax(candidate_action_logits, dim=-1)  # [B, num_actions]
        action_entropy = -torch.sum(action_probs * torch.log(action_probs + 1e-8), dim=-1, keepdim=True)

        # 归一化熵（最大熵为 log(num_actions)）
        num_actions = candidate_action_logits.size(-1)
        max_entropy = torch.log(torch.tensor(num_actions, dtype=torch.float32, device=state_features.device))
        normalized_entropy = action_entropy / max_entropy.unsqueeze(0)

        # 综合不确定得分：结合熵和状态编码
        confidence_logit = self.confidence_head(encoded)  # [B, 1]
        # 不确定性 = 归一化熵 + 可学习的置信度调整
        uncertainty = normalized_entropy + torch.sigmoid(confidence_logit) * 0.2

        return uncertainty  # [batch_size, 1]


class QuestionSelector(nn.Module):
    """
    问题选择器
    从候选问题中选择期望信息增益最高的问题
    """

    def __init__(self, state_dim, num_question_types=7, hidden_dim=128):
        super().__init__()
        self.num_question_types = num_question_types

        # 状态编码
        self.state_encoder = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )

        # 候选问题的价值网络
        self.question_value_head = nn.Linear(hidden_dim, num_question_types)

    def forward(self, state_features):
        """
        参数：
        - state_features: 状态特征，[batch_size, state_dim]

        返回：
        - question_values: 每个候选问题的期望价值，[batch_size, num_question_types]
        """
        encoded = self.state_encoder(state_features)
        question_values = self.question_value_head(encoded)  # [B, num_question_types]
        return question_values

    def select_question(self, state_features, epsilon=0.1):
        """
        使用epsilon-greedy策略选择问题

        参数：
        - state_features: [batch_size, state_dim]
        - epsilon: 随机探索概率

        返回：
        - selected_indices: 选择的问题索引，[batch_size]
        """
        with torch.no_grad():
            question_values = self.forward(state_features)  # [B, num_q]
            batch_size = state_features.size(0)

            # epsilon-greedy
            random_actions = torch.randint(0, self.num_question_types, (batch_size,))
            best_actions = torch.argmax(question_values, dim=-1)  # [B]

            # 以epsilon概率随机选择
            is_random = torch.rand(batch_size) < epsilon
            selected = torch.where(is_random, random_actions, best_actions)

            return selected


class ActiveQuestioningPolicy(nn.Module):
    """
    主动询问策略网络（顶层模块）
    整合不确定性估计和问题选择，决定是导航还是询问
    """

    def __init__(self, state_dim, num_nav_actions=4, num_question_types=7, hidden_dim=128):
        super().__init__()
        self.num_nav_actions = num_nav_actions
        self.num_question_types = num_question_types

        # 不确定性估计器
        self.uncertainty_estimator = UncertaintyEstimator(state_dim, hidden_dim)

        # 问题选择器
        self.question_selector = QuestionSelector(state_dim, num_question_types, hidden_dim)

        # 状态编码器（共享）
        self.state_encoder = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )

        # 导航动作预测头
        self.nav_action_head = nn.Linear(hidden_dim, num_nav_actions)

        # 询问阈值（可学习）
        self.ask_threshold = nn.Parameter(torch.tensor(0.5))

    def forward(self, state_features):
        """
        参数：
        - state_features: 状态特征，[batch_size, state_dim]
        - candidate_nav_logits: 导航动作的logits，[batch_size, num_nav_actions]

        返回：
        - uncertainty: 不确定性得分，[batch_size, 1]
        - question_values: 问题价值，[batch_size, num_question_types]
        """
        encoded = self.state_encoder(state_features)
        nav_logits = self.nav_action_head(encoded)
        uncertainty = self.uncertainty_estimator(state_features, nav_logits)
        question_values = self.question_selector(state_features)

        return uncertainty, question_values

    def get_action(self, state_features, epsilon=0.1):
        """
        获取动作：导航或询问

        返回：
        - action_type: 0=导航, 1=询问
        - nav_action / question_index: 具体动作
        """
        uncertainty, question_values = self.forward(state_features)

        # 决定是导航还是询问
        # uncertainty > threshold -> 询问
        should_ask = (uncertainty > self.ask_threshold).squeeze(-1)  # [B]

        nav_actions = torch.argmax(self.nav_action_head(self.state_encoder(state_features)), dim=-1)
        question_indices = self.question_selector.select_question(state_features, epsilon)

        # 根据should_ask选择
        action_type = should_ask.long()  # 0或1
        chosen_actions = torch.where(should_ask, question_indices.float(), nav_actions.float())

        return action_type, chosen_actions.long(), uncertainty


class MultiTurnNavAgent(nn.Module):
    """
    多轮导航Agent（整合模块）
    结合对话上下文编码和主动询问策略
    """

    def __init__(self, vocab_size=10000, embed_dim=128, hidden_dim=256,
                 vision_dim=512, num_history=10, num_nav_actions=4, num_question_types=7):
        super().__init__()
        self.hidden_dim = hidden_dim

        # 对话上下文编码器
        self.context_encoder = DialogueContextEncoder(
            vocab_size, embed_dim, hidden_dim, vision_dim, num_history
        )

        # 主动询问策略
        self.questioning_policy = ActiveQuestioningPolicy(
            state_dim=hidden_dim,
            num_nav_actions=num_nav_actions,
            num_question_types=num_question_types,
            hidden_dim=hidden_dim
        )

        # 导航动作预测
        self.nav_policy = nn.Sequential(
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_dim, num_nav_actions)
        )

    def forward(self, dialogue_history, speaker_ids, action_ids,
                vision_history, camera_poses, past_actions, current_obs):
        """
        完整前向传播

        参数：
        - dialogue_history: [batch_size, max_dial_len]
        - speaker_ids: [batch_size, max_dial_len]
        - action_ids: [batch_size, max_dial_len]
        - vision_history: 历史视觉特征列表
        - camera_poses: 相机姿态列表
        - past_actions: 历史动作列表
        - current_obs: [batch_size, vision_dim]

        返回：
        - action_type: 动作类型（导航/询问），[batch_size]
        - nav_logits: 导航动作logits，[batch_size, num_nav_actions]
        - uncertainty: 不确定性，[batch_size, 1]
        - question_values: 问题价值，[batch_size, num_question_types]
        """
        # 1. 编码上下文
        context_features = self.context_encoder(
            dialogue_history, speaker_ids, action_ids,
            vision_history, camera_poses, past_actions, current_obs
        )

        # 2. 导航动作预测
        nav_logits = self.nav_policy(context_features)

        # 3. 主动询问策略
        uncertainty, question_values = self.questioning_policy(context_features)

        return nav_logits, uncertainty, question_values
```

### 简答题答案

**6. CVDN 中目标指代的含义与对话消除歧义的作用**

**目标指代（Goal Grounding）** 是指将自然语言描述的导航目标（如"红色的椅子"）准确地映射到物理环境中的具体位置或物体。这是 VLN 任务的核心挑战之一。

**问题**：语言描述往往存在歧义——"红色的椅子"可能出现在多个房间中。

**对话消除歧义的过程**：

```
初始指令：去"红色的椅子"那里
  ↓
智能体到达客厅，发现有两把椅子（一红一白）
  ↓
智能体不确定目标，询问：您说的"红色的椅子"是指哪个房间的？
  ↓
用户澄清：就是你现在所在的客厅里，靠窗的那个
  ↓
歧义消除，智能体精确知道目标是"客厅窗户旁的红色椅子"
```

这个过程体现了 CVDN 中对话的核心价值：**通过多轮交互将模糊的语言指代逐步具体化**。

**7. 不确定性估计器与问题选择器的协同工作**

**不确定性估计器**：
- **输入**：状态特征 $s_t$ + 候选动作的 logits
- **输出**：不确定性得分 $	ext{uncertainty}(s_t)$（策略熵的归一化值）
- **作用**：判断当前决策的置信度——当熵高时表示动作预测不确定

**问题选择器**：
- **输入**：状态特征 $s_t$
- **输出**：每个候选问题的期望价值 $[V_1, V_2, ..., V_K]$
- **作用**：评估每个候选问题对任务成功的贡献

**协同工作流程**：

```
状态 s_t
    │
    ├──→ 不确定性估计器 ──→ uncertainty = 0.85 (> threshold=0.5)
    │                                    │
    │                                    ↓ 决定：应该询问
    │                                    │
    ├──→ 问题选择器 ──→ question_values = [0.2, 0.8, 0.5, 0.9, 0.3, 0.6, 0.4]
    │                       │
    │                       ↓ 选择价值最高的问题（索引3）
    │
    └──→ 最终决策：执行问题AskColor（颜色询问）
```

**关键**：当不确定性估计器判断需要询问时，问题选择器从候选问题中选择期望价值最高的一个，两者顺序协同工作。

**8. 对话状态跟踪的作用与贝叶斯更新**

**作用**：

对话状态跟踪（Dialogue State Tracking, DST）是维护智能体对**目标位置 belief（信念）**的核心机制。每轮对话后，智能体根据新的对话信息更新对目标位置的 belief，从而逐步消除歧义、精确目标。

**状态表示**：

每个槽位（如 goal_location、landmark_color 等）维护一个概率分布：

$$P_t(	ext{slot}_k) = 	ext{Softmax}(W_{	ext{slot}_k} \cdot [v_t; L; D_t])$$

**贝叶斯更新过程**：

当用户给出反馈时，用贝叶斯规则更新目标位置的 belief：

$$
P(	ext{goal}_i | D_t) \propto P(D_t | 	ext{goal}_i) \cdot P(	ext{goal}_i | D_{t-1})
$$

**具体例子**：

```
初始：P(goal_kitchen) = 0.5, P(goal_bedroom) = 0.5（不确定）

用户确认：P(D_t="yes" | goal_kitchen) = 0.9, P(D_t="yes" | goal_bedroom) = 0.1

贝叶斯更新：
P(goal_kitchen | D_t) ∝ 0.9 × 0.5 = 0.45
P(goal_bedroom | D_t) ∝ 0.1 × 0.5 = 0.05

归一化后：
P(goal_kitchen | D_t) = 0.45 / (0.45 + 0.05) = 0.9
P(goal_bedroom | D_t) = 0.05 / (0.45 + 0.05) = 0.1
```

通过贝叶斯更新，智能体能够逐步将 belief 集中在正确目标上。

**9. 生成式询问的优势与训练挑战**

**优势**：

1. **表达多样性**：可以生成无限多种问题形式，不受预定义问题池的限制
2. **上下文相关性**：生成的问题与当前具体场景紧密相关，而非泛化的模板问题
3. **自然交互**：更像人类的自然对话，用户体验更好

**训练挑战**：

1. **奖励稀疏性**：生成的问题是否"好"难以直接评估（需要任务成功后的延迟奖励）
2. **语法正确性**：需要额外的语法约束或后处理确保生成的问题是合法的
3. **信息价值评估**：难以精确计算生成问题的期望信息增益
4. **训练不稳定**：RL训练中，探索空间大，策略容易崩溃
5. **用户理解**：自由生成的问题可能超出用户的理解范围或回答能力
