## 1. VLN概述

### 1.1 视觉语言导航定义

**视觉语言导航（Vision-Language Navigation，VLN）** 是指智能体根据自然语言指令，在未知的三维环境中自主导航到目标位置的任务。其核心挑战在于**跨模态理解**——将视觉感知与语言语义对齐，并基于此做出序列决策。

从数学角度，VLN任务可以形式化为：

$$\pi(a_t | o_t, I_t, L) \rightarrow a_t$$

其中：
- $I_t$：智能体在时刻 $t$ 接收的视觉观测（RGB图像或深度图）
- $L$：自然语言指令（如"走过厨房，左转进入卧室"）
- $a_t$：智能体在时刻 $t$ 采取的动作（如 `MoveForward`、`TurnLeft`、`Stop`）
- $\pi$：策略网络，从观测和指令预测动作

**VLN的三大核心问题**：

| 问题 | 含义 | 示例 |
|------|------|------|
| **Where am I?** | 视觉定位——理解当前看到了什么 | "这是厨房" |
| **Where am I going?** | 目标理解——指令要求去哪里 | "目标是卧室" |
| **How do I get there?** | 动作规划——下一步怎么走 | "需要左转" |

### 1.2 应用场景

VLN技术在以下场景中具有重要应用价值：

**（1）家庭服务机器人**

```
用户指令："去客厅把茶几上的遥控器拿给我"
机器人流程：
  1. 理解"客厅"和"茶几"的空间位置
  2. 视觉导航到客厅区域
  3. 识别茶几并抓取遥控器
  4. 返回用户位置
```

**（2）医院/酒店导览机器人**

```
场景：医院大厅
指令："请带我到三楼内科诊室"
挑战：大型室内环境，导航路线长，语义地标多
```

**（3）仓库/物流机器人**

```
场景：电商仓库
指令："去B区第三排货架，取最上层左边第二个箱子"
挑战：密集货架、精确目标定位
```

**（4）博物馆/展览导览**

```
场景：科技馆
指令："沿着蓝色路线走，参观完太空展区后停在前台"
挑战：多区域导航、复杂指令理解
```

### 1.3 挑战与难点

VLN任务面临以下核心挑战：

#### （1）跨模态对齐难

视觉信息和语言信息是异构的——图像是连续的高维信号，语言是离散的结构化符号。将两者对齐需要模型理解：
- "厨房"在视觉上长什么样？
- "左转"对应的视觉特征变化是什么？
- "经过"这个动作在视觉上如何体现？

#### （2）长程序列决策

从起点到目标可能需要数十步动作，误差会累积。即使每一步准确率是90%，50步后的成功率也仅剩 $0.9^{50} \approx 0.5\%$。这要求模型具有强大的**长程记忆和规划能力**。

#### （3）未知环境泛化

训练数据和测试环境通常不同。模型需要泛化到：
- 新的房间布局
- 新的家具摆设
- 新的物体外观

#### （4）指令多样性

自然语言指令的表达方式多种多样：
- 直接指令："往前走5米"
- 间接指令："跟着光源走"
- 模糊指令："去那个看起来温馨的房间"

#### （5）Sim2Real差距

在仿真器中训练的模型，直接部署到真实机器人上往往效果大幅下降，原因包括：
- 视觉渲染差异（光照、纹理、阴影）
- 物理执行差异（电机误差、地形变化）
- 传感器噪声差异

## 2. VLN核心组件

### 2.1 视觉编码器

视觉编码器负责将RGB图像或深度图转换为特征向量，是VLN的"眼睛"。

#### （1）CNN-based 编码器

**ResNet** 是经典的视觉编码器，通过残差连接解决深层网络梯度消失问题：

$$y = \mathcal{F}(x, \{W_i\}) + x$$

其中 $\mathcal{F}$ 是残差函数，$x$ 是输入特征。

对于VLN任务，通常使用 **ResNet-50** 或 **ResNet-101** 预训练的ImageNet权重作为初始化。

#### （2）ViT-based 编码器

**Vision Transformer（ViT）** 将图像划分为固定大小的patches，通过Transformer编码：

```python
# ViT 视觉编码的核心逻辑（伪代码）
def vit_encode(image, patch_size=16):
    """
    ViT 将图像划分为 patches 并编码
    
    输入: [H, W, 3] 的图像
    输出: 图像块的特征表示
    """
    # 1. 划分 patches
    # 224x224 图像，16x16 patch → 14x14=196 个 patches
    patches = split_into_patches(image, patch_size)
    
    # 2. 线性投影 + 位置编码
    patch_embeddings = [Linear(patch) + positional_encoding(i) 
                       for i, patch in enumerate(patches)]
    
    # 3. 添加 [CLS] token
    cls_token = LearnableToken()
    tokens = [cls_token] + patch_embeddings
    
    # 4. 通过 Transformer 编码
    encoded = transformer_encoder(tokens)
    
    # 5. [CLS] token 对应整个图像的全局特征
    return encoded[0]
```

**VLN中常用的ViT模型**：

| 模型 | 参数量 | 视觉特征维度 | 特点 |
|------|--------|-------------|------|
| ViT-B/32 | 86M | 768 | 速度快，精度适中 |
| ViT-B/16 | 86M | 768 | 精度更高 |
| ViT-L/14 | 304M | 1024 | 精度最高 |
| DINOv2 | 304M | 1024 | 自监督学习，泛化强 |

#### （3）CLIP视觉编码器

CLIP的视觉编码器在**大规模图文对**上预训练，天然具有**视觉-语言对齐**能力，非常适合VLN任务：

| 特性 | 说明 |
|------|------|
| 预训练数据 | 4亿图文对 |
| 对齐能力 | 视觉特征与语言特征天然对齐 |
| 零样本能力 | 能识别训练时未见过的物体类别 |
| 适用任务 | VLN、机器人操控、多模态理解 |

**CLIP视觉编码器的使用**：

```python
import clip
import torch

# 加载 CLIP 视觉编码器
model, preprocess = clip.load("ViT-B/32", device="cuda")

# 输入预处理
image = preprocess(pil_image)  # 自动归一化到 [-1, 1]

# 编码图像
with torch.no_grad():
    visual_features = model.encode_image(image.unsqueeze(0))
    # 输出: [1, 512] 的特征向量

# CLIP 输出的特征已归一化，可直接用于跨模态计算
visual_features = visual_features / visual_features.norm(dim=-1, keepdim=True)
```

### 2.2 语言编码器

语言编码器负责将自然语言指令转换为特征向量，是VLN的"耳朵"。

#### （1）LSTM-based 编码器

早期VLN方法使用LSTM作为语言编码器：

$$h_t = \text{LSTM}(w_t, h_{t-1})$$

其中 $w_t$ 是第 $t$ 个词的词嵌入，$h_t$ 是隐藏状态。

**局限性**：LSTM是序列模型，难以并行计算，且对长序列建模能力有限。

#### （2）Transformer-based 编码器

当前主流方法使用 **Transformer** 作为语言编码器，具有更强的并行性和长程建模能力：

```python
# Transformer 语言编码器核心逻辑
class TextEncoder(nn.Module):
    """
    基于 Transformer 的语言编码器
    将自然语言指令编码为特征向量
    """
    
    def __init__(self, vocab_size, embed_dim=512, num_heads=8, num_layers=6):
        super().__init__()
        # 词嵌入层
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        
        # 位置编码
        self.pos_encoding = PositionalEncoding(embed_dim)
        
        # Transformer 编码器层
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=embed_dim,
            nhead=num_heads,
            dim_feedforward=2048,
            dropout=0.1
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers)
        
    def forward(self, input_ids):
        """
        Args:
            input_ids: [B, L]  token IDs
        Returns:
            text_features: [B, D] 全局语言特征
        """
        # 词嵌入 + 位置编码
        x = self.embedding(input_ids)  # [B, L, D]
        x = self.pos_encoding(x)      # [B, L, D]
        
        # Transformer 编码
        x = x.transpose(0, 1)          # [L, B, D]
        x = self.transformer(x)        # [L, B, D]
        
        # 取最后一个 token 的输出作为全局特征
        text_features = x[-1]          # [B, D]
        return text_features
```

### 2.3 跨模态融合

跨模态融合是VLN的核心，其目标是将视觉特征和语言特征统一到同一表示空间。

#### （1）早期融合（Early Fusion）

将视觉特征和语言特征在输入层直接拼接：

$$F_{fused} = [F_{vision}; F_{text}]$$

**优点**：融合彻底，信息损失少
**缺点**：难以训练，容易被主导模态带偏

#### （2）晚期融合（Late Fusion）

各模态独立编码，在决策层进行融合：

$$a = \pi_v(F_{vision}) + \pi_l(F_{text})$$

**优点**：各模态独立优化
**缺点**：缺乏深层次的交互

#### （3）跨模态注意力（Cross-Modal Attention）

当前主流方法，使用注意力机制让不同模态相互"查询"：

$$F_{fused} = \text{Attention}(Q=\text{LN}(F_{text}), K=\text{LN}(F_{vision}), V=\text{LN}(F_{vision}))$$

```python
# 跨模态注意力融合（核心实现）
class CrossModalAttention(nn.Module):
    """
    跨模态注意力：语言特征作为 Query 查询视觉特征
    
    数学表达：
    F_fused = softmax(Q @ K^T / sqrt(d)) @ V
    
    其中：
    - Q（Query）来自语言特征：语言在"问"什么
    - K（Key）来自视觉特征：视觉"回答"什么
    - V（Value）来自视觉特征：视觉提供什么信息
    """
    
    def __init__(self, embed_dim=512, num_heads=8):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = embed_dim // num_heads
        self.scale = self.head_dim ** -0.5
        
        # 投影层
        self.q_proj = nn.Linear(embed_dim, embed_dim)  # 语言 -> Query
        self.k_proj = nn.Linear(embed_dim, embed_dim)  # 视觉 -> Key
        self.v_proj = nn.Linear(embed_dim, embed_dim)  # 视觉 -> Value
        self.out_proj = nn.Linear(embed_dim, embed_dim)
        
        self.layer_norm = nn.LayerNorm(embed_dim)
        
    def forward(self, text_features, visual_features):
        """
        Args:
            text_features: [B, D] 语言特征
            visual_features: [B, N, D] 视觉特征（N 个 patches）
        Returns:
            fused_features: [B, D] 融合后特征
        """
        B = text_features.shape[0]
        
        # 投影到 Q, K, V
        Q = self.q_proj(text_features).unsqueeze(1)    # [B, 1, D]
        K = self.k_proj(visual_features)               # [B, N, D]
        V = self.v_proj(visual_features)                 # [B, N, D]
        
        # 多头注意力
        Q = Q.view(B, 1, self.num_heads, self.head_dim).transpose(1, 2)
        K = K.view(B, -1, self.num_heads, self.head_dim).transpose(1, 2)
        V = V.view(B, -1, self.num_heads, self.head_dim).transpose(1, 2)
        
        # 注意力分数
        attn = (Q @ K.transpose(-2, -1)) * self.scale  # [B, H, 1, N]
        attn = F.softmax(attn, dim=-1)                   # [B, H, 1, N]
        
        # 加权聚合
        out = attn @ V                                   # [B, H, 1, D_h]
        out = out.transpose(1, 2).contiguous().view(B, 1, -1)  # [B, 1, D]
        
        # 输出投影 + 残差
        out = self.out_proj(out.squeeze(1))              # [B, D]
        out = out + text_features                        # 残差连接
        out = self.layer_norm(out)
        
        return out
```

## 3. VLN方法分类

### 3.1 基于注意力的方法

基于注意力的方法通过**跨模态注意力机制**对齐视觉和语言，是VLN的基础方法。

#### 3.1.1 Seq2Seq + Attention

最早期的VLN方法，将导航建模为序列到序列问题：

$$\hat{a}_t = \arg\max_{a} P(a | s_{t}, L; \theta)$$

其中 $s_t$ 是当前状态。

**架构**：
```
指令（LSTM编码）→ 全局意图
当前观测（CNN编码）→ 视觉特征
注意力融合 → 动作预测
```

#### 3.1.2 Speaker-Follower

Speaker-Follower 是VLN的经典方法，包含两个模型：

| 模型 | 作用 | 输入 | 输出 |
|------|------|------|------|
| **Speaker** | 生成指令 | 轨迹 | 自然语言描述 |
| **Follower** | 执行导航 | 图像 + 指令 | 动作序列 |

**Follower的注意力机制**：

```python
# Speaker-Follower 的跨模态注意力
class SpeakerFollowerAttention(nn.Module):
    """
    Speaker-Follower 的核心：语言引导的视觉注意力
    
    功能：
    1. 语言编码器：编码指令序列
    2. 视觉编码器：编码全景图像
    3. 跨模态注意力：语言查询视觉，生成导航动作
    """
    
    def __init__(self, embed_dim=512):
        super().__init__()
        self.lang_encoder = LSTMEncoder(embed_dim)
        self.vis_encoder = CNNEncoder(embed_dim)
        self.attention = CrossModalAttention(embed_dim)
        self.action_head = nn.Linear(embed_dim, 4)  # 4个动作
        
    def forward(self, image, instruction, history=None):
        """
        Args:
            image: [B, H, W, 3] 全景图像
            instruction: [B, L] 语言指令 token
            history: [B, T, D] 历史特征（可选）
        Returns:
            action_probs: [B, 4] 动作概率分布
        """
        # 1. 编码语言指令
        lang_features = self.lang_encoder(instruction)  # [B, D]
        
        # 2. 编码视觉图像
        vis_patches = self.vis_encoder(image)  # [B, N, D]
        
        # 3. 跨模态注意力融合
        fused = self.attention(lang_features, vis_patches)  # [B, D]
        
        # 4. 结合历史（如果有）
        if history is not None:
            fused = fused + history.mean(dim=1)  # 加入历史信息
        
        # 5. 预测动作
        action_logits = self.action_head(fused)  # [B, 4]
        action_probs = F.softmax(action_logits, dim=-1)
        
        return action_probs
```

### 3.2 基于图的方法

基于图的方法将环境建模为拓扑图，利用图神经网络（GNN）进行导航推理。

#### 3.2.1 拓扑地图表示

在VLN中，环境通常被建模为**有向无环图（DAG）**：

$$G = (V, E)$$

- $V = \{v_1, v_2, ..., v_n\}$：节点，表示位置
- $E \subseteq V \times V$：边，表示可通行的路径

**节点属性**：
- 视觉特征（RGB-D图像）
- 语义标签（厨房、卧室等）
- 几何信息（位置坐标）

**边属性**：
- 距离
- 方向
- 可通行性

#### 3.2.2 图神经网络导航

```python
# 基于 GNN 的 VLN 导航（核心逻辑）
class GNN navigator(nn.Module):
    """
    使用图神经网络进行 VLN 导航
    
    核心思想：
    1. 将环境建模为拓扑图
    2. 使用 GNN 在图上传播信息
    3. 根据语言指令在图上做路径规划
    """
    
    def __init__(self, embed_dim=512, num_layers=3):
        super().__init__()
        
        # 节点编码器：视觉 + 位置 -> 节点嵌入
        self.node_encoder = nn.Sequential(
            nn.Linear(embed_dim + 3, embed_dim),  # 视觉特征 + 位置
            nn.ReLU(),
            nn.Linear(embed_dim, embed_dim)
        )
        
        # 图卷积层
        self.gnn_layers = nn.ModuleList([
            GraphConvLayer(embed_dim) for _ in range(num_layers)
        ])
        
        # 语言-图注意力：语言指令在图上查询
        self.lang_graph_attention = CrossModalAttention(embed_dim)
        
        # 动作预测头：选择下一个节点
        self.action_head = nn.Linear(embed_dim * 2, 1)
        
    def build_graph(self, observations):
        """
        从观测构建拓扑图
        
        Args:
            observations: 各位置的视觉观测
        Returns:
            graph: 包含节点和边的图结构
        """
        nodes = []
        edges = []
        
        for obs in observations:
            # 提取节点特征
            node_feat = self.node_encoder(obs)
            nodes.append(node_feat)
            
        # 构建边（相邻位置有边连接）
        for i in range(len(nodes) - 1):
            edges.append([i, i+1, 1.0])  # 距离作为边权重
        
        return {'nodes': nodes, 'edges': edges}
    
    def forward(self, graph, instruction, current_node):
        """
        在图上进行导航决策
        
        Args:
            graph: 拓扑图
            instruction: 语言指令特征
            current_node: 当前节点索引
        Returns:
            next_node: 下一个节点索引
        """
        # 1. 图神经网络信息传播
        node_features = torch.stack(graph['nodes'])  # [N, D]
        
        for gnn_layer in self.gnn_layers:
            node_features = gnn_layer(node_features, graph['edges'])
        
        # 2. 语言-图注意力
        current_feat = node_features[current_node]  # [D]
        fused = self.lang_graph_attention(instruction, node_features)
        
        # 3. 预测下一个节点
        neighbors = self.get_neighbors(current_node, graph['edges'])
        
        scores = []
        for neighbor_idx in neighbors:
            # 拼接当前特征和邻居特征
            concat_feat = torch.cat([current_feat, node_features[neighbor_idx]])
            score = self.action_head(concat_feat)
            scores.append((neighbor_idx, score))
        
        # 选择得分最高的邻居
        next_node = max(scores, key=lambda x: x[1])[0]
        return next_node


class GraphConvLayer(nn.Module):
    """
    图卷积层：节点从邻居节点聚合信息
    """
    
    def __init__(self, embed_dim):
        super().__init__()
        self.linear = nn.Linear(embed_dim, embed_dim)
        self.norm = nn.LayerNorm(embed_dim)
        
    def forward(self, node_features, edges):
        """
        Args:
            node_features: [N, D] 节点特征
            edges: 边列表
        Returns:
            updated_features: [N, D] 更新后节点特征
        """
        N, D = node_features.shape
        aggregated = torch.zeros_like(node_features)
        
        # 聚合邻居信息
        for src, dst, weight in edges:
            aggregated[dst] += weight * self.linear(node_features[src])
        
        # 残差连接 + 归一化
        out = self.norm(node_features + aggregated)
        return out
```

### 3.3 基于强化学习的方法

基于强化学习（RL）的方法通过与环境交互学习最优策略，适合长程导航任务。

#### 3.3.1 强化学习基础

VLN中的强化学习框架：

$$L_{RL} = \mathbb{E}_{\pi_\theta}[\sum_{t=0}^{T} r_t]$$

其中 $r_t$ 是奖励函数：

$$r_t = \begin{cases} +10 & \text{任务成功} \\ -0.01 \cdot \text{NE}_t & \text{奖励递减} \\ -1 & \text{碰撞} \end{cases}$$

#### 3.3.2 PPO算法在VLN中的应用

```python
# PPO 算法用于 VLN 训练
class PPO_VLN:
    """
    PPO（Proximal Policy Optimization）算法训练 VLN 智能体
    
    PPO 的核心思想：限制策略更新幅度，保证训练稳定性
    """
    
    def __init__(self, actor_critic, config):
        self.actor_critic = actor_critic
        self.optimizer = torch.optim.Adam(actor_critic.parameters(), lr=3e-4)
        
        # PPO 超参数
        self.clip_epsilon = 0.2       # 裁剪范围
        self.value_coef = 0.5          # 价值损失系数
        self.entropy_coef = 0.01       # 熵正则系数
        self.gamma = 0.99             # 折扣因子
        self.lam = 0.95               # GAE 系数
        
    def compute_returns(self, rewards, values, dones):
        """
        计算回报和优势函数
        
        使用 GAE（Generalized Advantage Estimation）：
        A_t = δ_t + γλδ_{t+1} + ... + (γλ)^{T-t}δ_{T-1}
        其中 δ_t = r_t + γV(s_{t+1}) - V(s_t)
        """
        T = len(rewards)
        advantages = torch.zeros(T)
        returns = torch.zeros(T)
        
        # 从后向前计算
        last_gae = 0
        for t in reversed(range(T)):
            if t == T - 1:
                next_value = 0
            else:
                next_value = values[t + 1]
            
            # TD 误差
            delta = rewards[t] + self.gamma * next_value * (1 - dones[t]) - values[t]
            
            # GAE
            last_gae = delta + self.gamma * self.lam * (1 - dones[t]) * last_gae
            advantages[t] = last_gae
            returns[t] = advantages[t] + values[t]
        
        return returns, advantages
    
    def update(self, rollout):
        """
        PPO 更新步骤
        
        Args:
            rollout: 收集的轨迹数据
        """
        states, actions, old_log_probs, rewards, dones = rollout
        
        # 1. 计算当前策略的动作概率和价值
        action_probs, values = self.actor_critic(states)
        
        # 2. 计算新策略的动作概率和价值
        dist = torch.distributions.Categorical(action_probs)
        new_log_probs = dist.log_prob(actions)
        
        # 3. 计算 PPO 损失
        ratio = torch.exp(new_log_probs - old_log_probs)
        
        # 裁剪损失
        surr1 = ratio * advantages
        surr2 = torch.clamp(ratio, 1 - self.clip_epsilon, 1 + self.clip_epsilon) * advantages
        policy_loss = -torch.min(surr1, surr2).mean()
        
        # 价值损失
        value_loss = F.mse_loss(values, returns)
        
        # 熵正则（鼓励探索）
        entropy = dist.entropy().mean()
        
        # 总损失
        total_loss = policy_loss + self.value_coef * value_loss - self.entropy_coef * entropy
        
        # 4. 更新策略
        self.optimizer.zero_grad()
        total_loss.backward()
        torch.nn.utils.clip_grad_norm_(self.actor_critic.parameters(), 0.5)
        self.optimizer.step()
```

## 4. 典型VLN模型

### 4.1 PREVALENT

**PREVALENT**（PRETRAINing for VLN）是第一个将大规模预训练应用于VLN的工作。

#### 核心思想

PREVALENT在**大规模视觉-语言数据**上预训练，然后在VLN任务上微调：

| 阶段 | 数据 | 任务 |
|------|------|------|
| 预训练 | COCO、CC等图文对 | 图像-文本匹配、掩码建模 |
| 微调 | R2R/R2R-CE | 行为克隆、强化学习 |

#### 预训练任务

$$\mathcal{L}_{pretrain} = \mathcal{L}_{itm} + \mathcal{L}_{mlm} + \mathcal{L}_{mae}$$

其中：
- $\mathcal{L}_{itm}$：图像-文本匹配损失
- $\mathcal{L}_{mlm}$：掩码语言建模损失
- $\mathcal{L}_{mae}$：掩码图像建模损失

#### 架构

```python
# PREVALENT 核心架构
class PREVALENT(nn.Module):
    """
    PREVALENT: 大规模预训练 VLN 模型
    
    核心组件：
    1. 视觉编码器：ViT 或 ResNet（预训练）
    2. 语言编码器：Transformer（预训练）
    3. 跨模态融合：Cross-Attention
    4. 导航头：动作预测
    """
    
    def __init__(self, config):
        super().__init__()
        
        # 视觉编码器
        self.vision_encoder = ViT.from_pretrained('vit-base-patch16-224')
        
        # 语言编码器
        self.text_encoder = TransformerTextEncoder(
            vocab_size=30000,
            embed_dim=768,
            num_layers=6
        )
        
        # 跨模态融合
        self.cross_attention = CrossModalAttention(embed_dim=768)
        
        # 导航头
        self.navigation_head = nn.Linear(768, 4)  # 4 个动作
        
    def forward(self, image, instruction, mask=None):
        """
        前向传播
        """
        # 视觉特征
        vis_features = self.vision_encoder(image)  # [B, N, D]
        
        # 语言特征
        text_features = self.text_encoder(instruction)  # [B, D]
        
        # 跨模态融合
        fused = self.cross_attention(text_features, vis_features)
        
        # 预测动作
        action_logits = self.navigation_head(fused)
        
        return action_logits
```

### 4.2 SERL

**SERL**（Sample Efficient Reinforcement Learning）是港中文提出的高效强化学习VLN方法。

#### 核心创新

1. **Efficient RL Training**：使用经验回放和策略梯度结合
2. **Success Detector**：训练一个成功检测器用于提前停止
3. **Backbone**：基于Transformer的跨模态编码器

#### 成功检测器

SERL的核心创新是训练一个**成功检测器**来判断任务是否即将成功：

$$P(\text{success} | s_t, L) = \sigma(W \cdot [h_t; f_L] + b)$$

其中 $h_t$ 是当前状态特征，$f_L$ 是语言特征。

#### 训练策略

```python
# SERL 的强化学习训练
class SERL_Trainer:
    """
    SERL 训练器：高效强化学习训练 VLN 智能体
    """
    
    def __init__(self, vln_model, success_detector):
        self.model = vln_model
        self.success_detector = success_detector
        self.replay_buffer = ReplayBuffer(capacity=100000)
        
    def collect_experience(self, env, policy, num_steps=1000):
        """
        收集交互经验
        """
        trajectories = []
        
        for _ in range(num_steps):
            obs = env.reset()
            done = False
            episode_data = []
            
            while not done:
                # 策略网络采样动作
                action = policy.sample_action(obs)
                
                # 执行动作
                next_obs, reward, done, info = env.step(action)
                
                # 记录经验
                episode_data.append({
                    'obs': obs,
                    'action': action,
                    'reward': reward,
                    'next_obs': next_obs,
                    'done': done
                })
                
                obs = next_obs
            
            trajectories.append(episode_data)
            self.replay_buffer.push(trajectories)
        
        return trajectories
    
    def train_step(self, batch_size=256):
        """
        一步训练更新
        """
        # 从回放缓冲区采样
        batch = self.replay_buffer.sample(batch_size)
        
        # 计算奖励（包含成功检测器的辅助奖励）
        enhanced_rewards = []
        for obs, action, reward, done in batch:
            success_prob = self.success_detector.predict(obs)
            enhanced_reward = reward + 0.1 * success_prob  # 辅助奖励
            enhanced_rewards.append(enhanced_reward)
        
        # PPO 更新
        self.model.update(enhanced_rewards)
        
        # 更新成功检测器
        self.success_detector.update(batch)
```

### 4.3 HAMPT

**HAMPT**（Hierarchical Ambiguous instruction-aware Policy Transform）是分层规划方法。

#### 核心思想

HAMPT将VLN任务分解为**三层**：

| 层次 | 名称 | 功能 | 输出 |
|------|------|------|------|
| **高层** | Route Planner | 理解指令，规划路线 | 航点序列 |
| **中层** | View Analyzer | 分析视角，选择目标 | 可通行方向 |
| **低层** | Action Controller | 执行具体动作 | 电机指令 |

#### 分层架构

```python
# HAMPT 分层规划架构
class HAMPT(nn.Module):
    """
    HAMPT: 分层自适应多粒度 VLN 模型
    
    三层架构：
    1. 指令理解层（Route Planner）：理解指令，规划路径
    2. 视角分析层（View Analyzer）：分析当前视野，选择目标
    3. 动作控制层（Action Controller）：输出具体动作
    """
    
    def __init__(self, config):
        super().__init__()
        
        # ===== 高层：路径规划器 =====
        self.route_planner = RoutePlanner(
            embed_dim=512,
            num_layers=6
        )
        
        # ===== 中层：视角分析器 =====
        self.view_analyzer = ViewAnalyzer(
            embed_dim=512,
            num_candidates=36  # 36个方向的候选
        )
        
        # ===== 低层：动作控制器 =====
        self.action_controller = ActionController(
            embed_dim=512,
            action_space=['forward', 'left', 'right', 'stop']
        )
        
        # 状态记忆（用于长程导航）
        self.memory = None
        
    def forward(self, instruction, obs_history, current_view):
        """
        分层决策
        
        Args:
            instruction: 语言指令
            obs_history: 历史观测列表
            current_view: 当前视角图像
        Returns:
            action: 具体动作
        """
        # 1. 高层：更新路径规划
        route_plan = self.route_planner(
            instruction, 
            obs_history
        )
        
        # 2. 中层：分析当前视角，选择下一个航点
        target_waypoint = self.view_analyzer(
            current_view,
            route_plan
        )
        
        # 3. 低层：输出动作使机器人朝向航点
        action = self.action_controller(
            current_view,
            target_waypoint
        )
        
        return action


class RoutePlanner(nn.Module):
    """
    高层路径规划器
    
    功能：根据指令预测导航路径上的航点序列
    """
    
    def __init__(self, embed_dim, num_layers):
        super().__init__()
        self.text_encoder = TransformerEncoder(num_layers, embed_dim)
        self.vision_encoder = CNNEncoder(embed_dim)
        self.route_predictor = nn.GRU(embed_dim, embed_dim, batch_first=True)
        
    def forward(self, instruction, obs_history):
        """
        预测航点序列
        """
        # 编码指令
        lang_feat = self.text_encoder(instruction)  # [B, D]
        
        # 编码历史观测
        vis_feats = [self.vision_encoder(obs) for obs in obs_history]
        vis_feats = torch.stack(vis_feats)  # [B, T, D]
        
        # 预测航点
        route, _ = self.route_predictor(vis_feats)
        
        return route  # 预测的航点序列
```

## 5. 导航策略

### 5.1 距离估计

距离估计是VLN的基础能力，帮助智能体判断"离目标还有多远"。

#### 视觉里程计

基于连续两帧图像的差异估计移动距离：

$$d_t = f(I_t, I_{t-1})$$

其中 $f$ 是深度网络或光流网络。

#### 目标距离预测

根据当前观测预测到目标的最短路径距离：

$$P(\text{dist} | I_t, L) = \text{MLP}([F_{vis}; F_{text}])$$

```python
# 距离估计模块
class DistanceEstimator(nn.Module):
    """
    基于视觉和语言特征的距离估计器
    
    功能：预测从当前位置到目标的估计距离
    这对判断"是否接近目标"和"何时停止"非常重要
    """
    
    def __init__(self, embed_dim=512):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(embed_dim * 2, 256),
            nn.ReLU(),
            nn.Linear(256, 64),
            nn.ReLU(),
            nn.Linear(64, 1)  # 输出标量距离
        )
        
    def forward(self, visual_features, text_features):
        """
        估计到目标的距离
        
        Args:
            visual_features: [B, D] 当前视觉特征
            text_features: [B, D] 语言指令特征
        Returns:
            distance: [B, 1] 估计距离（米）
        """
        # 拼接视觉和语言特征
        fused = torch.cat([visual_features, text_features], dim=-1)
        
        # 预测距离
        distance = self.encoder(fused)
        distance = F.relu(distance)  # 距离非负
        
        return distance
```

### 5.2 方向预测

方向预测帮助智能体判断"下一步应该往哪个方向走"。

#### 动作方向建模

VLN中常用的动作是离散的转向动作：
- `TurnLeft`：左转15°
- `TurnRight`：右转15°
- `MoveForward`：前进0.25m

#### 地标方向预测

预测"目标地标在哪个方向"：

$$P(\text{direction} | I_t, L) = \text{Softmax}(W \cdot [F_{vis}; F_{text}])$$

```python
# 方向预测模块
class DirectionPredictor(nn.Module):
    """
    预测目标地标的方向
    
    将360度全景分成N个方向桶，
    输出每个方向的概率分布
    """
    
    def __init__(self, embed_dim=512, num_directions=36):
        super().__init__()
        self.num_directions = num_directions
        
        self.predictor = nn.Sequential(
            nn.Linear(embed_dim * 2, 256),
            nn.ReLU(),
            nn.Linear(256, num_directions)
        )
        
    def forward(self, visual_features, text_features):
        """
        预测方向概率分布
        
        Args:
            visual_features: [B, D] 视觉特征
            text_features: [B, D] 语言特征
        Returns:
            direction_probs: [B, N] N个方向桶的概率
        """
        fused = torch.cat([visual_features, text_features], dim=-1)
        logits = self.predictor(fused)
        probs = F.softmax(logits, dim=-1)
        
        return probs
    
    def get_direction(self, direction_probs):
        """
        从概率分布中提取方向角
        
        Returns:
            angle: 方向角度（度数，0-360）
        """
        # 加权平均求方向
        angles = torch.arange(0, 360, 360 / self.num_directions).to(device)
        direction = (direction_probs * angles).sum(dim=-1)
        return direction
```

### 5.3 探索策略

在未知环境中，探索策略决定了智能体如何平衡"探索"和"利用"。

#### ε-greedy 策略

$$\pi(a | s) = \begin{cases} \text{random} & \text{if } \epsilon \\ \arg\max Q(s, a) & \text{otherwise} \end{cases}$$

```python
# ε-greedy 探索策略
class EpsilonGreedyExploration:
    """
    ε-greedy 探索策略：平衡探索与利用
    
    - 以概率 ε 随机选择动作（探索）
    - 以概率 1-ε 选择 Q 值最大的动作（利用）
    
    随着训练进行，逐渐减小 ε，鼓励利用
    """
    
    def __init__(self, epsilon_start=1.0, epsilon_end=0.05, epsilon_decay=0.995):
        self.epsilon = epsilon_start
        self.epsilon_end = epsilon_end
        self.epsilon_decay = epsilon_decay
        
    def select_action(self, q_values, training=True):
        """
        选择动作
        
        Args:
            q_values: [A] 各动作的 Q 值
            training: 是否训练模式（训练时探索，推理时利用）
        Returns:
            action: 选择的动作索引
        """
        if training and random.random() < self.epsilon:
            # 探索：随机选择
            return random.randint(0, len(q_values) - 1)
        else:
            # 利用：选择 Q 值最大的动作
            return q_values.argmax().item()
    
    def update_epsilon(self):
        """衰减 epsilon，鼓励更多利用"""
        self.epsilon = max(self.epsilon_end, self.epsilon * self.epsilon_decay)
```

#### 基于不确定性的探索

利用**认知不确定性**指导探索：

$$a^* = \arg\max_a [Q(s, a) + \beta \cdot \sigma(s, a)]$$

其中 $\sigma(s, a)$ 是估计的不确定性。

```python
# 基于不确定性的探索
class UncertaintyExploration:
    """
    基于不确定性的探索策略
    
    核心思想：
    1. Q(s,a) 代表对动作价值的估计
    2. σ(s,a) 代表估计的不确定性
    3. 两者相加，平衡利用和探索
    """
    
    def __init__(self, beta=0.5):
        self.beta = beta
        
        # 主 Q 网络
        self.q_network = QNetwork()
        # 目标 Q 网络（用于计算目标值）
        self.target_network = TargetNetwork()
        
        # 不确定性估计网络
        self.uncertainty_network = UncertaintyNetwork()
        
    def select_action(self, state, instruction, training=True):
        """
        选择动作
        
        Args:
            state: 当前状态（视觉观测）
            instruction: 语言指令
            training: 是否训练模式
        """
        with torch.no_grad():
            # 估计 Q 值
            q_values = self.q_network(state, instruction)
            
            # 估计不确定性
            uncertainty = self.uncertainty_network(state, instruction)
            
            # 综合评分：Q 值 + β * 不确定性
            scores = q_values + self.beta * uncertainty
            
            # 选择评分最高的动作
            action = scores.argmax().item()
            
        return action
```

## 6. 代码实战

### 6.1 VLN模型实现

以下是完整的VLN模型实现，包括视觉编码器、语言编码器、跨模态融合和动作预测：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import clip  # OpenAI CLIP

class VLNAgent(nn.Module):
    """
    VLN 智能体：完整的视觉语言导航模型
    
    架构：
    1. 视觉编码器：CLIP Vision Transformer
    2. 语言编码器：CLIP Text Transformer
    3. 跨模态融合：Cross-Attention
    4. 导航头：动作预测（4个离散动作）
    
    动作空间：
    - 0: MoveForward（前进 0.25m）
    - 1: TurnLeft（左转 15°）
    - 2: TurnRight（右转 15°）
    - 3: Stop（停止）
    """
    
    def __init__(
        self,
        clip_model_name="ViT-B/32",
        action_space=['forward', 'left', 'right', 'stop'],
        embed_dim=512
    ):
        super().__init__()
        
        # 动作空间定义
        self.action_space = action_space
        self.num_actions = len(action_space)
        
        # ========== 1. 视觉编码器（CLIP）==========
        # CLIP 提供了预训练的视觉-语言统一表示
        # 非常适合 VLN 这种需要跨模态理解的任务
        self.clip_model, self.preprocess = clip.load(clip_model_name, jit=False)
        
        # 冻结 CLIP 参数（训练时只更新后续层）
        for param in self.clip_model.parameters():
            param.requires_grad = False
        
        # CLIP 视觉特征维度：ViT-B/32 为 512
        self.vision_dim = self.clip_model.visual.output_dim
        
        # ========== 2. 语言编码器（CLIP Text）==========
        # 复用 CLIP 的文本编码器，与视觉编码器天然对齐
        self.text_encoder = self.clip_model.transformer
        self.vocab_size = self.clip_model.vocab_size
        
        # ========== 3. 跨模态融合层 ==========
        # 将语言特征和视觉特征融合为统一表示
        self.fusion = CrossModalFusion(
            vision_dim=self.vision_dim,
            embed_dim=embed_dim
        )
        
        # ========== 4. 历史记忆层 ==========
        # 用于存储历史观测，帮助长程导航
        self.memory_dim = embed_dim
        self.memory_rnn = nn.LSTM(
            input_size=embed_dim,
            hidden_size=embed_dim,
            num_layers=2,
            batch_first=True
        )
        
        # ========== 5. 动作预测头 ==========
        self.action_head = nn.Sequential(
            nn.Linear(embed_dim, 256),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(256, self.num_actions)
        )
        
        # 隐藏状态
        self.hidden_state = None
        
    def encode_visual(self, images):
        """
        编码视觉图像
        
        Args:
            images: [B, C, H, W] 输入图像（CLIP 预处理后）
        Returns:
            visual_features: [B, D] 视觉特征向量
        """
        # 使用 CLIP 视觉编码器
        visual_features = self.clip_model.encode_image(images)
        
        # L2 归一化
        visual_features = visual_features / visual_features.norm(dim=-1, keepdim=True)
        
        return visual_features.float()
    
    def encode_text(self, text_tokens):
        """
        编码文本指令
        
        Args:
            text_tokens: [B, L] token IDs
        Returns:
            text_features: [B, D] 文本特征向量
        """
        # CLIP 文本编码
        text_features = self.clip_model.encode_text(text_tokens)
        
        return text_features.float()
    
    def reset_memory(self, batch_size):
        """
        重置历史记忆
        
        Args:
            batch_size: batch 大小
        """
        device = next(self.parameters()).device
        # LSTM 隐藏状态：(h_0, c_0)
        h_0 = torch.zeros(2, batch_size, self.memory_dim).to(device)
        c_0 = torch.zeros(2, batch_size, self.memory_dim).to(device)
        self.hidden_state = (h_0, c_0)
    
    def forward(self, images, text_tokens, return_attention=False):
        """
        VLN 前向传播
        
        Args:
            images: [B, C, H, W] 图像
            text_tokens: [B, L] 文本 token
            return_attention: 是否返回注意力权重
        Returns:
            action_logits: [B, num_actions] 动作 logits
            attention_weights: 可选的注意力权重
        """
        B = images.shape[0]
        
        # 1. 编码视觉特征
        visual_features = self.encode_visual(images)  # [B, D]
        
        # 2. 编码文本特征
        text_features = self.encode_text(text_tokens)  # [B, D]
        
        # 3. 跨模态融合
        fused_features = self.fusion(visual_features, text_features)  # [B, D]
        
        # 4. 更新历史记忆
        if self.hidden_state is None:
            self.reset_memory(B)
        
        # 将融合特征扩展为序列（适应 LSTM 输入）
        fused_expanded = fused_features.unsqueeze(1)  # [B, 1, D]
        
        # 通过 LSTM 更新记忆
        memory_out, self.hidden_state = self.memory_rnn(fused_expanded, self.hidden_state)
        memory_out = memory_out.squeeze(1)  # [B, D]
        
        # 5. 融合当前观测和历史记忆
        final_features = fused_features + memory_out  # [B, D]
        
        # 6. 预测动作
        action_logits = self.action_head(final_features)  # [B, num_actions]
        
        if return_attention:
            return action_logits, self.fusion.attention_weights
        return action_logits
    
    def get_action(self, images, text_tokens, epsilon=0.0, training=False):
        """
        获取动作（支持 ε-greedy 探索）
        
        Args:
            images: 图像
            text_tokens: 文本 token
            epsilon: 探索概率
            training: 是否训练模式
        Returns:
            action: 动作索引
            action_probs: 动作概率分布
        """
        action_logits = self.forward(images, text_tokens)
        action_probs = F.softmax(action_logits, dim=-1)
        
        if training and random.random() < epsilon:
            # 探索：随机动作
            action = random.randint(0, self.num_actions - 1)
        else:
            # 利用：选择概率最高的动作
            action = action_probs.argmax(dim=-1).item()
        
        return action, action_probs


class CrossModalFusion(nn.Module):
    """
    跨模态融合模块
    
    核心思想：语言特征作为 Query，查询视觉特征
    实现语义引导的视觉注意力
    """
    
    def __init__(self, vision_dim=512, embed_dim=512, num_heads=8):
        super().__init__()
        
        self.num_heads = num_heads
        self.head_dim = embed_dim // num_heads
        self.scale = self.head_dim ** -0.5
        
        # 语言特征投影为 Query
        self.q_proj = nn.Linear(vision_dim, embed_dim)
        
        # 视觉特征投影为 Key 和 Value
        self.k_proj = nn.Linear(vision_dim, embed_dim)
        self.v_proj = nn.Linear(vision_dim, embed_dim)
        
        # 输出投影
        self.out_proj = nn.Linear(embed_dim, embed_dim)
        
        # 层归一化
        self.layer_norm = nn.LayerNorm(embed_dim)
        
        # FFN（前馈网络）
        self.ffn = nn.Sequential(
            nn.Linear(embed_dim, embed_dim * 4),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(embed_dim * 4, embed_dim)
        )
        
        self.attention_weights = None
        
    def forward(self, visual_features, text_features):
        """
        跨模态注意力融合
        
        Args:
            visual_features: [B, D] 视觉特征
            text_features: [B, D] 语言特征
        Returns:
            fused: [B, D] 融合后特征
        """
        B = visual_features.shape[0]
        
        # 1. 投影 Q, K, V
        Q = self.q_proj(text_features).unsqueeze(1)    # [B, 1, D]
        K = self.k_proj(visual_features).unsqueeze(1)  # [B, 1, D]
        V = self.v_proj(visual_features).unsqueeze(1)  # [B, 1, D]
        
        # 2. 多头注意力
        Q = Q.view(B, 1, self.num_heads, self.head_dim).transpose(1, 2)
        K = K.view(B, 1, self.num_heads, self.head_dim).transpose(1, 2)
        V = V.view(B, 1, self.num_heads, self.head_dim).transpose(1, 2)
        
        # 注意力分数
        attn = (Q @ K.transpose(-2, -1)) * self.scale  # [B, H, 1, 1]
        attn = F.softmax(attn, dim=-1)
        self.attention_weights = attn
        
        # 加权聚合
        out = attn @ V  # [B, H, 1, D_h]
        out = out.transpose(1, 2).contiguous().view(B, 1, -1)  # [B, 1, D]
        
        # 3. 输出投影
        out = self.out_proj(out.squeeze(1))  # [B, D]
        
        # 4. 残差连接 + 层归一化
        out = self.layer_norm(out + text_features)
        
        # 5. FFN
        out = self.layer_norm(out + self.ffn(out))
        
        return out
```

### 6.2 导航控制

以下是VLN导航控制器的实现，用于在仿真环境中执行导航任务：

```python
import numpy as np
import habitat_sim

class VLNNavigationController:
    """
    VLN 导航控制器
    
    负责：
    1. 将 VLN 模型与仿真环境（Habitat）连接
    2. 管理导航循环
    3. 计算奖励和判断成功
    """
    
    # 动作映射：模型动作 -> Habitat 动作
    ACTION_MAP = {
        0: habitat_sim.nav.ShortestPathPolicy.action.MOVE_FORWARD,
        1: habitat_sim.nav.ShortestPathPolicy.action.TURN_LEFT,
        2: habitat_sim.nav.ShortestPathPolicy.action.TURN_RIGHT,
        3: 'STOP'  # 特殊停止动作
    }
    
    def __init__(self, vln_model, device='cuda'):
        """
        初始化导航控制器
        
        Args:
            vln_model: VLNAgent 模型实例
            device: 计算设备
        """
        self.model = vln_model.to(device)
        self.model.eval()
        self.device = device
        
        # 动作历史（用于重置）
        self.action_history = []
        
    def setup_environment(self, scene_path, goal_position):
        """
        设置仿真环境
        
        Args:
            scene_path: .glb 场景文件路径
            goal_position: 目标位置 [x, y, z]
        """
        # 创建 Habitat 模拟器
        sim_config = habitat_sim.SimulatorConfiguration()
        sim_config.scene.id = scene_path
        
        # 创建代理
        agent_config = habitat_sim.AgentConfiguration()
        agent_config.height = 1.5  # 代理高度（米）
        agent_config.radius = 0.1  # 碰撞半径
        
        # 初始化模拟器
        self.sim = habitat_sim.Simulator(sim_config)
        self.agent = self.sim.initialize_agent(0, agent_config)
        
        # 设置目标
        self.goal_position = np.array(goal_position)
        
        # 重置模型记忆
        self.model.reset_memory(batch_size=1)
        
    def get_observation(self):
        """
        获取当前观测
        
        Returns:
            obs: 包含 RGB 图像的字典
        """
        # 从 Habitat 获取传感器数据
        obs = self.sim.get_sensor_observations()
        
        # 转换为 torch tensor
        rgb_image = torch.from_numpy(obs['rgb']).to(self.device)
        rgb_image = rgb_image.permute(2, 0, 1)  # [H, W, C] -> [C, H, W]
        rgb_image = rgb_image.float() / 255.0  # 归一化到 [0, 1]
        
        return rgb_image.unsqueeze(0)  # [1, C, H, W]
    
    def compute_reward(self, agent_position):
        """
        计算奖励
        
        Args:
            agent_position: 代理当前位置 [x, y, z]
        Returns:
            reward: 奖励值
            done: 是否结束
        """
        dist_to_goal = np.linalg.norm(agent_position - self.goal_position)
        
        # 奖励设计
        if dist_to_goal < 0.5:
            # 成功：到达目标附近
            reward = 10.0
            done = True
        else:
            # 步进惩罚（鼓励高效导航）
            reward = -0.01
            done = False
        
        return reward, done
    
    def step(self, instruction):
        """
        执行一步导航
        
        Args:
            instruction: 自然语言指令
        Returns:
            result: 包含 reward, done, info 的字典
        """
        # 1. 获取观测
        obs = self.get_observation()
        
        # 2. Tokenize 指令
        text_tokens = clip.tokenize([instruction]).to(self.device)
        
        # 3. 模型推理
        with torch.no_grad():
            action_idx, action_probs = self.model.get_action(
                obs, text_tokens, training=False
            )
        
        # 4. 转换为 Habitat 动作
        habitat_action = self.ACTION_MAP[action_idx]
        
        # 5. 执行动作
        if habitat_action == 'STOP':
            # 停止动作
            done = True
        else:
            # 执行导航动作
            self.agent.act(habitat_action)
            done = False
        
        # 6. 计算奖励
        agent_pos = self.agent.get_state().position
        reward, episode_done = self.compute_reward(agent_pos)
        done = done or episode_done
        
        return {
            'action_idx': action_idx,
            'action_probs': action_probs.cpu().numpy(),
            'reward': reward,
            'done': done,
            'position': agent_pos,
            'distance_to_goal': np.linalg.norm(agent_pos - self.goal_position)
        }
    
    def reset(self, instruction):
        """
        重置环境，开始新 episode
        
        Args:
            instruction: 自然语言指令
        Returns:
            obs: 初始观测
        """
        self.sim.reset()
        self.agent.set_state(habitat_sim.AgentState())
        
        # 重置模型记忆
        self.model.reset_memory(batch_size=1)
        
        # 重置动作历史
        self.action_history = []
        
        return self.get_observation()


class HabitatVLNTrainer:
    """
    Habitat 环境下的 VLN 训练器
    
    支持：
    1. 行为克隆（Behavior Cloning）
    2. PPO 强化学习
    """
    
    def __init__(self, model, env_config, optimizer_config):
        self.model = model
        self.env_config = env_config
        
        # 优化器
        self.optimizer = torch.optim.Adam(
            model.parameters(),
            lr=optimizer_config['lr']
        )
        
        # 经验回放缓冲区
        self.replay_buffer = []
        self.buffer_capacity = 100000
        
    def train_step_bc(self, batch):
        """
        行为克隆训练步骤
        
        Args:
            batch: 包含 (image, text, action) 的批次数据
        """
        images, text_tokens, actions = batch
        
        # 前向传播
        action_logits = self.model(images, text_tokens)
        
        # 交叉熵损失
        loss = F.cross_entropy(action_logits, actions)
        
        # 反向传播
        self.optimizer.zero_grad()
        loss.backward()
        torch.nn.utils.clip_grad_norm_(self.model.parameters(), 1.0)
        self.optimizer.step()
        
        return loss.item()
    
    def collect_rollout(self, env, policy, num_steps=1000):
        """
        收集 rollout 数据用于 PPO 训练
        
        Args:
            env: VLN 环境
            policy: 导航策略
            num_steps: 收集的步数
        """
        rollout_data = []
        
        for _ in range(num_steps):
            # 获取当前状态
            obs = env.get_observation()
            instruction = env.get_instruction()
            
            # 策略采样动作
            action, action_probs = policy.sample_action(obs, instruction)
            
            # 执行动作
            next_obs, reward, done, info = env.step(action)
            
            # 存储经验
            rollout_data.append({
                'obs': obs,
                'instruction': instruction,
                'action': action,
                'action_probs': action_probs,
                'reward': reward,
                'done': done,
                'info': info
            })
            
            if done:
                env.reset()
        
        return rollout_data
```

## 7. 练习题

### 基础题

**1. VLN 的全称是什么？它的三大输入是什么？**

<details>
<summary>点击查看答案</summary>

**VLN = Vision-Language Navigation（视觉语言导航）**

三大输入：
1. **视觉输入**：RGB图像或深度图，智能体感知周围环境
2. **语言输入**：自然语言指令，描述导航目标
3. **状态输入**：当前位置、历史轨迹、当前朝向

</details>

**2. 跨模态注意力（Cross-Modal Attention）的作用是什么？**

<details>
<summary>点击查看答案</summary>

跨模态注意力的核心作用是**让视觉和语言两种异构模态在特征空间中对齐和融合**。

具体来说：
- 语言特征作为 Query，去"查询"视觉特征
- 视觉特征作为 Key 和 Value，提供视觉信息
- 通过注意力权重，语言可以"告诉"模型应该关注图像的哪个区域

数学表达：
$$F_{fused} = \text{Attention}(Q=\text{LN}(F_{text}), K=\text{LN}(F_{vision}), V=\text{LN}(F_{vision}))$$

</details>

**3. PREVALENT 模型的核心思想是什么？**

<details>
<summary>点击查看答案</summary>

PREVALENT 的核心思想是**大规模视觉-语言预训练**：

1. 在大规模图文数据（CC、COCO等）上进行预训练
2. 预训练任务包括：图像-文本匹配、掩码语言建模、掩码图像建模
3. 在 VLN 任务上微调

这使得模型能够利用预训练学到的丰富视觉-语言对应关系，提高 VLN 任务的泛化能力。

</details>

**4. 什么是 ε-greedy 探索策略？**

<details>
<summary>点击查看答案</summary>

ε-greedy 是一种平衡探索和利用的策略：

- 以概率 $\epsilon$：**探索**——随机选择一个动作
- 以概率 $1-\epsilon$：**利用**——选择当前 Q 值最大的动作

随着训练进行，$\epsilon$ 逐渐衰减（从 1.0 衰减到 0.05），使策略逐渐从探索转向利用。

</details>

### 进阶题

**1. 比较基于注意力的方法、基于图的方法和基于强化学习的方法的优缺点。**

<details>
<summary>点击查看答案</summary>

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **基于注意力** | 架构简单，可解释性强，训练稳定 | 难以处理长程依赖，泛化能力有限 | 短程导航，数据充足 |
| **基于图** | 利用环境拓扑结构，可做路径规划 | 需要预构建地图，计算复杂度高 | 有先验地图的导航 |
| **基于强化学习** | 能自我纠正错误，可处理复杂任务 | 训练不稳定，样本效率低 | 长程导航，需要探索 |

</details>

**2. 为什么说"分层规划"对长程VLN任务特别重要？**

<details>
<summary>点击查看答案</summary>

长程VLN任务（如"从客厅穿过餐厅去主卧"可能需要50+步动作）中，分层规划的重要性体现在：

1. **减少决策复杂度**：
   - 不分层：每一步都要决定"前进多少度"，决策空间大
   - 分层：高层只决定"下一个航点"，低层负责到达航点

2. **缓解误差累积**：
   - 不分层：每一步误差累积，最终位置可能偏离目标很远
   - 分层：每个子目标都是可检验的中间状态，容易纠错

3. **提高指令相关性**：
   - 分层方法可以显式地建模指令中的子目标
   - 高层规划器可以根据指令选择性地关注特定地标

4. **可解释性强**：
   - 可以清楚地解释"当前在执行指令的哪个部分"
   - 调试和干预更容易

</details>

**3. 解释 VLN-CE 和传统 R2R 的主要区别，以及为什么 VLN-CE 更难。**

<details>
<summary>点击查看答案</summary>

**主要区别**：

| 维度 | R2R（离散） | VLN-CE（连续） |
|------|------------|---------------|
| **环境** | 预定义导航图 | 连续3D空间 |
| **动作** | 跳跃到相邻节点 | 前进0.25m、左转15°等原语 |
| **状态** | 离散节点ID | 连续位置+朝向 |
| **碰撞** | 无需处理 | 必须处理 |

**VLN-CE 更难的原因**：

1. **动作空间连续**：智能体需要决定"走多少"、"转多少"，而不是简单选择节点
2. **物理真实性**：需要真实处理障碍物碰撞
3. **定位误差**：连续空间中没有节点作为参考，定位误差会累积
4. **路径多样性**：同一起点终点间有无数条路径，需要智能体自己找到可行的

</details>

**4. 在 VLN 中，为什么需要"距离估计"能力？它是如何实现的？**

<details>
<summary>点击查看答案</summary>

**距离估计的作用**：

1. **判断何时停止**：智能体需要知道"我是否已经到达目标附近"，距离估计提供了判断依据
2. **辅助奖励函数**：在强化学习中，距离变化可以作为奖励信号
3. **路径规划**：估计到目标的距离有助于选择正确的导航方向

**实现方式**：

1. **视觉里程计**：基于连续帧的光流或深度变化估计移动距离
2. **学习式估计**：训练一个网络从"当前观测+目标描述"预测距离

$$\hat{d} = \text{MLP}([F_{vis}; F_{text}])$$

3. **几何计算**：如果知道环境地图，可以用 A* 等算法估计最短路径距离

</details>
