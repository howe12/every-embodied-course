## 1. RAG概述

### 1.1 什么是检索增强生成

**RAG**（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索系统与大语言模型（LLM）相结合的技术架构。其核心思想是：在模型生成回答之前，先从外部知识库中检索出与问题相关的内容，将检索到的文档作为上下文（Context）一并提供给LLM，由LLM基于这些真实内容生成更准确、更可靠的答案。

RAG的工作流程可以用四步概括：

```
用户问题(Query)
    │
    ▼
┌─────────────┐
│  检索(Retrieve)  │ ──→ 从向量数据库查询Top-K相关文档
└─────────────┘
    │
    ▼
┌─────────────┐
│  增强(Augment)  │ ──→ 将检索内容拼入Prompt模板
└─────────────┘
    │
    ▼
┌─────────────┐
│  生成(Generate) │ ──→ LLM基于上下文生成答案
└─────────────┘
    │
    ▼
  最终回答
```

> **来源**：RAG最早由Meta AI研究团队在2020年发表的论文《Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks》中提出。

### 1.2 为什么需要RAG

大语言模型虽已具备强大的通用能力，但在实际应用中仍存在明显瓶颈：

| 问题 | 描述 | RAG的解决方式 |
|------|------|--------------|
| **幻觉问题** | LLM可能生成看似流畅但事实错误的内容 | 提供真实文档作为依据，约束生成范围 |
| **知识时效性** | 模型训练数据有截止日期，无法获取实时信息 | 实时检索最新知识库，无需重新训练 |
| **知识盲区** | 垂直领域（如医疗、法律、机器人）专业知识不足 | 接入领域知识库，补充专业内容 |
| **无法溯源** | 模型的输出无法给出具体来源 | 显式返回检索文档，使回答可溯源 |
| **数据安全** | 企业不愿将私有数据上传用于训练 | 私有数据留存在本地，仅作检索使用 |
| **参数量大** | 知识全部编码在参数中，成本高 | 知识外置到知识库，减少模型参数依赖 |

### 1.3 RAG vs Fine-tuning vs Prompt Engineering

| 对比维度 | Prompt Engineering | Fine-tuning | RAG |
|----------|---------------------|-------------|-----|
| **实现方式** | 优化输入提示词 | 在特定数据上继续训练 | 检索外部知识作为上下文 |
| **知识更新** | 无法更新知识 | 需重新训练模型 | 更新知识库即可 |
| **计算成本** | 极低 | 较高（GPU训练） | 中等（向量检索） |
| **知识容量** | 受Prompt长度限制 | 有限 | 可接入超大规模知识库 |
| **适用场景** | 通用对话、风格控制 | 任务模式学习 | 知识密集型问答 |

**最佳实践**：在许多实际场景中，RAG + Fine-tuning 组合使用效果最佳——先Fine-tune让模型学会任务格式，再用RAG注入最新知识。

## 2. 向量数据库

### 2.1 文本向量化（Embedding）

文本向量化是将自然语言文本转换为稠密向量（Dense Vector）的过程，使得语义相似的文本在向量空间中距离相近。

#### 2.1.1 传统方法：TF-IDF

**TF-IDF**（Term Frequency - Inverse Document Frequency）是一种基于统计的词权重表示方法：

$$
\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)
$$

其中：
- $\text{TF}(t, d)$：词 $t$ 在文档 $d$ 中出现的频率
- $\text{IDF}(t) = \log\frac{N}{1 + \text{DF}(t)}$：词 $t$ 的逆文档频率，$N$ 为文档总数

```python
from sklearn.feature_extraction.text import TfidfVectorizer

# TF-IDF 示例：文本向量化
documents = [
    "机器人运动控制使用PID算法",
    "深度学习模型训练需要GPU算力",
    "具身智能结合机器人与人工智能"
]

vectorizer = TfidfVectorizer()  # 创建TF-IDF向量化器
tfidf_matrix = vectorizer.fit_transform(documents)  # 转换文本为TF-IDF矩阵

print("词汇表:", vectorizer.get_feature_names_out())
print("第一个文档向量:", tfidf_matrix[0].toarray())
```

#### 2.1.2 神经网络方法：Word2Vec

**Word2Vec**通过神经网络将词映射到低维连续向量空间，捕捉语义关系：

```python
from gensim.models import Word2Vec
import jieba

# 中文分词
sentences = [
    jieba.lcut("机器人可以自主导航"),
    jieba.lcut("人工智能让机器人更智能"),
    jieba.lcut("深度学习用于图像识别")
]

# 训练Word2Vec模型
model = Word2Vec(sentences, vector_size=64, window=3, min_count=1, epochs=100)

# 获取词向量
word_vector = model.wv["机器人"]
print("'机器人' 的词向量维度:", word_vector.shape)
print("与'人工智能'最相似的词:", model.wv.most_similar("机器人"))
```

#### 2.1.3 现代方法：BERT Embedding

**BERT**（Bidirectional Encoder Representations from Transformers）能够生成上下文相关的句子级嵌入，是当前RAG系统中最常用的Embedding方案：

```python
from sentence_transformers import SentenceTransformer

# 使用 sentence-transformers 加载预训练Embedding模型
# sentence-transformers 封装了BERT等模型，提供便捷的句子向量提取
model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')

# 将中文句子编码为向量
sentences = [
    "如何调整PID控制器的参数",
    "机器人的运动控制方法有哪些",
    "深度强化学习的训练策略"
]

embeddings = model.encode(sentences)  # 返回形状为 (3, 384) 的向量矩阵
print("向量维度:", embeddings.shape)  # (num_sentences, embedding_dim)
```

> **推荐Embedding模型**（中文场景）：
> - `paraphrase-multilingual-MiniLM-L12-v2`：多语言支持，速度快
> - `text2vec-base-chinese`：专门针对中文训练
> - `m3e-base`：国产开源模型，效果优秀

### 2.2 向量相似度度量

在向量空间中，通过计算向量之间的距离或相似度来衡量文本的相关性。

| 度量方法 | 公式 | 特点 | 适用场景 |
|----------|------|------|----------|
| **余弦相似度** | $\cos(\theta) = \frac{\mathbf{A} \cdot \mathbf{B}}{\|\mathbf{A}\|\|\mathbf{B}\|}$ | 衡量方向差异，不受模长影响 | 文本相似度（最常用） |
| **点积（内积）** | $\mathbf{A} \cdot \mathbf{B} = \sum A_i B_i$ | 兼顾方向与模长 | 已归一化向量时等价于余弦 |
| **欧氏距离** | $d = \|\mathbf{A} - \mathbf{B}\|_2 = \sqrt{\sum(A_i-B_i)^2}$ | 衡量绝对距离 | 几何距离敏感场景 |

```python
import numpy as np
from numpy.linalg import norm

def cosine_similarity(A, B):
    """计算余弦相似度"""
    return np.dot(A, B) / (norm(A) * norm(B))

def dot_product(A, B):
    """计算点积"""
    return np.dot(A, B)

def euclidean_distance(A, B):
    """计算欧氏距离"""
    return norm(A - B)

# 示例
vec_a = np.array([0.1, 0.5, 0.9])
vec_b = np.array([0.2, 0.6, 0.85])

print("余弦相似度:", cosine_similarity(vec_a, vec_b))
print("点积:", dot_product(vec_a, vec_b))
print("欧氏距离:", euclidean_distance(vec_a, vec_b))
```

### 2.3 近似最近邻搜索（ANN）

精确的最近邻搜索在海量向量中复杂度极高（$O(N)$），**近似最近邻（ANN — Approximate Nearest Neighbor）**通过牺牲少量精度换取查询速度的大幅提升。

#### 2.3.1 HNSW（Hierarchical Navigable Small World）

**HNSW**是一种基于图的多层近似搜索算法，具有 $O(\log N)$ 的查询复杂度：

```
层2:  ● ──────── ●          ← 顶层，宽泛搜索
          │
层1:  ● ─ ● ─ ● ─ ●  ●      ← 中层，缩小范围
          │
层0:  ● ● ● ● ● ● ● ● ●    ← 底层，最精细搜索
```

```python
# 使用 hnswlib 库实现HNSW近似最近邻搜索
import hnswlib
import numpy as np

# 生成随机10000个128维向量作为数据
dim = 128
num_elements = 10000
data = np.random.rand(num_elements, dim).astype('float32')

# 初始化HNSW索引
index = hnswlib.Index(space='cosine', dim=dim)  # 使用余弦距离空间
index.init_index(max_elements=num_elements, ef_construction=200, M=16)
index.set_num_threads(4)

# 添加数据到索引
index.add_items(data)

# 设置搜索参数（ef越大越精确但越慢）
index.set_ef(100)

# 查询：找到与查询向量最相似的5个
query = np.random.rand(dim).astype('float32')
labels, distances = index.knn_query(query, k=5)

print("最相似向量的索引:", labels)
print("对应距离:", distances)
```

#### 2.3.2 FAISS（Facebook AI Similarity Search）

**FAISS**是Facebook开源的向量检索库，支持多种索引类型，适合大规模向量搜索：

```python
import faiss
import numpy as np

# 生成10000个128维向量
dim = 128
n = 10000
data = np.random.rand(n, dim).astype('float32')

# 归一化向量（用于余弦相似度搜索）
norms = np.linalg.norm(data, axis=1, keepdims=True)
data_normalized = data / norms

# 创建IndexFlatIP（内积索引，适合归一化向量即余弦相似度）
index = faiss.IndexFlatIP(dim)
index.add(data_normalized)

# 查询：找到最相似的5个
query = np.random.rand(dim).astype('float32')
query = query / np.linalg.norm(query)  # 归一化查询向量
distances, indices = index.search(query.reshape(1, -1), k=5)

print("最相似向量索引:", indices[0])
print("相似度得分:", distances[0])
```

## 3. RAG核心组件

### 3.1 文档切分策略

文档切分（Chunking）是将长文档划分为较小文本块的过程，直接影响检索质量。

#### 3.1.1 固定长度切分

最简单的策略，按字符数或词数硬切分：

```python
def chunk_by_length(text, chunk_size=500, overlap=50):
    """
    固定长度切分（字符级）
    chunk_size: 每个块的字符数
    overlap: 相邻块之间的重叠字符数
    """
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start += chunk_size - overlap  # 移动窗口，留有重叠
    return chunks

# 示例
text = "这是一段很长的机器人操作手册内容..."
chunks = chunk_by_length(text, chunk_size=500, overlap=50)
```

#### 3.1.2 语义切分

基于句子或段落的自然边界切分，保持语义完整性：

```python
import re

def chunk_by_sentence(text, max_chars=500):
    """
    按句子切分，保证每个块不超过最大字符数
    """
    # 按常见句末标点分割句子
    sentences = re.split(r'(?<=[。！？.!?])', text)
    
    chunks = []
    current_chunk = ""
    
    for sentence in sentences:
        if len(current_chunk) + len(sentence) <= max_chars:
            current_chunk += sentence
        else:
            if current_chunk:
                chunks.append(current_chunk.strip())
            current_chunk = sentence
    
    if current_chunk:
        chunks.append(current_chunk.strip())
    
    return chunks
```

#### 3.1.3 递归切分（推荐）

先按大段落切分，再递归拆分超出限制的部分，是目前最推荐的策略：

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

# LangChain提供的递归切分器
# 先尝试按段落切分（双换行），段落太长则按句子切分（单换行）
# 句子还太长则按字符切分
text_splitter = RecursiveCharacterTextSplitter(
    separators=["\n\n", "\n", "。", "！", "？", ". ", " ", ""],  # 按优先级尝试的分隔符
    chunk_size=500,      # 每个块的最大字符数
    chunk_overlap=50,    # 块之间的重叠字符数
    length_function=len  # 计算文本长度的函数
)

# 示例文档
document = """
具身智能是近年来人工智能领域的重要研究方向。
它强调智能体通过身体与环境交互来获取感知和理解世界。
典型的具身智能系统包括机器人、自动驾驶车辆等。
"""

chunks = text_splitter.split_text(document)
for i, chunk in enumerate(chunks):
    print(f"Chunk {i}: {chunk}")
```

### 3.3 检索策略

#### 3.3.1 稀疏检索 vs 密集检索

| 类型 | 原理 | 代表方法 | 优缺点 |
|------|------|----------|--------|
| **稀疏检索** | 基于词频/倒排索引 | BM25、TF-IDF | 可解释性好，但对同义词、语义相似性差 |
| **密集检索** | 基于向量语义表示 | BERT、Dense Passage Retriever | 语义理解强，但需要大量训练数据 |
| **混合检索** | 结合两者优势 | BM25 + Vector Search | 兼顾关键词精确性和语义相似性（**推荐**） |

```python
# 混合检索示例（以ChromaDB为例）
import chromadb
from chromadb.utils import embedding_functions

# 加载Embedding模型
embedding_fn = embedding_functions.SentenceTransformerEmbeddingFunction(
    model_name="paraphrase-multilingual-MiniLM-L12-v2"
)

# 创建ChromaDB客户端
client = chromadb.Client()
collection = client.create_collection(
    name="robot_knowledge_base",
    embedding_function=embedding_fn
)

# 添加文档
collection.add(
    documents=[
        "机器人的PID控制参数调节方法",
        "激光雷达SLAM算法原理",
        "ROS2话题通信机制"
    ],
    ids=["doc1", "doc2", "doc3"]
)

# 检索（支持元数据过滤）
results = collection.query(
    query_texts=["如何调节PID参数"],
    n_results=3,
    where={"category": "control"}  # 元数据过滤条件
)
```

#### 3.3.2 重排序（Rerank）

初检阶段使用向量检索快速筛选候选文档，复检阶段使用交叉编码器（如Cohere Rerank、BGE Reranker）对候选文档进行精细排序：

```python
# Rerank示例（使用BGE-Reranker）
from sentence_transformers import CrossEncoder

# 初始化重排序模型
reranker = CrossEncoder('BAAI/bge-reranker-base')

# 原始查询和候选文档
query = "机器人PID调参方法"
candidate_docs = [
    "PID控制是一种经典的反馈控制算法",
    "机器人的运动学正逆解",
    "PID参数包括比例、积分、微分三个部分",
    "具身智能的感知-决策-执行循环"
]

# 构建(query, document)对的输入格式
pairs = [[query, doc] for doc in candidate_docs]

# 计算重排序得分
scores = reranker.predict(pairs)

# 按得分排序
ranked_indices = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)
ranked_docs = [(candidate_docs[i], scores[i]) for i in ranked_indices]

print("重排序结果:")
for doc, score in ranked_docs:
    print(f"  得分 {score:.4f}: {doc}")
```

## 4. RAG进阶技术

### 4.1 对话式RAG（Conversational RAG）

传统的RAG每次查询是独立的，无法处理多轮对话中的上下文关联。对话式RAG通过**历史消息管理**和**查询改写（Query Rewriting）**来解决这个问题。

```
用户: 具身智能是什么？         (Query 1)
助手: 具身智能是...
用户: 它有哪些应用场景？        (Query 2，需要理解"它"指代上一轮)
         ↓
     查询改写: "具身智能有哪些应用场景"
         ↓
     拼接历史上下文 → 检索 → 生成
```

```python
# 对话式RAG：历史消息管理示例
class ConversationBuffer:
    """维护对话历史，自动完成查询改写"""
    
    def __init__(self, max_turns=5):
        self.history = []  # 存储历史对话 [(user, assistant), ...]
        self.max_turns = max_turns
    
    def add_turn(self, user_msg, assistant_msg):
        self.history.append((user_msg, assistant_msg))
        if len(self.history) > self.max_turns:
            self.history.pop(0)
    
    def rewrite_query(self, current_query, llm):
        """
        使用LLM将当前查询结合历史改写为完整问题
        例如: "它的应用" → "具身智能的应用场景"
        """
        if not self.history:
            return current_query
        
        # 构建包含历史的提示词
        history_text = "\n".join([
            f"用户: {u}\n助手: {a}" for u, a in self.history
        ])
        
        rewrite_prompt = f"""根据以下对话历史，将当前问题改写为一个完整、独立的问题。

对话历史:
{history_text}

当前问题: {current_query}

改写后的完整问题:"""
        
        rewritten = llm.invoke(rewrite_prompt)
        return rewritten.content

# 使用示例
buffer = ConversationBuffer(max_turns=3)
buffer.add_turn("具身智能是什么？", "具身智能是...")
```

### 4.3 Self-RAG

**Self-RAG**（Self-Reflective Retrieval-Augmented Generation）是由卡内基梅隆大学提出的高级RAG框架。其核心思想是让LLM学会**自我反思**：在生成过程中主动判断是否需要检索、检索的内容是否相关、生成的内容是否得到支持。

```
用户问题 ──→ [IsRELENT? 需要检索吗？] ──→ [检索模块]
                                             │
                                    [IsRLEV? 检索内容相关吗？]
                                             │
                                    [IsSUPPORT? 生成内容有依据吗？]
                                             │
                                    ──→ 最终回答（带引用标记）
```

```python
# Self-RAG 核心思想演示（伪代码）
class SelfRAG:
    """Self-RAG 自我反思RAG框架"""
    
    def __init__(self, retriever, generator):
        self.retriever = retriever
        self.generator = generator
    
    def is_retrieval_needed(self, query):
        """判断是否需要检索"""
        # 提示LLM判断：这个问题是否需要外部知识
        prompt = f"判断以下问题是否需要检索外部知识来回答：{query}"
        decision = self.generator.decide(prompt)
        return decision == "需要检索"
    
    def grade_retrieval(self, query, retrieved_doc):
        """评估检索内容的相关性"""
        prompt = f"问题：{query}\n文档：{retrieved_doc}\n评估：文档对回答问题有多大帮助？"
        grade = self.generator.grade(prompt)  # 返回 [Relevant / Partially Relevant / Irrelevant]
        return grade in ["Relevant", "Partially Relevant"]
    
    def grade_support(self, generated_text, retrieved_docs):
        """评估生成内容是否被检索内容支持"""
        prompt = f"生成内容：{generated_text}\n依据文档：{retrieved_docs}\n评估：生成内容是否有文档依据？"
        return self.generator.judge(prompt) == "有依据"
```

### 4.4 Corrective-RAG（CRAG）

**Corrective-RAG**（纠错型RAG）在Self-RAG的基础上增加了主动纠错机制。当检索结果质量不高时，自动触发知识库更新或网络搜索进行补救：

```
检索结果 ──→ [质量评估] ──→ 高质量 ──→ 直接生成
                │
                ├──→ 低质量 ──→ 尝试网络搜索 ──→ 补充结果 ──→ 生成
                │
                └──→ 完全不相关 ─→ 触发知识库更新 ──→ 重新检索
```

### 4.5 RAG系统评估指标

评估RAG系统质量的常用指标：

| 指标 | 描述 | 评估方式 |
|------|------|----------|
| **Precision@K** | Top-K检索结果中有多少相关 | 人工标注或LLM判断 |
| **MRR（平均倒数排名）** | 第一个相关结果排名的倒数均值 | 公式计算 |
| **Hit Rate@K** | Top-K中至少有一个相关结果的比例 | 统计计算 |
| **Faithfulness** | 生成内容与检索内容的契合度 | LLM判断或NLP指标 |
| **Answer Relevance** | 生成答案与原始问题的相关程度 | LLM判断或嵌入相似度 |
| **RAGAS** | 综合评估（Faithfulness + Answer Relevance + Context Relevance） | 多维度LLM评估 |

```python
# ============================================================

    chunk_size=500,          # 每个块最大500字符
    chunk_overlap=50,        # 块之间重叠50字符，保持上下文连贯性
    separators=["\n\n", "\n", "。", "！", "？", ". ", " ", ""]

print(f"切分后共 {len(chunks)} 个文本块")

# ============================================================
# 第3步：生成向量嵌入并存储到ChromaDB
# ============================================================

# 初始化Embedding模型（使用多语言模型，支持中文）
embedding_model = HuggingFaceEmbeddings(
    model_name="paraphrase-multilingual-MiniLM-L12-v2",
    model_kwargs={'device': 'cpu'},  # 使用CPU推理
    encode_kwargs={'normalize_embeddings': True}  # 归一化，便于余弦相似度计算
)

# 创建ChromaDB向量数据库
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embedding_model,
    persist_directory="./chroma_db"  # 持久化存储路径
)

print("向量数据库构建完成，已保存到 ./chroma_db")

# ============================================================

llm = ChatOpenAI(
    model_name="gpt-4",
    temperature=0.3,          # 低温度，保证回答准确性

)

如果检索内容中没有提到相关信息，请如实告知，不要编造答案。

检索到的文档内容：
{context}

用户问题：{question}

请基于以上内容给出专业、准确的回答："""

    template=prompt_template,
    input_variables=["context", "question"]
)

# 创建基于检索的问答链
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",         # stuff类型：将所有检索内容拼接后一起输入
    retriever=vectorstore.as_retriever(
        search_kwargs={"k": 3}  # 检索Top-3最相关文档
    ),
    chain_type_kwargs={"prompt": PROMPT},
    return_source_documents=True  # 返回源文档，便于溯源
)

# 第5步：问答演示
# ============================================================

# 示例问答
questions = [

    "ROS2的话题通信机制是什么？",
    "具身智能有哪些应用场景？"
]

for question in questions:
    print(f"\n{'='*50}")
    print(f"问题: {question}")
    result = qa_chain({"query": question})
    print(f"回答: {result['result']}")
    print(f"\n参考来源文档数: {len(result['source_documents'])}")
```

### 答案2

**答案：C**

余弦相似度公式为：

$$\cos(\theta) = \frac{\mathbf{A} \cdot \mathbf{B}}{\|\mathbf{A}\| \|\mathbf{B}\|}$$

由于同时除以两个向量的模长，余弦相似度**只与方向有关，不受模长影响**。这使得它特别适合比较不同长度的文本向量。

- 点积：受模长影响，模长越大，点积通常越大
- 欧氏距离：衡量绝对距离，自然受模长影响

---

### 答案3

**答案：B**

HNSW的搜索过程是**从最顶层（粗粒度）向底层（细粒度）逐层收敛**的：

1. 从最顶层开始，因为顶层节点稀疏，可以快速定位到大致的搜索区域
2. 在当前层找到最近邻后，跳转到下一层继续搜索
3. 直到最底层，找到最终的最近邻

这种分层设计使得HNSW的查询复杂度降为 $O(\log N)$，兼顾了搜索效率和准确性。

### 答案4

**答案：C**

**递归切分（Recursive Character Text Splitting）** 是目前最推荐的策略，因为它：

1. **按语义优先级切分**：先尝试按段落（双换行）切分，保持语义完整性
2. **逐级细分**：段落太长时自动降级到按句子、单句切分
3. **块间重叠**：通过 `chunk_overlap` 参数保留上下文，避免块边界切断语义
4. **灵活可控**：可通过参数调整切分粒度

相比之下，固定长度切分可能切断句子，破坏语义完整性。

### 答案8

**为什么混合检索效果更好：**

稀疏检索（BM25）和密集检索（向量检索）各有优缺点，混合检索通过融合两者来弥补彼此的不足：

| | 稀疏检索（BM25） | 密集检索（向量） |
|---|---|---|
| **原理** | 基于词频统计和倒排索引 | 基于语义向量表示 |
| **优势** | 关键词精确匹配、专业术语检索 | 同义词、语义相似性理解 |
| **劣势** | 无法理解语义、无法处理同义词 | 对专有名词检索不精确 |
| **典型场景** | 搜索"ROS2话题通信" | 搜索"机器人通信方式有哪些"（语义查询） |

**混合检索的融合方式**：

1. **分数加权求和**：$score_{hybrid} = \alpha \cdot score_{sparse} + (1-\alpha) \cdot score_{dense}$
2. **倒数排名融合（RRF）**：$RRF_{final}(d) = \sum_{i} \frac{1}{k + rank_i(d)}$，其中 $k$ 为常数（通常设为60）

通过混合检索，当用户搜索"机器人PID调节"时，既能通过稀疏检索命中"PID"关键词，又能通过密集检索找到"参数整定"等语义相关的内容，从而获得更全面、更准确的检索结果。
