## 1. 大模型微调概述

### 1.1 为什么需要微调

大语言模型（LLM）在预训练阶段学习到了海量的通用知识——从互联网文本中掌握了语言规律、世界常识甚至编程能力。然而，这些通用能力并不能直接满足特定垂直场景的需求。例如：

- 通用模型无法精准回答"本公司的报销流程是什么"
- 通用模型无法按照特定风格（如客服话术、技术文档）生成内容
- 通用模型在特定领域术语理解上存在偏差

**微调（Fine-tuning）** 的核心思想是：利用少量领域数据，在预训练模型的基础上继续训练，使模型适配特定任务或领域。与从头训练相比，微调具备以下优势：

| 对比维度 | 从头训练 | 全量微调 | 高效微调 |
|----------|----------|----------|----------|
| 训练数据量 | 数十万亿Token | 数百万~数千万Token | 数千~数万Token |
| 计算成本 | 极高（数百万美元） | 高 | 低（几百~几千美元）|
| 过拟合风险 | 低 | 高 | 低 |
| 领域适配性 | 差 | 好 | 好 |
| 训练时间 | 数月 | 数天~数周 | 数小时~数天 |

### 1.2 全量微调 vs 参数高效微调

**全量微调（Full Fine-tuning）** 是最直觉的方式：更新模型的所有参数。以一个7B（70亿参数）规模的模型为例，全量微调需要：

- 优化器状态（Adam）：$$4 \times \text{参数量} \times 2 = 8 \times 70\text{B} = 560\text{GB}$$
- 梯度：$$2 \times \text{参数量} = 140\text{GB}$$
- 模型参数：$$2 \times \text{参数量} = 140\text{GB}$$

总计约 **840GB GPU显存**，至少需要 8×80GB 的 A100 集群。

**参数高效微调（Parameter-Efficient Fine-tuning，PEFT）** 的核心思路是：冻结预训练模型的绝大部分参数，只训练少量额外参数。根据技术路线不同，PEFT 主要分为以下几类：

| 方法 | 核心思想 | 训练参数量 | 代表工作 |
|------|----------|------------|----------|
| LoRA | 低秩矩阵适配 | 0.1%~1% | LoRA、QLoRA、DoRA |
| Adapter | 瓶颈层插入 | 1%~5% | AdapterH、AdapterDrop |
| Prefix Tuning | 软提示词 | <0.1% | Prefix Tuning、P-Tuning v2 |
| Prompt Tuning | 连续提示 | <0.1% | Prompt Tuning |

### 1.3 主流微调方法一览

#### LoRA（Low-Rank Adaptation）

LoRA 通过在原始权重矩阵旁添加低秩分解矩阵来实现参数高效微调。假设原始权重为 $\mathbf{W}_0 \in \mathbb{R}^{d \times k}$，LoRA 冻结 $\mathbf{W}_0$，只训练 $\mathbf{A} \in \mathbb{R}^{d \times r}$ 和 $\mathbf{B} \in \mathbb{R}^{r \times k}$，其中 $r \ll \min(d, k)$。

$$h = \mathbf{W}_0 x + \frac{\alpha}{r} \mathbf{B}\mathbf{A} x$$

其中 $\alpha$ 为缩放因子，通常设为 $r$ 的初始值。

#### QLoRA（Quantized LoRA）

QLoRA 在 LoRA 基础上引入 4-bit 量化，将基座模型的显存需求降低约 75%。核心创新包括：
- **NF4（4-bit NormalFloat）** 量化数据类型
- **双重量化（Double Quantization）**：对量化常数也做量化
- **分页优化器（Paged Optimizers）**：处理梯度检查点内存峰值

#### Adapter

Adapter 在 Transformer 层的注意力模块和前馈网络之间插入瓶颈层：

```
原: ... → Attention → FFN → ...
改: ... → Attention → Adapter(↓r → ↑d) → FFN → ...
```

适配器内部为两层全连接网络，中间层维度为 $r$（远小于 $d$），通过跳跃连接保持主路径畅通。

## 2. LoRA原理与实现

### 2.1 LoRA核心思想

LoRA 的数学基础来自 **低秩重构假设**：对于一个在特定任务上微调后的大矩阵 $\mathbf{W}_{ft}$，其与原始权重 $\mathbf{W}_0$ 的差值 $\Delta \mathbf{W} = \mathbf{W}_{ft} - \mathbf{W}_0$ 通常是低秩的。

这意味着 $\Delta \mathbf{W}$ 可以分解为两个小矩阵的乘积：

$$\Delta \mathbf{W} = \mathbf{B}\mathbf{A}, \quad \mathbf{B} \in \mathbb{R}^{d \times r}, \mathbf{A} \in \mathbb{R}^{r \times k}$$

其中 $r$ 称为**秩（Rank）**，远小于 $d$ 和 $k$。

为什么低秩假设是合理的？当预训练模型已经学会了大量通用特征后，针对特定任务的调整往往是有限的——只需要在已学到的特征空间中进行小幅度的线性组合，而非重建全新的特征表示。

### 2.2 低秩矩阵分解的物理意义

从特征视角理解：矩阵 $\mathbf{A}$ 将输入特征投影到低维空间（"任务压缩"），矩阵 $\mathbf{B}$ 再从低维空间重建输出（"任务重构"）。

```
输入 x (维度 d)
    ↓
[冻结] W₀ (d×k) → 加到 LoRA 分支
    ↓
矩阵 A (d×r) → 降维到 r
    ↓
矩阵 B (r×k) → 升维到 k
    ↓
输出 h = W₀x + (α/r)BAx
```

### 2.3 LoRA的超参数

| 超参数 | 说明 | 推荐值 |
|--------|------|--------|
| `r`（秩） | 低秩分解的维度，越大越强但越慢 | 4~64，常用 8/16/32 |
| `alpha` | 缩放因子，控制LoRA分支的权重 | 通常设为 r 或 2r |
| `dropout` | LoRA权重的Dropout率 | 0~0.1，常用 0.05 |
| `target_modules` | 应用LoRA的模块 | q_proj, v_proj 常用 |
| `lora_dropout` | LoRA层的Dropout | 0.05 |

### 2.4 训练稳定性

LoRA 训练中的稳定性问题及应对策略：

1. **梯度爆炸/消失**：当 $r$ 较大且 $\alpha$ 较小时，LoRA 分支的梯度可能过小。建议设置 $\alpha = 2r$ 或使用 `scale_mask` 机制。

2. **学习率设置**：LoRA 参数通常需要比原模型更高的学习率（10x~100x），因为 LoRA 参数空间远小于原模型。

3. **秩的选择**：`r=4~8` 适合简单任务（如风格迁移），`r=32~64` 适合复杂推理任务。

## 3. QLoRA

### 3.1 4-bit量化原理

QLoRA 的核心是将预训练模型的权重从 FP16/BF16 量化到 4-bit 以大幅降低显存占用。

**量化**是将连续值映射到离散值的过程。对于 4-bit 量化，每个参数只能用 $2^4 = 16$ 个离散值表示。

量化过程：
$$\mathbf{W}_{int4} = \text{round}\left( \frac{\mathbf{W}_{fp16}}{s} + z \right)$$

其中 $s$ 为缩放因子（scale），$z$ 为零点（zero-point）。

**反量化**：
$$\mathbf{W}_{fp16}^{\prime} = s \times (\mathbf{W}_{int4} - z)$$

量化误差：$\epsilon = \mathbf{W}_{fp16} - \mathbf{W}_{fp16}^{\prime}$

### 3.2 NF4数据类型

标准 4-bit 整数（INT4）的问题在于：其离散值是均匀分布的 $\{0, 1, ..., 15\}$，但神经网络权重通常服从**非均匀分布**（如正态分布）。

**NF4（4-bit NormalFloat）** 是一种针对神经网络权重分布优化的量化格式。其量化边界基于分位数确定，使得量化后的表示在原权重分布上具有均匀的信息密度。

NF4 的 16 个量化中心（以正态分布为例）：

| 索引 | NF4 值（近似） | 对应正态分布分位数 |
|------|---------------|-------------------|
| 0 | -2.83 | 0.0625 |
| 1 | -1.79 | 0.1875 |
| 2 | -1.23 | 0.3125 |
| ... | ... | ... |
| 15 | +2.83 | 0.9375 |

### 3.3 分页优化器（Paged Optimizers）

大模型训练中，梯度清空（gradient accumulation）步骤可能导致显存峰值达到优化器状态大小的数倍。分页优化器将优化器状态卸载到 **CPU 内存**（或磁盘），只在需要时分页调入 GPU：

```
GPU显存不足 → 将部分优化器状态分页到CPU内存
需要更新参数 → 从CPU分页调入 → 执行优化器步骤 → 分页换出
```

这使得在单卡（如 A100 80GB）上微调 65B 参数模型成为可能。

### 3.4 QLoRA vs LoRA 显存对比

| 模型规模 | 全量FP16 | LoRA FP16 | QLoRA NF4 |
|----------|----------|-----------|-----------|
| 7B | ~14GB | ~2GB | ~6GB |
| 13B | ~26GB | ~4GB | ~10GB |
| 33B | ~66GB | ~8GB | ~18GB |
| 65B | ~130GB | ~16GB | ~34GB |

QLoRA 在大幅降低显存的同时，模型效果与全量 FP16 微调基本持平。

## 4. 常用微调框架

### 4.1 PEFT（Parameter-Efficient Fine-Tuning）

**PEFT** 是 Hugging Face 官方维护的参数高效微调库，提供了 LoRA、Prefix Tuning、Prompt Tuning、Adapter 等多种方法的统一接口。

```python
# 使用 PEFT 进行 LoRA 配置
from peft import LoraConfig, get_peft_model

# 定义 LoRA 配置
lora_config = LoraConfig(
    r=16,                          # 秩
    lora_alpha=32,                 # 缩放因子
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# 将 LoRA 应用到基础模型
model = get_peft_model(base_model, lora_config)
model.print_trainable_parameters()
# 输出: trainable params: 4,194,304 || all params: 6,738,415,616 || 0.062%
```

### 4.2 LLaMA-Factory

**LLaMA-Factory** 是一个一站式大模型微调框架，支持 WebUI 和命令行两种交互方式，内置多种数据集格式和训练策略。

```bash
# 使用 LLaMA-Factory 启动 LoRA 微调
llamafactory-cli train examples/train_full/llama3_lora.yaml
```

```yaml
# llama3_lora.yaml 配置示例
model:
  name_or_path: meta-llama/Meta-Llama-3-8B-Instruct
  quantization_bit: 4             # 启用 QLoRA 4-bit 量化

data:
  dataset: alpaca_zh              # 中文 Alpaca 数据集
  template: llama3
  cutoff_len: 2048

lora:
  target_modules: all             # 所有 Linear 层
  r: 16
  alpha: 32
  dropout: 0.05

train:
  output_dir: ./output/llama3_lora
  per_device_train_batch_size: 2
  gradient_accumulation_steps: 4
  learning_rate: 1e-4
  num_train_epochs: 3
  fp16: true
  ddp_timeout: 180000000
```

**Axolotl** 是一个专注于大模型微调的配置驱动框架，使用 YAML 配置文件定义训练流程，支持多模态、多数据源混合训练。

## 5. 数据准备

### 5.1 指令数据格式

高质量的指令微调数据通常包含三个核心字段：**指令（Instruction）**、**输入（Input）** 和 **输出（Output）**。

**Alpaca 格式**：
```json
{
  "instruction": "请将以下英文翻译成中文",
  "input": "Hello, how are you?",
  "output": "你好，你好吗？"
}
```

**ShareGPT 格式**（对话型）：
```json
{
  "conversations": [
    {"from": "human", "value": "请介绍一下北京"},
    {"from": "gpt", "value": "北京是中国的首都..."}
  ]
}
```

**UltraChat 格式**（多轮对话）：
```json
{
  "id": "unique_id",
  "conversations": [
    {"role": "user", "content": "第一轮问题"},
    {"role": "assistant", "content": "第一轮回答"},
    {"role": "user", "content": "第二轮问题"},
    {"role": "assistant", "content": "第二轮回答"}
  ]
}
```

### 5.2 数据清洗

原始数据通常存在以下问题，需要系统性清洗：

| 问题类型 | 具体表现 | 清洗策略 |
|----------|----------|----------|
| 重复数据 | 完全重复或近似重复 | SimHash/MinHash 去重 |
| 格式错误 | JSON结构不完整、字段缺失 | 解析校验+修复 |
| 低质量内容 | 截断文本、乱码、占位符 | 长度阈值+语言检测 |
| 隐私信息 | 手机号、身份证、邮箱等 | 正则匹配+脱敏 |
| 噪声标签 | 标注错误、标签不一致 | 交叉验证+众包审核 |

```python
# 数据清洗代码示例
import re
import json
from typing import List, Dict

def clean_instruction_data(raw_data: List[Dict]) -> List[Dict]:
    """清洗指令微调数据"""
    cleaned = []
    
    for item in raw_data:
        # 1. 校验必要字段
        if not all(k in item for k in ["instruction", "output"]):
            continue
            
        # 2. 去除截断内容（以"..."、"。"等异常结尾）
        output = item["output"].rstrip()
        if output.endswith("...") or len(output) < 5:
            continue
            
        # 3. 过滤隐私信息（手机号、邮箱）
        privacy_patterns = [
            r"1[3-9]\d{9}",          # 手机号
            r"\w+@\w+\.\w+",          # 邮箱
            r"\d{18}",                # 身份证
        ]
        for pattern in privacy_patterns:
            if re.search(pattern, output):
                output = re.sub(pattern, "[隐私信息]", output)
                
        # 4. 过滤乱码（非中文、英文字符过多）
        chinese_chars = len(re.findall(r'[\u4e00-\u9fff]', output))
        total_chars = len(output)
        if total_chars > 0 and chinese_chars / total_chars < 0.3:
            continue  # 中文占比过低，可能是乱码
            
        item["output"] = output
        cleaned.append(item)
        
    return cleaned
```

### 5.3 数据增强

当特定领域数据稀缺时，可以采用以下数据增强策略：

**1. 语义一致性增强（Paraphrasing）**：利用大模型生成语义等价的不同表述：

```python
# 使用 LLM 进行数据增强
prompt_template = """
请将下面的问答对改写成{num_variants}个不同的版本，
保持语义完全相同，但表达方式不同：

原始问答：
问题：{question}
答案：{answer}

改写版本（用数字列表标注）：
"""
```

**2. 领域术语替换**：在保持语法正确的前提下，替换领域相关实体：

```
原始：糖尿病患者应该控制血糖
改写（心脏病场景）：心脏病患者应该控制血脂
```

**3. Back-Translation（回译增强）**：

```
中文 → 英文 → 中文（得到不同表达）
```

**4. 指令多样性**：同一组输入-输出，用不同措辞的指令描述：

```
输入：{"input": "北京天气晴朗", "output": "北京今天阳光明媚"}
指令1: 描述一下北京的天气
指令2: 简单介绍一下北京的气候
指令3: 用一句话概括北京的天气状况
```

## 6. 训练技巧

### 6.1 学习率调度

学习率是微调中最关键的超参数之一。合理的学习率调度策略能显著提升训练稳定性。

**常用学习率调度器对比**：

| 调度器 | 特点 | 适用场景 |
|--------|------|----------|
| Cosine Annealing | 平滑下降，周期性重启 | 长训练步数 |
| Linear Warmup + Cosine | 预热 + 余弦衰减 | 通用场景 |
| Constant | 保持恒定 | 短训练、快速实验 |
| Polynomial Decay | 多项式下降 | 长上下文模型 |

**Cosine Annealing 公式**：
$$lr_t = \eta_{min} + \frac{1}{2}(\eta_{max} - \eta_{min}) \left(1 + \cos\left(\frac{t\pi}{T}\right)\right)$$

其中 $\eta_{max}$ 为最大学习率，$T$ 为总步数。

**带 Warmup 的 Cosine 调度**：

```python
# 使用 transformers 的调度器配置
from transformers import TrainingArguments

training_args = TrainingArguments(
    learning_rate=1e-4,
    warmup_ratio=0.1,              # 前10%的步数用于warmup
    lr_scheduler_type="cosine",
    warmup_steps=100,             # 或者直接指定warmup步数
)
```

### 6.2 Warmup策略

Warmup（预热）是指在训练初期将学习率从极小值逐渐提高到目标学习率的过程。其核心作用是：

1. **防止局部最优**：避免在训练初期因学习率过大而跳出较优区域
2. **减少梯度震荡**：训练初期模型处于不稳定状态，大学习率会导致梯度爆炸
3. **稳定优化器状态**：Adam等优化器的二阶动量需要一定步数才能准确估计

**Warmup 配置建议**：

| 训练规模 | warmup_ratio | warmup_steps（估算） |
|----------|-------------|---------------------|
| < 1000 步 | 0.05~0.1 | 50~100 |
| 1000~10000 步 | 0.03~0.06 | 100~500 |
| > 10000 步 | 0.01~0.03 | 200~1000 |

### 6.3 梯度裁剪

梯度裁剪（Gradient Clipping）通过限制梯度的最大范数，防止训练过程中梯度爆炸：

$$\mathbf{g}_{clipped} = \mathbf{g} \cdot \frac{\text{clip\_norm}}{\|\mathbf{g}\|_2} \quad \text{当} \ \|\mathbf{g}\|_2 > \text{clip\_norm}$$

```python
# PyTorch 梯度裁剪
torch.nn.utils.clip_grad_norm_(
    parameters=model.parameters(),
    max_norm=1.0,         # 最大梯度范数
    norm_type=2            # L2范数
)

# 在训练循环中
optimizer.step()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.zero_grad()
```

**梯度裁剪阈值建议**：

| 模型规模 | 推荐 clip_norm |
|----------|---------------|
| 7B | 1.0 ~ 2.0 |
| 13B | 0.5 ~ 1.5 |
| 33B+ | 0.3 ~ 1.0 |

### 6.4 其他训练技巧

| 技巧 | 说明 | 注意事项 |
|------|------|----------|
| 混合精度训练（FP16/BF16） | 减少显存，加速训练 | 梯度溢出时切换BF16 |
| 梯度检查点（Gradient Checkpointing） | 用时间换显存 | 会使训练速度降低约20%~30% |
| DeepSpeed ZeRO | 分布式优化器状态分片 | Stage 2/3 最常用 |
| 早停（Early Stopping） | 监控验证集 loss 停止训练 | 防止过拟合 |
| 权重衰减（Weight Decay） | L2正则化，防止过拟合 | LoRA 中通常只对 A、B 设置 |

| 1 | B | LoRA 通过低秩矩阵分解（$\Delta W = BA$）减少训练参数量；秩 r 越大训练越慢；LoRA 广泛用于 LLM。 |

| 3 | B | `target_modules` 指定在哪些模块的 Linear 层添加 LoRA 低秩适配器。 |

### 二、简答题答案

**5. LoRA 的核心数学原理**

LoRA 的核心是将预训练权重矩阵的变化量 $\Delta \mathbf{W}$ 假设为低秩矩阵。设原始权重 $\mathbf{W}_0 \in \mathbb{R}^{d \times k}$，LoRA 冻结 $\mathbf{W}_0$，仅训练 $\mathbf{A} \in \mathbb{R}^{d \times r}$ 和 $\mathbf{B} \in \mathbb{R}^{r \times k}$（$r \ll \min(d, k)$），最终权重为：

$$\mathbf{W}_{ft} = \mathbf{W}_0 + \frac{\alpha}{r} \mathbf{B}\mathbf{A}$$

**低秩假设合理的理由**：
- 预训练模型已掌握了大量通用特征表示
- 特定任务的微调是对已有特征的**线性组合调整**，而非重建全新特征
- 实证研究（Aghajanyan et al., 2020）表明：微调后权重的变化量 $\Delta \mathbf{W}$ 的本征维度（intrinsic dimension）远小于原始维度
- 例如 7B 模型的全参数量为 $7 \times 10^9$，但仅需 $r=8$（约 800 万参数）即可达到全量微调 90% 以上的效果

---

**6. QLoRA 的三大核心技术创新**

**① NF4（4-bit NormalFloat）数据类型**

标准 INT4 使用均匀分布的 16 个离散值 $\{0,1,...,15\}$，但神经网络权重服从非均匀分布（接近正态分布）。NF4 基于分位数设计非均匀量化边界，使每个量化值代表的信息量基本相等，从而在相同比特数下大幅减少量化误差。

$$\text{NF4}_i = \text{quantile}\left(\mathcal{N}(0,1), \frac{i}{16}\right), \quad i = 0, 1, ..., 15$$

**② 双重量化（Double Quantization）**

量化过程需要存储缩放因子 $s$（每个量化组一个，通常为 FP32）。双重量化进一步对缩放因子本身做量化（用 INT8），将每个缩放因子的存储从 4 字节降至 1 字节。

**③ 分页优化器（Paged Optimizers）**

传统 Adam 优化器的动量参数大小约为模型参数的 2 倍。在梯度累积清空时，显存峰值可能达到优化器状态的 3 倍。分页优化器将优化器状态卸载到 CPU 内存，在 GPU 显存不足时自动换入换出，避免 OOM。

| 技术 | 显存节省 | 精度影响 |
|------|----------|----------|
| 4-bit NF4 | ~75% vs FP16 | < 0.5% 性能损失 |
| 双重量化 | ~0.4 bits/参数 | 可忽略 |
| 分页优化器 | 峰值显存 ↓50% | 无 |

**7. 两个训练配置的作用分析**

**`gradient_checkpointing=True`（梯度检查点）**

核心思想：以 **时间换空间**。Transformer 的每一层都保留中间激活值用于反向传播。开启梯度检查点后，每隔 $n$ 层才保存一次激活值，反向传播时重新计算被跳过的激活值。

- 显存节省：约 **30%~50%**
- 计算开销增加：约 **20%~30%**

**`per_device_train_batch_size=2, gradient_accumulation_steps=8`**

- 每设备实际 batch：2
- 梯度累积步数：8
- **有效 batch size** = $2 \times 8 = 16$

**两者如何配合**：梯度检查点腾出显存空间 → 允许设置更大的 `per_device_train_batch_size` 或增加模型规模 → 梯度累积确保等效的大 batch 训练效果。

**9. 数据增强函数实现**

```python
import re
from typing import List, Dict

def augment_qa(
    instruction: str, 
    output: str, 
    n_variants: int
) -> List[Dict]:
    """
    将问答对改写为多个不同版本的表述
    策略：同义词替换 + 句式变换 + 指令改写
    """
    
    # 中文同义词词典（部分）
    synonyms = {
        "请": ["麻烦请", "劳烦", "麻烦", "请您"],
        "解释": ["说明", "阐述", "讲解", "解释一下"],
        "什么": ["啥", "什么东西", "哪些"],
        "是": ["叫做", "称为", "指的是"],
        "如何": ["怎样", "怎么样", "怎么"],
        "为什么": ["为何", "缘由是", "原因是"],
    }
    
    # 句式模板
    instruction_templates = [
        "{keyword}，{rest}",
        "请问{keyword}吗？{rest}",
        "我想了解{keyword}，{rest}",
        "关于{keyword}，{rest}",
        "能否{keyword}？{rest}",
    ]
    
    # 标点符号变化
    punctuation_variants = ["。", "？", "，然后呢？", "具体说说。"]
    
    results = []
    
    for i in range(n_variants):
        new_instruction = instruction
        new_output = output
        
        # 策略1：同义词替换（随机替换1~2个关键词）
        words = list(synonyms.keys())
        num_replacements = min(i % 3 + 1, 2)  # 1~2次替换
        
        for _ in range(num_replacements):
            for word in words:
                if word in new_instruction:
                    replacement = synonyms[word][i % len(synonyms[word])]
                    new_instruction = new_instruction.replace(word, replacement, 1)
                    break
        
        # 策略2：句式重组（仅第一个版本使用）
        if i == 0 and "请" in new_instruction:
            new_instruction = new_instruction.replace("请", "请问")
        
        # 策略3：标点变化（影响回答的语气）
        if i % 2 == 0 and not new_output.endswith("。"):
            new_output = new_output + "。"
        
        # 策略4：添加确认词
        confirmations = ["", "以上是答案。", "希望对你有帮助。", "具体如下："]
        new_output = new_output + confirmations[i % len(confirmations)]
        
        # 限制输出长度，避免过长
        if len(new_output) > len(output) * 1.5:
            new_output = output  # 保持原始输出
        
        results.append({
            "instruction": new_instruction,
            "output": output  # 回答内容保持不变，增强的是问题多样性
        })
    
    # 确保至少有 n_variants 个（通过简单复制填充）
    while len(results) < n_variants:
        results.append({"instruction": instruction, "output": output})
    
    return results[:n_variants]

# ==================== 使用示例 ====================
if __name__ == "__main__":
    instruction = "请解释什么是人工智能"
    output = "人工智能是指由人工制造的机器所表现出来的智能行为，包括学习、推理、规划等能力。"
    
    augmented = augment_qa(instruction, output, n_variants=5)
    
    for i, item in enumerate(augmented, 1):
        print(f"版本 {i}:")
        print(f"  问题: {item['instruction']}")
        print(f"  回答: {item['output']}")
        print()
```
