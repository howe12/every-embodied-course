## 1. RAG概述

### 1.1 什么是RAG

**RAG**（Retrieval-Augmented Generation，检索增强生成）是一种结合了信息检索与大语言模型生成能力的技术架构。其核心思想是：在生成回答之前，先从外部知识库中检索出与问题相关的内容，然后将检索到的内容作为上下文提供给LLM，由LLM基于这些内容生成更准确、更可靠的答案。

RAG的工作流程可以用下图概括：

```
用户问题 ──► 检索模块 ──► 相关文档 ──► 生成模块 ──► 最终回答
              │                            ▲
              └──── 外部知识库（向量库）─────┘
```

RAG最早由 Meta AI 研究团队在2020年发表的论文《Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks》中提出，旨在解决大语言模型在知识密集型任务中的局限性。

### 1.2 为什么需要RAG

大语言模型虽然具备强大的通用能力，但在实际应用中仍面临以下挑战：

| 问题 | 描述 | RAG的解决方式 |
|------|------|--------------|
| **知识过时** | 模型的训练数据有截止日期，无法获取最新信息 | 实时从外部知识库检索最新内容 |
| **幻觉问题** | 模型可能生成看似合理但错误的内容（幻觉） | 提供真实文档作为依据，减少幻觉 |
| **知识盲区** | 模型对垂直领域专业知识掌握不足 | 接入领域知识库，补充专业知识 |
| **无法溯源** | 模型的输出难以验证来源 | 显式检索文档，使回答可溯源 |
| **数据安全** | 企业不希望将私有数据用于训练 | 私有数据留存在本地，仅用于检索 |

### 1.3 RAG vs Fine-tuning

在提升LLM在特定领域表现的方法中，RAG和Fine-tuning（微调）是最常用的两种手段。

| 对比维度 | RAG | Fine-tuning |
|----------|-----|-------------|
| **实现原理** | 检索外部知识，注入上下文 | 在特定数据上继续训练模型 |
| **知识更新** | 实时更新知识库即可 | 需要重新训练模型 |
| **计算成本** | 较低，无需重新训练 | 较高，需要GPU和训练时间 |
| **时效性** | 高，可随时更新 | 低，训练数据有截止日期 |
| **适用场景** | 知识密集型、动态数据 | 任务特定、模式学习 |

**选择建议**：
- **选RAG**：数据频繁更新、知识库庞大、需要可解释性、注重数据安全
- **选Fine-tuning**：任务模式固定、需要特定的输出风格
- **两者结合**：在Fine-tuned模型基础上叠加RAG，可取得最佳效果

## 2. RAG架构

RAG系统的核心架构分为三个主要阶段：**检索（Retrieval）**、**增强（Augmentation）**、**生成（Generation）**。

### 2.1 Retrieval（检索）

检索是RAG的第一步，负责从知识库中找到与用户问题最相关的文档片段。其核心流程如下：

```
用户问题 ──► 向量化（Embedding）──► 向量相似度搜索 ──► Top-K 相关文档
```

**检索的三个关键步骤**：
1. **向量化（Embedding）**：将用户问题转换为稠密向量表示
2. **向量相似度计算**：使用余弦相似度、点积等度量方法
3. **排序筛选**：根据相似度得分排序，返回Top-K个最相关的文档片段

### 2.2 Generation（生成）

生成阶段将检索到的文档与用户问题组合成Prompt，交给LLM生成最终答案。

### 2.3 完整流程

完整的RAG pipeline分为离线阶段和在线阶段：

```
离线阶段：文档加载 → 文本分割 → 向量化 → 存入向量数据库
在线阶段：用户问题 → 向量化 → 检索 → 构造Prompt → LLM生成
```

## 3. 文档处理

文档处理是RAG系统的基础环节，文档质量直接影响检索效果。

### 3.1 文档加载

LangChain提供了丰富的文档加载器：

```python
from langchain_community.document_loaders import (
    PyPDFLoader,          # PDF文件
    TextLoader,           # 纯文本文件

    Docx2txtLoader,      # Word文档
)

pages = loader.load_and_split()  # 按页分割

# 文本文件加载
loader = TextLoader("example.txt", encoding="utf-8")
documents = loader.load()
```

### 3.2 文本分割

文本分割（Chunking）是将长文档切分为小片段的关键步骤。

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,      # 每个片段的目标字符数
    chunk_overlap=50,   # 片段间的重叠字符数（保留上下文）
    length_function=len,
)

chunks = text_splitter.split_documents(documents)
text_chunks = text_splitter.split_text(raw_text)
```

**chunk_size 和 chunk_overlap 的选择**：
- **chunk_size过小**：丢失上下文，检索片段可能不完整
- **chunk_size过大**：超出模型上下文，检索效率降低
- **chunk_overlap作用**：保留片段边界处的上下文，避免关键信息被切断

一般建议chunk_size在300-1000字符之间，overlap在50-100字符之间。

### 3.3 向量化

向量化是将文本转换为稠密向量的过程，是RAG检索的核心。

### 4.4 相似度检索

```python
results = vectorstore.similarity_search(query="机器人控制的相关内容", k=5)

results_with_scores = vectorstore.similarity_search_with_score(
    query="机器人的运动控制原理", k=5
)

for doc, score in results_with_scores:
    print(f"相似度分数: {score:.4f}")
    print(f"内容: {doc.page_content}")

# 带过滤条件的检索
results = vectorstore.similarity_search(
    query="激光雷达的原理", k=5,
    filter={"source": "sensor.md"}
)
```

| 方法 | 公式 | 特点 |
|------|------|------|
| **余弦相似度** | $$\cos(\theta) = \frac{A \cdot B}{\|A\|\|B\|}$$ | 关注方向，不受长度影响 |
| **点积** | $$A \cdot B = \sum_{i=1}^{n} A_i \times B_i$$ | 计算简单，对长度敏感 |
| **欧氏距离** | $$d(A,B) = \sqrt{\sum_{i=1}^{n}(A_i - B_i)^2}$$ | 物理距离，与尺度相关 |

## 5. RAG优化

### 5.1 Query改写

Query改写通过对用户问题进行优化，提升检索质量。

```python
from langchain_openai import ChatOpenAI
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate

rewrite_prompt = PromptTemplate.from_template(
    """你是一个Query改写专家。请将用户的问题改写得更清晰、更具体，
使其更适合进行信息检索。改写时注意：
1. 保留原问题的核心意图
2. 补充必要的上下文信息
3. 使用更精确的专业术语
4. 分解复杂问题为简单问题

原始问题：{original_query}

改写后的问题："""
)

llm = ChatOpenAI(model="gpt-4o", temperature=0)
rewrite_chain = LLMChain(llm=llm, prompt=rewrite_prompt)
rewritten_query = rewrite_chain.run("机器人的定位")
print(f"改写后: {rewritten_query}")
```

| 策略 | 说明 | 示例 |
|------|------|------|
| **Query扩展** | 添加同义词和相关概念 | "机器人定位" → "机器人定位 OR 机器人SLAM" |
| **Query分解** | 将复杂问题拆分为子问题 | "机器人的感知、规划和控制" → 分别检索 |
| **Query消歧** | 消除歧义，明确指代 | "它的精度" → "UR5机械臂的定位精度" |
| **HyDE方法** | 让LLM生成假设性答案再检索 | 先猜答案，再基于答案检索 |

### 5.2 混合检索

混合检索结合多种检索方法，弥补单一检索的不足。

```python
from langchain.retrievers import EnsembleRetriever
from langchain_community.retrievers import BM25Retriever
from langchain_community.vectorstores import FAISS

# 稀疏检索：BM25
bm25_retriever = BM25Retriever.from_texts(text_chunks, k=5)

# 稠密检索：向量相似度
faiss_vectorstore = FAISS.from_texts(text_chunks, embeddings)
faiss_retriever = faiss_vectorstore.as_retriever(search_kwargs={"k": 5})

# 混合检索：加权组合
ensemble_retriever = EnsembleRetriever(
    retrievers=[bm25_retriever, faiss_retriever],
    weights=[0.3, 0.7]
)

results = ensemble_retriever.get_relevant_documents("机器人的运动控制")
```

### 5.3 重排序（Re-ranking）

重排序是在初步检索后，使用更精确的模型对结果进行二次排序。

```python
from langchain_community.retrievers import CohereRerank

reranker = CohereRerank(
    retriever=vectorstore.as_retriever(search_kwargs={"k": 20}),
    cohere_api_key="your-api-key",
    top_n=5
)

results = reranker.get_relevant_documents("机器人的路径规划算法有哪些？")
```

| 模型 | 说明 | 特点 |
|------|------|------|
| **Cohere Rerank** | 商用量化服务 | 效果好，付费使用 |
| **BGE Reranker** | 国产开源重排序模型 | 开源免费，需本地部署 |

```python
def cosine_similarity(v1, v2):
    return np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2) + 1e-8)

def search_mock(query, top_k=3):
    query_seed = int(hashlib.md5(query.encode()).hexdigest(), 16) % (2**32)
    np.random.seed(query_seed)

    similarities = []
    for i, chunk_vec in enumerate(mock_vecs):

        similarities.append((i, sim))
```

### 6.5 完整RAG Pipeline

```python
class SimpleRAG:
    def __init__(self, vectorstore, llm_model="gpt-4o"):
        self.vectorstore = vectorstore
        self.llm = ChatOpenAI(model=llm_model, temperature=0.3)
    
    def retrieve(self, query, top_k=3):
        return self.vectorstore.similarity_search(query, k=top_k)
    
    def generate(self, query, retrieved_docs):
        context = "\n\n".join([doc.page_content for doc in retrieved_docs])
        prompt = PromptTemplate.from_template(
            """你是一个专业的机器人技术助手。请根据以下参考文档回答用户的问题。
要求：
1. 只基于参考文档内容回答，不要编造信息
2. 如果参考文档中没有相关信息，请如实说明

## 参考文档：
{context}

## 用户问题：
{question}

## 回答："""
        )
        chain = prompt | self.llm
        response = chain.invoke({"context": context, "question": query})
        return response.content
    
    def answer(self, query, top_k=3):
        retrieved_docs = self.retrieve(query, top_k)
        answer = self.generate(query, retrieved_docs)
        return {"answer": answer, "sources": [doc.page_content for doc in retrieved_docs]}
```

**4. 简答题答案**

**6. 简答题答案**  
chunk_overlap是相邻文本片段之间的重叠字符数，作用是保留片段边界处的上下文，避免关键信息被切断。设置建议：chunk_size在300-1000字符，overlap在50-100字符之间，需根据实际效果调整。

**7. 综合设计题答案**（要点）
- a) 文档处理：PDF/Word等加载器、递归字符分割、chunk_size 500左右
- b) 向量数据库：FAISS或Chroma、中小规模用Flat索引、大规模用HNSW
- c) 检索策略：混合检索（BM25+向量），权重分配0.3:0.7
- d) Query改写：需要，尤其对于模糊或口语化问题
- e) 评估：检索召回率/精确率、生成回答质量（人工评估+RAGAS）、A/B测试
