## 1. 多模态概述

### 1.1 什么是多模态

**多模态（Multimodal）** 是指同时涉及两种或两种以上感官信息通道的计算与推理能力。在人工智能领域，"模态"通常指数据类型或信息表达形式，如文本、图像、音频、视频等。多模态智能的核心目标是让机器像人类一样，能够综合理解并生成多种模态的信息。

传统的人工智能系统往往只能处理单一模态的数据——例如专门处理文本的NLP模型或专门处理图像的计算机视觉模型。而多模态大模型（Multimodal Large Language Model，MLLM）则将多种模态的信息统一映射到一个共享的特征空间中，使模型能够同时理解和生成不同类型的数据。

多模态大模型的关键能力包括：

- **跨模态理解**：根据一种模态的信息理解另一种模态的内容（如看图说话）
- **跨模态生成**：根据一种模态的输入生成另一种模态的输出（如文生图）
- **模态融合**：综合多种模态的信息做出更全面的判断（如视频理解需要同时处理画面和声音）

### 1.2 模态类型

| 模态类型 | 数据形式 | 特征 | 典型任务 |
|----------|----------|------|----------|
| **文本（Text）** | 字符串、词序列 | 高语义密度、结构化 | 文本分类、机器翻译、文本生成 |
| **图像（Image）** | 像素矩阵、2D数组 | 空间结构性强、视觉丰富 | 图像分类、目标检测、图像分割 |
| **音频（Audio）** | 时序波形、频谱图 | 时序性强、包含语音/音乐/音效 | 语音识别、语音合成、情感识别 |
| **视频（Video）** | 图像序列+音频 | 时空结构、动态信息 | 动作识别、视频描述、视频生成 |
| **3D（Point Cloud/Depth）** | 3D坐标、体素 | 空间感知、深度信息 | 3D检测、SLAM、场景重建 |
| **触觉（Tactile）** | 压力阵列、力反馈 | 物理交互信息 | 灵巧操作、接触感知 |

在具身智能场景中，机器人需要同时处理视觉、触觉、本体感知（Proprioception）等多种模态，以实现精细的操作与导航任务。

### 1.3 多模态研究的意义

多模态研究的重要性体现在以下几个维度：

**第一，逼近人类感知方式。** 人类通过多通道感官协同认识世界——我们看到一只猫的同时会听到它的叫声，触摸物体会获得触觉反馈。单一模态的AI系统无法复现这种完整的感知体验，多模态是实现真正通用人工智能（AGI）的必经之路。

**第二，释放新的应用场景。** 许多真实世界的任务天然是多模态的：
- 具身智能机器人需要视觉+触觉+语言指令协同控制
- 医疗诊断需要结合医学影像（X光片、CT、MRI）和文字病历
- 自动驾驶需要融合摄像头画面、激光雷达点云、地图信息

**第三，提升模型鲁棒性与理解深度。** 多模态信号之间存在互补性。例如，仅依靠文本描述无法精确还原图像中的空间关系，而结合视觉信息则可以更准确地理解语义。跨模态的联合学习能够弥补单一模态的盲区。

**第四，推动内容创作革命。** 文生图（DALL·E、Stable Diffusion）、文生视频（Sora、Runway）等应用正在重塑创意产业的生产方式，多模态生成模型让"每个人都能成为艺术家"成为可能。

## 2. 多模态融合架构

多模态融合（Multimodal Fusion）是指将来自不同模态的数据或特征整合到统一表示中的技术路线。根据融合发生的阶段和方式的不同，可以分为早期融合、晚期融合、中期融合和注意力融合四大类。

### 2.1 早期融合（Early Fusion）

**早期融合**又称特征级融合，是在模型的输入层或浅层将不同模态的原始数据或低级特征拼接在一起，再送入统一的网络进行学习。

```
[图像特征] ──┐
            ├──→ [拼接层] → [共享Transformer] → 输出
[文本特征] ──┘
```

早期融合的优点：
- 能够学习模态间的底层关联（如颜色与描述词汇的对应关系）
- 端到端优化，整体性能潜力高
- 适合模态间存在强相关性的任务

早期融合的缺点：
- 要求不同模态的原始数据在时间或空间上对齐（Alignment）
- 模态之间数据结构差异大时，融合困难
- 原始数据维度高，计算开销大

数学上，若两个模态的特征向量为 $\mathbf{x}_v$ 和 $\mathbf{x}_t$，早期融合后的联合表示为：

$$\mathbf{x}_{fuse} = [\mathbf{x}_v; \mathbf{x}_t] = \text{Concat}(\mathbf{x}_v, \mathbf{x}_t)$$

其中 $[;]$ 表示向量拼接操作。

### 2.2 晚期融合（Late Fusion）

**晚期融合**又称决策级融合，是让各个模态分别通过独立的编码器得到各自的输出（如类别概率、特征向量），再在决策层进行融合。融合方式可以是简单平均、加权投票或另一个轻量级学习器。

```
[图像] → [图像Encoder] → [图像分类头] ──┐
                                        ├──→ [融合层] → 最终决策
[文本] → [文本Encoder] → [文本分类头] ──┘
```

晚期融合的优点：
- 各模态独立训练，灵活性高，支持异构模型（如CNN处理图像、RNN处理文本）
- 模态之间无需严格对齐
- 某一模态缺失时仍能工作（鲁棒性强）

晚期融合的缺点：
- 无法捕获模态之间的底层交互信息
- 融合层设计不当容易成为性能瓶颈

晚期融合的融合公式（以加权平均为例）：

$$P_{final}(y|\mathbf{x}_v, \mathbf{x}_t) = w_v \cdot P_v(y|\mathbf{x}_v) + w_t \cdot P_t(y|\mathbf{x}_t)$$

其中 $w_v + w_t = 1$ 为融合权重。

### 2.3 中期融合（Mid-level Fusion）

**中期融合**介于早期融合和晚期融合之间，是在各模态完成初步编码后、进入深层网络之前进行融合。这种方式既保留了一定的模态交互能力，又避免了早期融合的复杂度。

```
[图像] → [图像Encoder(浅层)] → [图像中层特征] ──┐
                                                  ├──→ [共享深层网络] → 输出
[文本] → [文本Encoder(浅层)] → [文本中层特征] ──┘
```

中期融合的典型应用包括双塔检索模型（如CLIP）：图像和文本分别通过独立的Encoder得到中等维度的表示，然后通过一个跨模态对比损失函数进行对齐。

### 2.4 注意力融合（Attention-based Fusion）

**注意力融合**是当前多模态模型最主流的融合方式，核心思想是通过注意力机制动态地决定在处理某一模态时，应该关注另一模态的哪些部分。

跨模态注意力的计算过程如下：

给定图像特征序列 $\mathbf{V} = \{\mathbf{v}_1, \mathbf{v}_2, ..., \mathbf{v}_n\}$ 和文本特征序列 $\mathbf{T} = \{\mathbf{t}_1, \mathbf{t}_2, ..., \mathbf{t}_m\}$，跨模态注意力输出为：

$$\alpha_{ij} = \frac{\exp(\mathbf{t}_i^\top \mathbf{W}_q^\top \mathbf{W}_k \mathbf{v}_j)}{\sum_{k=1}^{n} \exp(\mathbf{t}_i^\top \mathbf{W}_q^\top \mathbf{W}_k \mathbf{v}_k)}$$

$$\mathbf{t}_i' = \sum_{j=1}^{n} \alpha_{ij} \mathbf{W}_v \mathbf{v}_j$$

其中 $\mathbf{W}_q, \mathbf{W}_k, \mathbf{W}_v$ 为可学习的投影矩阵，$\alpha_{ij}$ 表示文本第 $i$ 个位置对图像第 $j$ 个位置的注意力权重。

注意力融合的优势在于：
- 可以处理任意长度和结构的跨模态序列
- 灵活建模细粒度的跨模态对应关系
- 与Transformer架构天然兼容，易于端到端训练

## 3. 视觉-语言模型

视觉-语言模型（Vision-Language Model，VLM）是指能够同时理解图像和文本的深度学习模型，是多模态研究中最活跃的分支。本节介绍当前最具代表性的几个VLM架构。

### 3.1 CLIP模型

**CLIP**（Contrastive Language-Image Pre-training）由OpenAI于2021年提出，是视觉-语言预训练的里程碑工作。CLIP的核心创新在于**大规模对比学习**：通过收集4亿个图像-文本配对数据，训练一个双塔（Dual-Encoder）结构，使得匹配的图像和文本在特征空间中距离相近，不匹配的则距离遥远。

CLIP的架构包含两个分支：
- **图像Encoder**：通常为ViT（Vision Transformer）或ResNet，负责将图像映射为特征向量
- **文本Encoder**：基于Transformer，将文本映射为特征向量

CLIP的对比学习损失函数定义为：

$$\mathcal{L} = -\frac{1}{2N}\sum_{i=1}^{N} \left[ \log \frac{\exp(\mathbf{v}_i^\top \mathbf{t}_i / \tau)}{\sum_{j=1}^{N} \exp(\mathbf{v}_i^\top \mathbf{t}_j / \tau)} + \log \frac{\exp(\mathbf{t}_i^\top \mathbf{v}_i / \tau)}{\sum_{j=1}^{N} \exp(\mathbf{t}_i^\top \mathbf{v}_j / \tau)} \right]$$

其中 $\tau$ 为温度参数，$\mathbf{v}_i$ 和 $\mathbf{t}_i$ 分别为归一化后的图像和文本特征向量。

CLIP的零样本（Zero-shot）分类能力是其最引人注目的特点：无需在任何特定数据集上微调，只需提供类别名称的文本描述，就能对图像进行分类。这打破了传统视觉模型"一类别一分类器"的范式。

### 3.2 BLIP系列

**BLIP**（Bootstrapping Language-Image Pre-training）由Salesforce研究院提出，其核心贡献是提出了**Bootstrapping（自举）**训练策略，并通过统一的框架同时支持理解和生成两类任务。

BLIP的两个主要版本：

| 版本 | 提出时间 | 主要改进 |
|------|----------|----------|
| BLIP | 2022年 | 统一框架（ITC+ITM+LM损失联合训练），支持图文检索和描述生成 |
| BLIP-2 | 2023年 | 引入Q-Former架构，冻结LLM和视觉Encoder，大幅减少训练参数量 |

BLIP-2的关键创新是**Q-Former**（Querying Transformer）模块：
- Q-Former由Transformer编码器和解码器组成
- 从图像Encoder输出的特征中 Queries（查询）出与文本最相关的视觉信息
- 适配器（Adaptor）将Q-Former的输出投影到LLM的输入空间

这种"冻结主干、训练适配器"的设计，使得BLIP-2只需训练约1%的参数，就能快速适配新的LLM。

### 3.3 LLaVA

**LLaVA**（Large Language and Vision Assistant）是开源社区最具影响力的开源多模态模型之一，由威斯康星大学麦迪逊分校和微软研究院联合提出。LLaVA的核心设计原则是**极简架构 + 大规模数据**。

LLaVA采用"视觉Encoder + 投影层 + LLM"的简单范式：

```
图像 → [CLIP ViT-L] → [线性投影层] → [文本token序列] → [Vicuna LLM] → 文本输出
```

LLaVA-1.5通过以下改进大幅提升了性能：
- 使用MLP投影层替代线性投影，增强表示能力
- 引入学术任务数据集（VQA、OCR、Region Understanding）进行指令微调
- 增加图像分辨率至336×336，提升细节感知能力

LLaVA的开源特性使得它在学术研究和工业落地中被广泛采用，是多模态入门和实践的首选模型之一。

### 3.4 MiniGPT-4

**MiniGPT-4**是由阿布杜拉国王科技大学（KAUST）提出的轻量级多模态模型，其核心目标是以最小的训练代价将强大的LLM能力迁移到视觉领域。

MiniGPT-4的设计同样极简：
- **视觉编码器**：使用与BLIP-2相同的CLIP ViT
- **投影层**：一个线性层，将视觉特征投影到LLM的输入空间
- **语言模型**：使用Vicuna（基于LLaMA微调的对话模型）

MiniGPT-4的训练分为两个阶段：
1. **预训练阶段**：在大量图像-文本对上训练投影层，学习跨模态对齐
2. **指令微调阶段**：使用高质量的图像-描述数据微调，提升对话能力

MiniGPT-4的论文还特别强调了**语言生成幻觉问题**（Hallucination）——即模型可能生成与图像内容不符的文本描述，这是当前多模态模型面临的重要挑战之一。

### 3.5 主流VLM对比

| 模型 | 视觉Encoder | LLM | 训练策略 | 特点 |
|------|------------|-----|---------|------|
| CLIP | ViT/ResNet | 无 | 对比学习 | 零样本分类最强，开源权重 |
| BLIP-2 | EVA-CLIP | Flan-T5/Vicuna | 预训练+适配 | Q-Former桥接，参数高效 |
| LLaVA | CLIP ViT-L | Vicuna | 指令微调 | 开源易用，社区活跃 |
| MiniGPT-4 | CLIP ViT | Vicuna | 两阶段训练 | 轻量级，效果惊艳 |
| GPT-4V | 闭源 | GPT-4 | 闭源训练 | 性能最强，API调用 |
| Gemini | 闭源 | Gemini | 多模态原生 | 原生多模态，视频理解 |

## 4. 多模态预训练

多模态预训练是指在大规模多模态数据（通常是图像-文本对或视频-文本对）上，通过自监督学习的方式训练模型，使其习得通用的跨模态表示能力。根据学习范式的不同，主流的多模态预训练方法可以分为三大类：对比学习、遮蔽重建和跨模态对齐。

### 4.1 对比学习（Contrastive Learning）

对比学习的核心思想是：**让正样本对（匹配的跨模态数据）的特征表示在特征空间中相互靠近，让负样本对（不匹配的数据）的特征表示相互远离**。

在图像-文本场景中，给定一个批次（Batch）中的 $N$ 个图像-文本配对样本，对比学习的目标是最大化对角线上的相似度（正样本对），最小化非对角线上的相似度（负样本对）。

对比学习的目标函数（InfoNCE）为：

$$\mathcal{L}_{contrastive} = -\frac{1}{N}\sum_{i=1}^{N} \log \frac{\exp(\text{sim}(\mathbf{v}_i, \mathbf{t}_i)/\tau)}{\sum_{j=1}^{N} \exp(\text{sim}(\mathbf{v}_i, \mathbf{t}_j)/\tau)}$$

其中 $\text{sim}(\mathbf{v}, \mathbf{t}) = \frac{\mathbf{v}^\top \mathbf{t}}{\|\mathbf{v}\|\|\mathbf{t}\|}$ 为余弦相似度，$\tau > 0$ 为温度系数。

对比学习的优势在于：
- **自监督信号天然丰富**：不需要人工标注的语义标签
- **批次越大、负样本越多、效果越好**（但计算成本也随之上升）
- 能够学习到细粒度的跨模态语义对应关系

CLIP、ALIGN等模型就是对比学习路线的代表。

### 4.2 遮蔽重建（Masked Reconstruction）

遮蔽重建是自编码思想的跨模态扩展，核心任务是**随机遮蔽（Mask）输入的一部分（如图像块或文本词），然后让模型重建被遮蔽的内容**。

**遮蔽图像建模（Masked Image Modeling，MIM）**：

在视觉侧随机遮蔽一定比例的图像块（如ViT中的patch），然后用未被遮蔽的块来预测被遮蔽块的内容。典型代表如BEiT、MAE等。

$$\mathcal{L}_{MIM} = \mathbb{E}_{m \sim M} \left[ -\log p_{\theta}(\mathbf{x}_m | \mathbf{x}_{\setminus m}) \right]$$

其中 $m$ 为被遮蔽的位置集合，$\mathbf{x}_{\setminus m}$ 为未被遮蔽的观测。

**遮蔽语言建模（Masked Language Modeling，MLM）**：

在文本侧随机遮蔽一些词元，让模型根据上下文预测被遮蔽的词。BERT首先提出这一范式。

**跨模态遮蔽建模**：

在多模态场景下，可以遮蔽文本让模型根据图像信息重建（如Flamingo），或遮蔽图像块让模型根据文本重建，从而强迫模型学习跨模态的互补信息。

### 4.3 跨模态对齐（Cross-modal Alignment）

跨模态对齐的目标是将不同模态的数据映射到同一个共享的特征空间，使得语义相近的跨模态样本在该空间中距离接近。

**对比对齐（Contrastive Alignment）**：

使用对比损失（如前文所述的InfoNCE）拉近配对样本的特征距离，推开非配对样本。这是CLIP、ALIGN等模型的核心方法。

**匹配预测（Modal Matching）**：

将图像-文本配对预测作为一个二分类任务：给定图像和文本对，预测它们是否匹配（match/unmatch）。这对应于BLIP中的ITM（Image-Text Matching）损失：

$$\mathcal{L}_{ITM} = -\left[ y \log p_{match} + (1-y) \log (1-p_{match}) \right]$$

其中 $y=1$ 表示正样本对（匹配），$y=0$ 表示负样本对（不匹配）。

**Captioning Loss（描述生成损失）**：

让模型根据图像特征生成对应的文本描述，训练一个从视觉到语言的生成模型。这是LLaVA等生成式多模态模型的核心训练目标：

$$\mathcal{L}_{cap} = -\sum_{t=1}^{T} \log p_{\theta}(w_t | \mathbf{v}, w_{1:t-1})$$

其中 $w_t$ 为第 $t$ 个生成的词。

## 5. 多模态应用

多模态技术已经在各行各业实现了广泛的应用落地。本节重点介绍四种最具代表性的应用场景：图像描述生成、视觉问答、图文检索和文本到图像生成。

### 5.1 图像描述生成（Image Captioning）

图像描述生成是指为给定图像自动生成一段自然语言描述的任务。它是图像理解和语言生成的交叉领域，要求模型同时具备视觉感知和语言组织能力。

一个典型的图像描述由以下几部分构成：

| 描述粒度 | 示例 | 特点 |
|----------|------|------|
| 简单描述 | "一只狗在草地上奔跑" | 描述主要对象和动作 |
| 细节描述 | "一只金毛犬在阳光明媚的公园草地上欢快地奔跑" | 增加颜色、环境、情感等细节 |
| 故事描述 | "午后，一只金毛犬在公园追着飞盘，尽情享受着夏日的欢乐时光" | 赋予场景叙事性 |

图像描述模型通常采用编码器-解码器（Encoder-Decoder）架构：
- **编码器**：CNN或ViT负责从图像中提取视觉特征
- **解码器**：LSTM或Transformer负责将视觉特征解码为文本序列

### 5.2 视觉问答（Visual Question Answering，VQA）

视觉问答是指根据给定图像内容，自动回答相关自然语言问题的任务。与图像描述不同，VQA需要模型理解问题的语义并针对性地从图像中提取相关信息。

VQA问题的类型可以分为：

| 问题类型 | 示例 | 所需能力 |
|----------|------|----------|
| 开放域 | "这张图中发生了什么？" | 综合理解 |
| 计数类 | "图中有几只鸟？" | 目标检测+计数 |
| 是非类 | "天是蓝色的吗？" | 属性识别 |
| 位置类 | "猫在哪里？" | 空间推理 |
| 属性类 | "这件衬衫是什么颜色？" | 细粒度识别 |

现代VQA系统通常将图像特征、问题文本特征一起输入到联合编码器中，通过多模态融合得到统一表示，再解码出答案。

### 5.3 图文检索（Image-Text Retrieval）

图文检索包括两个方向：**文搜图**（Text-to-Image）和**图搜文**（Image-to-Text）。其核心思想是将图像和文本分别编码为向量，通过向量相似度（如余弦相似度）进行匹配排序。

```
查询文本 → [文本Encoder] → 文本向量 → 余弦相似度排序 → Top-K 图像
候选图像 → [图像Encoder] → 图像向量 ─────────────────┘
```

图文检索的评价指标通常为Recall@K（召回率），即正确结果出现在Top-K结果中的比例。

### 5.4 文本到图像生成（Text-to-Image Generation）

文本到图像生成是指根据自然语言描述自动生成对应图像的技术。近年来，扩散模型（Diffusion Model）取代GAN成为该领域的主导技术。

主流的文生图模型包括：

| 模型 | 机构 | 特点 |
|------|------|------|
| DALL·E 3 | OpenAI | GPT-4V加持，理解复杂提示词 |
| Stable Diffusion | Stability AI | 开源，LoRA微调生态丰富 |
| Midjourney | Midjourney | 艺术风格强，社区活跃 |
| Imagen | Google | 高分辨率，逼真度高 |
| SDXL | Stability AI | 1024×1024大分辨率，细节丰富 |

扩散模型的工作原理是逐步从噪声中恢复出图像：

$$p_{\theta}(\mathbf{x}_0 | \mathbf{c}) = \prod_{t=1}^{T} p_{\theta}(\mathbf{x}_{t-1} | \mathbf{x}_t, \mathbf{c})$$

其中 $\mathbf{c}$ 为文本条件（Condition），$\mathbf{x}_T$ 是纯噪声，$\mathbf{x}_0$ 是生成的图像。

### 6.1 CLIP图像特征提取

本节演示如何使用CLIP模型提取图像和文本的特征向量，并进行相似度计算。

```python
# 导入必要的库
import torch
import numpy as np
from PIL import Image
import clip  # OpenAI CLIP模型库

# 检查设备（优先使用GPU）
device = "cuda" if torch.cuda.is_available() else "cpu"
print(f"使用设备: {device}")

# 加载CLIP模型和预处理
# clip.available_models() 可以查看可用的模型列表
model, preprocess = clip.load("ViT-L/14", device=device)
print("CLIP模型加载完成: ViT-L/14")

# ==================== 示例1：单张图像特征提取 ====================
def extract_image_features(image_path, model, preprocess, device):
    """
    提取图像的特征向量
    
    参数:
        image_path: 图像文件路径
        model: CLIP模型
        preprocess: 图像预处理模块
        device: 运行设备
    返回:
        image_features: 归一化后的图像特征向量 [1, embed_dim]
    """
    # 加载并预处理图像
    image = Image.open(image_path).convert("RGB")
    # preprocess会自动完成: 调整大小 + 中心裁剪 + 归一化
    image_input = preprocess(image).unsqueeze(0).to(device)
    
    with torch.no_grad():
        # 通过CLIP图像编码器提取特征
        image_features = model.encode_image(image_input)
        # 归一化特征向量（用于余弦相似度计算）
        image_features = image_features / image_features.norm(dim=-1, keepdim=True)
    
    return image_features, image_input.shape

# ==================== 示例2：文本特征提取 ====================
def extract_text_features(texts, model, device):
    """
    提取文本的特征向量
    
    参数:
        texts: 文本列表（支持多文本批量处理）
        model: CLIP模型
        device: 运行设备
    返回:
        text_features: 归一化后的文本特征向量 [N, embed_dim]
    """
    # 对文本进行Tokenize编码
    text_tokens = clip.tokenize(texts).to(device)
    
    with torch.no_grad():
        # 通过CLIP文本编码器提取特征
        text_features = model.encode_text(text_tokens)
        # 归一化
        text_features = text_features / text_features.norm(dim=-1, keepdim=True)
    
    return text_features

# ==================== 示例3：图文相似度计算 ====================
def compute_similarity(image_path, texts, model, preprocess, device):
    """
    计算图像与多个文本描述之间的相似度
    
    参数:
        image_path: 图像路径
        texts: 文本描述列表
        model: CLIP模型
        preprocess: 预处理模块
        device: 运行设备
    返回:
        similarities: 每个文本与图像的余弦相似度分数
    """
    # 提取图像特征
    image_features = extract_image_features(image_path, model, preprocess, device)
    # 提取多个文本的特征
    text_features = extract_text_features(texts, model, device)
    
    # 计算余弦相似度：矩阵乘法等价于余弦相似度（特征已归一化）
    # image_features: [1, 768], text_features: [N, 768]
    # 相乘结果: [1, N]，每行表示该图像与每个文本的相似度
    similarity = image_features[0] @ text_features.T
    
    return similarity.cpu().numpy()

# ==================== 示例4：零样本图像分类 ====================
def zero_shot_classify(image_path, class_names, model, preprocess, device):
    """
    基于CLIP的零样本图像分类
    
    参数:
        image_path: 图像路径
        class_names: 类别名称列表，如 ["猫", "狗", "鸟"]
        model: CLIP模型
        preprocess: 预处理模块
        device: 运行设备
    返回:
        predicted_class: 预测的类别名称
        probs: 每个类别的概率
    """
    # 构建完整的文本描述（可选策略，此处直接用类别名）
    # 更强的做法是加入模板："一张{class_name}的照片"
    texts = [f"a photo of a {c}" for c in class_names]
    
    # 提取图像和文本特征
    image_features = extract_image_features(image_path, model, preprocess, device)
    text_features = extract_text_features(texts, model, device)
    
    # 计算相似度并通过softmax转为概率
    logits_per_image = image_features[0] @ text_features.T
    probs = logits_per_image.softmax(dim=-1).cpu().numpy()
    
    # 获取最高概率的类别
    predicted_idx = np.argmax(probs)
    predicted_class = class_names[predicted_idx]
    
    return predicted_class, probs[0]

# ==================== 运行示例 ====================
if __name__ == "__main__":
    print("=== CLIP 特征提取演示 ===")

    ]
    
    print(f"\n候选文本: {sample_texts}")
```

```python
import torch

def load_llava_model(model_id="llava-hf/llava-1.5-7b-hf"):
    """
    加载LLaVA多模态对话模型
    
    参数:
        model_id: HuggingFace上的模型名称

        model: LLaVA模型实例
        processor: 图像和文本的预处理处理器

    device = "cuda" if torch.cuda.is_available() else "cpu"
    print(f"加载模型到设备: {device}")
    
    # 加载处理器（负责图像预处理和文本tokenize）
    processor = AutoProcessor.from_pretrained(model_id)
    
    # 加载LLaVA模型（因果语言模型 + 视觉编码器）
    model = LlavaForConditionalGeneration.from_pretrained(
        model_id,
        torch_dtype=torch.float16 if device == "cuda" else torch.float32,

        # load_in_4bit=True,
    )
    model = model.to(device)
    
    return model, processor, device

# ==================== 单轮对话 ====================
def llava_single_turn(model, processor, device, image_path, user_prompt):
    """
    LLaVA单轮对话：输入图像+文本问题，返回文本回答
    
    参数:
        model: LLaVA模型
        processor: 预处理处理器
        device: 运行设备
        image_path: 图像路径或URL
        user_prompt: 用户输入的文本问题
    返回:
        response_text: 模型生成的文本回答
    """
    # 加载图像（支持本地路径和URL）
    if image_path.startswith("http"):
        image = Image.open(requests.get(image_path, stream=True).raw).convert("RGB")
    else:
        image = Image.open(image_path).convert("RGB")
    
    # 构建对话模板
    # LLaVA使用特殊的对话格式：[USER]...[/USER][ASST]...[/ASST]
    conversation = [
        {
            "role": "user",
            "content": [
                {"type": "image"},
                {"type": "text", "text": user_prompt}
            ]
        }
    ]
    
    # 处理器将对话和图像转换为模型输入格式
    inputs = processor(
        conversation,
        images=image,
        return_tensors="pt",
        padding=True
    ).to(device)
    
    # 生成回答
    with torch.no_grad():
        output_ids = model.generate(
            **inputs,
            max_new_tokens=256,       # 最大生成长度
            do_sample=True,           # 采样而非贪心解码
            temperature=0.7,          # 采样温度
            top_p=0.9,                # nucleus sampling
            repetition_penalty=1.1,   # 防止重复生成
        )
    
    # 解码输出（跳过输入部分的token）
    input_length = inputs.input_ids.shape[1]
    generated_ids = output_ids[:, input_length:]
    response_text = processor.batch_decode(
        generated_ids, 
        skip_special_tokens=True
    )[0].strip()
    
    return response_text

    """
    LLaVA多轮对话：支持上下文连贯的对话

    conversation = []
    for role, content in dialogue_history:

            })

                "role": "assistant",
```

### 6.3 多模态检索示例

```python
import torch
import numpy as np
from PIL import Image
import clip
from sklearn.metrics.pairwise import cosine_similarity

# ==================== CLIP 特征提取工具类 ====================
class CLIPFeatureExtractor:
    """
    CLIP特征提取器封装
    支持图像和文本的特征提取与缓存
    """
    def __init__(self, model_name="ViT-L/14", device=None):
        # 初始化设备
        self.device = device or ("cuda" if torch.cuda.is_available() else "cpu")
        # 加载CLIP模型
        self.model, self.preprocess = clip.load(model_name, device=self.device)
        self.model.eval()  # 设置为评估模式
        # ViT-L/14 的嵌入维度为 768
        self.embed_dim = 768 if "L" in model_name else 512
        print(f"CLIP特征提取器就绪，嵌入维度: {self.embed_dim}")
    
    @torch.no_grad()
    def extract_image(self, image_path):
        """
        提取单张图像的特征向量
        """
        image = Image.open(image_path).convert("RGB")
        image_input = self.preprocess(image).unsqueeze(0).to(self.device)
        features = self.model.encode_image(image_input)
        features = features / features.norm(dim=-1, keepdim=True)
        return features.cpu().numpy().flatten()
    
    @torch.no_grad()
    def extract_text(self, text):
        """
        提取文本的特征向量
        """
        text_input = clip.tokenize([text]).to(self.device)
        features = self.model.encode_text(text_input)
        features = features / features.norm(dim=-1, keepdim=True)
        return features.cpu().numpy().flatten()
    
    @torch.no_grad()
    def extract_images(self, image_paths):
        """
        批量提取多张图像的特征向量
        """
        images = []
        for path in image_paths:
            img = Image.open(path).convert("RGB")
            images.append(self.preprocess(img))
        # 堆叠为批次 [N, C, H, W]
        image_batch = torch.stack(images).to(self.device)
        features =        features = self.model.encode_image(image_batch)
        features = features / features.norm(dim=-1, keepdim=True)
        return features.cpu().numpy()

# ==================== 图文检索引擎 ====================
class MultimodalRetriever:
    """
    基于CLIP的多模态检索引擎
    支持: 文搜图(text_search) 和 图搜文(image_search)
    """
    def __init__(self, extractor: CLIPFeatureExtractor):
        self.extractor = extractor
        # 图像库特征矩阵: [N, embed_dim]
        self.image_features = None
        # 图像ID列表
        self.image_ids = []
        # 文本库特征矩阵: [M, embed_dim]
        self.text_features = None
        # 文本ID列表
        self.text_ids = []
    
    def build_image_index(self, image_paths, batch_size=32):
        """
        构建图像索引
        
        参数:
            image_paths: 图像路径列表
            batch_size: 批量处理的尺寸
        """
        all_features = []
        for i in range(0, len(image_paths), batch_size):
            batch_paths = image_paths[i:i+batch_size]
            batch_features = self.extractor.extract_images(batch_paths)
            all_features.append(batch_features)
            print(f"已处理 {min(i+batch_size, len(image_paths))}/{len(image_paths)} 张图像")
        
        self.image_features = np.vstack(all_features)
        self.image_ids = list(range(len(image_paths)))
        print(f"图像索引构建完成: {self.image_features.shape}")
    
    def build_text_index(self, texts):
        """
        构建文本索引
        
        参数:
            texts: 文本列表
        """
        self.text_ids = list(range(len(texts)))
        self.text_features = np.array([
            self.extractor.extract_text(t) for t in texts
        ])
        print(f"文本索引构建完成: {self.text_features.shape}")
    
    def text_to_image_search(self, query_text, top_k=5):
        """
        文搜图: 根据文本查询最相关的图像
        
        参数:
            query_text: 查询文本
            top_k: 返回前k个最相关结果
        返回:
            results: [(image_id, score), ...] 按相似度降序排列
        """
        # 提取查询文本特征
        query_vec = self.extractor.extract_text(query_text).reshape(1, -1)
        
        # 计算与所有图像的余弦相似度
        similarities = cosine_similarity(query_vec, self.image_features)[0]
        
        # 排序并返回Top-K
        top_indices = np.argsort(similarities)[::-1][:top_k]
        results = [(self.image_ids[i], similarities[i]) for i in top_indices]
        
        return results
    
    def image_to_text_search(self, query_image_path, top_k=5):
        """
        图搜文: 根据图像查询最相关的文本
        
        参数:
            query_image_path: 查询图像路径
            top_k: 返回前k个最相关结果
        返回:
            results: [(text_id, score), ...] 按相似度降序排列
        """
        # 提取查询图像特征
        query_vec = self.extractor.extract_image(query_image_path).reshape(1, -1)
        
        # 计算与所有文本的余弦相似度
        similarities = cosine_similarity(query_vec, self.text_features)[0]
        
        # 排序并返回Top-K
        top_indices = np.argsort(similarities)[::-1][:top_k]
        results = [(self.text_ids[i], similarities[i]) for i in top_indices]
        
        return results
```

### 选择题

**1. 以下关于多模态融合的说法，错误的是？**

A. 早期融合在输入层进行模态拼接，适合模态数据结构差异大的场景
B. 晚期融合允许各模态使用独立的Encoder，灵活性高

### 简答题答案

**11. 早期融合与晚期融合对比：**

| | 早期融合 | 晚期融合 |
|---|---|---|
| 融合位置 | 输入层/浅层特征拼接 | 决策层/输出层融合 |
| 优点 | 端到端优化；能学习底层跨模态关联 | 模态独立，灵活性高；支持异构模型；某一模态缺失仍可工作 |
| 缺点 | 模态需对齐；数据差异大时融合困难；计算开销大 | 无法捕获底层跨模态交互；融合层设计复杂 |
| 适用场景 | 模态结构相近、数据对齐的场景（如音频+视频） | 模态异构、各自成熟的场景（如医学影像+病历文本） |

**12. 多模态是AGI必经之路的原因：**

人类感知世界是天然的**多通道**过程——视觉、听觉、触觉、本体感知共同构成对环境的完整认知。单一模态的AI系统存在根本局限：仅靠文本无法感知物理世界的空间和物理属性，仅靠图像无法理解因果和动态过程。多模态让AI具备：
- 综合理解多种信息源的能力
- 与真实物理世界交互的基础（具身智能）
- 更接近人类认知方式的通用推理能力

因此，多模态融合被认为是通向AGI的关键里程碑之一。

**13. Q-Former工作原理：**

Q-Former（Querying Transformer）由一个Transformer Encoder和一个Cross-Attention Decoder组成：
1. 冻结视觉Encoder（如EVA-CLIP）输出的图像特征作为Cross-Attention的Key和Value
2. 可学习的Queries（32个）通过Self-Attention建模query间关系
3. Cross-Attention让queries从图像特征中选择性地聚合信息
4. 输出的query表示（维度远小于原始图像特征）作为LLM的软提示（soft prompt）

这种设计使得Q-Former可以高效地将视觉信息压缩并对齐到LLM的输入空间，同时只需训练约1%的参数（适配器+Queries），大大降低了训练成本。

**14. 幻觉问题：**

**定义**：多模态大模型在生成文本描述时，可能产生与图像实际内容不符的描述，例如描述不存在的物体或不正确的属性。

**主要原因**：
- **LLM的先验偏见**：语言模型在预训练中积累了强大的文本知识，可能优先依赖语言先验而非视觉信号
- **训练数据噪声**：图文配对数据中存在弱相关或错误匹配，模型可能学到错误的对应关系
- **视觉编码器容量不足**：视觉特征未能充分捕获图像细节，导致模型过度依赖语言生成

**缓解方法**：增加高质量对齐数据的比例、引入检测幻觉的辅助损失、采用强化学习对齐人类偏好（RLHF）等。

**15. 具身智能机器人感知系统的多模态设计：**

| 模态 | 传感器 | 主要任务 |
|------|--------|----------|
| 视觉 | RGB相机、深度相机 | 目标识别、场景理解、物体位姿估计 |
| 触觉 | 触觉传感器阵列 | 接触力感知、抓取稳定性判断 |
| 本体感知 | IMU、关节编码器 | 自身姿态、运动状态估计 |
| 听觉 | 麦克风阵列 | 语音指令识别、异常声音检测 |
| 激光雷达（可选） | LiDAR | 导航避障、地图构建 |
| 语言 | 麦克风+LLM | 自然语言指令理解、人机对话 |

多模态融合的方式可以是**注意力融合**：在决策层让视觉特征、触觉特征和语言指令通过Cross-Attention交互，使机器人能够综合"看到"、"摸到"和"听到"的信息做出协调的控制决策。
