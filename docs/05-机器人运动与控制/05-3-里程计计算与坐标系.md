## 1. 里程计概念与原理

### 1.1 什么是里程计

**里程计（Odometry）** 是机器人导航中的一种基本定位方法，它通过积分（累加）传感器测量到的运动增量来估计机器人位置的变化。术语"里程计"源自希腊语"hodos"（路径）和"metron"（测量），字面意思就是"路径的测量"。

在移动机器人中，里程计的作用类似于汽车上的里程表。汽车里程表通过计算车轮转动的圈数来估算行驶的距离，机器人里程计也是类似的原理——通过跟踪车轮的转动来估算机器人移动的距离和方向。

里程计的主要优点是：
- **计算简单**：只需要基本的数学运算，不需要复杂的滤波算法
- **实时性好**：每次传感器更新都可以立即计算位置变化
- **成本低廉**：只需要车轮编码器，不需要外部感知设备

里程计的主要缺点是：
- **误差累积**：积分过程会导致误差不断累积，长期运行会产生较大的漂移
- **累积误差**：车轮打滑、地面不平整等因素会导致累积误差
- **需要初始化**：需要知道初始位置才能计算后续位置

### 1.2 差速驱动机器人的里程计计算

差速驱动（Differential Drive）是移动机器人最常用的运动模型之一。这种机器人有两个独立驱动的轮子，分别安装在机器人主体的两侧，通过控制两个轮子的速度差来实现转向。

**运动学模型**：

对于差速驱动机器人，其运动学方程如下：

$$\begin{pmatrix} \dot{x} \\ \dot{y} \\ \dot{\theta} \end{pmatrix} = \begin{pmatrix} \cos\theta & 0 \\ \sin\theta & 0 \\ 0 & 1 \end{pmatrix} \begin{pmatrix} v \\ \omega \end{pmatrix}$$

其中：
- $\dot{x}, \dot{y}$ 是机器人线速度在 world 坐标系下的分量
- $\dot{\theta}$ 是机器人角速度
- $v$ 是机器人线速度（m/s）
- $\omega$ 是机器人角速度（rad/s）

**线速度和角速度计算**：

假设机器人的左右轮中心距离（轮距）为 $L$，左轮速度为 $v_L$，右轮速度为 $v_R$，则：

$$v = \frac{v_R + v_L}{2}$$

$$\omega = \frac{v_R - v_L}{L}$$

**编码器脉冲到距离的转换**：

电机编码器通常输出脉冲信号，需要转换为轮子移动的距离：

$$D = \frac{2\pi r \cdot N}{C}$$

其中：
- $D$ 是轮子移动的距离（m）
- $r$ 是轮子半径（m）
- $N$ 是编码器脉冲计数
- $C$ 是编码器每转脉冲数（常见为 4096 或 500 CPR）

**离散时间积分**：

在实际系统中，我们每隔固定时间间隔 $\Delta t$ 更新一次里程计：

```python
# 伪代码
delta_left = (encoder_left - prev_encoder_left) * distance_per_pulse
delta_right = (encoder_right - prev_encoder_right) * distance_per_pulse

# 计算运动增量
delta_distance = (delta_left + delta_right) / 2
delta_angle = (delta_right - delta_left) / wheel_base

# 更新位置
x += delta_distance * cos(theta)
y += delta_distance * sin(theta)
theta += delta_angle
```

### 1.3 里程计误差模型

里程计的误差主要分为两类：**系统误差**和**非系统误差**。

**系统误差（确定性误差）**：

| 误差类型 | 描述 | 原因 |
|----------|------|------|
| 比例误差 | 实际距离与测量距离存在固定比例 | 轮子半径标定不准、编码器分辨率 |
| 偏置误差 | 测量值存在固定偏移 | 机械装配不对称 |
| 非线性误差 | 传感器输出与实际值呈非线性关系 | 电机特性、齿轮间隙 |

**非系统误差（随机误差）**：

| 误差类型 | 描述 | 原因 |
|----------|------|------|
| 轮子打滑 | 轮子转动但机器人未移动 | 地面湿滑、低摩擦表面 |
| 颠簸抖动 | 机器人通过不平整地面 | 地面不平、障碍物 |
| 积分噪声 | 传感器测量噪声积分 | 编码器精度限制 |

为了抑制误差累积，通常会将里程计与IMU（惯性测量单元）或GPS等外部传感器数据进行融合，这部分内容将在后续的传感器融合课程中详细讲解。

## 2. 坐标系变换

### 2.1 ROS2中的常用坐标系

在ROS2机器人系统中，存在多个重要的坐标系，它们之间有着明确的层级关系。理解这些坐标系及其变换关系，是正确实现里程计和导航的基础。

**常用坐标系**：

| 坐标系 | 名称 | 描述 |
|--------|------|------|
| `map` | 全局坐标系 | 固定的全局参考坐标系，相当于世界坐标 |
| `odom` | 里程计坐标系 | 里程计计算的参考坐标系，原点为起点 |
| `base_link` | 机器人基座坐标系 | 固定在机器人本体上的坐标系 |
| `base_footprint` | 机器人脚底坐标系 | 机器人投影到地面的坐标系 |
| `laser_link` | 激光雷达坐标系 | 激光雷达传感器的坐标系 |
| `imu_link` | IMU坐标系 | IMU传感器的坐标系 |

**坐标系层级关系**：

```
map
  │
  └── odom (里程计坐标系)
        │
        └── base_link (机器人基座)
              │
              ├── laser_link (激光雷达)
              │     └── 传感器原点
              │
              └── imu_link (IMU)
                    └── 传感器原点
```

### 2.2 坐标系变换链

坐标系之间的变换通过 **tf2** 库进行管理。tf2是ROS2中处理坐标变换的核心库，它维护了一个坐标变换树，允许查询任意两个坐标系之间的变换关系。

**坐标变换的含义**：

1. **map → odom**：这是全局修正变换，通常由SLAM算法或GPS定位提供。由于里程计存在漂移，需要周期性地修正odom坐标系的位置，使其与真实位置保持一致。

2. **odom → base_link**：这是里程计计算的输出，表示机器人相对于起始位置的位移和朝向。这个变换是连续发布的，即使机器人在移动，也能实时反映当前位置。

3. **base_link → sensor**：这是传感器相对于机器人基座的固定安装位置，在机器人装配时就确定了，不会随时间变化。

### 2.3 坐标变换的数学表示

坐标变换通过 **齐次变换矩阵** 表示，它包含旋转和平移两个部分。

**二维齐次变换矩阵**：

$$T = \begin{pmatrix} \cos\theta & -\sin\theta & x \\ \sin\theta & \cos\theta & y \\ 0 & 0 & 1 \end{pmatrix}$$

其中 $(x, y)$ 是平移向量，$\theta$ 是旋转角度。

**三维齐次变换矩阵**（更常用）：

$$T = \begin{pmatrix} R_{11} & R_{12} & R_{13} & t_x \\ R_{21} & R_{22} & R_{23} & t_y \\ R_{31} & R_{32} & R_{33} & t_z \\ 0 & 0 & 0 & 1 \end{pmatrix}$$

其中 $R$ 是 $3\times3$ 的旋转矩阵，$t$ 是 $3\times1$ 的平移向量。

**坐标变换示例**：

假设在 `base_link` 坐标系下有一个点 $P_{base} = (x, y, z)$，要转换到 `odom` 坐标系：

$$P_{odom} = T_{odom}^{base\_link} \cdot P_{base}$$

在ROS2中，使用tf2库可以轻松完成这种变换：

```python
# 伪代码
from tf_transformations import quaternion_from_euler

# 创建变换
transform = TransformStamped()
transform.header.frame_id = 'odom'
transform.child_frame_id = 'base_link'
transform.transform.translation.x = x
transform.transform.translation.y = y
transform.transform.translation.z = z
transform.transform.rotation = quaternion_from_euler(0, 0, theta)

# 发布变换
self.tf_broadcaster.sendTransform(transform)
```

## 3. ROS2代码实现

### 3.1 项目结构

本节我们将实现一个完整的里程计计算节点，功能包括：
- 订阅左右轮编码器数据
- 计算里程计（位置和朝向）
- 发布里程计消息
- 发布tf变换

### 3.2 里程计节点实现

```python
# odometry_node/odometry_node/odometry_node.py
import rclpy
from rclpy.node import Node
from rclpy.qos import QoSProfile, ReliabilityPolicy, DurabilityPolicy
from geometry_msgs.msg import TransformStamped
from nav_msgs.msg import Odometry
from sensor_msgs.msg import JointState
import math
from tf_transformations import quaternion_from_euler


class OdometryNode(Node):
    """
    里程计计算节点
    
    订阅关节状态（编码器数据），计算并发布里程计和TF变换
    """
    
    def __init__(self):
        super().__init__('odometry_node')
        
        # 声明参数
        self.declare_parameter('wheel_base', 0.3)        # 轮距 (m)
        self.declare_parameter('wheel_radius', 0.05)     # 轮子半径 (m)
        self.declare_parameter('ticks_per_rev', 4096)    # 编码器分辨率
        self.declare_parameter('odom_frame', 'odom')     # 里程计坐标系
        self.declare_parameter('base_frame', 'base_link')  # 机器人基座坐标系
        
        # 获取参数值
        self.wheel_base = self.get_parameter('wheel_base').value
        self.wheel_radius = self.get_parameter('wheel_radius').value
        self.ticks_per_rev = self.get_parameter('ticks_per_rev').value
        self.odom_frame = self.get_parameter('odom_frame').value
        self.base_frame = self.get_parameter('base_frame').value
        
        # 里程计状态变量
        self.x = 0.0           # 机器人X位置 (m)
        self.y = 0.0           # 机器人Y位置 (m)
        self.theta = 0.0       # 机器人朝向角 (rad)
        
        # 编码器状态
        self.prev_left_ticks = 0
        self.prev_right_ticks = 0
        self.first_reading = True
        
        # 计算常量
        self.ticks_per_meter = 1.0 / (2 * math.pi * self.wheel_radius / self.ticks_per_rev)
        
        # 创建QoS配置
        qos = QoSProfile(
            reliability=ReliabilityPolicy.RELIABLE,
            durability=DurabilityPolicy.VOLATILE,
            depth=10
        )
        
        # 订阅关节状态（编码器数据）
        self.joint_sub = self.create_subscription(
            JointState,
            '/joint_states',
            self.joint_callback,
            qos
        )
        
        # 发布里程计消息
        self.odom_pub = self.create_publisher(
            Odometry,
            '/odom',
            qos
        )
        
        # TF变换发布器
        from tf2_ros import TransformBroadcaster
        self.tf_broadcaster = TransformBroadcaster(self)
        
        # 里程计消息
        self.odom_msg = Odometry()
        self.odom_msg.header.frame_id = self.odom_frame
        self.odom_msg.child_frame_id = self.base_frame
        
        self.get_logger().info('Odometry node started')
        self.get_logger().info(f'Wheel base: {self.wheel_base}m, Wheel radius: {self.wheel_radius}m')
    
    def joint_callback(self, msg: JointState):
        """处理关节状态消息，更新里程计"""
        
        # 获取左右轮的位置（编码器计数）
        try:
            left_index = msg.name.index('left_wheel_joint')
            right_index = msg.name.index('right_wheel_joint')
        except ValueError as e:
            self.get_logger().warn(f'Wheel joint not found: {e}')
            return
        
        left_ticks = int(msg.position[left_index])
        right_ticks = int(msg.position[right_index])
        
        # 首次读取，初始化编码器值
        if self.first_reading:
            self.prev_left_ticks = left_ticks
            self.prev_right_ticks = right_ticks
            self.first_reading = False
            return
        
        # 计算编码器变化
        delta_left = left_ticks - self.prev_left_ticks
        delta_right = right_ticks - self.prev_right_ticks
        
        # 更新编码器值
        self.prev_left_ticks = left_ticks
        self.prev_right_ticks = right_ticks
        
        # 计算运动增量（米）
        left_distance = delta_left / self.ticks_per_meter
        right_distance = delta_right / self.ticks_per_meter
        
        # 计算里程计更新
        self.update_odometry(left_distance, right_distance)
    
    def update_odometry(self, left_dist: float, right_dist: float):
        """更新里程计位置"""
        
        # 计算左右轮平均移动距离
        distance = (left_dist + right_dist) / 2.0
        
        # 计算角度变化
        angle = (right_dist - left_dist) / self.wheel_base
        
        # 更新位置
        if abs(distance) > 1e-6:
            self.x += distance * math.cos(self.theta)
            self.y += distance * math.sin(self.theta)
        
        # 更新朝向
        self.theta += angle
        
        # 归一化角度到 [-π, π]
        while self.theta > math.pi:
            self.theta -= 2 * math.pi
        while self.theta < -math.pi:
            self.theta += 2 * math.pi
        
        # 发布里程计消息和TF
        self.publish_odometry(distance, angle)
    
    def publish_odometry(self, linear_velocity: float, angular_velocity: float):
        """发布里程计消息和TF变换"""
        
        current_time = self.get_clock().now()
        
        # 更新里程计消息头
        self.odom_msg.header.stamp = current_time.to_msg()
        self.odom_msg.pose.pose.position.x = self.x
        self.odom_msg.pose.pose.position.y = self.y
        self.odom_msg.pose.pose.position.z = 0.0
        
        # 设置四元数
        quat = quaternion_from_euler(0, 0, self.theta)
        self.odom_msg.pose.pose.orientation.x = quat[0]
        self.odom_msg.pose.pose.orientation.y = quat[1]
        self.odom_msg.pose.pose.orientation.z = quat[2]
        self.odom_msg.pose.pose.orientation.w = quat[3]
        
        # 设置速度
        self.odom_msg.twist.twist.linear.x = linear_velocity
        self.odom_msg.twist.twist.angular.z = angular_velocity
        
        # 发布里程计
        self.odom_pub.publish(self.odom_msg)
        
        # 发布TF变换
        transform = TransformStamped()
        transform.header.stamp = current_time.to_msg()
        transform.header.frame_id = self.odom_frame
        transform.child_frame_id = self.base_frame
        
        transform.transform.translation.x = self.x
        transform.transform.translation.y = self.y
        transform.transform.translation.z = 0.0
        
        transform.transform.rotation.x = quat[0]
        transform.transform.rotation.y = quat[1]
        transform.transform.rotation.z = quat[2]
        transform.transform.rotation.w = quat[3]
        
        self.tf_broadcaster.sendTransform(transform)


def main(args=None):
    rclpy.init(args=args)
    node = OdometryNode()
    
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 3.3 使用IMU修正航向角

为了提高里程计的准确性，可以使用IMU数据来修正航向角：

```python
# odometry_node/odometry_node/odometry_with_imu.py
import rclpy
from rclpy.node import Node
from rclpy.qos import QoSProfile, ReliabilityPolicy, DurabilityPolicy
from geometry_msgs.msg import TransformStamped
from nav_msgs.msg import Odometry
from sensor_msgs.msg import JointState, Imu
from tf_transformations import quaternion_from_euler, euler_from_quaternion
import math


class OdometryWithImu(Node):
    """带IMU修正的里程计计算节点"""
    
    def __init__(self):
        super().__init__('odometry_with_imu_node')
        
        # 参数
        self.declare_parameter('wheel_base', 0.3)
        self.declare_parameter('wheel_radius', 0.05)
        self.declare_parameter('ticks_per_rev', 4096)
        self.declare_parameter('use_imu', True)
        self.declare_parameter('imu_weight', 0.5)
        
        self.wheel_base = self.get_parameter('wheel_base').value
        self.wheel_radius = self.get_parameter('wheel_radius').value
        self.ticks_per_rev = self.get_parameter('ticks_per_rev').value
        self.use_imu = self.get_parameter('use_imu').value
        self.imu_weight = self.get_parameter('imu_weight').value
        
        # 里程计状态
        self.x = 0.0
        self.y = 0.0
        self.theta = 0.0
        self.prev_theta_imu = 0.0
        
        # 编码器状态
        self.prev_left_ticks = 0
        self.prev_right_ticks = 0
        self.first_reading = True
        
        # 计算常量
        self.ticks_per_meter = 1.0 / (2 * math.pi * self.wheel_radius / self.ticks_per_rev)
        
        # QoS配置
        qos = QoSProfile(
            reliability=ReliabilityPolicy.RELIABLE,
            durability=DurabilityPolicy.VOLATILE,
            depth=10
        )
        
        # 订阅关节状态
        self.joint_sub = self.create_subscription(
            JointState, '/joint_states', self.joint_callback, qos)
        
        # 订阅IMU数据
        self.imu_sub = self.create_subscription(
            Imu, '/imu', self.imu_callback, qos)
        
        # 发布者
        self.odom_pub = self.create_publisher(Odometry, '/odom', qos)
        
        from tf2_ros import TransformBroadcaster
        self.tf_broadcaster = TransformBroadcaster(self)
        
        # 初始化消息
        self.odom_msg = Odometry()
        self.odom_msg.header.frame_id = 'odom'
        self.odom_msg.child_frame_id = 'base_link'
        
        self.get_logger().info('Odometry with IMU node started')
    
    def imu_callback(self, msg: Imu):
        """处理IMU数据"""
        q = msg.orientation
        _, _, self.theta_imu = euler_from_quaternion([q.x, q.y, q.z, q.w])
    
    def joint_callback(self, msg: JointState):
        """处理编码器数据"""
        try:
            left_idx = msg.name.index('left_wheel_joint')
            right_idx = msg.name.index('right_wheel_joint')
        except ValueError:
            return
        
        left_ticks = int(msg.position[left_idx])
        right_ticks = int(msg.position[right_idx])
        
        if self.first_reading:
            self.prev_left_ticks = left_ticks
            self.prev_right_ticks = right_ticks
            self.first_reading = False
            return
        
        delta_left = left_ticks - self.prev_left_ticks
        delta_right = right_ticks - self.prev_right_ticks
        
        self.prev_left_ticks = left_ticks
        self.prev_right_ticks = right_ticks
        
        left_dist = delta_left / self.ticks_per_meter
        right_dist = delta_right / self.ticks_per_meter
        
        distance = (left_dist + right_dist) / 2.0
        angle = (right_dist - left_dist) / self.wheel_base
        
        if abs(distance) > 1e-6:
            self.x += distance * math.cos(self.theta)
            self.y += distance * math.sin(self.theta)
        
        self.theta += angle
        
        # IMU融合
        if self.use_imu and hasattr(self, 'theta_imu'):
            self.theta = self.theta * (1 - self.imu_weight) + self.theta_imu * self.imu_weight
        
        self.publish_odometry()
    
    def publish_odometry(self):
        """发布里程计"""
        now = self.get_clock().now()
        
        self.odom_msg.header.stamp = now.to_msg()
        self.odom_msg.pose.pose.position.x = self.x
        self.odom_msg.pose.pose.position.y = self.y
        
        quat = quaternion_from_euler(0, 0, self.theta)
        self.odom_msg.pose.pose.orientation.x = quat[0]
        self.odom_msg.pose.pose.orientation.y = quat[1]
        self.odom_msg.pose.pose.orientation.z = quat[2]
        self.odom_msg.pose.pose.orientation.w = quat[3]
        
        self.odom_pub.publish(self.odom_msg)
        
        # TF
        t = TransformStamped()
        t.header.stamp = now.to_msg()
        t.header.frame_id = 'odom'
        t.child_frame_id = 'base_link'
        t.transform.translation.x = self.x
        t.transform.translation.y = self.y
        t.transform.translation.z = 0.0
        t.transform.rotation.x = quat[0]
        t.transform.rotation.y = quat[1]
        t.transform.rotation.z = quat[2]
        t.transform.rotation.w = quat[3]
        
        self.tf_broadcaster.sendTransform(t)


def main(args=None):
    rclpy.init(args=args)
    node = OdometryWithImu()
    
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
```

## 4. 练习题

### 选择题

1. 里程计的主要误差来源是什么？
   - A) 传感器测量精度
   - B) 误差累积（漂移）
   - C) 通信延迟
   - D) CPU计算能力
   
   **答案：B**。里程计通过积分运动增量来计算位置，误差会随时间累积，这是里程计最主要的误差来源。

2. 在差速驱动机器人中，如果左轮速度小于右轮，机器人会向哪个方向转动？
   - A) 左转
   - B) 右转
   - C) 直行
   - D) 停止
   
   **答案：B**。差速驱动机器人通过左右轮速度差实现转向。左轮慢、右轮快时，机器人向右转。

3. ROS2中，`odom`坐标系和`base_link`坐标系之间的变换通常由哪个节点发布？
   - A) SLAM节点
   - B) 里程计节点
   - C) 导航节点
   - D) 传感器驱动节点
   
   **答案：B**。里程计节点计算机器人相对于起始位置的运动，并发布`odom`到`base_link`的tf变换。

4. 下列哪个坐标系是全局参考坐标系？
   - A) odom
   - B) base_link
   - C) map
   - D) laser_link
   
   **答案：C**。`map`坐标系是全局固定的参考坐标系，相当于世界坐标。`odom`坐标系的原点在机器人启动位置，会随时间漂移。

5. 编写一个简单的模拟节点，发布模拟的编码器数据（JointState消息），用于测试里程计节点。
   
   **参考答案**：
   
   ```python
   import rclpy
   from rclpy.node import Node
   from sensor_msgs.msg import JointState

   class EncoderSimulator(Node):
       def __init__(self):
           super().__init__('encoder_simulator')
           self.publisher = self.create_publisher(JointState, '/joint_states', 10)
           self.timer = self.create_timer(0.05, self.publish_encoder)
           self.left_ticks = 0
           self.right_ticks = 0
           self.velocity = 0.2
           self.ticks_per_meter = 10000
           
       def publish_encoder(self):
           msg = JointState()
           msg.header.stamp = self.get_clock().now().to_msg()
           msg.name = ['left_wheel_joint', 'right_wheel_joint']
           dt = 0.05
           distance = self.velocity * dt
           ticks = int(distance * self.ticks_per_meter)
           self.left_ticks += ticks
           self.right_ticks += ticks
           msg.position = [float(self.left_ticks), float(self.right_ticks)]
           self.publisher.publish(msg)

   def main(args=None):
       rclpy.init(args=args)
       node = EncoderSimulator()
       rclpy.spin(node)
       node.destroy_node()
       rclpy.shutdown()

   if __name__ == '__main__':
       main()
   ```
