## 1. 电机基础与驱动

### 1.1 直流电机工作原理

直流电机（DC Motor）是机器人领域最常用的执行器。其工作原理基于**安培力**——通电导体在磁场中会受到力的作用。

```
┌─────────────────────────────────────────────────────┐
│                   直流电机原理                        │
│                                                     │
│              N  ╔═══════════╗  S                     │
│                ║           ║                        │
│           ◄────║  线圈(a)  ║─────►  力 F           │
│                ║           ║                        │
│              S ╚═══════════╝  N                     │
│                  │     │                            │
│                  │     │                            │
│               换向器（机械开关）                      │
│                                                     │
│         电流方向：+ → a → b → -                      │
│         受力方向：F = BIL（左手定则）                 │
└─────────────────────────────────────────────────────┘
```

**左手定则判断受力方向**：
- 掌心朝向磁场方向（N→S）
- 四指指向电流方向
- 大拇指即为受力方向

电机轴的旋转速度与电压成正比，改变电压即可改变转速。

### 1.2 电机关键参数

| 参数 | 含义 | 典型值（FishBot） |
|------|------|------------------|
| **额定电压** | 正常工作电压 | 6V~12V |
| **额定电流** | 正常工作电流 | 0.5A~2A |
| **空载转速** | 无负载时最高转速 | 10000 RPM |
| **额定转速** | 额定负载下的转速 | 5000 RPM |
| **额定扭矩** | 额定负载产生的力矩 | 0.1 N·m |
| **减速比** | 电机轴转速与输出轴转速的比值 | 30:1、50:1 |
| **堵转电流** | 电机卡死时的电流 | 额定电流的3~5倍 |

**减速箱的作用**：
- 降低转速、增加扭矩（扭矩 = 原始扭矩 × 减速比）
- 减速比越大，带负载能力越强，但转速越低

### 1.3 电机驱动器

单片机（如STM32）的IO口无法直接驱动电机，原因：
1. IO口输出电流极小（mA级），无法驱动电机
2. 反电动势可能损坏单片机

因此需要**电机驱动器**作为桥梁。

#### 1.3.1 TB6612FNG 双H桥驱动器

TB6612FNG 是机器人领域最常用的小型电机驱动芯片：

```
         TB6612FNG 引脚分布
    ┌─────────────────────────────┐
    │  AIN1  AIN2  PWMA  STBY     │  ← 控制信号（接单片机IO）
    │                             │
    │       TB6612FNG             │
    │                             │
    │  BIN1  BIN2  PWMB  GND VCC  │
    └─────────────────────────────┘
         │      │      │
         ▼      ▼      ▼
      ┌─────────────────────┐
      │    电机A    电机B    │  ← 驱动两个直流电机
      └─────────────────────┘
```

**控制逻辑**：

| AIN1 | AIN2 | PWMA | 电机状态 |
|------|------|------|----------|
| 0 | 0 | X | 停止（高阻态） |
| 0 | 1 | PWM | 反转（PWM调速） |
| 1 | 0 | PWM | 正转（PWM调速） |
| 1 | 1 | X | 制动（快速停止） |

```c
// TB6612FNG 电机驱动代码
#define AIN1_PIN  GPIO_Pin_0   // PC0
#define AIN2_PIN  GPIO_Pin_1   // PC1
#define PWMA_PIN  GPIO_Pin_6   // PA6 (TIM3 Ch1)

// 电机初始化
void Motor_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    TIM_TimeBaseInitTypeDef TIM_TimeBaseStructure;
    TIM_OCInitTypeDef TIM_OCInitStructure;

    // 使能GPIO和定时器时钟
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA | RCC_APB2Periph_GPIOC, ENABLE);
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM3, ENABLE);

    // 配置AIN1、AIN2为推挽输出
    GPIO_InitStructure.GPIO_Pin = AIN1_PIN | AIN2_PIN;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_PP;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_Init(GPIOC, &GPIO_InitStructure);

    // 配置PWMA为复用推挽输出（PWM通道）
    GPIO_InitStructure.GPIO_Pin = PWMA_PIN;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    // 配置定时器3，PWM频率20kHz
    TIM_TimeBaseStructure.TIM_Period = 999;      // ARR = 1000，计数0~999
    TIM_TimeBaseStructure.TIM_Prescaler = 71;   // PSC = 72，预分频后1MHz
    TIM_TimeBaseStructure.TIM_ClockDivision = 0;
    TIM_TimeBaseStructure.TIM_CounterMode = TIM_CounterMode_Up;
    TIM_TimeBaseInit(TIM3, &TIM_TimeBaseStructure);

    // 配置PWM模式1：占空比越大输出越"正"
    TIM_OCInitStructure.TIM_OCMode = TIM_OCMode_PWM1;
    TIM_OCInitStructure.TIM_OutputState = TIM_OutputState_Enable;
    TIM_OCInitStructure.TIM_Pulse = 0;           // 初始占空比为0
    TIM_OCInitStructure.TIM_OCPolarity = TIM_OCPolarity_High;
    TIM_OC1Init(TIM3, &TIM_OCInitStructure);
    TIM_OC1PreloadConfig(TIM3, TIM_OCPreload_Enable);

    TIM_ARRPreloadConfig(TIM3, ENABLE);
    TIM_Cmd(TIM3, ENABLE);
}

// 设置电机速度与方向
// speed: -1000 ~ 1000，负数为反转
void Motor_SetSpeed(int16_t speed)
{
    if (speed > 0) {
        GPIO_SetBits(GPIOC, AIN1_PIN);    // 正转
        GPIO_ResetBits(GPIOC, AIN2_PIN);
    } else if (speed < 0) {
        GPIO_ResetBits(GPIOC, AIN1_PIN); // 反转
        GPIO_SetBits(GPIOC, AIN2_PIN);
        speed = -speed;                  // 取绝对值
    } else {
        GPIO_ResetBits(GPIOC, AIN1_PIN | AIN2_PIN); // 停止
    }
    TIM_SetCompare1(TIM3, speed);         // 设置PWM占空比
}
```

## 2. PWM调速原理

### 2.1 PWM信号概念

PWM（Pulse Width Modulation，脉宽调制）是一种用数字信号模拟模拟量输出的技术。

```
    PWM信号示意图（占空比50%）
    
    高电平 ─┐       ┌───────┐       ┌────
            │       │       │       │
            └───────┘       └───────┘
    ────────────────────────────────────────────── 低电平
    
    │←─ 周期 T ─→│←─ 周期 T ─→│← 周期 T →│
    
    占空比 = t_high / T = 50%
    
    PWM信号示意图（占空比25%）
    
    高电平 ─┐   ┌───────┐   ┌───────┐
            │   │       │   │       │
            └───┘       └───┘       └────
    
    占空比 = t_high / T = 25%
```

### 2.2 占空比与转速的关系

对于直流电机，PWM的占空比直接决定了电机两端的平均电压：

```
平均电压 = Vcc × 占空比

例如：Vcc = 12V，占空比 = 60%
平均电压 = 12V × 60% = 7.2V
```

转速与平均电压近似成正比：

```
转速 RPM ∝ 平均电压 ∝ 占空比
```

### 2.3 PWM频率选择

PWM频率选择需要权衡**响应速度**和**电机效率**：

| PWM频率 | 优点 | 缺点 | 适用场景 |
|---------|------|------|----------|
| < 1kHz | 开关损耗低 | 电机抖动、噪音大 | 大功率直流电机 |
| 1kHz~20kHz | 平衡方案 | 轻微噪音 | 通用场景 |
| > 20kHz | 无噪音（超音频） | 开关损耗高 | 低功率精密控制 |

**STM32定时器配置示例（20kHz PWM）**：

```c
// PWM频率计算公式：F PWM = APB总线频率 / ((PSC+1) × (ARR+1))
// 以APB1 = 72MHz为例，要得到20kHz：
// F = 72,000,000 / ((71+1) × (999+1)) = 72,000,000 / 72,000 = 1000Hz
// 重新计算：
// F = 72,000,000 / ((3599+1) × (999+1)) = 72,000,000 / 3,600,000 = 20kHz

void PWM_Init(void)
{
    TIM_TimeBaseInitTypeDef TIM_TimeBaseStructure;

    // APB1 = 72MHz，设置PSC=3599，ARR=999
    // PWM频率 = 72,000,000 / 3600 / 1000 = 20kHz
    TIM_TimeBaseStructure.TIM_Period = 999;      // ARR = 1000
    TIM_TimeBaseStructure.TIM_Prescaler = 3599;  // PSC = 3600
    TIM_TimeBaseStructure.TIM_ClockDivision = 0;
    TIM_TimeBaseStructure.TIM_CounterMode = TIM_CounterMode_Up;
    TIM_TimeBaseInit(TIM3, &TIM_TimeBaseStructure);
    
    // ... GPIO和OC配置见上一节
}
```

### 2.4 PWM调速与电机扭矩

```
         扭矩-转速特性曲线
        
扭矩
 ↑
 │    ╭─────────────── 空载转速（最高）
 │   ╱
 │  ╱
 │ ╱  额定扭矩点
 │╱───────────────────────────► 转速
 │  ╲
 │   ╲  负载越大，转速越低
 │
```

> **重要**：电机在低速时扭矩会下降，因为PWM占空比太低时，反电动势不足，电流有限。某些场景需要使用**六步换向（Six-Step Commutation）**或**空间矢量PWM（SVPWM）**来改善低速性能。

## 3. 编码器测速

### 3.1 增量式编码器原理

增量式编码器（Incremental Encoder）是机器人测速最常用的传感器：

```
         增量式编码器原理示意图
        
          码盘（带透光槽）
             ┌──┐
          ┌──┘  └──┐
          │   ◎    │  ← 光源
          │   LED  │
          └──┐  └──┘
             └──┘
             传感器A、B（相隔90°相位）
```

**工作原理**：
- 码盘随电机轴旋转
- 光线穿过槽时，光敏传感器输出高电平
- 光线被遮挡时，输出低电平
- 产生两路相位相差90°的方波信号（A、B相）

### 3.2 正转与反转判断

```
正转（A相超前B相90°）：
    A ──┐   ┌───────┐   ┌────
        └───┘       └───┘
    B ─────┐   ┌───────┐   ┌
           └───┘       └───┘
        ↑
     A上升沿时，B为低电平 → 正转

反转（B相超前A相90°）：
    A ─────┐   ┌───────┐   ┌
           └───┘       └───┘
    B ──┐   ┌───────┐   ┌────
        └───┘       └───┘
        ↑
     A上升沿时，B为高电平 → 反转
```

### 3.3 编码器参数

| 参数 | 含义 | 说明 |
|------|------|------|
| **PPR** | Pulses Per Revolution，每转脉冲数 | 码盘一圈产生的脉冲数 |
| **线数** | 码盘上的槽数量 | 通常等于PPR |
| **分辨率** | 能分辨的最小角度 | 360° / PPR |

**常见规格**：
- 13线编码器（13PPR）：每转13个脉冲
- 100线编码器（100PPR）：每转100个脉冲
- 600线编码器（600PPR）：每转600个脉冲

> FishBot 四驱版 V2 使用的是 **13PPR 霍尔编码器**，配合减速箱减速比30:1，输出轴每转产生 13×30 = 390 个脉冲。

### 3.4 STM32编码器接口配置

```c
// 编码器接口初始化（使用TIM2作为编码器计数器）
void Encoder_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    TIM_TimeBaseInitTypeDef TIM_TimeBaseStructure;
    TIM_ICInitTypeDef TIM_ICInitStructure;

    // 使能TIM2和GPIOA时钟
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2, ENABLE);
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);

    // 配置PA0和PA1为复用推挽输出（TIM2_CH1和TIM2_CH2）
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_0 | GPIO_Pin_1;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    // 配置TIM2为编码器模式
    // TI1FP1 = TI1, TI2FP2 = TI2
    TIM_EncoderInterfaceConfig(TIM2, TIM_EncoderMode_TI12,
                               TIM_ICPolarity_Rising,
                               TIM_ICPolarity_Rising);

    // 配置输入捕获
    TIM_ICStructInit(&TIM_ICInitStructure);
    TIM_ICInit(TIM2, &TIM_ICInitStructure);

    // 清除更新标志位
    TIM_ClearFlag(TIM2, TIM_FLAG_Update);
    TIM_ITConfig(TIM2, TIM_IT_Update, ENABLE);

    // 重置计数器
    TIM_SetCounter(TIM2, 0);
    TIM_Cmd(TIM2, ENABLE);
}

// 获取编码器计数值（带方向）
int16_t Encoder_GetCount(void)
{
    return TIM_GetCounter(TIM2);  // STM32F1的TIM2是16位
}
```

### 3.5 转速计算（RPM）

转速计算需要在固定时间窗口内统计脉冲数：

```c
// 编码器相关参数（根据实际硬件配置）
#define ENCODER_PPR        13      // 电机轴每转脉冲数
#define REDUCTION_RATIO    30      // 减速比
#define OUTPUT_PPR         (ENCODER_PPR * REDUCTION_RATIO)  // 输出轴每转脉冲数 = 390

// 测速周期（ms）
#define SPEED_SAMPLE_PERIOD   50   // 每50ms计算一次速度

// 速度计算
// 全局变量
volatile int16_t encoder_count = 0;     // 本周期编码器计数
volatile float motor_speed_rpm = 0.0f;   // 电机转速（RPM）

// 在定时器中断中计算速度（每50ms执行一次）
void TIM3_IRQHandler(void)
{
    if (TIM_GetITStatus(TIM3, TIM_IT_Update) != RESET) {
        static int16_t last_count = 0;
        int16_t current_count;
        
        // 获取当前编码器计数值
        current_count = Encoder_GetCount();
        
        // 计算本周期内的脉冲数
        // 处理计数器溢出/下溢（16位有符号范围 -32768~32767）
        int16_t delta = current_count - last_count;
        if (delta > 32767) delta -= 65536;  // 负向溢出
        if (delta < -32768) delta += 65536; // 正向溢出
        
        encoder_count = delta;
        last_count = current_count;
        
        // 计算转速：RPM = (脉冲数 / 每转脉冲数) / 时间(分钟)
        // 简化：RPM = (脉冲数 / OUTPUT_PPR) * (1000ms / 采样周期ms) * 60
        motor_speed_rpm = (float)delta / OUTPUT_PPR * (1000.0f / SPEED_SAMPLE_PERIOD) * 60.0f;
        
        TIM_ClearITPendingBit(TIM3, TIM_IT_Update);
    }
}

// 获取当前电机转速（外部调用）
float Get_Motor_Speed(void)
{
    return motor_speed_rpm;
}
```

### 3.6 位置计算

利用编码器脉冲累计可以计算电机转过的角度：

```c
// 累计脉冲计数（需要使用32位变量避免溢出）
volatile int32_t total_pulses = 0;

// 在定时器中断中更新位置
void TIM3_IRQHandler(void)
{
    if (TIM_GetITStatus(TIM3, TIM_IT_Update) != RESET) {
        int16_t delta = Encoder_GetCount() - last_encoder_count;
        total_pulses += delta;
        last_encoder_count = Encoder_GetCount();
        
        TIM_ClearITPendingBit(TIM3, TIM_IT_Update);
    }
}

// 获取累计角度（度）
float Get_Motor_Angle_Deg(void)
{
    // 角度 = (累计脉冲数 / 每转脉冲数) × 360°
    return (float)total_pulses / OUTPUT_PPR * 360.0f;
}

// 获取累计角度（弧度）
float Get_Motor_Angle_Rad(void)
{
    return Get_Motor_Angle_Deg() * 3.1415926f / 180.0f;
}
```

## 4. PID控制原理

### 4.1 为什么要用PID？

开环控制（只给PWM信号）无法应对以下问题：
- 电机负载变化（地面摩擦、上坡等）
- 电池电压波动（满电12V vs 低电10V）
- 电机参数不一致（四台电机特性略有差异）

PID闭环控制通过**实时检测实际速度，调整PWM输出**，使实际速度逼近目标速度。

### 4.2 PID三环节详解

#### 4.2.1 比例（Proportional）控制

**思想**：误差越大，控制器输出越大

```
P控制器输出 = Kp × 误差(e)

例如：Kp = 2.0
  目标速度 = 100 RPM
  实际速度 = 80 RPM
  误差 e = 20 RPM
  P输出 = 2.0 × 20 = 40

实际速度 = 60 RPM → e = 40 → P输出 = 80（更大修正力度）
```

**Kp的作用**：
- Kp太小：响应慢，调节时间长
- Kp太大：震荡剧烈，系统不稳定

#### 4.2.2 积分（Integral）控制

**思想**：消除稳态误差（小误差长期累积也能产生修正）

```
I控制器输出 = Ki × ∫e(t)dt  ≈ Ki × 累积误差

积分的作用：
  - 当系统存在持续的小误差时
  - P控制可能输出一个"差不多"的力，无法完全消除误差
  - I控制将所有历史误差累加，逐渐增大输出直到误差为零
```

**Ki的作用**：
- Ki太小：消除稳态误差时间长
- Ki太大：超调量大，可能震荡

#### 4.2.3 微分（Differential）控制

**思想**：预测误差变化趋势，提前"刹车"

```
D控制器输出 = Kd × de/dt

微分的物理意义：
  - 当误差在减小（D为负）→ 减少输出，防止超调
  - 当误差在增大（D为正）→ 增大输出，加速响应
```

### 4.3 PID完整公式

```
         ┌─────────────────────────────────────────────┐
         │              PID 控制器                       │
         │                                              │
目标值 r ──► (+)   ┌────────────────────────────────►  │ 控制量 u
              ─    │   ┌─────┐   ┌─────┐   ┌─────┐   │
                   └─► │ P  │ + │ I  │ + │ D  │ ──┘   │
                       └──┬─┘   └──┬─┘   └──┬─┘      │
                         │        │        │         │
                         ▼        ▼        ▼         │
                       ┌─────┐  ┌─────┐  ┌─────┐      │
                       │ Kp  │  │ Ki×∫ │  │ Kd×d│      │
                       └─────┘  └─────┘  └─────┘      │
                         │        │        │         │
                         └────────┼────────┘         │
                                  ▼                    │
                    ┌──────────────────────┐          │
                    │    被控对象（电机）    │          │
                    └──────────────────────┘          │
                              │                        │
                              ▼                        │
实际值 y ─────────────────────────────────────────────┘
         ▲
         │ 误差 e = r - y
         │
```

**连续形式**：
```
         u(t) = Kp × e(t) + Ki × ∫e(t)dt + Kd × de(t)/dt
```

**离散形式（单片机中使用）**：
```
         u(k) = Kp × e(k) + Ki × Σe(i) + Kd × (e(k) - e(k-1))
```

### 4.4 PID参数整定方法

#### 4.4.1 试凑法（工程最常用）

**口诀**：**先P后I最后D，逐步增加找最佳**

| 步骤 | 操作 | 目标 |
|------|------|------|
| 1 | 设 Ki=0, Kd=0，逐渐增大 Kp | 系统开始振荡但能稳定 |
| 2 | 减小 Kp 至振荡消失的 60~80% | 消除剧烈振荡 |
| 3 | 逐渐增大 Ki，消除稳态误差 | 误差逐渐收敛到零 |
| 4 | 逐渐增大 Kd，抑制超调 | 减少振荡和超调 |

#### 4.4.2 Ziegler-Nichols法（经典方法）

1. 将 Ki 和 Kd 设为 0
2. 逐渐增大 Kp 直到系统开始持续振荡（临界振荡）
3. 记录此时的临界增益 Kcu 和振荡周期 Pcu
4. 按下表取值：

| 控制器类型 | Kp | Ti | Td |
|-----------|-----|------|------|
| P | 0.5 × Kcu | ∞ | 0 |
| PI | 0.45 × Kcu | Pcu / 1.2 | 0 |
| PID | 0.6 × Kcu | Pcu / 2 | Pcu / 8 |

#### 4.4.3 常见问题与解决

| 现象 | 原因 | 解决 |
|------|------|------|
| 震荡剧烈 | Kp 过大 | 减小 Kp |
| 响应太慢 | Kp 过小 | 增大 Kp |
| 稳态误差大 | Ki 太小 | 增大 Ki |
| 超调过大 | Kd 太小 | 增大 Kd |
| 噪声敏感 | Kd 过大 | 增大微分滤波器 |

### 5.2 PID控制器实现

```c
// PID控制器结构体
typedef struct {
    float target;      // 目标值
    float feedback;    // 反馈值（实际测量值）
    
    float Kp;          // 比例系数
    float Ki;          // 积分系数
    float Kd;          // 微分系数
    
    float error;       // 当前误差
    float error_prev;  // 上次误差
    float error_sum;   // 累积误差（积分项）
    
    float output;      // PID输出
    float output_max;  // 输出上限
    float output_min;  // 输出下限
    
    // 微分项滤波（降低噪声敏感度）
    float derivative_filter_alpha;  // 滤波系数 0~1，越大越敏感
    float derivative_filtered;       // 滤波后的微分项
} PID_Controller;

// PID初始化
void PID_Init(PID_Controller *pid, float Kp, float Ki, float Kd,
              float output_max, float output_min)
{
    pid->Kp = Kp;
    pid->Ki = Ki;
    pid->Kd = Kd;
    pid->output_max = output_max;
    pid->output_min = output_min;
    pid->error_sum = 0.0f;
    pid->error_prev = 0.0f;
    pid->derivative_filtered = 0.0f;
    pid->derivative_filter_alpha = 0.3f;  // 微分滤波系数
}

// PID计算（需要定期调用，如每10ms）
float PID_Calculate(PID_Controller *pid, float target, float feedback, float dt)
{
    pid->target = target;
    pid->feedback = feedback;
    
    // 计算当前误差
    pid->error = target - feedback;
    
    // ========== 比例项 ==========
    float p_term = pid->Kp * pid->error;
    
    // ========== 积分项（带积分限幅，防止积分饱和）==========
    // 累积误差前先限幅，防止积分项过大
    pid->error_sum += pid->error * dt;
    
    // 积分限幅：累积误差不超过 (output_max / Ki) 的某个比例
    float max_integral = pid->output_max / (pid->Ki + 0.0001f);  // 防止除零
    if (pid->error_sum > max_integral)  pid->error_sum = max_integral;
    if (pid->error_sum < -max_integral) pid->error_sum = -max_integral;
    
    float i_term = pid->Ki * pid->error_sum;
    
    // ========== 微分项（带低通滤波）==========
    float d_term_raw = pid->Kd * (pid->error - pid->error_prev) / dt;
    
    // 微分项滤波：避免高频噪声放大
    pid->derivative_filtered = pid->derivative_filter_alpha * d_term_raw
                              + (1.0f - pid->derivative_filter_alpha) * pid->derivative_filtered;
    
    // ========== 总输出 ==========
    pid->output = p_term + i_term + pid->derivative_filtered;
    
    // 输出限幅
    if (pid->output > pid->output_max) pid->output = pid->output_max;
    if (pid->output < pid->output_min) pid->output = pid->output_min;
    
    // 保存误差用于下次计算
    pid->error_prev = pid->error;
    
    return pid->output;
}

// PID重置（停止时调用，清除累积误差）
void PID_Reset(PID_Controller *pid)
{
    pid->error_sum = 0.0f;
    pid->error_prev = 0.0f;
    pid->derivative_filtered = 0.0f;
    pid->output = 0.0f;
}
```

### 5.4 PID调参实战技巧

#### 5.4.1 防积分饱和

**问题**：当目标值突然改变时（如急停），误差突然变大，积分项会快速累积，导致控制器"过度"响应，即使实际值已追上目标，积分项也要很长时间才能消退。

**解决方案**：
1. **积分限幅**：限制累积误差的最大值
2. **积分分离**：当误差很大时，暂时关闭积分项
3. **抗积分饱和**：输出超限时，积分项不再累积

```c
// 积分分离PID实现
float PID_Calculate_withWindup(PID_Controller *pid, float target, 
                                float feedback, float dt)
{
    pid->error = target - feedback;
    
    float p_term = pid->Kp * pid->error;
    
    // 积分分离：当误差超过阈值时，不累积积分项
    float error_threshold = 20.0f;  // 可调整
    if (fabs(pid->error) < error_threshold) {
        pid->error_sum += pid->error * dt;
    }
    
    // 积分限幅
    float max_integral = pid->output_max / (pid->Ki + 0.0001f);
    if (pid->error_sum > max_integral)  pid->error_sum = max_integral;
    if (pid->error_sum < -max_integral) pid->error_sum = -max_integral;
    
    float i_term = pid->Ki * pid->error_sum;
    
    // 微分项
    float d_term = pid->Kd * (pid->error - pid->error_prev) / dt;
    
    pid->output = p_term + i_term + d_term;
    
    // 输出限幅
    if (pid->output > pid->output_max) pid->output = pid->output_max;
    if (pid->output < pid->output_min) pid->output = pid->output_min;
    
    // 抗积分饱和：如果输出已饱和，不累积积分
    if (pid->output == pid->output_max || pid->output == pid->output_min) {
        pid->error_sum -= pid->error * dt;  // 回退本次积分
    }
    
    pid->error_prev = pid->error;
    return pid->output;
}
```

#### 5.4.2 微分项滤波

高频噪声会放大微分项，导致输出抖动：

```c
// 低通滤波器
// filtered_new = alpha × raw_new + (1-alpha) × filtered_old
// alpha 越小，滤波越强，但响应越慢

#define DERIVATIVE_FILTER_ALPHA  0.3f  // 调整此值，0.1~0.5之间

float derivative_filter(float raw_derivative)
{
    static float filtered = 0.0f;
    filtered = DERIVATIVE_FILTER_ALPHA * raw_derivative 
             + (1.0f - DERIVATIVE_FILTER_ALPHA) * filtered;
    return filtered;
}
```

直接阶跃目标速度会造成机械冲击，使用**梯形速度曲线**或**S曲线**平滑过渡：

```c

{
    ramp->target = target;
    ramp->current = 0.0f;
    ramp->max_accel = max_accel;
}

float Speed_Ramp_Update(Speed_Ramp *ramp, float dt)
{

float Speed_Ramp_Update(Speed_Ramp *ramp, float dt)
{
    float error = ramp->target - ramp->current;
    
    if (fabs(error) < 0.1f) {
        // 已到达目标
        ramp->current = ramp->target;
        return ramp->current;
    }
    
    // 计算需要的加速度（限制最大加速度）
    float accel = ramp->max_accel;
    if (error < 0) accel = -accel;  // 减速时方向相反
    
    // 限制加速度，防止过冲
    float delta = accel * dt;
    if (delta > error) delta = error;   // 过冲保护
    if (delta < -error) delta = -error;  // 负向过冲保护
    
    ramp->current += delta;
    return ramp->current;
}
```

## 6. 多电机协同控制

### 6.1 差速驱动原理

差速驱动是轮式机器人最常用的运动方式，通过控制左右轮的速度差实现转向：

```
         差速驱动示意图

              前进（左右同速）
         ◄──────────────────►
         
              左转（左轮慢，右轮快）
         ◄───
              ───►
              
              右转（左轮快，右轮慢）
                   ───►
              ◄───

              原地左转（左轮反转，右轮正转）
                ⊙
              ◄───   ───►

              原地右转（左轮正转，右轮反转）
                   ⊙
              ───►   ◄───
```

**运动学公式**：

```
线速度 v = (v_left + v_right) / 2
角速度 ω = (v_right - v_left) / L

其中 L = 左右轮中心间距

给定 v 和 ω，反解左右轮速度：
v_left  = v - ω × L / 2
v_right = v + ω × L / 2
```

### 6.3 同步协调控制

四个电机特性不可能完全一致，需要同步控制避免偏移：

```c
// 电机速度同步校正
typedef struct {
    float measured_rpm;     // 实测转速
    float error;            // 与目标的误差
    float correction;       // 补偿量
} Motor_Sync;

typedef struct {
    Motor_Sync FL, FR, BL, BR;  // 四个电机
    
    float sync_Kp;          // 同步校正系数
} Sync_Control;

void Sync_Update(Sync_Control *sync, Motor_Control *motors)
{
    // 计算四个电机的误差
    sync->FL.error = motors[0].target_rpm - motors[0].current_rpm;
    sync->FR.error = motors[1].target_rpm - motors[1].current_rpm;
    sync->BL.error = motors[2].target_rpm - motors[2].current_rpm;
    sync->BR.error = motors[3].target_rpm - motors[3].current_rpm;
    
    // 计算平均误差
    float avg_error = (sync->FL.error + sync->FR.error + 
                       sync->BL.error + sync->BR.error) / 4.0f;
    
    // 慢速电机补偿（加快），快速电机抑制（减慢）
    // 这是一种简单的负载均衡策略
    motors[0].target_rpm += sync->sync_Kp * (avg_error - sync->FL.error);
    motors[1].target_rpm += sync->sync_Kp * (avg_error - sync->FR.error);
    motors[2].target_rpm += sync->sync_Kp * (avg_error - sync->BL.error);
    motors[3].target_rpm += sync->sync_Kp * (avg_error - sync->BR.error);
}
```

### 6.4 ROS2与电机控制集成

FishBot 的电机控制最终由 ROS2 指令驱动：

```c
// ROS2 接收速度指令回调函数
void cmd_vel_callback(const geometry_msgs__msg__Twist *msg)
{
    // 将 ROS2 的 geometry_msgs/Twist 转换为电机速度
    // vx = msg->linear.x
    // wz = msg->angular.z
    
    float v = msg->linear.x;    // 线速度 (m/s)
    float w = msg->angular.z;   // 角速度 (rad/s)
    
    // 转换为左右轮转速（RPM）
    // 需要知道轮子半径 r 和轮距 L
    float wheel_radius = 0.03f;  // FishBot 轮子半径约 3cm
    float wheel_base = 0.10f;   // 左右轮距约 10cm
    
    float v_left  = (v - w * wheel_base / 2.0f) / wheel_radius * 60.0f / (2.0f * 3.14159f);
    float v_right = (v + w * wheel_base / 2.0f) / wheel_radius * 60.0f / (2.0f * 3.14159f);
    
    // 设置电机目标转速
    Tank_Set_Velocity(v_left, v_right);
}
```

## 8. 答案

### 答案1：PWM占空比计算

**PWM占空比**：60%（即定时器CCR寄存器值 = ARR × 60%）

**平均电压**：
```
V_avg = Vcc × 占空比 = 12V × 60% = 7.2V
```

---

### 答案2：编码器转速计算

**输出轴每转脉冲数**：
```
PPR_out = 13 × 30 = 390 脉冲/转
```

**转速计算**：
```
采样周期 T = 50ms = 50/1000 s

输出轴转速 n = (117 脉冲 / 390 脉冲/转) / (50/1000 分)
             = 0.3 转 / 0.05 分
             = 6 转/分
             = 6 RPM
```

---

### 答案3：PID参数分析

| 现象 | Kp调整 | Ki调整 | Kd调整 | 可能原因 |
|------|--------|--------|--------|---------|
| 响应非常慢 | **增大** Kp | 可适当增大 | - | P太小，系统响应不足 |
| 持续振荡 | **减小** Kp | 检查是否过大 | 可适当减小 | P太大，系统过激 |
| 稳态误差存在 | 可适当增大 | **增大** Ki | - | 积分作用不足 |
| 对噪声敏感 | - | - | **减小** Kd | D太大，放大噪声 |

---

### 答案4：差速驱动计算

**直线前进时**：
```
左右轮同速，无转向：
v_left = v_right = v / r = 0.2 / 0.03 ≈ 6.67 m/s（电机轴）
转换为RPM需考虑减速比，或直接用输出轴计算：
v_left_rpm = v_right_rpm = (v / (2πr)) × 60 / 减速比
           = (0.2 / (2×3.14×0.03)) × 60 / 30
           ≈ 2.12 × 60 / 30
           ≈ 212 / 30 ≈ 7.1 RPM
```

**原地左转时**（ω=2rad/s，L=0.1m）：
```
v_left  = v - ω×L/2 = 0 - 2×0.1/2 = -0.1 m/s（反转）
v_right = v + ω×L/2 = 0 + 2×0.1/2 = +0.1 m/s（正转）

v_left_rpm  ≈ -7.1 RPM
v_right_rpm ≈ +7.1 RPM
```

---

### 答案5：积分饱和分析

**积分项累积计算**：
```
每步误差 e ≈ 500RPM
积分项 I = Ki × Σe × dt = 0.2 × 500 × 10 = 1000

或等效为：I = Ki × 累积误差
Ki = 0.2，累积误差 = 1000/0.2 = 5000（10秒内累计的误差值）
```

**问题**：
- 输出被限幅在1000，但控制器认为需要1000的输出才能消除误差
- 当目标突然变为0时，积分项仍然是1000，需要很长时间才能消退
- 这导致系统**超调严重**，甚至产生**反向超调**

**解决方案**：
1. **积分限幅**：限制累积误差不超过 output_max / Ki = 1000/0.2 = 5000
2. **积分分离**：当误差很大时，关闭积分项
3. **抗积分饱和**：输出饱和时停止积分累积
4. **微分先行**：对目标变化做滤波，不直接作用于微分项

### 答案6：设计题（参考方案）

#### 1. 硬件选型

| 组件 | 推荐型号 | 说明 |
|------|---------|------|
| 主控 | STM32F103C8T6 | 72MHz，足够处理4路PID |
| 电机驱动 | TB6612FNG | 双H桥，3A峰值，支持PWM调速 |
| 编码器 | 13线霍尔编码器 | 配合减速箱，输出390 PPR |
| 电源 | 3S锂电池(11.1V) | 经LDO降压到5V给单片机供电 |

#### 2. 软件架构

```
┌─────────────────────────────────────────────┐
│                  主循环 (1ms)                │
│  • 接收ROS2指令（USB CDC / UART）            │
│  • 更新目标速度                              │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│            速度控制任务 (10ms)               │
│  • 读取编码器（定时器计数）                   │
│  • 计算实际转速                             │
│  • PID计算                                  │
│  • 输出PWM                                  │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│            通信任务 (50ms)                   │
│  • 上报当前速度给上位机                      │
│  • 接收上位机指令                           │
└─────────────────────────────────────────────┘
```

#### 3. PID参数整定步骤

**Step 1：初始化参数**
```
Kp = 0, Ki = 0, Kd = 0
目标速度设为50 RPM
```

**Step 2：整定比例P**
```
逐渐增大Kp，直到电机开始持续小幅振荡
记录此时的Kp0 ≈ 1.0
设置Kp = 0.6 × Kp0 ≈ 0.6
```

**Step 3：整定积分I**
```
逐渐增大Ki，消除稳态误差
直到响应速度和超调量达到平衡
Ki ≈ 0.1 ~ 0.2
```

**Step 4：整定微分D**
```
逐渐增大Kd，减少超调和振荡
注意：如果系统噪声大，Kd过大会导致抖动
Kd ≈ 0.05 ~ 0.1
```

**Step 5：微调**
```
观察实际运行效果，精细调整
记录最佳参数组合
```
