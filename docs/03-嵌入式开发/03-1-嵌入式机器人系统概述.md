## 1. 什么是嵌入式系统

### 1.1 定义

**嵌入式系统**（Embedded System）是一种专用于完成特定功能的计算机系统，通常被嵌入在更大的设备或产品中。与通用计算机（如 PC）不同，嵌入式系统具有以下特点：

| 特性 | 通用计算机 | 嵌入式系统 |
|------|-----------|-----------|
| 用途 | 多任务、通用 | 单一或有限任务 |
| 资源 | 资源丰富 | 资源受限（CPU/内存/存储） |
| 成本 | 较高 | 低 |
| 功耗 | 较高 | 低 |
| 可靠性 | 一般 | 高（实时性要求） |
| 定制性 | 低 | 高（专为某产品定制） |

### 1.2 嵌入式系统的组成

```
┌─────────────────────────────────────┐
│           嵌入式系统                 │
│  ┌───────────┐  ┌─────────────────┐ │
│  │  硬件层    │  │     软件层       │ │
│  │ ───────── │  │ ──────────────  │ │
│  │ CPU/MCU   │  │ 操作系统/RTOS   │ │
│  │ 存储器     │  │  驱动程序        │ │
│  │ 外设接口   │  │  应用程序        │ │
│  │ 传感器/执行器│ │  中间件         │ │
│  └───────────┘  └─────────────────┘ │
└─────────────────────────────────────┘
```

### 1.3 常见嵌入式处理器类型

| 类型 | 代表产品 | 特点 | 适用场景 |
|------|---------|------|---------|
| **MCU**（微控制器） | STM32、ATmega328 | 低成本、低功耗、资源有限 | 简单控制、传感器采集 |
| **MPU**（微处理器） | ARM Cortex-A 系列 | 较强性能、可运行 Linux | 复杂控制、图形显示 |
| **DSP**（数字信号处理器） | TMS320 | 擅长数值运算 | 信号处理、音频处理 |
| **SoC**（片上系统） | ESP32、Jetson Nano | 高度集成、多核 | 无线通信、AI推理 |

## 2. 嵌入式系统在机器人中的应用

### 2.1 机器人的分层架构

机器人系统通常采用分层架构，嵌入式系统位于**底层控制层**：

```
┌─────────────────────────────────────────┐
│          应用层（Application）            │
│   任务规划、路径规划、SLAM、视觉识别       │
├─────────────────────────────────────────┤
│          中间层（Middleware）            │
│   ROS/ROS2 通信、导航栈、运动控制库       │
├─────────────────────────────────────────┤
│          嵌入式层（Embedded）            │ ← 本课程重点
│   电机控制、传感器采集、通信、实时控制     │
├─────────────────────────────────────────┤
│          硬件层（Hardware）              │
│   电机、传感器、主控芯片、执行机构         │
└─────────────────────────────────────────┘
```

### 2.2 嵌入式系统在机器人中的职责

1. **运动控制**：PWM 控制电机转速与方向
2. **传感器采集**：读取编码器、超声波、红外等传感器数据
3. **通信管理**：与上位机（ROS2）进行数据交换
4. **实时响应**：对紧急情况进行毫秒级响应
5. **安全监控**：过流保护、堵转检测、电池监测

## 3. 常见嵌入式平台

### 3.1 平台对比

| 平台 | 类型 | 主频 | RAM | Flash | 功耗 | 价格 | 典型机器人应用 |
|------|------|------|-----|-------|------|------|--------------|
| **STM32** | MCU | 72-180MHz | 20-256KB | 64KB-2MB | 低 | ¥10-100 | 电机控制、传感器采集 |
| **Arduino** | MCU | 8-84MHz | 2-32KB | 32-256KB | 低 | ¥20-80 | 简单机器人、教学 |
| **ESP32** | SoC | 160-240MHz | 520KB | 4MB | 中 | ¥15-40 | WiFi/蓝牙通信、遥控 |
| **Raspberry Pi** | MPU | 1.2-1.8GHz | 1-8GB | SD卡 | 高 | ¥200-600 | 视觉处理、边缘计算 |
| **Jetson Nano** | SoC | 1.43GHz 四核 | 4GB | SD卡 | 高 | ¥700-900 | AI推理、SLAM |

### 3.2 STM32 系列详解

STM32 是机器人嵌入式开发中最常用的 MCU 系列：

```c
/**
 * STM32 系列分类（按内核）
 * - STM32F1: Cortex-M3, 基础型, 适合简单控制
 * - STM32F4: Cortex-M4, 性能型, 带 DSP/FPU
 * - STM32F7: Cortex-M7, 高性能型
 * - STM32H7: Cortex-M7, 超高性能型
 * - STM32WB: 双核, 集成蓝牙
 */

// STM32 常用外设
// - GPIO: 通用输入输出
// - TIM: 定时器（生成 PWM）
// - UART: 串口通信
// - SPI/I2C: 高速传感器通信
// - ADC: 模拟信号采集
// - DMA: 直接内存访问
```

### 3.3 ESP32 详解

ESP32 是性价比极高的 WiFi+蓝牙 SoC，非常适合需要无线通信的机器人项目：

```c
/**
 * ESP32 特性
 * - 双核 Tensilica Xtensa LX6, 最高 240MHz
 * - 520KB SRAM + 4MB Flash
 * - WiFi 802.11 b/g/n + 蓝牙 v4.2/BLE
 * - 34 个 GPIO 引脚
 * - 支持 FreeRTOS
 */

// ESP32 常用开发框架
// - Arduino IDE: 简单易用，适合入门
// - ESP-IDF: 官方框架，功能完整，适合专业开发
// - PlatformIO: VSCode 插件，跨平台
```

## 4. 机器人嵌入式架构

### 4.1 主控层与执行层分离

现代机器人通常采用**主控层 + 执行层**的分离架构：

```
┌──────────────────────────────────────────────────┐
│              主控层（High Level）                 │
│  ┌────────────────┐  ┌────────────────────────┐  │
│  │  Jetson Nano/  │  │       ROS 2             │  │
│  │  Raspberry Pi  │  │  Navigation/SLAM/Vision │  │
│  └───────┬────────┘  └───────────┬────────────┘  │
│          │                      │                │
│          │    UART / SPI / USB  │                │
└──────────┼──────────────────────┼────────────────┘
           │                      │
           ▼                      ▼
┌──────────────────────────────────────────────────┐
│              执行层（Low Level）                   │
│  ┌────────────────┐  ┌────────────────────────┐  │
│  │    STM32       │  │    电机驱动/传感器      │  │
│  │  实时控制      │  │    PWM/ADC/GPIO         │  │
│  └────────────────┘  └────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

**为什么需要分层？**

- **主控层**：计算能力强，适合复杂算法（SLAM、路径规划）
- **执行层**：实时性强，适合精确控制（电机调速、传感器采集）
- **分层解耦**：上下层独立开发，维护性高

### 4.2 实时控制要求

机器人控制对**实时性**有严格要求：

| 控制类型 | 响应时间要求 | 示例 |
|---------|------------|------|
| 硬实时 | < 1ms | 紧急制动、安全停机 |
| 软实时 | < 10ms | 电机转速控制 |
| 非实时 | 无严格要求 | 日志记录、参数配置 |

**实时性影响因素：**
1. 操作系统调度延迟
2. 总线通信延迟
3. 中断处理时间
4. 代码执行时间

### 4.3 通信协议

#### UART（串口通信）

最常用的主控与嵌入式芯片之间的通信方式：

```c
/**
 * UART 通信简介
 * - 全称: Universal Asynchronous Receiver/Transmitter
 * - 异步通信，无需时钟线
 * - 常用波特率: 9600, 115200, 460800 bps
 * - 接线: TX(发送) ↔ RX(接收), GND 共地
 */

// STM32 UART 初始化示例
void uart_init(UART_HandleTypeDef *huart) {
    huart->Instance = USART1;
    huart->Init.BaudRate = 115200;      // 波特率 115200
    huart->Init.WordLength = UART_WORDLENGTH_8B;  // 8位数据位
    huart->Init.StopBits = UART_STOPBITS_1;       // 1位停止位
    huart->Init.Parity = UART_PARITY_NONE;         // 无校验
    huart->Init.Mode = UART_MODE_TX_RX;            // 收发模式
    HAL_UART_Init(huart);
}

// 发送数据
uint8_t tx_data[] = "Hello ROS2!";
HAL_UART_Transmit(&huart1, tx_data, sizeof(tx_data), HAL_MAX_DELAY);

// 接收数据（中断方式回调）
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    // 处理接收到的数据
    process_received_byte(rx_buffer[rx_index++]);
}
```

#### SPI（串行外设接口）

高速同步通信协议，适合连接显示模块、存储芯片、高分辨率传感器：

```c
/**
 * SPI 通信简介
 * - 同步通信，需要时钟线 SCK
 * - 全双工（同时发送和接收）
 * - 主从架构（一主多从）
 * - 速度可达数十 MHz
 */

// SPI 连接示意
// MISO ──── MISO  (主入从出)
// MOSI ──── MOSI  (主出从入)
// SCK  ──── SCK   (时钟线)
// CS   ──── CS    (片选，低有效)
```

#### I2C（集成电路总线）

适合连接低速传感器和外设，支持多设备挂在同一总线上：

```c
/**
 * I2C 通信简介
 * - 两线制：SDA（数据线）+ SCL（时钟线）
 * - 地址寻址，支持多个从设备
 * - 速度：标准模式 100kHz，快速模式 400kHz
 * - 需要上拉电阻
 */

// I2C 读取传感器示例（MPU6050 加速度计）
#define MPU6050_ADDR 0x68

int16_t mpu6050_read_axis(I2C_HandleTypeDef *hi2c, uint8_t reg_addr) {
    uint8_t data[2];
    // 发送寄存器地址
    HAL_I2C_Master_Transmit(hi2c, MPU6050_ADDR << 1, &reg_addr, 1, HAL_MAX_DELAY);
    // 读取 2 字节数据
    HAL_I2C_Master_Receive(hi2c, MPU6050_ADDR << 1, data, 2, HAL_MAX_DELAY);
    return (int16_t)(data[0] << 8 | data[1]);  // 合成 16 位数据
}
```

#### CAN 总线（控制器局域网）

在汽车和工业机器人中广泛使用，抗干扰能力强，支持多主通信：

```c
/**
 * CAN 总线简介
 * - 高可靠性，适合恶劣环境
 * - 长距离通信（可达数公里）
 * - 多主通信，无中心节点
 * - 内置错误检测和重传机制
 * - 典型应用：汽车电子、工业机器人
 */

// CAN 报文发送示例
CAN_TxHeaderTypeDef tx_header;
uint8_t tx_data[8];

tx_header.StdId = 0x123;        // 标准 ID
tx_header.DLC = 4;              // 数据长度 4 字节
tx_header.IDE = CAN_ID_STD;     // 标准帧

tx_data[0] = motor_speed >> 8;  // 高字节
tx_data[1] = motor_speed & 0xFF; // 低字节

HAL_CAN_AddTxMessage(&hcan, &tx_header, tx_data, &tx_mailbox);
```

### 5.3 电机驱动与 PWM 控制

#### 电机驱动芯片

通常使用 H 桥驱动芯片，如 TB6612FNG 或 L298N：

```
        ┌─────────────────┐
        │   TB6612FNG     │
        │                 │
   AIN1─┤                 ├─ BIN1
   AIN2─┤                 ├─ BIN2
  PWMA─┤                 ├─ PWMB
   STBY─┤                 │
        └─────────────────┘

  AIN1/AIN2: 控制 A 电机方向
  PWMA: 控制 A 电机速度（PWM）
  STBY: 待机引脚，高电平使能
```

#### PWM 控制原理

PWM（脉宽调制）通过调整脉冲宽度来控制电机平均电压：

```c
/**
 * PWM 控制电机速度原理
 * 占空比 = 高电平时间 / 周期时间
 * 占空比越大，平均电压越高，电机转速越快
 *
 * 示例（假设频率 1kHz，周期 1ms）：
 * 占空比 0%   → 平均电压 0V  → 电机停止
 * 占空比 50%  → 平均电压 6V  → 电机半速
 * 占空比 100% → 平均电压 12V → 电机全速
 */

// ESP32 PWM 配置示例
#define MOTOR_PWM_FREQ 1000    // PWM 频率 1kHz
#define MOTOR_PWM_RESOLUTION 8 // 8 位分辨率，0-255
#define MOTOR_L_PWM_PIN 2      // 左电机 PWM 引脚

void motor_init(void) {
    // 配置 LEDC（PWM）通道
    ledc_timer_config_t timer_conf = {
        .speed_mode = LEDC_LOW_SPEED_MODE,
        .timer_num = LEDC_TIMER_0,
        .freq_hz = MOTOR_PWM_FREQ,
        .resolution = MOTOR_PWM_RESOLUTION,
    };
    ledc_timer_config(&timer_conf);

    ledc_channel_config_t channel_conf = {
        .gpio_num = MOTOR_L_PWM_PIN,
        .speed_mode = LEDC_LOW_SPEED_MODE,
        .channel = LEDC_CHANNEL_0,
        .timer_sel = LEDC_TIMER_0,
        .duty = 0,
    };
    ledc_channel_config(&channel_conf);
}

// 设置电机速度（0-255）
void motor_set_speed(uint8_t speed) {
    // ESP32 最大占空比 = 2^resolution - 1 = 255
    ledc_set_duty(LEDC_LOW_SPEED_MODE, LEDC_CHANNEL_0, speed);
    ledc_update_duty(LEDC_LOW_SPEED_MODE, LEDC_CHANNEL_0);
}
```

### 5.4 传感器接口

#### 编码器（测速）

通过读取编码器脉冲计算电机转速：

```c
/**
 * 编码器原理
 * - 霍尔编码器：每转产生固定数量的脉冲
 * - AB 相编码器：可检测转向
 * 
 * 转速计算：
 * RPM = (脉冲数 / 每转脉冲数) * (60 / 采样时间秒)
 */

// ESP32 编码器读取示例（外部中断方式）
#define ENCODER_A_PIN 4  // A 相引脚
#define ENCODER_B_PIN 5  // B 相引脚

volatile int encoder_pulses = 0;  // 脉冲计数器（使用 volatile 防止编译器优化）

void IRAM_ATTR encoder_isr(void) {
    // 读取 B 相判断转向
    if (gpio_get_level(ENCODER_B_PIN)) {
        encoder_pulses++;  // 正转
    } else {
        encoder_pulses--;  // 反转
    }
}

void encoder_init(void) {
    gpio_set_direction(ENCODER_A_PIN, GPIO_MODE_INPUT);
    gpio_set_direction(ENCODER_B_PIN, GPIO_MODE_INPUT);
    // 上升沿触发中断
    gpio_set_intr_type(ENCODER_A_PIN, GPIO_INTR_POSEDGE);
    gpio_install_isr_service(0);
    gpio_isr_handler_add(ENCODER_A_PIN, encoder_isr, NULL);
}

// 定时读取转速（每 100ms 读取一次）
void calculate_rpm(void) {
    int pulses = encoder_pulses;
    encoder_pulses = 0;  // 清零计数器
    
    // 假设：每转 390 脉冲，100ms 内记录脉冲数
    float rpm = (pulses * 600.0f) / 390.0f;  // 转换为 RPM
    printf("Motor RPM: %.1f\n", rpm);
}
```

#### 超声波传感器（测距）

```c
/**
 * HC-SR04 超声波测距原理
 * 1. 发送 10us 触发脉冲
 * 2. 模块自动发送 8 个 40kHz 超声波脉冲
 * 3. 测量回波信号宽度
 * 4. 距离 = 声速 * 回波时间 / 2
 * 
 * 声速约 340m/s，换算后：距离(cm) = 回波时间(μs) / 58
 */

#define TRIG_PIN 12
#define ECHO_PIN 13

float ultrasonic_read_distance(void) {
    // 发送触发信号
    gpio_set_level(TRIG_PIN, 1);
    ets_delay_us(10);
    gpio_set_level(TRIG_PIN, 0);
    
    // 等待回波开始
    uint32_t start_time = 0;
    while (gpio_get_level(ECHO_PIN) == 0) {
        start_time = esp_timer_get_time();
    }
    
    // 等待回波结束
    uint32_t end_time = 0;
    while (gpio_get_level(ECHO_PIN) == 1) {
        end_time = esp_timer_get_time();
    }
    
    // 计算距离（单位：cm）
    float distance = (end_time - start_time) / 58.0f;
    return distance;
}
```

#### 红外传感器（巡线）

```c
/**
 * 红外巡线传感器原理
 * - 红外发射管发射红外光
 * - 红外光被地面反射后被接收管接收
 * - 黑线不反射红外光，白色地面反射
 * 
 * 输出：数字信号（0=检测到黑线，1=检测到白线）
 */

#define IR_SENSOR_LEFT_PIN 21
#define IR_SENSOR_RIGHT_PIN 22

bool ir_detect_line(void) {
    // 读取左右两个红外传感器
    bool left = !gpio_get_level(IR_SENSOR_LEFT_PIN);   // 低电平表示检测到黑线
    bool right = !gpio_get_level(IR_SENSOR_RIGHT_PIN);
    
    // 巡线逻辑
    if (left && right) {
        return true;  // 两侧都检测到黑线，机器人居中
    } else if (left) {
        // 偏左，需要右转
        motor_turn_right();
    } else if (right) {
        // 偏右，需要左转
        motor_turn_left();
    } else {
        // 未检测到黑线，沿直线前进
        motor_forward();
    }
    return false;
}
```

### 5.5 无线通信（蓝牙/WiFi）

FishBot 使用 ESP32 内置的无线模块进行通信：

```c
/**
 * ESP32 无线通信方式
 * 1. WiFi: 适合大流量数据传输，可作为 AP 或 Station
 * 2. 蓝牙: 适合低功耗、近距离控制
 * 3. BLE: 更低功耗，支持少量数据传输
 */

// WiFi 作为 Station 连接 ROS2
#define WIFI_SSID "FishBot_AP"
```

## 7. 实时操作系统（RTOS）入门

### 7.1 什么是 RTOS

**RTOS**（Real-Time Operating System，实时操作系统）是一种专门为实时应用设计的操作系统，能够在确定的时间内完成响应。

**实时 vs 非实时：**

| 特性 | 硬实时系统 | 软实时系统 | 非实时系统 |
|------|----------|----------|-----------|
| 响应时间 | 必须在截止时间前完成 | 尽量在截止时间前完成 | 无截止时间要求 |
| 超时后果 | 系统失败 | 性能下降 | 无影响 |
| 示例 | 安全气囊、飞机控制 | 视频播放 | Windows、Linux |

### 7.2 FreeRTOS 简介

FreeRTOS 是最流行的开源 RTOS，被广泛应用于嵌入式开发：

```c
/**
 * FreeRTOS 特性
 * - 开源免费（MIT 许可证）
 * - 轻量级（内核仅 3-9KB）
 * - 支持 40+ MCU 平台
 * - 任务调度、队列、信号量、互斥量
 * - 官方支持 ESP32
 */

// FreeRTOS 核心概念
// 1. 任务 (Task): 独立运行的代码单元
// 2. 队列 (Queue): 任务间通信
// 3. 信号量 (Semaphore): 事件同步
// 4. 互斥量 (Mutex): 防止竞态条件
```

### 7.3 任务与调度

```c
/**
 * FreeRTOS 任务示例
 * 
 * 任务状态：
 * - Running: 正在运行
 * - Ready: 就绪（等待 CPU）
 * - Blocked: 阻塞（等待事件）
 * - Suspended: 挂起（暂停）
 */

#include "freertos/FreeRTOS.h"
#include "freertos/task.h"

// 定义任务函数
void motor_control_task(void *pvParameters) {
    // pvParameters: 传入参数
    while (1) {
        // 读取传感器
        int16_t encoder_count = read_encoder();
        
        // 计算 PID
        int16_t pwm = pid_calculate(encoder_count);
        
        // 设置电机 PWM
        motor_set_speed(pwm);
        
        // 延时 10ms（100Hz 控制频率）
        vTaskDelay(pdMS_TO_TICKS(10));
    }
}

void sensor_read_task(void *pvParameters) {
    while (1) {
        // 读取超声波距离
        float distance = ultrasonic_read_distance();
        
        // 发送到消息队列（供其他任务使用）
        xQueueSend(distance_queue, &distance, 0);
        
        // 延时 50ms（20Hz 采样频率）
        vTaskDelay(pdMS_TO_TICKS(50));
    }
}

void app_main(void) {
    // 创建消息队列
    distance_queue = xQueueCreate(10, sizeof(float));
    
    // 创建任务（名称, 栈大小, 参数, 优先级, 句柄）
    xTaskCreate(motor_control_task, "motor_ctrl", 2048, NULL, 5, &motor_task_handle);
    xTaskCreate(sensor_read_task, "sensor", 2048, NULL, 3, &sensor_task_handle);
}
```

### 7.4 任务优先级

```c
/**
 * FreeRTOS 优先级说明
 * - 数值越大，优先级越高
 * - 高优先级任务会抢占低优先级任务
 * - configMAX_PRIORITIES: 最大优先级数（通常 5-24）
 * 
 * 示例优先级分配：
 * 优先级 5 (最高): 安全控制任务（急停、碰撞检测）
 * 优先级 4: 电机控制任务（100Hz+）
 * 优先级 3: 传感器采集任务（50Hz）
 * 优先级 2: 通信任务（UART/WiFi）
 * 优先级 1 (最低): 日志记录任务

// 时间片轮转
// 同优先级的任务轮流执行，时间片长度由 configTICK_RATE_HZ 决定
// 例如 configTICK_RATE_HZ = 1000，则时间片 = 1ms
```

### 7.5 任务间通信

```c
/**
 * 任务间通信方式
 * 1. 队列 (Queue): 发送和接收数据
 * 2. 信号量 (Semaphore): 事件同步
 * 3. 互斥量 (Mutex): 共享资源保护

// 队列通信示例
QueueHandle_t cmd_queue;  // 命令队列
```

### 8.2 常用通信方式

#### 方式一：串口 + 自定义协议

```c
/**
 * 串口通信协议设计
 * 
 * 速度指令格式（ROS2 → 嵌入式）：
 * | 帧头 | 长度 | CMD | 数据 | 校验 | 帧尾 |
 * | 0xFF  | 1B   | 1B  | N B | 1B   | 0xFE |
 * 
 * 里程计数据格式（嵌入式 → ROS2）：
 * | 帧头 | 长度 | CMD | 左编码器 | 右编码器 | 校验 | 帧尾 |
 * | 0xFF  | 1B   | 1B  | 2B       | 2B       | 1B   | 0xFE |
 */

// 发送速度指令给电机
#define CMD_SET_VELOCITY 0x01

typedef struct {
    int16_t linear;   // 线速度 (mm/s)
    int16_t angular;  // 角速度 (mrad/s)
} velocity_cmd_t;

void send_velocity_cmd(velocity_cmd_t *cmd) {
    uint8_t frame[9];
    frame[0] = 0xFF;                    // 帧头
    frame[1] = 5;                        // 长度
    frame[2] = CMD_SET_VELOCITY;         // 命令字
    frame[3] = (cmd->linear >> 8) & 0xFF;  // 线速度高字节
    frame[4] = cmd->linear & 0xFF;         // 线速度低字节
    frame[5] = (cmd->angular >> 8) & 0xFF; // 角速度高字节
    frame[6] = cmd->angular & 0xFF;         // 角速度低字节
    frame[7] = calculate_checksum(frame, 7); // 校验和
    frame[8] = 0xFE;                    // 帧尾
    
    HAL_UART_Transmit(&huart1, frame, 9, HAL_MAX_DELAY);
}
```

#### 方式二：WiFi + Socket（ESP32）

```c
/**
 * ESP32 WiFi Socket 通信
 * - 建立 TCP/UDP 连接
 * - ROS2 端使用 socket_bridge 或 rosserial
 */

// ESP32 TCP 服务器示例
#include "lwip/sockets.h"

#define PORT 8888

void tcp_server_task(void *pvParameters) {
    int server_fd, client_fd;
    struct sockaddr_in server_addr, client_addr;
    socklen_t client_len = sizeof(client_addr);
    
    // 创建 socket（IPv4, TCP）
    server_fd = socket(AF_INET, SOCK_STREAM, 0);
    
    // 配置服务器地址
    server_addr.sin_family = AF_INET;         // IPv4
    server_addr.sin_addr.s_addr = INADDR_ANY; // 监听所有网卡
    server_addr.sin_port = htons(PORT);       // 端口号
    
    // 绑定 socket 到地址
    bind(server_fd, (struct sockaddr *)&server_addr, sizeof(server_addr));
    
    // 开始监听（最大 5 个连接）
    listen(server_fd, 5);
    
    while (1) {
        // 接受客户端连接
        client_fd = accept(server_fd, (struct sockaddr *)&client_addr, &client_len);
        
        // 接收数据
        uint8_t rx_buffer[1024];
        int len = recv(client_fd, rx_buffer, sizeof(rx_buffer), 0);
        
        if (len > 0) {
            // 处理接收到的协议数据
            protocol_parse(rx_buffer, len);
        }
        
        // 发送响应（里程计数据等）
        uint8_t tx_buffer[64];
        int tx_len = protocol_build(tx_buffer, sizeof(tx_buffer));
        send(client_fd, tx_buffer, tx_len, 0);
        
        close(client_fd);
    }
}
```

### 9.1 GPIO 控制示例（PWM 调速）

```c
/**
 * 文件名: motor_pwm_control.c
 * 功能: 使用 STM32 HAL 库实现 PWM 控制电机速度
 * 硬件: STM32F103 + TB6612FNG 电机驱动
 */

#include "stm32f1xx_hal.h"

// 定义电机控制引脚
#define MOTOR_L_PWM_PIN  GPIO_PIN_0   // 左电机 PWM (TIM2_CH1)
#define MOTOR_L_IN1_PIN   GPIO_PIN_2   // 左电机方向控制
#define MOTOR_L_IN2_PIN   GPIO_PIN_3
#define MOTOR_R_PWM_PIN  GPIO_PIN_1   // 右电机 PWM (TIM2_CH2)
#define MOTOR_R_IN1_PIN   GPIO_PIN_4   // 右电机方向控制
#define MOTOR_R_IN2_PIN   GPIO_PIN_5

// PWM 配置参数
#define PWM_FREQUENCY    1000          // PWM 频率 1kHz
#define PWM_RESOLUTION    100          // 100% 占空比

TIM_HandleTypeDef htim2;               // 定时器句柄

/**
 * 电机初始化
 */
void motor_init(void) {
    GPIO_InitTypeDef GPIO_InitStruct = {0};
    
    // 使能时钟
    __HAL_RCC_TIM2_CLK_ENABLE();
    __HAL_RCC_GPIOA_CLK_ENABLE();
    
    // 配置 GPIO 为复用推挽输出
    GPIO_InitStruct.Pin = MOTOR_L_PWM_PIN | MOTOR_R_PWM_PIN |
                         MOTOR_L_IN1_PIN | MOTOR_L_IN2_PIN |
                         MOTOR_R_IN1_PIN | MOTOR_R_IN2_PIN;
    GPIO_InitStruct.Mode = GPIO_MODE_AF_PP;
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_HIGH;
    HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);
    
    // 配置定时器为 PWM 模式
    htim2.Instance = TIM2;
    htim2.Init.Prescaler = 72 - 1;              // 72MHz / 72 = 1MHz
    htim2.Init.CounterMode = TIM_COUNTERMODE_UP;
    htim2.Init.Period = 1000 - 1;                // 1MHz / 1000 = 1kHz
    htim2.Init.AutoReloadPreload = TIM_AUTORELOAD_PRELOAD_ENABLE;
    HAL_TIM_PWM_Init(&htim2);
    
    // 配置 PWM 通道
    TIM_OC_InitTypeDef sConfig = {0};
    sConfig.OCMode = TIM_OCMODE_PWM1;
    sConfig.Pulse = 0;                           // 初始占空比 0
    sConfig.OCPolarity = TIM_OCPOLARITY_HIGH;
    sConfig.OCFastMode = TIM_OCFAST_DISABLE;
    
    HAL_TIM_PWM_ConfigChannel(&htim2, &sConfig, TIM_CHANNEL_1);  // 左电机
    HAL_TIM_PWM_ConfigChannel(&htim2, &sConfig, TIM_CHANNEL_2);  // 右电机
    
    // 启动 PWM 输出
    HAL_TIM_PWM_Start(&htim2, TIM_CHANNEL_1);
    HAL_TIM_PWM_Start(&htim2, TIM_CHANNEL_2);
}

/**
 * 设置单个电机速度
 * @param motor: 电机编号 (0=左, 1=右)
 * @param speed: 速度值 (-100 到 100, 负数为反转)
 */
void motor_set_speed(uint8_t motor, int16_t speed) {
    uint32_t channel;
    GPIO_TypeDef* gpio;
    uint16_t in1_pin, in2_pin;
    
    if (motor == 0) {
        // 左电机
        channel = TIM_CHANNEL_1;
        gpio = GPIOA;
        in1_pin = MOTOR_L_IN1_PIN;
        in2_pin = MOTOR_L_IN2_PIN;
    } else {
        // 右电机
        channel = TIM_CHANNEL_2;
        gpio = GPIOA;
        in1_pin = MOTOR_R_IN1_PIN;
        in2_pin = MOTOR_R_IN2_PIN;
    }
    
    // 限制速度范围
    if (speed > 100) speed = 100;
    if (speed < -100) speed = -100;
    
    // 设置方向
    if (speed >= 0) {
        HAL_GPIO_WritePin(gpio, in1_pin, GPIO_PIN_SET);   // 正转
        HAL_GPIO_WritePin(gpio, in2_pin, GPIO_PIN_RESET);
    } else {
        HAL_GPIO_WritePin(gpio, in1_pin, GPIO_PIN_RESET); // 反转
        HAL_GPIO_WritePin(gpio, in2_pin, GPIO_PIN_SET);
        speed = -speed;  // 取绝对值
    }
    
    // 设置 PWM 占空比
    uint32_t pulse = (speed * 10);  // 0-100 → 0-1000
    __HAL_TIM_SET_COMPARE(&htim2, channel, pulse);
}

/**
 * 机器人直行控制
 * @param speed: 直行速度 (-100 到 100)
 */
void robot_move_forward(int16_t speed) {
    motor_set_speed(0, speed);  // 左电机
    motor_set_speed(1, speed); // 右电机
}

/**
 * 机器人原地旋转
 * @param angular: 旋转速度 (-100 到 100, 正值左转，负值右转)
 */
void robot_rotate(int16_t angular) {
    motor_set_speed(0, -angular); // 左电机反转
    motor_set_speed(1, angular); // 右电机正转
}
```

### 9.2 传感器数据读取示例

```c
/**
 * 文件名: sensor_read.c
 * 功能: 读取多种传感器的数据
 * 硬件: HC-SR04 超声波、编码器、红外巡线传感器
 */

#include "stm32f1xx_hal.h"

// ==================== 超声波传感器 ====================
#define ULTRASONIC_TRIG_PIN  GPIO_PIN_0
#define ULTRASONIC_ECHO_PIN  GPIO_PIN_1
#define ULTRASONIC_GPIO_PORT GPIOA

/**
 * 读取超声波距离（单位：cm）
 * 原理：发送触发信号，测量回波宽度
 */
float ultrasonic_read_distance(void) {
    uint32_t start_time, end_time;
    float distance;
    
    // 发送 10us 触发脉冲
    HAL_GPIO_WritePin(ULTRASONIC_GPIO_PORT, ULTRASONIC_TRIG_PIN, GPIO_PIN_SET);
    HAL_Delay_us(10);
    HAL_GPIO_WritePin(ULTRASONIC_GPIO_PORT, ULTRASONIC_TRIG_PIN, GPIO_PIN_RESET);
    
    // 等待回波开始（Echo 变高）
    uint32_t timeout = 30000;  // 30ms 超时
    while (HAL_GPIO_ReadPin(ULTRASONIC_GPIO_PORT, ULTRASONIC_ECHO_PIN) == GPIO_PIN_RESET) {
        if (timeout-- == 0) return -1;  // 超时返回 -1
    }
    start_time = __HAL_TIM_GET_COUNTER(&htim3);  // 读取定时器计数值
    
    // 等待回波结束（Echo 变低）
    timeout = 30000;
    while (HAL_GPIO_ReadPin(ULTRASONIC_GPIO_PORT, ULTRASONIC_ECHO_PIN) == GPIO_PIN_SET) {
        if (timeout-- == 0) return -1;
    }
    end_time = __HAL_TIM_GET_COUNTER(&htim3);
    
    // 计算距离：声速 340m/s，往返距离
    // 定时器频率 1MHz，每计数 1us
    uint32_t time_us = end_time - start_time;
    distance = (time_us * 340.0f) / 20000.0f;  // cm
    
    return distance;
}

 * @param encoder_count: 采样周期内的脉冲数

}
```

```c
/**
 * 文件名: uart_communication.c
 * 功能: STM32 与上位机之间的 UART 通信
 * 协议: 自定义帧格式 | 帧头 | 长度 | CMD | 数据 | 校验 | 帧尾 |
 */

#include "stm32f1xx_hal.h"

#define FRAME_HEAD      0xFF
#define FRAME_TAIL      0xFE
#define CMD_VELOCITY    0x01  // 速度控制指令
#define CMD_LED         0x02  // LED 控制指令
#define CMD_GET_SENSOR  0x03  // 读取传感器指令

UART_HandleTypeDef huart1;
uint8_t rx_buffer[256];       // 接收缓冲区
uint8_t rx_index = 0;         // 接收索引
uint8_t frame_buffer[256];    // 完整帧缓冲区

/**
 * UART 初始化
 * 波特率: 115200, 8N1
 */
void uart_init(void) {
    huart1.Instance = USART1;
    huart1.Init.BaudRate = 115200;
    huart1.Init.WordLength = UART_WORDLENGTH_8B;
    huart1.Init.StopBits = UART_STOPBITS_1;
    huart1.Init.Parity = UART_PARITY_NONE;
    huart1.Init.Mode = UART_MODE_TX_RX;
    huart1.Init.HwFlowCtl = UART_HWCONTROL_NONE;
    HAL_UART_Init(&huart1);
    
    // 使能接收中断
    HAL_UART_Receive_IT(&huart1, &rx_buffer[rx_index], 1);
}

/**
 * UART 接收中断回调
 * 解析自定义协议
 */
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    uint8_t data = rx_buffer[rx_index];
    
    if (data == FRAME_HEAD && rx_index == 0) {
        // 帧头，开始接收
        frame_buffer[rx_index++] = data;
    } else if (rx_index > 0) {
        frame_buffer[rx_index++] = data;
        
        // 检查帧尾
        if (data == FRAME_TAIL && rx_index >= 4) {
            // 完整帧接收完成，解析
            protocol_parse(frame_buffer, rx_index);
            rx_index = 0;  // 重置接收索引
        }
        
        // 防止缓冲区溢出
        if (rx_index >= sizeof(rx_buffer)) {
            rx_index = 0;
        }
    }
    
    // 继续接收下一个字节
    HAL_UART_Receive_IT(&huart1, &rx_buffer[rx_index], 1);
}

/**
 * 计算校验和（简单 XOR）
 */
uint8_t calculate_checksum(uint8_t *data, uint8_t len) {
    uint8_t checksum = 0;
    for (uint8_t i = 0; i < len; i++) {
        checksum ^= data[i];
    }
    return checksum;
}

/**
 * 发送数据帧
 */
void uart_send_frame(uint8_t cmd, uint8_t *data, uint8_t len) {
    uint8_t frame[256];
    uint8_t index = 0;
    
    frame[index++] = FRAME_HEAD;      // 帧头
    frame[index++] = len + 2;          // 长度（数据+CMD+校验）
    frame[index++] = cmd;             // 命令字
    
    // 复制数据
    for (uint8_t i = 0; i < len; i++) {
        frame[index++] = data[i];
    }
    
    // 计算并添加校验和
    frame[index++] = calculate_checksum(&frame[1], index - 1);
    frame[index++] = FRAME_TAIL;      // 帧尾
    
    // 发送
    HAL_UART_Transmit(&huart1, frame, index, HAL_MAX_DELAY);
}

/**
 * 解析接收到的协议帧
 */
void protocol_parse(uint8_t *frame, uint8_t len) {
    if (len < 5) return;  // 最小帧长度检查
    
    uint8_t frame_len = frame[1];
    uint8_t cmd = frame[2];
    uint8_t *data = &frame[3];
    uint8_t received_checksum = frame[len - 2];
    
    // 校验
    uint8_t calculated_checksum = calculate_checksum(&frame[1], len - 3);
    if (received_checksum != calculated_checksum) {
        // 校验失败，忽略
        return;
    }
    
    // 处理命令
    switch (cmd) {
        case CMD_VELOCITY: {
            // 速度控制指令：data[0]=左速度, data[1]=右速度（有符号）
            int16_t left_speed = (int8_t)data[0];
            int16_t right_speed = (int8_t)data[1];
            motor_set_speed(0, left_speed);
            motor_set_speed(1, right_speed);
            break;
        }
        case CMD_LED: {
            // LED 控制：data[0]=状态（0=关，1=开）
            HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13, data[0] ? GPIO_PIN_SET : GPIO_PIN_RESET);
            break;
        }
        case CMD_GET_SENSOR: {
            // 读取传感器数据并上报
            float distance = ultrasonic_read_distance();
            int32_t left_encoder = encoder_left_count;
            int32_t right_encoder = encoder_right_count;
            
            uint8_t response[12];
            response[0] = (uint8_t)((uint16_t)distance >> 8);  // 距离高字节
            response[1] = (uint8_t)((uint16_t)distance & 0xFF); // 距离低字节
            response[2] = (uint8_t)(left_encoder >> 24);
            response[3] = (uint8_t)(left_encoder >> 16);
            response[4] = (uint8_t)(left_encoder >> 8);
            response[5] = (uint8_t)(left_encoder & 0xFF);
            response[6] = (uint8_t)(right_encoder >> 24);
            response[7] = (uint8_t)(right_encoder >> 16);
            response[8] = (uint8_t)(right_encoder >> 8);
            response[9] = (uint8_t)(right_encoder & 0xFF);
            
            uart_send_frame(CMD_GET_SENSOR, response, 10);
            break;
        }
    }
}

/**
 * 主函数示例
 */
int main(void) {
    HAL_Init();
    SystemClock_Config();
    uart_init();
    motor_init();
    
    while (1) {
        // 主循环可以处理其他任务
        // UART 中断会在后台接收数据
        HAL_Delay(100);
    }
}
```

### 二、简答题答案

**6. 嵌入式系统在机器人中的作用：**

1. **运动控制**：通过 PWM 信号精确控制电机转速和方向
2. **传感器采集**：实时读取编码器、超声波、红外、IMU 等传感器数据
3. **通信管理**：与上位机或 ROS2 进行数据交换（WiFi/蓝牙/UART）
4. **实时响应**：对紧急情况进行毫秒级响应（如急停、碰撞检测）
5. **安全监控**：过流保护、堵转检测、电池电压监测等
6. **数据处理**：对传感器原始数据进行滤波、融合等预处理

**7. UART、SPI、I2C 对比：**

| 特性 | UART | SPI | I2C |
|------|------|-----|-----|
| 线数 | 2（TX+RX） | 4（MISO/MOSI/SCK/CS） | 2（SDA/SCL） |
| 通信模式 | 异步 | 同步 | 同步 |
| 速度 | 最高 ~10Mbps | 最高数十 MHz | 最高 ~5Mbps |
| 主从 | 点对点 | 一主多从 | 一主多从 |
| 优点 | 简单、只需两根线 | 高速、全双工 | 仅需两根线、可接多设备 |
| 缺点 | 速度较低、无多主 | 线多、无流控 | 速度较低、需要上拉电阻 |
| 典型应用 | 主控与 ESP32 通信 | 显示模块、存储芯片 | 传感器芯片（MPU6050） |

**8. FishBot 选择 ESP32-C3 的原因：**

1. **内置无线**：集成 WiFi 802.11b/g/n + 蓝牙 5.0，无需额外通信模块，降低成本和复杂度
2. **RISC-V 内核**：开源免费，避免 ARM 授权费用
3. **低功耗**：适合电池供电的移动机器人
4. **丰富的外设**：GPIO、PWM、I2C、SPI、UART 等，满足机器人控制需求
5. **FreeRTOS 支持**：官方支持，开发效率高
6. **成本控制**：性价比高，适合教育和小批量产品

**9. RTOS 与通用操作系统的区别：**

| 特性 | RTOS | 通用操作系统 |
|------|------|-------------|
| 设计目标 | 实时响应，确定性 | 吞吐量和资源利用率 |
| 响应时间 | 硬实时有严格保证 | 不保证 |
| 内核大小 | 极小（3-100KB） | 庞大（数百MB） |
| 调度方式 | 优先级抢占、时间片 | 多种调度算法 |
| 应用场景 | 嵌入式、机器人、汽车 | PC、服务器、手机 |

**10. 嵌入式与 ROS2 通信的两种方式：**

| 方式 | 优点 | 缺点 |
|------|------|------|
| **串口+自定义协议** | 简单可靠、延迟低 | 协议不标准、需要手动解析 |
| **WiFi+Micro-ROS** | 与 ROS2 原生集成、开发快 | 需要 WiFi 环境、有一定延迟 |
| **蓝牙/BLE** | 低功耗、近距离 | 速度慢、不适合大数据 |
| **CAN 总线** | 高可靠、抗干扰 | 需要 CAN 转换器 |

```c
// 巡线状态
typedef enum {
    TRACK_CENTER,
    TRACK_LEFT,
    TRACK_RIGHT,
    TRACK_LOST
} TrackState;

// 读取巡线状态
TrackState read_track_state(void) {
    bool left = !gpio_get_level(IR_LEFT_PIN);   // 低=检测到黑线
    bool right = !gpio_get_level(IR_RIGHT_PIN);
    
    if (left && right) return TRACK_CENTER;
    if (left) return TRACK_LEFT;
    if (right) return TRACK_RIGHT;
    return TRACK_LOST;
}

// 巡线控制任务
void tracking_control_task(void *pvParameters) {
    while (1) {
        TrackState state = read_track_state();
        int16_t left_speed, right_speed;
        
        switch (state) {
            case TRACK_CENTER:
                // 居中，直行
                left_speed = 50;
                right_speed = 50;
                break;
            case TRACK_LEFT:
                // 偏左，右转修正
                left_speed = 60;
                right_speed = 20;
                break;
            case TRACK_RIGHT:
                // 偏右，左转修正
                left_speed = 20;
                right_speed = 60;
                break;
            case TRACK_LOST:
                // 丢失目标，减速搜索
                left_speed = 30;
                right_speed = 30;
                break;
        }
        
        motor_set_speed(0, left_speed);
        motor_set_speed(1, right_speed);
        
        vTaskDelay(pdMS_TO_TICKS(10));  // 100Hz 控制频率
    }
```
