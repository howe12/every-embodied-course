## 1. 传感器基础概念

### 1.1 传感器的定义与作用

传感器（Sensor）是机器人的"感官"，是将外界物理量转换为电信号的装置。在机器人系统中，传感器提供两类关键信息：

- **本体感知**：机器人自身状态（位置、速度、姿态）
- **环境感知**：外部环境信息（距离、障碍、亮度、温度）

```
         传感器在机器人系统中的地位

    ┌──────────────────────────────────────┐
    │           上位机（ROS2）              │
    │                                      │
    │    感知 → 决策 → 控制（闭环）         │
    └──────────────┬───────────────────────┘
                   │
                   ▼
    ┌──────────────────────────────────────┐
    │           下位机（STM32）             │
    │                                      │
    │  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐ │
    │  │编码器│  │IMU  │  │超声 │  │红外 │ │
    │  │测速 │  │测姿 │  │测距 │  │避障 │ │
    │  └─────┘  └─────┘  └─────┘  └─────┘ │
    └──────────────────────────────────────┘
                   │
                   ▼
    ┌──────────────────────────────────────┐
    │           执行器（电机）              │
    └──────────────────────────────────────┘
```

### 1.2 传感器分类

#### 按输出信号类型分类

| 类型 | 说明 | 示例 | 接口 |
|------|------|------|------|
| **模拟传感器** | 输出连续电压/电流信号 | 电位器、热敏电阻 | ADC |
| **数字传感器** | 输出离散数字信号 | 编码器、数字温度计 | GPIO、UART、I2C、SPI |

#### 按感知方式分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **主动传感器** | 自己发射能量，检测反射/响应 | 超声波、红外测距、激光雷达 |
| **被动传感器** | 接收外界自然能量 | 光敏电阻、摄像头、磁力计 |

#### 按测量物理量分类

| 类别 | 物理量 | 典型传感器 |
|------|--------|-----------|
| **位置/位移** | 距离、角度 | 编码器、电位器、超声波 |
| **速度** | 线速度、角速度 | 编码器、光流计 |
| **姿态** | 加速度、角速度、磁场 | IMU、磁力计 |
| **环境** | 温度、湿度、光强 | 热敏电阻、光敏电阻 |
| **接触** | 按压、碰撞 | 触碰开关、微动开关 |

### 1.3 传感器关键性能指标

#### 精度（Accuracy）

传感器测量值与真实值的接近程度，通常用误差表示：

```
误差 = |测量值 - 真实值|

例如：某温度传感器精度为 ±0.5°C
表示测量值与真实温度的偏差不超过 0.5°C
```

#### 分辨率（Resolution）

传感器能分辨的最小变化量：

```
模拟传感器分辨率取决于 ADC 位数：
  12位ADC，分辨率 = 1 / 2^12 = 1 / 4096 ≈ 0.024%

数字传感器分辨率由传感器本身决定：
  16位陀螺仪，分辨率 = 1 / 2^16 = 1 / 65536 （量程相关）
```

> **注意**：精度和分辨率是两个不同概念。高分辨率不等于高精度（可能高分辨率但偏差大），但高精度通常需要足够分辨率。

#### 量程（Range）

传感器能正常工作的测量范围：

```
例如：MPU6050加速度计量程可选 ±2g、±4g、±8g、±16g
选择量程时应使待测量接近量程中点，以获得最佳精度
```

#### 采样率（Sampling Rate）

传感器每秒能提供测量值的次数：

```
奈奎斯特采样定理：
采样率 ≥ 2 × 信号最高频率

例如：检测电机振动（最高100Hz）
最小采样率应 ≥ 200 Hz
实际应用中通常选择 5~10 倍信号最高频率
```

#### 响应时间（Response Time）

传感器从检测到变化到输出稳定所需的时间：

```
响应时间越短，传感器动态性能越好

典型响应时间：
  超声波传感器：约 10~50ms
  热敏电阻：约 100ms~1s
  红外测距：约 10~50ms
```

### 1.4 传感器选型依据

| 考虑因素 | 说明 | 示例 |
|----------|------|------|
| **测量范围** | 待测量的大小范围 | 测距选 0.1~4m 还是 0.1~8m |
| **精度要求** | 应用场景需要的测量精度 | 室内定位 vs 工业测量 |
| **响应速度** | 动态测量还是静态测量 | 实时避障 vs 环境监测 |
| **环境适应性** | 防水、防尘、抗干扰能力 | 户外机器人需要IP67 |
| **接口类型** | 与主控的接口兼容性 | 有哪些接口可用 |
| **功耗** | 电池供电需考虑功耗 | 低功耗传感器适合长时间续航 |
| **成本** | 预算限制 | 消费级 vs 工业级 |

## 2. 常用传感器接口

### 2.1 模拟接口（ADC）

#### 2.1.1 ADC原理

ADC（Analog-to-Digital Converter）将连续模拟电压转换为数字量。STM32F1系列通常配备12位ADC：

```
12位ADC：输出范围 0 ~ 4095（2^12 - 1）

模拟电压 → ADC数值转换公式：
ADC_value = Vin / Vref × 4095

例如：Vref = 3.3V，测量 1.65V
ADC_value = 1.65 / 3.3 × 4095 ≈ 2047（中间值）
```

#### 2.1.2 电位器分压电路

电位器是一种模拟位置传感器，通过旋转改变输出电压：

```
         电位器分压原理

    VCC ──┬─────────────────────── 3.3V
          │
        ┌─┴─┐
        │   │  电位器（10kΩ）
        └───┘
          │
          ├──● 输出到 ADC 引脚
          │
        ┌─┴─┐
        │   │  可变部分
        └───┘
          │
          │
         GND

输出电压 = VCC × (阻值比例)

旋钮在中间：输出 1.65V
旋钮在 1/4：输出 0.825V
旋钮在 3/4：输出 2.475V
```

#### 2.1.3 STM32 ADC配置示例

```c
// STM32 ADC 配置——读取电位器电压
#include "stm32f1xx_hal.h"

ADC_HandleTypeDef hadc1;

// ADC初始化——配置PA0为模拟输入
void ADC_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;

    // 使能ADC1和GPIOA时钟
    __HAL_RCC_ADC1_CLK_ENABLE();
    __HAL_RCC_GPIOA_CLK_ENABLE();

    // 配置PA0为模拟输入（无需上下拉）
    GPIO_InitStructure.Pin = GPIO_PIN_0;
    GPIO_InitStructure.Mode = GPIO_MODE_ANALOG;
    GPIO_InitStructure.Pull = GPIO_NOPULL;
    HAL_GPIO_Init(GPIOA, &GPIO_InitStructure);

    // 配置ADC参数
    hadc1.Instance = ADC1;
    hadc1.Init.DataAlign = ADC_DATAALIGN_RIGHT;   // 数据右对齐
    hadc1.Init.ScanConvMode = ADC_SCAN_DISABLE;   // 单通道扫描
    hadc1.Init.ContinuousConvMode = DISABLE;     // 单次转换
    hadc1.Init.NbrOfConversion = 1;              // 转换通道数
    HAL_ADC_Init(&hadc1);

    // 配置ADC通道
    ADC_ChannelConfTypeDef sConfig;
    sConfig.Channel = ADC_CHANNEL_0;      // PA0 对应通道0
    sConfig.Rank = ADC_REGULAR_RANK_1;   // 排名第1
    sConfig.SamplingTime = ADC_SAMPLETIME_55CYCLES_5; // 采样时间
    HAL_ADC_ConfigChannel(&hadc1, &sConfig);
}

// 读取ADC值（轮询方式）
uint16_t ADC_Read(void)
{
    uint16_t adc_value;

    // 启动ADC转换
    HAL_ADC_Start(&hadc1);

    // 等待转换完成（超时10ms）
    if (HAL_ADC_PollForConversion(&hadc1, 10) == HAL_OK) {
        adc_value = HAL_ADC_GetValue(&hadc1);
    }

    HAL_ADC_Stop(&hadc1);

    return adc_value;
}

// 读取电压值（浮点）
float ADC_Read_Voltage(void)
{
    uint16_t adc_value = ADC_Read();
    // 转换为电压值：Vin = ADC / 4095 × Vref
    return (float)adc_value / 4095.0f * 3.3f;
}
```

### 2.2 数字接口（GPIO / PWM输入）

#### 2.2.1 GPIO数字输入

简单的数字传感器（如触碰开关、行程开关）通过GPIO读取高低电平：

```c
// GPIO数字输入配置——读取触碰开关
void GPIO_Digital_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;

    // 使能GPIOB时钟
    __HAL_RCC_GPIOB_CLK_ENABLE();

    // 配置PB1为输入（带内部上拉）
    GPIO_InitStructure.Pin = GPIO_PIN_1;
    GPIO_InitStructure.Mode = GPIO_MODE_INPUT;
    GPIO_InitStructure.Pull = GPIO_PULLUP;    // 内部上拉
    GPIO_InitStructure.Speed = GPIO_SPEED_FREQ_LOW;
    HAL_GPIO_Init(GPIOB, &GPIO_InitStructure);
}

// 读取按键状态（按下为低电平）
uint8_t GPIO_Read_Button(void)
{
    // 返回 1 表示松开，0 表示按下
    return HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_1) == GPIO_PIN_SET ? 1 : 0;
}
```

#### 2.2.2 PWM输入测量

PWM输入用于测量占空比可变的数字信号（如红外测距传感器的模拟输出模式）：

```c
// 使用定时器输入捕获测量PWM占空比和频率
// 配置TIM3_CH1（PA6）为输入捕获
void PWM_Input_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;
    TIM_IC_InitTypeDef TIM_ICInitStructure;

    __HAL_RCC_TIM3_CLK_ENABLE();
    __HAL_RCC_GPIOA_CLK_ENABLE();

    // 配置PA6为复用功能（TIM3_CH1）
    GPIO_InitStructure.Pin = GPIO_PIN_6;
    GPIO_InitStructure.Mode = GPIO_MODE_AF_PP;
    GPIO_InitStructure.Pull = GPIO_PULLDOWN;
    GPIO_InitStructure.Speed = GPIO_SPEED_FREQ_HIGH;
    HAL_GPIO_Init(GPIOA, &GPIO_InitStructure);

    // 配置TIM3为输入捕获模式
    TIM3->PSC = 71;          // 预分频：72MHz/72 = 1MHz
    TIM3->ARR = 0xFFFF;      // 最大计数值

    // 配置通道1：上升沿捕获
    TIM_ICInitStructure.ICPolarity = TIM_ICPOLARITY_RISING;
    TIM_ICInitStructure.ICSelection = TIM_ICSELECTION_DIRECTTI;
    TIM_ICInitStructure.ICPrescaler = TIM_ICPSC_DIV1;
    TIM_ICInitStructure.ICFilter = 0;
    HAL_TIM_IC_ConfigChannel(&htim3, &TIM_ICInitStructure, TIM_CHANNEL_1);

    HAL_TIM_IC_Start_IT(&htim3, TIM_CHANNEL_1);
}

// 定时器中断中计算PWM占空比
volatile uint32_t capture_rising = 0;  // 上升沿时刻
volatile uint32_t capture_falling = 0; // 下降沿时刻
volatile float pwm_duty = 0.0f;         // 占空比（0.0~1.0）

void TIM3_IRQHandler(void)
{
    static uint8_t capture_state = 0;  // 0=等待上升沿，1=等待下降沿

    if (__HAL_TIM_GET_FLAG(&htim3, TIM_FLAG_CC1)) {
        if (capture_state == 0) {
            // 捕获上升沿
            capture_rising = HAL_TIM_ReadCapturedValue(&htim3, TIM_CHANNEL_1);
            // 切换为下降沿捕获
            __HAL_TIM_SET_POLARITY(&htim3, TIM_CHANNEL_1, TIM_ICPOLARITY_FALLING);
            capture_state = 1;
        } else {
            // 捕获下降沿
            capture_falling = HAL_TIM_ReadCapturedValue(&htim3, TIM_CHANNEL_1);
            // 计算占空比
            uint32_t period = capture_falling - capture_rising;
            // 注意：这里需要知道一个完整周期的计数值
            // 通过另一个通道捕获上升沿来计算周期
            pwm_duty = (float)capture_falling / (float)period; // 简化计算
            // 复位为上升沿捕获
            __HAL_TIM_SET_POLARITY(&htim3, TIM_CHANNEL_1, TIM_ICPOLARITY_RISING);
            capture_state = 0;
        }
        __HAL_TIM_CLEAR_FLAG(&htim3, TIM_FLAG_CC1);
    }
}
```

### 2.3 UART接口

UART（Universal Asynchronous Receiver/Transmitter）是一种异步串行通信接口，常用于连接GPS模块、蓝牙模块等：

```
UART通信接线示意

    STM32                    GPS模块
  ┌────────┐              ┌────────┐
  │   TX   │──────────────│   RX   │
  │   RX   │◄─────────────│   TX   │
  │   GND  │──────────────│   GND  │
  └────────┘              └────────┘

通信参数：波特率 9600/115200，数据位8位，停止位1位，无校验
```

#### GPS模块NMEA数据解析示例

```c
// GPS数据结构体
typedef struct {
    float latitude;     // 纬度（度）
    float longitude;    // 经度（度）
    float altitude;     // 海拔（米）
    uint8_t valid;      // 数据有效标志
} GPS_Data;

// 解析NMEA语句中的RMC信息
// 示例：$GPRMC,123519.00,A,3958.54,N,11627.42,E,0.0,0.0,261220,,*1E
uint8_t GPS_Parse_RMC(char *nmea, GPS_Data *gps)
{
    char *p = nmea;

    // 检查帧头
    if (strncmp(p, "$GPRMC", 6) != 0) {
        return 0;  // 不是RMC语句
    }

    // 查找时间字段（逗号分隔第1个字段）
    p = strchr(p, ',') + 1;  // 跳过时间
    // 跳过状态字段
    p = strchr(p, ',') + 1;
    if (*p == 'A') {
        gps->valid = 1;
    } else {
        gps->valid = 0;
        return 0;
    }

    // 解析纬度（ddmm.mmmm格式）
    p = strchr(p, ',') + 1;
    float lat_raw = atof(p);
    int lat_deg = (int)(lat_raw / 100);
    float lat_min = lat_raw - lat_deg * 100;
    gps->latitude = lat_deg + lat_min / 60.0f;

    // 解析N/S标识
    p = strchr(p, ',') + 1;
    if (*p == 'S') {
        gps->latitude = -gps->latitude;
    }

    // 解析经度（ddmm.mmmm格式）
    p = strchr(p, ',') + 1;
    float lon_raw = atof(p);
    int lon_deg = (int)(lon_raw / 100);
    float lon_min = lon_raw - lon_deg * 100;
    gps->longitude = lon_deg + lon_min / 60.0f;

    // 解析E/W标识
    p = strchr(p, ',') + 1;
    if (*p == 'W') {
        gps->longitude = -gps->longitude;
    }

    return 1;
}
```

### 2.4 I2C接口

I2C（Inter-Integrated Circuit）是一种两线制同步串行总线，适合连接低速外设：

```
I2C总线接线示意

    STM32                    MPU6050
  ┌────────┐              ┌────────┐
  │   SCL  │──────────────│   SCL  │
  │   SDA  │──────────────│   SDA  │
  │   GND  │──────────────│   GND  │
  └────────┘              └────────┘
          │                   │
          └─── 上拉电阻 4.7kΩ ─┘

I2C地址（MPU6050）：
  AD0=0 → 0x68 (110 1000)
  AD0=1 → 0x69 (110 1001)
```

#### I2C读写操作示例

```c
// STM32 HAL库 I2C操作函数
// I2C句柄 hi2c1（在CubeMX中配置）

// I2C写入寄存器
HAL_StatusTypeDef I2C_WriteReg(I2C_HandleTypeDef *hi2c,
                                 uint8_t addr,
                                 uint8_t reg,
                                 uint8_t data)
{
    return HAL_I2C_Mem_Write(hi2c,          // I2C句柄
                             addr << 1,     // 从机地址（7位左移）
                             reg,           // 寄存器地址
                             I2C_MEMADD_SIZE_8BIT,  // 地址长度
                             &data,         // 数据缓冲区
                             1,            // 数据长度
                             100);         // 超时时间ms
}

// I2C读取寄存器
HAL_StatusTypeDef I2C_ReadReg(I2C_HandleTypeDef *hi2c,
                                uint8_t addr,
                                uint8_t reg,
                                uint8_t *data)
{
    return HAL_I2C_Mem_Read(hi2c,
                            addr << 1,
                            reg,
                            I2C_MEMADD_SIZE_8BIT,
                            data,
                            1,
                            100);
}

// I2C读取多个字节
HAL_StatusTypeDef I2C_ReadMulti(I2C_HandleTypeDef *hi2c,
                                  uint8_t addr,
                                  uint8_t reg,
                                  uint8_t *data,
                                  uint16_t len)
{
    return HAL_I2C_Mem_Read(hi2c,
                            addr << 1,
                            reg,
                            I2C_MEMADD_SIZE_8BIT,
                            data,
                            len,
                            100);
}
```

### 2.5 SPI接口

SPI（Serial Peripheral Interface）是一种高速同步串行总线，适合连接高带宽传感器：

```
SPI总线接线示意（四线制）

    STM32                  更高带宽传感器（如OLED）
  ┌────────┐            ┌────────┐
  │   SCK  │────────────│   SCK  │
  │   MOSI │────────────│   MOSI │
  │   MISO │◄───────────│   MISO │
  │   NSS  │────────────│   NSS  │
  └────────┘            └────────┘

SPI模式：
  CPOL：时钟极性（空闲时高/低电平）
  CPHA：时钟相位（上升/下降沿采样）
  通常使用 mode 0（CPOL=0, CPHA=0）
```

#### SPI读取示例

```c
// STM32 HAL库 SPI操作函数
// SPI句柄 hspi1

// SPI发送接收一个字节
uint8_t SPI_TransmitReceive(uint8_t data)
{
    uint8_t received;
    HAL_SPI_TransmitReceive(&hspi1, &data, &received, 1, 100);
    return received;
}

// SPI读取寄存器（发送地址，接收数据）
uint8_t SPI_ReadReg(uint8_t reg_addr)
{
    uint8_t tx_data = reg_addr | 0x80;  // 最高位1表示读操作
    uint8_t rx_data;
    HAL_SPI_TransmitReceive(&hspi1, &tx_data, &rx_data, 1, 100);
    return rx_data;
}
```

### 2.6 各接口对比

| 接口 | 接线数 | 速度 | 典型设备 | 特点 |
|------|--------|------|----------|------|
| **ADC** | 1（模拟） | 慢（<1MHz） | 电位器、热敏电阻 | 简单，精度有限 |
| **GPIO** | 1（数字） | 最快 | 开关、按键 | 最简单，只区分0/1 |
| **UART** | 2 | 中（115kbps~1Mbps） | GPS、蓝牙 | 点对点，无需寻址 |
| **I2C** | 2 | 慢（100k~400kHz） | IMU、OLED、磁力计 | 总线式，多设备 |
| **SPI** | 4 | 快（>10MHz） | 高带宽传感器、SD卡 | 高速，需片选线 |

## 3. 机器人常用传感器

### 3.1 超声波传感器（HC-SR04）

#### 3.1.1 测距原理

超声波测距基于**飞行时间（TOF, Time of Flight）**原理：

```
         HC-SR04 超声波测距原理

    机器人        障碍物
       │             │
       │   超声波脉冲  │
       │    ───────►  │
       │   ◄───────   │
       │   反射波      │
       │             │
       └─────────────┘

距离计算公式：
  D = v_sound × t / 2

其中：
  v_sound ≈ 340 m/s（声速，受温度影响）
  t = 发射到接收的时间差

示例：t = 10ms 时，D = 340 × 0.01 / 2 = 1.7m
```

#### 3.1.2 时序图

```
HC-SR04 控制时序

    Trig ──┐     ┌──────────────────────
          │     │
          └─────┘
          │←10µs→│

    Echo ──────────┐                    ┌────
                    │                    │
                    └────────────────────┘
                    │←────── t ──────→│

时序说明：
1. Trig 发送 ≥10µs 的高电平触发信号
2. 模块自动发送 8 个 40kHz 超声波脉冲
3. Echo 输出高电平，持续时间 = 超声波飞行时间
4. 若超过 60ms 无反射，Echo 自动变低（表示无障碍）
```

#### 3.1.3 HC-SR04参数

| 参数 | 值 | 说明 |
|------|----|------|
| 工作电压 | 5V | 需要5V供电和5V兼容IO |
| 测量范围 | 2cm ~ 400cm | 最佳 2cm ~ 300cm |
| 测量角度 | ±15° | 声波发散角 |
| 精度 | ±3mm | 实际受温度影响较大 |
| 盲区 | < 2cm | 过近无法测量 |

#### 3.1.4 代码实现（STM32）

```c
// HC-SR04 超声波测距模块驱动
// Trig: PC6, Echo: PC7
#include "stm32f1xx_hal.h"

#define TRIG_PIN  GPIO_PIN_6
#define TRIG_PORT GPIOC
#define ECHO_PIN  GPIO_PIN_7
#define ECHO_PORT GPIOC

// 超声波测距初始化
void HC_SR04_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;

    // 使能GPIOC时钟
    __HAL_RCC_GPIOC_CLK_ENABLE();

    // 配置Trig为推挽输出
    GPIO_InitStructure.Pin = TRIG_PIN;
    GPIO_InitStructure.Mode = GPIO_MODE_OUTPUT_PP;
    GPIO_InitStructure.Pull = GPIO_NOPULL;
    GPIO_InitStructure.Speed = GPIO_SPEED_FREQ_LOW;
    HAL_GPIO_Init(TRIG_PORT, &GPIO_InitStructure);
    HAL_GPIO_WritePin(TRIG_PORT, TRIG_PIN, GPIO_PIN_RESET);

    // 配置Echo为输入（带下拉）
    GPIO_InitStructure.Pin = ECHO_PIN;
    GPIO_InitStructure.Mode = GPIO_MODE_INPUT;
    GPIO_InitStructure.Pull = GPIO_PULLDOWN;
    HAL_GPIO_Init(ECHO_PORT, &GPIO_InitStructure);
}

// 测量距离（单位：cm）
// 使用定时器2测量Echo高电平时间
float HC_SR04_Read(void)
{
    uint32_t timeout;
    float distance;

    // 1. 发送触发信号（≥10µs高电平）
    HAL_GPIO_WritePin(TRIG_PORT, TRIG_PIN, GPIO_PIN_SET);
    delay_us(10);    // 延时10µs
    HAL_GPIO_WritePin(TRIG_PORT, TRIG_PIN, GPIO_PIN_RESET);

    // 2. 等待Echo变为高电平（开始计时）
    timeout = 0;
    while (HAL_GPIO_ReadPin(ECHO_PORT, ECHO_PIN) == GPIO_PIN_RESET) {
        timeout++;
        if (timeout > 10000) {
            return -1.0f;  // 超时，无障碍物
        }
    }

    // 3. 启动定时器计数（TIM2，1MHz，即1µs计数器+1）
    __HAL_TIM_SET_COUNTER(&htim2, 0);
    __HAL_TIM_ENABLE(&htim2);

    // 4. 等待Echo变为低电平（停止计时）
    timeout = 0;
    while (HAL_GPIO_ReadPin(ECHO_PORT, ECHO_PIN) == GPIO_PIN_SET) {
        timeout++;
        if (timeout > 60000) {  // 60ms超时（对应约10m）
            __HAL_TIM_DISABLE(&htim2);
            return -1.0f;
        }
    }

    // 5. 读取定时器计数值（单位：µs）
    uint32_t echo_time = __HAL_TIM_GET_COUNTER(&htim2);
    __HAL_TIM_DISABLE(&htim2);

    // 6. 计算距离： D = v × t / 2
    // 声速随温度变化：v ≈ 331.3 + 0.606 × T(°C)
    // 假设室温25°C：v ≈ 346.5 m/s = 0.03465 cm/µs
    // 所以：D(cm) = echo_time(µs) × 0.03465 / 2 ≈ echo_time × 0.0173
    distance = echo_time * 0.0173f;

    // 7. 简单滤波：距离过小或过大视为无效
    if (distance < 2.0f || distance > 400.0f) {
        return -1.0f;
    }

    return distance;
}

// 微秒延时函数（基于SysTick）
void delay_us(uint32_t us)
{
    uint32_t ticks;
    uint32_t told, tnow, tcnt = 0;

    ticks = us * (HAL_RCC_GetHCLKFreq() / 1000000);  // 微秒对应的滴答数
    told = SysTick->VAL;
    tcnt = 0;

    while (1) {
        tnow = SysTick->VAL;
        if (tnow != told) {
            if (tnow < told) {
                tcnt += told - tnow;
            } else {
                tcnt += SysTick->LOAD - tnow + told;
            }
            told = tnow;
            if (tcnt >= ticks) {
                break;
            }
        }
    }
}
```

### 3.2 红外测距传感器（夏普GP2Y0A21）

#### 3.2.1 测距原理

夏普GP2Y0A21是一种基于三角测量原理的红外测距传感器：

```
         夏普红外测距原理（三角测量）

         发射的红外光
              \
               \  反射光
                \ /
                 ● ← PSD（位置敏感探测器）
                / \
               /   \
          红外LED    镜头
              ↑       ↑
           发射器     接收器

    障碍近 → 反射光角度大 → PSD中心位置偏移大 → 输出电压高
    障碍远 → 反射光角度小 → PSD中心位置偏移小 → 输出电压低
```

#### 3.2.2 输出特性

GP2Y0A21 输出模拟电压，与距离呈非线性关系：

| 距离 | 输出电压（约） |
|------|---------------|
| 10cm | 3.0V |
| 20cm | 1.9V |
| 40cm | 1.0V |
| 60cm | 0.65V |
| 80cm | 0.5V |

> **注意**：GP2Y0A21 在 10cm~80cm 范围内线性度较好，小于10cm有盲区，大于80cm信号微弱。

#### 3.2.3 代码实现

```c
// 夏普GP2Y0A21 红外测距
// 连接方式：VCC(5V)、GND、GND(外壳)、Vo(模拟输出到ADC)
// 使用ADC1通道1（PA1）读取

#define GP2Y0A21_ADC_CHANNEL  ADC_CHANNEL_1
#define GP2Y0A21_VCC         5.0f    // 工作电压5V

// 查找表法：将ADC值转换为距离
// 基于实测数据建立的非线性映射表
static const uint16_t adc_lookup[] = {
    // ADC值（12位），对应约80cm~10cm的adc值（倒序）
    50,   55,   60,   70,   80,   95,   110,  130,  // 80cm~70cm
    155,  185,  220,  260,  310,  370,  440,  520,  // 60cm~50cm
    600,  690,  780,  870,  950, 1020, 1080, 1130,  // 40cm~30cm
    1170, 1200, 1230, 1260, 1290, 1310, 1330, 1350, // 20cm~10cm
};

// 对应距离（cm），与查找表索引对应
static const float distance_lookup[] = {
    80, 75, 70, 65, 60, 55, 50, 45,
    40, 35, 30, 28, 25, 22, 20, 18,
    16, 15, 14, 13, 12, 11, 10, 9
};

// 读取红外测距距离（cm）
float GP2Y0A21_Read(void)
{
    uint16_t adc_value;
    float distance;
    int i;

    // 读取ADC值
    adc_value = ADC_Read();  // 使用前文ADC_Read函数

    // 边界检查
    if (adc_value <= adc_lookup[0]) {
        return distance_lookup[0];  // 最远距离
    }
    if (adc_value >= adc_lookup[sizeof(adc_lookup)-1]) {
        return distance_lookup[sizeof(distance_lookup)-1];  // 最近距离
    }

    // 线性插值查找对应距离
    for (i = 0; i < sizeof(adc_lookup)/sizeof(adc_lookup[0]) - 1; i++) {
        if (adc_value >= adc_lookup[i] && adc_value < adc_lookup[i+1]) {
            // 在adc_lookup[i]和adc_lookup[i+1]之间，进行线性插值
            float ratio = (float)(adc_value - adc_lookup[i]) /
                          (float)(adc_lookup[i+1] - adc_lookup[i]);
            distance = distance_lookup[i] +
                       ratio * (distance_lookup[i+1] - distance_lookup[i]);
            return distance;
        }
    }

    return -1.0f;  // 异常
}

// 公式法（基于经验公式）
// 注意：实际标定参数需要根据实测数据拟合
float GP2Y0A21_Formula(uint16_t adc_value)
{
    float voltage;
    float distance;

    // ADC值转换为电压：Vin = ADC / 4095 × 3.3V
    // 但由于GP2Y0A21使用5V供电，而ADC参考电压是3.3V
    // 需要用电阻分压（100k + 100k）将5V分压到2.5V输入ADC
    // 实际电压 = ADC / 4095 × 3.3V × 2

    voltage = (float)adc_value / 4095.0f * 3.3f * 2.0f;  // 还原实际电压

    // 经验公式（Sharp GP2Y0A21）
    // distance = a / (voltage - b)
    // 标定参数：a ≈ 27.0, b ≈ 0.42（需要实测调整）
    if (voltage > 0.42f) {
        distance = 27.0f / (voltage - 0.42f);
    } else {
        distance = 80.0f;  // 超出范围
    }

    return distance;
}
```

#### 3.3.4 MPU6050驱动代码

```c
// MPU6050 驱动结构体
typedef struct {
    float accel_x;      // 加速度 X (g)
    float accel_y;      // 加速度 Y (g)
    float accel_z;      // 加速度 Z (g)

    float gyro_x;       // 角速度 X (°/s)
    float gyro_y;       // 角速度 Y (°/s)
    float gyro_z;       // 角速度 Z (°/s)

    float temperature;  // 温度 (°C)

    // 量程配置
    uint8_t accel_scale;   // 0=±2g, 1=±4g, 2=±8g, 3=±16g
    uint8_t gyro_scale;    // 0=±250, 1=±500, 2=±1000, 3=±2000 °/s
} MPU6050_Data;

// I2C句柄（外部声明）
extern I2C_HandleTypeDef hi2c1;

// MPU6050 初始化
uint8_t MPU6050_Init(void)
{
    uint8_t whoami;
    uint8_t data;

    // 读取WHO_AM_I寄存器（应返回0x68）
    if (I2C_ReadReg(&hi2c1, MPU6050_ADDR, 0x75, &whoami) != HAL_OK) {
        return 0;
    }
    if (whoami != 0x68) {
        return 0;  // 设备ID错误
    }

    // 唤醒MPU6050（退出睡眠模式）
    data = 0x00;  // 内部8MHz振荡器，正常工作
    I2C_WriteReg(&hi2c1, MPU6050_ADDR, MPU6050_PWR_MGMT_1, data);

    // 配置采样率：SAMPLE_RATE = 1kHz / (1 + SMPLRT_DIV)
    data = 0x07;  // SMPLRT_DIV = 7 → 1000/(1+7) = 125Hz
    I2C_WriteReg(&hi2c1, MPU6050_ADDR, MPU6050_SMPLRT_DIV, data);

    // 配置低通滤波器（DLPF）
    data = 0x03;  // DLPF = 3（Accel Bandwidth 44Hz, Gyro Bandwidth 42Hz）
    I2C_WriteReg(&hi2c1, MPU6050_ADDR, MPU6050_CONFIG, data);

    // 配置加速度计量程：±2g
    data = 0x00;  // ACCEL_FS_SEL = 0 → ±2g
    I2C_WriteReg(&hi2c1, MPU6050_ADDR, MPU6050_ACCEL_CONFIG, data);

    // 配置陀螺仪量程：±250°/s
    data = 0x00;  // FS_SEL = 0 → ±250°/s
    I2C_WriteReg(&hi2c1, MPU6050_ADDR, MPU6050_GYRO_CONFIG, data);

    return 1;  // 初始化成功
}

// 读取原始数据（16位有符号）
void MPU6050_Read_Raw(int16_t *accel, int16_t *gyro, int16_t *temp)
{
    uint8_t buf[14];

    // 连续读取14字节（从ACCEL_XOUT_H开始）
    I2C_ReadMulti(&hi2c1, MPU6050_ADDR, MPU6050_ACCEL_XOUT_H, buf, 14);

    // 合并高低字节（高位在前）
    accel[0] = (int16_t)((buf[0] << 8) | buf[1]);   // X
    accel[1] = (int16_t)((buf[2] << 8) | buf[3]);   // Y
    accel[2] = (int16_t)((buf[4] << 8) | buf[5]);   // Z
    *temp    = (int16_t)((buf[6] << 8) | buf[7]);   // 温度
    gyro[0]  = (int16_t)((buf[8] << 8) | buf[9]);  // X
    gyro[1]  = (int16_t)((buf[10] << 8) | buf[11]); // Y
    gyro[2]  = (int16_t)((buf[12] << 8) | buf[13]); // Z
}

// 读取物理量（转换为标准单位）
void MPU6050_Read_Physics(MPU6050_Data *data)
{
    int16_t accel_raw[3], gyro_raw[3], temp_raw;

    MPU6050_Read_Raw(accel_raw, gyro_raw, &temp_raw);

    // 加速度转换：±2g量程，分辨率 = 2×2 / 65536 = 1/16384 g/LSB
    // 输出值除以16384得到加速度（单位g），再乘以9.8得到m/s²
    data->accel_x = (float)accel_raw[0] / 16384.0f * 9.80665f;
    data->accel_y = (float)accel_raw[1] / 16384.0f * 9.80665f;
    data->accel_z = (float)accel_raw[2] / 16384.0f * 9.80665f;

    // 陀螺仪转换：±250°/s量程，分辨率 = 500 / 65536 ≈ 1/131 °/s/LSB
    data->gyro_x = (float)gyro_raw[0] / 131.0f;
    data->gyro_y = (float)gyro_raw[1] / 131.0f;
    data->gyro_z = (float)gyro_raw[2] / 131.0f;

    // 温度转换：公式：T(°C) = (temp_raw / 340) + 36.53
    data->temperature = (float)temp_raw / 340.0f + 36.53f;
}
```

### 3.4 磁力计（HMC5883L）

#### 3.4.1 简介

HMC5883L是一款三轴磁力计，用于测量地球磁场方向，作为电子罗盘：

```
         HMC5883L 电子罗盘原理

              北
              ▲
              │
         ┌────┴────┐
         │ 磁场强度 │
         │ B ≈ 25~65 µT │
         └────┬────┘
              │
```

#### 3.4.2 HMC5883L驱动代码

```c

// HMC5883L 寄存器
#define HMC5883L_CONFIG_A  0x00
#define HMC5883L_CONFIG_B  0x01
#define HMC5883L_MODE       0x02
#define HMC5883L_DATA_X_H   0x03
#define HMC5883L_DATA_X_L   0x04
#define HMC5883L_DATA_Z_H   0x05
#define HMC5883L_DATA_Z_L   0x06
#define HMC5883L_DATA_Y_H   0x07
#define HMC5883L_DATA_Y_L   0x08

// HMC5883L 初始化

{
    uint8_t data;

    // 配置寄存器A：采样平均数=8，测量模式=正常
    data = 0x70;  // [7:6]平均次数=8，[5:4]测量模式=正常，[3:2]速率=15Hz，[1:0]=正常
    I2C_WriteReg(&hi2c1, HMC5883L_ADDR, HMC5883L_CONFIG_A, data);

    data = 0x20;  // [7:5]量程=±1.3Ga
    I2C_WriteReg(&hi2c1, HMC5883L_ADDR, HMC5883L_CONFIG_B, data);

    data = 0x00;
    I2C_WriteReg(&hi2c1, HMC5883L_ADDR, HMC5883L_MODE, data);
}

// 读取磁场数据
void HMC5883L_Read(int16_t *mx, int16_t *my, int16_t *mz)
{
    uint8_t buf[6];

    I2C_ReadMulti(&hi2c1, HMC5883L_ADDR, HMC5883L_DATA_X_H, buf, 6);

    // 注意：HMC5883L的X和Z轴是反的，需要根据实际安装调整
    *mx = (int16_t)((buf[0] << 8) | buf[1]);
    *mz = (int16_t)((buf[2] << 8) | buf[3]);  // Z在Y之前读
    *my = (int16_t)((buf[4] << 8) | buf[5]);
}

// 计算航向角（度）
float HMC5883L_Get_Heading(void)
{
    int16_t mx, my, mz;
    float heading;

    HMC5883L_Read(&mx, &my, &mz);

    // atan2得到弧度，转换为度
    heading = atan2((float)my, (float)mx) * 180.0f / 3.1415926f;

    // 修正到0~360范围
    if (heading < 0) {
        heading += 360.0f;
    }

    return heading;
}
```

## 4. 传感器数据采集

### 4.1 轮询方式

轮询是最简单的数据采集方式，CPU主动查询传感器状态：

```c
// 轮询方式读取超声波距离
void Ultrasonic_Polling(void)
{
    while (1) {
        float distance = HC_SR04_Read();
        if (distance > 0) {
            printf("Distance: %.2f cm\r\n", distance);

```

**优点**：逻辑简单，CPU完全控制  
**缺点**：浪费CPU资源，响应延迟不确定

### 4.2 中断驱动采集

中断方式在传感器数据就绪时触发CPU处理：

```c
// 中断驱动方式读取超声波
// 配置外部中断EXTI（Echo引脚上升沿触发）

void EXTI9_5_IRQHandler(void)
{
    static uint32_t echo_start = 0;
    static uint8_t measure_state = 0;  // 0=等待触发，1=等待回波

    if (__HAL_GPIO_EXTI_GET_FLAG(ECHO_PIN)) {
        if (measure_state == 0) {
            // 上升沿：开始计时
            echo_start = __HAL_TIM_GET_COUNTER(&htim2);
            measure_state = 1;

            // 切换到下降沿检测
            __HAL_TIM_SET_CAPTUREPOLARITY(&htim2, TIM_CHANNEL_1,
                                          TIM_ICPOLARITY_FALLING);
        } else {
            // 下降沿：计算时间
            uint32_t echo_end = __HAL_TIM_GET_COUNTER(&htim2);
            uint32_t echo_time = echo_end - echo_start;
            float distance = echo_time * 0.0173f;  // cm

            // 恢复初始状态
            __HAL_TIM_SET_CAPTUREPOLARITY(&htim2, TIM_CHANNEL_1,
                                          TIM_ICPOLARITY_RISING);
            measure_state = 0;

            // 处理距离数据
            if (distance > 2.0f && distance < 400.0f) {
                printf("Distance: %.2f cm\r\n", distance);
            }
        }
        __HAL_GPIO_EXTI_CLEAR_FLAG(ECHO_PIN);
    }
}
```

**优点**：CPU效率高，响应及时  
**缺点**：中断频繁时CPU开销大

## 5. 传感器数据滤波

### 5.1 均值滤波

最简单的滤波方法，对连续N次采样取平均：

```c
// 均值滤波
#define FILTER_WINDOW  10

typedef struct {
    float buffer[FILTER_WINDOW];
    uint8_t index;
    uint8_t count;
    float sum;
} MeanFilter;

// 均值滤波初始化
void MeanFilter_Init(MeanFilter *filter)
{
    memset(filter->buffer, 0, sizeof(filter->buffer));
    filter->index = 0;
    filter->count = 0;
    filter->sum = 0.0f;
}

// 添加新数据，返回滤波结果
float MeanFilter_Add(MeanFilter *filter, float new_value)
{
    // 如果缓冲区未满，直接累加
    if (filter->count < FILTER_WINDOW) {
        filter->count++;
    } else {
        // 缓冲区已满，减去最旧的值
        filter->sum -= filter->buffer[filter->index];
    }

    // 添加新值
    filter->buffer[filter->index] = new_value;
    filter->sum += new_value;
    filter->index = (filter->index + 1) % FILTER_WINDOW;

    // 返回平均值
    return filter->sum / (float)filter->count;
}
```

### 5.3 一阶低通滤波

一阶低通滤波对快速变化有抑制作用，适用于平滑噪声：

```c
// 一阶低通滤波（指数加权移动平均）
typedef struct {
    float alpha;   // 滤波系数 0~1，越小越平滑
    float output;  // 滤波输出
} LowPassFilter;

// 初始化（alpha越小越平滑，但响应越慢）
void LowPassFilter_Init(LowPassFilter *filter, float alpha)
{
    filter->alpha = alpha;
    filter->output = 0.0f;
}

// 添加数据，返回滤波结果
// formula: y(k) = α·x(k) + (1-α)·y(k-1)
float LowPassFilter_Update(LowPassFilter *filter, float input)
{
    filter->output = filter->alpha * input +
                     (1.0f - filter->alpha) * filter->output;
    return filter->output;
}

// 使用示例：
// alpha = 0.1：对噪声抑制强，但响应慢（适合速度平滑）
// alpha = 0.3：平衡方案（适合姿态角度）
// alpha = 0.8：对快速变化敏感（适合碰撞检测）
```

### 5.4 卡尔曼滤波

卡尔曼滤波是最优递归状态估计，适用于传感器融合：

```c
// 一维卡尔曼滤波器
typedef struct {
    float x;       // 状态估计值
    float p;       // 估计协方差（不确定性）
    float q;       // 过程噪声协方差（模型不确定度）
    float r;       // 测量噪声协方差（传感器噪声）
    float k;       // 卡尔曼增益
} KalmanFilter;

// 卡尔曼滤波器初始化
// q: 过程噪声（模型不确定度，越大越信任测量）
// r: 测量噪声（传感器噪声，越大越信任预测）
void KalmanFilter_Init(KalmanFilter *kf, float q, float r)
{
    kf->q = q;  // 过程噪声
    kf->r = r;  // 测量噪声
    kf->p = 1.0f;  // 初始估计协方差
    kf->x = 0.0f;  // 初始状态
    kf->k = 0.0f;  // 初始增益
}

// 卡尔曼滤波更新
// measurement: 当前测量值
// 返回: 最优估计值
float KalmanFilter_Update(KalmanFilter *kf, float measurement)
{
    // ===== 预测步骤 =====
    // 状态预测（这里假设是恒定模型，即不预测变化）
    // 协方差预测：P(k|k-1) = P(k-1|k-1) + Q
    kf->p = kf->p + kf->q;

    // ===== 更新步骤 =====
    // 计算卡尔曼增益：
    // K = P / (P + R)
    // K越小 → 更信任预测；K越大 → 更信任测量
    kf->k = kf->p / (kf->p + kf->r);

    // 状态更新：
    // x(k|k) = x(k|k-1) + K·(z(k) - x(k|k-1))
    kf->x = kf->x + kf->k * (measurement - kf->x);

    // 协方差更新：
    // P(k|k) = (1 - K) · P(k|k-1)
    kf->p = (1.0f - kf->k) * kf->p;

    return kf->x;
}

// 使用示例：
// 超声波测距卡尔曼滤波
KalmanFilter ultra_kf;
KalmanFilter_Init(&ultra_kf, 0.01f, 0.5f);  // q=0.01, r=0.5

float distance;
while (1) {
    float raw_distance = HC_SR04_Read();
    if (raw_distance > 0) {
        distance = KalmanFilter_Update(&ultra_kf, raw_distance);
        printf("Distance: %.2f cm\r\n", distance);
    }
    HAL_Delay(50);
}
```

### 5.5 互补滤波

互补滤波将不同传感器优势互补，常用于IMU姿态融合：

```
         互补滤波原理

    加速度计 ──────► 低通滤波 ──┐
                               ├─► 融合角度
    陀螺仪积分 ────► 高通滤波 ──┘

    低频噪声（加速度计）─── 通过低通滤波保留
    高频噪声（陀螺仪漂移）── 通过高通滤波消除
```

```c
// 互补滤波（用于IMU姿态融合）
typedef struct {
    float tau;    // 时间常数，越大越信任加速度计
    float angle;  // 融合后的角度
} ComplementaryFilter;

// 互补滤波初始化
void CompFilter_Init(ComplementaryFilter *cf, float tau)
{
    cf->tau = tau;
    cf->angle = 0.0f;
}

// 互补滤波更新
// accel_angle: 加速度计计算的角度（低频）
// gyro_rate: 陀螺仪角速度（高频）
// dt: 时间步长
float CompFilter_Update(ComplementaryFilter *cf,
                         float accel_angle,
                         float gyro_rate,
                         float dt)
{
    // 互补滤波公式：
    // angle = α * (angle + gyro_rate * dt) + (1-α) * accel_angle
    // 其中 α = tau / (tau + dt)

    float alpha = cf->tau / (cf->tau + dt);

    // 陀螺仪积分（预测）
    float gyro_predict = cf->angle + gyro_rate * dt;

    // 加速度计校正（融合）
    cf->angle = alpha * gyro_predict + (1.0f - alpha) * accel_angle;

    return cf->angle;
}

// 使用示例：
ComplementaryFilter roll_filter, pitch_filter;
CompFilter_Init(&roll_filter, 0.98f);   // tau=0.98s
CompFilter_Init(&pitch_filter, 0.98f);

MPU6050_Data imu;
float dt = 0.01f;  // 10ms周期

while (1) {
    MPU6050_Read_Physics(&imu);

    // 由加速度计计算角度（反正切）
    float accel_roll = atan2(imu.accel_y, imu.accel_z) * 180.0f / 3.14159f;
    float accel_pitch = atan2(-imu.accel_x,
                               sqrt(imu.accel_y*imu.accel_y + imu.accel_z*imu.accel_z))
                        * 180.0f / 3.14159f;

    // 互补滤波融合
    float roll = CompFilter_Update(&roll_filter, accel_roll, imu.gyro_x, dt);
    float pitch = CompFilter_Update(&pitch_filter, accel_pitch, imu.gyro_y, dt);

    printf("Roll: %.2f°, Pitch: %.2f°\r\n", roll, pitch);

    HAL_Delay(10);
}
```

```c
// 超声波传感器配置
typedef struct {
    GPIO_TypeDef *trig_port;
    uint16_t trig_pin;
    GPIO_TypeDef *echo_port;
    uint16_t echo_pin;
    TIM_HandleTypeDef *timer;
    uint8_t timer_channel;
} UltrasonicSensor;

// 三个传感器：前、左、右
static UltrasonicSensor us_front = {
    .trig_port = GPIOC, .trig_pin = GPIO_PIN_6,
    .echo_port = GPIOC, .echo_pin = GPIO_PIN_7,
    .timer = &htim2, .timer_channel = TIM_CHANNEL_1,
};

static UltrasonicSensor us_left = {
    .trig_port = GPIOC, .trig_pin = GPIO_PIN_8,
    .echo_port = GPIOC, .echo_pin = GPIO_PIN_9,
    .timer = &htim2, .timer_channel = TIM_CHANNEL_2,
};

static UltrasonicSensor us_right = {
    .trig_port = GPIOD, .trig_pin = GPIO_PIN_2,
    .echo_port = GPIOD, .echo_pin = GPIO_PIN_3,
    .timer = &htim4, .timer_channel = TIM_CHANNEL_1,
};

// 距离数据（cm）
volatile float us_front_dist = 0;
volatile float us_left_dist = 0;
volatile float us_right_dist = 0;

// 触发一个超声波传感器
void Ultrasonic_Trigger(UltrasonicSensor *us)
{
    // 发送10us触发脉冲
    HAL_GPIO_WritePin(us->trig_port, us->trig_pin, GPIO_PIN_SET);
    delay_us(10);
    HAL_GPIO_WritePin(us->trig_port, us->trig_pin, GPIO_PIN_RESET);
}

// 读取一个超声波传感器的距离（轮询方式）
float Ultrasonic_Read_Polling(UltrasonicSensor *us)
{
    uint32_t timeout;
    uint32_t start_tick, end_tick;

    // 触发
    Ultrasonic_Trigger(us);

    // 等待Echo高电平
    timeout = 60000;  // 60ms超时
    while (HAL_GPIO_ReadPin(us->echo_port, us->echo_pin) == GPIO_PIN_RESET) {
        if (timeout-- == 0) return -1.0f;
    }

    // 开始计时
    start_tick = __HAL_TIM_GET_COUNTER(us->timer);

    // 等待Echo低电平
    timeout = 60000;
    while (HAL_GPIO_ReadPin(us->echo_port, us->echo_pin) == GPIO_PIN_SET) {
        if (timeout-- == 0) return -1.0f;
    }

    // 停止计时
    end_tick = __HAL_TIM_GET_COUNTER(us->timer);

    // 计算距离
    uint32_t echo_time = end_tick - start_tick;
    if (echo_time > 60000) return -1.0f;

    float distance = echo_time * 0.0173f;  // cm

    // 简单滤波
    if (distance < 2.0f || distance > 400.0f) {
        return -1.0f;
    }

    return distance;
}

// 周期性任务（每50ms调用一次）
void Ultrasonic_Task(void)
{
    static uint8_t sensor_index = 0;
    static uint32_t last_trigger_time = 0;

    // 每个周期依次读取一个传感器
    switch (sensor_index) {
        case 0:
            us_front_dist = Ultrasonic_Read_Polling(&us_front);
            break;
        case 1:
            us_left_dist = Ultrasonic_Read_Polling(&us_left);
            break;
        case 2:
            us_right_dist = Ultrasonic_Read_Polling(&us_right);
            break;
    }

}

float Get_Min_Distance(void)

        min_dist = us_front_dist;
    if (us_left_dist > 0 && us_left_dist < min_dist)
        min_dist = us_left_dist;
    if (us_right_dist > 0 && us_right_dist < min_dist)

}
```

### 6.3 IMU数据读取完整代码

```c
// fishbot_imu.c - FishBot IMU数据读取与融合
// 使用MPU6050 + HMC5883L

#include "stm32f1xx_hal.h"

// IMU数据结构
typedef struct {
    // 加速度计（m/s²）
    float accel[3];

    // 陀螺仪（°/s）
    float gyro[3];

    // 磁场（处理后单位）
    float mag[3];

    // 融合后的姿态角（°）
    float roll;
    float pitch;
    float yaw;  // 需要磁力计
} IMU_Data;

static IMU_Data imu_data;

// 互补滤波
static ComplementaryFilter cf_roll;
static ComplementaryFilter cf_pitch;
static ComplementaryFilter cf_yaw;

// 卡尔曼滤波
static KalmanFilter kf_roll;
static KalmanFilter kf_pitch;

// IMU初始化
void IMU_Init(void)
{
    // 初始化MPU6050
    if (!MPU6050_Init()) {
        Error_Handler();  // MPU6050初始化失败
    }

    // 初始化HMC5883L
    HMC5883L_Init();

    // 初始化滤波
    CompFilter_Init(&cf_roll, 0.98f);
    CompFilter_Init(&cf_pitch, 0.98f);
    CompFilter_Init(&cf_yaw, 0.95f);  // yaw融合需要更信任磁力计

    KalmanFilter_Init(&kf_roll, 0.01f, 0.3f);
    KalmanFilter_Init(&kf_pitch, 0.01f, 0.3f);
}

// IMU更新（每10ms调用一次）
void IMU_Update(float dt)
{
    MPU6050_Data mpu_data;
    int16_t mag_x, mag_y, mag_z;

    // 读取MPU6050
    MPU6050_Read_Physics(&mpu_data);

    // 读取磁力计
    HMC5883L_Read(&mag_x, &mag_y, &mag_z);
    imu_data.mag[0] = (float)mag_x;
    imu_data.mag[1] = (float)mag_y;
    imu_data.mag[2] = (float)mag_z;

    // 存储加速度和陀螺仪
    imu_data.accel[0] = mpu_data.accel_x;
    imu_data.accel[1] = mpu_data.accel_y;
    imu_data.accel[2] = mpu_data.accel_z;

    imu_data.gyro[0] = mpu_data.gyro_x;
    imu_data.gyro[1] = mpu_data.gyro_y;
    imu_data.gyro[2] = mpu_data.gyro_z;

    // 由加速度计计算Roll和Pitch
    float accel_roll = atan2(mpu_data.accel_y, mpu_data.accel_z) * 180.0f / 3.14159f;
    float accel_pitch = atan2(-mpu_data.accel_x,
                               sqrt(mpu_data.accel_y * mpu_data.accel_y +
                                    mpu_data.accel_z * mpu_data.accel_z))
                        * 180.0f / 3.14159f;

    // 互补滤波融合
    imu_data.roll = CompFilter_Update(&cf_roll, accel_roll, mpu_data.gyro_x, dt);
    imu_data.pitch = CompFilter_Update(&cf_pitch, accel_pitch, mpu_data.gyro_y, dt);

    // 卡尔曼滤波（可选，与互补滤波二选一）
    // imu_data.roll = KalmanFilter_Update(&kf_roll, accel_roll);
    // imu_data.pitch = KalmanFilter_Update(&kf_pitch, accel_pitch);

    // 由磁力计计算Yaw
    float mag_heading = atan2((float)mag_y, (float)mag_x) * 180.0f / 3.14159f;
    if (mag_heading < 0) mag_heading += 360.0f;

    // 互补滤波融合Yaw（融合陀螺仪积分和磁力计）
    imu_data.yaw = CompFilter_Update(&cf_yaw, mag_heading, mpu_data.gyro_z, dt);

    if (imu_data.yaw < 0) imu_data.yaw += 360.0f;
    if (imu_data.yaw >= 360.0f) imu_data.yaw -= 360.0f;
}

// 获取IMU数据
IMU_Data* IMU_Get_Data(void)
{
    return &imu_data;
}
```

```c
    float heading;

    // 障碍物地图（二维栅格，简化版）
    float obstacle_front;   // 前方障碍距离
    float obstacle_left;    // 左方障碍距离
    float obstacle_right;   // 右方障碍距离
} SensorFusion_Data;

static SensorFusion_Data fusion_data;

// 传感器数据来源
extern volatile float us_front_dist;
extern volatile float us_left_dist;
extern volatile float us_right_dist;
extern volatile float motor_left_rpm;
extern volatile float motor_right_rpm;
extern IMU_Data* IMU_Get_Data(void);

// 航迹推算（Dead Reckoning）
// 基于编码器速度积分 + IMU航向
void SensorFusion_Update(float dt)
{
    IMU_Data *imu = IMU_Get_Data();

    // 更新超声波数据
    fusion_data.obstacle_front = (us_front_dist > 0) ? us_front_dist / 100.0f : -1.0f;
    fusion_data.obstacle_left  = (us_left_dist > 0)  ? us_left_dist / 100.0f : -1.0f;
    fusion_data.obstacle_right = (us_right_dist > 0) ? us_right_dist / 100.0f : -1.0f;

    // 编码器速度转换为线速度（m/s）
    // RPM → m/s: v = RPM / 60 × 2π × wheel_radius
    float wheel_radius = 0.03f;  // FishBot轮子半径 3cm
    float left_speed = motor_left_rpm / 60.0f * 2.0f * 3.14159f * wheel_radius;
    float right_speed = motor_right_rpm / 60.0f * 2.0f * 3.14159f * wheel_radius;

    // 差速驱动运动学
    float v = (left_speed + right_speed) / 2.0f;  // 线速度
    float omega = (right_speed - left_speed) / 0.1f;  // 角速度，轮距0.1m

    // 更新航向（使用IMU航向）
    fusion_data.heading = imu->yaw;

    // 坐标更新（航迹推算）
    // 在机器人坐标系下：
    //   vx_robot = v（前进方向）
    //   vy_robot = 0（无侧向滑动）
    // 转换到世界坐标系需要旋转heading角度
    float heading_rad = fusion_data.heading * 3.14159f / 180.0f;
    fusion_data.vx = v * cos(heading_rad);
    fusion_data.vy = v * sin(heading_rad);

    // 积分更新位置
    fusion_data.x += fusion_data.vx * dt;
    fusion_data.y += fusion_data.vy * dt;
}

// 获取融合数据
SensorFusion_Data* SensorFusion_Get_Data(void)
{
    return &fusion_data;
}

// 简单避障决策
// 返回：0=直行，1=左转，2=右转，3=停止
uint8_t Obstacle_Avoidance_Decision(void)
{
    float front = fusion_data.obstacle_front;
    float left = fusion_data.obstacle_left;
    float right = fusion_data.obstacle_right;

    // 阈值设置
    float danger_dist = 0.15f;   // 危险距离 15cm
    float warning_dist = 0.30f;  // 警告距离 30cm

    // 严重障碍，紧急停止
    if (front > 0 && front < danger_dist) {
        return 3;  // 停止
    }

    // 中等距离障碍
    if (front > 0 && front < warning_dist) {
        // 优先转向障碍物少的一侧
        if (left > 0 && left > right) {
            return 1;  // 左转
        } else {
            return 2;  // 右转
        }
    }

    // 正常直行
    return 0;
}
```

```c
// 超声波主循环
void ultrasonic_loop(void)
{
    static uint32_t last_update = 0;
    uint32_t now = HAL_GetTick();

    // 每50ms更新一次
    if (now - last_update < 50) return;
    last_update = now;

    // 读取三个超声波距离
    float dist_front = HC_SR04_Read();
    float dist_left = HC_SR04_Read();   // 实际应用中需要不同引脚
    float dist_right = HC_SR04_Read();

    // 更新时间戳
    uint64_t stamp = now * 1000000ULL;  // 转换为纳秒

    // 发布前超声波
    range_front_msg.header.stamp.sec = stamp / 1000000000ULL;
    range_front_msg.header.stamp.nanosec = stamp % 1000000000ULL;
    range_front_msg.range = (dist_front > 0) ? dist_front / 100.0f : -1.0f;
    rcl_publish(&range_front_pub, &range_front_msg, NULL);

    // 发布左超声波
    range_left_msg.header.stamp = range_front_msg.header.stamp;
    range_left_msg.range = (dist_left > 0) ? dist_left / 100.0f : -1.0f;
    rcl_publish(&range_left_pub, &range_left_msg, NULL);

    // 发布右超声波
    range_right_msg.header.stamp = range_front_msg.header.stamp;
    range_right_msg.range = (dist_right > 0) ? dist_right / 100.0f : -1.0f;
    rcl_publish(&range_right_pub, &range_right_msg, NULL);
}
```

### 7.3 IMU ROS2发布节点

```c
// fishbot_imu_node.c
// FishBot IMU ROS2发布节点

#include <micro_ros_arduino.h>
#include <sensor_msgs/msg/imu.h>
#include <sensor_msgs/msg/magnetic_field.h>
#include <geometry_msgs/msg/quaternion.h>
#include <rmw_microxrcedds_c/config.h>
#include <rclc/executor.h>
#include <rclc/rclc.h>

// ROS2节点和发布者
rcl_node_t imu_node;
rcl_publisher_t imu_pub;
rcl_publisher_t mag_pub;

sensor_msgs__msg__Imu imu_msg;
sensor_msgs__msg__MagneticField mag_msg;

// 四元数转换（欧拉角 → 四元数）
void euler_to_quaternion(float roll, float pitch, float yaw,
                          float *qx, float *qy, float *qz, float *qw)
{
    float cr = cos(roll * 0.5f);
    float sr = sin(roll * 0.5f);
    float cp = cos(pitch * 0.5f);
    float sp = sin(pitch * 0.5f);
    float cy = cos(yaw * 0.5f);
    float sy = sin(yaw * 0.5f);

    *qw = cr * cp * cy + sr * sp * sy;
    *qx = sr * cp * cy - cr * sp * sy;
    *qy = cr * sp * cy + sr * cp * sy;
    *qz = cr * cp * sy - sr * sp * cy;
}

// IMU ROS2发布
void imu_publish(float roll_deg, float pitch_deg, float yaw_deg,
                 float gyro_x, float gyro_y, float gyro_z,
                 float accel_x, float accel_y, float accel_z,
                 float mag_x, float mag_y, float mag_z)
{
    uint64_t stamp = HAL_GetTick() * 1000000ULL;

    // 填充IMU消息
    imu_msg.header.stamp.sec = stamp / 1000000000ULL;
    imu_msg.header.stamp.nanosec = stamp % 1000000000ULL;
    imu_msg.header.frame_id.data = "imu_link";
    imu_msg.header.frame_id.size = strlen("imu_link");

    // 转换欧拉角为四元数
    float qx, qy, qz, qw;
    float roll_rad = roll_deg * 3.14159f / 180.0f;
    float pitch_rad = pitch_deg * 3.14159f / 180.0f;
    float yaw_rad = yaw_deg * 3.14159f / 180.0f;
    euler_to_quaternion(roll_rad, pitch_rad, yaw_rad, &qx, &qy, &qz, &qw);

    imu_msg.orientation.x = qx;
    imu_msg.orientation.y = qy;
    imu_msg.orientation.z = qz;
    imu_msg.orientation.w = qw;

    // 角速度（rad/s）
    imu_msg.angular_velocity.x = gyro_x * 3.14159f / 180.0f;
    imu_msg.angular_velocity.y = gyro_y * 3.14159f / 180.0f;
    imu_msg.angular_velocity.z = gyro_z * 3.14159f / 180.0f;

    // 线加速度（m/s²）
    imu_msg.linear_acceleration.x = accel_x;
    imu_msg.linear_acceleration.y = accel_y;
    imu_msg.linear_acceleration.z = accel_z;

    rcl_publish(&imu_pub, &imu_msg, NULL);

    // 填充磁场消息
    mag_msg.header.stamp = imu_msg.header.stamp;
    mag_msg.header.frame_id.data = "mag_link";
    mag_msg.magnetic_field.x = mag_x * 0.00000015f;  // μT转换
    mag_msg.magnetic_field.y = mag_y * 0.00000015f;
    mag_msg.magnetic_field.z = mag_z * 0.00000015f;

    rcl_publish(&mag_pub, &mag_msg, NULL);
}
```

## 9. 答案

### 答案1：传感器接口识别

| 传感器 | 接口类型 | 理由 |
|--------|---------|------|
| HC-SR04超声波 | **GPIO + 定时器** | Trig输入（数字PWM），Echo输出（数字PWM测宽） |
| MPU6050 IMU | **I2C** | 内置I2C接口，三轴加速度+三轴陀螺仪可通过I2C配置和读取 |
| 夏普GP2Y0A21红外 | **ADC** | 输出模拟电压（0.4~3.1V），与距离成非线性关系 |
| GPS模块 | **UART** | NMEA协议是串行协议，标准UART通信 |
| 触碰开关 | **GPIO输入** | 简单的通/断数字信号，只需判断高低电平 |

---

### 答案2：超声波测距计算

**声速**：v = 346.5 m/s = 0.3465 m/ms = 0.03465 cm/μs

**计算**：
```
t = 5.8ms = 5800μs

距离 = v × t / 2
     = 0.03465 cm/μs × 5800 μs / 2
     = 200.97 cm / 2
     = 100.5 cm

≈ 1.005 m
```

---

### 答案3：ADC分辨率计算

**分辨率**：
```
分辨率 = Vref / (2^n - 1) = 3.3V / 4095 ≈ 0.806 mV/位
```

**2048对应电压**：
```
电压 = 2048 / 4095 × 3.3V ≈ 1.65V
```

**1mV分辨率所需位数**：
```
2^n ≥ 3.3V / 0.001V = 3300
n ≥ log2(3300) ≈ 11.7 → 至少12位
（12位ADC分辨率正好约 0.806mV/位，满足要求）
```

---

### 答案4：互补滤波参数分析

**α计算**：
```
α = tau / (tau + dt) = 0.98 / (0.98 + 0.01) = 0.98 / 0.99 ≈ 0.99
```

**α的意义**：
- α越大 → 更信任陀螺仪积分 → 响应快但漂移累积
- α越小 → 更信任加速度计 → 响应慢但稳定

**噪声场景下的选择**：
- 机器人震动大（加速度噪声大）→ 应该**增大α**
- 理由：减小加速度计权重，减少噪声影响；但代价是陀螺仪漂移累积更多

---

### 答案5：卡尔曼滤波分析

**计算步骤**：

**① 预测步骤**（假设恒定模型）：
```
P_pred = P + Q = 1.0 + 0.01 = 1.01
```

**② 卡尔曼增益**：
```
K = P_pred / (P_pred + R) = 1.01 / (1.01 + 0.5) = 1.01 / 1.51 ≈ 0.669
```

**③ 状态更新**（假设当前估计 x_pred = 上一时刻的 x = 150）：
```
x = x_pred + K × (z - x_pred)
  = 150 + 0.669 × (150 - 150)
  = 150 + 0 = 150 cm
```
（测量值=估计值，校正量为0）

**④ 协方差更新**：
```
P = (1 - K) × P_pred = (1 - 0.669) × 1.01 ≈ 0.331 × 1.01 ≈ 0.334
```

**协方差从1.0降低到0.334**，说明滤波器的估计不确定性在减小。

```
         │           └──────────────────┘                     │
         └────────────────────────────────────────────────────┘
```

**融合算法选择**：
- **简单场景**：互补滤波 + 航迹推算
- **复杂场景**：扩展卡尔曼滤波（EKF）或因子图优化

---

### 答案7：编码器接口复习

**每转计数次数**：
```
13脉冲 × 4跳变/脉冲 × 30减速比 = 1560 计数/转（电机轴）
编码器接口模式下：电机轴每转 13 × 4 = 52 个计数
输出轴每转 = 52 × 30 = 1560 个计数
```

**转速计算**：
```
每转1560计数，50ms内计数195个
每秒计数 = 195 / 0.05 = 3900 计数/秒
转速 = 3900 / 1560 = 2.5 转/秒 = 150 RPM
```

**接口类型**：编码器接口使用的是 **定时器编码器模式**（本质上还是GPIO的定时器外设功能），属于 **GPIO输入的增强模式**。

### 答案8：I2C地址与通信

**MPU6050地址**：
- AD0接地 → 7位地址 = `110_1000` = **0x68**
- 写地址（8位）= 0x68 << 1 | 0 = **0xD0**
- 读地址（8位）= 0x68 << 1 | 1 = **0xD1**

**I2C读取时序**：

```
    START
      │
      ▼
    SDA ──────┐                    ┌────────
               │ ADD7(1101000)+W(0) │ ACK
               └───────────────────┘
               │
               ▼
    SDA ────────────────────────┐
               │ REG8(0x75)       │ ACK
               └───────────────────┘
               │
           RESTART
               │
               ▼
    SDA ──────┐                    ┌────────
               │ ADD7(1101000)+R(1) │ ACK
               └───────────────────┘
               │
               ▼
    SDA ──────────────────────DATA(0x68)─► NACK ──┐
               └───────────────────────────────────┘
                                                      │
    STOP ◄────────────────────────────────────────────┘
```
