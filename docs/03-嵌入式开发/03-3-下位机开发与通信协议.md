## 1. 下位机开发概述

### 1.1 什么是下位机

**下位机**（Lower Computer / Slave Device）是机器人系统中直接与硬件交互的控制器，通常指直接控制传感器和执行器的嵌入式设备。下位机负责实时数据采集、运动控制等底层任务。

典型例子：
- **STM32 单片机**：实时性好，价格低廉，广泛用于电机控制和传感器采集
- **ESP32**：集成 WiFi/Bluetooth，适合无线机器人项目
- **Arduino**：入门级控制器，适合教学和快速原型

### 1.2 下位机在机器人系统中的角色

```
┌─────────────────────────────────────────────────────┐
│                   机器人系统架构                     │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │              上位机（决策层）                  │   │
│  │    • ROS2 主控计算机（Jetson Nano/树莓派）     │   │
│  │    • 路径规划、SLAM、导航                     │   │
│  │    • 传感器融合、决策制定                      │   │
│  └──────────────────┬──────────────────────────┘   │
│                     │  通信（USB/UART/CAN）          │
│  ┌──────────────────▼──────────────────────────┐   │
│  │              下位机（控制层）                  │   │
│  │    • STM32F1/F4 单片机                        │   │
│  │    • 电机PWM控制、编码器读取                   │   │
│  │    • 传感器数据采集（超声波、红外、IMU）        │   │
│  │    • 实时闭环控制（PID）                       │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 1.3 下位机 vs 上位机

| 对比项 | 上位机 | 下位机 |
|--------|--------|--------|
| **硬件平台** | x86/ARM Cortex-A（运行 Linux） | MCU（STM32、ESP32、ATmega） |
| **操作系统** | Linux / RTOS | 裸机或轻量级 RTOS |
| **计算能力** | 强（GHz 级 CPU） | 弱（72MHz~200MHz） |
| **实时性** | 一般（非实时 Linux 内核） | 高（硬件级实时中断） |
| **主要任务** | 路径规划、SLAM、视觉处理 | 电机控制、传感器采集、PID 闭环 |
| **编程语言** | Python、C++ | C/C++ |
| **通信方式** | 网络、串口 | UART、I2C、SPI、CAN |

> **关键理解**：上位机负责"大脑"的智能决策，下位机负责"小脑"的实时反应。两者通过通信协议交换数据，共同构成完整的机器人系统。

## 2. 常用通信协议

### 2.1 UART/串口通信

#### 2.1.1 原理概述

UART（Universal Asynchronous Receiver/Transmitter）是最常用的异步串行通信协议，只需要两根数据线（TX/RX）即可实现全双工通信。

```
┌──────────┐         TX          ┌──────────┐
│   MCU1   │ ─────────────────► │   MCU2   │
│  (发送)   │                    │  (接收)   │
│          │ ◄───────────────── │          │
│          │         RX         │          │
└──────────┘                    └──────────┘
```

**关键参数**：
- **波特率**（Baud Rate）：9600、115200、460800 等，双方必须一致
- **数据位**（Data Bits）：通常 8 位
- **停止位**（Stop Bits）：1 位或 2 位
- **校验位**（Parity）：无校验、奇校验、偶校验

#### 2.1.2 数据帧格式

```
┌──────┬────────┬─────────┬──────┬──────┐
│ 起始位 │ 数据位0-7 │  校验位  │ 停止位 │  空闲   │
│ (1bit)│ (5-8bit)│ (可选)   │(1-2bit)│       │
└──────┴────────┴─────────┴──────┴──────┘
```

标准的 8N1 格式（8 数据位、无校验、1 停止位）最为常用。

#### 2.1.3 代码实现（STM32 HAL）

```c
// STM32 HAL 串口初始化代码
#include "stm32f1xx_hal.h"

UART_HandleTypeDef huart1;  // 串口句柄

// 串口初始化函数
void MX_USART1_UART_Init(void)
{
    huart1.Instance = USART1;           // 使用 USART1 外设
    huart1.Init.BaudRate = 115200;       // 波特率 115200
    huart1.Init.WordLength = UART_WORDLENGTH_8B;  // 8 位数据位
    huart1.Init.StopBits = UART_STOPBITS_1;       // 1 位停止位
    huart1.Init.Parity = UART_PARITY_NONE;        // 无校验位
    huart1.Init.Mode = UART_MODE_TX_RX;           // 全双工模式
    huart1.Init.HwFlowCtl = UART_HWCONTROL_NONE;  // 无硬件流控
    huart1.Init.OverSampling = UART_OVERSAMPLING_16;  // 16 倍过采样

    if (HAL_UART_Init(&huart1) != HAL_OK)
    {
        // 初始化失败处理
        Error_Handler();
    }
}

// 发送一个字节
void UART_Send_Byte(uint8_t data)
{
    HAL_UART_Transmit(&huart1, &data, 1, HAL_MAX_DELAY);
}

// 接收一个字节（阻塞式）
uint8_t UART_Receive_Byte(void)
{
    uint8_t data;
    HAL_UART_Receive(&huart1, &data, 1, HAL_MAX_DELAY);
    return data;
}

// 发送字符串
void UART_Send_String(uint8_t *str)
{
    while (*str != '\0')
    {
        HAL_UART_Transmit(&huart1, (uint8_t *)str, 1, HAL_MAX_DELAY);
        str++;
    }
}
```

### 2.2 I2C 通信协议

#### 2.2.1 原理概述

I2C（Inter-Integrated Circuit）是一种两线制同步串行总线，只需要 SCL（时钟线）和 SDA（数据线）两根线，支持多个从设备挂在同一总线上。

```
         ┌─────────┐
    SCL ─┤         ├─── SCL ─┐
         │  Master  │          │
    SDA ─┤         ├─── SDA ─┤
         └─────────┘          │
                       ┌──────┴──────┐
                       │  Slave1  │  Slave2  │  Slave3
                       └──────────┘  └──────────┘
```

**特点**：
- 主从架构（Master 控制通信）
- 每个从设备有唯一的 7 位或 10 位地址
- 支持多个主设备（multi-master）
- 速度：标准模式 100kHz，快速模式 400kHz，高速模式 3.4MHz

#### 2.2.2 I2C 读写时序

```c
// I2C 读取传感器示例（以 MPU6050 为例）
#include "stm32f1xx_hal.h"

#define MPU6050_ADDR 0xD0  // MPU6050 I2C 地址（写）

// I2C 初始化
void I2C_Init(void)
{
    hi2c1.Instance = I2C1;
    hi2c1.Init.ClockSpeed = 400000;      // 400kHz 快速模式
    hi2c1.Init.DutyCycle = I2C_DUTYCYCLE_2;
    hi2c1.Init.OwnAddress1 = 0x00;
    hi2c1.Init.AddressingMode = I2C_ADDRESSINGMODE_7BIT;
    hi2c1.Init.DualAddressMode = I2C_DUALADDRESS_DISABLE;
    hi2c1.Init.OwnAddress2 = 0x00;
    hi2c1.Init.GeneralCallMode = I2C_GENERALCALL_DISABLE;
    hi2c1.Init.NoStretchMode = I2C_NOSTRETCH_DISABLE;

    HAL_I2C_Init(&hi2c1);
}

// 向 I2C 设备写入一个字节
HAL_StatusTypeDef I2C_Write_Byte(uint8_t addr, uint8_t reg, uint8_t data)
{
    uint8_t tx_data[2] = {reg, data};  // [寄存器地址, 数据]
    return HAL_I2C_Master_Transmit(&hi2c1, addr, tx_data, 2, HAL_MAX_DELAY);
}

// 从 I2C 设备读取一个字节
HAL_StatusTypeDef I2C_Read_Byte(uint8_t addr, uint8_t reg, uint8_t *data)
{
    // 先发送寄存器地址
    HAL_I2C_Master_Transmit(&hi2c1, addr, &reg, 1, HAL_MAX_DELAY);
    // 再读取数据
    return HAL_I2C_Master_Receive(&hi2c1, addr, data, 1, HAL_MAX_DELAY);
}

// 读取 MPU6050 原始加速度数据
void MPU6050_Read_Accel(int16_t *ax, int16_t *ay, int16_t *az)
{
    uint8_t raw_data[6];
    uint8_t reg_addr = 0x3B;  // 加速度数据首寄存器

    // 发送寄存器起始地址
    HAL_I2C_Master_Transmit(&hi2c1, MPU6050_ADDR, &reg_addr, 1, HAL_MAX_DELAY);
    // 读取 6 字节加速度数据（X、Y、Z 各占 2 字节）
    HAL_I2C_Master_Receive(&hi2c1, MPU6050_ADDR, raw_data, 6, HAL_MAX_DELAY);

    // 数据高字节在前，组合成 16 位有符号整数
    *ax = (int16_t)(raw_data[0] << 8 | raw_data[1]);
    *ay = (int16_t)(raw_data[2] << 8 | raw_data[3]);
    *az = (int16_t)(raw_data[4] << 8 | raw_data[5]);
}
```

### 2.3 SPI 通信协议

#### 2.3.1 原理概述

SPI（Serial Peripheral Interface）是一种高速同步串行接口，使用四根信号线：MOSI（主出从入）、MISO（主入从出）、SCK（时钟）、CS/SS（片选）。

```
         ┌─────────┐
    MOSI ├─┤         ├─ MOSI ─┐
    MISO ├─┤  Master  │        │
    SCK  ├─┤         ├─ SCK  ─┤
    CS   ├─┤         ├─ CS   ─┤
         └─────────┘          │
                       ┌───────┴───────┐
                       │    Slave      │
                       └───────────────┘
```

**特点**：
- 全双工通信（可同时发送和接收）
- 速度高（可达数十 MHz）
- 没有硬件流控和应答机制
- 一个主设备可接多个从设备（通过不同 CS 线）

#### 2.3.2 代码实现

```c
// SPI 发送接收一个字节
uint8_t SPI_Transfer_Byte(SPI_HandleTypeDef *hspi, uint8_t data)
{
    uint8_t received;
    HAL_SPI_TransmitReceive(hspi, &data, &received, 1, HAL_MAX_DELAY);
    return received;
}

// 配置 NRF24L01 无线模块为发送模式
void NRF24L01_Set_TX_Mode(void)
{
    // 拉低 CE 引脚，使模块进入待机模式
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_5, GPIO_PIN_RESET);

    // 打开接收管道 0（接收地址与发送地址相同）
    uint8_t en_rxaddr[] = {0x01};
    SPI_Write_Register(NRF24_REG_EN_RXADDR, en_rxaddr);

    // 设置自动重发次数和延迟
    uint8_t setup_retr[] = {0x1A};  // 自动重发 15 次，延迟 500us
    SPI_Write_Register(NRF24_REG_SETUP_RETR, setup_retr);

    // 拉高 CE，进入发送模式
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_5, GPIO_PIN_SET);
}
```

### 2.4 CAN 总线协议

#### 2.4.1 原理概述

CAN（Controller Area Network）是一种广泛应用于汽车和工业控制的差分总线协议，具有高可靠性和抗干扰能力。

**特点**：
- 多主从架构，所有节点都可以主动发送
- 内置 CRC 校验和错误检测
- 差分信号，抗干扰能力强
- 速度：高速 CAN（1Mbps）、低速 CAN（125kbps）
- 总线长度可达数公里（低速）

```
    CAN_H ──────────────────────────────
                                   终端
    CAN_L ──────────────────────────────   电阻 120Ω
                   [终端电阻]
```

#### 2.4.2 CAN 消息格式

```
┌────────┬────┬────┬────┬─────────────┬─────┬────┐
│ SOF   │ ID  │ RTR│ DLC│   Data (0-8B)│ CRC │ EOF│
│ 1bit  │11bit│1bit│4bit│             │15bit│2bit│
└────────┴────┴────┴────┴─────────────┴─────┴────┘
```

- **SOF**：帧起始位
- **ID**：11 位标准标识符（可扩展为 29 位）
- **RTR**：远程帧标识（数据帧/远程帧）
- **DLC**：数据长度（0-8 字节）
- **CRC**：15 位循环冗余校验

### 2.5 自定义通信协议设计

为了在串口通信中可靠传输数据，通常需要设计一个结构化的自定义协议。

#### 2.5.1 协议帧格式

推荐使用 **帧头 + 长度 + 数据 + 校验 + 帧尾** 的格式：

```
┌────────┬────────┬──────────────┬────────┬────────┐
│ 帧头   │ 长度   │    数据      │ 校验码  │  帧尾   │
│ 2字节  │ 1字节  │  N 字节      │ 1字节   │  2字节  │
└────────┴────────┴──────────────┴────────┴────────┘
 0xAA 0x55   Len    data[0..N-1]   XOR     0x0D 0x0A
```

- **帧头**（2 字节）：固定值 `0xAA 0x55`，用于标识帧起始
- **长度**（1 字节）：数据长度
- **数据**（N 字节）：有效载荷
- **校验码**（1 字节）：XOR 异或校验
- **帧尾**（2 字节）：固定值 `0x0D 0x0A`（即 `\r\n`）

#### 2.5.2 协议实现代码

```c
#include <stdint.h>
#include <string.h>

#define FRAME_HEADER1  0xAA  // 帧头第 1 字节
#define FRAME_HEADER2  0x55  // 帧头第 2 字节
#define FRAME_TAIL1    0x0D  // 帧尾第 1 字节 (\r)
#define FRAME_TAIL2    0x0A  // 帧尾第 2 字节 (\n)

#define MAX_DATA_LEN   32    // 最大数据长度

// 协议数据结构
typedef struct {
    uint8_t header[2];       // 帧头 0xAA 0x55
    uint8_t len;             // 数据长度
    uint8_t data[MAX_DATA_LEN];  // 数据区
    uint8_t checksum;        // XOR 校验码
    uint8_t tail[2];         // 帧尾 0x0D 0x0A
} __attribute__((packed)) Protocol_Frame_t;

// 计算 XOR 校验码
static uint8_t calc_checksum(uint8_t *data, uint8_t len)
{
    uint8_t checksum = 0;
    for (uint8_t i = 0; i < len; i++)
    {
        checksum ^= data[i];  // 异或运算
    }
    return checksum;
}

// 发送协议帧
void Protocol_Send_Frame(uint8_t *data, uint8_t len)
{
    uint8_t frame[MAX_DATA_LEN + 6];
    uint8_t index = 0;

    // 1. 写入帧头
    frame[index++] = FRAME_HEADER1;
    frame[index++] = FRAME_HEADER2;

    // 2. 写入长度
    frame[index++] = len;

    // 3. 写入数据
    memcpy(&frame[index], data, len);
    index += len;

    // 4. 计算并写入校验码（校验范围：长度 + 数据）
    uint8_t checksum_data[MAX_DATA_LEN + 1];
    checksum_data[0] = len;
    memcpy(&checksum_data[1], data, len);
    frame[index++] = calc_checksum(checksum_data, len + 1);

    // 5. 写入帧尾
    frame[index++] = FRAME_TAIL1;
    frame[index++] = FRAME_TAIL2;

    // 6. 通过串口发送
    HAL_UART_Transmit(&huart1, frame, index, HAL_MAX_DELAY);
}

// 接收协议帧（状态机方式）
typedef enum {
    RECV_STATE_IDLE,      // 空闲状态，等待帧头
    RECV_STATE_HEADER2,   // 已收到帧头第 1 字节，等待第 2 字节
    RECV_STATE_LEN,       // 等待长度字节
    RECV_STATE_DATA,      // 接收数据
    RECV_STATE_CHECKSUM,  // 接收校验码
    RECV_STATE_TAIL1,     // 接收帧尾第 1 字节
    RECV_STATE_TAIL2      // 接收帧尾第 2 字节
} Recv_State_t;

static Recv_State_t recv_state = RECV_STATE_IDLE;
static uint8_t recv_len = 0;       // 数据长度
static uint8_t recv_data[MAX_DATA_LEN];  // 接收数据缓冲区
static uint8_t recv_index = 0;     // 当前接收数据索引
static uint8_t recv_checksum = 0;  // 计算的校验码

// 串口接收中断处理（每收到一个字节调用一次）
void Protocol_UART_RxHandler(uint8_t byte)
{
    switch (recv_state)
    {
        case RECV_STATE_IDLE:
            if (byte == FRAME_HEADER1)
                recv_state = RECV_STATE_HEADER2;
            break;

        case RECV_STATE_HEADER2:
            if (byte == FRAME_HEADER2)
                recv_state = RECV_STATE_LEN;
            else
                recv_state = RECV_STATE_IDLE;  // 帧头错误，复位
            break;

        case RECV_STATE_LEN:
            if (byte <= MAX_DATA_LEN) {
                recv_len = byte;
                recv_index = 0;
                recv_state = RECV_STATE_DATA;
            } else {
                recv_state = RECV_STATE_IDLE;  // 长度非法，复位
            }
            break;

        case RECV_STATE_DATA:
            recv_data[recv_index++] = byte;
            if (recv_index >= recv_len)
                recv_state = RECV_STATE_CHECKSUM;
            break;

        case RECV_STATE_CHECKSUM:
            // 校验码 = len ^ data[0] ^ data[1] ^ ... ^ data[N-1]
            recv_checksum = recv_len;
            for (uint8_t i = 0; i < recv_len; i++)
                recv_checksum ^= recv_data[i];

            if (byte == recv_checksum)
                recv_state = RECV_STATE_TAIL1;
            else
                recv_state = RECV_STATE_IDLE;  // 校验失败
            break;

        case RECV_STATE_TAIL1:
            if (byte == FRAME_TAIL1)
                recv_state = RECV_STATE_TAIL2;
            else
                recv_state = RECV_STATE_IDLE;
            break;

        case RECV_STATE_TAIL2:
            if (byte == FRAME_TAIL2) {
                // 成功接收完整帧！在这里处理接收到的数据
                Protocol_Handle_Frame(recv_data, recv_len);
            }
            recv_state = RECV_STATE_IDLE;
            break;
    }
}

// 处理接收到的有效数据（在主循环或中断中调用）
void Protocol_Handle_Frame(uint8_t *data, uint8_t len)
{
    // 示例：第一个字节为命令字
    switch (data[0])
    {
        case 0x01:  // 电机控制命令
            // data[1] = 左轮速度, data[2] = 右轮速度
            Motor_Set_Speed(data[1], data[2]);
            break;
        case 0x02:  // LED 控制命令
            // data[1] = LED 状态
            LED_Set_State(data[1]);
            break;
        default:
            break;
    }
}
```

### 3.1 驱动开发基础

#### 3.1.1 GPIO 控制

```c
// GPIO 初始化：配置 PB0 为推挽输出（控制 LED）
void GPIO_Init(void)
{
    __HAL_RCC_GPIOB_CLK_ENABLE();  // 使能 GPIOB 时钟

    GPIO_InitTypeDef GPIO_InitStruct = {0};
    GPIO_InitStruct.Pin = GPIO_PIN_0;        // PB0 引脚
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;  // 推挽输出
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW;  // 低速
    HAL_GPIO_Init(GPIOB, &GPIO_InitStruct);
}

// 点亮 LED
void LED_On(void)
{
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_SET);
}

// 熄灭 LED
void LED_Off(void)
{
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_RESET);
}

// 翻转 LED 状态
void LED_Toggle(void)
{
    HAL_GPIO_TogglePin(GPIOB, GPIO_PIN_0);
}
```

#### 3.1.2 PWM 输出（电机调速）

```c
// 配置 TIM3 CH2 (PA7) 输出 PWM，控制电机速度
void PWM_Init(void)
{
    __HAL_RCC_TIM3_CLK_ENABLE();  // 使能 TIM3 时钟

    TIM_OC_InitTypeDef oc_config = {0};
    oc_config.OCMode = TIM_OCMODE_PWM1;      // PWM 模式 1
    oc_config.Pulse = 0;                     // 初始占空比为 0
```

### 3.2 电机控制

#### 3.2.1 PID 控制算法

PID（比例-积分-微分）控制是最经典的闭环控制算法，广泛应用于电机速度控制。

```
     ┌──────────┐
r ──►│          │     e      ┌───────┐     u     ┌────────┐    y
     │   PID    ├────────►   │       ├────────►  │  电机   ├───┬──►
     │          │            │ Plant │           │         │   │
     └──────────┘            └───────┘           └────────┘   │
        ▲  -                     │                         │   │
        │  │                     │                         │   │
        └──┘                     │                         └───┘
           │                     │                            │
           │                     ▼                            │
           │                传感器反馈                         │
           │                                               反馈
           │                                               回路
           │
           设定值 r: 目标速度
           误差 e:  r - y（设定值与实际值的差）
           控制器输出 u: PWM 占空比
           Plant: 电机 + 减速器 + 编码器
```

**PID 公式**：

```
u(t) = Kp * e(t) + Ki * ∫e(t)dt + Kd * de(t)/dt
```

离散化（数字实现）：

```
P = Kp * error
I = I + Ki * error * dt
D = Kd * (error - prev_error) / dt
output = P + I + D
```

#### 3.2.2 PID 速度控制代码

```c
// PID 参数结构体
typedef struct {
    float target;       // 目标值（设定速度）
    float current;      // 当前值（实际速度）
    float error;        // 误差 = target - current
    float prev_error;   // 上一次误差
    float integral;     // 积分项（累积误差）
    float derivative;   // 微分项
    float output;       // PID 输出
    float Kp;           // 比例系数
    float Ki;           // 积分系数
    float Kd;           // 微分系数
    float dt;           // 控制周期（秒）
    float max_output;   // 输出上限
    float min_output;   // 输出下限
} PID_Controller_t;

// PID 初始化
void PID_Init(PID_Controller_t *pid, float Kp, float Ki, float Kd,
              float max_out, float min_out, float dt)
{
    memset(pid, 0, sizeof(PID_Controller_t));
    pid->Kp = Kp;
    pid->Ki = Ki;
    pid->Kd = Kd;
    pid->max_output = max_out;
    pid->min_output = min_out;
    pid->dt = dt;
}

// PID 计算（每控制周期调用一次）
void PID_Calculate(PID_Controller_t *pid)
{
    // 计算误差
    pid->error = pid->target - pid->current;

    // 比例项
    float P = pid->Kp * pid->error;

    // 积分项（带积分限幅，防止积分饱和）
    pid->integral += pid->Ki * pid->error * pid->dt;
    // 积分限幅：限制累积误差，避免输出过大
    if (pid->integral > pid->max_output)  pid->integral = pid->max_output;
    if (pid->integral < pid->min_output)  pid->integral = pid->min_output;

    // 微分项（抑制误差变化过快）
    float D = pid->Kd * (pid->error - pid->prev_error) / pid->dt;

    // 总输出
    pid->output = P + pid->integral + D;

    // 输出限幅
    if (pid->output > pid->max_output)  pid->output = pid->max_output;
    if (pid->output < pid->min_output)  pid->output = pid->min_output;

    // 保存误差值，供下次微分计算使用
    pid->prev_error = pid->error;
}

// 电机 PID 速度控制器
typedef struct {
    PID_Controller_t pid;    // PID 控制器
    int16_t encoder_prev;   // 上次编码器值
    int16_t speed_rpm;      // 当前速度（RPM）
} Motor_Controller_t;

// 电机控制器初始化
void Motor_PID_Init(Motor_Controller_t *motor, float Kp, float Ki, float Kd)
{
    // 电机 PWM 满占空比对应最大速度约 200 RPM
    PID_Init(&motor->pid, Kp, Ki, Kd, 1000, -1000, 0.01f);  // 10ms 控制周期
}

// 更新电机速度测量值（每 10ms 从编码器读取）
void Motor_Update_Speed(Motor_Controller_t *motor, int16_t encoder_count)
{
    // 计算编码器变化量（处理编码器溢出）
    int16_t delta = encoder_count - motor->encoder_prev;
    if (delta > 32768)  delta -= 65536;   // 负向溢出
    if (delta < -32768)  delta += 65536;  // 正向溢出

    // 将编码器变化量转换为 RPM
    // 假设：电机每转产生 390 个编码器脉冲，减速比 30:1
    // 测量周期 10ms = 0.01s
    // 速度(RPM) = delta / 390 / 30 * (60 / 0.01)
    motor->speed_rpm = (int16_t)((float)delta * 60 * 100 / 390 / 30);
    motor->encoder_prev = encoder_count;

    // 更新 PID 当前值
    motor->pid.current = (float)motor->speed_rpm;
}

// 执行电机 PID 控制（每 10ms 调用一次）
int16_t Motor_PID_Control(Motor_Controller_t *motor)
{
    PID_Calculate(&motor->pid);
    // 将 PID 输出（-1000~1000）转换为 PWM 占空比（0~1000）
    return (int16_t)(motor->pid.output + 1000) / 2;
}

// 实例化左轮和右轮电机控制器
Motor_Controller_t left_motor, right_motor;
```

```c
// ============ 超声波传感器（HC-SR04）============
// HC-SR04 使用 TRIG 触发测距，ECHO 接收回响
// 距离 = 高电平时间 * 声速 / 2

#define TRIG_PIN  GPIO_PIN_6
#define ECHO_PIN  GPIO_PIN_7
#define SOUND_SPEED 0.034f  // 声速 340m/s

// 发送超声波触发信号
void Ultrasonic_Trigger(void)
{
    HAL_GPIO_WritePin(GPIOB, TRIG_PIN, GPIO_PIN_SET);
    HAL_Delay_US(10);   // 发送 10us 高电平
    HAL_GPIO_WritePin(GPIOB, TRIG_PIN, GPIO_PIN_RESET);
}

// 读取超声波距离（厘米）
float Ultrasonic_Read(void)
{
    uint32_t timeout;
    // 等待 ECHO 高电平（开始）
    timeout = HAL_GetTick() + 100;
    while (HAL_GPIO_ReadPin(GPIOB, ECHO_PIN) == GPIO_PIN_RESET) {
        if (HAL_GetTick() > timeout) return -1;  // 超时
    }

    uint32_t start_tick = HAL_GetTick();  // 记录高电平开始时刻

    // 等待 ECHO 低电平（结束）
    timeout = HAL_GetTick() + 100;
    while (HAL_GPIO_ReadPin(GPIOB, ECHO_PIN) == GPIO_PIN_SET) {
        if (HAL_GetTick() > timeout) return -1;  // 超时
    }

    uint32_t duration = HAL_GetTick() - start_tick;  // 高电平持续时间（ms）
    // 转换为距离：时间(us) * 声速(cm/us) / 2
    return (float)duration * SOUND_SPEED * 50.0f;  // 近似计算
}

// ============ 红外巡线传感器 =============
// 红外传感器输出数字信号，检测地面黑白线

// 定时器回调（每 10ms 执行一次，采集传感器数据）

        sensor_data.ir_sensors = IR_Sensor_Read();
    }
}
```

```c
#define UL_TRIG_PIN   GPIO_PIN_5
#define UL_ECHO_PORT  GPIOB
#define UL_ECHO_PIN   GPIO_PIN_4

void Ultrasonic_Init(void)
{

    GPIO_InitTypeDef GPIO_InitStruct = {0};
    GPIO_InitStruct.Pin = UL_TRIG_PIN;
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_HIGH;
    HAL_GPIO_Init(UL_TRIG_PORT, &GPIO_InitStruct);

    // ECHO 配置为输入
    GPIO_InitStruct.Pin = UL_ECHO_PIN;

    GPIO_InitStruct.Pull = GPIO_NOPULL;
    HAL_GPIO_Init(UL_ECHO_PORT, &GPIO_InitStruct);

// 测量距离（阻塞式，最长约 60ms）
float Ultrasonic_Get_Distance(void)
{
    uint32_t timeout;

    // 1. 发送 10us 触发脉冲
    HAL_GPIO_WritePin(UL_TRIG_PORT, UL_TRIG_PIN, GPIO_PIN_SET);
    for(volatile int i = 0; i < 80; i++);  // 约 10us
    HAL_GPIO_WritePin(UL_TRIG_PORT, UL_TRIG_PIN, GPIO_PIN_RESET);

    // 2. 等待 ECHO 高电平开始
    timeout = 10000;
    while (HAL_GPIO_ReadPin(UL_ECHO_PORT, UL_ECHO_PIN) == GPIO_PIN_RESET) {
        if (--timeout == 0) return -1;  // 超时返回 -1
    }

    // 3. 记录高电平开始时刻（使用 DWT 计数器，精度更高）
    uint32_t tick_start = DWT->CYCCNT;

    // 4. 等待 ECHO 低电平结束
    timeout = 10000;
    while (HAL_GPIO_ReadPin(UL_ECHO_PORT, UL_ECHO_PIN) == GPIO_PIN_SET) {
        if (--timeout == 0) return -1;
    }
    uint32_t tick_end = DWT->CYCCNT;

    // 5. 计算距离
    // 声速约 340m/s = 0.034cm/us
    // 距离 = 时间(us) * 0.034 / 2（往返）
    float time_us = (float)(tick_end - tick_start) / (SystemCoreClock / 1000000UL);
    return time_us * 0.034f / 2.0f;
}

    if (HAL_GPIO_ReadPin(IR_SENSOR_PORT, GPIO_PIN_5) == GPIO_PIN_RESET) val |= 0x04;  // 左
    if (HAL_GPIO_ReadPin(IR_SENSOR_PORT, GPIO_PIN_6) == GPIO_PIN_RESET) val |= 0x02;  // 中
    if (HAL_GPIO_ReadPin(IR_SENSOR_PORT, GPIO_PIN_7) == GPIO_PIN_RESET) val |= 0x01;  // 右
    return val;
}
```

### 4.5 串口协议实现

```c
// ============== FishBot 串口通信协议 =============
// 上位机（ROS2）<--USB转串口--> 下位机（STM32）

// 协议格式（与第 2.5 节相同）
// 帧头(2B) + 长度(1B) + 数据(NB) + 校验(1B) + 帧尾(2B)
// 帧头：0xAA 0x55
// 帧尾：0x0D 0x0A

// ---------- 发送给上位机的数据帧 ----------

// 主动上报：传感器数据（每 50ms 上报一次）
void Report_Sensor_Data(void)
{
    uint8_t frame_data[16];
    uint8_t idx = 0;

    // 帧类型 = 0x01（传感器数据）
    frame_data[idx++] = 0x01;

    // 电池电压（2字节，高字节在前）
    frame_data[idx++] = (uint8_t)(sensor_data.battery_voltage >> 8);
    frame_data[idx++] = (uint8_t)(sensor_data.battery_voltage & 0xFF);

    // 超声波距离（2字节，厘米，×100 转成定点数）
    uint16_t dist = (uint16_t)(sensor_data.ultrasonic_distance * 100);
    frame_data[idx++] = (uint8_t)(dist >> 8);
    frame_data[idx++] = (uint8_t)(dist & 0xFF);

    // 左轮速度（2字节，有符号 RPM）
    frame_data[idx++] = (uint8_t)(sensor_data.left_speed >> 8);
    frame_data[idx++] = (uint8_t)(sensor_data.left_speed & 0xFF);

    // 右轮速度（2字节，有符号 RPM）
    frame_data[idx++] = (uint8_t)(sensor_data.right_speed >> 8);
    frame_data[idx++] = (uint8_t)(sensor_data.right_speed & 0xFF);

    // 红外传感器状态
    frame_data[idx++] = sensor_data.ir_sensors;

    // 填充预留字节
    frame_data[idx++] = 0x00;
    frame_data[idx++] = 0x00;

    Protocol_Send_Frame(frame_data, idx);
}

// ---------- 接收上位机指令 ----------

// 指令类型
typedef enum {
    CMD_MOTOR = 0x10,   // 电机控制指令
    CMD_LED   = 0x11,   // LED 控制指令
    CMD_PID   = 0x12,   // PID 参数调整指令
} Command_Type_t;

// 电机控制指令格式：
// [0x10][左轮速度(1B)][右轮速度(1B)]
// 速度范围：-100 ~ +100，0=停止

void Protocol_Handle_Frame(uint8_t *data, uint8_t len)
{
    if (len < 2) return;

    switch (data[0])
    {
        case CMD_MOTOR:  // 电机控制
        {
            // 直接控制模式（速度值 -100~100）
            int16_t left_speed = (int8_t)data[1];   // 有符号速度
            int16_t right_speed = (int8_t)data[2];

            // 转换为 PWM 占空比（0~1000）
            Motor_Set_Left(left_speed * 10);
            Motor_Set_Right(right_speed * 10);
            break;
        }

        case CMD_PID:  // PID 参数调整
        {
            // data[1]=Kp_x10, data[2]=Ki_x100, data[3]=Kd_x10
            float Kp = data[1] / 10.0f;
            float Ki = data[2] / 100.0f;
            float Kd = data[3] / 10.0f;
            left_motor.pid.Kp = Kp;
            left_motor.pid.Ki = Ki;
            left_motor.pid.Kd = Kd;
            right_motor.pid.Kp = Kp;
            right_motor.pid.Ki = Ki;
            right_motor.pid.Kd = Kd;
            break;
        }

        case CMD_LED:  // LED 控制
        {
            if (data[1] == 0)
                LED_Off();
            else
                LED_On();
            break;
        }
    }
}

// 串口接收中断服务函数（放在 stm32f1xx_it.c 的 USART1_IRQHandler 中）
void USART1_IRQHandler(void)
{
    if (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_RXNE))
    {
        uint8_t byte = (uint8_t)(USART1->DR);  // 读取数据
        Protocol_UART_RxHandler(byte);          // 交给协议状态机处理
    }
    HAL_UART_IRQHandler(&huart1);
}
```

### 6.2 话题转发节点

#### 6.2.1 串口数据打包/解包

```python
# ros2_serial_node.py - ROS2 与 STM32 串口通信节点
import rclpy
from rclpy.node import Node
from rclpy.parameter import Parameter
from sensor_msgs.msg import JointState  # 电机速度话题
from geometry_msgs.msg import Twist    # /cmd_vel 话题
import serial
import threading

class Ros2SerialNode(Node):
    def __init__(self):
        super().__init__('ros2_serial_node')

        # 串口参数
        self.declare_parameter('serial_port', '/dev/ttyUSB0')
        self.declare_parameter('baudrate', 115200)
        port = self.get_parameter('serial_port').value
        baud = self.get_parameter('baudrate').value

        # 打开串口
        try:
            self.ser = serial.Serial(port, baud, timeout=0.1)
            self.get_logger().info(f'串口已打开: {port} @ {baud}')
        except Exception as e:
            self.get_logger().error(f'串口打开失败: {e}')
            self.ser = None

        # 接收缓冲区
        self.rx_buffer = bytearray()

        # 创建发布者：发布从下位机接收的传感器数据
        self.sensor_pub = self.create_publisher(JointState, '/fishbot/sensors', 10)

        # 创建订阅者：订阅 /cmd_vel，转换为电机速度
        self.cmd_vel_sub = self.create_subscription(
            Twist, '/cmd_vel', self.cmd_vel_callback, 10
        )

        # 定时器：每 20ms 处理一次串口接收
        self.timer = self.create_timer(0.02, self.timer_callback)

        # 接收线程
        if self.ser:
            self.rx_thread = threading.Thread(target=self.serial_read_thread)
            self.rx_thread.daemon = True
            self.rx_thread.start()

    def cmd_vel_callback(self, msg: Twist):
        """将 Twist 消息转换为电机速度并发送"""
        # 差速模型：线速度 + 角速度 → 左右轮速度
        wheel_separation = 0.2  # 轮间距 0.2 米
        left_speed = (msg.linear.x - msg.angular.z * wheel_separation / 2) * 10  # 转成约 -100~100
        right_speed = (msg.linear.x + msg.angular.z * wheel_separation / 2) * 10

        # 发送电机控制指令
        self.send_motor_cmd(int(left_speed), int(right_speed))

    def send_motor_cmd(self, left: int, right: int):
        """发送电机控制命令"""
        if not self.ser:
            return

        # 协议帧：AA 55 10 len data... checksum 0D 0A
        cmd = bytes([0x10])  # 电机控制命令字
        left_byte = bytes([max(-100, min(100, left))])  # 限幅 -100~100
        right_byte = bytes([max(-100, min(100, right))])
        data = cmd + left_byte + right_byte

        frame = self.build_frame(data)
        self.ser.write(frame)

    def build_frame(self, data: bytes) -> bytes:
        """构建协议帧：帧头+长度+数据+XOR校验+帧尾"""
        frame = bytearray()
        frame += bytes([0xAA, 0x55])          # 帧头
        frame.append(len(data))               # 长度
        frame += data                          # 数据

        # XOR 校验
        checksum = len(data)
        for b in data:
            checksum ^= b
        frame.append(checksum)

        frame += bytes([0x0D, 0x0A])          # 帧尾
        return bytes(frame)

    def serial_read_thread(self):
        """后台线程持续读取串口数据"""
        while rclpy.ok() and self.ser and self.ser.is_open:
            if self.ser.in_waiting:
                data = self.ser.read(self.ser.in_waiting)
                self.rx_buffer.extend(data)
            threading.Event().wait(0.001)

    def timer_callback(self):
        """定时处理接收缓冲区"""
        if not self.ser:
            return

        # 查找并解析完整帧
        while len(self.rx_buffer) >= 6:
            # 查找帧头 0xAA 0x55
            try:
                idx = self.rx_buffer.index(0xAA)
                if idx > 0:
                    # 丢弃帧头前的无效数据
                    del self.rx_buffer[:idx]
                    continue

                if len(self.rx_buffer) < 6:
                    break

                # 检查帧头第 2 字节
                if self.rx_buffer[1] != 0x55:
                    del self.rx_buffer[0]
                    continue

                # 读取长度
                pkg_len = self.rx_buffer[2]
                total_len = 4 + pkg_len + 2  # 帧头2 + 长度1 + 数据N + 校验1 + 帧尾2

                if len(self.rx_buffer) < total_len:
                    break  # 数据不完整，等待下一批

                # 提取完整帧
                frame = self.rx_buffer[:total_len]
                del self.rx_buffer[:total_len]

                # 解析数据
                self.parse_frame(frame)

            except ValueError:
                del self.rx_buffer[0]
                continue

    def parse_frame(self, frame: bytes):
        """解析传感器数据帧"""
        # 简单的校验检查
        checksum_data = frame[2:2+frame[2]]
        calc_checksum = frame[2]
        for b in checksum_data:
            calc_checksum ^= b

        if calc_checksum != frame[3 + frame[2]]:
            self.get_logger().warn('校验失败')
            return

        data = frame[3:3+frame[2]]  # 去掉帧头(2) + 长度(1) + 校验(1)

        if data[0] == 0x01:  # 传感器数据帧
            voltage = (data[1] << 8) | data[2]
            dist = ((data[3] << 8) | data[4]) / 100.0  # 厘米
            left_rpm = int.from_bytes(data[5:7], byteorder='big', signed=True)
            right_rpm = int.from_bytes(data[7:9], byteorder='big', signed=True)
            ir_state = data[9]

            # 发布 JointState 消息
            msg = JointState()
            msg.header.stamp = self.get_clock().now().to_msg()
            msg.name = ['left_wheel_joint', 'right_wheel_joint']
            msg.velocity = [left_rpm * 2 * 3.14159 / 60.0, right_rpm * 2 * 3.14159 / 60.0]
            self.sensor_pub.publish(msg)

def main(args=None):
    rclpy.init(args=args)
    node = Ros2SerialNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        if hasattr(node, 'ser') and node.ser:
            node.ser.close()
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 8.3 简答题答案

**11. 答案要点：**
- **上位机**：运行复杂算法（如 SLAM、路径规划），通常使用高性能处理器（如 ARM Cortex-A、x86），运行操作系统（Linux），编程语言为 Python/C++，计算能力强但实时性一般
- **下位机**：直接控制执行器和传感器，使用 MCU（如 STM32），通常裸机或轻量 RTOS，编程语言为 C/C++，计算能力有限但实时性好
- **举例**：上位机在机器人中负责用激光雷达数据进行地图构建（SLAM），下位机负责根据电机编码器反馈实时调整 PWM 使电机转速稳定在设定值

**12. 答案要点：**  
直接发送原始数据存在以下问题：
- **无法确定数据边界**：接收方不知道一帧何时开始、何时结束
- **数据中的特殊字节可能造成误判**：如数据中恰好出现与帧定义冲突的字节
- **无校验机制**：无法检测传输错误  
自定义协议通过帧头固定标识（0xAA 0x55）、长度字段明确数据范围、XOR/CRC 校验检测错误、帧尾标识（0x0D 0x0A）确认帧完整，有效解决了以上问题。

**13. 答案要点：**
- PID 根据设定速度与实际速度的误差计算控制量：  
  `output = Kp×error + Ki×∫error·dt + Kd×derror/dt`
- **Kp（比例）**：根据当前误差大小产生控制作用，误差越大控制越强，响应快但有稳态误差
- **Ki（积分）**：累积历史误差，消除稳态误差，但可能引起超调和振荡
- **Kd（微分）**：根据误差变化趋势提前控制，抑制超调和振荡，提高系统稳定性

**14. 答案要点：**  
串口数据是**字节流**，没有消息边界，状态机用于从连续字节流中解析出完整的数据帧：
1. `RECV_WAIT_HEADER1`：等待 0xAA
2. `RECV_WAIT_HEADER2`：已收到 0xAA，等待 0x55（确认帧头正确性）
3. `
3. `RECV_WAIT_LEN`：读取长度字节，确定后续数据量
4. `RECV_WAIT_DATA`：按长度接收数据字节并存入缓冲区
5. `RECV_WAIT_CHECKSUM`：接收校验码并验证
6. `RECV_WAIT_TAIL`：确认帧尾 0x0D 0x0A，验证通过后提交完整帧处理

状态机保证了即使数据不连续到来、或数据流中出现噪声字节，也能正确找到并解析每一帧。

### 8.4 编程题答案

**15. 答案：**

```c
typedef enum {
    RECV_WAIT_HEADER1,
    RECV_WAIT_HEADER2,
    RECV_WAIT_LEN,
    RECV_WAIT_DATA,
    RECV_WAIT_CHECKSUM,
    RECV_WAIT_TAIL
} RecvState;

void on_uart_byte(uint8_t byte) {
    static RecvState state = RECV_WAIT_HEADER1;
    static uint8_t pkg_len = 0;
    static uint8_t data_buf[32];
    static uint8_t data_idx = 0;

    switch (state) {
        case RECV_WAIT_HEADER1:
            if (byte == 0xAA)
                state = RECV_WAIT_HEADER2;
            break;

        case RECV_WAIT_HEADER2:
            if (byte == 0x55)
                state = RECV_WAIT_LEN;   // ✅ 帧头匹配，进入长度接收状态
            else
                state = RECV_WAIT_HEADER1;  // 帧头不匹配，重新等待
            break;

        case RECV_WAIT_LEN:
            if (byte <= 32) {           // 限制最大长度，防止溢出
                pkg_len = byte;
                data_idx = 0;
                state = RECV_WAIT_DATA;
            } else {
                state = RECV_WAIT_HEADER1;  // 长度非法，复位
            }
            break;

        case RECV_WAIT_DATA:
            data_buf[data_idx++] = byte;
            if (data_idx >= pkg_len)
                state = RECV_WAIT_CHECKSUM;
            break;

        case RECV_WAIT_CHECKSUM:
            // 校验码 = len XOR data[0] XOR data[1] ... XOR data[N-1]
            {
                uint8_t expected = pkg_len;
                for (uint8_t i = 0; i < pkg_len; i++)
                    expected ^= data_buf[i];

                if (byte == expected)
                    state = RECV_WAIT_TAIL;
                else
                    state = RECV_WAIT_HEADER1;  // 校验失败，复位
            }
            break;

        case RECV_WAIT_TAIL:
            if (byte == 0x0D)
                state = RECV_WAIT_TAIL;   // 收到 \r，等待 \n
            else if (byte == 0x0A) {
                // ✅ 完整帧接收成功！
                // 在这里调用 frame_handler(data_buf, pkg_len)
                on_frame_received(data_buf, pkg_len);
                state = RECV_WAIT_HEADER1;
            } else {
                state = RECV_WAIT_HEADER1;  // 帧尾错误
            }
            break;
    }
}
```

> **关键点**：状态机确保即使字节流中出现噪声或数据丢失，也能自动重新同步到正确的帧头位置，而不会死锁或处理错误的帧。
