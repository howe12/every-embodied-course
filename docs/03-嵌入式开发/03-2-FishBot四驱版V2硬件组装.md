## 1. FishBot V2 硬件概述

### 1.1 产品定位

FishBot 四驱版 V2 是一款基于 STM32 主控的**四轮差速机器人平台**，专为 ROS2 机器人开发设计。相比上一代 V1 版本，V2 在以下方面进行了升级：

- **四驱系统**：四轮独立驱动，通过差速实现转向，结构更稳定
- **内置黑盒**：板载数据记录模块，方便调试与数据回放
- **增强供电**：双电池供电设计，动力与逻辑分离
- **丰富接口**：预留了超声波、红外、编码器、IMU 等传感器接口
- **ROS2 原生支持**：固件内置 MicroROS，可直接与 ROS2 通信

```
┌─────────────────────────────────────────┐
│           FishBot V2 四驱版              │
│                                         │
│    [前轮]              [前轮]           │
│      ○                   ○              │
│         ┌───────────────┐               │
│         │   主控板      │               │
│         │  STM32F103    │               │
│         │ + 黑盒模块    │               │
│         └───────────────┘               │
│      ○                   ○              │
│    [后轮]              [后轮]           │
│                                         │
│  尺寸: 200×150×80mm  重量: 1.2kg       │
└─────────────────────────────────────────┘
```

### 1.2 主要特点

| 特性 | 说明 |
|------|------|
| 驱动方式 | 四轮差速驱动，独立编码器反馈 |
| 主控芯片 | STM32F103RCT6，72MHz，Cortex-M3 |
| 通信方式 | USB/UART，支持 MicroROS |
| 供电方案 | 7.4V 锂电池（动力）+ 5V LDO（逻辑） |
| 电机类型 | 直流有刷减速电机 + AB 相编码器 |
| 预留接口 | 超声波×4、红外测障×3、IMU(I2C)、遥控器PPM |

### 1.3 应用场景

- ROS2 机器人学习与科研项目
- 室内自主导航算法验证
- 多传感器融合算法开发
- 机器人竞赛原型开发

### 2.2 电机参数详解

FishBot V2 采用的直流有刷减速电机是关键执行部件，其参数直接影响运动性能：

| 参数 | 数值 | 说明 |
|------|------|------|
| 额定电压 | 12V DC | 推荐工作电压 |
| 空载转速 | 200 RPM | 12V 时空载转速 |
| 额定转速 | 150 RPM | 额定负载下转速 |
| 额定扭矩 | 0.3 N·m | 持续运行最大扭矩 |
| 减速比 | 1:30 | 电机轴到输出轴减速比 |
| 编码器类型 | AB 相磁编码器 | 每转 12 个脉冲 |
| 重量 | 约 85g | 单个电机重量 |

**编码器脉冲计算：**

```
输出轴每转脉冲数 = 电机自带脉冲 × 减速比 = 12 × 30 = 360 P/R
四倍频后 = 360 × 4 = 1440 P/R（软件四倍频可提高分辨率）
```

## 3. 机械结构组装

### 3.1 组装流程概览

```
组装顺序：
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 第一步  │ →  │ 第二步  │ →  │ 第三步  │ →  │ 第四步  │
│ 底盘   │    │ 电机    │    │ 轮子    │    │ 万向轮  │
│ 检查   │    │ 安装    │    │ 安装    │    │ 安装    │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↓
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 第五步  │ →  │ 第六步  │ →  │ 第七步  │
│ 主控板  │    │ 电池    │    │ 传感器  │
│ 安装    │    │ 固定    │    │ 安装    │
└─────────┘    └─────────┘    └─────────┘
```

### 3.2 第一步：底盘检查与准备

组装前先检查底盘外观和尺寸：

1. **外观检查**：底盘表面无划伤、无变形，铝材切口平整
2. **孔位检查**：对照图纸，逐一核对安装孔位是否正确
   - 电机安装孔：4组，每组2孔，间距 30mm
   - 主控板安装孔：4个，M3 螺纹孔
   - 电池仓安装孔：2组长圆孔
   - 传感器安装孔：前部超声波×4，侧部红外×3
3. **清理工作**：用锉刀轻轻打磨切口毛刺，用酒精擦拭表面油污

### 3.3 第二步：电机安装

电机安装是整个机械结构的关键步骤，安装质量直接影响机器人运动性能。

#### 安装步骤

1. **准备电机支架**：将铝合金角码与底盘预留孔位对齐
2. **放置电机**：将电机放入支架凹槽，确保电机轴朝向底盘外侧
3. **固定螺丝**：
   - 使用 M3×8mm 平头螺丝从支架侧面穿过
   - 配合 M3 尼龙锁紧螺母，扭力适中（约 0.3 N·m）
   - **注意**：不要过紧，会压伤电机外壳
4. **检查电机轴**：转动电机轴，确认无卡滞、无异响

#### 电机安装位置

```
底盘俯视图：
         前方
    ┌─────────────────┐
    │   [超声波支架]   │
    │                 │
    │  [M1]     [M2]  │  ← M1/M2 为前轮电机
    │                 │
    │  [M3]     [M4]  │  ← M3/M4 为后轮电机
    │                 │
    │   [电池仓]      │
    └─────────────────┘
         后方

注意：M1/M2（前方）电机轴朝向车头前方
      M3/M4（后方）电机轴朝向车头后方
      两侧电机轴朝向相反，形成镜像安装
```

#### 常见问题与解决

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 电机轴转动有明显阻力 | 支架与电机外壳干涉 | 添加 M3 垫片调整间隙 |
| 螺丝拧紧后电机异响 | 螺丝压到电机线圈 | 放松螺丝，移位重装 |
| 电机轴偏心 | 轴承损坏 | 更换电机 |

### 3.4 第三步：轮子安装

轮子通过联轴器与电机轴连接：

1. **清理轴面**：用干布擦拭电机轴，去除油污
2. **安装联轴器**：
   - 联轴器内孔尺寸需与电机轴匹配（通常为 6mm）
   - 使用内六角螺丝顶紧（拧紧力矩约 0.5 N·m）
   - **重要**：必须使用乐泰 243 螺纹胶，防止松动
3. **安装轮胎**：
   - 将橡胶轮胎压在轮毂上，用手按压嵌入
   - 确认轮胎安装牢固，无歪斜

> ⚠️ **警告**：轮胎未安装牢固时禁止通电测试，否则轮胎可能飞脱伤人！

### 3.5 第四步：万向轮安装

万向轮安装在底盘后部两侧，用于支撑和辅助转向：

1. 在底盘后部两侧标记孔位
2. 使用 M3×15mm 螺丝 + 螺母固定万向轮支架
3. 调整万向轮高度，确保底盘水平
4. 万向轮应能灵活转动 360°，无卡滞

**水平度检查方法**：
- 将机器人放在平面上
- 用水平尺检查底盘前后、左右方向的水平度
- 调整万向轮高度，使底盘达到水平状态

### 3.6 第五步：主控板安装

1. 在主控板背面贴上 3M 导热双面胶
2. 对准底盘上的 M3 螺纹孔，轻轻按压粘贴
3. 从主板上方用 M3×6mm 螺丝固定四角
4. 检查主板与其他金属件之间是否有短路风险

### 3.7 第六步：电池固定

1. 将电池放入电池仓，使用魔术贴带固定
2. 连接电池 XT30 接口到主控板 J7 电源接口
3. **注意极性**：红线接正极（VBAT+），黑线接负极（VBAT-）
4. 整理线缆，避免电池压在接插件上

### 3.8 第七步：传感器安装

#### 超声波传感器安装（4个）

| 位置 | 角度 | 说明 |
|------|------|------|
| 前左 | 45° | 检测左前方障碍 |
| 前中 | 0° | 检测正前方障碍 |
| 前右 | -45° | 检测右前方障碍 |
| 后中 | 180° | 检测后方障碍 |

安装步骤：
1. 将 HC-SR04 模块插入超声波支架
2. 使用 M2×8mm 螺丝固定
3. 连接 4Pin 排线到主控板 J3 接口

#### 红外测障模块安装（3个）

| 位置 | 说明 |
|------|------|
| 左侧 | 检测左方近距离障碍 |
| 正前方 | 检测正前方近距离障碍 |
| 右侧 | 检测右方近距离障碍 |

安装步骤：
1. 使用 M2 螺丝将 TCRT5000 固定在底盘侧面
2. 调整传感器角度，使其朝向车身外侧
3. 连接 3Pin 排线到主控板 J4 接口

### 4.2 电源系统详解

FishBot V2 采用**双电源分区供电**方案，这是保证系统稳定运行的关键设计：

#### 供电架构

| 区域 | 电源 | 说明 |
|------|------|------|
| 电机驱动 | 7.4V 直通 | 锂电池直接供电，电流可达 5A |
| 逻辑电路 | 5V LDO | STM32、编码器、传感器供电 |
| 通信模块 | 5V/3.3V | 蓝牙/WiFi 模块供电 |

#### 电源连接步骤

**步骤 1：检查电池电量**

```python
# 使用万用表测量电池电压，判断电量状态
# 2S锂电池满电电压：8.4V (每节4.2V)
# 2S锂电池截止电压：6.4V (每节3.2V)

# 电压与电量对照表
voltage_table = {
    8.4: "100% - 满电",
    8.0: "80%  - 良好",
    7.6: "50%  - 需充电",
    7.2: "20%  - 即将耗尽",
    6.4: "0%   - 必须充电"
}
```

**步骤 2：连接电池到主控板**

1. 将 XT30 插头插入主控板 J7 接口
2. 确认插头方向正确（防反插设计）
3. 轻轻按压，直到听到"咔哒"声

**步骤 3：测量各路电源输出**

| 测试点 | 正常电压 | 说明 |
|--------|---------|------|
| 主控板 5V 引脚 | 4.9～5.1V | 逻辑电源 |
| 主控板 3.3V 引脚 | 3.3V | 芯片内核电压 |
| 传感器供电引脚 | 5.0V | 超声波等传感器 |

> ⚠️ **重要**：通电前必须用万用表确认电压！电压过高会烧毁芯片！

### 4.3 电机驱动板连接

TB6612FNG 是一款双通道 H 桥电机驱动芯片，每块芯片可驱动 2 个电机：

#### TB6612FNG 引脚定义

| 引脚 | 功能 | 说明 |
|------|------|------|
| VMOT | 电机电源 | 接 7.4V |
| GND | 电源地 | 共地 |
| AIN1/AIN2 | 电机A方向 | STM32 GPIO 输出 |
| PWMA | 电机A调速 | STM32 PWM 输出 |
| AOUT1/AOUT2 | 电机A输出 | 接电机红线/黑线 |
| BIN1/BIN2 | 电机B方向 | STM32 GPIO 输出 |
| PWMB | 电机B调速 | STM32 PWM 输出 |
| BOUT1/BOUT2 | 电机B输出 | 接电机红线/黑线 |
| STBY | 使能脚 | 高电平使能，接 3.3V |

#### 驱动原理

```
TB6612FNG H桥工作原理（单通道）：

正转：AIN1=H, AIN2=L  →  AOUT1=VMOT, AOUT2=GND
反转：AIN1=L, AIN2=H  →  AOUT1=GND, AOUT2=VMOT
停止：AIN1=L, AIN2=L  →  AOUT1=GND, AOUT2=GND（短路制动）
```

#### 连接表

| 主控板 GPIO | TB6612FNG | 功能 |
|-------------|-----------|------|
| PA0 | PWMA | 电机A速度（PWM） |
| PA1 | AIN1 | 电机A方向1 |
| PA2 | AIN2 | 电机A方向2 |
| PB0 | PWMB | 电机B速度（PWM） |
| PB1 | BIN1 | 电机B方向1 |
| PB2 | BIN2 | 电机B方向2 |
| PC13 | STBY | 使能（高电平） |

#### 连接步骤

1. **焊接排针**：在 TB6612FNG 板上焊接 2.54mm 间距排针
2. **连接信号线**：用杜邦线连接主控板与驱动板各信号脚
3. **连接电源线**：
   - VMOT 接 7.4V（电池正极）
   - GND 接 GND（电池负极）
4. **连接电机线**：
   - AOUT 接 M1/M2 电机（两侧前轮）
   - BOUT 接 M3/M4 电机（两侧后轮）

### 4.4 编码器接线

编码器采用 AB 相输出，需要连接到主控板的定时器编码器接口：

#### 编码器接口定义

| 编码器线色 | 功能 | 说明 |
|-----------|------|------|

#### 超声波模块（HC-SR04）

| HC-SR04 | 主控板 | 说明 |
|---------|--------|------|

| GND | GND | 地线 |

```c
/**
 * HC-SR04 超声波测距原理
 * 
 * 1. MCU 向 Trig 脚发送至少 10us 的高电平脉冲
 * 2. HC-SR04 自动发送 8 个 40kHz 的超声波脉冲
 * 3. 如果超声波遇到障碍物反射回来，Echo 脚输出高电平
 * 4. 高电平持续时间 = 超声波往返时间
 * 
 * 距离计算公式：
 *   距离 = (高电平时间 × 声速) / 2
 *       = (高电平时间 × 340m/s) / 2
 *       = 高电平时间(us) × 0.017 cm
 * 
 * 例如：高电平持续 1000us → 距离 = 1000 × 0.017 = 17cm
 */

// 超声波测距代码示例（伪代码）
float ultrasonic_measure(uint8_t trig_pin, uint8_t echo_pin) {
    // 1. 发送触发信号
    digital_write(trig_pin, HIGH);
    delay_us(15);          // 触发脉冲宽度 ≥10us
    digital_write(trig_pin, LOW);
    
    // 2. 等待回响信号开始
    uint32_t timeout = 25000;  // 最大测量距离约4m
    while (digital_read(echo_pin) == LOW) {
        if (timeout-- == 0) return -1;  // 超时返回-1
    }
    
    // 3. 记录回响开始时间
    uint32_t start_time = micros();
    
    // 4. 等待回响信号结束
    while (digital_read(echo_pin) == HIGH) {
        if (timeout-- == 0) return -1;
    }
    
    // 5. 计算距离
    uint32_t echo_time = micros() - start_time;  // 回响持续时间(us)
    float distance = echo_time * 0.017;          // 计算距离(cm)
    
    return distance;
}
```

#define ECHO_PIN GPIO_PIN_1   // 假设 Echo 接 PA1

float ultrasonic_read(uint8_t us_id) {

     * us_id: 0=前左, 1=前中, 2=前右, 3=后中
     * 返回值：距离值（cm），-1 表示无效
     */
    uint8_t trig = trig_pins[us_id];

    HAL_GPIO_WritePin(GPIOA, trig, GPIO_PIN_RESET);
    
    // 等待回响开始（超时检测）
    uint32_t tick_start = HAL_GetTick();
    while (HAL_GPIO_ReadPin(GPIOA, echo) == GPIO_PIN_RESET) {
        if (HAL_GetTick() - tick_start > 100) return -1;  // 超时
    }
    
    // 记录回响开始时间
    uint32_t us_start = __HAL_TIM_GET_COUNTER(&htim7);
    
    // 等待回响结束
    while (HAL_GPIO_ReadPin(GPIOA, echo) == GPIO_PIN_SET) {
        if (HAL_GetTick() - tick_start > 100) return -1;
    }
    
    // 计算距离
    uint32_t us_end = __HAL_TIM_GET_COUNTER(&htim7);
    uint32_t duration = us_end - us_start;  // us
    float distance = duration * 0.017;      // cm
    
    return distance;
}

## 6. 组装注意事项

### 6.1 防短路措施

> ⚠️ **重中之重！短路是嵌入式系统最常见的损坏原因！**

#### 常见短路原因

| 原因 | 后果 | 预防措施 |
|------|------|---------|
| 金属螺丝压到PCB走线 | 烧毁主控芯片 | 使用绝缘垫片或缠胶带 |
| 电池极性反接 | 电容/芯片爆炸 | XT30防反插设计，仔细核对 |
| 5V/3.3V 与 GND 短接 | 芯片烧毁 | 通电前测量各路阻值 |
| 焊接桥连 | 芯片烧毁 | 焊接后仔细检查焊点 |
| 线缆破皮 | 短路起火 | 使用热缩管保护 |

#### 防短路检查清单

- [ ] 所有金属螺丝与 PCB 之间有绝缘垫片或胶带
- [ ] 电池接口防反插定位准确
- [ ] 通电前用万用表测量 VCC 与 GND 之间阻值（应 > 1kΩ）
- [ ] 检查所有排针是否与邻近焊盘桥连
- [ ] 线缆破皮处用热缩管保护

```python
# 防短路检查代码（万用表测量指导）
def check_short_circuit():
    """
    使用万用表二极管/连续性档位检查短路
    正常阻值：VCC-GND 之间应有 100Ω～几MΩ（取决于电容充电）
    短路时：阻值 < 10Ω，万用表会发出蜂鸣声
    """
    checks = [
        ("主控板 5V-GND", ">100Ω 或 蜂鸣不响"),
        ("主控板 3.3V-GND", ">100Ω 或 蜂鸣不响"),
        ("TB6612 VMOT-GND", ">10Ω 或 蜂鸣不响"),
        ("STM32 VDD-GND", ">1kΩ 或 蜂鸣不响"),
    ]
    
    for name, expected in checks:
        print(f"测量 {name}...")
        print(f"  预期：{expected}")
        result = input("  实测（好/坏）：")
        if result == "坏":
            print("  ⚠️ 短路风险！请排查后再通电！")
    
    print("短路检查完成。")
```

### 6.2 线缆整理

良好的线缆整理不仅美观，更能：
- 防止线缆被轮子卷入
- 避免信号线与电源线干扰
- 便于维护和故障排查

#### 线缆整理规范

| 线缆类型 | 整理要求 |
|---------|---------|
| 电源线（粗） | 使用扎带固定，沿底盘边缘走线 |
| 电机线 | 经过活动关节处留有余量，使用螺旋管保护 |
| 信号线（细） | 用扎带捆扎成束，与电源线分开走线 |
| 排线 | 使用排线座连接，不要直接焊接 |

#### 线缆标签规范

| 线缆 | 标签内容 | 示例 |
|------|---------|------|
| 电机线 | M1/M2/M3/M4 | M1-A, M1-B |
| 编码器线 | 电机编号 | ENC1, ENC2 |
| 传感器线 | 位置+类型 | US-FL, IR-L |
| 电源线 | 电压值 | 7.4V, 5V, 3.3V |

### 6.3 散热处理

#### 发热源分析

| 发热源 | 发热程度 | 散热措施 |
|--------|---------|---------|
| 电机驱动芯片 TB6612 | 中等 | 添加散热片 |
| 稳压模块 LM2596 | 中等 | 保持通风，添加散热片 |
| STM32 主芯片 | 较低 | 通常无需散热 |
| 电机本体 | 高 | 避免长时间堵转 |

#### 散热安装步骤

1. **清洁表面**：用酒精擦拭芯片背面和散热片接触面
2. **涂导热硅脂**：薄层均匀涂抹，不要太厚
3. **固定散热片**：使用导热双面胶或螺丝固定
4. **注意绝缘**：如果散热片金属部分接触 PCB，需加绝缘垫片

FishBot 室内使用一般不需要防水，但需要注意防尘：

- **电池仓**：使用 3D 打印件保护电池
- **主控板**：使用亚克力板或 3D 打印外壳覆盖
- **传感器接口**：使用热缩管或硅胶套保护

### 填空题答案

**1.** 2S；7.4V  
> 2S 表示 2 串，即 2 节 3.7V 锂电池串联，额定电压 7.4V

**2.** 7.4V  
> VMOT 接电池电压，为电机提供动力

**3.** 编码器分辨率  
> 四倍频后脉冲数变为原来的 4 倍，等效分辨率提高 4 倍

**4.** 0.017  
> 声速 340m/s = 0.034cm/us，往返除以 2 = 0.017cm/us

**5.** 5V；3.3V  
> 5V 供给传感器和外设，3.3V 供给 STM32 芯片内核

### 简答题答案

**1. 供电架构设计答案：**

FishBot V2 采用分区供电方案，将动力电源（7.4V）和逻辑电源（5V/3.3V）分开：

- **动力部分（7.4V）**：直接供给 TB6612FNG 的 VMOT 引脚，为电机供电。电机启动时电流可达数安培，会产生较大纹波。

- **逻辑部分（5V→3.3V）**：通过 LM2596 降压芯片将 7.4V 降至 5V，再通过 LDO（如 AMS1117）降至 3.3V。逻辑电路对电源稳定性要求高，需要独立稳压。

**分区供电的好处：**
  - 防止电机启停时的大电流波动干扰主控芯片
  - 便于分别管理不同电压域的功耗
  - 故障隔离：某一区域短路不会影响另一区域

**2. H 桥工作原理答案：**

H 桥电路由 4 个 MOSFET/三极管组成，形似字母"H"：

```
     VCC
      |
    ┌─┴─┐
    │   │
  ┌─┴─┐ ┌─┴─┐
  │   │ │   │
  │   │ │   │
  └───┘ └───┘
    │   │
  电机  │
    │   │
    └─┬─┘
      |
     GND
```

**正常工作状态：**
- 正转：左上ON、右下ON（左高右低）
- 反转：右上ON、左下ON（右高左低）
- 停止：全部OFF 或 左上+右上ON（短路制动）

**危险状态：**
如果左上ON、左上ON（即左右两侧都接高电平），会形成 VCC→GND 短路，电流极大，会立即烧毁 H 桥！

**3. 硬件组装安全注意事项答案：**

① **防短路**：所有金属螺丝与 PCB 之间必须有绝缘措施  
② **防反接**：电池接口注意极性，接反会损坏元件  
③ **架空测试**：电机测试时必须架空机器人，防止失控伤人  
④ **检查电压**：通电前必须用万用表确认各路电压  
⑤ **烫伤防护**：电烙铁、热风枪温度高，注意防烫  
⑥ **静电防护**：焊接时注意防静电，损坏芯片  
⑦ **工具安全**：内六角扳手等工具注意正确使用，防止滑脱伤手  

**4. 电机转速过低排查方法答案：**

| 序号 | 可能原因 | 排查方法 |
|------|---------|---------|
| 1 | PWM 占空比过低 | 检查程序中 PWM 值设置 |
| 2 | 电池电量不足 | 用万用表测量电池电压（应 > 7V） |
| 3 | 机械卡阻 | 断电后手动转动轮子，检查是否顺畅 |
| 4 | 电机线圈短路 | 用万用表测量电机线圈阻值是否正常 |
| 5 | 驱动芯片过热 | 用手触摸 TB6612FNG 是否烫手，可能进入热保护 |
| 6 | 驱动芯片损坏 | 测量 AOUT 输出电压是否随 PWM 变化 |
| 7 | 负载过大 | 减少机器人载重，测试空载转速 |
